# fa_jitter_buffer 仕様書

## 目的

`fa_jitter_buffer` は network / remote audio stream の到着揺らぎを吸収するため、`AudioFrame.epoch` を基準に frame を一時保持し、指定した depth を超えた分だけ epoch 昇順で publish する streaming package である。

## 入力 topic

| topic | type | 必須 |
| --- | --- | --- |
| `input_topic` | `fa_interfaces::msg::AudioFrame` | yes |

`input_topic` は起動 parameter で明示する。コード内の有効 default は持たない。

## 出力 topic

| topic | type | 内容 |
| --- | --- | --- |
| `output_topic` | `fa_interfaces::msg::AudioFrame` | jitter buffer を通過した frame |
| `diagnostics` | `diagnostic_msgs::msg::DiagnosticArray` | 設定、buffer 状態、counter |

## parameters

| name | type | validation |
| --- | --- | --- |
| `input_topic` | string | non-empty |
| `output_topic` | string | non-empty |
| `input_stream_id` | string | non-empty、raw / resolved ROS topic identity と異なる |
| `output.stream_id` | string | non-empty、raw / resolved ROS topic identity と異なり、入力 stream id と異なる |
| `expected.sample_rate` | int | `> 0` |
| `expected.channels` | int | `> 0` |
| `expected.encoding` | string | non-empty |
| `expected.bit_depth` | int | `> 0` and byte-aligned |
| `expected.layout` | string | non-empty |
| `jitter.target_depth_frames` | int | `>= 0` |
| `jitter.max_depth_frames` | int | `> target_depth_frames` |
| `jitter.reset_on_epoch_regression` | bool | required |
| `qos.depth` | int | `> 0` |
| `qos.reliable` | bool | required |
| `diagnostics.publish_period_ms` | int | `> 0` |
| `diagnostics.qos.depth` | int | `> 0` |
| `diagnostics.qos.reliable` | bool | required |

invalid config は起動時に例外で fail closed する。

## Runtime validation

受信 frame は buffer を変更する前に検証する。

- `source_id` と `stream_id` は non-empty。
- `stream_id` は `input_stream_id` と一致する。
- `sample_rate`、`channels`、`encoding`、`bit_depth`、`layout` は `expected.*` と完全一致する。
- `data` は non-empty。
- `data.size()` は `channels * bit_depth / 8` の倍数である。
- `FLOAT32LE` + `interleaved` の場合、全 sample は finite かつ `[-1.0, 1.0]` の範囲内である。

不正 frame は drop し、buffer、active stream、publish 済み epoch を変更しない。

## Buffer behavior

`fa_jitter_buffer` は同時に 1 つの active source / format contract だけを扱う。

1. 初回 valid frame で active stream を確定する。
2. active stream の `source_id` または format contract が変わった場合は buffer を reset する。
3. duplicate epoch は drop する。
4. publish 済み epoch より古い epoch は late drop する。
5. `jitter.reset_on_epoch_regression=true` の場合、publish 済み epoch より古い epoch を受けた時点で reset し、新しい epoch sequence として受け入れる。
6. valid frame は `epoch` を key とする buffer に入れる。
7. `buffered_frames > jitter.target_depth_frames` の間、最古 epoch の frame を publish する。

publish 時は入力 frame を copy し、`stream_id` だけを `output.stream_id` に更新する。ROS topic 名は transport identity であり、`AudioFrame.stream_id` には使わない。

## Non-goals

- resample
- format conversion
- gain / normalize
- padding
- packet loss concealment
- 欠損 frame 生成
- device I/O
- network I/O
- legacy alias / compatibility layer

## 仕様 ID

| SPEC ID | 内容 |
| --- | --- |
| `FA-JITTER-BUFFER-SPEC-001` | default config は required parameter を明示する。 |
| `FA-JITTER-BUFFER-SPEC-002` | package layout は標準 streaming layout を満たす。 |
| `FA-JITTER-BUFFER-SPEC-003` | required parameter にコード default を持たず、invalid config を例外にする。 |
| `FA-JITTER-BUFFER-SPEC-004` | runtime validation は invalid frame と `stream_id != input_stream_id` を buffer mutation 前に drop する。 |
| `FA-JITTER-BUFFER-SPEC-005` | `FLOAT32LE` + `interleaved` で finite / normalized sample を検証する。 |
| `FA-JITTER-BUFFER-SPEC-006` | `std::map` で epoch 昇順 publish する。 |
| `FA-JITTER-BUFFER-SPEC-007` | duplicate と publish 済みより古い epoch を drop する。 |
| `FA-JITTER-BUFFER-SPEC-008` | source / format change と configured epoch regression reset を明示的に扱う。 |
| `FA-JITTER-BUFFER-SPEC-009` | diagnostics は config、buffered count、counter を publish する。 |
| `FA-JITTER-BUFFER-SPEC-010` | CMake/package deps は `ament_cmake_gtest`、`ament_cmake_pytest`、`ament_lint_auto`、pytest/yaml deps を持つ。 |
| `FA-JITTER-BUFFER-SPEC-011` | integration contract は data-plane config と runtime source contract を検証する。 |
| `FA-JITTER-BUFFER-SPEC-012` | launch contract は node_name / config_file のみを公開し、backend 選択を漏らさない。 |
| `FA-JITTER-BUFFER-SPEC-013` | ROS graph contract は epoch 昇順 publish、invalid sample drop、duplicate drop、source contract reset を検証する。 |
| `FA-JITTER-BUFFER-SPEC-014` | resample、format conversion、gain、padding、PLC、device/network I/O、legacy alias は非目標である。 |
