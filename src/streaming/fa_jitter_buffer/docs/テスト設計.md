# fa_jitter_buffer テスト設計

## 方針

初期実装では、pytest による text / contract test と、C++ gtest による ROS2 pub/sub contract test を置く。package-local integration / launch contract test は実行可能な pytest として置き、より長時間の reconnect / transport integration は未検証項目として分離する。

## Unit contract tests

| test | 観点 |
| --- | --- |
| default config | required parameter が明示されていること |
| package layout | 標準 streaming layout を満たすこと |
| startup validation | required parameter にコード default を持たず、invalid config を例外にすること |
| runtime validation | invalid frame を buffer mutation 前に drop すること |
| sample validation | `FLOAT32LE` + `interleaved` で finite / normalized sample を検証すること |
| epoch ordering | `std::map` で epoch 昇順 publish すること |
| duplicate / late handling | duplicate と publish 済みより古い epoch を drop すること |
| reset behavior | source / format change と configured epoch regression reset を明示的に扱うこと |
| diagnostics | config、buffered count、counter を publish すること |
| CMake/package deps | `ament_cmake_gtest`、`ament_cmake_pytest`、`ament_lint_auto`、pytest/yaml deps を持つこと |

## Integration / launch contract tests

`test/integration/test_fa_jitter_buffer_streaming_config_contract.py` で以下を確認する。

- default config が明示的な input / output topic、QoS、diagnostics contract を持つ
- runtime source が `AudioFrame`、validation、diagnostics を含む

`test/launch/test_fa_jitter_buffer_streaming_launch_contract.py` で以下を確認する。

- launch file は `node_name` と `config_file` だけを公開する
- package / executable / default config path が `fa_jitter_buffer` の契約に一致する
- streaming node の launch に `backend.name` を漏らさない

## ROS graph contract tests

`test/cpp/test_fa_jitter_buffer_node_contract.cpp` で実 ROS2 pub/sub を使い、次を確認する。

- `input_topic` へ `AudioFrame` を publish したとき、`output_topic` へ epoch 昇順に出ること。
- finite / normalized ではない `FLOAT32LE` sample は output に出ないこと。
- publish 済み epoch の duplicate は output に再送されないこと。
- source contract change では buffered frame を reset し、別 source と混ぜないこと。

## 未検証の ROS graph integration

今後の長時間 integration test では次を確認する。

- invalid frame が diagnostics counter に反映されること。
- reconnect / launch parameter mismatch 時に fail closed すること。
