# fa_frame_buffer アルゴリズム詳細説明書

## 概要

内部 buffer は `uint8_t` byte buffer として保持する。入力は `FLOAT32LE` interleaved のみを受け付けるため、1 frame の byte 数は `channels * sizeof(float)` である。

## 処理手順

1. 入力 frame 数を count する。
2. `AudioFrame` 契約を検証する。
3. partial buffer が存在する場合、入力 frame の stream identity と format が buffer の identity と一致するか確認する。
4. 不一致なら warning を出して buffer を clear する。
5. buffer が空なら、入力 frame の `header`、`source_id`、format、`epoch` を chunk identity として記録する。
6. 入力 `data` を byte buffer 末尾に追加する。
7. `max_buffered_chunks * chunk_bytes` を超えた場合、古い chunk 1 個分を破棄して `overflow_count` を増やす。
8. `chunk_bytes` 以上の data がある間、先頭 `chunk_bytes` を出力 frame として publish する。
9. 残りが `chunk_bytes` 未満なら partial chunk として保持する。

## chunk identity

chunk identity は「その chunk の先頭 byte に寄与した frame」の metadata で決まる。出力は入力 stream の identity を保持し、`stream_id` のみ `output_topic` に更新する。

## overflow

overflow は transport loss として扱う。これは意味を変える fallback ではなく、buffer 上限を超えたことを明示する破棄である。

破棄単位は古い whole chunk worth of bytes である。破棄した回数は `overflow_count`、破棄した frame 数は `frames_dropped` に反映する。

## partial chunk

partial chunk は publish しない。無音 padding、zero 埋め、前回 sample の繰り返しは行わない。
