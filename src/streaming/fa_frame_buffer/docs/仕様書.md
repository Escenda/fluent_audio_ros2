# fa_frame_buffer 仕様書

## 目的

`fa_frame_buffer` は、可変長で到着する `FLOAT32LE` interleaved `AudioFrame` stream を、後段 node が扱いやすい固定フレーム数の chunk に整形する streaming node である。

## ノード

| 項目 | 値 |
| --- | --- |
| package | `fa_frame_buffer` |
| executable | `fa_frame_buffer_node` |
| node name | `fa_frame_buffer` |
| input | `input_topic` |
| output | `output_topic` |

## 入力契約

入力 frame は以下をすべて満たす必要がある。

| 項目 | 条件 |
| --- | --- |
| `source_id` | non-empty |
| `stream_id` | non-empty かつ `input_topic` と一致 |
| `sample_rate` | `expected.sample_rate` と一致し、設定値は `> 0` |
| `channels` | `expected.channels` と一致し、設定値は `> 0` |
| `encoding` | `FLOAT32LE` |
| `bit_depth` | `32` |
| `layout` | `interleaved` |
| `data` | non-empty かつ `channels * sizeof(float)` で割り切れる |

起動時設定が不正な場合、node は `std::runtime_error` で fail closed する。
runtime frame が不正な場合、その frame を drop し、warning を出す。

## buffering parameters

| parameter | default | 条件 |
| --- | ---: | --- |
| `buffering.frames_per_chunk` | `512` | `> 0` |
| `buffering.max_buffered_chunks` | `4` | `> 0` |

## 出力契約

出力 frame は以下を満たす。

- `data.size() == frames_per_chunk * channels * sizeof(float)`
- `stream_id == output_topic`
- `header`、`source_id`、`sample_rate`、`channels`、`encoding`、`bit_depth`、`layout`、`epoch` は chunk の先頭に寄与した frame から引き継ぐ
- partial chunk は publish しない
- padding は行わない

## stream compatibility

buffer に partial data が存在する間に、以下のいずれかが変わった場合、既存 buffer を clear して warning を出す。

- `source_id`
- `sample_rate`
- `channels`
- `encoding`
- `bit_depth`
- `layout`

互換でない stream を混ぜて chunk を作らない。

## diagnostics

`diagnostics` topic に以下を含める。

- `input_topic`
- `output_topic`
- `frames_per_chunk`
- `max_buffered_chunks`
- `chunk_bytes`
- `buffered_bytes`
- `partial_frames_buffered`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `chunks_out`
- `overflow_count`
- `buffer_resets`
