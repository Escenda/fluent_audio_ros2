# fa_frame_buffer テスト設計

## unit contract tests

`test/unit/test_fa_frame_buffer_audio_frame_contract.py` は package files を静的に検査し、以下を確認する。

- default config が `FLOAT32LE` interleaved fixed chunk contract を持つ
- device I/O、resampling、format conversion、gain、limiter、filter、denoise を実装していない
- 起動時設定の invalid 値を `std::runtime_error` で fail closed する
- runtime frame validation が `source_id`、`stream_id`、format、data size を検査する
- stream compatibility 変更時に buffer を clear する
- fixed chunk algorithm が先頭 contributing frame の identity を保持する
- overflow が古い chunk 単位で破棄され、diagnostics に出る
- package layout と colcon pytest registration が存在する

## integration tests

今後の integration test では、ROS2 node を launch し、以下を確認する。

- 512 frame 未満では publish されない
- 複数入力 frame の合計が 512 frame に達したら 1 chunk が publish される
- 1024 frame 入力で 2 chunks が publish される
- format 変更時に partial buffer が破棄される
- overflow 時に古い chunk が破棄される

## launch tests

launch test では `launch/fa_frame_buffer.launch.py` が default config を読み、node name と executable が契約どおりになることを確認する。
