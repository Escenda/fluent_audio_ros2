# fa_frame_buffer テスト設計

## unit contract tests

`test/unit/test_fa_frame_buffer_audio_frame_contract.py` は package files を静的に検査し、以下を確認する。

| TC ID | 対応 SPEC | 観点 |
| --- | --- | --- |
| `FA-FRAME-BUFFER-TC-001` -> `FA-FRAME-BUFFER-SPEC-001` | `FA-FRAME-BUFFER-SPEC-001` | default config が `FLOAT32LE` interleaved fixed chunk contract を持つ |
| `FA-FRAME-BUFFER-TC-002` -> `FA-FRAME-BUFFER-SPEC-002` | `FA-FRAME-BUFFER-SPEC-002` | device I/O、resampling、format conversion、gain、limiter、filter、denoise を実装していない |
| `FA-FRAME-BUFFER-TC-003` -> `FA-FRAME-BUFFER-SPEC-003` | `FA-FRAME-BUFFER-SPEC-003` | 起動時設定の invalid 値を `std::runtime_error` で fail closed する |

| `FA-FRAME-BUFFER-TC-005` -> `FA-FRAME-BUFFER-SPEC-005` | `FA-FRAME-BUFFER-SPEC-005` | `input_stream_id` / `output.stream_id` が必須で、ROS topic identity と同値なら起動失敗する |
| `FA-FRAME-BUFFER-TC-006` -> `FA-FRAME-BUFFER-SPEC-006` | `FA-FRAME-BUFFER-SPEC-006` | stream compatibility 変更時に buffer を clear する |

| `FA-FRAME-BUFFER-TC-008` -> `FA-FRAME-BUFFER-SPEC-008` | `FA-FRAME-BUFFER-SPEC-008` | overflow が古い chunk 単位で破棄され、diagnostics に出る |
| `FA-FRAME-BUFFER-TC-009` -> `FA-FRAME-BUFFER-SPEC-009` | `FA-FRAME-BUFFER-SPEC-009` | package layout と colcon pytest registration が存在する |
| `FA-FRAME-BUFFER-TC-010` -> `FA-FRAME-BUFFER-SPEC-010` | `FA-FRAME-BUFFER-SPEC-010` | integration contract が data-plane config と runtime source contract を検証する |
| `FA-FRAME-BUFFER-TC-011` -> `FA-FRAME-BUFFER-SPEC-011` | `FA-FRAME-BUFFER-SPEC-011` | launch contract が node_name / config_file のみを公開する |

## integration / launch contract tests

`test/integration/test_fa_frame_buffer_streaming_config_contract.py` で以下を確認する。

- default config が明示的な input / output topic、stream identity、QoS、diagnostics contract を持つ
- runtime source が `AudioFrame`、validation、diagnostics を含む

`test/launch/test_fa_frame_buffer_streaming_launch_contract.py` で以下を確認する。

- launch file は `node_name` と `config_file` だけを公開する
- package / executable が `fa_frame_buffer` の契約に一致し、config path は launch argument で明示する
- streaming node の launch に `backend.name` を漏らさない

## ROS graph contract tests

`test/cpp/test_fa_frame_buffer_node_contract.cpp` で実 ROS2 pub/sub を使い、以下を確認する。

- configured `frames_per_chunk` 未満では publish されない
- 複数入力 frame の合計が configured `frames_per_chunk` に達したら `stream_id == output.stream_id` の 1 chunk が publish される
- configured `frames_per_chunk * 2` の入力で 2 chunks が publish される
- format 変更時に partial buffer が破棄される
- invalid format frame は drop され、chunk として publish されない
