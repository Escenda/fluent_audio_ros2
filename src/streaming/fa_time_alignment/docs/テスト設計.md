# fa_time_alignment テスト設計

## 1. 対象

`fa_time_alignment` の package-local contract を対象とする。

## 2. Unit Contract Tests

`test/unit/test_fa_time_alignment_audio_frame_contract.py` で以下を確認する。

- package layout が標準構成を満たす
- default config が明示的な入出力、format、alignment、QoS、diagnostics を持つ
- CMake が `ament_cmake_gtest`、`ament_cmake_pytest`、`ament_lint_auto` を有効にする
- node core が `rclcpp::NodeOptions` で起動でき、executable `main()` と分離されている
- startup validation が invalid config を例外で fail-closed する
- runtime validation が identity、format、payload byte alignment を検証する
- algorithm が nearest grid を使い、`max_adjust_ms` 超過を drop する
- output が `header.stamp` と `stream_id` だけを更新する
- device I/O、sample editing、resampling、legacy alias が入っていない

## 3. Integration / launch contract tests

`test/integration/test_fa_time_alignment_streaming_config_contract.py` で以下を確認する。

- default config が明示的な input / output topic、QoS、diagnostics contract を持つ
- runtime source が `AudioFrame`、validation、diagnostics を含む

`test/launch/test_fa_time_alignment_streaming_launch_contract.py` で以下を確認する。

- launch file は `node_name` と `config_file` だけを公開する
- package / executable / default config path が `fa_time_alignment` の契約に一致する
- streaming node の launch に `backend.name` を漏らさない

親 system package が package registration と launch graph を担当するため、この package では standalone contract を先に固定する。

## 4. ROS graph contract tests

`test/cpp/test_fa_time_alignment_node_contract.cpp` は C++ node core を起動し、ROS2 pub/sub 経由で以下を確認する。

- nearest grid alignment が timestamp だけを補正し、payload を保持する
- `alignment.phase_ms` が grid phase として効く
- `max_adjust_ms` を超える補正は publish しない
- invalid format は publish しない

## 5. 未検証項目

- parent system config からの起動
- VLAbor profile からの launch

これらは parent agent が package registration を行った後に system-level test で確認する。
