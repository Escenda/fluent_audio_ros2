# fa_time_alignment 仕様書

## 1. 目的

`fa_time_alignment` は `AudioFrame.header.stamp` を明示的な時間グリッドへ揃える ROS2 streaming node である。

音声 payload は変更しない。変更対象は timestamp と stream identity のみである。

## 2. 入力

Topic は `input_topic` パラメータで指定する。空文字は禁止する。

Message type:

```text
fa_interfaces/msg/AudioFrame
```

必須条件:

- `source_id` が空でない
- `stream_id` が空でない
- `stream_id == input_stream_id`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == expected.encoding`
- `bit_depth == expected.bit_depth`
- `layout == expected.layout`
- `data` が空でない
- `data.size()` が `channels * bit_depth / 8` の倍数である

## 3. 出力

Topic は `output_topic` パラメータで指定する。空文字は禁止する。

出力 `AudioFrame` は入力をコピーし、以下だけを更新する。

- `header.stamp`
- `stream_id = output.stream_id`

`data` は byte 列として完全に保持する。

## 4. パラメータ

| パラメータ | 型 | 制約 |
| --- | --- | --- |
| `input_topic` | string | 必須、非空 |
| `output_topic` | string | 必須、非空 |
| `input_stream_id` | string | 必須、raw / resolved ROS topic identity と異なる |
| `output.stream_id` | string | 必須、raw / resolved ROS topic identity と異なり、入力 stream id と異なる |
| `expected.sample_rate` | int | `> 0` |
| `expected.channels` | int | `> 0` |
| `expected.encoding` | string | 非空 |
| `expected.bit_depth` | int | `> 0` |
| `expected.layout` | string | 非空 |
| `alignment.period_ms` | double | 有限、`> 0` |
| `alignment.phase_ms` | double | 有限、`>= 0`、`< period_ms` |
| `alignment.max_adjust_ms` | double | 有限、`>= 0` |
| `qos.depth` | int | `> 0` |
| `qos.reliable` | bool | true なら reliable、false なら best effort |
| `diagnostics.publish_period_ms` | int | `> 0` |
| `diagnostics.qos.depth` | int | `> 0` |
| `diagnostics.qos.reliable` | bool | required |

## 5. Drop 条件

以下の条件では frame を publish しない。

- 入力 message pointer が null
- 入力 identity または format が契約に一致しない
- payload byte 境界が契約に一致しない
- 計算した grid timestamp が非有限
- 計算した grid timestamp が負
- 計算した grid timestamp が int64 nanosecond 範囲外
- `abs(aligned_stamp - input_stamp) > max_adjust_ms`

## 6. Diagnostics

`diagnostic_msgs/msg/DiagnosticArray` を `diagnostics` topic に publish する。

Counters:

- `frames_in`
- `frames_out`
- `frames_dropped`
- `frames_aligned`
- `frames_excess_adjust`

## 7. 非目標

- device 入出力
- audio sample の変換、補正、正規化
- stream の再構成
- timestamp 欠損や契約不一致を補う fallback

## 8. 仕様 ID

| SPEC ID | 内容 |
| --- | --- |
| `FA-TIME-ALIGNMENT-SPEC-001` | package layout は標準構成を満たす。 |
| `FA-TIME-ALIGNMENT-SPEC-002` | default config は明示的な入出力 topic、stream identity、format、alignment、QoS、diagnostics を持つ。 |
| `FA-TIME-ALIGNMENT-SPEC-003` | CMake は `ament_cmake_gtest`、`ament_cmake_pytest`、`ament_lint_auto` を有効にする。 |
| `FA-TIME-ALIGNMENT-SPEC-004` | node core は `rclcpp::NodeOptions` で起動でき、executable `main()` と分離されている。 |
| `FA-TIME-ALIGNMENT-SPEC-005` | startup validation は invalid config を例外で fail closed する。 |
| `FA-TIME-ALIGNMENT-SPEC-006` | runtime validation は identity、format、payload byte alignment を検証する。 |
| `FA-TIME-ALIGNMENT-SPEC-007` | algorithm は nearest grid を使い、`max_adjust_ms` 超過を drop する。 |
| `FA-TIME-ALIGNMENT-SPEC-008` | output は `header.stamp` と `stream_id` だけを更新する。 |
| `FA-TIME-ALIGNMENT-SPEC-009` | device I/O、sample editing、resampling、legacy alias が入っていない。 |
| `FA-TIME-ALIGNMENT-SPEC-010` | integration contract は data-plane config と runtime source contract を検証する。 |
| `FA-TIME-ALIGNMENT-SPEC-011` | launch contract は node_name / config_file のみを公開し、backend 選択を漏らさない。 |
| `FA-TIME-ALIGNMENT-SPEC-012` | ROS graph contract は nearest grid alignment、phase、max_adjust drop、invalid format drop を検証する。 |
| `FA-TIME-ALIGNMENT-SPEC-013` | parent system config と VLAbor profile からの起動は system-level test で扱う未検証項目である。 |
