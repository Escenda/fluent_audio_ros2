# fa_time_alignment アルゴリズム詳細説明書

## 1. 入力検証

受信時に `frames_in` を加算する。

`AudioFrame` は publish 前に以下を検証する。

1. `source_id` と `stream_id` が空でない
2. `stream_id` が `input_stream_id` と一致する
3. format fields が `expected.*` と一致する
4. `data` が空でない
5. `data.size()` が `channels * bit_depth / 8` の倍数である

検証に失敗した frame は drop し、音声データや timestamp の推定補完は行わない。

## 2. グリッド定義

時刻グリッドは次式で定義する。

```text
grid(k) = phase_ms + k * period_ms
```

`period_ms` は有限かつ正、`phase_ms` は有限かつ `0 <= phase_ms < period_ms` である。

実装では millisecond を nanosecond に変換し、`header.stamp` も nanosecond 表現に変換して計算する。

## 3. Nearest Grid

入力 timestamp `t` に対して、nearest grid index を次式で求める。

```text
k = round((t - phase) / period)
aligned = phase + k * period
```

`aligned` が負の場合は drop する。これは負時刻を有効な現在時刻として扱わないためである。

## 4. 最大補正量

補正量を次式で求める。

```text
adjustment = aligned - t
```

`abs(adjustment) > max_adjust_ms` の場合、frame は drop する。この drop は `frames_excess_adjust` にも計上する。

## 5. 出力生成

許容範囲内の場合、入力 `AudioFrame` をコピーし、以下だけを変更する。

- `header.stamp = aligned`
- `stream_id = output.stream_id`

`data` は resize、decode、encode、sample 単位の読み取りを行わない。

## 6. Diagnostics

一定周期で以下を publish する。

- topic / stream identity 設定
- alignment 設定
- in/out/drop/aligned/excess_adjust counters
- backend 名 `no_runtime_backend`
