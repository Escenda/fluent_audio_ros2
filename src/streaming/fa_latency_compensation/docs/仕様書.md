# fa_latency_compensation 仕様書

## 目的

`fa_latency_compensation` は FluentAudio の `AudioFrame` stream に対して、測定済み latency offset を ROS timestamp に反映する streaming node です。音声 sample 自体は変更対象ではありません。

## 非目的

- device capture / playback
- audio sample の編集、変換、補間
- resample / normalize / gain / filter
- 欠損 timestamp や format mismatch の推測補完
- legacy topic alias

## 入出力

| Direction | Topic | Type | Contract |
| --- | --- | --- | --- |
| Subscribe | `input_topic` | `fa_interfaces/msg/AudioFrame` | `stream_id == input_stream_id` |
| Publish | `output_topic` | `fa_interfaces/msg/AudioFrame` | `stream_id == output.stream_id` |
| Publish | `diagnostics` | `diagnostic_msgs/msg/DiagnosticArray` | counters / static config |

## Parameters

| Name | Type | Required | Validation |
| --- | --- | --- | --- |
| `input_topic` | string | yes | non-empty |
| `output_topic` | string | yes | non-empty |
| `input_stream_id` | string | yes | raw / resolved ROS topic 名と同値不可 |
| `output.stream_id` | string | yes | raw / resolved ROS topic 名と同値不可 |
| `compensation.offset_ms` | double | yes | finite and representable as int64 nanoseconds |
| `expected.sample_rate` | int | yes | `> 0` |
| `expected.channels` | int | yes | `> 0` |
| `expected.encoding` | string | yes | non-empty |
| `expected.bit_depth` | int | yes | `> 0` and byte-aligned |
| `expected.layout` | string | yes | non-empty |
| `qos.depth` | int | yes | `> 0` |
| `qos.reliable` | bool | yes | explicit bool |
| `diagnostics.publish_period_ms` | int | yes | `> 0` |
| `diagnostics.qos.depth` | int | yes | `> 0` |
| `diagnostics.qos.reliable` | bool | yes | explicit bool |

## Runtime Frame Contract

入力 frame は publish 前に次を満たす必要があります。

- `source_id` が空でない
- `stream_id` が空でない
- `stream_id == input_stream_id`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == expected.encoding`
- `bit_depth == expected.bit_depth`
- `layout == expected.layout`
- `data` が空でない
- `data.size()` が `channels * bit_depth / 8` の倍数

違反した frame は warning を出して drop します。値を推測して継続しません。

## Timestamp Compensation

出力 frame は入力 frame の全 field をコピーした後、次だけを更新します。

- `header.stamp = input.header.stamp + compensation.offset_ms`
- `stream_id = output.stream_id`

補正後 timestamp が負になる場合、または int64 nanosecond range を超える場合、その frame は fail-closed として drop します。`header.stamp` を 0 に丸める fallback は行いません。

## Diagnostics

`diagnostics` には少なくとも次を出力します。

- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `offset_ms`
- expected format fields
- `frames_in`
- `frames_out`
- `frames_dropped`
- `negative_timestamp_drops`
- `timestamp_overflow_drops`

## 仕様 ID

| SPEC ID | 内容 |
| --- | --- |
| `FA-LATENCY-COMPENSATION-SPEC-001` | default config は required parameter を明示する。 |
| `FA-LATENCY-COMPENSATION-SPEC-002` | device I/O や audio sample editing token を含まない。 |
| `FA-LATENCY-COMPENSATION-SPEC-003` | node identity は `fa_latency_compensation` / `fa_latency_compensation_node` / `FaLatencyCompensationNode` に一致する。 |
| `FA-LATENCY-COMPENSATION-SPEC-004` | startup validation は fail closed である。 |
| `FA-LATENCY-COMPENSATION-SPEC-005` | runtime validation は `source_id` / `stream_id` / format / data alignment を確認する。 |
| `FA-LATENCY-COMPENSATION-SPEC-006` | compensation は `header.stamp` と `stream_id` だけを更新し、topic 名ではなく `output.stream_id` を使う。 |
| `FA-LATENCY-COMPENSATION-SPEC-007` | `input_stream_id` / `output.stream_id` は必須で、ROS topic identity と同値なら起動失敗する。 |
| `FA-LATENCY-COMPENSATION-SPEC-008` | 負 timestamp と timestamp overflow を drop し、diagnostics counter に反映する。 |
| `FA-LATENCY-COMPENSATION-SPEC-009` | package layout は required files を含む。 |
| `FA-LATENCY-COMPENSATION-SPEC-010` | CMake は pytest と `ament_lint_auto` を登録する。 |
| `FA-LATENCY-COMPENSATION-SPEC-011` | CMake は C++ ROS graph contract test を登録する。 |
| `FA-LATENCY-COMPENSATION-SPEC-012` | integration contract は data-plane config と runtime source contract を検証する。 |
| `FA-LATENCY-COMPENSATION-SPEC-013` | launch contract は node_name / config_file のみを公開し、backend 選択を漏らさない。 |
| `FA-LATENCY-COMPENSATION-SPEC-014` | ROS graph contract は offset adjustment、payload/source/epoch 保持、negative/overflow timestamp drop、invalid format drop を検証する。 |
