# fa_latency_compensation 仕様書

## 目的

`fa_latency_compensation` は FluentAudio の `AudioFrame` stream に対して、測定済み latency offset を ROS timestamp に反映する streaming node です。音声 sample 自体は処理対象ではありません。

## 非目的

- device capture / playback
- audio sample の編集、変換、補間
- resample / normalize / gain / filter
- 欠損 timestamp や format mismatch の推測補完
- legacy topic alias

## 入出力

| Direction | Topic | Type | Contract |
| --- | --- | --- | --- |
| Subscribe | `input_topic` | `fa_interfaces/msg/AudioFrame` | `stream_id == input_topic` |
| Publish | `output_topic` | `fa_interfaces/msg/AudioFrame` | `stream_id == output_topic` |
| Publish | `diagnostics` | `diagnostic_msgs/msg/DiagnosticArray` | counters / static config |

## Parameters

| Name | Type | Required | Validation |
| --- | --- | --- | --- |
| `input_topic` | string | yes | non-empty |
| `output_topic` | string | yes | non-empty |
| `compensation.offset_ms` | double | yes | finite and representable as int64 nanoseconds |
| `expected.sample_rate` | int | yes | `> 0` |
| `expected.channels` | int | yes | `> 0` |
| `expected.encoding` | string | yes | non-empty |
| `expected.bit_depth` | int | yes | `> 0` and byte-aligned |
| `expected.layout` | string | yes | non-empty |
| `qos.depth` | int | yes | `> 0` |
| `qos.reliable` | bool | yes | explicit bool |
| `diagnostics.publish_period_ms` | int | yes | `> 0` |

## Runtime Frame Contract

入力 frame は publish 前に次を満たす必要があります。

- `source_id` が空でない
- `stream_id` が空でない
- `stream_id == input_topic`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == expected.encoding`
- `bit_depth == expected.bit_depth`
- `layout == expected.layout`
- `data` が空でない
- `data.size()` が `channels * bit_depth / 8` の倍数

違反した frame は warning を出して drop します。値を推測して継続しません。

## Timestamp Compensation

出力 frame は入力 frame の全 field をコピーした後、次だけを更新します。

- `header.stamp = input.header.stamp + compensation.offset_ms`
- `stream_id = output_topic`

補正後 timestamp が負になる場合、または int64 nanosecond range を超える場合、その frame は fail-closed として drop します。`header.stamp` を 0 に丸める fallback は行いません。

## Diagnostics

`diagnostics` には少なくとも次を出力します。

- `input_topic`
- `output_topic`
- `offset_ms`
- expected format fields
- `frames_in`
- `frames_out`
- `frames_dropped`
- `negative_timestamp_drops`
- `timestamp_overflow_drops`
