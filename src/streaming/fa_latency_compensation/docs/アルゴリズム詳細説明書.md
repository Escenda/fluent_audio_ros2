# fa_latency_compensation アルゴリズム詳細説明書

## Startup Validation

node は parameter declaration 後に全 required parameter を取得し、起動時に検証します。`compensation.offset_ms` は `std::isfinite` で有限値のみを許可します。format expectation は explicit field として扱い、空文字や非正値は起動失敗にします。

## Frame Timing Flow

1. 入力 callback で `frames_in` を増加する。
2. null message は publish せず `frames_dropped` を増加する。
3. `AudioFrame` contract を検証する。
4. `header.stamp` を nanoseconds に変換する。
5. `compensation.offset_ms` を nanoseconds に丸めて加算する。
6. 加算結果が負または int64 nanosecond range を超える場合は publish せず、drop counter を増加する。
7. 入力 frame を出力 frame にコピーする。
8. 出力 frame の `header.stamp` と `stream_id` だけを更新する。
9. publish 成功 path で `frames_out` を増加する。

## Audio Payload

`AudioFrame.data` は読み取り専用の byte payload として扱います。node は sample value を decode せず、byte order も変更しません。

frame alignment 検証は `channels * bit_depth / 8` の倍数であることだけを確認します。alignment 違反を padding で補正することは禁止です。

## Negative Timestamp

負 timestamp と int64 nanosecond overflow は ROS time として意味が壊れるため drop します。0 への丸め、latest timestamp への置換、入力 stamp の維持は行いません。
