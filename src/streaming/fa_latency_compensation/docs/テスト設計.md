## Integration / launch contract tests

`test/integration/test_fa_latency_compensation_streaming_config_contract.py` で以下を確認する。

- default config が明示的な input / output topic、stream identity、QoS、diagnostics contract を持つ
- runtime source が `AudioFrame`、validation、diagnostics を含む

`test/launch/test_fa_latency_compensation_streaming_launch_contract.py` で以下を確認する。

- launch file は `node_name` と `config_file` だけを公開する
- package / executable が `fa_latency_compensation` の契約に一致し、config path は launch argument で明示する
- streaming node の launch に `backend.name` を漏らさない

## 未検証の ROS graph integration

`test/cpp/test_fa_latency_compensation_node_contract.cpp` は C++ node core を起動し、ROS2 pub/sub 経由で以下を確認する。

- positive offset が `header.stamp` だけを進め、payload / source / epoch を保持する
- negative offset が非負 timestamp の範囲で `header.stamp` だけを戻す
- negative adjusted timestamp は publish せず drop する
- builtin_interfaces/Time 範囲を超える timestamp は publish せず drop する
- invalid format は publish せず drop する

親 system 登録後に残る未検証項目は、実際の system profile からこの package を include した状態で同じ data-plane 契約が成立すること。

## Non-Goals

この package-local test は actual audio device、speaker、microphone、sample transform quality を検証しません。
