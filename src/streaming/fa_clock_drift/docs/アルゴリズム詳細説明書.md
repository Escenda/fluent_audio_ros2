# fa_clock_drift アルゴリズム詳細説明書

## 1. Timing Pipeline

1. `AudioFrame` を受信する。
2. `source_id` / `stream_id` / format / byte alignment を検証する。
3. active stream identity の有無と変更を確認する。
4. baseline frame または drift correction frame として timestamp を計算する。
5. timestamp が有効な場合のみ publish する。
6. diagnostics counter と timeline state を更新する。

validation は drift state の前に実行する。契約違反 frame によって active stream や drift estimate が変化してはいけない。

## 2. State Machine

| state | description |
| --- | --- |
| `inactive` | active stream identity と previous output timestamp がない |
| `active` | active stream identity、previous output timestamp、drift estimate を保持している |
| `reset` | source / format change、threshold exceed、timestamp invalid により state を破棄する |

`inactive` で valid frame を受けると、その incoming `header.stamp` を baseline として `active` へ遷移する。`active` で source / format change を受けると state を reset し、その frame を新 baseline にする。

## 3. Frame Duration

frame duration は payload length と expected format から求める。

```text
bytes_per_sample_frame = expected.channels * expected.bit_depth / 8
sample_count = data.size / bytes_per_sample_frame
duration_seconds = sample_count / expected.sample_rate
```

payload が空、または `bytes_per_sample_frame` の倍数でない frame は validation で drop する。

## 4. Drift Estimate

後続 frame では、前回出力 timestamp に前回 accepted frame duration を加えた値を expected timestamp とする。current frame duration は、その frame が publish された後に次回予測用 state として保存する。

```text
expected_timestamp = previous_output_timestamp + previous_frame_duration
observed_drift = incoming_timestamp - expected_timestamp
estimate = (1 - alpha) * estimate + alpha * observed_drift
```

`alpha` は `0.0 < alpha <= 1.0` の finite 値に限定する。`alpha=1.0` では最新 observed drift をそのまま estimate とする。

## 5. Correction Limit

1 frame で出力 timestamp を大きく動かしすぎないため、EMA estimate は `+/- drift.max_correction_ms_per_frame` に制限する。C++ 実装では `std::clamp` は使わず、明示的な上限・下限比較で制限する。

負方向の補正は、前回 frame duration も下限として扱う。つまり output timestamp が前回 output timestamp 以下へ戻らない範囲でだけ負方向補正を許可する。

```text
negative_limit = min(max_correction_per_frame, previous_frame_duration - 1ns)
bounded_correction = max(estimate, -negative_limit)
output_timestamp = expected_timestamp + bounded_correction
```

制限が発生した frame は `correction_limited_frames` を increment する。

## 6. Reset Threshold

`abs(observed_drift) > drift.reset_threshold_ms` の場合、clock drift として補正可能な範囲を超えた discontinuity と判断する。この frame は drop せず、incoming timestamp を新 baseline として publish する。

これは hidden fallback ではなく、drift correction timeline の明示的な区切りである。reset は `timeline_resets` に記録され、drift estimate は 0 に戻る。

## 7. Timestamp Range

timestamp 計算は nanoseconds 単位で行う。次の場合は frame を drop し、timeline state を reset する。

- input timestamp の `nanosec >= 1_000_000_000`
- 計算値が non-finite
- 計算値が negative
- 計算値が `builtin_interfaces/Time` の最大範囲を超える

現在時刻への置換、zero timestamp への置換、latest timestamp の再利用は行わない。

## 8. Payload / Epoch

この node は payload を編集しない。出力は入力 frame を copy し、`header.stamp` と `stream_id` だけを設定する。`epoch` は通過させる。clock drift correction は timestamp metadata の処理であり、session generation を変更しないためである。

## 9. Realtime Constraints

処理は 1 frame あたり O(1) であり、buffer を持たない。state は active stream identity、previous output timestamp、drift estimate、counter のみである。
