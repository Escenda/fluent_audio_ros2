# fa_clock_drift 仕様書

## 1. 目的

`fa_clock_drift` は、`AudioFrame` stream の sample-clock drift によって少しずつずれる `header.stamp` metadata を bounded correction する ROS2 streaming node である。

## 2. 非目標

- sample rate conversion
- sample の drop / add
- padding
- normalize / gain / filter
- device I/O
- packet reordering
- latency の固定 offset 補正

## 3. 所属分類

- package: `src/streaming/fa_clock_drift`
- 分類: `streaming`
- node: `fa_clock_drift`
- executable: `fa_clock_drift_node`
- class: `fa_clock_drift::FaClockDriftNode`

## 4. Topic

| direction | name | type | contract |
| --- | --- | --- | --- |
| subscribe | `input_topic` | `fa_interfaces/msg/AudioFrame` | 補正前の frame。`stream_id` は `input_topic` と一致する |
| publish | `output_topic` | `fa_interfaces/msg/AudioFrame` | 補正後の frame。`stream_id` は `output_topic` に置き換える |
| publish | `diagnostics` | `diagnostic_msgs/msg/DiagnosticArray` | 設定値、counter、drift 推定値 |

## 5. Message Contract

入力 frame は次を満たす必要がある。

- `source_id` が空でない
- `stream_id` が空でない
- `stream_id == input_topic`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == expected.encoding`
- `bit_depth == expected.bit_depth`
- `layout == expected.layout`
- `data` が空でない
- `data.size()` が `channels * bit_depth / 8` の倍数

出力 frame は次を保証する。

- `source_id` は入力を保持する
- `data` は入力を保持する
- `epoch` は入力を保持する
- `stream_id` は `output_topic` に設定する
- `header.stamp` は clock drift 補正後の timestamp に設定する

## 6. Parameters

すべて必須であり、コード上の有効な default は持たない。

| parameter | valid range |
| --- | --- |
| `input_topic` | non-empty string |
| `output_topic` | non-empty string |
| `expected.sample_rate` | `> 0` |
| `expected.channels` | `> 0` |
| `expected.encoding` | non-empty string |
| `expected.bit_depth` | `> 0` and byte-aligned |
| `expected.layout` | non-empty string |
| `drift.ema_alpha` | finite and `0.0 < alpha <= 1.0` |
| `drift.max_correction_ms_per_frame` | finite and `>= 0.0` |
| `drift.reset_threshold_ms` | finite and `> 0.0` |
| `qos.depth` | `> 0` |
| `qos.reliable` | bool |
| `diagnostics.publish_period_ms` | `> 0` |

## 7. Lifecycle / State

node は 1 つの active stream identity だけを保持する。active stream identity は `source_id`, `stream_id`, `sample_rate`, `channels`, `encoding`, `bit_depth`, `layout` で構成する。

初回 valid frame または source / format change では、incoming `header.stamp` を baseline として sample-clock timeline を開始し、その frame を baseline frame として publish する。

## 8. Drift Correction

後続 valid frame では次の処理を行う。

1. `sample_count = data.size / (channels * bit_depth / 8)` を求める。
2. `duration = sample_count / expected.sample_rate` を求める。
3. `expected_timestamp = previous_output_timestamp + previous_frame_duration` を求める。
4. `observed_drift = incoming_header_stamp - expected_timestamp` を求める。
5. `abs(observed_drift) > drift.reset_threshold_ms` の場合は timeline を reset し、incoming frame を新 baseline として publish する。
6. それ以外では `estimate = (1-alpha)*estimate + alpha*observed_drift` を求める。
7. `estimate` を `+/- drift.max_correction_ms_per_frame` に制限する。ただし負方向補正は `previous_frame_duration - 1ns` 以下に制限し、output timestamp が前回 output timestamp 以前へ戻らないようにする。
8. `output_stamp = expected_timestamp + bounded_correction` として publish する。

current frame の `duration` は、その frame が publish された後に次 frame の `previous_frame_duration` として保存する。可変 chunk 長でも、次 frame の開始時刻予測に「現在 frame の長さ」を先取りして使わない。

## 9. Fail Closed

起動時 parameter が不正な場合は node を起動しない。

runtime では、契約違反 frame は drift state 変更前に drop する。計算した timestamp が negative、non-finite、または `builtin_interfaces/Time` 範囲外の場合は frame を drop し、timeline state を reset する。現在時刻や zero timestamp への代替は行わない。

## 10. Diagnostics

diagnostics には次を含める。

- config: input / output topic、expected format、drift parameter、QoS
- `frames_in`
- `frames_out`
- `frames_dropped`
- `timeline_resets`
- `drift_estimate_ms`
- `last_observed_drift_ms`
- `correction_limited_frames`

## 11. Config Example

```yaml
fa_clock_drift:
  ros__parameters:
    input_topic: "audio/sample_format/mic"
    output_topic: "audio/clock_drift_corrected/mic"
    expected:
      sample_rate: 16000
      channels: 1
      encoding: "FLOAT32LE"
      bit_depth: 32
      layout: "interleaved"
    drift:
      ema_alpha: 0.1
      max_correction_ms_per_frame: 2.0
      reset_threshold_ms: 50.0
```

## 12. Backend

runtime backend は持たない。sample-clock timeline は node 内部 state として管理する。
