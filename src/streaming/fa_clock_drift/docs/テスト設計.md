# fa_clock_drift テスト設計

## 1. Scope

本 package の unit test は repository-local contract test として、C++ node の構造、parameter validation、runtime validation 順序、drift correction logic、diagnostics を検証する。

package-local integration / launch contract test は実行可能な pytest として置く。ROS2 pub/sub を伴う runtime graph contract は `ament_cmake_gtest` で C++ node core を直接起動し、`AudioFrame` の data-plane 契約を検証する。

## 2. Specification IDs

| ID | Spec |
| --- | --- |
| FA-CLOCK-DRIFT-SPEC-001 | package は標準 streaming layout を持つ |
| FA-CLOCK-DRIFT-SPEC-002 | node identity は `fa_clock_drift` / `fa_clock_drift_node` / `FaClockDriftNode` |
| FA-CLOCK-DRIFT-SPEC-003 | startup parameters は default なしで宣言され、不正値では起動失敗する |
| FA-CLOCK-DRIFT-SPEC-004 | runtime validation は drift state mutation より前に実行される |
| FA-CLOCK-DRIFT-SPEC-005 | 初回 valid frame と source / format change は baseline timeline を開始する |
| FA-CLOCK-DRIFT-SPEC-006 | observed drift は EMA で推定し、per-frame correction は上限で制限する |
| FA-CLOCK-DRIFT-SPEC-007 | threshold 超過は timeline reset と baseline publish になる |
| FA-CLOCK-DRIFT-SPEC-008 | computed timestamp invalid は drop と timeline reset になる |
| FA-CLOCK-DRIFT-SPEC-009 | source_id、payload、epoch を保持し、stream_id だけを `output.stream_id` に変える |
| FA-CLOCK-DRIFT-SPEC-010 | diagnostics は config、counter、drift estimate を含む |
| FA-CLOCK-DRIFT-SPEC-011 | device I/O、sample conversion、payload edit、legacy path を持たない |
| FA-CLOCK-DRIFT-SPEC-012 | CMake / package.xml は pytest contract test dependencies を持つ |
| FA-CLOCK-DRIFT-SPEC-013 | integration contract は data-plane config と runtime source contract を検証する |
| FA-CLOCK-DRIFT-SPEC-014 | launch contract は node_name / config_file のみを公開し、backend 選択を漏らさない |
| FA-CLOCK-DRIFT-SPEC-015 | ROS graph contract は baseline publish、bounded correction、reset threshold、source switch、invalid format drop を検証する |
