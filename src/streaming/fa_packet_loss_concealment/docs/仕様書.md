# fa_packet_loss_concealment 仕様書

## 1. 目的

`fa_packet_loss_concealment` は、FluentAudio の `streaming` に属する明示的な PLC（packet loss concealment）node である。

入力 `AudioFrame.epoch` の欠落を検出した場合に限り、直前の有効 frame を減衰付きで繰り返して、短い欠落を埋める。system config でこの node を pipeline に挟んだ場合だけ有効になる処理であり、欠損を隠す暗黙 fallback ではない。

ROS topic は transport identity としてのみ扱う。data-plane の音声 stream identity は `input_stream_id` / `output.stream_id` で指定し、`AudioFrame.stream_id` に topic 名を流用しない。

## 2. 非目標

- device I/O
- resampling
- channel / bit depth / layout 変換
- padding
- arbitrary sample generation
- large gap の推測補完
- legacy alias

- package: `fa_packet_loss_concealment`

| 種別 | topic | message | 条件 |
| --- | --- | --- | --- |
| subscribe | `input_topic` | `fa_interfaces/msg/AudioFrame` | 必須 parameter |

## 5. AudioFrame contract

入力 frame は次を満たす必要がある。

- `source_id` が空でない。
- `stream_id` が空でない。
- `stream_id == input_stream_id`。
- `sample_rate == expected.sample_rate`。
- `channels == expected.channels`。
- `encoding == FLOAT32LE`。
- `bit_depth == 32`。
- `layout == interleaved`。
- `data` が空でない。
- `data.size()` が `channels * sizeof(float)` で割り切れる。
- 全 sample が finite かつ `[-1.0, 1.0]` に収まる。

不正 frame は PLC state を変更する前に drop する。

すべて required parameter であり、C++ 側に有効な code default は持たない。

| parameter | 型 | 制約 |

| `output_topic` | string | 空文字禁止 |
| `input_stream_id` | string | 空文字禁止。raw / resolved ROS topic identity と異なる |
| `output.stream_id` | string | 空文字禁止。raw / resolved ROS topic identity と異なり、`input_stream_id` と異なる |
| `expected.sample_rate` | int | `> 0` |
| `expected.channels` | int | `> 0` |
| `expected.encoding` | string | `FLOAT32LE` のみ |

| `expected.layout` | string | `interleaved` のみ |
| `plc.max_gap_frames` | int | `>= 0` |

## 7. PLC behavior

前回の有効 frame があり、次の有効 frame の `epoch` が前回より 2 以上進んでいる場合、欠落 frame 数を `missing_frame_count = current.epoch - previous.epoch - 1` とする。

- `missing_frame_count <= plc.max_gap_frames`: 欠落 epoch ごとに synthetic frame を publish する。
- `missing_frame_count > plc.max_gap_frames`: synthetic frame は publish しない。古い baseline を破棄し、現在 frame だけを新 baseline として publish する。
- 前回 frame がない場合: synthetic frame は publish しない。
- duplicate または regression epoch: frame を drop し、baseline は変更しない。

synthetic frame は次を満たす。

- `stream_id == output.stream_id`
- `epoch == missing_epoch`
- `data == previous.data * pow(plc.attenuation_per_gap, gap_index)`
- format field は previous frame と同じ

current frame を pass-through publish する場合も、payload の `stream_id` は `output.stream_id` に置き換える。入力 stream identity を継承しない。

## 8. Diagnostics

`diagnostics` には少なくとも次を含める。

- config: `input_topic`, `output_topic`, `input_stream_id`, `output_stream_id`, `expected_*`, `plc_*`, `qos_*`, `diagnostics_publish_period_ms`, `diagnostics_qos_depth`, `diagnostics_qos_reliable`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `concealed_frames`
- `duplicate_drops`
- `gap_resets`
- `backend.name=repeat_attenuation_plc`

## 9. Fail closed 条件

- required parameter が未設定または不正。
- `input_stream_id` / `output.stream_id` が空、ROS topic identity と同値、または互いに同値。
- `FLOAT32LE / 32bit / interleaved` 以外が指定された。
- `plc.attenuation_per_gap` が finite でない、または範囲外。
- 入力 frame contract に違反した。
- PLC timestamp advance が ROS2 `builtin_interfaces/Time` の範囲を超える。

起動 parameter の不正は例外で起動失敗にする。runtime frame の不正は drop し、counter と warning に出す。
