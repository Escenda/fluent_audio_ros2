# fa_packet_loss_concealment テスト設計

## 1. 方針

この package の unit test は既存 streaming package と同じく、text / contract test を中心にする。package-local integration / launch contract test は実行可能な pytest として置く。ROS2 pub/sub を伴う runtime graph contract は `ament_cmake_gtest` で C++ node core を直接起動し、PLC の data-plane 契約を検証する。

## 2. 仕様 ID

| ID | 仕様 |
| --- | --- |
| FA-PLC-SPEC-001 | 標準 package layout を満たす |
| FA-PLC-SPEC-002 | default config は required parameter を明示する |
| FA-PLC-SPEC-003 | 起動 parameter 不正時は fail closed する |
| FA-PLC-SPEC-004 | 入力 frame は state mutation 前に contract validation される |
| FA-PLC-SPEC-005 | PLC は `FLOAT32LE / 32bit / interleaved` のみを扱う |
| FA-PLC-SPEC-006 | 欠落 epoch は previous frame の repeat attenuation で合成する |
| FA-PLC-SPEC-007 | 上限を超える gap は合成せず、現在 frame だけを publish する |
| FA-PLC-SPEC-008 | duplicate / regression epoch は drop し、baseline を変えない |
| FA-PLC-SPEC-009 | diagnostics は config と counters を publish する |
| FA-PLC-SPEC-010 | CMake / package.xml は pytest と lint dependencies を持つ |
| FA-PLC-SPEC-011 | integration contract は data-plane config と runtime source contract を検証する |
| FA-PLC-SPEC-012 | launch contract は node_name / config_file のみを公開し、backend 選択を漏らさない |
| FA-PLC-SPEC-013 | ROS graph contract は欠落 epoch 合成、oversized gap、duplicate drop、source switch、invalid sample drop を検証する |
| FA-PLC-SPEC-014 | ROS topic identity と `AudioFrame.stream_id` を分離し、入力は `input_stream_id`、出力は `output.stream_id` を使う |
