# fa_packet_loss_concealment テスト設計

## 1. 方針

この package の unit test は既存 streaming package と同じく、text / contract test を中心にする。ROS2 launch 経路と実 runtime の結合 test は `test/integration` / `test/launch` の拡張対象とする。

## 2. 仕様 ID

| ID | 仕様 |
| --- | --- |
| FA-PLC-SPEC-001 | 標準 package layout を満たす |
| FA-PLC-SPEC-002 | default config は required parameter を明示する |
| FA-PLC-SPEC-003 | 起動 parameter 不正時は fail closed する |
| FA-PLC-SPEC-004 | 入力 frame は state mutation 前に contract validation される |
| FA-PLC-SPEC-005 | PLC は `FLOAT32LE / 32bit / interleaved` のみを扱う |
| FA-PLC-SPEC-006 | 欠落 epoch は previous frame の repeat attenuation で合成する |
| FA-PLC-SPEC-007 | 上限を超える gap は合成せず、現在 frame だけを publish する |
| FA-PLC-SPEC-008 | duplicate / regression epoch は drop し、baseline を変えない |
| FA-PLC-SPEC-009 | diagnostics は config と counters を publish する |
| FA-PLC-SPEC-010 | CMake / package.xml は pytest と lint dependencies を持つ |

## 3. Test case 対応

| Test case | 対応仕様 |
| --- | --- |
| `test_package_layout_matches_required_processing_layout` | FA-PLC-SPEC-001 |
| `test_default_config_declares_explicit_plc_contract` | FA-PLC-SPEC-002, FA-PLC-SPEC-005 |
| `test_startup_validation_fails_closed_for_invalid_config` | FA-PLC-SPEC-003 |
| `test_runtime_validation_rejects_invalid_frames_before_state_mutation` | FA-PLC-SPEC-004, FA-PLC-SPEC-005 |
| `test_plc_synthesizes_missing_epochs_with_repeat_attenuation` | FA-PLC-SPEC-006 |
| `test_gap_larger_than_max_resets_and_publishes_current_only` | FA-PLC-SPEC-007 |
| `test_duplicate_and_regressing_epochs_drop_without_baseline_update` | FA-PLC-SPEC-008 |
| `test_diagnostics_publish_config_and_required_counters` | FA-PLC-SPEC-009 |
| `test_colcon_runs_pytest_contracts_and_lint_auto` | FA-PLC-SPEC-010 |
