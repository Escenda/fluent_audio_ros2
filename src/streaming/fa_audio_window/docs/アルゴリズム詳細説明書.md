# fa_audio_window アルゴリズム詳細説明書

## 0. この資料の位置づけ

この資料は `fa_audio_window` の current implementation に基づき、AudioFrame retention、range export、archive metadata、決定的 identity の処理手順を説明する。

ここで説明するアルゴリズムは local PCM16LE/WAV export / archive に限定する。ASR、hidden conversion、durable storage、World Station artifact 化は対象外である。

## 1. 全体フロー

`fa_audio_window` の処理は次の順序で進む。

1. 起動時に config を読み、expected AudioFrame format、retention、scope、export format、window identity を検証する。
2. `AudioFrame` subscription で frame を受け取る。
3. source / stream identity、format、payload alignment を検証する。
4. frame timestamp と sample count から frame time range を算出する。
5. retention window に frame を追加し、保持期限外の frame を削除する。
6. export / archive service request で numeric time range を parse する。
7. scope と codec / container / payload format を検証する。
8. retained window から連続する PCM16LE samples を抽出する。
9. resolved exported range を確定する。
10. 決定的 `clip_id` と target path を生成する。
11. temporary WAV を書き、既存 target があれば bytes 一致を検証する。
12. archive の場合は metadata JSON も決定的に検証または書き込む。
13. `AudioClipRef` と resolved time range を返す。

## 2. 起動時 validation

起動時に次を検証する。

- `input_topic`、`service_name`、`archive_service_name` が空でない。
- export service と archive service の名前が同一でない。
- `input.source_id` と `input.stream_id` が空でない。
- expected format が PCM16LE / 16-bit / interleaved である。
- sample rate と channels が 0 でない。
- `window.retention_seconds` が 0 より大きい。
- `audio.default_scope` が空でなく、`audio.supported_scopes` に含まれる。
- export codec / container / payload format が空でない。
- QoS depth が 0 より大きい。

起動時に静的に壊れている設定は、後続 frame や service request で吸収しない。

## 3. AudioFrame retention

受信した `AudioFrame` は、保持前に検証する。

- `source_id` / `stream_id` が config と一致すること。
- encoding、sample rate、channels、bit depth、layout が expected format と一致すること。
- payload が空でなく、sample-frame boundary に整列していること。

検証に失敗した frame は retention window に入れない。`fa_audio_window` は frame を resample / downmix / format conversion して受け入れない。

frame duration は sample count と configured sample rate から nanosecond 単位で算出する。timestamp overflow や duration 0 は frame rejection になる。

retention buffer は timestamp 順に frame を保持し、overlap frame を拒否する。新しい frame の end timestamp を基準に `retention_ns` より古い frame を削除する。

## 4. Range query

service request の `time_range_spec` は numeric `start_unix_ns..end_unix_ns` のみを受ける。

parse 失敗、数値以外、separator 不正、`0 <= start < end` を満たさない range は `time_range_unresolved` とする。

window query は次を判定する。

- window が空なら `window_not_found`。
- requested range が retained range の外なら `range_outside_window`。
- requested range を連続 coverage できない gap があれば non-continuous failure。
- sample selection が空なら export / archive failure。

抽出範囲は sample boundary に合わせた resolved exported range として返す。requested range と resolved exported range が sample boundary の都合で完全一致しない場合でも、返却値は実際に export した sample range を表す。

## 5. Scope / format validation

`audio_scope` が空の場合は configured default scope に解決する。非空の場合はその値を使う。

解決後 scope が `audio.supported_scopes` に含まれない場合は `unsupported_audio_scope` とする。unsupported scope を default scope に落とす fallback は行わない。

現在の export / archive format は config の codec / container / payload format と完全一致する値だけを受ける。既定値は `pcm_s16le` / `wav` / `audio/wav` である。unsupported format を別 format に変換して成功扱いにしない。

## 6. 決定的 clip identity

`clip_id` は request と resolved window fact から決定的に生成する。

identity source は length-delimited field sequence として連結し、stable FNV-1a 64-bit hash を取る。basename には human-readable な operation、window identity、source / stream identity、range、format と hash を含める。file path は `export.output_directory/<clip_id>.wav` である。

export identity に含める値:

- operation: `export_audio_window`
- `window_id`
- `window_epoch`
- `source_id`
- `stream_id`
- resolved audio scope
- resolved exported `start_unix_ns`
- resolved exported `end_unix_ns`
- codec
- container
- payload_format

archive identity に追加で含める値:

- operation: `archive_audio_window`
- reason
- related artifact id count
- related artifact ids

archive の reason と related artifact ids は metadata identity の一部であるため、同じ音声 range でも reason や related artifact ids が異なれば別 identity になる。

## 7. WAV publish algorithm

WAV publish は target と同じ directory に per-attempt temporary path を作成して使う。

1. `O_CREAT | O_EXCL` で temporary path を予約する。
2. requested window から抽出した PCM16LE samples を temporary WAV として書く。
3. POSIX `link(temporary, target)` で no-clobber publish する。
4. link が成功した場合は temporary file を削除し、新規 publish 成功とする。
5. target path が存在する場合、target bytes と temporary bytes を比較する。
6. bytes が同一なら temporary file を削除し、既存 target を成功として使う。
7. bytes が異なるなら temporary file を削除し、export / archive failure とする。
8. target path が存在しないのに link が失敗した場合は write failure とする。

この手順により、同じ request の再実行は同じ `AudioClipRef` を返す。既存 target の内容が異なる場合は上書きせず fail closed する。

## 8. Archive metadata algorithm

archive の場合は WAV target に加えて `*.metadata.json` を扱う。

metadata には次を含める。

- schema
- operation
- reason
- related artifact ids
- source id
- stream id
- window id
- window epoch
- audio scope
- resolved time range
- audio clip ref

WAV target が既存 bytes と一致した場合も、metadata path は同じ no-clobber publish 手順で検証または作成する。metadata が期待値と同一なら成功、欠損していれば atomically 作成、異なる場合は archive failure とする。

metadata は temporary path に書き、hard link による no-clobber publish を行う。既存 metadata が同一なら成功、異なるなら failure、target が存在しないのに publish できなければ write failure にする。metadata publish に失敗した場合、新規 publish した WAV target は削除する。

metadata だけが存在し、対応する WAV clip が存在しない状態は archive failure とする。

## 9. Failure mapping

failure は service response の `success=false` と明示 `error_code` / `message` で返す。

| 内部状態 | ExportAudioWindow | ArchiveAudioWindow |
| --- | --- | --- |
| time range parse failure | `time_range_unresolved` | `time_range_unresolved` |
| empty window | `window_not_found` | `window_not_found` |
| outside window | `range_outside_window` | `range_outside_window` |
| non-continuous window | `range_outside_window` | `range_not_continuous` |
| unsupported scope | `unsupported_audio_scope` | `unsupported_audio_scope` |
| unsupported format | `unsupported_export_format` | `unsupported_archive_format` |
| no samples selected | `export_failed` | `archive_failed` |
| bytes / metadata conflict | `export_failed` | `archive_failed` |
| missing archive reason | 対象外 | `invalid_archive_request` |

失敗時に空 `AudioClipRef` を正常結果として扱わない。

## 10. 検証済み範囲

検証済み:

- window retention と range query。
- WAV writer の PCM16LE header / payload。
- service failure contract。
- repeated export request の deterministic `AudioClipRef`。
- deterministic export target conflict の fail-closed。
- duplicate export/archive service name の起動時拒否。
- repeated archive request の deterministic WAV bytes / metadata。
- deterministic archive target / metadata conflict の fail-closed。
- archive service の success / explicit error contract。

Docker/ROS で `colcon build --packages-select fa_audio_window --event-handlers console_direct+ && colcon test --packages-select fa_audio_window --event-handlers console_direct+` は成功している。service contract test は 8 tests、package test summary は 3 test binaries all passed である。
