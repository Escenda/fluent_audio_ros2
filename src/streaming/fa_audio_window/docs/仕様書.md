# fa_audio_window 仕様書

## 0. この資料の位置づけ

この資料は `src/streaming/fa_audio_window` package の現在実装済み slice を記述する仕様書である。

ここで扱う範囲は、保持済み `AudioFrame` window から evidence 用 PCM16LE/WAV clip を export / archive する挙動である。durable object storage、World Station への artifact 書き込み、親 Agent Runtime MCP client、親 VLAbor profile integration、実 SO101 device / model provisioning の完了を示す資料ではない。

## 1. 責務

`fa_audio_window` は、evidence 音声 window の archive / export owner である。

- `AudioFrame` を短期 rolling window として保持する。
- numeric `start_unix_ns..end_unix_ns` の time range を受ける。
- 保持済み window から対象範囲の PCM16LE sample を切り出す。
- PCM16LE/WAV clip を local file として書き出す。
- export / archive 結果として `AudioClipRef` と resolved time range を返す。
- archive では reason / related artifact を含む sidecar metadata を書く。

## 2. 非目標

`fa_audio_window` は ASR ではない。

- 文字起こしを行わない。
- ASR backend を呼ばない。
- PCM16LE から FLOAT32LE への hidden conversion を行わない。
- resample、downmix、bit-depth conversion、sample format conversion を隠して実行しない。
- Agent LLM の保存判断、World Station artifact 化、durable storage への upload を担当しない。

変換が必要な場合は、`fa_sample_format`、`fa_resample`、`fa_channel_convert` などの専用 node を pipeline に明示する。

## 3. 入力 contract

### 3.1 AudioFrame

現在の実装は、設定された source / stream identity と PCM16LE format に一致する `AudioFrame` だけを保持する。

| 項目 | 現在の既定値 |
| --- | --- |
| `source_id` | `mic` |
| `stream_id` | `audio/mic` |
| encoding | `PCM16LE` |
| sample rate | `16000` |
| channels | `1` |
| bit depth | `16` |
| layout | `interleaved` |

source / stream identity、format、payload alignment が一致しない frame は保持しない。`fa_audio_window` は不一致 frame を変換して受け入れない。

### 3.2 Retention

rolling window の既定 retention は `window.retention_seconds=1800` 秒である。config/profile から変更できるが、0 以下は起動時に拒否する。

rolling window は永続保存ではない。指定された time range が保持範囲外の場合、空 clip 成功ではなく明示失敗にする。

## 4. Service contract

### 4.1 `ExportAudioWindow`

`ExportAudioWindow` は、保持済み window の numeric time range を WAV clip として export する。

| request | 内容 |
| --- | --- |
| `time_range_spec` | numeric `start_unix_ns..end_unix_ns` |
| `audio_scope` | 空の場合は configured default scope。非空の場合は supported set に含まれる必要がある |
| `codec` | 現在は configured `pcm_s16le` のみ |
| `container` | 現在は configured `wav` のみ |
| `payload_format` | 現在は configured `audio/wav` のみ |

成功時は `success=true`、`error_code=none`、`AudioClipRef`、resolved time range を返す。

### 4.2 `ArchiveAudioWindow`

`ArchiveAudioWindow` は、Agent Runtime が保存すべきと判断した音声範囲を evidence clip として archive する。

| request | 内容 |
| --- | --- |
| `time_range_spec` | numeric `start_unix_ns..end_unix_ns` |
| `audio_scope` | export と同じ scope contract |
| `reason` | archive 理由。空白のみは失敗 |
| `related_artifact_ids` | archive metadata identity に含める関連 artifact id |
| `codec` / `container` / `payload_format` | export と同じ format contract |

成功時は WAV clip と同じ basename の `*.metadata.json` を local に書く。metadata には reason、related artifact ids、source / stream / window identity、resolved time range、`AudioClipRef` を含める。

## 5. 決定的 media identity

export / archive の `clip_id` と local file path は決定的に生成する。sequence 採番による別名生成は行わない。

export identity は次の値から導出する。

- operation
- `window_id`
- `window_epoch`
- `source_id`
- `stream_id`
- resolved audio scope
- resolved exported range
- codec
- container
- payload_format

archive identity は上記に加えて、metadata identity の一部である次の値を含める。

- reason
- related_artifact_ids

同じ export / archive request を同じ window 状態に対して繰り返した場合、同じ `AudioClipRef` を返し、sequence-based な別 clip ref は作らない。

## 6. Conflict policy

決定的 target が既に存在する場合の扱いは fail closed である。

- 既存 WAV bytes が今回生成される bytes と同一なら成功扱いにし、同じ `AudioClipRef` を返す。
- 既存 WAV bytes が異なる場合は export / archive failure にする。
- archive で既存 metadata が今回期待する metadata と同一なら成功扱いにする。
- archive で metadata が欠損している場合は no-clobber publish で atomically 作成する。
- archive で既存 metadata が異なる場合は archive failure にする。
- archive metadata だけが存在し、対応する WAV clip がない場合は archive failure にする。

既存 target の差異を上書きで吸収しない。

## 7. 明示失敗

次は成功扱いにしない。

| 条件 | 扱い |
| --- | --- |
| time range parse failure | `time_range_unresolved` |
| empty window | `window_not_found` |
| requested range outside retained window | `range_outside_window` |
| non-continuous retained window | export は range outside 系、archive は `range_not_continuous` |
| selected sample なし | export / archive failure |
| unsupported audio scope | `unsupported_audio_scope` |
| unsupported codec / container / payload format | unsupported format |
| archive reason 欠損 | `invalid_archive_request` |
| WAV / metadata 書き込み失敗 | export / archive failure |
| deterministic target conflict | export / archive failure |

`fa_audio_window` は、range 解決失敗を `now` に置き換えたり、空 clip を成功として返したりしない。

## 8. 検証状況

確認済み:

- Docker/ROS 環境で `colcon build --packages-select fa_audio_window --event-handlers console_direct+ && colcon test --packages-select fa_audio_window --event-handlers console_direct+` が成功。
- service contract test は 8 tests。
- package test summary は 3 test binaries すべて passed。

未検証:

- durable object storage への保存。
- World Station への evidence / timeline artifact 書き込み。
- 親 Agent Runtime MCP client からの呼び出し。
- 親 VLAbor profile integration。
- 実 SO101 device / model provisioning。

この package-local slice は、PCM16LE/WAV window export / archive の deterministic identity と fail-closed service contract が検証済みである。上記未検証範囲を含む system 完了はまだ主張しない。
