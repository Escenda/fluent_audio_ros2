# fa_overlap_add 仕様書

## 目的

`fa_overlap_add` は、重複を持つ `FLOAT32LE` interleaved audio chunk を synthesis window で overlap-add し、hop-sized frame として連続 stream に戻す streaming node である。

device I/O、resample、bit-depth conversion、channel conversion、gain、limiter、clamp、tail padding はこの package の責務外とする。

## 所属分類

`src/streaming`

同期・ストリーミング系の overlap-add package として扱う。

## 入力

Topic は `input_topic` parameter で指定する。message 型は `fa_interfaces/msg/AudioFrame`。

必須条件:

- `source_id` が空でない
- `stream_id` が空でなく、`input_topic` と一致する
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == "FLOAT32LE"`
- `bit_depth == 32`
- `layout == "interleaved"`
- `data.size() == window.frame_samples * channels * sizeof(float)`
- 全 sample が finite かつ `[-1.0, 1.0]` の範囲内

条件を満たさない frame は drop する。drop 時に既存 overlap-add state は変更しない。

## 出力

Topic は `output_topic` parameter で指定する。message 型は `fa_interfaces/msg/AudioFrame`。

出力 frame:

- `source_id` は active source を引き継ぐ
- `stream_id` は `output_topic` に更新する
- `encoding`、`sample_rate`、`channels`、`bit_depth`、`layout` は入力 contract を保持する
- `data` は `window.hop_samples * channels` sample を含む
- `epoch` は node lifetime 内で publish ごとに単調増加する
- `header.stamp` は出力 hop 先頭 sample frame の timestamp とする

## Parameters

全 parameter は起動時必須であり、コード上の有効 default は持たない。

| parameter | 条件 |
| --- | --- |
| `input_topic` | non-empty |
| `output_topic` | non-empty |
| `expected.sample_rate` | `> 0` |
| `expected.channels` | `> 0` |
| `expected.encoding` | `FLOAT32LE` |
| `expected.bit_depth` | `32` |
| `expected.layout` | `interleaved` |
| `window.frame_samples` | `> 0` |
| `window.hop_samples` | `> 0` and `<= window.frame_samples` |
| `window.type` | `rectangular` or `hann` |
| `overlap.max_buffered_chunks` | `> 0` |
| `qos.depth` | `> 0` |
| `qos.reliable` | bool |
| `diagnostics.publish_period_ms` | `> 0` |
| `diagnostics.qos.depth` | `> 0` |
| `diagnostics.qos.reliable` | bool |

不正 parameter は起動失敗とする。

## Window

`window.type=rectangular` は全 sample に weight `1.0` を適用する。

`window.type=hann` は sample-center の Hann synthesis window を使う。端点 weight が 0 にならないため、stream 先頭の hop も非 finite な正規化に落ちない。

## State / reset

次の場合、overlap-add state を reset する。

- valid frame の `source_id` が active source と異なる
- valid frame の format fields が active stream と異なる
- valid frame の `epoch` が次に期待する input epoch より未来へ飛んだ
- 正規化済み出力 sample が non-finite または `[-1.0, 1.0]` 外になる
- accumulator が `overlap.max_buffered_chunks` を超える

duplicate / regressing input `epoch` は、古い音声を新しい output epoch として再発行しないため drop する。invalid frame は reset trigger として扱わない。

## Diagnostics

`diagnostics` に以下を含む:

- config: `input_topic`, `output_topic`, `expected_*`, `frame_samples`, `hop_samples`, `window_type`, `max_buffered_chunks`
- counters: `frames_in`, `frames_out`, `frames_dropped`, `epoch_regression_drops`, `chunks_accumulated`, `resets`
- state: `buffered_sample_frames`
- backend: `internal_overlap_add`

## Backend

runtime backend は ROS2 node 内の `internal_overlap_add` のみである。外部 worker、device backend、resampler は使わない。
