# fa_overlap_add テスト設計

## Unit contract test

`test/unit/test_fa_overlap_add_audio_frame_contract.py` で以下を確認する。

| test case | 対応仕様 |
| --- | --- |
| FA-OA-TC-001 default config | required parameter と `FLOAT32LE`/interleaved/window/overlap contract |
| FA-OA-TC-002 startup validation | invalid config は起動失敗し、コード上の有効 default を持たない |
| FA-OA-TC-003 runtime validation | identity、topic、format、exact chunk size、finite normalized sample を確認する |
| FA-OA-TC-004 mutation order | invalid frame は accumulator mutation 前に return する |
| FA-OA-TC-005 window handling | `rectangular` と `hann` のみ許可し、unknown window を reject する |
| FA-OA-TC-006 overlap-add | synthesis window weight を sample sum / weight sum に積み、output は weight で正規化する |
| FA-OA-TC-007 output safety | output range 違反は clamp せず reset/drop する |
| FA-OA-TC-008 reset behavior | source / format / future epoch gap で state reset し、output epoch は単調増加を保つ |
| FA-OA-TC-009 epoch regression | duplicate / regressing input epoch は stale audio replay を避けるため drop する |
| FA-OA-TC-010 diagnostics | config、frames、drop、epoch regression、chunk、reset、buffer counter を publish する |
| FA-OA-TC-011 package layout | required files と CMake/package test deps が存在する |
| FA-OA-TC-012 ROS graph contract | rectangular window の hop 復元、invalid frame state 保持、future gap reset、duplicate drop を検証する |

## Integration / launch contract tests

`test/integration/test_fa_overlap_add_streaming_config_contract.py` で以下を確認する。

- default config が明示的な input / output topic、QoS、diagnostics contract を持つ
- runtime source が `AudioFrame`、validation、diagnostics を含む

`test/launch/test_fa_overlap_add_streaming_launch_contract.py` で以下を確認する。

- launch file は `node_name` と `config_file` だけを公開する
- package / executable が `fa_overlap_add` の契約に一致し、config path は launch argument で明示する
- streaming node の launch に `backend.name` を漏らさない

## 未検証の ROS graph integration

`test/cpp/test_fa_overlap_add_node_contract.cpp` は C++ node core を起動し、ROS2 pub/sub 経由で以下を確認する。

- `fa_chunk_overlap` の出力を入力し、元の hop sequence が復元される
- invalid frame 送信後、直前の valid accumulator state が変更されない
- future input epoch gap 後、古い tail が新 stream に混入しない
- duplicate / regressing input epoch が新しい output epoch として再発行されない

親 system 登録後に残る未検証項目は、実際の system profile から include した状態で `hann` と `rectangular` の diagnostics が publish されること。

## 非目標

この package では device I/O、resample、format conversion、limiter、音質主観評価は行わない。
