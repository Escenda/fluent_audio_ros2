# fa_overlap_add アルゴリズム詳細説明書

## 前提

入力は `FLOAT32LE`、32 bit、interleaved、正規化済み sample の byte array である。各 input `AudioFrame` は `window.frame_samples` sample frames ちょうどの overlapped chunk を表す。

## Streaming 手順

1. 受信 frame の `source_id`、`stream_id`、format fields、data size、sample 値を検証する。
2. 検証に失敗した場合は drop し、accumulator を変更しない。
3. active stream が存在し、valid frame の `source_id` または format が変わった場合は accumulator を reset する。
4. 同一 stream で duplicate / regressing `epoch` を受けた場合は stale audio replay を避けるため drop する。
5. 同一 stream で future `epoch` gap を受けた場合は accumulator を reset する。
6. active stream が未設定の場合は、受信 frame の identity と `header.stamp` を保存する。
7. chunk の開始位置を、未 publish buffer 先頭から `next_chunk_start_sample_frames` として決める。
8. synthesis window を input chunk に掛け、sample sum と weight sum に加算する。
9. `next_chunk_start_sample_frames` を `window.hop_samples` だけ進める。
10. `next_chunk_start_sample_frames >= window.hop_samples` の間、先頭 hop は将来 chunk によって更新されないため publish 対象にする。
11. 各 output sample は `sample_sum / weight_sum` で正規化する。
12. output sample が non-finite または `[-1.0, 1.0]` 外になる場合は publish せず state を reset する。
13. publish 後、先頭 `window.hop_samples` sample frames を accumulator から削除し、timestamp を hop 分進める。

## Accumulator

Accumulator は 2 種類の配列を持つ。

- `sample_sums`: channel ごとの weighted sample sum
- `weight_sums`: sample frame ごとの synthesis window weight sum

channel ごとの正規化では、同じ sample frame の `weight_sums` を全 channel に適用する。

## Window

`rectangular`:

```text
w[n] = 1.0
```

`hann`:

```text
w[n] = 0.5 - 0.5 * cos(2π(n + 0.5) / frame_samples)
```

sample-center の Hann を使う理由は、stream 境界で weight 0 による非 finite 正規化を発生させないためである。これは missing sample を 0 で埋める fallback ではなく、起動時に明示された synthesis window の定義である。

## Output readiness

Chunk を 1 つ受け取ると、その chunk の先頭から次 chunk 開始位置までの hop 分だけが確定する。未確定 tail は buffer に残し、zero padding で出力しない。

例: `frame_samples=512`, `hop_samples=256`

- chunk 0 を受信後: `[0, 256)` を publish 可能、`[256, 512)` は残す
- chunk 1 を受信後: `[256, 512)` を publish 可能、`[512, 768)` は残す

## Reset

Reset は accumulator、active stream、次 input epoch、次 output timestamp、chunk 開始位置を消去する。`next_output_epoch` は node lifetime の単調増加 counter なので reset しない。

## fail-closed

invalid frame、buffer overflow、非 finite output、range 外 output は、値を補完せず drop または reset する。clamp、zero padding、tail frame 合成は行わない。
