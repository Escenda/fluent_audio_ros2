# fa_chunk_overlap アルゴリズム詳細説明書

## 前提

入力は `FLOAT32LE`、32 bit、interleaved、正規化済み sample の byte array である。node は byte列の意味を変更しない。

## Streaming 手順

1. 受信 frame の `source_id`、`stream_id`、format fields、data alignment、sample 値を検証する。
2. 検証に失敗した場合は drop し、既存 buffer を変更しない。
3. active source が存在し、受信 frame の `source_id` が異なる場合は buffer を clear する。
4. active source が未設定の場合は、受信 frame の identity を active stream として保存する。
5. 受信 `data` を buffer 末尾に追加する。
6. buffer が `window.frame_samples * channels * sizeof(float)` bytes 以上ある間、chunk を publish する。
7. publish 後、buffer 先頭から `window.hop_samples * channels * sizeof(float)` bytes を削除する。
8. partial segment が残る場合は、消費した sample frame 数に応じて segment 先頭 `header.stamp` を進める。

## sample 検証

`FLOAT32LE` の 4 bytes を little-endian の `uint32_t` として読み、同じ bit pattern を `float` として解釈する。各 sample は以下を満たす必要がある。

- `std::isfinite(sample)`
- `sample >= -1.0F`
- `sample <= 1.0F`

範囲外、NaN、Inf は streaming contract 違反として drop する。

## overlap

`frame_samples=512`、`hop_samples=256` の場合:

- chunk 0: sample frame `[0, 512)`
- chunk 1: sample frame `[256, 768)`
- chunk 2: sample frame `[512, 1024)`

zero padding は行わないため、最後に 512 sample frames 未満しか残らない場合は publish しない。

## fail-closed

invalid frame は処理継続用の fallback 値に変換しない。data 欠損、format mismatch、非正規化 sample は全て drop し、既存 valid buffer を保持する。
