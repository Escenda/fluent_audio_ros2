## Integration / launch contract tests

`test/integration/test_fa_chunk_overlap_streaming_config_contract.py` で以下を確認する。

- default config が明示的な input / output topic、stream identity、QoS、diagnostics contract を持つ
- runtime source が `AudioFrame`、validation、diagnostics を含む

`test/launch/test_fa_chunk_overlap_streaming_launch_contract.py` で以下を確認する。

- launch file は `node_name` と `config_file` だけを公開する
- package / executable が `fa_chunk_overlap` の契約に一致し、config path は launch argument で明示する
- streaming node の launch に `backend.name` を漏らさない

## 未検証の ROS graph integration

`test/cpp/test_fa_chunk_overlap_node_contract.cpp` は C++ node core を起動し、ROS2 pub/sub 経由で以下を確認する。

- 1 source から複数 frame を送信し、期待数の overlapped chunk が publish される
- source change 後に旧 source の partial buffer が混入しない
- invalid frame 送信後も直前の valid partial buffer が保持される
- mismatched stream は publish せず drop する

親 system 登録後に残る未検証項目は、実際の system profile から include した状態で diagnostics counter が publish されること。

## 非目標

この package では device I/O、resample、format conversion、padding、AGC、filter の品質評価は行わない。
