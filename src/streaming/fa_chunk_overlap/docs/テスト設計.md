# fa_chunk_overlap テスト設計

## Unit contract test

`test/unit/test_fa_chunk_overlap_audio_frame_contract.py` で以下を確認する。

- default config が `FLOAT32LE`、32 bit、interleaved、明示 window/hop を持つ
- startup parameter validation が fail-closed である
- runtime validation が `source_id`、`stream_id`、format fields、data alignment、finite normalized sample を確認する
- invalid frame drop path が buffer mutation より前に return する
- source change が buffer reset を行い、source を混ぜない
- publish 後の消費量が window size ではなく hop size である
- output `stream_id` と `epoch` が更新される
- package layout と CMake pytest integration が存在する

## Integration test

今後の integration test では、ROS2 graph 上で以下を確認する。

- 1 source から複数 frame を送信し、期待数の overlapped chunk が publish される
- source change 後に旧 source の partial buffer が混入しない
- invalid frame 送信後も直前の valid partial buffer が保持される
- diagnostics counter が publish される

## 非目標

この package では device I/O、resample、format conversion、padding、AGC、filter の品質評価は行わない。
