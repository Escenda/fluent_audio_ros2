# fa_chunk_overlap 仕様書

## 目的

`fa_chunk_overlap` は、正規化済み `FLOAT32LE` interleaved audio stream を、固定 window と hop に基づく重複 chunk に変換する streaming node である。

device I/O、resample、bit-depth conversion、channel conversion、gain、filter、zero padding はこの package の責務外とする。

## 入力

Topic は `input_topic` parameter で指定する。message 型は `fa_interfaces/msg/AudioFrame`。

必須条件:

- `source_id` が空でない
- `stream_id` が空でなく、`input_stream_id` と一致する
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == "FLOAT32LE"`
- `bit_depth == 32`
- `layout == "interleaved"`
- `data` が空でない
- `data.size()` が `channels * sizeof(float)` の倍数
- 全 sample が finite かつ `[-1.0, 1.0]` の範囲内

条件を満たさない frame は drop する。drop 時に既存 buffer は変更しない。

## 出力

Topic は `output_topic` parameter で指定する。message 型は `fa_interfaces/msg/AudioFrame`。

出力 frame:

- `source_id` は active source を引き継ぐ
- `stream_id` は `output.stream_id` に更新する
- `header.stamp` は chunk 先頭 sample frame の timestamp とする
- format fields は入力 contract と同じ値を保持する
- `data` は `window.frame_samples * channels` sample を含む
- `epoch` は emitted chunk ごとに 1 ずつ増加する

## Window

- `window.frame_samples > 0`
- `window.hop_samples > 0`
- `window.hop_samples <= window.frame_samples`
- overlap は `window.frame_samples - window.hop_samples`

buffer に `frame_samples` sample frames 以上が存在する限り、0 個以上の chunk を publish する。

publish 後は `hop_samples` sample frames だけ buffer 先頭から削除する。`frame_samples - hop_samples` sample frames は次 chunk の overlap として残す。

`input_topic` / `output_topic` は ROS transport identity、`input_stream_id` / `output.stream_id` は audio stream identity として扱う。stream identity が raw / resolved ROS topic identity と同値の場合は起動失敗する。

## Source 切替

valid frame の `source_id` が active source と異なる場合、既存 buffer を破棄し、新 source の buffer として開始する。異なる source の sample を同一 chunk に混ぜない。

invalid frame は source 切替として扱わない。

## Diagnostics

`diagnostics.publish_period_ms`, `diagnostics.qos.depth`, `diagnostics.qos.reliable` は必須 parameter とする。
`diagnostics.qos.depth` は `> 0` でなければならない。

`diagnostics` に以下を含む:

- `input_stream_id`
- `output_stream_id`
- `frame_samples`
- `hop_samples`
- `overlap_samples`
- `window_bytes`
- `hop_bytes`
- `buffered_bytes`
- `buffered_sample_frames`
- `frames_in`
- `input_frames_accepted`
- `input_frames_dropped`
- `chunks_out`
- `sample_frames_out`
- `source_resets`

## 仕様 ID

| SPEC ID | 内容 |
| --- | --- |
| `FA-CHUNK-OVERLAP-SPEC-001` | default config は `FLOAT32LE`、32 bit、interleaved、明示 topic / stream identity / window / hop を持つ。 |
| `FA-CHUNK-OVERLAP-SPEC-002` | node identity は `fa_chunk_overlap` / `fa_chunk_overlap_node` / `FaChunkOverlapNode` に一致する。 |
| `FA-CHUNK-OVERLAP-SPEC-003` | startup parameter validation は fail closed である。 |
| `FA-CHUNK-OVERLAP-SPEC-004` | runtime validation は `source_id`、`stream_id == input_stream_id`、format fields、data alignment、finite normalized sample を確認する。 |
| `FA-CHUNK-OVERLAP-SPEC-005` | invalid frame drop path は buffer mutation より前に return する。 |
| `FA-CHUNK-OVERLAP-SPEC-006` | source change は buffer reset を行い、異なる source を同一 chunk に混ぜない。 |
| `FA-CHUNK-OVERLAP-SPEC-007` | publish 後は window size ではなく hop size だけ buffer を消費する。 |
| `FA-CHUNK-OVERLAP-SPEC-008` | output は `stream_id == output.stream_id` と chunk ごとの `epoch` 更新を行う。 |
| `FA-CHUNK-OVERLAP-SPEC-009` | package layout と CMake pytest integration が存在する。 |
| `FA-CHUNK-OVERLAP-SPEC-010` | C++ ROS graph contract は overlap window、source reset、invalid frame handling を確認する。 |
| `FA-CHUNK-OVERLAP-SPEC-011` | integration contract は data-plane config と runtime source contract を検証する。 |
| `FA-CHUNK-OVERLAP-SPEC-012` | launch contract は node_name / config_file のみを公開し、backend 選択を漏らさない。 |
| `FA-CHUNK-OVERLAP-SPEC-013` | device I/O、resample、format conversion、padding、AGC、filter の品質評価は非目標である。 |
