# fa_chunk_overlap 仕様書

## 目的

`fa_chunk_overlap` は、正規化済み `FLOAT32LE` interleaved audio stream を、固定 window と hop に基づく重複 chunk に変換する processing node である。

device I/O、resample、bit-depth conversion、channel conversion、gain、filter、zero padding はこの package の責務外とする。

## 入力

Topic は `input_topic` parameter で指定する。message 型は `fa_interfaces/msg/AudioFrame`。

必須条件:

- `source_id` が空でない
- `stream_id` が空でなく、`input_topic` と一致する
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == "FLOAT32LE"`
- `bit_depth == 32`
- `layout == "interleaved"`
- `data` が空でない
- `data.size()` が `channels * sizeof(float)` の倍数
- 全 sample が finite かつ `[-1.0, 1.0]` の範囲内

条件を満たさない frame は drop する。drop 時に既存 buffer は変更しない。

## 出力

Topic は `output_topic` parameter で指定する。message 型は `fa_interfaces/msg/AudioFrame`。

出力 frame:

- `source_id` は active source を引き継ぐ
- `stream_id` は `output_topic` に更新する
- `header.stamp` は chunk 先頭 sample frame の timestamp とする
- format fields は入力 contract と同じ値を保持する
- `data` は `window.frame_samples * channels` sample を含む
- `epoch` は emitted chunk ごとに 1 ずつ増加する

## Window

- `window.frame_samples > 0`
- `window.hop_samples > 0`
- `window.hop_samples <= window.frame_samples`
- overlap は `window.frame_samples - window.hop_samples`

buffer に `frame_samples` sample frames 以上が存在する限り、0 個以上の chunk を publish する。

publish 後は `hop_samples` sample frames だけ buffer 先頭から削除する。`frame_samples - hop_samples` sample frames は次 chunk の overlap として残す。

## Source 切替

valid frame の `source_id` が active source と異なる場合、既存 buffer を破棄し、新 source の buffer として開始する。異なる source の sample を同一 chunk に混ぜない。

invalid frame は source 切替として扱わない。

## Diagnostics

`diagnostics` に以下を含む:

- `frame_samples`
- `hop_samples`
- `overlap_samples`
- `window_bytes`
- `hop_bytes`
- `buffered_bytes`
- `buffered_sample_frames`
- `frames_in`
- `input_frames_accepted`
- `input_frames_dropped`
- `chunks_out`
- `sample_frames_out`
- `source_resets`
