# fa_network_in 仕様書

## 目的

`fa_network_in` は、network endpoint から raw PCM packet を受け取り
`AudioFrame` を publish する将来の standalone source adapter の設計枠である。

## 現在の状態

現時点では ROS 2 package ではない。`package.xml`、`CMakeLists.txt`、launch
entrypoint を持たない。

## 非目標

- jitter buffer、packet loss concealment、clock drift correction を隠すこと
- network codec decode を隠すこと
- resample、gain、denoise を行うこと
- `fa_in` の `network_pcm_receiver` backend の互換 alias になること

## 現行契約

- system config は `package: fa_network_in` を参照しない。
- raw PCM UDP source は `fa_in` の `backend.name: network_pcm_receiver` で明示する。
- jitter / PLC / clock drift が必要な場合は `src/streaming` の専用 node を pipeline に置く。

## Fail Closed

standalone package 化するまでは、`fa_network_in` を build / launch 対象として扱ってはならない。
