# fa_network_in 仕様書

## 1. 目的

`fa_network_in` は明示 UDP endpoint から raw PCM packet を受け取り、configured `AudioFrame` contract として publish する source adapter である。
network jitter absorption、PLC、clock drift correction は `src/streaming` の責務とする。

`fa_network_in` は入力 source adapter であり、音声を編集しない。packet payload は expected format として検証した上で `AudioFrame.data` へそのまま渡す。

## 2. Runtime Parameters

| parameter | required | description |
| --- | --- | --- |
| `backend.name` | yes | `network_pcm_receiver` のみ |
| `endpoint.uri` | yes | bind する `udp://<IPv4>:<port>` |
| `transport.identity` | yes | site / transport binding の明示 identity |
| `output_topic` | yes | publish 先 `AudioFrame` topic |
| `audio.source_id` | yes | publish する `AudioFrame.source_id` と `header.frame_id` |
| `audio.stream_id` | yes | publish する `AudioFrame.stream_id` |
| `expected.sample_rate` | yes | packet の sample rate contract |
| `expected.channels` | yes | interleaved channel count |
| `expected.encoding` | yes | `PCM16LE` / `PCM32LE` / `FLOAT32LE` |
| `expected.bit_depth` | yes | encoding に一致する bit depth |
| `expected.layout` | yes | `interleaved` のみ |
| `network.max_packet_bytes` | yes | 受信 datagram の最大 byte 数 |
| `polling.period_ms` | yes | non-blocking receive の polling interval |
| `qos.depth` | yes | publish QoS depth |
| `qos.reliable` | yes | publish QoS reliability |
| `diagnostics.publish_period_ms` | yes | diagnostics publish interval |
| `diagnostics.qos.depth` | yes | diagnostics publish QoS depth |
| `diagnostics.qos.reliable` | yes | diagnostics publish QoS reliability |

## 3. 責務外

- jitter buffer / PLC / clock drift correction
- codec decode
- resample / channel conversion / gain
- endpoint 推測
- missing endpoint fallback
- packet reorder / retry / time alignment
- hidden format negotiation

## 4. fail closed 条件

- endpoint 欠損
- backend 未指定
- unknown backend
- transport identity 欠損
- expected format 欠損
- packet payload が expected format と一致しない
- source identity 欠損
- socket open / bind / receive failure
