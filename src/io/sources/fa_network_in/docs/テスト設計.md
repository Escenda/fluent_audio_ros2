# fa_network_in テスト設計

## Traceability

- `FA-NETWORK-IN-SPEC-001`: system config の IO group は network input capability を `fa_in` + `backend.name=network_pcm_receiver` として解決する。
- `FA-NETWORK-IN-SPEC-002`: `network_pcm_receiver` backend は accepted UDP packet を raw PCM `AudioFrame.data` として publish し、packet boundary を hidden buffer で変形しない。
- `FA-NETWORK-IN-SPEC-003`: jitter / PLC / clock drift は source adapter に隠さず、unsupported endpoint/packet/no-data timeout は起動失敗、runtime fatal、または明示 error result にする。
