# fa_network_in テスト設計

## Unit Test

- `FA-NETWORK-IN-SPEC-001`: README、仕様書、アルゴリズム詳細説明書、テスト設計、backend docs が存在する。
- `FA-NETWORK-IN-SPEC-002`: `package.xml`、`CMakeLists.txt`、launch、default config が存在する。
- `FA-NETWORK-IN-SPEC-003`: 仕様書が jitter / PLC / drift correction、codec decode、format negotiation を責務外にする。
- `FA-NETWORK-IN-NODE-001`: unknown backend は起動失敗する。
- `FA-NETWORK-IN-NODE-002`: missing endpoint は起動失敗する。
- `FA-NETWORK-IN-NODE-003`: missing transport identity は起動失敗する。
- `FA-NETWORK-IN-NODE-004`: DNS / hostname endpoint は起動失敗する。
- `FA-NETWORK-IN-NODE-005`: valid UDP packet は byte 列を変えずに `AudioFrame` として publish する。
- `FA-NETWORK-IN-NODE-006`: packet byte size が expected frame size と一致しない場合は fail closed する。
- `FA-NETWORK-IN-NODE-007`: `audio.stream_id` は `output_topic` と同値にしない。

## Integration Test

`fluent_audio_system` と組み合わせた network source / sink E2E は、system pipeline の検証として別 package で行う。
この package では loopback UDP socket を使った C++ contract test で raw packet boundary を検証する。

## Launch Test

- launch file は `node_name` と `config_file` だけを受け取り、endpoint / backend / expected format を inline argument に出さない。
- default config は raw PCM network source contract を宣言するが、site 固有 `endpoint.uri` と `transport.identity` は空にして起動失敗させる。
- missing endpoint、missing transport identity が launch 経由で fail closed する。
- `audio.stream_id == output_topic` が launch 経由で fail closed する。
