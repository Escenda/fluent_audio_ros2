# fa_network_in アルゴリズム詳細説明書

1. startup config で endpoint、backend、transport identity、`audio.stream_id`、expected format、source identity を検証する。
2. `audio.stream_id` が raw / resolved `output_topic` と同値ではないことを検証する。
3. `network_pcm_receiver` backend で `udp://<IPv4>:<port>` を bind する。
4. timer callback で backend を non-blocking polling する。
5. packet が無い場合は何も publish せず次 tick に戻る。
6. packet が空、または `channels * bit_depth / 8` で割り切れない場合は fail closed する。
7. packet が `network.max_packet_bytes` を超える場合は fail closed する。
8. configured `source_id`、`stream_id`、expected format metadata と packet payload を使って `AudioFrame` を作る。
9. `AudioFrame.data` には受信 byte 列をそのまま入れ、format conversion、resample、gain、decode は行わない。
10. `AudioFrame` を publish し、diagnostics 用 counter を更新する。

transport instability はこの node で補正しない。必要な場合は `src/streaming` の node を pipeline に明示する。
