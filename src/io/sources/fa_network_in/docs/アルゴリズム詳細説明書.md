# fa_network_in アルゴリズム詳細説明書

## 現在の処理境界

実装はまだ持たない。raw PCM UDP の現在の受信処理は `fa_in` の
`network_pcm_receiver` source backend に閉じる。

## 将来 standalone 化する場合

1. endpoint、payload encoding、sample rate、channels、bit depth、layout を起動時に検証する。
2. packet payload を raw PCM chunk として受け取る。
3. payload contract が `AudioFrame` contract と一致しない場合は frame を落として明示的に診断する。
4. jitter buffer、PLC、clock drift correction は `src/streaming` の node に分離する。

## Backend 境界

backend は network receive と packet-to-chunk boundary だけを担当し、ROS 2
topic/message を知らない。
