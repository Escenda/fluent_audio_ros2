# fa_in 仕様書

## 責務

`fa_in` は入力 source adapter です。ALSA の raw hardware capture source を開き、取得した PCM byte stream を `fa_interfaces/msg/AudioFrame` として `audio/frame` に publish します。

ROS2 node は parameter / service / publish / diagnostics の adapter です。ALSA device enumeration / open / read / close は `include/fa_in/backends` と `src/backends` の source backend に置き、backend は ROS2 topic、ROS message、`rclcpp` を知りません。

`fa_in_node` は実行 target と node core を分けます。node core は `SourceBackend` factory を注入できるため、契約テストは実 ALSA device に依存せず、backend 境界から返る PCM bytes / device enumeration / read failure / reopen failure を検証します。production 実行 target は `backend.name=alsa_capture` のときだけ ALSA backend factory を使います。

## 非責務

- gain / normalize / AGC / compressor / limiter
- resample / bit depth convert / channel convert
- RMS / peak / loudness 計測
- VAD / KWS / ASR / turn detection
- ノイズ除去 / AEC / filter
- file decode / network receive

これらは dedicated processing node または別 source package で扱います。

## Input

| 種別 | 名前 | 内容 |
| --- | --- | --- |
| ALSA PCM | `audio.device_selector` | 明示された capture source |

## Output

| Topic | 型 | 内容 |
| --- | --- | --- |
| `audio/frame` | `fa_interfaces/msg/AudioFrame` | 入力 PCM frame |
| `diagnostics` | `diagnostic_msgs/msg/DiagnosticArray` | capture 状態、XRUN、publish count |

## Service

| Service | 型 | 内容 |
| --- | --- | --- |
| `list_devices` | `fa_interfaces/srv/ListDevices` | ALSA capture source 一覧 |
| `switch_device` | `fa_interfaces/srv/SwitchDevice` | 明示 source への切替 |

`list_devices` の `max_input_channels` / `default_sample_rates` は backend が列挙時に得た capability だけを返す。ALSA hint enumeration で安全に取得できない場合は `0` を返し、起動時に要求している `audio.channels` / `audio.sample_rate` を device capability として合成しない。

`switch_device` は `target_identifier` または `target_index` のどちらか一方だけを受け付ける。両方指定、両方未指定、または index 指定が範囲外の場合は `success=false` を返し、identifier lookup などの別 selector へ fallback しない。

## Parameters

必須 parameter は node 内の runtime default で補完しません。launch/config 境界で明示されていない場合は起動失敗します。

| Parameter | 必須 | 内容 |
| --- | --- | --- |
| `backend.name` | yes | 現在は `alsa_capture` のみ。未知 backend は起動失敗 |
| `audio.device_selector.mode` | yes | `index` または `name` |
| `audio.device_selector.identifier` | `name` 時 yes | `hw:` で始まる ALSA raw hardware id または一意に解決できる表示名の完全一致 |
| `audio.device_selector.index` | `index` 時 yes | ALSA device list index |
| `audio.sample_rate` | yes | device に要求する sample rate |
| `audio.channels` | yes | channel count |
| `audio.bit_depth` | yes | `PCM16LE` は `16`、`PCM32LE` / `FLOAT32LE` は `32` |
| `audio.encoding` | yes | `PCM16LE`, `PCM32LE`, `FLOAT32LE`。`audio.bit_depth` と不一致なら起動失敗 |
| `audio.stream_id` | yes | publish する `AudioFrame.stream_id` |
| `audio.layout` | yes | 現在は `interleaved` のみ。ALSA access と一致しない値は起動失敗 |
| `audio.chunk_ms` | yes | publish frame duration |
| `diagnostics.publish_period_ms` | yes | diagnostics publish period |

## Fail Closed

- ALSA device enumeration が空
- 指定 source が見つからない
- device open に失敗
- 指定 format / rate / channel を device が受け付けない
- `audio.encoding` と `audio.bit_depth` の組が未対応
- `audio.sample_rate`, `audio.channels`, `audio.bit_depth`, `audio.chunk_ms`, `diagnostics.publish_period_ms` が 0 以下
- `audio.sample_rate * audio.chunk_ms` が整数 frame 数にならない
- bytes per frame / capture buffer byte count が表現可能範囲を超える
- requested capture chunk が ALSA frame count 型の表現可能範囲を超える
- ALSA period size negotiation が requested capture chunk から変更された
- runtime 中の `snd_pcm_readi` が configured capture chunk と異なる frame 数を返した
- `audio.stream_id` が空
- `audio.layout` が `interleaved` 以外
- runtime 中の `snd_pcm_readi` が XRUN / error / zero-frame read を返した
- runtime 中の明示 device switch で reopen に失敗した
- ALSA plugin PCM (`default`, `plug*`, `plughw*`, `sysdefault`, `pulse`, `pipewire` など) が指定された
- `switch_device` に `target_identifier` と `target_index` が同時に指定された
- `switch_device` に `target_identifier` と `target_index` がどちらも指定されなかった
- `switch_device` の明示 `target_index` が存在しない
- 表示名指定が複数の ALSA source に一致する
- `device_selector.mode` が未対応
- `backend.name` が空
- `backend.name` が未対応

runtime 中に required input source が失敗した場合、`snd_pcm_prepare` での自動復旧や別 source への fallback は行いません。fatal 状態に入り、ROS shutdown によりプロセスを非ゼロ終了させます。

ALSA software resampling は明示的に無効化します。device が指定 sample rate を受け付けない場合、`fa_in` 内で暗黙 resample せず起動失敗します。

`audio.chunk_ms` から算出した capture frame count は暗黙の下限補正や丸めを行いません。requested chunk が sample rate と整合しない場合や ALSA が別 period size へ変更した場合は、後段の timestamp / frame duration 契約が変わるため起動失敗します。

## Test IDs

- `FA-IN-SPEC-001`: 明示 source がない場合に起動失敗する
- `FA-IN-SPEC-002`: `AudioFrame` に source PCM data と format metadata だけを publish する
- `FA-IN-SPEC-003`: RMS / peak / VAD を publish しない
- `FA-IN-SPEC-004`: device switch は存在する source だけ成功する
- `FA-IN-SPEC-005`: runtime read / reopen failure は fail closed し、自動 prepare / 継続をしない
- `FA-IN-SPEC-006`: `ament_cmake_pytest` / `ament_cmake_gtest` により unit contract test が `colcon test` から実行される
- `FA-IN-SPEC-007`: 必須 audio / diagnostics parameter を node 内 runtime default で補完しない
- `FA-IN-SPEC-008`: `AudioFrame` publish path が `source_id` / `stream_id` / `layout` を必ず設定する
- `FA-IN-SPEC-009`: backend implementation file が `rclcpp` / ROS message / `fa_interfaces` に依存しない
- `FA-IN-SPEC-010`: `fa_in_node.hpp` は `snd_pcm_t` / ALSA header / ALSA handle member を保持しない
- `FA-IN-SPEC-011`: numeric parameter は 0 以下、暗黙丸め、byte count overflow を許さず fail closed する
- `FA-IN-SPEC-012`: ALSA period size negotiation が requested capture chunk を変えた場合は fail closed する
- `FA-IN-SPEC-013`: requested capture chunk が ALSA frame count 型を超える場合は ALSA 呼び出し前に fail closed する
- `FA-IN-SPEC-014`: bytes per frame / capture buffer byte count overflow を fail closed する
- `FA-IN-SPEC-015`: runtime read が configured capture chunk と異なる frame 数を返した場合は fail closed する
- `FA-IN-SPEC-016`: ALSA plugin PCM は enumeration だけでなく backend open 境界でも fail closed する
- `FA-IN-SPEC-017`: `switch_device` は exactly-one selector を要求し、ambiguous selector / invalid index で fallback しない
- `FA-IN-SPEC-018`: 表示名指定は一意に解決できる場合だけ許可し、重複表示名では fail closed する
- `FA-IN-SPEC-019`: node core は source backend factory を注入でき、fake backend で publish metadata / raw payload / enumeration failure / read failure / reopen failure を検証できる
- `FA-IN-SPEC-020`: `list_devices` は backend-reported capability だけを返し、configured stream format を device capability として合成しない
