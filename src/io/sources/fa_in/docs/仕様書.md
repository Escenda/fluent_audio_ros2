# fa_in 仕様書

## 責務

`fa_in` は入力 source adapter です。明示された source backend を開き、取得した PCM byte stream を `fa_interfaces/msg/AudioFrame` として `output_topic` に publish します。

ROS2 node は parameter / service / publish / diagnostics の adapter です。ALSA device enumeration / file open / read / close は `include/fa_in/backends` と `src/backends` の source backend に置き、backend は ROS2 topic、ROS message、`rclcpp` を知りません。

`fa_in_node` は実行 target と node core を分けます。node core は `SourceBackend` factory を注入できるため、契約テストは実 ALSA device に依存せず、backend 境界から返る PCM bytes / device enumeration / read failure / reopen failure を検証します。concrete source backend の生成は ROS-free な `include/fa_in/backends/factory.hpp` / `src/backends/factory.cpp` に置き、node は `SourceBackendSettings` と `SourceBackend` だけを扱います。production 実行 target は `backend.name=alsa_capture` のとき ALSA backend factory、`backend.name=pcm_file_reader` のとき raw PCM file backend、`backend.name=network_pcm_receiver` のとき raw PCM UDP receiver backend を使います。

ROS node 名と config root の例は `fa_in` です。`fa_in_node` は executable / core library の名前としてだけ使い、profile や system config から参照する node identity は `fa_in` に揃えます。launch file は `node_name` / `config_file` の default を持たず、呼び出し元が明示的に渡します。

## 非責務

- gain / normalize / AGC / compressor / limiter
- resample / bit depth convert / channel convert
- RMS / peak / loudness 計測
- VAD / KWS / ASR / turn detection
- ノイズ除去 / AEC / filter
- file decode / network jitter absorption / packet loss concealment / clock drift correction

file source は `pcm_file_reader` backend として raw PCM payload だけを扱います。network source は `network_pcm_receiver` backend として raw PCM UDP payload だけを扱います。encoded file/network decode は `src/processing/format/fa_decode`、jitter buffer / PLC / clock drift correction は `src/streaming`、gain / normalize / resample などは dedicated processing node で扱います。

## Input

| 種別 | 名前 | 内容 |
| --- | --- | --- |
| ALSA PCM | `audio.device_selector` | 明示された capture source |
| Raw PCM file | `file.path` | 明示された regular file |
| Raw PCM UDP | `endpoint.uri` | 明示された `udp://IPv4:port` bind endpoint |

## Output

| Topic | 型 | 内容 |
| --- | --- | --- |
| `output_topic` | `fa_interfaces/msg/AudioFrame` | 入力 PCM frame |
| `diagnostics` | `diagnostic_msgs/msg/DiagnosticArray` | capture 状態、XRUN、publish count |

## Service

| Service | 型 | 内容 |
| --- | --- | --- |
| `list_devices` | `fa_interfaces/srv/ListDevices` | ALSA capture source 一覧 |
| `switch_device` | `fa_interfaces/srv/SwitchDevice` | 明示 source への切替 |

`list_devices` の `max_input_channels` / `default_sample_rates` は backend が列挙時に得た capability だけを返す。ALSA hint enumeration で安全に取得できない場合は `0` を返し、起動時に要求している `audio.channels` / `audio.sample_rate` を device capability として合成しない。

`switch_device` は `target_selector_mode` に `id` / `name` / `index` のいずれかを要求する。`id` は raw ALSA hardware id の完全一致だけを扱い、`name` の場合だけ表示名の完全一致を許可する。`index` は `target_index` だけを使う。mode と field の組が不正な場合、ID miss、name miss、ambiguous name、または index 範囲外では `success=false` を返し、別 selector へ fallback しない。成功時は選択 source を必ず reopen し、no-op の選択成功は返さない。

## Parameters

必須 parameter は node 内の runtime default で補完しません。launch/config 境界で明示されていない場合は起動失敗します。

| Parameter | 必須 | 内容 |
| --- | --- | --- |
| `backend.name` | yes | `alsa_capture` / `pcm_file_reader` / `network_pcm_receiver`。未知 backend は起動失敗 |
| `output_topic` | yes | publish 先の `AudioFrame` ROS topic。`audio.stream_id` と同一視しない |
| `audio.device_selector.mode` | `alsa_capture` 時 yes | `id`, `index`, `name` |
| `audio.device_selector.identifier` | `id` / `name` 時 yes | `id` は `hw:` で始まる ALSA raw hardware id の完全一致。`name` は一意に解決できる表示名の完全一致であり、raw `hw:` id は受け付けない |
| `audio.device_selector.index` | `index` 時 yes | ALSA device list index |
| `file.path` | `pcm_file_reader` 時 yes | 既存 regular file。empty / missing は起動失敗 |
| `endpoint.uri` | `network_pcm_receiver` 時 yes | bind する `udp://IPv4:port`。empty / invalid は起動失敗 |
| `transport.identity` | `network_pcm_receiver` 時 yes | network transport の明示 identity。ROS topic とは同一視しない |
| `audio.source_id` | `pcm_file_reader` / `network_pcm_receiver` 時 yes | source backend が publish する `AudioFrame.source_id` |
| `playback.loop` | `pcm_file_reader` 時 yes | EOF 後に先頭へ戻るか。暗黙 default は持たない |
| `network.max_packet_bytes` | `network_pcm_receiver` 時 yes | 受信 UDP packet の最大 byte 数。expected frame byte size で割り切れない場合は起動失敗 |
| `polling.period_ms` | `network_pcm_receiver` 時 yes | non-blocking UDP receive で packet が無い場合の poll interval |
| `audio.sample_rate` | yes | device に要求する sample rate |
| `audio.channels` | yes | channel count |
| `audio.bit_depth` | yes | `PCM16LE` は `16`、`PCM32LE` / `FLOAT32LE` は `32` |
| `audio.encoding` | yes | `PCM16LE`, `PCM32LE`, `FLOAT32LE`。`audio.bit_depth` と不一致なら起動失敗 |
| `audio.stream_id` | yes | publish する `AudioFrame.stream_id` |
| `audio.layout` | yes | 現在は `interleaved` のみ。ALSA access と一致しない値は起動失敗 |
| `audio.chunk_ms` | yes | publish frame duration |
| `audio.qos.depth` | yes | `AudioFrame` publisher QoS depth |
| `audio.qos.reliable` | yes | `AudioFrame` publisher reliability |
| `diagnostics.qos.depth` | yes | diagnostics publisher QoS depth |
| `diagnostics.qos.reliable` | yes | diagnostics publisher reliability |
| `diagnostics.publish_period_ms` | yes | diagnostics publish period |

## Fail Closed

- ALSA device enumeration が空
- `pcm_file_reader` の `file.path` が空、存在しない、regular file ではない、または空 file
- `pcm_file_reader` の `audio.source_id` が空
- file payload byte size が expected frame byte size で割り切れない
- `network_pcm_receiver` の `endpoint.uri` / `transport.identity` / `audio.source_id` が空
- `network.max_packet_bytes` が expected frame byte size で割り切れない
- 受信 UDP packet が空、`network.max_packet_bytes` 超過、または expected frame byte size で割り切れない
- 指定 source が見つからない
- device open に失敗
- 指定 format / rate / channel を device が受け付けない
- `audio.encoding` と `audio.bit_depth` の組が未対応
- `audio.sample_rate`, `audio.channels`, `audio.bit_depth`, `audio.chunk_ms`, `diagnostics.publish_period_ms` が 0 以下
- `audio.sample_rate * audio.chunk_ms` が整数 frame 数にならない
- bytes per frame / capture buffer byte count が表現可能範囲を超える
- requested capture chunk が ALSA frame count 型の表現可能範囲を超える
- ALSA period size negotiation が requested capture chunk から変更された
- runtime 中の `snd_pcm_readi` が configured capture chunk と異なる frame 数を返した
- `audio.stream_id` が空
- `audio.layout` が `interleaved` 以外
- runtime 中の `snd_pcm_readi` が XRUN / error / zero-frame read を返した
- runtime 中の明示 device switch で reopen に失敗した
- ALSA plugin PCM (`default`, `plug*`, `plughw*`, `sysdefault`, `pulse`, `pipewire` など) が指定された
- `switch_device` の `target_selector_mode` が未対応
- `switch_device` の `target_selector_mode` と `target_identifier` / `target_index` の組が不正
- `switch_device` の明示 `target_index` が存在しない
- `switch_device` の明示 `target_identifier` id が存在しない
- 表示名指定が複数の ALSA source に一致する
- `id` 指定が `hw:` で始まる raw ALSA hardware id ではない
- `name` 指定に raw `hw:` id が渡された
- `switch_device` で選択 source の reopen に失敗した
- `device_selector.mode` が未対応
- `backend.name` が空
- `backend.name` が未対応
- `output_topic` が空
- `audio.stream_id` が raw / resolved `output_topic` と一致する
- `audio.qos.depth` が 0 以下
- `diagnostics.qos.depth` が 0 以下

runtime 中に required live input source が失敗した場合、`snd_pcm_prepare` での自動復旧や別 source への fallback は行いません。fatal 状態に入り、ROS shutdown によりプロセスを非ゼロ終了させます。`pcm_file_reader` で `playback.loop=false` の EOF は source 完了として capture thread を停止し、別 source への fallback は行いません。`network_pcm_receiver` で packet が未到着の状態は `polling.period_ms` だけ待って再 poll しますが、これは別 source への fallback ではなく同一 endpoint の non-blocking receive 待機です。

ALSA software resampling は明示的に無効化します。device が指定 sample rate を受け付けない場合、`fa_in` 内で暗黙 resample せず起動失敗します。

`audio.chunk_ms` から算出した capture frame count は暗黙の下限補正や丸めを行いません。requested chunk が sample rate と整合しない場合や ALSA が別 period size へ変更した場合は、後段の timestamp / frame duration 契約が変わるため起動失敗します。

- `FA-IN-SPEC-001`: 明示 source がない場合に起動失敗する
- `FA-IN-SPEC-002`: `AudioFrame` に source PCM data と format metadata だけを publish する
- `FA-IN-SPEC-003`: RMS / peak / VAD を publish しない
- `FA-IN-SPEC-004`: device switch は存在する source だけ成功する
- `FA-IN-SPEC-005`: runtime read / reopen failure は fail closed し、自動 prepare / 継続をしない
- `FA-IN-SPEC-006`: `ament_cmake_pytest` / `ament_cmake_gtest` により unit contract test が `colcon test` から実行される
- `FA-IN-SPEC-007`: 必須 audio / diagnostics parameter を node 内 runtime default で補完しない
- `FA-IN-SPEC-008`: `AudioFrame` publish path が `source_id` / `stream_id` / `layout` を必ず設定する
- `FA-IN-SPEC-009`: backend implementation file が `rclcpp` / ROS message / `fa_interfaces` に依存しない
- `FA-IN-SPEC-010`: `fa_in_node.hpp` は `snd_pcm_t` / ALSA header / ALSA handle member を保持しない
- `FA-IN-SPEC-011`: numeric parameter は 0 以下、暗黙丸め、byte count overflow を許さず fail closed する

- `FA-IN-SPEC-013`: requested capture chunk が ALSA frame count 型を超える場合は ALSA 呼び出し前に fail closed する
- `FA-IN-SPEC-014`: bytes per frame / capture buffer byte count overflow を fail closed する
- `FA-IN-SPEC-015`: runtime read が configured capture chunk と異なる frame 数を返した場合は fail closed する
- `FA-IN-SPEC-016`: ALSA plugin PCM は enumeration だけでなく backend open 境界でも fail closed する
- `FA-IN-SPEC-017`: `switch_device` は explicit selector mode を要求し、id / name / index の別 selector へ fallback しない
- `FA-IN-SPEC-018`: 表示名指定は一意に解決できる場合だけ許可し、重複表示名では fail closed する

- `FA-IN-SPEC-020`: `list_devices` は backend-reported capability だけを返し、configured stream format を device capability として合成しない
- `FA-IN-SPEC-021`: `switch_device` は成功時に必ず選択 source を reopen し、no-op 成功を返さない

- `FA-IN-SPEC-023`: `fa_in` は file source を `pcm_file_reader` backend、network source を `network_pcm_receiver` backend として扱うが、decode / gain / normalize / limiter / resample / volume / format conversion surface を提供しない
- `FA-IN-SPEC-024`: launch は `node_name` / `config_file` を必須引数にし、node 名、diagnostics status 名、config root は `fa_in` に一致する
- `FA-IN-SPEC-025`: `output_topic` と `audio.stream_id` は分離し、同一 identity になる設定は fail closed する
- `FA-IN-SPEC-026`: audio / diagnostics QoS は `audio.qos.*` / `diagnostics.qos.*` で明示し、system default QoS に fallback しない
- `FA-IN-SPEC-027`: `network_pcm_receiver` は raw PCM UDP payload を 1 `AudioFrame` として publish し、jitter buffer / PLC / clock drift correction を隠さない
- `FA-IN-SPEC-028`: concrete source backend の生成は ROS-free な `backends/factory` に置き、node は `SourceBackendSettings` と `SourceBackend` だけを扱う
