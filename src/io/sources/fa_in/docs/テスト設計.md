# fa_in テスト設計

## Unit Test

- `FA-IN-SPEC-001`: `device_selector.mode` が未対応値の場合に起動失敗する。
- `FA-IN-SPEC-002`: `name` mode で ALSA id 完全一致を優先する。
- `FA-IN-SPEC-003`: `AudioFrame` publish path が RMS / peak / VAD を計算しない。
- `FA-IN-SPEC-004`: `default` / `plug*` / `sysdefault` などの ALSA plugin PCM を候補に出さない。
- `FA-IN-SPEC-005`: `captureLoop` の read failure path が `snd_pcm_prepare` / retry 継続を使わず、fatal shutdown を要求する。
- `FA-IN-SPEC-006`: `ament_cmake_pytest` により unit contract test が `colcon test` から実行される。

## Colcon Test

- `colcon test --packages-select fa_in` は `fa_in_pytest` を実行する。
- ROS image 側の ambient pytest plugin に依存しないよう、`PYTEST_DISABLE_PLUGIN_AUTOLOAD=1` をテスト環境へ明示する。

## Integration Test

- ALSA loopback device を使い、指定 format の frame が `audio/frame` に出ることを確認する。
- device が存在しない config で起動失敗することを確認する。
- runtime 中に loopback capture device を切断した場合、node が fail closed し、read retry を継続しないことを確認する。

## Launch Test

- `fa_in.launch.py` が `config_file` を受け取り、missing params file で失敗することを確認する。

## Fixtures

- ALSA loopback setup script
- 16kHz mono PCM16LE sample
- 48kHz mono PCM16LE sample
