# fa_in テスト設計

## Unit Test

- `FA-IN-SPEC-001`: 明示 source がない場合に起動失敗する。
- `FA-IN-SPEC-002`: `AudioFrame` に source PCM data と format metadata だけを publish する。
- `FA-IN-SPEC-003`: `AudioFrame` publish path が RMS / peak / VAD を計算しない。
- `FA-IN-SPEC-004`: device switch は存在する source だけ成功する。
- `FA-IN-SPEC-005`: `captureLoop` の read failure path が `snd_pcm_prepare` / retry 継続を使わず、fatal shutdown を要求する。
- `FA-IN-SPEC-006`: `ament_cmake_pytest` / `ament_cmake_gtest` により unit contract test が `colcon test` から実行される。
- `FA-IN-SPEC-007`: 必須 audio/diagnostics parameter を node 内 runtime default で補完しない。
- `FA-IN-SPEC-008`: `AudioFrame` publish path が `source_id` / `stream_id` / `layout` を必ず設定する。
- `FA-IN-SPEC-009`: backend implementation file が `rclcpp` / ROS message / `fa_interfaces` に依存しない。
- `FA-IN-SPEC-010`: `fa_in_node.hpp` は `snd_pcm_t` / ALSA header / ALSA handle member を保持しない。
- `FA-IN-SPEC-011`: numeric parameter は 0 以下や暗黙丸めを許さず fail closed する。
- `FA-IN-SPEC-012`: ALSA period size negotiation が requested capture chunk を変えた場合は fail closed する。
- `FA-IN-SPEC-013`: requested capture chunk が ALSA frame count 型を超える場合は ALSA 呼び出し前に fail closed する。
- `FA-IN-SPEC-014`: bytes per frame / capture buffer byte count overflow を fail closed する。
- `FA-IN-SPEC-015`: runtime read が configured capture chunk と異なる frame 数を返した場合は fail closed する。
- `FA-IN-SPEC-016`: `default` / `plug*` / `pulse` / `pipewire` などの ALSA plugin PCM を backend open 境界で reject する。
- `FA-IN-SPEC-017`: `switch_device` が explicit selector mode を要求し、id / name / index の別 selector へ fallback しない。
- `FA-IN-SPEC-018`: 表示名指定は一意に解決できる場合だけ許可し、重複表示名では fail closed する。
- `FA-IN-SPEC-019`: `fa_in_node_core` に fake `SourceBackend` を注入し、実 ALSA device なしで publish metadata / raw payload / enumeration failure / read failure / reopen failure を検証する。
- `FA-IN-SPEC-020`: `list_devices` が backend-reported capability を返し、`config_.channels` / `config_.sample_rate` を device capability として合成しない。
- `FA-IN-SPEC-021`: `switch_device` が成功応答を返す場合、選択 source を必ず reopen し、no-op 成功を返さない。
- `FA-IN-SPEC-022`: `audio.device_selector.mode=id` が raw ALSA hardware id の完全一致だけを許可し、表示名解決を混ぜないことを検証する。
- `FA-IN-SPEC-023`: `fa_in` node / config surface は file source を `pcm_file_reader` backend、network source を `network_pcm_receiver` backend として扱うが、decode / gain / normalize / limiter / resample / volume / format conversion を提供しない。
- `FA-IN-SPEC-024`: launch が `node_name` / `config_file` を必須引数にし、node 名、diagnostics status 名、config root が `fa_in` に一致することを検証する。
- `FA-IN-SPEC-025`: `output_topic` と `audio.stream_id` を分離し、同一 identity になる設定を fail closed することを検証する。
- `FA-IN-SPEC-026`: audio / diagnostics QoS が `audio.qos.*` / `diagnostics.qos.*` で明示され、`SensorDataQoS` や system default QoS に fallback しないことを検証する。
- `FA-IN-SPEC-027`: `network_pcm_receiver` が raw PCM UDP payload を 1 `AudioFrame` として publish し、packet 未到着は poll 待機、packet size mismatch は fail closed になることを検証する。

## Colcon Test

- `colcon test --packages-select fa_in` は `fa_in_pytest`、validation gtest、node contract gtest を実行する。
- ROS image 側の ambient pytest plugin に依存しないよう、`PYTEST_DISABLE_PLUGIN_AUTOLOAD=1` をテスト環境へ明示する。

## Behavior Contract Test

- `test_fa_in_node_contract.cpp` は `SourceBackend` factory injection を使い、production ALSA backend を開かずに `fa_in_node_core` を起動する。
- unsupported `backend.name` では backend factory が呼ばれる前に起動失敗することを確認する。
- fake source から返った PCM bytes が hidden resample / gain / normalize / analysis を通らず、`output_topic` 上の `AudioFrame.data` と metadata にそのまま publish されることを確認する。
- `pcm_file_reader` backend で raw PCM file chunks が hidden decode / resample / gain を通らず、`audio.source_id` と configured format metadata で publish されることを確認する。
- `network_pcm_receiver` backend で raw PCM UDP packet が hidden decode / resample / gain / jitter buffer を通らず、`audio.source_id` と configured format metadata で publish されることを確認する。
- `list_devices` の backend enumeration failure は service response の `success=false` と message で明示し、別 device へ fallback しないことを確認する。
- `list_devices` の `max_input_channels` / `default_sample_rates` は fake backend が返した値を使い、node config から合成しないことを確認する。
- `switch_device` は `target_selector_mode=id` の ID miss を表示名探索へ fallback せず、`target_selector_mode=name` の場合だけ表示名選択を許可することを確認する。
- runtime read failure と runtime switch reopen failure は `hasFatalError()` を立て、retry / prepare / 別 source reopen を行わず fail closed することを確認する。
- `network_pcm_receiver` backend で missing `endpoint.uri`、missing `transport.identity`、missing `audio.source_id`、invalid endpoint、`network.max_packet_bytes` frame alignment 違反、invalid `polling.period_ms` が起動時 fail closed することを確認する。
- `network_pcm_receiver` backend で packet 未到着は poll 待機になり、empty / partial / oversized UDP packet は runtime 中に fail closed することを確認する。

## Planned Integration Test

- ALSA loopback device を使い、指定 format の frame が `output_topic` に出ることを確認する。
- device が存在しない config で起動失敗することを確認する。
- runtime 中に loopback capture device を切断した場合、node が fail closed し、read retry を継続しないことを確認する。
- raw PCM file source を `backend.name=pcm_file_reader` で起動し、EOF / loop の挙動と source identity を確認する。
- raw PCM network source を `backend.name=network_pcm_receiver` で起動し、UDP packet boundary と source identity を確認する。

## Launch Test

- `fa_in.launch.py` は `node_name` / `config_file` の default を持たないことを確認する。
- `config/default.yaml` は site 固有 source binding を持たないため、明示 `config_file` 起動が `audio.device_selector.identifier` 欠落で fail closed することを確認する。
- `fa_in.launch.py` が `config_file` を受け取り、missing params file で失敗することを確認する。
- `fa_in.launch.py` が `network_pcm_receiver` config を受け取り、missing `endpoint.uri`、missing `transport.identity`、missing `audio.source_id`、invalid endpoint、`network.max_packet_bytes` frame alignment 違反、invalid `polling.period_ms` を fail closed にすることを確認する。
