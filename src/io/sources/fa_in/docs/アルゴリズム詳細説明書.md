# fa_in アルゴリズム詳細説明書

## Source Selection

1. ALSA の PCM capture source を列挙する。
2. `audio.device_selector.mode` が `name` の場合は ALSA id または表示名の完全一致で source を決める。
3. `index` の場合は列挙結果の index で source を決める。
4. source が決まらない場合は起動を失敗させる。

`auto` source selection は使いません。入力源が変わると後段のデータ意味が変わるためです。

## Device Configuration

1. 指定 ALSA id を `SND_PCM_STREAM_CAPTURE` で open する。
2. access は interleaved PCM に固定する。
3. bit depth から ALSA format を決定する。
4. channel count / sample rate / period size を設定する。
5. 設定に失敗した場合は起動を失敗させる。

## Frame Publishing

1. capture thread が `snd_pcm_readi` で chunk を読む。
2. XRUN / negative error / zero-frame read は required input source の破断として扱い、fatal 状態へ移行する。
3. byte count を計算する。
4. `AudioFrame.header.stamp`, `encoding`, `sample_rate`, `channels`, `bit_depth`, `data` を設定する。
5. `audio/frame` へ publish する。

`fa_in` は波形値の解析を行いません。RMS / peak / VAD は専用 processing node で扱います。

## Runtime Failure Handling

runtime read failure では `snd_pcm_prepare` による自動復旧や別 source への reopen を行いません。`fa_in` は adapter であり、source path の破断を後段に隠すと timestamp と入力意味が壊れるため、fatal log を出して ROS shutdown を要求します。
