# fa_file_in 仕様書

## 1. 目的

`fa_file_in` は明示設定された file target から audio frame contract に一致する PCM payload を読み、`AudioFrame` として publish する source adapter である。
codec decode、resample、gain、format negotiation は行わない。

## 2. 責務外

- encoded file decode
- resample / channel conversion / bit-depth conversion
- gain / normalize
- file path 推測
- missing file fallback

## 3. fail closed 条件

- file path 欠損または存在しない
- backend 未指定
- unknown backend
- expected format 欠損
- file payload が expected format と一致しない
- output metadata 欠損

## 4. Topic

| 種別 | 既定値 | 内容 |
| --- | --- | --- |
| publish | `audio/file_in` | file から読んだ raw PCM payload を載せた `fa_interfaces/msg/AudioFrame` |

## 5. Parameters

| parameter | 必須 | 内容 |
| --- | --- | --- |
| `backend.name` | yes | `pcm_file_reader` のみ許可 |
| `file.path` | yes | 既存の regular file。空文字は fail closed |
| `output_topic` | yes | `AudioFrame` publish topic |
| `audio.source_id` | yes | frame の `source_id` / `header.frame_id` |
| `audio.stream_id` | yes | frame の `stream_id` |
| `audio.frames_per_chunk` | yes | 1 publish あたりの frame 数 |
| `expected.sample_rate` | yes | file payload の sample rate |
| `expected.channels` | yes | file payload の channel 数 |
| `expected.encoding` | yes | `PCM16LE`, `PCM32LE`, `FLOAT32LE` のいずれか |
| `expected.bit_depth` | yes | encoding と一致する bit depth |
| `expected.layout` | yes | `interleaved` のみ |
| `playback.loop` | yes | EOF 後に先頭へ戻るか |
| `playback.publish_period_ms` | yes | chunk publish 周期 |
| `qos.depth` | yes | output QoS depth |
| `qos.reliable` | yes | reliable QoS を使うか |
| `diagnostics.publish_period_ms` | yes | diagnostics publish 周期 |
| `diagnostics.qos.depth` | yes | diagnostics publish QoS depth |
| `diagnostics.qos.reliable` | yes | diagnostics reliable QoS を使うか |
