# fa_file_in 仕様書

## 目的

`fa_file_in` は、raw PCM file を source として `AudioFrame` を publish する
将来の standalone source adapter の設計枠である。

## 現在の状態

現時点では ROS 2 package ではない。`package.xml`、`CMakeLists.txt`、launch
entrypoint を持たない。

## 非目標

- WAV / MP3 / Opus / FLAC decode を隠すこと
- resample、channel convert、bit depth convert、gain を行うこと
- `fa_in` の `pcm_file_reader` backend の互換 alias になること

## 現行契約

- system config は `package: fa_file_in` を参照しない。
- raw PCM file source は `fa_in` の `backend.name: pcm_file_reader` で明示する。
- codec decode / resample / gain は source adapter に隠さない。必要な場合は
  `fa_decode` などの専用 processing node を pipeline に明示的に置く。

## Fail Closed

standalone package 化するまでは、`fa_file_in` を build / launch 対象として扱ってはならない。
