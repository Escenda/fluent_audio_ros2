# fa_file_in テスト設計

## Unit Test

- `FA-FILE-IN-SPEC-001`: README、仕様書、アルゴリズム詳細説明書、テスト設計、backend docs が存在する。
- `FA-FILE-IN-SPEC-002`: `package.xml` と `CMakeLists.txt` を持つ ROS2 package である。
- `FA-FILE-IN-SPEC-003`: 仕様書が decode / resample / gain を責務外にする。
- `FA-FILE-IN-SPEC-004`: unknown backend では起動失敗する。
- `FA-FILE-IN-SPEC-005`: file path 欠損では起動失敗する。
- `FA-FILE-IN-SPEC-006`: file payload byte size が expected frame byte size と一致しない場合は起動失敗する。
- `FA-FILE-IN-SPEC-007`: valid raw PCM fixture を `AudioFrame` として publish し、format metadata を変更しない。

## Integration Test

- `FA-FILE-IN-TC-001 -> FA-FILE-IN-SPEC-007`: gtest で fixture PCM file から 2 chunk を publish し、metadata と payload を検証する。

## Launch Test

- `FA-FILE-IN-TC-002 -> FA-FILE-IN-SPEC-001..003`: pytest で launch/default config/doc contract を確認する。
- `FA-FILE-IN-TC-003 -> FA-FILE-IN-SPEC-004`: unknown backend で launch が fail closed する。
- `FA-FILE-IN-TC-004 -> FA-FILE-IN-SPEC-005`: missing file path で launch が fail closed する。
