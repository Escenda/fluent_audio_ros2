# fa_file_in テスト設計

## Traceability

- `FA-FILE-IN-SPEC-001`: system config の IO group は file input capability を `fa_in` + `backend.name=pcm_file_reader` として解決する。
- `FA-FILE-IN-SPEC-002`: `pcm_file_reader` backend は raw PCM file chunk を `AudioFrame.data` として publish し、file path を stream identity に流用しない。
- `FA-FILE-IN-SPEC-003`: decode / resample / gain は source adapter に隠さず、unsupported file/config は起動失敗または明示 error result にする。

## Test Cases

- `FA-FILE-IN-TC-001` -> `FA-FILE-IN-SPEC-001`: `src/system/fluent_audio_system/test/unit/test_config_schema.py::test_io_group_accepts_source_sink_and_io_utility_packages`, `src/system/fluent_audio_system/test/unit/test_config_schema.py::test_runtime_backend_packages_require_backend_name`
- `FA-FILE-IN-TC-002` -> `FA-FILE-IN-SPEC-002`: `test/cpp/test_source_backend_factory.cpp::BuildsPcmFileReaderBackend`, `test/integration/test_fa_in_node_contract.cpp::FileBackendPublishesRawPcmChunksWithoutFormatMutation`
- `FA-FILE-IN-TC-003` -> `FA-FILE-IN-SPEC-003`: `test/integration/test_fa_in_node_contract.cpp::FileBackendFailsClosedWhenFilePathIsMissing`, `test/launch/test_fa_in_launch_fail_closed.py::test_default_launch_fails_closed_without_explicit_source_binding`
