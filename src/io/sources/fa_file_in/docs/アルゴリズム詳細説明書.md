# fa_file_in アルゴリズム詳細説明書

## 現在の処理境界

実装はまだ持たない。raw PCM file の現在の読み取り処理は `fa_in` の
`pcm_file_reader` source backend に閉じる。

## 将来 standalone 化する場合

1. file path、encoding、sample rate、channels、bit depth、layout を起動時に検証する。
2. raw PCM bytes を chunk size に従って読む。
3. file の byte contract と `AudioFrame` contract が一致しない場合は fail closed する。
4. decode / resample / gain / normalize は行わず、必要な場合は専用 processing node に渡す。

## Backend 境界

backend は file I/O と byte chunking だけを担当し、ROS 2 topic/message を知らない。
