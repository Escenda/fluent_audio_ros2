# fa_file_in アルゴリズム詳細説明書

1. startup config で file path、backend、expected sample rate、channels、encoding、bit depth、layout を検証する。
2. `pcm_file_reader` backend で regular file を binary mode で開く。
3. file size が expected bytes-per-frame で割り切れることを検証する。
4. timer callback で `audio.frames_per_chunk` 分の raw PCM payload を読む。
5. payload size と expected format contract を検証する。
6. `source_id`、`stream_id`、timestamp、epoch を明示して `AudioFrame` を publish する。
7. EOF では `playback.loop=false` の場合は publish timer を停止し、`true` の場合だけ先頭へ seek する。

encoded file の場合は `fa_decode` を前段または別 pipeline stage として明示する。

`pcm_file_reader` backend は ROS2 topic/message を知らない。file open、seek、read、size 取得だけを担当し、node が `AudioFrame` contract と diagnostics を担当する。
