# fa_file_in アルゴリズム詳細説明書

1. startup config で file path、backend、expected sample rate、channels、encoding、bit depth、layout を検証する。
2. backend で file を開き、configured frame size ごとに raw PCM payload を読む。
3. payload size と expected format contract を検証する。
4. `source_id`、`stream_id`、timestamp、epoch を明示して `AudioFrame` を publish する。

encoded file の場合は `fa_decode` を前段または別 pipeline stage として明示する。
