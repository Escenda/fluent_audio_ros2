# fa_stream 仕様書

## 目的

`fa_stream` は `AudioFrame` stream を network / radio style stream へ渡す sink utility である。device playback ではなく、外部監視・配信・遠隔操作向けの出力経路を担う。

この package は `src/io/utilities` に置く。`src/streaming` は jitter buffer、clock drift correction、packet loss concealment、time alignment などのリアルタイム伝送安定化 node の置き場であり、network endpoint へ送出する sink utility の置き場ではない。

## 入力 topic

| topic | type | 内容 |
| --- | --- | --- |
| `input_topic` | `fa_interfaces/msg/AudioFrame` | 配信対象の音声 frame |

## 出力 topic

なし。出力は backend が管理する network stream である。

## parameters

| parameter | 意味 |
| --- | --- |
| `input_topic` | 購読する `AudioFrame` topic |
| `ffmpeg_path` | 実行する ffmpeg binary |
| `output_url` | stream endpoint。必須 |
| `audio_codec` | ffmpeg の audio codec |
| `bitrate` | audio bitrate |
| `container_format` | ffmpeg output format |
| `content_type` | stream content type |
| `loglevel` | ffmpeg loglevel |

## lifecycle

起動時に `input_topic` を検証し、stream backend を初期化して subscription を作る。stream endpoint が未設定の場合は起動失敗にする。

## QoS

streaming は欠落より遅延を優先する場合がある。QoS は system config で用途ごとに決め、node が device や network 状態から暗黙に変更しない。

## error handling

endpoint 未設定、codec 不一致、network 接続失敗、write failure は明示 error とする。別 endpoint や local file への暗黙 fallback はしない。

## backend selection

network streamer backend を使用する。backend は validated audio chunk と endpoint config を受け取る。

ROS2 node は topic / parameter / `AudioFrame` validation のみを担当する。`ffmpeg` process の起動、stdin write、停止処理は ROS2 非依存の `NetworkStreamerBackend` に閉じる。

`audio_codec` / `container_format` は network endpoint protocol packaging のための sink-local 設定である。汎用的な encode/decode や sample rate / channel / bit depth 変換は `fa_stream` の責務ではない。事前に別 codec 表現へ変換したい場合は `src/processing/format` の encode / format node を pipeline に明示的に挟む。

## non-goals

- microphone capture
- resampling
- codec 自動判定
- jitter buffer
- packet loss concealment
