# fa_stream テスト設計

## traceability

| test ID | 対応仕様 | 観点 |
| --- | --- | --- |
| `FA-STREAM-TC-001` -> `FA-STREAM-SPEC-002` | required stream config | default config が必要な stream parameter を明示し、runtime default に依存しないこと。 |
| `FA-STREAM-TC-002` -> `FA-STREAM-SPEC-009` | launch contract | launch file が `node_name` / `config_file` だけを要求し、package-local default config を推測しないこと。 |
| `FA-STREAM-TC-003` -> `FA-STREAM-SPEC-002` | fail closed startup | 必須 parameter の欠落や型違いを起動時 error にすること。 |
| `FA-STREAM-TC-004` -> `FA-STREAM-SPEC-005` | input frame contract | PCM16LE / 16 bit / interleaved 以外の frame を送信成功扱いしないこと。 |
| `FA-STREAM-TC-005` -> `FA-STREAM-SPEC-007` | no hidden format conversion | codec 設定を sink-local packaging として扱い、resample / normalize / gain を行わないこと。 |
| `FA-STREAM-TC-006` -> `FA-STREAM-SPEC-006` | backend boundary | `NetworkStreamerBackend` が ROS2 topic / message を知らず、node が ffmpeg process 管理を持たないこと。 |
| `FA-STREAM-TC-007` -> `FA-STREAM-SPEC-004` | no endpoint fallback | endpoint 未設定、network failure、write timeout を別 endpoint や local file に逃がさないこと。 |
| `FA-STREAM-TC-008` -> `FA-STREAM-SPEC-008` | streaming correction non-goals | jitter buffer や packet loss concealment を `fa_stream` に入れず、`src/streaming` node に分離すること。 |
| `FA-STREAM-TC-009` -> `FA-STREAM-SPEC-010` | colcon coverage | package の pytest contract が `colcon test` から実行されること。 |

## unit test

- invalid `input_topic` を拒否する。
- endpoint config が欠落した場合に起動失敗する。
- unsupported format を送信成功扱いにしない。

## integration test

- local test endpoint へ frame を送信できる。
- endpoint 切断時に error state へ遷移する。

## launch test

- launch file は `node_name` / `config_file` を明示 argument とし、package-local default config を推測しない。
- endpoint 必須化後は missing endpoint が fail closed になる。

## failure test

- network connect failure。
- write timeout。
- invalid frame size。

## latency test

frame 受信から backend write までの遅延と queue depth を測る。

## real device smoke test

`fa_in -> fa_stream` を実 microphone で起動し、外部 receiver で sample rate / channel / duration が一致することを確認する。
