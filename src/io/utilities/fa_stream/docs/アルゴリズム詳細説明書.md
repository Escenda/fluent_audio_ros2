# fa_stream アルゴリズム詳細説明書

## 処理手順

```text
AudioFrame
  ↓
validate stream format
  ↓
encode or packetize if backend requires it
  ↓
write to stream endpoint
```

## frame / buffer 前提

- network streamer backend の expected sample format と `AudioFrame` metadata が一致すること。
- jitter buffer や clock drift correction は `streaming` package の責務であり、この sink に混ぜない。

## known limitations

- 現在の default config は `input_topic` のみを持つ最小形である。endpoint / codec / transport は backend contract と合わせて拡張する。
