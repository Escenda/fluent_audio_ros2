# fa_record アルゴリズム詳細説明書

## 処理手順

```text
AudioFrame
  ↓
validate sample metadata
  ↓
file_writer backend open / header write
  ↓
append PCM chunk to active recording
  ↓
flush / close on stop
```

## file writer 境界

ROS node は topic、service、frame validation を担当する。file writer backend は ROS2 topic / message を知らず、`AudioFormat`、output path、PCM byte chunk だけを受ける。backend は output file open、WAV header 生成、payload append、header finalize、close を担当する。

file writer は `AudioFrame` の `encoding`、`sample_rate`、`channels`、`bit_depth` を受け、出力 file header と PCM payload を作る。format が録音開始後に変わった場合は同一 file に混ぜず error とする。unsupported encoding は backend 起動前の validation または backend の `startFormat()` で fail closed する。

## frame / buffer 前提

- `data.size()` は frame size の倍数であること。
- WAV backend は `PCM16LE` / 16 bit または `FLOAT32LE` / 32 bit のみを受ける。
- recording session 中は sample rate / channels / bit depth / encoding が一定であること。
- output path の parent directory は事前に存在していること。backend は directory を自動作成しない。

## known limitations

- 現在の default config は `input_topic` のみを持つ最小形である。output path / format policy は recorder API の設計と合わせて拡張する。
