# fa_record テスト設計

## traceability

| test ID | 対応仕様 | 観点 |
| --- | --- | --- |
| `FA-RECORD-TC-001` -> `FA-RECORD-SPEC-002` | required input config | `input_topic` と input QoS が runtime default ではなく明示 config から読まれること。 |
| `FA-RECORD-TC-002` -> `FA-RECORD-SPEC-009` | launch contract | launch file が `node_name` / `config_file` だけを要求し、package-local default config を推測しないこと。 |
| `FA-RECORD-TC-003` -> `FA-RECORD-SPEC-002` | fail closed startup | 空 topic、型違い、`input.qos.depth <= 0` が起動時 error になること。 |
| `FA-RECORD-TC-004` -> `FA-RECORD-SPEC-003` | active recording format | 録音中に format が変わった frame を同一 file に混ぜないこと。 |
| `FA-RECORD-TC-005` -> `FA-RECORD-SPEC-004` | explicit QoS | `SensorDataQoS` や fixed reliability を使わず、明示 QoS だけを使うこと。 |
| `FA-RECORD-TC-006` -> `FA-RECORD-SPEC-006` | backend boundary | file writer backend が ROS2 topic / message を import しないこと。 |
| `FA-RECORD-TC-007` -> `FA-RECORD-SPEC-007` | WAV format contract | WAV backend が `PCM16LE` / 16 bit と `FLOAT32LE` / 32 bit 以外を拒否すること。 |
| `FA-RECORD-TC-008` -> `FA-RECORD-SPEC-005` | no hidden file fallback | parent directory 欠落、file open / write failure、invalid frame size を成功扱いしないこと。 |
| `FA-RECORD-TC-009` -> `FA-RECORD-SPEC-008` | no audio processing | resampling、gain、normalize を録音 node に入れないこと。 |
| `FA-RECORD-TC-010` -> `FA-RECORD-SPEC-010` | colcon coverage | package の pytest contract が `colcon test` から実行されること。 |

## unit test

- invalid `input_topic` を拒否する。
- `input.qos.depth` / `input.qos.reliable` を必須 parameter とし、`SensorDataQoS` や fixed reliability を使わない。
- `AudioFrame` metadata から file header 情報を作れる。
- recording 中の format change を error にする。
- `file_writer` backend code が ROS2 topic / message を import しない。
- WAV backend が `PCM16LE` / 16 bit と `FLOAT32LE` / 32 bit 以外を拒否する。
- parent directory が存在しない output path を失敗にする。

## integration test

- topic から PCM16LE frame を受け、file に保存できる。
- topic から FLOAT32LE frame を受け、IEEE float WAV として保存できる。
- stop 後に file が close / flush される。

## launch test

- launch file は `node_name` / `config_file` を明示 argument とし、package-local default config を推測しない。
- missing params file は system 側で fail closed される。

## failure test

- file open failure。
- disk write failure。
- invalid frame size。

## latency test

録音自体は低 latency 要求ではないが、subscription callback が audio graph を詰まらせないことを測る。

## real device smoke test

`fa_in -> fa_record` を実 microphone で起動し、録音 file の sample rate / channels / duration が topic と一致することを確認する。
