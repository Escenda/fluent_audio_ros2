# fa_record テスト設計

## unit test

- invalid `input_topic` を拒否する。
- `input.qos.depth` / `input.qos.reliable` を必須 parameter とし、`SensorDataQoS` や fixed reliability を使わない。
- `AudioFrame` metadata から file header 情報を作れる。
- recording 中の format change を error にする。
- `file_writer` backend code が ROS2 topic / message を import しない。
- WAV backend が `PCM16LE` / 16 bit と `FLOAT32LE` / 32 bit 以外を拒否する。
- parent directory が存在しない output path を失敗にする。

## integration test

- topic から PCM16LE frame を受け、file に保存できる。
- topic から FLOAT32LE frame を受け、IEEE float WAV として保存できる。
- stop 後に file が close / flush される。

## launch test

- launch file は `node_name` / `config_file` を明示 argument とし、package-local default config を推測しない。
- missing params file は system 側で fail closed される。

## failure test

- file open failure。
- disk write failure。
- invalid frame size。

## latency test

録音自体は低 latency 要求ではないが、subscription callback が audio graph を詰まらせないことを測る。

## real device smoke test

`fa_in -> fa_record` を実 microphone で起動し、録音 file の sample rate / channels / duration が topic と一致することを確認する。
