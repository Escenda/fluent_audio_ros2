# fa_record 仕様書

## 目的

`fa_record` は `AudioFrame` stream を file recording へ渡す sink utility である。`fa_out` の speaker sink とは別に、検証用・dataset 作成用の録音経路を提供する。

## 仕様 ID

| ID | 内容 |
| --- | --- |
| `FA-RECORD-SPEC-001` | `fa_record` は `AudioFrame` stream を file recording へ渡す sink utility であり、microphone capture や speaker playback を行わない。 |
| `FA-RECORD-SPEC-002` | `input_topic`、`input.qos.depth`、`input.qos.reliable` は起動時に明示設定され、空 topic や `input.qos.depth <= 0` は起動失敗にする。 |
| `FA-RECORD-SPEC-003` | 録音 file の header は valid `AudioFrame` format から作り、録音中の format change は同一 file に混ぜず drop / warning とする。 |
| `FA-RECORD-SPEC-004` | QoS は system config で選択し、`SensorDataQoS` や fixed reliability へ暗黙 fallback しない。 |
| `FA-RECORD-SPEC-005` | unsupported encoding、sample format 不一致、file open / write failure、invalid frame size は明示 error とし、別 path 書き込み、parent directory 自動作成、format 推測はしない。 |
| `FA-RECORD-SPEC-006` | file writer backend は ROS2 topic / message を知らず、validated audio chunk と output path のみを受け取る。 |
| `FA-RECORD-SPEC-007` | WAV backend は `PCM16LE` / 16 bit と `FLOAT32LE` / 32 bit の interleaved PCM chunk のみを受け入れる。 |
| `FA-RECORD-SPEC-008` | `fa_record` は resampling、gain / normalize、dataset manifest 管理を行わない。 |
| `FA-RECORD-SPEC-009` | launch file は `node_name` / `config_file` を明示引数とし、package-local default config を推測しない。 |
| `FA-RECORD-SPEC-010` | pytest contract は `colcon test` から実行される。 |

## 入力 topic

| topic | type | 内容 |
| --- | --- | --- |
| `input_topic` | `fa_interfaces/msg/AudioFrame` | 録音対象の音声 frame |

## 出力 topic

なし。録音結果は file または service response として扱う。

## parameters

| parameter | 意味 |
| --- | --- |
| `input_topic` | 購読する `AudioFrame` topic |
| `input.qos.depth` | `AudioFrame` subscription QoS depth。0 以下は起動失敗 |
| `input.qos.reliable` | `AudioFrame` subscription reliability |

## lifecycle

起動時に `input_topic` を検証し、subscription を作る。録音開始 / 停止は `Record` service または node 内 state で扱う。

録音開始時点で直近の valid frame format がある場合は、その format で WAV header を先に作る。まだ valid frame を受けていない場合は、最初の valid payload 到着時に header を作る。録音中に format が変わった frame は同一 file に混ぜず drop し、warning を出す。

## QoS

録音用途では欠落検出が重要になるため、system config 側で reliable QoS を選べるようにする。node が勝手に他 topic へ切り替えず、`SensorDataQoS` や fixed reliability に fallback しない。

## error handling

空 topic、`input.qos.depth <= 0`、unsupported encoding、sample format 不一致、file open 失敗、write 失敗は明示 error として扱う。別 path への暗黙書き込み、parent directory の自動作成、format 推測はしない。

## backend selection

file writer backend を使用する。backend は ROS2 topic を知らず、validated `AudioFrame` chunk と output path を受け取る境界にする。現在の WAV backend は `PCM16LE` / 16 bit と `FLOAT32LE` / 32 bit の interleaved PCM chunk のみを受け入れる。

## non-goals

- microphone capture
- speaker playback
- resampling
- gain / normalize
- dataset manifest 管理
