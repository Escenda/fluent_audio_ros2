# fa_out テスト設計

## Unit Test

- `FA-OUT-SPEC-001`: `queue.max_frames <= 0` で起動失敗する。
- `FA-OUT-SPEC-002`: `audio.chunk_duration_ms <= 0` または `> 1000` で起動失敗する。
- `FA-OUT-SPEC-003`: `audio.qos.depth <= 0` で起動失敗する。
- `FA-OUT-SPEC-004`: encoding / sample rate / channels / bit depth mismatch frame を reject する。
- `FA-OUT-SPEC-005`: `default` / `plug*` / `sysdefault` などの ALSA plugin PCM を device id として拒否する。
- `FA-OUT-SPEC-006`: playback thread の write failure path が自動 reopen / prepare retry を使わず、fatal shutdown を要求する。
- `FA-OUT-SPEC-007`: `ament_cmake_pytest` により unit contract test が `colcon test` から実行される。

## Colcon Test

- `colcon test --packages-select fa_out` は `fa_out_pytest` を実行する。
- ROS image 側の ambient pytest plugin に依存しないよう、`PYTEST_DISABLE_PLUGIN_AUTOLOAD=1` をテスト環境へ明示する。

## Integration Test

- ALSA loopback device へ PCM16LE frame を write できることを確認する。
- stop topic 後に queue が空になり、古い epoch frame を再生しないことを確認する。
- runtime 中に loopback playback device を切断した場合、node が fail closed し、frame drop 継続や reopen retry をしないことを確認する。

## Launch Test

- `fa_out.launch.py` が `config_file` を受け取り、missing params file で失敗することを確認する。

## Fixtures

- 48kHz mono PCM16LE frame
- format mismatch frame
- stale epoch frame
