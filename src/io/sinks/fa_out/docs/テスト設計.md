# fa_out テスト設計

## Unit Test

- `FA-OUT-SPEC-001`: `queue.max_frames <= 0` で起動失敗する。
- `FA-OUT-SPEC-002`: `audio.chunk_duration_ms <= 0` または `> 1000` で起動失敗する。
- `FA-OUT-SPEC-003`: `audio.qos.depth <= 0` で起動失敗する。
- `FA-OUT-SPEC-004`: encoding / sample rate / channels / bit depth mismatch frame を reject する。
- `FA-OUT-SPEC-005`: `default` / `plug*` / `sysdefault` などの ALSA plugin PCM を device id として拒否する。
- `FA-OUT-SPEC-006`: playback thread の write failure path が自動 reopen / prepare retry を使わず、fatal shutdown を要求する。
- `FA-OUT-SPEC-007`: `ament_cmake_pytest` / `ament_cmake_gtest` により unit contract test が `colcon test` から実行される。
- `FA-OUT-SPEC-008`: sink encoding は `audio.encoding` config で明示し、frame validation は config と照合する。
- `FA-OUT-SPEC-009`: ALSA backend 実装が ROS2 header / message header に依存しない。
- `FA-OUT-SPEC-010`: `fa_out` node header が `snd_pcm_t` を保持しない。
- `FA-OUT-SPEC-011`: playback queue overflow で oldest frame drop 継続せず fail closed する。
- `FA-OUT-SPEC-012`: playback thread と ROS callback thread の backend access が mutex で直列化される。
- `FA-OUT-SPEC-013`: signed ROS parameter を unsigned config に変換する前に `> 0` と範囲を検証する。
- `FA-OUT-SPEC-014`: playback chunk frame count が整数にならない設定を起動失敗にする。
- `FA-OUT-SPEC-015`: ALSA buffer / period frames が要求値から negotiation で変化した場合に起動失敗する。
- `FA-OUT-SPEC-016`: ALSA software parameter 設定失敗を warning 継続せず起動失敗にする。
- `FA-OUT-SPEC-017`: bytes per frame / playback chunk byte count overflow を fail closed する。
- `FA-OUT-SPEC-018`: ALSA buffer / period frames が ALSA frame count 型を超える場合は device open 前に fail closed する。
- `FA-OUT-SPEC-019`: runtime 中に required sink backend が欠落した場合は「停止中」扱いにせず fail closed する。
- `FA-OUT-SPEC-020`: decode / gain / normalize / limiter / resample / volume parameter や service surface を提供しない。
- `FA-OUT-SPEC-021`: backend open result は info-only であり、warning 継続チャネルを持たない。
- `FA-OUT-SPEC-022`: `AudioFrame.stream_id` が `input_stream_id` と一致しない frame を reject する。
- `FA-OUT-SPEC-023`: stop / pause / resume service と playback_done topic を持ち、stop 後は旧 epoch frame を stale として reject する。
- `FA-OUT-SPEC-024`: `input_topic` と `input_stream_id` が同一、または先頭 `/` の有無だけが違う同一 identity の場合に起動失敗する。
- `FA-OUT-SPEC-025`: playback_done は元 `AudioFrame.header`、`header.frame_id` 由来の request_id、`AudioFrame.epoch` を転送する。
- `FA-OUT-SPEC-026`: `pcm_file_writer` backend は raw PCM payload を file へ書き、codec / resample / gain を隠さない。
- `FA-OUT-SPEC-027`: `network_pcm_sender` backend は accepted `AudioFrame` 1 件を 1 UDP packet として送り、jitter buffer / PLC / packet loss handling を隠さない。

## Colcon Test

- `colcon test --packages-select fa_out` は `fa_out_pytest`、validation gtest、node contract gtest を実行する。
- ROS image 側の ambient pytest plugin に依存しないよう、`PYTEST_DISABLE_PLUGIN_AUTOLOAD=1` をテスト環境へ明示する。

## Behavior Test

- fake sink backend を注入し、unsupported `backend.name` と ALSA plugin sink が backend open 前に拒否されることを確認する。
- fake sink backend を注入し、configured sink contract と一致しない `AudioFrame` や別 stream の `AudioFrame` が write されないことを確認する。
- fake sink backend を注入し、`input_topic` と `input_stream_id` の衝突が backend open 前に拒否されることを確認する。
- fake sink backend と playback_done subscriber を使い、1 frame write 完了後に `PlaybackDone` が publish されることを確認する。
- `pcm_file_writer` backend を使い、target file へ raw payload が同一 byte sequence で書かれることを確認する。
- `pcm_file_writer` backend で missing `file.path` と `overwrite.enabled=false` の既存 target が fail closed することを確認する。
- `network_pcm_sender` backend を使い、accepted `AudioFrame.data` が 1 UDP packet として同一 byte sequence で送信されることを確認する。
- ROS-free `network_pcm_sender` backend test で invalid endpoint、invalid `network.max_packet_bytes`、packet size limit、UDP byte preservation を確認する。
- `network_pcm_sender` backend で missing `endpoint.uri`、missing `transport.identity`、invalid endpoint、`network.max_packet_bytes` frame alignment 違反が起動時 fail closed することを確認する。
- `network_pcm_sender` backend で accepted `AudioFrame.data` が `network.max_packet_bytes` を超える場合、runtime 中に fail closed することを確認する。
- playback control service の static contract と command handling source を確認し、stop が active epoch を進めて queue を clear し、pause / resume が chunk write を止める state を持つことを確認する。
- fake sink backend の write failure を発生させ、runtime 中の required sink failure が fatal 状態へ移行することを確認する。

## Planned Integration Test

- ALSA loopback device へ PCM16LE frame を write できることを確認する。
- ALSA loopback device で要求した buffer / period frames が維持されることを確認する。
- producer が sink より速く `queue.max_frames` を超えた場合、node が fail closed し、古い frame を落として継続しないことを確認する。
- runtime 中に loopback playback device を切断した場合、node が fail closed し、frame drop 継続や reopen retry をしないことを確認する。
- network sink を `backend.name=network_pcm_sender` で起動し、1 accepted `AudioFrame` が 1 UDP packet として届くことを確認する。

## Launch Test

- `fa_out.launch.py` は `node_name` と `config_file` の暗黙 default を持たず、launch 呼び出し側が明示することを確認する。
- `fa_out.launch.py` に `config/default.yaml` を明示した場合、site 固有 sink binding を持たないため、`audio.device_id` 欠落で fail closed することを確認する。
- `fa_out.launch.py` が `config_file` を受け取り、missing params file で失敗することを確認する。
- `fa_out.launch.py` が `pcm_file_writer` config を受け取り、missing `file.path` と既存 target を fail closed にすることを確認する。
- `fa_out.launch.py` が `network_pcm_sender` config を受け取り、missing `endpoint.uri`、missing `transport.identity`、invalid endpoint、`network.max_packet_bytes` frame alignment 違反を fail closed にすることを確認する。
