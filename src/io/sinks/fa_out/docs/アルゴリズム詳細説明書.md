# fa_out アルゴリズム詳細説明書

## Device Configuration

1. ROS2 node が parameter を検証し、`alsa_playback` backend config を作る。
2. ROS-free backend が `audio.device_id` で ALSA playback device を open する。
3. backend が bit depth から ALSA format を決定する。
4. backend が ALSA software resampling を無効にする。
5. backend が interleaved access, format, channels, sample rate を設定する。
6. 設定できない場合は起動を失敗させる。

## Frame Validation

1. encoding が `PCM16LE` と一致することを確認する。
2. sample rate / channels / bit depth が node config と一致することを確認する。
3. data が空ではないことを確認する。
4. data byte count が frame size で割り切れることを確認する。

一致しない frame は playback queue に入れません。

## Playback Queue

1. valid frame を queue に入れる。
2. queue が上限を超える場合、realtime sink として最古 frame を破棄する。
3. playback thread が queue から frame を取り出し、chunk 単位で backend の write API に渡す。
4. `snd_pcm_writei` が XRUN / EAGAIN / negative error / zero-frame write を返した場合、required sink の破断として fatal 状態へ移行する。

playback thread は runtime 中に device handle が失われても自動 reopen しません。`fa_out` は sink adapter であり、別 device や ALSA plugin 経由の復旧で出力先の意味を変えないためです。

ALSA の open / write / drop / prepare / close は `include/fa_out/backends` と `src/backends` の backend class に閉じ込めます。backend は ROS2 topic、ROS message、`rclcpp` を include しません。

## Stop / Pause

- stop は epoch を進め、queue と ALSA buffer を破棄する。
- pause は ALSA buffer を破棄し、resume まで write を停止する。
- stop より古い timestamp または epoch の frame は破棄する。
- stop / pause / interruption の buffer discard または prepare が失敗した場合も fatal 状態へ移行する。
