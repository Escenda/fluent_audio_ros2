# fa_out 仕様書

## 責務

`fa_out` は出力 sink adapter です。`audio/output/frame` の PCM frame を購読し、指定 ALSA raw hardware playback device へ書き込みます。

## 非責務

- file playback service
- decode / encode
- gain / normalize / limiter
- resample / bit depth convert / channel convert
- routing / mixing
- TTS

これらは dedicated processing node または別 sink/source package で扱います。

## Input

| Topic | 型 | 内容 |
| --- | --- | --- |
| `audio/output/frame` | `fa_interfaces/msg/AudioFrame` | playback 対象 PCM frame |
| `audio/output/stop` | `std_msgs/msg/Empty` | 現在 queue と device buffer を停止 |
| `audio/output/pause` | `std_msgs/msg/Empty` | playback pause |
| `audio/output/resume` | `std_msgs/msg/Empty` | playback resume |

## Output

| Topic | 型 | 内容 |
| --- | --- | --- |
| `audio/output/playback_done` | `fa_interfaces/msg/PlaybackDone` | frame playback 完了通知 |
| `audio/output/paused` | `std_msgs/msg/Empty` | pause 完了通知 |

## Parameters

| Parameter | 必須 | 内容 |
| --- | --- | --- |
| `backend.name` | yes | 現在は `alsa_playback` のみ。未知 backend は起動失敗 |
| `audio.device_id` | yes | `hw:` で始まる ALSA raw hardware playback device id |
| `audio.encoding` | yes | `PCM16LE`。現行 `fa_out` は PCM16LE-only sink |
| `audio.sample_rate` | yes | frame と一致する sample rate |
| `audio.channels` | yes | frame と一致する channels |
| `audio.bit_depth` | yes | `16`。現行 `fa_out` は PCM16LE-only sink |
| `audio.chunk_duration_ms` | yes | stop / pause 応答性用の内部 write chunk |
| `audio.alsa.buffer_frames` | yes | ALSA hardware buffer frames。negotiation で値が変わる場合は起動失敗 |
| `audio.alsa.period_frames` | yes | ALSA period frames。`audio.alsa.buffer_frames` 以下で、negotiation で値が変わる場合は起動失敗 |
| `audio.qos.depth` | yes | input topic QoS depth |
| `audio.qos.reliable` | yes | input topic QoS reliability |
| `queue.max_frames` | yes | playback queue frame count |

## Fail Closed

- ALSA device open に失敗
- `audio.device_id` が空
- ALSA plugin PCM (`default`, `plug*`, `plughw*`, `sysdefault`, `pulse`, `pipewire` など) が指定された
- `audio.encoding` が `PCM16LE` ではない
- `audio.bit_depth` が `16` ではない
- numeric parameter が `<= 0`、または表現可能範囲を超える
- `audio.sample_rate * audio.chunk_duration_ms` が整数 frame count にならない
- bytes per frame / playback chunk byte count が表現可能範囲を超える
- ALSA buffer / period frames が ALSA frame count 型の表現可能範囲を超える
- `audio.alsa.period_frames > audio.alsa.buffer_frames`
- sample rate / channel / format 設定に失敗
- ALSA buffer / period negotiation が要求値を変更した
- ALSA software parameter 設定に失敗
- runtime 中に required playback handle が閉じていた
- runtime 中の `snd_pcm_writei` が XRUN / EAGAIN / error / zero-frame write を返した
- playback queue が `queue.max_frames` を超えた
- stop / pause / interruption の device buffer discard / prepare に失敗した
- queue / QoS / chunk parameter が不正
- `backend.name` が空
- `backend.name` が未対応

runtime 中に required sink が失敗した場合、device reopen や `snd_pcm_prepare` による自動復旧は行いません。fatal 状態に入り、ROS shutdown によりプロセスを非ゼロ終了させます。

playback chunk は隠れた下限補正や丸めを行いません。指定 sample rate と `audio.chunk_duration_ms` から整数 frame count を作れない設定は、出力タイミングの意味が変わるため起動失敗にします。

## Frame Rejection

`AudioFrame` の encoding が `audio.encoding` と一致しない場合、または sample rate / channels / bit depth / byte count が設定と一致しない場合、その frame は再生せず破棄します。

## Test IDs

- `FA-OUT-SPEC-001`: 無効な queue / QoS / chunk parameter で起動失敗する
- `FA-OUT-SPEC-002`: `audio.chunk_duration_ms <= 0` または `> 1000` で起動失敗する
- `FA-OUT-SPEC-003`: `audio.qos.depth <= 0` で起動失敗する
- `FA-OUT-SPEC-004`: encoding / sample rate / channels / bit depth mismatch frame を reject する
- `FA-OUT-SPEC-005`: `default` / `plug*` / `sysdefault` などの ALSA plugin PCM を device id として拒否する
- `FA-OUT-SPEC-006`: runtime write / missing-handle failure は fail closed し、自動 reopen / prepare 継続をしない
- `FA-OUT-SPEC-007`: `ament_cmake_pytest` / `ament_cmake_gtest` により unit contract test が `colcon test` から実行される
- `FA-OUT-SPEC-008`: sink encoding は `audio.encoding` config で明示し、frame validation は config と照合する
- `FA-OUT-SPEC-009`: ALSA backend 実装が ROS2 header / message header に依存しない
- `FA-OUT-SPEC-010`: `fa_out` node header が `snd_pcm_t` を保持しない
- `FA-OUT-SPEC-011`: playback queue overflow で oldest frame drop 継続せず fail closed する
- `FA-OUT-SPEC-012`: playback thread と ROS callback thread の backend access が mutex で直列化される
- `FA-OUT-SPEC-013`: signed ROS parameter を unsigned config に変換する前に `> 0` と範囲を検証する
- `FA-OUT-SPEC-014`: playback chunk frame count が整数にならない設定を起動失敗にする
- `FA-OUT-SPEC-015`: ALSA buffer / period frames が要求値から negotiation で変化した場合に起動失敗する
- `FA-OUT-SPEC-016`: ALSA software parameter 設定失敗を warning 継続せず起動失敗にする
- `FA-OUT-SPEC-017`: bytes per frame / playback chunk byte count overflow を fail closed する
- `FA-OUT-SPEC-018`: ALSA buffer / period frames が ALSA frame count 型を超える場合は device open 前に fail closed する
- `FA-OUT-SPEC-019`: runtime 中に required sink backend が欠落した場合は「停止中」扱いにせず fail closed する
- `FA-OUT-SPEC-020`: file playback / decode / gain / normalize / limiter / resample / volume parameter や service surface を提供しない
