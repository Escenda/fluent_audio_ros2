# fa_out 仕様書

## 責務

`fa_out` は出力 sink adapter です。`audio/output/frame` の PCM frame を購読し、指定 ALSA raw hardware playback device へ書き込みます。

## 非責務

- file playback service
- decode / encode
- gain / normalize / limiter
- resample / bit depth convert / channel convert
- routing / mixing
- TTS

これらは dedicated processing node または別 sink/source package で扱います。

## Input

| Topic | 型 | 内容 |
| --- | --- | --- |
| `audio/output/frame` | `fa_interfaces/msg/AudioFrame` | playback 対象 PCM frame |
| `audio/output/stop` | `std_msgs/msg/Empty` | 現在 queue と device buffer を停止 |
| `audio/output/pause` | `std_msgs/msg/Empty` | playback pause |
| `audio/output/resume` | `std_msgs/msg/Empty` | playback resume |

## Output

| Topic | 型 | 内容 |
| --- | --- | --- |
| `audio/output/playback_done` | `fa_interfaces/msg/PlaybackDone` | frame playback 完了通知 |
| `audio/output/paused` | `std_msgs/msg/Empty` | pause 完了通知 |

## Parameters

| Parameter | 必須 | 内容 |
| --- | --- | --- |
| `backend.name` | yes | 現在は `alsa_playback` のみ。未知 backend は起動失敗 |
| `audio.device_id` | yes | `hw:` で始まる ALSA raw hardware playback device id |
| `audio.sample_rate` | yes | frame と一致する sample rate |
| `audio.channels` | yes | frame と一致する channels |
| `audio.bit_depth` | yes | `16`。現行 `fa_out` は PCM16LE-only sink |
| `audio.chunk_duration_ms` | yes | stop / pause 応答性用の内部 write chunk |
| `audio.qos.depth` | yes | input topic QoS depth |
| `audio.qos.reliable` | yes | input topic QoS reliability |
| `queue.max_frames` | yes | playback queue frame count |

## Fail Closed

- ALSA device open に失敗
- `audio.device_id` が空
- ALSA plugin PCM (`default`, `plug*`, `plughw*`, `sysdefault`, `pulse`, `pipewire` など) が指定された
- `audio.bit_depth` が `16` ではない
- sample rate / channel / format 設定に失敗
- runtime 中に required playback handle が閉じていた
- runtime 中の `snd_pcm_writei` が XRUN / EAGAIN / error / zero-frame write を返した
- stop / pause / interruption の device buffer discard / prepare に失敗した
- queue / QoS / chunk parameter が不正
- `backend.name` が空
- `backend.name` が未対応

runtime 中に required sink が失敗した場合、device reopen や `snd_pcm_prepare` による自動復旧は行いません。fatal 状態に入り、ROS shutdown によりプロセスを非ゼロ終了させます。

## Frame Rejection

`AudioFrame` の encoding が `PCM16LE` ではない場合、または sample rate / channels / bit depth / byte count が設定と一致しない場合、その frame は再生せず破棄します。

## Test IDs

- `FA-OUT-SPEC-001`: 無効な queue / QoS / chunk parameter で起動失敗する
- `FA-OUT-SPEC-002`: format mismatch frame を再生しない
- `FA-OUT-SPEC-003`: ファイル再生 service を提供しない
- `FA-OUT-SPEC-004`: stop 後の古い epoch frame を破棄する
- `FA-OUT-SPEC-005`: runtime write / missing-handle failure は fail closed し、自動 reopen / prepare 継続をしない
