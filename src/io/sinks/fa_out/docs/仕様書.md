# fa_out 仕様書

## 責務

`fa_out` は出力 sink adapter です。`input_topic` の PCM frame を購読し、`input_stream_id` と一致する frame だけを、`backend.name` で明示された sink backend へ書き込みます。

## 非責務

- decode / encode
- gain / normalize / limiter
- resample / bit depth convert / channel convert
- routing / mixing
- TTS

これらは dedicated processing node で扱います。raw PCM file への出力は `pcm_file_writer` backend が扱いますが、container 生成や encode は行いません。raw PCM network endpoint への送出は `network_pcm_sender` backend が扱いますが、packet loss concealment / jitter buffer / clock drift correction は `src/streaming` の責務です。

## Input

| Topic | 型 | 内容 |
| --- | --- | --- |
| `input_topic` | `fa_interfaces/msg/AudioFrame` | playback 対象 PCM frame |

## Output

| Topic | 型 | 内容 |
| --- | --- | --- |
| `playback_done_topic` | `fa_interfaces/msg/PlaybackDone` | 1つの accepted `AudioFrame` を sink backend へ最後まで write したことを通知する |

## Service

| Service | 型 | 内容 |
| --- | --- | --- |
| `playback_control_service` | `fa_interfaces/srv/PlaybackControl` | `stop` / `pause` / `resume` による output lifecycle 制御 |

## Parameters

| Parameter | 必須 | 内容 |
| --- | --- | --- |
| `backend.name` | yes | `alsa_playback` / `pcm_file_writer` / `network_pcm_sender`。未知 backend は起動失敗 |
| `input_topic` | yes | playback 対象 frame を購読する ROS topic |
| `input_stream_id` | yes | speaker sink へ流せる `AudioFrame.stream_id`。ROS topic 名とは別 identity として扱い、`input_topic` と同一視できる値は起動失敗 |
| `playback_done_topic` | yes | playback 完了通知を publish する ROS topic。`input_topic` と同一視できる値は起動失敗 |
| `playback_control_service` | yes | stop / pause / resume を受ける ROS service 名 |
| `audio.device_id` | `alsa_playback` | `hw:` で始まる ALSA raw hardware playback device id |
| `file.path` | `pcm_file_writer` | raw PCM output file path |
| `overwrite.enabled` | `pcm_file_writer` | 既存 file を上書きするか |
| `endpoint.uri` | `network_pcm_sender` | 送信先 `udp://IPv4:port` |
| `transport.identity` | `network_pcm_sender` | network transport の明示 identity。ROS topic とは同一視しない |
| `network.max_packet_bytes` | `network_pcm_sender` | 1 UDP packet として送る最大 byte 数。expected frame byte size で割り切れない場合は起動失敗 |
| `audio.encoding` | yes | `alsa_playback` は `PCM16LE`。`pcm_file_writer` / `network_pcm_sender` は `PCM16LE` / `PCM32LE` / `FLOAT32LE` |
| `audio.sample_rate` | yes | frame と一致する sample rate |
| `audio.channels` | yes | frame と一致する channels |
| `audio.bit_depth` | yes | `alsa_playback` は `16`。`pcm_file_writer` / `network_pcm_sender` は encoding と一致する `16` または `32` |
| `audio.chunk_duration_ms` | yes | backend write を分割する内部 chunk duration |
| `audio.alsa.buffer_frames` | `alsa_playback` | ALSA hardware buffer frames。negotiation で値が変わる場合は起動失敗 |
| `audio.alsa.period_frames` | `alsa_playback` | ALSA period frames。`audio.alsa.buffer_frames` 以下で、negotiation で値が変わる場合は起動失敗 |
| `audio.qos.depth` | yes | input topic QoS depth |
| `audio.qos.reliable` | yes | input topic QoS reliability |
| `lifecycle.qos.depth` | yes | playback lifecycle topic QoS depth |
| `lifecycle.qos.reliable` | yes | playback lifecycle topic QoS reliability |
| `queue.max_frames` | yes | playback queue frame count |

## Fail Closed

- ALSA device open に失敗
- `audio.device_id` が空
- `file.path` が空
- `file.path` の親 directory が存在しない
- `file.path` が directory
- `overwrite.enabled=false` で target file が存在する
- `network_pcm_sender` の `endpoint.uri` / `transport.identity` が空
- `network.max_packet_bytes` が expected frame byte size で割り切れない
- `network_pcm_sender` に渡す `AudioFrame.data` が `network.max_packet_bytes` を超える
- ALSA plugin PCM (`default`, `plug*`, `plughw*`, `sysdefault`, `pulse`, `pipewire` など) が指定された
- `audio.encoding` が `PCM16LE` ではない
- `audio.bit_depth` が `16` ではない
- `pcm_file_writer` で encoding / bit depth pair が `PCM16LE/16`、`PCM32LE/32`、`FLOAT32LE/32` 以外
- `network_pcm_sender` で encoding / bit depth pair が `PCM16LE/16`、`PCM32LE/32`、`FLOAT32LE/32` 以外
- numeric parameter が `<= 0`、または表現可能範囲を超える
- `audio.sample_rate * audio.chunk_duration_ms` が整数 frame count にならない
- bytes per frame / playback chunk byte count が表現可能範囲を超える
- ALSA buffer / period frames が ALSA frame count 型の表現可能範囲を超える
- `audio.alsa.period_frames > audio.alsa.buffer_frames`
- sample rate / channel / format 設定に失敗
- ALSA buffer / period negotiation が要求値を変更した
- ALSA software parameter 設定に失敗
- backend open が warning 継続を必要とする状態になった
- runtime 中に required playback handle が閉じていた
- runtime 中の `snd_pcm_writei` が XRUN / EAGAIN / error / zero-frame write を返した
- playback queue が `queue.max_frames` を超えた
- queue / QoS / chunk parameter が不正
- `backend.name` が空
- `backend.name` が未対応
- `input_topic` が空
- `input_stream_id` が空
- `input_stream_id` が `input_topic` と同一、または先頭 `/` の有無だけが違う同一 identity
- `playback_done_topic` が空
- `playback_done_topic` が `input_topic` と同一、または解決後に同一 ROS topic
- `playback_control_service` が空
- stop により進める active epoch が `uint32` の上限を超える

runtime 中に required sink が失敗した場合、device reopen や `snd_pcm_prepare` による自動復旧は行いません。fatal 状態に入り、ROS shutdown によりプロセスを非ゼロ終了させます。

playback chunk は隠れた下限補正や丸めを行いません。指定 sample rate と `audio.chunk_duration_ms` から整数 frame count を作れない設定は、出力タイミングの意味が変わるため起動失敗にします。

## Frame Rejection

`AudioFrame` の `stream_id` が `input_stream_id` と一致しない場合、encoding が `audio.encoding` と一致しない場合、または sample rate / channels / bit depth / byte count が設定と一致しない場合、その frame は再生せず破棄します。topic subscription だけを根拠に別 stream の frame を speaker sink へ流しません。

`AudioFrame.epoch` が現在の active epoch より古い場合、その frame は stale として再生せず破棄します。`stop` は active epoch を1つ進め、queue 内 frame を破棄し、再生 thread は次の chunk boundary で旧 epoch frame の write を止めます。`pause` は新規 chunk write を止め、`resume` は再開します。`fa_out` は stop / pause / resume のために resample、decode、gain、routing を行いません。

`network_pcm_sender` は accepted `AudioFrame` 1 件を 1 UDP packet として送ります。speaker / file backend のような `audio.chunk_duration_ms` による複数 write 分割は行いません。これにより packet boundary を sink adapter の外へ隠さず、jitter buffer / PLC / packet loss handling を `src/streaming` 側で明示できます。

## Test IDs

- `FA-OUT-SPEC-001`: 無効な queue / QoS / chunk parameter で起動失敗する
- `FA-OUT-SPEC-002`: `audio.chunk_duration_ms <= 0` または `> 1000` で起動失敗する
- `FA-OUT-SPEC-003`: `audio.qos.depth <= 0` で起動失敗する
- `FA-OUT-SPEC-004`: encoding / sample rate / channels / bit depth mismatch frame を reject する
- `FA-OUT-SPEC-005`: `default` / `plug*` / `sysdefault` などの ALSA plugin PCM を device id として拒否する
- `FA-OUT-SPEC-006`: runtime write / missing-handle failure は fail closed し、自動 reopen / prepare 継続をしない
- `FA-OUT-SPEC-007`: `ament_cmake_pytest` / `ament_cmake_gtest` により unit contract test が `colcon test` から実行される
- `FA-OUT-SPEC-008`: sink encoding は `audio.encoding` config で明示し、frame validation は config と照合する
- `FA-OUT-SPEC-009`: ALSA backend 実装が ROS2 header / message header に依存しない
- `FA-OUT-SPEC-010`: `fa_out` node header が `snd_pcm_t` を保持しない
- `FA-OUT-SPEC-011`: playback queue overflow で oldest frame drop 継続せず fail closed する
- `FA-OUT-SPEC-012`: playback thread と ROS callback thread の backend access が mutex で直列化される
- `FA-OUT-SPEC-013`: signed ROS parameter を unsigned config に変換する前に `> 0` と範囲を検証する
- `FA-OUT-SPEC-014`: playback chunk frame count が整数にならない設定を起動失敗にする
- `FA-OUT-SPEC-015`: ALSA buffer / period frames が要求値から negotiation で変化した場合に起動失敗する
- `FA-OUT-SPEC-016`: ALSA software parameter 設定失敗を warning 継続せず起動失敗にする
- `FA-OUT-SPEC-017`: bytes per frame / playback chunk byte count overflow を fail closed する
- `FA-OUT-SPEC-018`: ALSA buffer / period frames が ALSA frame count 型を超える場合は device open 前に fail closed する
- `FA-OUT-SPEC-019`: runtime 中に required sink backend が欠落した場合は「停止中」扱いにせず fail closed する
- `FA-OUT-SPEC-020`: decode / gain / normalize / limiter / resample / volume parameter や service surface を提供しない
- `FA-OUT-SPEC-021`: backend open result は info-only であり、warning 継続チャネルを持たない
- `FA-OUT-SPEC-022`: `AudioFrame.stream_id` が `input_stream_id` と一致しない frame を reject する
- `FA-OUT-SPEC-023`: stop / pause / resume service と playback_done topic を持ち、stop 後は旧 epoch frame を stale として reject する
- `FA-OUT-SPEC-024`: `input_topic` と `input_stream_id` は別 identity として明示し、先頭 `/` の有無だけが違う同一値も起動失敗にする
- `FA-OUT-SPEC-025`: playback_done は元 `AudioFrame.header`、`header.frame_id` 由来の request_id、`AudioFrame.epoch` を転送する
- `FA-OUT-SPEC-026`: `pcm_file_writer` backend は raw PCM payload を file へ書き、codec / resample / gain を隠さない
- `FA-OUT-SPEC-027`: `network_pcm_sender` backend は accepted `AudioFrame` 1 件を 1 UDP packet として送り、jitter buffer / PLC / packet loss handling を隠さない
