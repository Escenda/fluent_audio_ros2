# fa_record アルゴリズム詳細説明書

## 処理手順

```text
AudioFrame
  ↓
validate sample metadata
  ↓
append PCM chunk to active recording
  ↓
flush / close on stop
```

## file writer 境界

file writer は `AudioFrame` の `encoding`、`sample_rate`、`channels`、`bit_depth` を受け、出力 file header と PCM payload を作る。format が録音開始後に変わった場合は同一 file に混ぜず error とする。

## frame / buffer 前提

- `data.size()` は frame size の倍数であること。
- recording session 中は sample rate / channels / bit depth / encoding が一定であること。

## known limitations

- 現在の default config は `input_topic` のみを持つ最小形である。output path / format policy は recorder API の設計と合わせて拡張する。
