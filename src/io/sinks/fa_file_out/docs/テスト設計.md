# fa_file_out テスト設計

## Traceability

- `FA-FILE-OUT-SPEC-001`: system config の IO group は file output capability を `fa_out` + `backend.name=pcm_file_writer` として解決する。
- `FA-FILE-OUT-SPEC-002`: `pcm_file_writer` backend は accepted `AudioFrame.data` を raw PCM bytes として file へ書き、payload byte sequence を変形しない。
- `FA-FILE-OUT-SPEC-003`: encode / limiter / normalize / fade は sink adapter に隠さず、unsupported file/config/frame は起動失敗、frame reject、runtime fatal、または明示 error result にする。

## Test Cases

- `FA-FILE-OUT-TC-001` -> `FA-FILE-OUT-SPEC-001`: `src/system/fluent_audio_system/test/unit/test_config_schema.py::test_io_group_accepts_source_sink_and_io_utility_packages`, `src/system/fluent_audio_system/test/unit/test_config_schema.py::test_runtime_backend_packages_require_backend_name`
- `FA-FILE-OUT-TC-002` -> `FA-FILE-OUT-SPEC-002`: `test/cpp/test_sink_backend_factory.cpp::BuildsPcmFileWriterBackend`, `test/integration/test_fa_out_node_contract.cpp::FileBackendWritesRawPcmPayloadWithoutFormatMutation`
