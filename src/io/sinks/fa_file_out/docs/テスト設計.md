# fa_file_out テスト設計

## Unit

- `FA-FILE-OUT-SPEC-001`: 標準資料が存在すること。
- `FA-FILE-OUT-SPEC-002`: `package.xml` と `CMakeLists.txt` を持つ ROS2 package である。
- `FA-FILE-OUT-SPEC-003`: backend contract document が encoding、resample、gain の暗黙処理を禁止していること。
- `FA-FILE-OUT-SPEC-004`: unknown backend では起動失敗する。
- `FA-FILE-OUT-SPEC-005`: file path 欠損では起動失敗する。
- `FA-FILE-OUT-SPEC-006`: overwrite disabled の既存 target は起動失敗する。
- `FA-FILE-OUT-SPEC-007`: expected format と一致する frame だけを書き出す。
- `FA-FILE-OUT-SPEC-008`: expected format 不一致は fail closed する。

## Integration

- `FA-FILE-OUT-TC-001 -> FA-FILE-OUT-SPEC-007`: gtest で incoming `AudioFrame` payload が target file にそのまま write されることを確認する。
- `FA-FILE-OUT-TC-002 -> FA-FILE-OUT-SPEC-008`: gtest で expected format mismatch が fail closed することを確認する。

## Launch

- `FA-FILE-OUT-TC-003 -> FA-FILE-OUT-SPEC-001..003`: pytest で launch/default config/doc contract を確認する。
- `FA-FILE-OUT-TC-004 -> FA-FILE-OUT-SPEC-005`: `file.path` 欠損で起動失敗すること。
- `FA-FILE-OUT-TC-005 -> FA-FILE-OUT-SPEC-006`: overwrite disabled の既存 target で起動失敗すること。
