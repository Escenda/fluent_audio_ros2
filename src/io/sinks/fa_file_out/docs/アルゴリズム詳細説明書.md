# fa_file_out アルゴリズム詳細説明書

## 現在の処理境界

実装はまだ持たない。raw PCM file の現在の書き出し処理は `fa_out` の
`pcm_file_writer` sink backend に閉じる。

## 将来 standalone 化する場合

1. output path、encoding、sample rate、channels、bit depth、layout を起動時に検証する。
2. 受信した `AudioFrame` の contract が file contract と一致することを確認する。
3. 一致しない frame は変換せず fail closed / reject する。
4. encode / limiter / normalize / fade は行わず、必要な場合は専用 processing node に渡す。

## Backend 境界

backend は file I/O と byte writing だけを担当し、ROS 2 topic/message を知らない。
