# fa_file_out アルゴリズム詳細説明書

1. startup config で file path、backend、expected sample rate、channels、encoding、bit depth、layout を検証する。
2. overwrite policy と parent directory の存在を検証する。
3. backend で file を開く。
4. incoming `AudioFrame` の metadata と payload size が expected format contract に一致することを検証する。
5. payload をそのまま file target へ write する。
6. write failure は sink error として扱い、後続 frame を成功扱いしない。

encoded file を出力したい場合は、`fa_encode` を前段の明示 pipeline stage として配置する。
