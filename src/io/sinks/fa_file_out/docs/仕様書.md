# fa_file_out 仕様書

## 1. 目的

`fa_file_out` は incoming `AudioFrame` payload を、明示設定された file target へ書き出す sink adapter である。
codec encode、resample、gain、normalize、format negotiation は行わない。

## 2. 責務外

- encoded file output
- resample / channel conversion / bit-depth conversion
- gain / normalize / limiter
- output path 推測
- missing target fallback

## 3. fail closed 条件

- file path 欠損
- backend 未指定
- unknown backend
- expected input format 欠損
- incoming `AudioFrame` が expected input format と一致しない
- overwrite disabled の target が既に存在する
- write failure

## 4. Topic

| 種別 | 既定値 | 内容 |
| --- | --- | --- |
| subscribe | `audio/file_out` | raw PCM file へ write する `fa_interfaces/msg/AudioFrame` |

## 5. Parameters

| parameter | 必須 | 内容 |
| --- | --- | --- |
| `backend.name` | yes | `pcm_file_writer` のみ許可 |
| `file.path` | yes | write target。空文字は fail closed |
| `input_topic` | yes | `AudioFrame` subscribe topic |
| `expected.sample_rate` | yes | incoming frame の sample rate |
| `expected.channels` | yes | incoming frame の channel 数 |
| `expected.encoding` | yes | `PCM16LE`, `PCM32LE`, `FLOAT32LE` のいずれか |
| `expected.bit_depth` | yes | encoding と一致する bit depth |
| `expected.layout` | yes | `interleaved` のみ |
| `overwrite.enabled` | yes | 既存 target の上書きを許可するか |
| `qos.depth` | yes | input QoS depth |
| `qos.reliable` | yes | reliable QoS を使うか |
| `diagnostics.publish_period_ms` | yes | diagnostics publish 周期 |
