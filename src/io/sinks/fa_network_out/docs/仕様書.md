# fa_network_out 仕様書

## 1. 目的

`fa_network_out` は incoming `AudioFrame` payload を、明示設定された network endpoint へ送信する sink adapter である。
jitter buffer、PLC、clock drift correction、codec encode、resample、gain は行わない。

## 2. 責務外

- jitter buffer / PLC / clock drift correction
- codec encode
- resample / channel conversion / gain
- endpoint 推測
- missing endpoint fallback

## 3. fail closed 条件

- endpoint 欠損
- backend 未指定
- unknown backend
- expected input format 欠損
- incoming `AudioFrame` が expected input format と一致しない
- send failure
- transport identity 欠損

## 4. Topic

| 種別 | 既定値 | 内容 |
| --- | --- | --- |
| subscribe | `audio/network_out` | UDP endpoint へ送る raw PCM `AudioFrame` |

## 5. Parameters

| parameter | 必須 | 内容 |
| --- | --- | --- |
| `backend.name` | yes | `network_pcm_sender` のみ許可 |
| `endpoint.uri` | yes | `udp://<IPv4>:<port>`。DNS や endpoint 推測はしない |
| `transport.identity` | yes | diagnostics 用の明示 transport identity |
| `input_topic` | yes | `AudioFrame` subscribe topic |
| `expected.sample_rate` | yes | incoming frame の sample rate |
| `expected.channels` | yes | incoming frame の channel 数 |
| `expected.encoding` | yes | `PCM16LE`, `PCM32LE`, `FLOAT32LE` のいずれか |
| `expected.bit_depth` | yes | encoding と一致する bit depth |
| `expected.layout` | yes | `interleaved` のみ |
| `qos.depth` | yes | input QoS depth |
| `qos.reliable` | yes | reliable QoS を使うか |
| `diagnostics.publish_period_ms` | yes | diagnostics publish 周期 |
| `diagnostics.qos.depth` | yes | diagnostics publish QoS depth |
| `diagnostics.qos.reliable` | yes | diagnostics reliable QoS を使うか |
