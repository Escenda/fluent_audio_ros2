# fa_network_out テスト設計

## Traceability

- `FA-NETWORK-OUT-SPEC-001`: system config の IO group は network output capability を `fa_out` + `backend.name=network_pcm_sender` として解決する。
- `FA-NETWORK-OUT-SPEC-002`: `network_pcm_sender` backend は accepted `AudioFrame` 1 件を raw PCM UDP packet 1 件として送信し、payload byte sequence を変形しない。
- `FA-NETWORK-OUT-SPEC-003`: jitter / PLC / clock drift は sink adapter に隠さず、unsupported endpoint/packet/frame は起動失敗、frame reject、runtime fatal、または明示 error result にする。

## Test Cases

- `FA-NETWORK-OUT-TC-001` -> `FA-NETWORK-OUT-SPEC-001`: `src/system/fluent_audio_system/test/unit/test_config_schema.py::test_io_group_accepts_source_sink_and_io_utility_packages`, `src/system/fluent_audio_system/test/unit/test_config_schema.py::test_runtime_backend_packages_require_backend_name`
- `FA-NETWORK-OUT-TC-002` -> `FA-NETWORK-OUT-SPEC-002`: `test/cpp/test_sink_backend_factory.cpp::BuildsNetworkPcmSenderBackend`, `test/cpp/test_network_pcm_sender_backend.cpp::SendsOneAcceptedFrameAsOneUdpPacketWithoutByteMutation`
- `FA-NETWORK-OUT-TC-003` -> `FA-NETWORK-OUT-SPEC-003`: `test/cpp/test_network_pcm_sender_backend.cpp::RejectsInvalidPacketSizeConfiguration`, `test/cpp/test_network_pcm_sender_backend.cpp::RejectsInvalidEndpointUriOnOpen`, `test/cpp/test_network_pcm_sender_backend.cpp::RejectsPacketLargerThanConfiguredMaximum`, `test/launch/test_fa_out_launch_fail_closed.py`
