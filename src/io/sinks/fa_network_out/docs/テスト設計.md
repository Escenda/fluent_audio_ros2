# fa_network_out テスト設計

## Unit

- `FA-NETWORK-OUT-SPEC-001`: 標準資料が存在すること。
- `FA-NETWORK-OUT-SPEC-002`: `package.xml` と `CMakeLists.txt` を持つ ROS2 package である。
- `FA-NETWORK-OUT-SPEC-003`: backend contract document が jitter、PLC、codec encode、endpoint guessing の暗黙処理を禁止していること。
- `FA-NETWORK-OUT-SPEC-004`: unknown backend では起動失敗する。
- `FA-NETWORK-OUT-SPEC-005`: endpoint 欠損では起動失敗する。
- `FA-NETWORK-OUT-SPEC-006`: transport identity 欠損では起動失敗する。
- `FA-NETWORK-OUT-SPEC-007`: expected format と一致する frame だけを UDP datagram として送信する。
- `FA-NETWORK-OUT-SPEC-008`: expected format 不一致は fail closed する。

## Integration

- `FA-NETWORK-OUT-TC-001 -> FA-NETWORK-OUT-SPEC-007`: gtest で loopback UDP receiver を使い、incoming `AudioFrame` payload が datagram としてそのまま送信されることを確認する。
- `FA-NETWORK-OUT-TC-002 -> FA-NETWORK-OUT-SPEC-008`: gtest で expected format mismatch が fail closed し、packet を送らないことを確認する。

## Launch

- `FA-NETWORK-OUT-TC-003 -> FA-NETWORK-OUT-SPEC-001..003`: pytest で launch/default config/doc contract を確認する。
- `FA-NETWORK-OUT-TC-004 -> FA-NETWORK-OUT-SPEC-005`: `endpoint.uri` 欠損で起動失敗すること。
- `FA-NETWORK-OUT-TC-005 -> FA-NETWORK-OUT-SPEC-006`: `transport.identity` 欠損で起動失敗すること。
