# fa_network_out アルゴリズム詳細説明書

## 現在の処理境界

実装はまだ持たない。raw PCM UDP の現在の送信処理は `fa_out` の
`network_pcm_sender` sink backend に閉じる。

## 将来 standalone 化する場合

1. endpoint、payload encoding、sample rate、channels、bit depth、layout を起動時に検証する。
2. 受信した `AudioFrame` の contract が packet contract と一致することを確認する。
3. 一致しない frame は変換せず fail closed / reject する。
4. jitter buffer、PLC、clock drift correction は `src/streaming` の node に分離する。

## Backend 境界

backend は packet send と raw payload boundary だけを担当し、ROS 2
topic/message を知らない。
