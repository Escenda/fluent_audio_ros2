# fa_network_out アルゴリズム詳細説明書

1. startup config で endpoint、backend、transport identity、expected sample rate、channels、encoding、layout を検証する。
2. backend で明示 endpoint への送信 session を開く。
3. incoming `AudioFrame` の metadata と payload size が expected format contract に一致することを検証する。
4. payload をそのまま endpoint へ送信する。
5. send failure は sink error として扱い、retry や alternate endpoint へ暗黙に切り替えない。

network jitter absorption、packet loss concealment、clock drift correction が必要な場合は、`src/streaming` の専用 node を pipeline に配置する。
