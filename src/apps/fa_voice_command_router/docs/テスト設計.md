# fa_voice_command_router テスト設計

## unit test

- `allowed_modes` にない初期 mode を拒否する。
- command から期待する state へ遷移する。
- 不明 command を成功扱いにしない。

## integration test

- command topic 入力で state topic が更新される。
- `announce_tts=true` のとき `Speak` service request が発行される。
- stop command で `output_stop_topic` が publish される。

## launch test

- `config/default.yaml` で起動できる。
- missing `tts_service` や空 topic を fail closed できる。

## failure test

- TTS service unavailable を検出する。
- invalid mode を暗黙変換しない。

## latency test

command 受信から state publish までの control-plane latency を測る。

## real device smoke test

ASR result から command bridge を経由し、router state と TTS announcement が robot session で観測できることを確認する。
