# fa_voice_command_router 仕様書

## 目的

`fa_voice_command_router` は上位 app 層の voice command routing node である。音声認識結果や operator command を受け、router state を publish し、必要に応じて TTS announcement や audio output stop を要求する。

## 入力 topic

| topic | type | 内容 |
| --- | --- | --- |
| `command_topic` | `std_msgs/msg/String` | command router に渡す command 文字列 |

## 出力 topic

| topic | type | 内容 |
| --- | --- | --- |
| `state_topic` | `std_msgs/msg/String` | router の mode / active 状態 |
| `output_stop_topic` | `std_msgs/msg/Empty` | stop command 時の audio output 停止要求 |

## command schema

`command_topic` では以下の text command だけを受理する。互換 alias は持たない。

| command | 動作 |
| --- | --- |
| `start` | router を active にする |
| `stop` | router を inactive にし、設定に応じて `output_stop_topic` を publish する |
| `mode <name>` | `allowed_modes` に含まれる mode へ切り替える |
| `status` | 現在 state を再 publish する |

## parameters

全 parameter は起動時に明示設定されている必要がある。node 内の runtime default で欠落を補完しない。

| parameter | type | 意味 |
| --- | --- | --- |
| `command_topic` | string | command 入力 topic。空文字は禁止 |
| `state_topic` | string | state 出力 topic。空文字は禁止 |
| `active` | bool | router 初期 active 状態 |
| `mode` | string | 初期 mode。空文字は禁止 |
| `allowed_modes` | string array | 許可する mode 一覧。空配列と重複は禁止 |
| `announce_tts` | bool | mode change を TTS で通知するか |
| `tts_service` | string | `fa_interfaces/srv/Speak` service 名。空文字は禁止 |
| `tts_voice_id` | string | TTS voice id。空文字は許可 |
| `stop_output_on_stop` | bool | stop command で出力停止 topic を publish するか |
| `output_stop_topic` | string | 出力停止 topic。`stop_output_on_stop=true` の場合は空文字禁止 |

## lifecycle

起動時に parameter を読み、command subscription、state publisher、TTS service client、必要に応じて stop publisher を作る。mode が `allowed_modes` に含まれない場合は起動失敗にする。

runtime parameter update は `active`、`mode`、`allowed_modes` だけを受理する。それ以外の topic / service / startup-only parameter は publisher / client 再生成を伴うため拒否する。

## QoS

command / state は control-plane message であり、data-plane audio frame ではない。QoS は package 実装側で定義し、音声 frame の best-effort policy と混同しない。

## error handling

不明 command、未許可 mode、TTS service unavailable は state と log で明示する。未許可 mode を暗黙に `standby` へ置き換えない。必須 parameter 欠落、型違い、空文字、重複 `allowed_modes` は起動時に fail closed する。

## backend selection

なし。TTS 合成自体は `fa_tts` の責務であり、この node は service を呼ぶだけで backend を知らない。TTS の playback routing や gain は `fa_mix` / dynamics node の責務なので、この node は `Speak` request の `play=false`、`volume_db=0.0` を固定して送る。

## non-goals

- ASR
- KWS
- VAD
- TTS waveform generation
- audio frame processing
