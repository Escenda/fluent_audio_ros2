# fa_audio_mcp テスト設計

## 方針

テストは tool adapter の振る舞いを証明します。source string や Markdown 文言の存在確認は、この package の正しさの証明として扱いません。

現行の package-local tests は、ROS service 実体を起動しない unit test です。Full ROS launch / service smoke は `fa_asr` / `fa_audio_window` service と接続した別検証として扱います。

## Unit Test

- `FA-MCP-SPEC-001`: numeric `<start_unix_ns>..<end_unix_ns>` range だけを受け付ける。
- `FA-MCP-SPEC-002`: `start_unix_ns >= end_unix_ns` を拒否する。
- `FA-MCP-SPEC-003`: archive / transcribe は別 scope mapping を使う。
- `FA-MCP-SPEC-004`: unsupported scope、未設定 scope、解決不能な default scope を fail closed にする。
- `FA-MCP-SPEC-005`: archive request は trim 後に空の `reason` を拒否する。
- `FA-MCP-SPEC-006`: archive request は format 未指定時に `pcm_s16le` / `wav` / `audio/wav` を明示 request 値にする。
- `FA-MCP-SPEC-007`: `success=false` response を tool error に変換する。
- `FA-MCP-SPEC-008`: 成功 response は service response の resolved range と tool input の requested range を両方保持する。
- `FA-MCP-SPEC-009`: transport、port、timeout、service name、scope mapping の環境変数を検証する。
- `FA-MCP-SPEC-010`: service unavailable / timeout を tool error にする。
- `FA-MCP-SPEC-011`: omitted / null / blank `audio_scope` は tool ごとの explicit default scope key でのみ解決する。

## 未検証 / 追加が必要なテスト

- MCP transport ごとの起動 smoke。
- `fa_asr` / `fa_audio_window` 実 service と接続した full ROS service smoke。

## Full ROS Smoke

Full ROS smoke では次を確認します。

- `fa_audio_mcp_server` が ROS graph 上の `archive_audio_window` service を呼び、audio clip ref を tool result に返す。
- `fa_audio_mcp_server` が ROS graph 上の `transcribe_audio` service を呼び、transcript segments と model ref を tool result に返す。
- service が `success=false` を返した場合に tool error になる。
- unsupported scope、range 外、不連続 window が成功扱いにならない。

この smoke は現行の package-local unit test と同等扱いしません。
