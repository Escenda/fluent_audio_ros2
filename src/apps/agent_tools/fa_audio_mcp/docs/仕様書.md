# fa_audio_mcp 仕様書

## 責務

`fa_audio_mcp` は FluentAudio の timeline service を MCP tool として公開する adapter です。

対象 tool は次の 2 つです。

| Tool | ROS2 service | owner |
| --- | --- | --- |
| `archive_audio_window` | `fa_interfaces/srv/ArchiveAudioWindow` | archive / evidence window service |
| `transcribe_audio` | `fa_interfaces/srv/TranscribeAudio` | ASR timeline service |

## 非責務

- LLM 判断
- tool 選択 policy
- dialogue orchestration
- safety gate
- audio decode / resample / format conversion
- ASR / TTS / KWS / VAD backend 実行
- archive storage の永続化実体
- 自然言語 time range 解決
- marker / turn / action id からの range 解決
- unsupported scope の fallback

## Input Contract

### `archive_audio_window`

| 入力 | 型 | 契約 |
| --- | --- | --- |
| `time_range` | string | `<start_unix_ns>..<end_unix_ns>`。両端は 0 以上の整数で、start < end |
| `reason` | string | trim 後に空でないこと |
| `related_artifact_ids` | string list | service request へそのまま渡す |
| `audio_scope` | string or null | `mic` / `system` / `mixed` のうち、archive scope mapping に設定されたもの。省略 / null / blank は archive default scope key で解決する |
| `codec` | string or null | 未指定時は `pcm_s16le` |
| `container` | string or null | 未指定時は `wav` |
| `payload_format` | string or null | 未指定時は `audio/wav` |

### `transcribe_audio`

| 入力 | 型 | 契約 |
| --- | --- | --- |
| `time_range` | string | `<start_unix_ns>..<end_unix_ns>`。両端は 0 以上の整数で、start < end |
| `audio_scope` | string or null | `mic` / `system` / `mixed` のうち、transcribe scope mapping に設定されたもの。省略 / null / blank は transcribe default scope key が設定されている場合だけ解決する |

## Environment Contract

| 環境変数 | default | fail closed 条件 |
| --- | --- | --- |
| `FLUENT_AUDIO_MCP_TRANSPORT` | `stdio` | `stdio`, `sse`, `streamable-http` 以外 |
| `FLUENT_AUDIO_MCP_HOST` | `0.0.0.0` | なし |
| `FLUENT_AUDIO_MCP_PORT` | `9110` | 整数でない、または 0 以下 |
| `FLUENT_AUDIO_MCP_SERVICE_TIMEOUT_SEC` | `10.0` | 数値でない、または 0 以下 |
| `FLUENT_AUDIO_ARCHIVE_AUDIO_WINDOW_SERVICE` | `archive_audio_window` | trim 後に空 |
| `FLUENT_AUDIO_TRANSCRIBE_AUDIO_SERVICE` | `transcribe_audio` | trim 後に空 |
| `FLUENT_AUDIO_ARCHIVE_SCOPE_MIC` | `mic` | 空なら未設定として扱う |
| `FLUENT_AUDIO_ARCHIVE_SCOPE_SYSTEM` | 未設定 | 空なら未設定として扱う |
| `FLUENT_AUDIO_ARCHIVE_SCOPE_MIXED` | 未設定 | 空なら未設定として扱う |
| `FLUENT_AUDIO_ARCHIVE_DEFAULT_SCOPE` | `mic` | `mic`, `system`, `mixed` 以外 |
| `FLUENT_AUDIO_TRANSCRIBE_SCOPE_MIC` | 未設定 | 空なら未設定として扱う |
| `FLUENT_AUDIO_TRANSCRIBE_SCOPE_SYSTEM` | 未設定 | 空なら未設定として扱う |
| `FLUENT_AUDIO_TRANSCRIBE_SCOPE_MIXED` | 未設定 | 空なら未設定として扱う |
| `FLUENT_AUDIO_TRANSCRIBE_DEFAULT_SCOPE` | 未設定 | `mic`, `system`, `mixed` 以外 |

## Scope Contract

`archive` と `transcribe` は別々の `AudioScopeConfig` を使います。

SO101 voice frontend では、archive は `fa_audio_window` の supported scope である `mic` を使い、transcribe は ASR-ready stream identity である `audio/high_pass/mic` を使います。

```bash
export FLUENT_AUDIO_ARCHIVE_SCOPE_MIC=mic
export FLUENT_AUDIO_TRANSCRIBE_SCOPE_MIC=audio/high_pass/mic
```

unsupported scope、空 scope、未設定 scope は `unsupported_audio_scope` tool error です。別 scope や default scope に補完しません。
ただし `audio_scope` が省略、null または blank の場合だけ、tool ごとの default scope key で解決します。default scope key が未設定、または key の指す scope mapping が未設定の場合は `unsupported_audio_scope` です。
archive の default scope key は config loading で明示的に `mic` に設定されます。transcribe は profile / environment で `FLUENT_AUDIO_TRANSCRIBE_DEFAULT_SCOPE` を明示しない限り、null / blank の `audio_scope` では利用可能になりません。

## Output Contract

### `archive_audio_window`

成功時は次を返します。

- `audio_clip_ref`
- `time_range`
- `requested_time_range`

ROS service が `success=false` を返した場合は、service response の `error_code` と `message` を tool error として返します。

### `transcribe_audio`

成功時は次を返します。

- `segments`
- `audio_window_ref`
- `model_ref`
- `time_range`
- `requested_time_range`

ROS service が `success=false` を返した場合は、service response の `error_code` と `message` を tool error として返します。

## Fail Closed

- `time_range` が numeric range でない場合は service を呼ばない。
- `start_unix_ns >= end_unix_ns` の場合は service を呼ばない。
- unsupported scope は service を呼ばない。
- null / blank scope が default scope key で解決できない場合は service を呼ばない。
- `reason` が空の archive request は service を呼ばない。
- service が timeout / unavailable の場合は tool error にする。
- service response の failure を正常 JSON result に変換しない。

## 実行

```bash
ros2 run fa_audio_mcp fa_audio_mcp_server
```

`sse` / `streamable-http` transport では host / port を環境変数で指定できます。

```bash
export FLUENT_AUDIO_MCP_TRANSPORT=streamable-http
export FLUENT_AUDIO_MCP_HOST=127.0.0.1
export FLUENT_AUDIO_MCP_PORT=9110
ros2 run fa_audio_mcp fa_audio_mcp_server
```

## Test IDs

- `FA-MCP-SPEC-001`: numeric `<start_unix_ns>..<end_unix_ns>` range だけを受け付ける。
- `FA-MCP-SPEC-002`: start >= end の range を拒否する。
- `FA-MCP-SPEC-003`: archive / transcribe は別 scope mapping を使う。
- `FA-MCP-SPEC-004`: unsupported scope を fail closed にする。
- `FA-MCP-SPEC-005`: archive request は空 reason を拒否する。
- `FA-MCP-SPEC-006`: archive request は format 省略時に明示 default を request 値にする。
- `FA-MCP-SPEC-007`: ROS service failure を tool error に変換する。
- `FA-MCP-SPEC-008`: 成功 response は requested range と resolved response range を保持する。
- `FA-MCP-SPEC-009`: transport / port / timeout / service name の環境変数を検証する。
- `FA-MCP-SPEC-010`: service unavailable / timeout を tool error にする。
- `FA-MCP-SPEC-011`: null / blank `audio_scope` は tool ごとの explicit default scope key でのみ解決する。
