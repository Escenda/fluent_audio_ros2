# fa_audio_mcp 仕様書

## 責務

`fa_audio_mcp` は FluentAudio の timeline service を MCP tool として公開する adapter です。

対象 tool は次の 3 つです。

| Tool | ROS2 service | owner |
| --- | --- | --- |
| `export_audio_window` | `fa_interfaces/srv/ExportAudioWindow` | temporary clip export service |
| `archive_audio_window` | `fa_interfaces/srv/ArchiveAudioWindow` | archive / evidence window service |
| `transcribe_audio` | `fa_interfaces/srv/TranscribeAudio` | ASR timeline service |

## 非責務

- LLM 判断
- tool 選択 policy
- dialogue orchestration
- safety gate
- audio decode / resample / format conversion
- ASR / TTS / KWS / VAD backend 実行
- archive storage の永続化実体
- 自然言語 time range 解決
- marker / turn / action id からの range 解決
- unsupported scope の fallback

## Time Range Contract

`fa_audio_mcp` は Agent-facing な `time_range` を service-facing な numeric range に変換します。下流の `ArchiveAudioWindow` / `TranscribeAudio` service には、必ず `<start_unix_ns>..<end_unix_ns>` 形式だけを渡します。

受け付ける形式:

- `<start_unix_ns>..<end_unix_ns>`
- `now-<duration>..now`
- `now-<duration>..now-<duration>`

`duration` の単位は `ms`、`s`、`m` です。

例:

```text
1700000000000000000..1700000010000000000
now-10s..now
now-1500ms..now-500ms
now-2m..now-1m
```

`now` は `fa_audio_mcp_server` の ROS node clock で解決します。別の wall clock へ silent fallback しません。解決後の start / end が不正な場合は service を呼ばずに `invalid_time_range` tool error にします。

`now..now`、`now..now-<duration>`、`now+<duration>` は解決後の範囲または構文が不正なため拒否します。marker、turn id、action id、自然言語表現は現行責務ではありません。

## Scope Contract

`export`、`archive`、`transcribe` は別々の `AudioScopeConfig` を使います。

SO101 voice frontend では、export / archive は `fa_audio_window` の supported scope である `mic` を使い、transcribe は ASR-ready stream identity である `audio/high_pass/mic` を使います。

```bash
export FLUENT_AUDIO_EXPORT_SCOPE_MIC=mic
export FLUENT_AUDIO_ARCHIVE_SCOPE_MIC=mic
export FLUENT_AUDIO_TRANSCRIBE_SCOPE_MIC=audio/high_pass/mic
```

unsupported scope、空 scope、未設定 scope は `unsupported_audio_scope` tool error です。別 scope や default scope に補完しません。
ただし `audio_scope` が省略、null または blank の場合だけ、tool ごとの default scope key で解決します。default scope key が未設定、または key の指す scope mapping が未設定の場合は `unsupported_audio_scope` です。
export / archive の default scope key は config loading で明示的に `mic` に設定されます。transcribe は profile / environment で `FLUENT_AUDIO_TRANSCRIBE_DEFAULT_SCOPE` を明示しない限り、null / blank の `audio_scope` では利用可能になりません。

## Output Contract

### `export_audio_window`

成功時は次を返します。

- `audio_clip_ref`
- `time_range`
- `requested_time_range`

ROS service が `success=false` を返した場合は、service response の `error_code` と `message` を tool error として返します。

### `archive_audio_window`

成功時は次を返します。

- `audio_clip_ref`
- `time_range`
- `requested_time_range`

ROS service が `success=false` を返した場合は、service response の `error_code` と `message` を tool error として返します。

### `transcribe_audio`

成功時は次を返します。

- `segments`
- `audio_window_ref`
- `model_ref`
- `time_range`
- `requested_time_range`

ROS service が `success=false` を返した場合は、service response の `error_code` と `message` を tool error として返します。

## MCP Tool Registration

`export_audio_window`、`archive_audio_window`、`transcribe_audio` は `FastMCP` tool として登録します。tool の返り値は `AudioClipRef` や `TranscriptSegment` を含む JSON-compatible dict です。Pydantic structured output schema でこの dict を再解釈させず、response formatter が作った JSON 契約をそのまま返すため、tool 登録では structured output を無効にします。

## Fail Closed

- `time_range` が numeric range または `now` relative range でない場合は service を呼ばない。
- `now` relative range を node clock で解決できない場合は service を呼ばない。
- 解決後に `start_unix_ns >= end_unix_ns` の場合は service を呼ばない。
- unsupported scope は service を呼ばない。
- null / blank scope が default scope key で解決できない場合は service を呼ばない。
- `reason` が空の archive request は service を呼ばない。
- service が timeout / unavailable の場合は tool error にする。
- service response の failure を正常 JSON result に変換しない。

## 実行

```bash
ros2 run fa_audio_mcp fa_audio_mcp_server
```

`sse` / `streamable-http` transport では host / port を環境変数で指定できます。

```bash
export FLUENT_AUDIO_MCP_TRANSPORT=streamable-http
export FLUENT_AUDIO_MCP_HOST=127.0.0.1
export FLUENT_AUDIO_MCP_PORT=9110
ros2 run fa_audio_mcp fa_audio_mcp_server
```

## Test IDs

- `FA-MCP-SPEC-001`: numeric `<start_unix_ns>..<end_unix_ns>` range を受け付ける。
- `FA-MCP-SPEC-002`: 解決後の start >= end の range を拒否する。
- `FA-MCP-SPEC-003`: archive / transcribe は別 scope mapping を使う。
- `FA-MCP-SPEC-004`: unsupported scope を fail closed にする。
- `FA-MCP-SPEC-005`: archive request は空 reason を拒否する。
- `FA-MCP-SPEC-006`: archive request は format 省略時に明示 default を request 値にする。
- `FA-MCP-SPEC-007`: ROS service failure を tool error に変換する。
- `FA-MCP-SPEC-008`: 成功 response は requested range と resolved response range を保持する。
- `FA-MCP-SPEC-009`: transport / port / timeout / service name の環境変数を検証する。
- `FA-MCP-SPEC-010`: service unavailable / timeout を tool error にする。
- `FA-MCP-SPEC-011`: null / blank `audio_scope` は tool ごとの explicit default scope key でのみ解決する。
- `FA-MCP-SPEC-012`: `now[-duration]..now[-duration]` range を node clock で numeric range に解決してから service request に渡す。
- `FA-MCP-SPEC-013`: `export_audio_window` は export service / export scope mapping を使い、archive reason や related artifact id を持たない。
- `FA-MCP-SPEC-014`: `FastMCP` tool 呼び出しは、実 `rclpy` client と生成済み service 型を通して ROS service request / response に到達する。
- `FA-MCP-SPEC-015`: 同じ ROS client node の service call は直列化し、複数 executor から同時に同一 node を spin しない。
- `FA-MCP-SPEC-016`: 実 `FastMCP` + `rclpy` 経路でも、service `success=false` は正常 result ではなく MCP tool error になる。
