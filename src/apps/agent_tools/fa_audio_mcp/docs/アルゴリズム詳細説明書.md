# fa_audio_mcp アルゴリズム詳細説明書

## 全体フロー

1. 起動時に `load_server_config()` が環境変数を読み、transport、ROS service 名、timeout、export / archive / transcribe scope mapping、tool ごとの default scope key を確定する。
2. `rclpy` node `fa_audio_mcp_server` を作成する。
3. `ExportAudioWindow`、`ArchiveAudioWindow`、`TranscribeAudio` の ROS service client を作成する。
4. `FastMCP("fluent-audio")` に `export_audio_window`、`archive_audio_window`、`transcribe_audio` tool を登録する。
5. tool 呼び出しごとに入力を deterministic な request 値へ変換し、ROS service を呼び、response を tool result または tool error に変換する。

## time range resolution

受け付ける形式は numeric range と `now` relative range です。

```text
<start_unix_ns>..<end_unix_ns>
now-10s..now
now-1500ms..now-500ms
now-2m..now-1m
```

numeric range の判定手順:

1. 正規表現で両端が 0 または先頭 0 なしの正整数表現であることを確認する。
2. `int` に変換する。
3. `start_unix_ns < end_unix_ns` を確認する。
4. `NumericTimeRange` と service-facing canonical `spec` を作る。

`now` relative range の判定手順:

1. 入力が numeric range でない場合にだけ `now` relative として解釈する。
2. `fa_audio_mcp_server` の ROS node clock から `now_unix_ns` を取得する。
3. 両端が `now`、`now-<N>ms`、`now-<N>s`、`now-<N>m` のいずれかであることを確認する。
4. `now_unix_ns - duration` によって各 endpoint を解決する。
5. endpoint が 0 未満になる場合は `invalid_time_range`。
6. 解決後に `start_unix_ns < end_unix_ns` を確認する。
7. service-facing `time_range_spec` は必ず `<start_unix_ns>..<end_unix_ns>` にする。
8. tool result の `requested_time_range.spec` には、Agent が入力した元の `time_range` を保持する。

marker 名、turn id、action id、負数、不完全 range、`now+<duration>` は受け付けません。自然言語や marker 解決はこの package の現行責務ではありません。

## scope resolution

`AudioScopeResolver` は `mic`、`system`、`mixed` だけを解決対象にします。export、archive、transcribe は別 resolver です。

解決手順:

1. 入力 `audio_scope` が省略または null なら default scope key を使う。
2. 入力 `audio_scope` が string なら trim する。
3. trim 後に空なら default scope key を使う。
4. `mic` / `system` / `mixed` のうち、対応する config 値が設定されている場合だけその値を返す。
5. default scope key が未設定、default scope key の指す mapping が未設定、または未知 scope は `unsupported_audio_scope`。

scope は ROS topic 名ではなく Agent-facing な論理名です。解決後の値は ROS service の `audio_scope` に渡す concrete scope / stream identity です。

## export request 生成

`export_audio_window` tool は次を行います。

1. `time_range` を numeric range または `now` relative range として解決する。
2. export scope resolver で `audio_scope` を解決する。
3. `codec` / `container` / `payload_format` が未指定なら、`pcm_s16le` / `wav` / `audio/wav` を request 値として明示する。
4. `ExportAudioWindow.Request` を作り、ROS service を呼ぶ。

export は一時 clip 取得の tool です。保存判断理由や関連 artifact id は持ちません。永続化の根拠を残す場合は `archive_audio_window` を使います。

## archive request 生成

`archive_audio_window` tool は次を行います。

1. `reason` を trim し、空なら `invalid_archive_request`。
2. `time_range` を numeric range または `now` relative range として解決する。
3. archive scope resolver で `audio_scope` を解決する。
4. `codec` / `container` / `payload_format` が未指定なら、`pcm_s16le` / `wav` / `audio/wav` を request 値として明示する。
5. `ArchiveAudioWindow.Request` を作り、ROS service を呼ぶ。

format default は request contract の明示値です。audio decode、resample、format conversion は行いません。

## transcribe request 生成

`transcribe_audio` tool は次を行います。

1. `time_range` を numeric range または `now` relative range として解決する。
2. transcribe scope resolver で `audio_scope` を解決する。
3. `TranscribeAudio.Request` を作り、ROS service を呼ぶ。

ASR-ready timeline の所有者は ROS service owner です。adapter は音声 window を取得して ASR 用に変換しません。

## ROS service call

`RosAudioTimelineClient` は service 呼び出しで次を保証します。

1. `wait_for_service(timeout_sec=...)` が失敗した場合は `service_unavailable`。
2. `call_async()` 後、同じ timeout で `spin_until_future_complete()` を実行する。
3. response が `None` の場合は `service_timeout`。
4. response が存在する場合は formatter に渡す。

## response formatting

export / archive response:

1. `success=false` なら response の `error_code` / `message` を `AudioToolError` にする。
2. `audio_clip_ref`、resolved `time_range`、requested `time_range` を JSON-compatible dict にする。

transcribe response:

1. `success=false` なら response の `error_code` / `message` を `AudioToolError` にする。
2. `segments`、`audio_window_ref`、`model_ref`、resolved `time_range`、requested `time_range` を JSON-compatible dict にする。

## hidden behavior の禁止

この adapter は次を行いません。

- scope の推測
- time range の推測
- unsupported scope の default fallback
- transcribe default scope の暗黙有効化
- PCM16 と FLOAT32 の相互変換
- sample rate / channel / layout 変換
- service failure の成功 result 化
- LLM による保存判断
