# fa_dialogue テスト設計

## Unit Test

- `FA-DIALOGUE-SPEC-001`: package 化前でも README、仕様書、アルゴリズム詳細説明書、テスト設計、backend docs が存在する。
- `FA-DIALOGUE-SPEC-002`: `package.xml` と `CMakeLists.txt` を package 化前に置かない。
- `FA-DIALOGUE-SPEC-003`: backend failure を canned response や default model へ fallback しないことを仕様書で固定する。
- `FA-DIALOGUE-SPEC-004`: safety policy の最終 accept / reject を dialogue の責務にしない。

## Integration Test

ROS2 package 化後に fake external dialogue service を使い、transcript、turn state、safety context、backend response schema、TTS text request、command proposal を検証する。

## Launch Test

ROS2 package 化後に missing backend、missing model id、missing safety context、stale turn state で起動または処理が fail closed することを確認する。

## Fixture

fixtures には valid transcript、stale transcript、backend timeout response、schema mismatch response、safety handoff required response を置く。
