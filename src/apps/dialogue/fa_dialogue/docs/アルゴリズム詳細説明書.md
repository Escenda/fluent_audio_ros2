# fa_dialogue アルゴリズム詳細説明書

## 処理手順

```text
wake / turn / transcript event
  ↓
validate typed input and freshness
  ↓
update dialogue session state
  ↓
build external dialogue request
  ↓
call configured external_dialogue_service
  ↓
validate response schema and safety handoff fields
  ↓
publish dialogue state / TTS text / command proposal
```

## state handling

session id、turn id、transcript timestamp、backend response timestamp を持つ。stale な state は現在の対話状態として再利用しない。

## backend handling

external dialogue backend は ROS 2 node 内の hidden model ではなく、明示的な service / worker / process 境界として扱う。backend timeout や unavailable は成功扱いにせず、action proposal を publish しない。

## safety handoff

dialogue output が command proposal を含む場合でも、その proposal は `fa_safety_policy` の判定前状態である。dialogue node は安全 policy の accept / reject を代行しない。

## forbidden behavior

- backend failure を canned response で隠す
- transcript 欠損を空文字として処理継続する
- stale turn state を current state として扱う
- ASR / KWS / TTS waveform generation をこの package に混入する
