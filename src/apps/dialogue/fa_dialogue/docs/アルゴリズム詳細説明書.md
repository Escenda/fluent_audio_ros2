# fa_dialogue アルゴリズム詳細説明書

## 処理手順

```text
wake event
  ↓
validate detected, keyword, stamp freshness
  ↓
start session when needed
  ↓
assign next user_turn_id
  ↓
publish TurnContext(active=true)
```

```text
ASR result / TurnEnd
  ↓
validate active turn exists
  ↓
validate session_id and user_turn_id match active turn
  ↓
validate terminal condition
  ↓
clear active turn
  ↓
publish TurnContext(active=false)
```

## state handling

session id は `session_prefix` と単調増加 sequence から生成する。乱数や時刻由来の値は使わない。

turn id は session 内で単調増加する。active session 中に再 wake を受けた場合は、新しい turn id を発行し、前 turn を置き換える。古い turn id に対する ASR / TurnEnd は mismatch として無視する。

## freshness handling

wake stamp が zero でない場合、受信時刻との差が `wake.max_age_ms` を超えた wake は拒否する。zero stamp は `wake.allow_zero_stamp=true` の場合だけ受理する。

## backend handling

今回の slice では external dialogue backend を呼ばない。backend 未設定を canned response や default model へ置き換える処理も持たない。

## forbidden behavior

- backend failure を canned response で隠す
- transcript 欠損を空文字として処理継続する
- stale turn state を current state として扱う
- ASR / KWS / TTS waveform generation をこの package に混入する
- session / turn mismatch を active turn として扱う
