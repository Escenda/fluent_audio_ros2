# fa_dialogue 仕様書

## 1. 目的

`fa_dialogue` は audio-driven interaction の turn context publisher である。
この最小実装では wake word、ASR result、turn end を明示的な ID 契約で接続し、`TurnContext` の現在値だけを出力する。

reasoning、TTS、safety policy、external dialogue backend、action proposal は今回の責務外である。

## 2. 入力

入力は `fa_interfaces` の型付き message とする。

- `WakeWordResult`
  - `detected=true` かつ `keyword` が空でない場合だけ turn 開始候補とする
  - `header.stamp` が stale な場合は拒否する
  - zero stamp は `wake.allow_zero_stamp=true` の場合だけ許可する
- `AsrResult`
  - active turn の `session_id` / `user_turn_id` と一致する結果だけ処理する
  - `STATUS_FINAL` / `STATUS_TIMEOUT` / `STATUS_ERROR` だけ turn 終了条件とする
- `TurnEnd`
  - active turn の `session_id` / `user_turn_id` と一致する結果だけ処理する
  - `is_end=true` だけ turn 終了条件とする

## 3. 出力

出力は `TurnContext` のみである。

- wake 受理時: `active=true`
- turn 終了時: `active=false`
- session id: `session_prefix` と単調増加 sequence から決定的に生成する
- user turn id: session 内で単調増加する

## 4. state handling

初回 wake で session を開始する。active session 中の再 wake は新しい `user_turn_id` を発行し、前 turn を置き換える。

`active=false` publish 後も session は保持してよい。ただし current turn は無効化し、以後の ASR / TurnEnd は新しい wake が来るまで処理しない。

## 5. fail closed 条件

以下は処理せず、推測や補完をしない。

- `detected=false`
- keyword が空の wake
- `wake.max_age_ms` を超えた stale wake
- 設定で許可されていない zero stamp wake
- turn 未開始の ASR / TurnEnd
- session mismatch / turn mismatch
- terminal ではない ASR status
- `is_end=false` の TurnEnd

## 6. 責務外

- KWS / ASR / VAD / turn detector model runtime
- TTS waveform generation
- audio frame processing
- robot command の最終安全判断
- external dialogue backend
- action proposal
- stale context を current state として扱うこと
