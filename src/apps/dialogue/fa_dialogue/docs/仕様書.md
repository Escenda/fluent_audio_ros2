# fa_dialogue 仕様書

## 1. 目的

`fa_dialogue` は audio-driven interaction の dialogue orchestration app である。wake word、ASR transcript、turn state、TTS request、external reasoning service を明示的な境界で接続し、dialogue state と action proposal を出力する。

## 2. 入力

入力は package 化時に専用 message / service contract として定義する。少なくとも次の情報を曖昧な dict ではなく型付き payload として受ける。

- wake / session event
- ASR transcript と confidence
- turn state
- safety context
- operator confirmation
- external reasoning service response

## 3. 出力

出力は dialogue state、TTS text request、command proposal、handoff reason を含む。robot command の最終 accept / reject は `fa_safety_policy` の責務であり、この package で安全判断済みとして扱わない。

## 4. backend contract

外部 reasoning service は `docs/backends/external_dialogue_service.md` の contract に従う。backend 名、endpoint または command、model id、timeout、response schema は必須設定とする。未設定の backend を default model や canned response へ置き換えない。

## 5. fail closed 条件

- transcript 欠損または confidence / timestamp 欠損
- session id 欠損
- stale な turn state
- safety context 欠損
- backend 未設定
- backend timeout / unavailable
- backend response schema mismatch

fail closed 時は action proposal を出さず、明示的な error state / reason を publish する。

## 6. 責務外

- KWS / ASR / VAD / turn detector model runtime
- TTS waveform generation
- audio frame processing
- robot command の最終安全判断
- missing backend fallback
- stale context を current state として扱うこと
