# fa_safety_policy テスト設計

## Unit Test

- `FA-SAFETY-POLICY-SPEC-001`: package 化前でも README、仕様書、アルゴリズム詳細説明書、テスト設計、backend absence docs が存在する。
- `FA-SAFETY-POLICY-SPEC-002`: backend / plugin responsibility がないことを `docs/backends/no_runtime_backend.md` で固定する。
- `FA-SAFETY-POLICY-SPEC-003`: `package.xml` と `CMakeLists.txt` を package 化前に置かない。
- `FA-SAFETY-POLICY-SPEC-004`: unknown action、missing state、stale state を default allow にしない。

## Integration Test

ROS2 package 化後に fake voice command router / dialogue proposal と fake robot state を使い、accept、reject、require_confirmation、block の decision contract を確認する。

## Launch Test

ROS2 package 化後に missing policy file、missing robot state topic、invalid confirmation TTL、unknown action で fail closed することを確認する。

## Fixture

fixtures には accepted command、rejected command、confirmation required command、stale state、unknown action、missing robot state を置く。
