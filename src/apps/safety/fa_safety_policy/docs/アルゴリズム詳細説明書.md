# fa_safety_policy アルゴリズム詳細説明書

## 処理手順

```text
command proposal
  ↓
validate typed input and timestamp freshness
  ↓
load deterministic policy rules
  ↓
evaluate deny / block rules
  ↓
evaluate confirmation requirements
  ↓
evaluate explicit allow rules
  ↓
publish decision and reason code
```

## rule order

deny / block rule は allow rule より優先する。confirmation が必要な action は operator confirmation が fresh な場合だけ allow できる。

## missing data handling

missing state、unknown action、stale timestamp は安全側の停止条件である。空値や default mode を補完して allow 判定へ進めない。

## backend handling

runtime backend は持たない。policy は deterministic rule evaluation として定義し、external model や plugin に safety decision を委譲しない。

## forbidden behavior

- default allow
- warning only continuation
- stale robot state を current state として扱う
- unknown action を low-risk action と推測する
- dialogue node の intent を安全判断済みとして扱う
