# fa_safety_policy 仕様書

## 1. 目的

`fa_safety_policy` は audio-driven command proposal を安全に accept / reject / require_confirmation / block へ分類する app layer policy である。dialogue や voice command router から来る proposal を、robot/session state と operator confirmation に照らして判定する。

## 2. 入力

入力は package 化時に専用 message / service contract として定義する。少なくとも次の情報を境界で型検証する。

- command proposal
- requested action
- source package
- session id
- robot state
- active mode
- operator confirmation state
- timestamp

## 3. 出力

出力は policy decision と reason code を持つ。

- `accept`
- `reject`
- `require_confirmation`
- `block`

`accept` は必須入力がすべて fresh で、policy rule が明示的に許可した場合だけ返す。unknown action、missing state、stale state は default allow にしない。

## 4. backend contract

この package は deterministic policy evaluation の責務を持つ。model runtime、external reasoning service、audio DSP backend、plugin backend は選択しない。
backend が無いこと自体を `docs/backends/no_runtime_backend.md` で明示する。backend / plugin responsibility が追加される場合は、deterministic policy の責務境界を見直したうえで具体 backend contract を追加する。

## 5. fail closed 条件

- command proposal 欠損
- session id 欠損
- robot state 欠損
- stale timestamp
- unknown action
- confirmation required だが confirmation がない
- policy rule が action を明示的に許可していない

fail closed 時は `reject` または `block` と reason code を返す。空 decision、default allow、警告のみで継続はしない。

## 6. 責務外

- KWS / ASR / VAD / dialogue reasoning
- TTS generation
- audio frame processing
- robot driver control
- backend selection
- unsafe action の fallback allow
