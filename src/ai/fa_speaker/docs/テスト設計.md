# fa_speaker テスト設計

## 現在の位置づけ

`fa_speaker` は package 化前の roadmap note である。現時点では文書ファイル、package、実装ファイルの存在確認を仕様の根拠にしない。
package 化までは、この文書は実装済みテストの一覧ではなく、package 化時に必須化する behavioral / capability contract を定義する。

## Package 化後に必要な Unit / Backend Test

- fake backend capability validation: `backend.name`、provider / worker command、model id または model path、supported sample rates、supported window、batch / streaming mode、output schema、embedding dimension、enrolled speaker index が欠損または矛盾した場合は起動失敗する。
- accepted `AudioFrame` contract: config と capability が明示する `FLOAT32LE`、`bit_depth=32`、mono、interleaved、configured sample rate、configured window だけを受け付ける。
- unsupported input rejection: `PCM16`、`PCM32`、stereo、planar、sample-rate mismatch、window mismatch、non-finite sample、range violation は変換せず frame reject または fail closed にする。
- output schema contract: speaker embedding、speaker id、verification score、model id、source identity、stream identity を必須にする。
- speaker contract: unknown speaker fallback と enrolled speaker index 外への既知 speaker 割り当てを禁止する。

## Package 化後に必要な Integration / Launch Test

- fake speaker backend で embedding dimension、speaker id、verification score、model id propagation を確認する。
- missing params file、missing backend、missing model capability、missing enrolled speaker index、unsupported input format で起動失敗または frame reject することを確認する。
