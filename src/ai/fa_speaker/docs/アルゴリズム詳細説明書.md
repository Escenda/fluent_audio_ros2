# fa_speaker アルゴリズム詳細説明書

1. startup config で `backend.name`、model path/model id、enrolled speaker index、expected input format を検証する。
2. input frame の source identity、stream identity、format metadata を検証する。
3. backend に canonical audio tensor を渡す。
4. backend output の embedding dimension、speaker id、verification score を検証する。
5. 契約を満たす場合だけ speaker result を publish する。

unknown speaker を default known speaker に割り当てない。
Python version / venv が異なる backend は external worker / process / container 境界にする。
