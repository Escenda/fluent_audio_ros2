# fa_asr 仕様書

## 責務

`fa_asr` は発話区間の audio buffer を ASR backend に渡し、文字起こし結果を publish する audio AI node です。

## 非責務

- VAD 推定
- KWS
- turn detection
- device source selection
- ASR backend fallback

## Input / Output

| 種別 | 名前 | 型 |
| --- | --- | --- |
| Sub | `audio_topic` | `fa_interfaces/msg/AudioFrame` |
| Sub | `turn_context_topic` | `fa_interfaces/msg/TurnContext` |
| Sub | `control.<id>.topic` | `control.<id>.msg_type`。現行既定は `control.speech_control.topic=voice/vad_state` / `fa_interfaces/msg/VadState` |

| Srv | `transcribe_service_name` | `fa_interfaces/srv/TranscribeAudio` |

## Backend Contract

- `backend.name` は必須。
- `backend.result_format` は non-streaming command backend を設定する場合だけ必須。
- `nemo_rnnt_streaming` は JSONL streaming protocol を使い、`backend.result_format` / `backend.args` / `backend.health_args` を使わない。
- `nemo_offline_transcribe` は local `.nemo` と NeMo offline `transcribe(...)` を使う non-streaming command / worker backend。working tree 上では backend / worker / tests / profile work が作成済みで、PO 検証により real worker health と raw float32le fixture transcription は通過済み。ただし file-source full ROS graph と accuracy 評価は未検証。
- default config は `backend.name` / command backend 用 `backend.result_format` を空にし、backend selection と output contract を暗黙指定しない。
- backend は ROS2 topic/message を知らない。
- backend input は mono float samples と sample rate。
- non-streaming command backend output は `backend.result_format` に従って `AsrTranscript` segments へ変換する。
- streaming backend output は backend 固有 protocol で `AsrStreamResult` へ変換する。

`backend.result_format` の対応値は `plain_text` と `segments_json_v1` です。この設定は `local_command`、`whisper.cpp`、`parakeet_worker`、`nemo_offline_transcribe`、`openai_realtime`、`openai_transcriptions` の command-process backend に適用します。`plain_text` は worker が transcript text を返す形式であり、空白 trim 後の text が空なら失敗です。`fa_asr` はその text を selected ASR request samples 全体 `[0, sample_count)` を覆う 1 segment にします。`segments_json_v1` は worker が strict JSON を返す形式です。top-level は `result_format` と `segments` だけを持ち、`result_format` は `segments_json_v1` と完全一致しなければなりません。`segments` は非空 list であり、各 segment は integer `start_sample` / `end_sample`、非空 `text`、任意の非空 `speaker_label` だけを持ちます。segment offset は selected ASR request samples からの相対 sample index です。

`segments_json_v1` の JSON parse failure、schema 不一致、unsupported field、sample offset が integer でない値、`start_sample < 0`、`start_sample >= end_sample`、`end_sample > sample_count`、segment overlap / unsorted order、空 text、空 `speaker_label` は fail closed とします。`fa_asr` は backend output を推測、補正、型 coercion、resample、downmix、PCM/int16 変換して成功扱いにしません。

`TranscribeAudio` では、backend segment offset を selected range の start からの相対 sample index として扱い、absolute unix ns へ変換します。segment start は `selected_start_unix_ns + floor(start_sample * 1e9 / sample_rate)`、segment end は `selected_start_unix_ns + ceil(end_sample * 1e9 / sample_rate)` として返します。

## Supported Backend Slots

- `local_command`
- `whisper.cpp`
- `parakeet_worker`
- `nemo_offline_transcribe`
- `nemo_rnnt_streaming`
- `openai_realtime`
- `openai_transcriptions`

各 backend は個別 class と loader を持ち、`LocalCommandAsrBackend` の名前差し替え alias として扱いません。共通 subprocess 実行は内部 helper に閉じます。

`local_command` と `whisper.cpp` は local process backend として `backend.model_path` を要求します。`nemo_offline_transcribe` は local `.nemo` を使う non-streaming command / worker backend として `backend.command`、local `.nemo` file の `backend.model_path`、`backend.language`、`backend.sample_rate_hz`、`backend.channels=1`、`backend.result_format` を要求します。`nemo_rnnt_streaming` は JSONL streaming worker backend として `backend.command`、local `.nemo` file の `backend.model_path`、`backend.sample_rate_hz`、`backend.channels`、`backend.chunk_size_samples`、`backend.emit_partial`、`backend.max_partial_interval_ms` を要求します。`parakeet_worker`、`openai_realtime`、`openai_transcriptions` は external worker / process / container backend として `backend.model` を要求します。OpenAI 系 backend はさらに対応する `backend.openai_*.api_key_env` を要求し、その環境変数が空でないことを起動時に検証します。OpenAI SDK/API は `fa_asr` に import せず、worker 側の責務にします。OpenAI worker には configured env の値を `OPENAI_API_KEY` として process environment で渡し、argv / stdout / stderr / ROS topic / log には key 値を渡しません。

未知 backend、backend command 欠落、model / model_path 欠落、invalid working directory は起動失敗にします。
backend runtime timeout/error は `AsrResult.STATUS_ERROR` を publish してから `rclpy.shutdown()` で fail closed します。error result は downstream への明示通知であり、別 backend への fallback や継続運転は行いません。
`backend.args` は non-streaming command 系 backend ごとの worker/CLI contract なので default config では空にし、該当 backend で ASR を有効化する profile または system config で必ず明示します。空配列、`{audio}` 欠落、`{model}` 欠落、`{sample_rate}` 欠落は起動失敗です。`nemo_rnnt_streaming` は JSONL streaming protocol を固定 contract として持つため、`backend.args` を使いません。
`backend.args` の placeholder は `{audio}`, `{model}`, `{language}`, `{output}`, `{sample_rate}` だけを許可します。未知 placeholder、壊れた format string、conversion、format spec は起動失敗です。
`backend.health_args` は non-streaming command 系 backend の startup health check 用 worker/CLI contract です。placeholder は `{model}`, `{language}` だけを許可します。`parakeet_worker`、`openai_realtime`、`openai_transcriptions` は external worker / process / container backend なので `backend.health_args` が必須で、`{model}` 欠落、未知 placeholder、壊れた format string、conversion、format spec は起動失敗です。health check が non-zero exit または timeout の場合も起動失敗です。`local_command` / `whisper.cpp` では package 単体の backend contract としては任意ですが、profile template で ASR を enable 可能にする場合は startup health check を明示するために system config 側へ設定します。指定された場合は同じ検証と startup health check を行います。`nemo_rnnt_streaming` は `backend.health_args` を使わず、backend 起動時に JSONL `health` message を worker へ送り、model class、cache-aware streaming capability、sample rate、channels、partial support、language、chunk size、partial interval が config と一致しなければ起動失敗にします。
`backend.args` が `{output}` を使う場合は `backend.output_text_path` が必須です。`backend.output_text_path` の placeholder は `{audio}`, `{model}`, `{language}`, `{sample_rate}`, `{session_id}`, `{user_turn_id}` だけを許可します。
`backend.command` と `backend.model_path` は起動時に絶対 path へ解決します。command は実行可能 file、model path は読み取り可能 file でなければ起動失敗です。

non-streaming command 系 backend は backend payload encoding として `float32le_raw` を明示します。これは external worker に渡す一時ファイルの serialization contract であり、`fa_asr` data plane では bit depth / value range 変換、hidden resample、downmix、clipping を行いません。入力 `AsrRequest` は `float32`、1 次元、非空、finite/normalized samples、正の sample rate でなければ backend に渡しません。`backend.args` は `{audio}`、`{model}`、`{sample_rate}` を含める必要があります。`nemo_rnnt_streaming` の streaming audio は JSONL `audio` command の `base64_float32le` payload であり、この `backend.args` contract ではありません。

### `nemo_offline_transcribe`

`nemo_offline_transcribe` は、local `.nemo` file を NeMo offline / full-context `transcribe(...)` API で呼ぶ backend です。
NIM / Riva / gRPC を使わず、`fa_asr` node 本体は NeMo / Torch / Parakeet を import しません。

contract は次です。

- non-streaming command / worker backend として実装されている。
- `backend.command`、local readable `.nemo` の `backend.model_path`、`backend.language`、`backend.sample_rate_hz`、`backend.channels=1`、`backend.result_format` を明示する。
- `fa_asr` は ASR-ready mono `FLOAT32LE` samples と sample rate だけを backend boundary に渡す。
- worker は validated raw normalized `FLOAT32LE` mono samples を NeMo offline API へ渡すため、temporary PCM16 mono WAV bridge を backend 境界内で明示的に作る。
- worker bridge は hidden resample / downmix / DSP / semantic correction ではない。別 sample rate、stereo、PCM16 input、non-finite samples、normalized range 外 samples は成功扱いしない。
- output は `plain_text` または `segments_json_v1` として検証し、empty text / invalid segments は fail closed にする。

runtime evidence として、`fluent-audio-runtime`、Torch `2.11.0+cu130`、CUDA available の状態で `ASRModel.restore_from(...)` により Parakeet 1.1B multilingual local `.nemo` を `EncDecRNNTBPEModel` として restore できることを確認しています。

これは non-empty Japanese output の証跡であり、accuracy、backend の本番相当完了、full ROS graph 成功は証明しません。

## Parameter Contract

ROS parameter は宣言した型と一致することを要求する。文字列 `"false"` を bool として扱う、文字列 `"16000"` を integer として扱う、integer `1` を double として扱う、配列要素を `str()` で変換する、といった起動時の暗黙変換は行わない。

`expected_source_id` / `expected_stream_id` は必須であり、受信 `AudioFrame.source_id` / `AudioFrame.stream_id` と完全一致しなければならない。`audio_topic` は ROS transport topic、`expected_stream_id` は frame identity として扱う。topic subscription だけを根拠に、別 source / stream の frame を同じ ASR buffer に混ぜない。

`expected_stream_id` が `audio_topic` / `turn_context_topic` / `asr_result_topic` / `asr_state_topic` / `asr_event_topic` / `control.<id>.topic` と同一、または先頭 `/` の有無だけが違う同一 identity の場合は起動失敗にする。launch の `node_name` / `config_file` は呼び出し側が明示し、launch file は package-local config を推測しない。

`control.inputs` は control input ID の配列です。各 ID は `control.<id>.action`、`control.<id>.topic`、`control.<id>.msg_type`、`control.<id>.source_id`、`control.<id>.stream_id`、`control.<id>.active_field`、`control.<id>.start_field`、`control.<id>.end_field`、`control.<id>.open_on`、`control.<id>.close_on`、`control.<id>.submit_on_close`、`control.<id>.pre_roll_ms`、`control.<id>.post_roll_ms`、`control.<id>.qos.depth`、`control.<id>.qos.reliable` を持つ。現行既定の `speech_control` は `fa_interfaces/msg/VadState` を購読する topic control です。

`control.<id>.source_id` / `stream_id` も同じ `expected_source_id` / `expected_stream_id` と完全一致させる必要がある。identity 不一致の control state は speech activity 更新や window close/finalize に使わない。現行 code は control config と `expected_*` の一致を起動時に強制していないため、profile / system config 側の binding contract として明示する。

型が一致しない場合は node 起動時に失敗する。ASR backend/model の欠落と同じく、誤った system config / profile config を data plane の変換で隠さない。

`target_sample_rate`、`min_audio_sec`、`silence_timeout_sec` は起動時に検証する。`target_sample_rate` は正の整数、`min_audio_sec` と `silence_timeout_sec` は正の有限 double でなければならない。0、負値、NaN、Infinity により timeout や最小発話長の意味を暗黙に変更しない。

`timeline.retention_sec`、`timeline.clock`、`timeline.window_id`、`timeline.window_epoch` は `TranscribeAudio` service が返す `AudioWindowRef` の正本 contract です。profile / system config はこれらを明示し、default window identity に依存しません。`timeline.clock` は `agent` / `robot` / `media` のいずれかでなければならず、ASR 用 rolling timeline は `AudioFrame.header.stamp` から作られる media time range を保持します。

`TranscribeAudio` は `time_range_spec` と `audio_scope` だけを受け取り、自然言語 question を受け取りません。service request の `time_range_spec` は numeric `<start_unix_ns>..<end_unix_ns>` として扱い、相対時刻や marker 解決は MCP adapter / Agent Runtime 側の責務です。`audio_scope` は空、または `expected_stream_id` と完全一致する値だけを受け付けます。

`TranscribeAudio` は ASR-ready `FLOAT32LE` timeline から指定範囲を切り出します。numeric request の start/end が sample boundary と完全一致しない場合は、requested `[start,end)` を完全に覆う最小の sample-boundary `[actual_start,actual_end)` を選択します。sample index は `floor(delta_ns * sample_rate / 1e9)` / `ceil(delta_ns * sample_rate / 1e9)` で決定し、response の整数 ns range は selected start index を `floor(index * 1e9 / sample_rate)`、selected end index を `ceil(index * 1e9 / sample_rate)` で frame start に加算して表現します。48 kHz / 44.1 kHz のように sample 境界が整数 ns で表現できない場合、隣接 frame の append は前 frame の exact end boundary を整数化した `[floor,ceil]` envelope 内の next start を contiguous として扱います。next start が floor より前なら overlap として reject し、ceil より後なら append は許可しますが slice 時に gap として `range_not_continuous` を返します。成功 response の `time_range`、`audio_window_ref.time_range`、transcript segment start/end は requested range ではなく、この actual selected range を返します。backend には selected range に対応する float32 samples だけを渡し、`min_audio_sec` は selected sample count から判定します。`fa_audio_window` の PCM16 archive/export window から hidden decode / format conversion を行いません。SO101 voice frontend のように archive/export window と ASR timeline を併用する場合は、upstream の同じ processed stream から分岐し、同じ `AudioFrame.header.stamp` を保つことで time range を共有します。

`TranscribeAudio` の timeline failure taxonomy は次の通りです。ASR-ready rolling timeline が空で、要求範囲を判定する保持 frame が未蓄積の場合は `window_not_found` を返します。要求範囲が現在の retention window より古い、または未来側へ外れている場合は `range_outside_window` を返します。保持済み frame は存在するが、その frame 群に gap があり selected range を連続 coverage できない場合は `range_not_continuous` を返します。`range_not_continuous` は「空/未蓄積」ではなく「保持済み timeline の coverage 不成立」を表します。

timeline failure 時は空 clip、`fa_audio_window` の archive/export window、または最新 frame への置き換えを行いません。sample-boundary 量子化後の selected range が保持済み連続 audio で覆えない場合は、`window_not_found`、`range_outside_window`、`range_not_continuous` の既存 taxonomy で backend を呼ばず、`success=false` と明示 error code を返します。unsupported `audio_scope` も backend を呼びません。

QoS は data/control/result/observability edge ごとに `audio.qos.depth` / `audio.qos.reliable`、`control.<id>.qos.depth` / `control.<id>.qos.reliable`、`turn_context.qos.depth` / `turn_context.qos.reliable`、`result.qos.depth` / `result.qos.reliable`、`observability.qos.depth` / `observability.qos.reliable` を明示する。depth は正の整数、reliable は bool でなければならない。node 内で topic 名から depth / reliability を推測したり、固定値へ戻したりしない。

## Frame Rejection

`AudioFrame` の data、source/stream identity、layout、encoding、sample rate、channels、bit depth、byte alignment、finite/normalized sample range が契約と一致しない場合、その frame は backend に渡さず破棄する。node 内で resample、downmix、bit-depth conversion、sample format conversion、clip、zero fill は行わない。
