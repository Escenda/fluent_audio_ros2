# fa_asr 仕様書

## 責務

`fa_asr` は発話区間の audio buffer を ASR backend に渡し、文字起こし結果を publish する audio AI node です。

## 非責務

- VAD 推定
- KWS
- turn detection
- device source selection
- ASR backend fallback

## Input / Output

| 種別 | 名前 | 型 |
| --- | --- | --- |
| Sub | `audio_topic` | `fa_interfaces/msg/AudioFrame` |
| Sub | `vad_topic` | `fa_interfaces/msg/VadState` |
| Sub | `turn_context_topic` | `fa_interfaces/msg/TurnContext` |
| Pub | `asr_result_topic` | `fa_interfaces/msg/AsrResult` |

## Backend Contract

- `backend.name` は必須。
- default config は `backend.name` を暗黙指定しない。
- backend は ROS2 topic/message を知らない。
- backend input は mono float samples と sample rate。
- backend output は transcript text。

## Supported Backend Slots

- `local_command`
- `whisper.cpp`
- `parakeet_worker`
- `openai_realtime`
- `openai_transcriptions`

各 backend は個別 class と loader を持ち、`LocalCommandAsrBackend` の名前差し替え alias として扱いません。共通 subprocess 実行は内部 helper に閉じます。

`local_command` と `whisper.cpp` は local process backend として `backend.model_path` を要求します。`parakeet_worker`、`openai_realtime`、`openai_transcriptions` は external worker / process / container backend として `backend.model` を要求します。OpenAI 系 backend はさらに対応する `backend.openai_*.api_key_env` を要求し、その環境変数が空でないことを起動時に検証します。OpenAI SDK/API は `fa_asr` に import せず、worker 側の責務にします。OpenAI worker には configured env の値を `OPENAI_API_KEY` として process environment で渡し、argv / stdout / stderr / ROS topic / log には key 値を渡しません。

未知 backend、backend command 欠落、model / model_path 欠落、invalid working directory は起動失敗にします。
`backend.args` は backend ごとの worker/CLI contract なので default config では空にし、ASR を有効化する profile または system config で必ず明示します。空配列、`{audio}` 欠落、`{model}` 欠落、`{sample_rate}` 欠落は起動失敗です。
`backend.args` の placeholder は `{audio}`, `{model}`, `{language}`, `{output}`, `{sample_rate}` だけを許可します。未知 placeholder、壊れた format string、conversion、format spec は起動失敗です。
`backend.health_args` は startup health check 用の worker/CLI contract です。placeholder は `{model}`, `{language}` だけを許可します。`parakeet_worker`、`openai_realtime`、`openai_transcriptions` は external worker / process / container backend なので `backend.health_args` が必須で、`{model}` 欠落、未知 placeholder、壊れた format string、conversion、format spec は起動失敗です。health check が non-zero exit または timeout の場合も起動失敗です。`local_command` / `whisper.cpp` では package 単体の backend contract としては任意ですが、profile template で ASR を enable 可能にする場合は startup health check を明示するために system config 側へ設定します。指定された場合は同じ検証と startup health check を行います。
`backend.args` が `{output}` を使う場合は `backend.output_text_path` が必須です。`backend.output_text_path` の placeholder は `{audio}`, `{model}`, `{language}`, `{sample_rate}`, `{session_id}`, `{user_turn_id}` だけを許可します。
`backend.command` と `backend.model_path` は起動時に絶対 path へ解決します。command は実行可能 file、model path は読み取り可能 file でなければ起動失敗です。

command 系 backend は backend payload encoding として `float32le_raw` を明示します。これは external worker に渡す一時ファイルの serialization contract であり、`fa_asr` data plane では bit depth / value range 変換、hidden resample、downmix、clipping を行いません。入力 `AsrRequest` は `float32`、1 次元、非空、finite/normalized samples、正の sample rate でなければ backend に渡しません。`backend.args` は `{audio}`、`{model}`、`{sample_rate}` を含める必要があります。

## Parameter Contract

ROS parameter は宣言した型と一致することを要求する。文字列 `"false"` を bool として扱う、文字列 `"16000"` を integer として扱う、integer `1` を double として扱う、配列要素を `str()` で変換する、といった起動時の暗黙変換は行わない。

`expected_source_id` / `expected_stream_id` は必須であり、受信 `AudioFrame.source_id` / `AudioFrame.stream_id` と完全一致しなければならない。`audio_topic` は ROS transport topic、`expected_stream_id` は frame identity として扱う。topic subscription だけを根拠に、別 source / stream の frame を同じ ASR buffer に混ぜない。

`expected_stream_id` が `audio_topic` / `vad_topic` / `turn_context_topic` / `asr_result_topic` と同一、または先頭 `/` の有無だけが違う同一 identity の場合は起動失敗にする。launch の `node_name` / `config_file` は呼び出し側が明示し、launch file は package-local config を推測しない。

`VadState.source_id` / `stream_id` も同じ `expected_source_id` / `expected_stream_id` と完全一致しなければならない。identity 不一致の VAD state は speech activity 更新や VAD end finalize に使わない。

型が一致しない場合は node 起動時に失敗する。ASR backend/model の欠落と同じく、誤った system config / profile config を data plane の変換で隠さない。

`target_sample_rate`、`min_audio_sec`、`silence_timeout_sec` は起動時に検証する。`target_sample_rate` は正の整数、`min_audio_sec` と `silence_timeout_sec` は正の有限 double でなければならない。0、負値、NaN、Infinity により timeout や最小発話長の意味を暗黙に変更しない。

QoS は data/control/result edge ごとに `audio.qos.depth` / `audio.qos.reliable`、`vad.qos.depth` / `vad.qos.reliable`、`turn_context.qos.depth` / `turn_context.qos.reliable`、`result.qos.depth` / `result.qos.reliable` を明示する。depth は正の整数、reliable は bool でなければならない。node 内で topic 名から depth / reliability を推測したり、固定値へ戻したりしない。

## Frame Rejection

`AudioFrame` の data、source/stream identity、layout、encoding、sample rate、channels、bit depth、byte alignment、finite/normalized sample range が契約と一致しない場合、その frame は backend に渡さず破棄する。node 内で resample、downmix、bit-depth conversion、sample format conversion、clip、zero fill は行わない。

## Test IDs

- `FA-ASR-SPEC-001`: backend.name 空で起動失敗
- `FA-ASR-SPEC-002`: unknown backend で起動失敗
- `FA-ASR-SPEC-003`: backend ごとに専用 class / required parameter を使う
- `FA-ASR-SPEC-004`: AudioFrame canonical format validation
- `FA-ASR-SPEC-005`: VAD end で buffer を finalize する
- `FA-ASR-SPEC-006`: backend timeout/error を AsrResult error として publish する
- `FA-ASR-SPEC-007`: default config の `backend.args` は空で、backend ごとに明示しないと起動失敗
- `FA-ASR-SPEC-008`: `encoding != FLOAT32LE` / `channels != 1` / `sample_rate != target_sample_rate` / `bit_depth != 32` を reject し、node 内変換しない
- `FA-ASR-SPEC-009`: normalized range 外または non-finite samples を reject する
- `FA-ASR-SPEC-010`: `openai_transcriptions` は external worker backend として明示選択された場合だけ有効
- `FA-ASR-SPEC-011`: empty `AudioFrame.data` を silent skip せず invalid frame として reject する
- `FA-ASR-SPEC-012`: command/model path は絶対 path へ解決し、command executable / model readable を起動時に検証する
- `FA-ASR-SPEC-013`: `backend.args` / `backend.output_text_path` は許可 placeholder だけを起動時に検証する
- `FA-ASR-SPEC-014`: command backend payload encoding は `float32le_raw` として明示し、`AsrRequest` の dtype / ndim / empty / sample_rate / finite / normalized range を検証する
- `FA-ASR-SPEC-015`: OpenAI backend は `backend.openai_*.api_key_env` と参照先環境変数の存在を起動時に検証し、worker process の `OPENAI_API_KEY` に渡す
- `FA-ASR-SPEC-016`: ROS parameter は型一致を要求し、起動時の `str()` / `bool()` / `int()` / `float()` 変換で誤設定を隠さない
- `FA-ASR-SPEC-017`: `AudioFrame.source_id` が `expected_source_id` と一致しない frame を reject する
- `FA-ASR-SPEC-018`: `AudioFrame.stream_id` が `expected_stream_id` と一致しない frame を reject する
- `FA-ASR-SPEC-019`: `VadState.source_id` / `stream_id` が `expected_source_id` / `expected_stream_id` と一致しない state を reject する
- `FA-ASR-SPEC-020`: `backends/` 配下の全 Python file を走査し、ROS2 topic/message 依存が入らないことを確認する
- `FA-ASR-SPEC-021`: model-id worker backend は `backend.health_args` を必須にし、startup health check failure で起動失敗する
- `FA-ASR-SPEC-022`: launch の `node_name` / `config_file` は暗黙 default を持たない
- `FA-ASR-SPEC-023`: `target_sample_rate` / `min_audio_sec` / `silence_timeout_sec` は正の有限値として起動時に fail closed validation する
- `FA-ASR-SPEC-024`: QoS depth/reliability は topic edge ごとの explicit config / ROS parameter とし、node 内 hardcode や不正値の丸めを行わない
