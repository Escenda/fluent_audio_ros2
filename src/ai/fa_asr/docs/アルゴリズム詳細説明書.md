# fa_asr アルゴリズム詳細説明書

1. node 起動時に `expected_stream_id` が ROS topic 名と衝突しないことを検証する。
2. `AudioFrame` が data を持ち、configured source / stream identity、mono / `FLOAT32LE` / 32-bit / normalized [-1.0, 1.0] / `target_sample_rate` と一致することを検証する。
3. 不一致の frame は backend に渡さず reject する。
4. `VadState.source_id == expected_source_id` かつ `VadState.stream_id == expected_stream_id` であることを検証する。
5. VAD active 中の samples を buffer に追加する。
6. VAD end または timeout で buffer を finalize する。
7. backend に samples と sample rate を渡す。
8. transcript を `AsrResult` として publish する。
9. `TranscribeAudio` request を受けた場合は、numeric `time_range_spec` を parse し、ASR-ready rolling timeline から exact sample boundary で slice する。
10. slice が保持範囲外、gap、sample boundary 不一致、unsupported `audio_scope`、backend timeout/error/empty transcript の場合は `success=false` と明示 error code を返す。
11. 成功時は transcript segment、explicit `timeline.*` parameter 由来の `AudioWindowRef`、backend model ref、resolved time range を返す。

Python version / venv が異なる ASR engine は node 内 import ではなく external worker / process / container backend として扱います。
Resampling / downmix / bit-depth conversion は `fa_asr` では行いません。必要な変換は upstream の format/processing node で明示します。
`source_id` は `expected_source_id` と一致し、`stream_id` は `expected_stream_id` と一致する必要があります。別 source / stream の frame を同じ ASR buffer に混在させません。
VAD state も同じ identity と照合し、別 stream の VAD end で現在の ASR buffer を finalize しません。
command 系 backend は起動時に command/model path と `backend.args` placeholder を検証します。model-id worker backend はさらに `backend.health_args` を必須にし、`{model}` を含む startup health check を通過しない限り backend を起動完了扱いしません。実行時には canonical `AsrRequest` を `float32le_raw` payload として一時 `.f32` に serialize し、external worker に `{audio}`、`{model}`、`{sample_rate}` を渡します。`AsrRequest` が float32 1D、非空、正の sample rate、finite/normalized samples を満たさない場合は worker を起動しません。bit depth / value range 変換はここで行いません。
`TranscribeAudio` service も同じ ASR-ready timeline と backend contract を使います。`fa_audio_window` が提供する PCM16 archive/export window は証拠 clip 用であり、`fa_asr` はそこから WAV decode や PCM16 -> float32 conversion を行いません。両者を同じ time range で参照する場合は、system profile で同じ upstream stream から分岐し、`AudioFrame.header.stamp` を保存することで time alignment を成立させます。
