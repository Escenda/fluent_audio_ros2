# fa_asr アルゴリズム詳細説明書

1. `AudioFrame` が data を持ち、configured source / stream identity、mono / `FLOAT32LE` / 32-bit / normalized [-1.0, 1.0] / `target_sample_rate` と一致することを検証する。
2. 不一致の frame は backend に渡さず reject する。
3. `VadState.source_id == expected_source_id` かつ `VadState.stream_id == expected_stream_id` であることを検証する。
4. VAD active 中の samples を buffer に追加する。
5. VAD end または timeout で buffer を finalize する。
6. backend に samples と sample rate を渡す。
7. transcript を `AsrResult` として publish する。

Python version / venv が異なる ASR engine は node 内 import ではなく external worker / process / container backend として扱います。
Resampling / downmix / bit-depth conversion は `fa_asr` では行いません。必要な変換は upstream の format/processing node で明示します。
`source_id` は `expected_source_id` と一致し、`stream_id` は `expected_stream_id` と一致する必要があります。別 source / stream の frame を同じ ASR buffer に混在させません。
VAD state も同じ identity と照合し、別 stream の VAD end で現在の ASR buffer を finalize しません。
command 系 backend は起動時に command/model path と `backend.args` placeholder を検証します。model-id worker backend はさらに `backend.health_args` を必須にし、`{model}` を含む startup health check を通過しない限り backend を起動完了扱いしません。実行時には canonical `AsrRequest` を `float32le_raw` payload として一時 `.f32` に serialize し、external worker に `{audio}`、`{model}`、`{sample_rate}` を渡します。`AsrRequest` が float32 1D、非空、正の sample rate、finite/normalized samples を満たさない場合は worker を起動しません。bit depth / value range 変換はここで行いません。
