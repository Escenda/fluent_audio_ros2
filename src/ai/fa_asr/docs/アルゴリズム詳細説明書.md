# fa_asr アルゴリズム詳細説明書

1. `AudioFrame` が data を持ち、mono / float32 / normalized [-1.0, 1.0] / `target_sample_rate` と一致することを検証する。
2. 不一致の frame は backend に渡さず reject する。
3. VAD active 中の samples を buffer に追加する。
4. VAD end または timeout で buffer を finalize する。
5. backend に samples と sample rate を渡す。
6. transcript を `AsrResult` として publish する。

Python version / venv が異なる ASR engine は node 内 import ではなく external worker / process / container backend として扱います。
Resampling / downmix / bit-depth conversion は `fa_asr` では行いません。必要な変換は upstream の format/processing node で明示します。
command 系 backend は起動時に command/model path と `backend.args` placeholder を検証します。実行時には canonical `AsrRequest` を `pcm16_wav` payload として一時 WAV に serialize し、external worker に `{audio}` と `{model}` を渡します。`AsrRequest` が float32 1D、非空、正の sample rate、finite/normalized samples を満たさない場合は worker を起動しません。
