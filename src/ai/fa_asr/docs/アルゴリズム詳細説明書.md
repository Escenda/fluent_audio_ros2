# fa_asr アルゴリズム詳細説明書

## Active Objective

`fa_asr` の現行標準 ASR backend は `parakeet_multilingual_buffered` です。
この backend は multilingual Parakeet 1.1B `.nemo` を `fa_asr` backend/session 内部の runner として保持し、16 kHz mono `FLOAT32LE` の ASR-ready stream だけを受け付けます。

unsupported model、Whisper、OpenAI、NIM、Riva、gRPC、stdin/stdout JSONL worker は標準経路、default、参照実装、検証代替ではありません。
これらを使う場合は明示的に選択された optional / legacy backend として扱い、`parakeet_multilingual_buffered` の失敗時 fallback にはしません。

## Node Algorithm

1. node 起動時に `backend.name`、source / stream identity、QoS、timeline、control、backend 固有 parameter を型一致で検証する。
2. `backend.name=parakeet_multilingual_buffered` の場合、`backend.model` または `backend.model_path` のどちらか一方だけを受け付ける。
3. model id / path が multilingual Parakeet 1.1B を識別することを検証し、multilingual Parakeet 1.1B ではない model を拒否する。
4. `backend.language` は空文字、`backend.language_policy` は `auto_detect` でなければ起動失敗にする。言語を保証できない場合に固定 language を推測して入れず、auto detect policy を contract とする。
5. `AudioFrame` が data を持ち、configured source / stream identity、mono、`FLOAT32LE`、32-bit、interleaved、16 kHz、finite normalized range と一致することを検証する。
6. 不一致の frame は backend state に触れる前に reject する。node 内で resample、downmix、PCM16 変換、clamp、normalize、denoise、zero fill は行わない。
7. control input は `source_id` / `stream_id` を照合し、一致する VAD / TurnDetector / timeout trigger だけを ASR stream の open / close に使う。
8. stream open 中は accepted samples を backend session の rolling buffer に追加する。
9. configured chunk 境界で retained rolling buffer 全体を再 decode する。
10. partial text は `STATUS_PARTIAL` の未確定 hypothesis として publish できるが、conversation history や committed transcript として扱わない。
11. VAD / TurnDetector / timeout trigger で stream が閉じたら `finish()` を呼び、final decode を実行する。
12. `finish()` の final result だけを committed transcript として扱う。
13. speech energy が十分ある buffer で final transcript が空の場合は ASR 成功扱いにせず、明示 error / fail-closed とする。
14. backend runtime error、invalid final、empty speech final は別 backend へ fallback せず、error result と node fail-closed shutdown の対象にする。
15. `TranscribeAudio` request は ASR-ready rolling timeline から numeric time range を sample-boundary 量子化して切り出す。timeline failure、unsupported scope、gap では backend を呼ばず、latest frame や `fa_audio_window` archive/export へ fallback しない。

## Full-Context Streaming Policy

multilingual Parakeet 1.1B `.nemo` は、現行 FluentAudio では cache-aware streaming model としてではなく full-context model として扱います。
低遅延 streaming 成功を装うために cache-aware backend へ fallback しません。

現行 policy は次です。

1. ASR-ready samples を rolling buffer に保持する。
2. chunk boundary ごとに retained buffer を full-context re-decode する。
3. 直前 partial と異なる text が得られた場合だけ uncommitted partial として publish する。
4. partial を final transcript へ昇格しない。
5. VAD / TurnDetector / timeout close で `finish()` を呼ぶ。
6. `finish()` の final decode result だけを commit する。
7. final empty text は、buffered speech energy が threshold 以上なら failure とする。

この方式は「cache-aware streaming に失敗した場合の代替」ではありません。
full-context multilingual Parakeet を streaming ASR として使うための明示 backend policy です。

## Input Contract

`parakeet_multilingual_buffered` が受け付ける audio は次だけです。

| field | required |
| --- | --- |
| encoding | `FLOAT32LE` |
| sample rate | `16000` |
| channels | `1` |
| bit depth | `32` |
| layout | interleaved mono |
| samples | non-empty, finite, normalized |

この contract 外の input は「変換できなかった入力」ではなく「backend capability 外の入力」です。
必要な変換は upstream の `fa_sample_format`、`fa_resample`、`fa_channel_convert`、correction / dynamics node で明示的に行います。

## Language Policy

ASR language を上流 profile が保証できない場合、`backend.language` を固定値にしません。
標準 backend は `backend.language=""` と `backend.language_policy=auto_detect` を要求します。

固定 language を要求する backend は、現行標準経路ではありません。
将来固定 language backend を使う場合は、model capability、profile contract、検証 fixture、failure condition を別 backend doc に分離します。

## Fail-Closed Conditions

次の場合は成功扱いにしません。

- `backend.model` と `backend.model_path` が両方空、または両方設定されている。
- model id / path が multilingual Parakeet 1.1B を識別しない。
- model id / path が multilingual Parakeet 1.1B ではない model を指す。
- `backend.language_policy` が `auto_detect` ではない。
- `backend.language` が空文字ではない。
- sample rate が 16 kHz ではない。
- channel count が 1 ではない。
- pushed payload が `FLOAT32LE` / finite normalized samples ではない。
- rolling buffer / chunk / energy threshold config が不正。
- stream close 時に buffered speech energy が十分あるのに final transcript が空。
- backend runner restore / decode / finish が error を返す。

fail-closed 時に unsupported model、Whisper、OpenAI、NIM、Riva、gRPC、legacy JSONL worker へ fallback しません。

## Legacy Optional Backends

以下は残っている documentation / code path がある場合でも、現行標準 ASR 経路ではありません。
明示的に選択された optional / legacy backend としてだけ扱います。

| backend/path | position |
| --- | --- |
| `nemo_rnnt_streaming` | legacy experimental local cache-aware streaming slot。現行 Parakeet full-context standard の成功条件ではない。 |
| `nemo_offline_transcribe` | legacy optional non-streaming command / worker backend。過去の file-source evidence はあるが、標準 streaming policy ではない。 |
| `whisper.cpp` | legacy optional external worker backend。標準 / fallback ではない。 |
| `parakeet_worker` | legacy optional external worker backend。標準 / fallback ではない。 |
| `local_command` | optional command backend abstraction。標準 / fallback ではない。 |
| `openai_realtime` / `openai_transcriptions` | optional external API worker slots。標準 / fallback ではない。 |
| NIM / Riva / gRPC | external serving boundary。現行標準 backend の内部実装や validation substitute ではない。 |

legacy path の docs は、historical evidence、failure phase、または optional backend contract を残すためのものです。
それらの成功・失敗を `parakeet_multilingual_buffered` の完了判定へ混ぜません。
