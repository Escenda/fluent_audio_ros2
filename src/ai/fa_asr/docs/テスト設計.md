# fa_asr テスト設計

## 目的

`fa_asr` のテストは、現行標準 ASR backend `parakeet_multilingual_buffered` の contract を実行経路で証明するためのものです。
Markdown、source text、package metadata の文字列検査では代替しません。

unsupported model、Whisper、OpenAI、NIM、Riva、gRPC、stdin/stdout JSONL worker は標準経路、default、参照実装、fallback、validation substitute ではありません。
それらの test が残る場合は legacy optional backend の contract test として扱います。

## Standard Backend Test IDs

- `FA-ASR-SPEC-001`: `backend.name` 空で起動失敗。
- `FA-ASR-SPEC-002`: unknown backend で起動失敗。
- `FA-ASR-SPEC-003`: backend ごとに専用 class / required parameter を使う。
- `FA-ASR-SPEC-004`: AudioFrame canonical format validation。
- `FA-ASR-SPEC-005`: VAD / TurnDetector / timeout close で ASR stream を finalize する。
- `FA-ASR-SPEC-006`: backend timeout/error を `AsrResult` error として publish し、node を fail closed shutdown する。
- `FA-ASR-SPEC-007`: default config は backend / worker / model / command result format を暗黙選択しない。
- `FA-ASR-SPEC-008`: `encoding != FLOAT32LE` / `channels != 1` / `sample_rate != 16000` / `bit_depth != 32` を reject し、node 内変換しない。
- `FA-ASR-SPEC-009`: normalized range 外または non-finite samples を reject する。
- `FA-ASR-SPEC-010`: optional external API backend は明示選択された場合だけ有効で、標準 backend の fallback ではない。
- `FA-ASR-SPEC-011`: empty `AudioFrame.data` を silent skip せず invalid frame として reject する。
- `FA-ASR-SPEC-012`: command/model path は絶対 path へ解決し、command executable / model readable を起動時に検証する。
- `FA-ASR-SPEC-013`: command backend の placeholder は許可 list だけを起動時に検証する。
- `FA-ASR-SPEC-014`: command backend payload encoding は `float32le_raw` として明示し、hidden resample / downmix / PCM16 変換を行わない。
- `FA-ASR-SPEC-015`: optional OpenAI backend は API key env を起動時に検証し、標準 backend の fallback ではない。
- `FA-ASR-SPEC-016`: ROS parameter は型一致を要求し、起動時の `str()` / `bool()` / `int()` / `float()` 変換で誤設定を隠さない。
- `FA-ASR-SPEC-017`: `AudioFrame.source_id` が `expected_source_id` と一致しない frame を reject する。
- `FA-ASR-SPEC-018`: `AudioFrame.stream_id` が `expected_stream_id` と一致しない frame を reject する。
- `FA-ASR-SPEC-019`: control message の `source_id` / `stream_id` が expected identity と一致しない state を reject する。
- `FA-ASR-SPEC-020`: backend public API は ROS2 topic/message を要求せず、ASR request と明示 payload encoding だけで実行できる。
- `FA-ASR-SPEC-021`: model-id worker backend は startup health check failure で起動失敗する。
- `FA-ASR-SPEC-022`: launch の `node_name` / `config_file` は暗黙 default を持たない。
- `FA-ASR-SPEC-023`: `target_sample_rate` / `min_audio_sec` / `silence_timeout_sec` は正の有限値として起動時に fail closed validation する。
- `FA-ASR-SPEC-024`: QoS depth/reliability は topic edge ごとの explicit config / ROS parameter とし、node 内 hardcode や不正値の丸めを行わない。
- `FA-ASR-SPEC-025`: `TranscribeAudio` は numeric time range を ASR-ready timeline から sample-boundary 量子化して切り出す。
- `FA-ASR-SPEC-026`: `TranscribeAudio` は `fa_audio_window` の PCM16 archive/export window から hidden conversion しない。
- `FA-ASR-SPEC-027`: `TranscribeAudio` は `window_not_found` / `range_outside_window` / `range_not_continuous` を区別し、timeline gap failure では backend を呼ばない。
- `FA-ASR-SPEC-028`: non-streaming command backend の `backend.result_format` は `plain_text` / `segments_json_v1` の明示値だけを受け付ける。
- `FA-ASR-SPEC-029`: `segments_json_v1` は strict JSON/schema/range/text validation に失敗した場合 fail closed する。
- `FA-ASR-SPEC-030`: `TranscribeAudio` は backend segment の相対 sample offset を selected range 起点の absolute unix ns へ floor start / ceil end で変換する。
- `FA-ASR-SPEC-031`: `parakeet_multilingual_buffered` は multilingual Parakeet 1.1B runner を backend/session 内部に保持し、外部 ASR worker command を要求しない。
- `FA-ASR-SPEC-032`: `parakeet_multilingual_buffered` は 16 kHz mono `FLOAT32LE` 以外を拒否し、node/backend 内で hidden resample / downmix / PCM16 変換を行わない。
- `FA-ASR-SPEC-033`: `parakeet_multilingual_buffered` は rolling buffer / chunked full-context re-decode を行う。
- `FA-ASR-SPEC-034`: partial は未確定 hypothesis であり、final transcript として commit しない。
- `FA-ASR-SPEC-035`: VAD / TurnDetector / timeout close の `finish()` で得た final だけを committed transcript とする。
- `FA-ASR-SPEC-036`: `backend.language=""` と `backend.language_policy=auto_detect` を要求し、固定 language を暗黙補完しない。
- `FA-ASR-SPEC-037`: multilingual Parakeet 1.1B ではない model を拒否する。
- `FA-ASR-SPEC-038`: speech energy が十分ある final empty transcript を成功扱いしない。
- `FA-ASR-SPEC-039`: `parakeet_multilingual_buffered` failure 時に Whisper、OpenAI、NIM、Riva、gRPC、legacy worker へ fallback しない。

## Integration Test Requirements

standard ASR path の representative integration では、最低限次を確認します。

- upstream pipeline が 16 kHz mono `FLOAT32LE` を生成している。
- `AudioFrame.source_id` / `stream_id` が `fa_asr` の expected identity と一致する。
- VAD / TurnDetector / timeout close が ASR stream の `finish()` を発火する。
- partial は未確定 result として publish され、final と混同されない。
- final result は non-empty text を持つ。
- speech energy が十分ある empty final は failure になる。
- ASR worker command、Whisper、OpenAI、NIM、Riva、gRPC へ fallback していない。

## Colcon / Pytest

- package-local unit / integration / launch contract test は `colcon test --packages-select fa_asr` または package-local pytest で実行する。
- real model / real ROS graph smoke は opt-in 環境変数と local model path を要求する。
- docs 更新だけでは test 実行済みや backend 実装完了を意味しない。
