# fa_asr テスト設計

## Unit Test

- `FA-ASR-SPEC-001`: backend.name validation。
- `FA-ASR-SPEC-002`: unknown backend validation。
- `FA-ASR-SPEC-003`: backend ごとの required parameter と class 分離。
- `FA-ASR-SPEC-004`: AudioFrame canonical format validation。
- `FA-ASR-SPEC-005`: VAD end finalize。
- `FA-ASR-SPEC-006`: backend timeout/error mapping。
- `FA-ASR-SPEC-007`: default config の `backend.args` は空で、backend ごとに明示しないと起動失敗。
- `FA-ASR-SPEC-008`: `channels != 1` / `sample_rate != target_sample_rate` / `bit_depth != 32` を reject し、node 内変換しない。
- `FA-ASR-SPEC-009`: normalized range 外または non-finite samples を reject する。
- `FA-ASR-SPEC-010`: `openai_transcriptions` は external worker backend として明示選択された場合だけ有効。
- `FA-ASR-SPEC-011`: empty `AudioFrame.data` を silent skip せず invalid frame として reject する。

## Integration Test

- `test/integration/test_asr_external_worker_pipeline.py`
  - `FA-ASR-TC-010`: fixture PCM を local command mock に渡し、stdout transcript を読む。
  - `FA-ASR-TC-011`: output file contract を使う backend でも transcript を読む。
  - `FA-ASR-TC-012`: 空 transcript は成功扱いせず `RuntimeError` にする。

## Launch Test

- `test/launch/test_fa_asr_launch_contract.py`
  - `FA-ASR-TC-020`: launch は `config_file` を明示的に node parameters へ渡す。
  - `FA-ASR-TC-021`: default config は backend / worker / model を暗黙選択しない。
- unknown backend で起動失敗することは `test/unit/test_backend_config.py` で確認する。

## Colcon Test

- `CMakeLists.txt` で `ament_cmake_pytest` に `test/` を登録し、`colcon test --packages-select fa_asr` で unit / integration / launch contract test を実行する。
- test 実行時は ambient pytest plugin を無効化し、host/container 側の無関係な pytest plugin によって ASR 契約テストが skip されないことを保証する。

## Fixtures

- `test/fixtures/fake_asr_worker.py`: mono PCM16 WAV を読み、model file の文字列を stdout または output file へ返す external worker mock。
