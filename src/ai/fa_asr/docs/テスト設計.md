# fa_asr テスト設計

## Unit Test

- `FA-ASR-SPEC-001`: backend.name validation。
- `FA-ASR-SPEC-002`: unknown backend validation。
- `FA-ASR-SPEC-003`: backend ごとの required parameter と class 分離。
- `FA-ASR-SPEC-004`: AudioFrame canonical format validation。
- `FA-ASR-SPEC-005`: VAD end finalize。
- `FA-ASR-SPEC-006`: backend timeout/error mapping と fail closed shutdown。
- `FA-ASR-SPEC-007`: default config の `backend.args` / `backend.result_format` は空で、backend ごとに明示しないと起動失敗。
- `FA-ASR-SPEC-008`: `encoding != FLOAT32LE` / `channels != 1` / `sample_rate != target_sample_rate` / `bit_depth != 32` を reject し、node 内変換しない。
- `FA-ASR-SPEC-009`: normalized range 外または non-finite samples を reject する。
- `FA-ASR-SPEC-010`: `openai_transcriptions` は external worker backend として明示選択された場合だけ有効。
- `FA-ASR-SPEC-011`: empty `AudioFrame.data` を silent skip せず invalid frame として reject する。
- `FA-ASR-SPEC-012`: command/model path は起動時に絶対 path と実行/読み取り権限を検証する。
- `FA-ASR-SPEC-013`: `backend.args` / `backend.output_text_path` は許可 placeholder のみを起動時に検証する。
- `FA-ASR-SPEC-014`: command backend payload encoding は `float32le_raw` として明示し、`AsrRequest` dtype / ndim / empty / sample_rate / finite / normalized range を検証する。
- `FA-ASR-SPEC-015`: OpenAI backend は `backend.openai_*.api_key_env` と参照先環境変数の存在を起動時に検証し、worker process の `OPENAI_API_KEY` に渡す。
- `FA-ASR-SPEC-016`: ROS parameter は型一致を要求し、起動時の `str()` / `bool()` / `int()` / `float()` 変換で誤設定を隠さない。
- `FA-ASR-SPEC-017`: `AudioFrame.source_id` が `expected_source_id` と一致しない frame を reject する。
- `FA-ASR-SPEC-018`: `AudioFrame.stream_id` が `expected_stream_id` と一致しない frame を reject する。
- `FA-ASR-SPEC-019`: `VadState.source_id` / `stream_id` が `expected_source_id` / `expected_stream_id` と一致しない state を reject する。
- `FA-ASR-SPEC-020`: backend public API と external worker protocol は ROS2 topic/message を要求せず、`AsrRequest` と明示 payload encoding だけで実行できる。
- `FA-ASR-SPEC-021`: model-id worker backend は `backend.health_args` を必須にし、startup health check failure で起動失敗する。
- `FA-ASR-SPEC-022`: launch の `node_name` / `config_file` は暗黙 default を持たず、package-local config を推測しない。
- `FA-ASR-SPEC-023`: `target_sample_rate` / `min_audio_sec` / `silence_timeout_sec` は正の有限値として起動時に fail closed validation する。
- `FA-ASR-SPEC-024`: QoS depth/reliability は topic edge ごとの explicit config / ROS parameter とし、node 内 hardcode や不正値の丸めを行わない。
- `FA-ASR-SPEC-025`: `TranscribeAudio` は numeric time range を ASR-ready timeline から sample-boundary 量子化して切り出し、`AudioWindowRef` は explicit `timeline.*` parameter と actual selected range から作る。
- `FA-ASR-SPEC-026`: `TranscribeAudio` は `fa_audio_window` の PCM16 archive/export window から hidden conversion しない。
- `FA-ASR-SPEC-027`: `TranscribeAudio` は `window_not_found` / `range_outside_window` / `range_not_continuous` を区別し、timeline gap failure では backend を呼ばない。
- `FA-ASR-SPEC-028`: `backend.result_format` は `plain_text` / `segments_json_v1` の明示値だけを受け付ける。
- `FA-ASR-SPEC-029`: `segments_json_v1` は strict JSON/schema/range/text validation に失敗した場合 fail closed する。
- `FA-ASR-SPEC-030`: `TranscribeAudio` は backend segment の相対 sample offset を selected range 起点の absolute unix ns へ floor start / ceil end で変換する。

## Integration Test

- `test/integration/test_asr_external_worker_pipeline.py`
- `FA-ASR-TC-011`: fixture float32 payload を local command mock に渡し、stdout の `plain_text` result を読む。
- `FA-ASR-TC-012`: output file contract を使う backend でも `plain_text` result を読む。
- `FA-ASR-TC-013`: 空 `plain_text` transcript は成功扱いせず `RuntimeError` にする。
- `FA-ASR-TC-018`: `segments_json_v1` result を複数 segment として読み、invalid JSON/schema/range/text を fail closed にする。

## Launch Test

- `test/launch/test_fa_asr_launch_contract.py`
  - `FA-ASR-TC-020`: launch は `node_name` と `config_file` を明示的に受け取り、`config_file` だけを node parameters へ渡す。
  - `FA-ASR-TC-021`: default config は backend / worker / model / result format を暗黙選択しない。
- unknown backend で起動失敗することは `test/unit/test_backend_config.py` で確認する。

## Colcon Test

- `colcon test --packages-select fa_asr` または package-local pytest で unit / integration / launch contract test を実行する。
- test 実行時は ambient pytest plugin を無効化し、host/container 側の無関係な pytest plugin によって ASR 契約テストが skip されないことを保証する。

## Fixtures

- `test/fixtures/fake_asr_worker.py`: mono raw float32le payload を読み、configured result format に従う結果を stdout または output file へ返す external worker mock。
