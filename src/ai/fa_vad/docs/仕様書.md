# fa_vad 仕様書

## 責務

`fa_vad` は PCM frame から発話区間を推定する audio AI node です。backend を明示選択し、VAD probability と speech start/end state を publish します。

## 非責務

- audio source selection
- resample node としての再配信
- KWS / ASR / turn detection
- device fallback
- online model download fallback

## Input / Output

| 種別 | 名前 | 型 |
| --- | --- | --- |
| Sub | `input_topic` | `fa_interfaces/msg/AudioFrame` |
| Pub | `output_topic` | `std_msgs/msg/Bool` |
| Pub | `vad_state_topic` | `fa_interfaces/msg/VadState` |
| Pub | `probability_topic` | `std_msgs/msg/Float32` |

## Backend Contract

- `backend.name` は必須。
- `backend.model_path` は必須。空値から torch hub cache や online source を推測しない。
- `backend.execution_provider` は必須。
- unknown backend は起動失敗。
- backend は ROS2 topic/message を知らない。
- backend input は `Float32MonoWindow(sample_rate, data)`。`sample_rate` は 8kHz / 16kHz、`data` は non-empty、float32 byte aligned、finite、normalized [-1.0, 1.0] の mono raw float32le window。
- backend output は probability, is_speech, start, end、または window 未満時の explicit no-decision。
- `silero` backend は external process boundary として実行する。ROS2 node process 内で PyTorch / Silero を import しない。
- `backend.command` は必須。command は probability float を stdout へ出力する。

## Fail Closed

- `backend.name` が空
- backend が未対応
- model path が backend 条件を満たさない
- execution provider が未指定または未対応
- `backend.command` が未指定または実行不能
- `backend.args` が ROS string array parameter として宣言されていない、または非 string 値を `str()` で丸めようとする
- `backend.args` に required placeholder がない
- `backend.args` に unknown placeholder / malformed format / conversion / format spec がある
- `target_sample_rate <= 0`
- `silero` backend の sample rate が `8000` / `16000` 以外
- `Float32MonoWindow` の sample rate / byte alignment / finite / normalized range が契約外
- `expected_source_id` が空
- `threshold_start` / `threshold_end` が `[0.0, 1.0]` の範囲外
- `threshold_end > threshold_start`
- `hangover_ms <= 0`
- `hangover_ms < frame_ms`
- `hangover_ms` が `frame_ms` で割り切れない
- `qos.depth <= 0`
- external command が timeout / non-zero exit / invalid probability を返す
- external command が NaN / inf probability を返す
- external worker が 8kHz / 16kHz 以外の sample rate で起動される
- backend runtime failure
- ROS parameter の型が宣言と一致しない。たとえば bool parameter に string `"false"`、integer parameter に string `"16000"`、double parameter に integer `1` を渡した場合は起動失敗にする

## Frame Rejection

`AudioFrame` の data、source/stream identity、layout、sample rate、channels、bit depth、byte alignment、finite/normalized sample range が契約と一致しない場合、その frame は publish せず破棄する。`source_id` は `expected_source_id`、`stream_id` は `input_topic` と一致しなければならない。topic subscription だけを根拠に別 source / stream の frame を受け入れない。node 内で resample、downmix、clip、zero fill は行わない。

- `FA-VAD-SPEC-006`: `backend.execution_provider` missing / unknown で起動失敗
- `FA-VAD-SPEC-007`: `sample_rate != target_sample_rate` を reject し、node 内 resample しない
- `FA-VAD-SPEC-008`: normalized range 外または non-finite samples を reject する
- `FA-VAD-SPEC-009`: `backend.command` missing / not executable で起動失敗
- `FA-VAD-SPEC-010`: `backend.args` に `{audio}`, `{model}`, `{provider}`, `{sample_rate}` がなければ起動失敗
- `FA-VAD-SPEC-011`: backend module は ROS2 node process 内で `torch` を import せず external process boundary を使う
- `FA-VAD-SPEC-012`: fake external worker で raw float32le / stdout probability contract を検証する
- `FA-VAD-SPEC-013`: `qos.depth <= 0` を暗黙補正せず起動失敗
- `FA-VAD-SPEC-014`: invalid threshold / hangover / sample rate は backend 側でも起動失敗
- `FA-VAD-SPEC-015`: backend constructor は sample rate / frame / threshold / hangover の暗黙 default を持たない
- `FA-VAD-SPEC-016`: empty `AudioFrame.data` を silent skip せず invalid frame として reject する
- `FA-VAD-SPEC-017`: backend runtime failure で node を fail closed する
- `FA-VAD-SPEC-018`: backend input は `Float32MonoWindow` として sample rate / float32 alignment / finite / normalized range を検証する
- `FA-VAD-SPEC-019`: backend window が必要 sample 数未満の場合は no-decision として publish しない
- `FA-VAD-SPEC-020`: external worker は 8kHz / 16kHz 以外の sample rate を reject する
- `FA-VAD-SPEC-021`: `backend.args` は ROS string array parameter として宣言し、非 string 値を `str()` で丸めない
- `FA-VAD-SPEC-022`: worker stdout probability が NaN / inf の場合は fail closed する
- `FA-VAD-SPEC-023`: startup parameter は ROS parameter type を検証し、`bool()` / `int()` / `float()` / `str()` cast で設定ミスを丸めない
- `FA-VAD-SPEC-024`: `AudioFrame.stream_id` が `input_topic` と一致しない frame を reject する
- `FA-VAD-SPEC-025`: rclpy logger は printf-style 引数に依存せず、実起動時に logger call で落ちない
- `FA-VAD-SPEC-026`: `AudioFrame.source_id` が `expected_source_id` と一致しない frame を reject する
