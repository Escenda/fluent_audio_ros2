# fa_vad 仕様書

## 責務

`fa_vad` は PCM frame から発話区間を推定する audio AI node です。backend を明示選択し、VAD probability と speech start/end state を publish します。

## 非責務

- audio source selection
- resample node としての再配信
- KWS / ASR / turn detection
- device fallback
- online model download fallback

## Input / Output

| 種別 | 名前 | 型 |
| --- | --- | --- |
| Sub | `input_topic` | `fa_interfaces/msg/AudioFrame` |
| Pub | `output_topic` | `std_msgs/msg/Bool` |
| Pub | `vad_state_topic` | `fa_interfaces/msg/VadState` |
| Pub | `probability_topic` | `std_msgs/msg/Float32` |

## Backend Contract

- `backend.name` は必須。
- `backend.model_path` は必須。空値から torch hub cache や online source を推測しない。
- `backend.execution_provider` は必須。
- unknown backend は起動失敗。
- backend は ROS2 topic/message を知らない。
- backend input は mono PCM16 bytes。
- backend output は probability, is_speech, start, end。
- `silero` backend は external process boundary として実行する。ROS2 node process 内で PyTorch / Silero を import しない。
- `backend.command` は必須。command は probability float を stdout へ出力する。

## Fail Closed

- `backend.name` が空
- backend が未対応
- model path が backend 条件を満たさない
- execution provider が未指定または未対応
- `backend.command` が未指定または実行不能
- `backend.args` に required placeholder がない
- `target_sample_rate <= 0`
- `silero` backend の sample rate が `8000` / `16000` 以外
- `threshold_start` / `threshold_end` が `[0.0, 1.0]` の範囲外
- `threshold_end > threshold_start`
- `hangover_ms <= 0`
- `hangover_ms < frame_ms`
- `hangover_ms` が `frame_ms` で割り切れない
- `qos.depth <= 0`
- external command が timeout / non-zero exit / invalid probability を返す

## Frame Rejection

`AudioFrame` の data、source/stream identity、layout、sample rate、channels、bit depth、byte alignment、finite/normalized sample range が契約と一致しない場合、その frame は publish せず破棄する。node 内で resample、downmix、clip、zero fill は行わない。

## Test IDs

- `FA-VAD-SPEC-001`: backend.name 空で起動失敗
- `FA-VAD-SPEC-002`: unknown backend で起動失敗
- `FA-VAD-SPEC-003`: invalid AudioFrame を publish しない
- `FA-VAD-SPEC-004`: backend result を VadState に変換する
- `FA-VAD-SPEC-005`: invalid runtime config は暗黙補正せず起動失敗する
