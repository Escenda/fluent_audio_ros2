# fa_vad アルゴリズム詳細説明書

1. node startup で `target_sample_rate`、threshold、hangover、QoS、backend config を検証する。
2. `backends/factory.py` が `backend.name` を検証し、対応する `VADBackend` adapter を生成する。
3. `AudioFrame` が `source_id == expected_source_id` / `stream_id == input_stream_id` / mono / float32 / normalized [-1.0, 1.0] / `target_sample_rate` と一致することを検証する。
4. 不一致の frame は backend に渡さず reject する。
5. `AudioFrame.data` の raw float32le bytes を `Float32MonoWindow(sample_rate, data)` として包み、sample rate / non-empty / float32 byte alignment / finite / normalized range を検証する。
6. backend `update()` に `Float32MonoWindow` を渡す。
7. `silero` backend は `backend.window_samples` と `backend.history_buffer_ms` を検証し、明示された rolling buffer から model input window を切り出す。buffer が `backend.window_samples` 未満の場合は explicit no-decision を返し、node は publish しない。
8. `silero` backend は window を一時 raw float32le `.f32` file に書き、`backend.command` を external process として実行する。
9. external process の stdout から probability float を読む。
10. probability / is_speech / start / end と、入力 `AudioFrame` の `source_id` / `stream_id` を `VadState` として publish する。
11. speech state が変化した場合のみ Bool topic を publish する。

Resampling / downmix / bit-depth conversion は `fa_vad` では行いません。必要な変換は upstream の format/processing node で明示します。
`input_topic` は ROS transport の接続点、`input_stream_id` は frame 内 identity として扱います。`source_id` と `stream_id` は `expected_source_id` / `input_stream_id` と一致しない frame は backend に渡しません。`input_stream_id` が ROS topic 名と同一視できる値の場合は起動失敗にします。
`VadState` の identity は VAD 判定元 frame の identity であり、後段の KWS / ASR / Turn Detector が別 stream の VAD state を reject するための契約です。
Silero backend は `backend.model_path` の local torch.hub repository だけを external worker に渡し、online source や torch cache path を推測しません。`backend.model_path` は起動時に実体解決し、directory として readable / executable であること、かつ `hubconf.py` を持つことを検証します。path 指定の `backend.command` も起動時に実体解決し、存在しない path / 実行不能 command は起動失敗にします。推論 device は `backend.execution_provider` として external worker へ明示的に渡します。ROS2 node process 内で PyTorch を import しません。
`qos.depth`、threshold、hangover、sample rate、window samples、history buffer duration は暗黙補正しません。不正値は起動失敗または backend 初期化失敗として扱います。`silero` backend は 8kHz / 16kHz のみ受け付け、`hangover_ms` は `frame_ms` 以上かつ `frame_ms` で割り切れる値だけを受け付けます。
`backend.args` は `{audio}`, `{model}`, `{provider}`, `{sample_rate}`, `{window_samples}` の required placeholder だけを使えます。unknown placeholder、malformed format、conversion、format spec は起動時に reject します。
