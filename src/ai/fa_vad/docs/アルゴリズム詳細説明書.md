# fa_vad アルゴリズム詳細説明書

1. node startup で `target_sample_rate`、threshold、hangover、QoS、backend config を検証する。
2. `AudioFrame` が mono / float32 / normalized [-1.0, 1.0] / `target_sample_rate` と一致することを検証する。
3. 不一致の frame は backend に渡さず reject する。
4. backend adapter 用に float samples を PCM16 bytes へ serialize する。
5. backend `update()` に PCM16 bytes を渡す。
6. `silero` backend は window を一時 WAV file に書き、`backend.command` を external process として実行する。
7. external process の stdout から probability float を読む。
8. probability / is_speech / start / end を `VadState` として publish する。
9. speech state が変化した場合のみ Bool topic を publish する。

Resampling / downmix / bit-depth conversion は `fa_vad` では行いません。必要な変換は upstream の format/processing node で明示します。
Silero backend は `backend.model_path` の local torch.hub repository だけを external worker に渡し、online source や torch cache path を推測しません。推論 device は `backend.execution_provider` として external worker へ明示的に渡します。ROS2 node process 内で PyTorch を import しません。
`qos.depth`、threshold、hangover、sample rate は暗黙補正しません。不正値は起動失敗または backend 初期化失敗として扱います。`silero` backend は 8kHz / 16kHz のみ受け付け、`hangover_ms` は `frame_ms` 以上かつ `frame_ms` で割り切れる値だけを受け付けます。
