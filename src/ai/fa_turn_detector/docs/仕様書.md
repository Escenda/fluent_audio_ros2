# fa_turn_detector 仕様書

## 責務

`fa_turn_detector` は audio buffer, VAD state, turn context から発話ターン終了を推定する audio AI node です。

## 非責務

- VAD 推定
- ASR
- KWS
- dialogue orchestration
- model fallback

## Input / Output

| 種別 | 名前 | 型 |
| --- | --- | --- |
| Sub | `audio_topic` | `fa_interfaces/msg/AudioFrame` |
| Sub | `vad_topic` | `fa_interfaces/msg/VadState` |
| Sub | `turn_context_topic` | `fa_interfaces/msg/TurnContext` |
| Pub | `output_topic` | `fa_interfaces/msg/TurnEnd` |

## Backend Contract

- `backend.name` は必須。
- `backend.model_path` は必須。空値から package share の model を推測しない。
- `backend.execution_provider` は必須。
- `backend.command` は必須。ROS2 node process とは別の Python / venv / container runtime を指す。
- `backend.args` は必須。`{audio}`, `{model}`, `{provider}` を含む。
- `backend.health_args` は必須。`{model}`, `{provider}` を含み、startup health check で model IO contract を検証する。
- backend は ROS2 topic/message を知らない。
- backend name の解決と backend-specific config assembly は `fa_turn_detector_py/backends/factory.py` に閉じ、ROS2 node は `TurnDetectorBackend` interface だけを保持する。
- backend input は mono `FLOAT32LE/32` samples と sample rate。
- backend output は turn-end probability。
- `smart_turn_onnx` backend は ONNX Runtime を ROS2 node process 内で import しない。ONNX 推論は external worker process で実行する。

## Parameter Contract

ROS parameter は宣言した型と一致することを要求する。文字列 `"false"` を bool として扱う、文字列 `"0.5"` や integer `1` を double として扱う、topic 名を `str()` で丸める、配列要素を `str()` で変換する、といった起動時の暗黙変換は行わない。

`expected_source_id` は必須であり、受信 `AudioFrame.source_id` と完全一致しなければならない。受信 `AudioFrame.stream_id` は `audio_topic` と完全一致しなければならない。topic subscription だけを根拠に、別 source / stream の frame を turn detection buffer に混ぜない。

`VadState.source_id` も `expected_source_id`、`VadState.stream_id` も `audio_topic` と完全一致しなければならない。identity 不一致の VAD state は `is_speech` 更新にも turn end trigger にも使わない。

`smart_turn_onnx` backend constructor に渡された config も同じ契約を維持し、`threshold` / `timeout_sec` / `cleanup_audio_files` を `float()` / `bool()` で再丸めしない。型が一致しない場合は起動時に失敗する。

## Fail Closed

- backend.name 空
- unknown backend
- model path missing
- execution provider missing / unavailable
- command missing / not executable
- args / health_args missing required placeholder
- args / health_args に unknown placeholder / malformed format / conversion / format spec がある
- invalid model file
- incompatible ONNX model input/output contract
- expected_source_id missing
- AudioFrame source_id / stream_id mismatch
- VadState source_id / stream_id mismatch
- invalid audio frame
- invalid worker output probability
- health check failure
- backend runtime failure

## Test IDs

- `FA-TD-SPEC-001`: backend.name 空で起動失敗
- `FA-TD-SPEC-002`: model path missing で起動失敗
- `FA-TD-SPEC-003`: execution provider missing / unavailable で起動失敗
- `FA-TD-SPEC-004`: invalid frame を publish しない
- `FA-TD-SPEC-005`: backend runtime failure で node を fail closed する
- `FA-TD-SPEC-006`: normalized range 外または non-finite samples を reject する
- `FA-TD-SPEC-007`: probability threshold 超過で TurnEnd を publish
- `FA-TD-SPEC-008`: ONNX Runtime は ROS2 node process ではなく external worker process に閉じ込める
- `FA-TD-SPEC-009`: startup health check で model IO contract 不一致を起動失敗にする
- `FA-TD-SPEC-010`: empty `AudioFrame.data` を invalid frame として reject する
- `FA-TD-SPEC-011`: worker output probability が NaN / 範囲外なら fail closed する
- `FA-TD-SPEC-012`: default config は worker command / model / provider / args を暗黙補完しない
- `FA-TD-SPEC-013`: `PCM32LE/32` は float32 として解釈せず、`FLOAT32LE/32` のみを backend input として受け付ける
- `FA-TD-SPEC-014`: `backend.args` / `backend.health_args` は ROS string array parameter として宣言し、非 string 値を `str()` で丸めない
- `FA-TD-SPEC-015`: ROS parameter / backend config は型一致を要求し、起動時の `str()` / `bool()` / `float()` 変換で誤設定を隠さない
- `FA-TD-SPEC-016`: `AudioFrame.source_id` が `expected_source_id` と一致しない frame を reject する
- `FA-TD-SPEC-017`: `AudioFrame.stream_id` が `audio_topic` と一致しない frame を reject する
- `FA-TD-SPEC-018`: `VadState.source_id` / `stream_id` が `expected_source_id` / `audio_topic` と一致しない state を reject する
