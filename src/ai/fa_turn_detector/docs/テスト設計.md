# fa_turn_detector テスト設計

## Unit Test

- `FA-TD-SPEC-001`: backend.name validation。
- `FA-TD-SPEC-002`: model path validation。
- `FA-TD-SPEC-003`: execution provider missing / unavailable validation。
- `FA-TD-SPEC-004`: AudioFrame canonical format validation。
- `FA-TD-SPEC-005`: backend runtime failure で fatal log / shutdown / exception により fail closed する。
- `FA-TD-SPEC-006`: normalized range 外または non-finite samples を reject する。
- `FA-TD-SPEC-007`: threshold decision。
- `FA-TD-SPEC-008`: ROS2 node / backend adapter が `onnxruntime` を import せず external worker boundary を使う。
- `FA-TD-SPEC-009`: startup health check failure は起動失敗にする。
- `FA-TD-SPEC-010`: empty `AudioFrame.data` を invalid frame として reject する。
- `FA-TD-SPEC-011`: worker output probability の NaN / 範囲外 / empty stdout を fail closed する。
- `FA-TD-SPEC-012`: `backend.args` / `backend.health_args` の required placeholder / unknown placeholder / malformed format を起動時に検証する。
- `FA-TD-SPEC-013`: `PCM32LE/32` payload は float32 として解釈せず、`_frame_to_float` 実行テストで reject を確認する。
- `FA-TD-SPEC-014`: `backend.args` / `backend.health_args` は ROS string array parameter と backend `tuple[str, ...]` contract として扱い、非 string 値を `str()` で丸めない。
- `FA-TD-SPEC-015`: ROS parameter / backend config は型一致を要求し、起動時の `str()` / `bool()` / `float()` 変換で誤設定を隠さない。
- `FA-TD-SPEC-016`: `AudioFrame.source_id` が `expected_source_id` と一致しない frame を reject する。
- `FA-TD-SPEC-017`: `AudioFrame.stream_id` が `expected_stream_id` と一致しない frame を reject する。
- `FA-TD-SPEC-018`: `VadState.source_id` / `stream_id` が `expected_source_id` / `expected_stream_id` と一致しない state を reject する。
- `FA-TD-SPEC-019`: launch の `node_name` / `config_file` は暗黙 default を持たず、呼び出し側が明示する。
- `FA-TD-SPEC-020`: Smart Turn runtime は 80 x 800 log-mel feature-window contract を守り、短い mel は左 `-4.0` pad、長い mel は最新 800 frame に truncate する。
- `FA-TD-SPEC-021`: Smart Turn runtime の log-mel floor / dynamic-range normalization policy を contract test で固定する。
- `FA-TD-SPEC-022`: active な `TurnContext.session_id` / `user_turn_id` が切り替わった場合、旧 turn の audio buffer と VAD speech state を新 turn に引き継がない。

## Integration Test

- `test/integration/test_fa_turn_detector_external_worker_contract.py`
  - fake external worker で `.npy` audio payload / stdout probability contract を確認する。
  - health command と inference command が分離され、health path に空 audio placeholder を渡さないことを確認する。
  - `backends/` 配下の全 Python file を走査し、ROS adapter backend は `rclpy`、`fa_interfaces`、ROS message 型へ依存しないことを確認する。
  - ROS adapter backend は `onnxruntime` を import せず、ONNX Runtime は worker runtime module に閉じることを確認する。
- `test/integration/test_fa_turn_detector_ros_graph_contract.py`
  - rclpy graph 上で `TurnContext`、`AudioFrame`、`VadState` を publish し、fake external worker の fixture probability から `TurnEnd` が publish されることを確認する。
  - session / user_turn_id / probability / threshold decision が output に反映されることを確認する。
  - identity mismatch の `VadState` では `TurnEnd` が publish されず、matching `VadState` で publish されることを確認する。

## Launch Test

- launch は `node_name` と `config_file` を呼び出し側から必須で受け取り、launch file 内に default を持たないことを確認する。
- launch は `FindPackageShare` / `PathJoinSubstitution` で package-local config を推測しないことを確認する。
- launch は config file だけを渡し、`backend.model_path` を空 launch argument で上書きしないことを確認する。
- missing expected source / model path / execution provider / command / args config で起動失敗することを確認する。

## Colcon Test

- `CMakeLists.txt` で `ament_cmake_pytest` に `test/` を登録し、`colcon test --packages-select fa_turn_detector` で unit / integration / launch contract test を実行する。
- test 実行時は ambient pytest plugin を無効化し、host/container 側の無関係な pytest plugin によって Turn Detector 契約テストが skip されないことを保証する。
