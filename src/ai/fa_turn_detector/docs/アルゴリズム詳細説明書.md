# fa_turn_detector アルゴリズム詳細説明書

1. `AudioFrame` が configured source / stream identity、mono / `FLOAT32LE` / bit depth 32 / normalized [-1.0, 1.0] / backend sample rate と一致することを検証する。
2. 不一致の frame は backend に渡さず reject する。
3. rolling buffer に samples を追加する。
6. 起動時に `backends/factory.py` が `backend.name` を検証し、対応する backend adapter を生成する。
7. 必要 sample 数が揃ったら backend に mono float32 window を渡す。
8. `smart_turn_onnx` backend は float32 window を `.npy` payload として workspace に書き出す。
9. backend は `backend.command` と `backend.args` から external worker process を起動する。
10. worker process は ONNX Runtime を import し、model input/output contract を検証したうえで probability を stdout に出力する。
11. backend は stdout の最終非空行から probability を読み、finite / `[0.0, 1.0]` を検証する。
12. backend runtime failure は fatal log / shutdown / exception により fail closed する。
13. VAD `speech_ended` を受けた時点で backend を呼び、probability が threshold を超えた場合に `TurnEnd` 候補を publish する。

Resampling / downmix / bit-depth conversion / normalization は `fa_turn_detector` では行いません。必要な変換は upstream の format/processing node で明示します。
`source_id` は `expected_source_id` と一致し、`stream_id` は `expected_stream_id` と一致する必要があります。`audio_topic` は ROS transport の接続点であり、frame identity ではありません。別 source / stream の frame を rolling buffer に混在させません。
ONNX Runtime は ROS2 node process 内で import しません。Python version / venv / provider 依存は `backend.command` で指定した external worker 側へ閉じ込めます。
startup 時に `backend.health_args` で worker の health command を起動し、model load、execution provider、input/output name / dtype / shape を検証します。inference command は `backend.args` に限定し、health command に空 audio path を流しません。health check が失敗した場合は起動失敗です。

Smart Turn worker runtime の model input は 80 mel bin x 800 frame の `input_features` です。STFT は `n_fft=400`、`hop_length=160`、16kHz 前提で行い、log mel は `log10(max(mel, 1e-10))` のあと dynamic range を最大値から 8.0 以内に floor し、`(log_mel + 4.0) / 4.0` で model 入力値へ正規化します。mel frame が 800 未満の場合は左側を `-4.0` で pad し、800 を超える場合は最新 800 frame だけを使います。この処理は audio sample rate / channel / bit depth の変換ではなく、Smart Turn model 入力の feature-window 契約です。
