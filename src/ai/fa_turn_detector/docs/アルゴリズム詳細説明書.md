# fa_turn_detector アルゴリズム詳細説明書

1. `AudioFrame` が mono / float32 / normalized [-1.0, 1.0] / backend sample rate と一致することを検証する。
2. 不一致の frame は backend に渡さず reject する。
3. rolling buffer に samples を追加する。
4. VAD state と turn context を保持する。
5. 必要 sample 数が揃ったら backend に mono float32 window を渡す。
6. `smart_turn_onnx` backend は float32 window を `.npy` payload として workspace に書き出す。
7. backend は `backend.command` と `backend.args` から external worker process を起動する。
8. worker process は ONNX Runtime を import し、model input/output contract を検証したうえで probability を stdout に出力する。
9. backend は stdout の最終非空行から probability を読み、finite / `[0.0, 1.0]` を検証する。
10. backend runtime failure は fatal log / shutdown / exception により fail closed する。
11. probability が threshold を超えた場合に `TurnEnd` を publish する。

Turn context は ASR / dialogue 側との相関に使います。turn detector は会話状態を直接変更しません。
Resampling / downmix / bit-depth conversion / normalization は `fa_turn_detector` では行いません。必要な変換は upstream の format/processing node で明示します。
ONNX Runtime は ROS2 node process 内で import しません。Python version / venv / provider 依存は `backend.command` で指定した external worker 側へ閉じ込めます。
startup 時に `backend.health_args` で worker を起動し、model load、execution provider、input/output name / dtype / shape を検証します。health check が失敗した場合は起動失敗です。
