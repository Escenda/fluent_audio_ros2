# fa_sed テスト設計

## 現在の位置づけ

`fa_sed` は package 化前の roadmap note である。現時点では文書ファイル、package、実装ファイルの存在確認を仕様の根拠にしない。
package 化までは、この文書は実装済みテストの一覧ではなく、package 化時に必須化する behavioral / capability contract を定義する。

## Package 化後に必要な Unit / Backend Test

- fake backend capability validation: `backend.name`、provider / worker command、model id または model path、supported sample rates、supported window、batch / streaming mode、output schema、label set が欠損または矛盾した場合は起動失敗する。
- accepted `AudioFrame` contract: config と capability が明示する `FLOAT32LE`、`bit_depth=32`、mono、interleaved、configured sample rate、configured window だけを受け付ける。
- unsupported input rejection: `PCM16`、`PCM32`、stereo、planar、sample-rate mismatch、window mismatch、non-finite sample、range violation は変換せず frame reject または fail closed にする。
- output schema contract: event label、confidence、time range、model id、source identity、stream identity を必須にする。
- label contract: backend output label は configured label set に含まれる値だけを許可し、unknown label fallback は禁止する。

## Package 化後に必要な Integration / Launch Test

- fake SED backend で configured label set、confidence range、time range、model id propagation を確認する。
- missing params file、missing backend、missing model capability、missing label set、unsupported input format で起動失敗または frame reject することを確認する。
