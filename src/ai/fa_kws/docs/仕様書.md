# fa_kws 仕様書

## 責務

`fa_kws` は audio frame から wake word を検出する audio AI node です。backend を明示選択し、wake word event を publish します。

## 非責務

- turn detection
- microphone source selection
- audio recording / dump file generation
- WAV / PCM encode
- model fallback

## Input / Output

| 種別 | 名前 | 型 |
| --- | --- | --- |
| Sub | `audio_topic` | `fa_interfaces/msg/AudioFrame` |
| Sub | `control.<id>.topic` | `fa_interfaces/msg/VoiceActivity` |
| Pub | `output_topic` | `fa_interfaces/msg/WakeWordResult` |

## Backend Contract

- `backend.name` は必須。
- `backend.execution_provider` は必須。
- topic binding、sample rate、cooldown、model files、worker command、worker args / stream_args、worker timeout、sherpa-onnx numeric config は ROS parameter として明示される必要がある。node code / backend config struct は実行時 default を持たず、config 欠落時は起動失敗する。
- audio / output topic QoS は `audio.qos.depth`、`audio.qos.reliable`、`output.qos.depth`、`output.qos.reliable` として明示される必要がある。node code に hidden depth / reliability を置かない。
- backend は ROS2 topic/message を知らない。
- node が保持する backend 型は `KwsBackend` interface とし、sherpa-onnx 固有 runtime は `backend.command` で指定された external worker 側へ閉じる。
- backend name validation と concrete backend construction は `backends/factory` に閉じ、node は `KwsBackendSettings` と `KwsBackend` interface だけを扱う。
- backend input は mono float samples, source sample rate, timestamp。
- backend output は detected flag, keyword, score。
- `sherpa_onnx_kws` backend は validated mono float samples を one-shot raw float32le file、または persistent worker への base64 raw float32le `PUSH` として渡す。stdout は `NO_DETECTION` または `DETECTED<TAB>keyword<TAB>score<TAB>start_time_sec` だけを受け付ける。
- `expected_source_id` は必須。`AudioFrame.source_id` はこの値と一致しなければならない。
- `expected_stream_id` は必須。`AudioFrame.stream_id` はこの値と一致しなければならない。
- `audio_topic` は ROS transport の接続点であり、frame identity ではない。`expected_stream_id` が `audio_topic` / `output_topic` と同一、または先頭 `/` の有無だけが違う同一 identity の場合は起動失敗にする。
- `control.inputs` に `VoiceActivity` gate を指定した場合、`VoiceActivity.is_speech` が true かつ configured probability field が `control.<id>.probability_gate` 以上の区間だけ `stream_gate` が backend に audio chunk を渡す。`stream_gate` は pre-roll、hangover、短い VAD gap の merge を扱い、閉区間で backend stream を reset する。
- VAD gate は node/controller 層の責務であり、`KwsBackend` / sherpa-onnx worker は VAD topic、turn lifecycle、ROS message を知らない。
- `kws_wav_tool` は offline 評価用であり、ROS topic を購読しない。
- `kws_wav_tool` も backend config field を未初期化値や hidden default にしない。`--sample_rate`, `--num_threads`, `--max_active_paths`, `--num_trailing_blanks`, `--keywords_score`, `--threshold` を CLI で明示し、`SherpaOnnxKwsBackendConfig` 全 field を `build_backend_config()` で埋める。

## Fail Closed

- unknown backend
- required ROS parameter missing
- execution provider missing / unsupported
- worker command missing / not executable
- worker args / stream_args / health_args missing required placeholder
- worker health check failure
- worker timeout / non-zero exit / invalid output
- model path missing / not a regular readable file
- tokens/keywords file missing / not a regular readable file
- invalid audio frame
- `expected_source_id` missing
- `expected_stream_id` missing
- `expected_stream_id` が ROS topic 名と衝突
- `AudioFrame.source_id` が `expected_source_id` と一致しない
- `AudioFrame.stream_id` が `expected_stream_id` と一致しない
- `control.inputs` が複数ある、または `control.<id>.msg_type` が `fa_interfaces/msg/VoiceActivity` でない
- VAD gate の `probability_gate` / `max_age_ms` / QoS が不正
- `kws_wav_tool` non-canonical WAV input
- `audio.qos.depth` / `output.qos.depth` が 0 以下
- `backend.command` / `backend.args` または `backend.stream_args` / `backend.health_args` / `backend.timeout_sec` / `backend.workspace_dir` が不正
- `kws_wav_tool` の backend numeric CLI が未指定または範囲外
- backend sample rate が configured `target_sample_rate` と一致しない
- backend が未初期化 spotter / stream で処理または reset される
- backend に空 sample window が渡される
- backend に non-finite または normalized range 外の sample window が渡される
- backend model / tokens / keywords path が regular readable file でない
- backend numeric config が範囲外または非有限値

backend runtime failure は callback 内で fatal log を出し、`rclcpp::shutdown()` により fail closed する。別 backend への fallback、no detection 扱い、audio frame drop 継続は行わない。

## Test IDs

- `FA-KWS-SPEC-001`: backend config validation
- `FA-KWS-SPEC-002`: AudioFrame canonical format validation
- `FA-KWS-SPEC-003`: worker no-detection output を publish しない
- `FA-KWS-SPEC-004`: worker detection output を WakeWordResult に変換する
- `FA-KWS-SPEC-005`: `backend.execution_provider` missing / unsupported validation
- `FA-KWS-SPEC-006`: missing audio binding では audio frame を処理しない
- `FA-KWS-SPEC-007`: `channels != 1` / `sample_rate != target_sample_rate` / `encoding != FLOAT32LE` / `bit_depth != 32` を reject し、node 内変換しない
- `FA-KWS-SPEC-008`: normalized range 外または non-finite samples を reject する
- `FA-KWS-SPEC-009`: `kws_wav_tool` は non-canonical WAV を resample/downmix/bit-depth convert せず reject する
- `FA-KWS-SPEC-016`: `fa_kws_node` は sherpa-onnx C API に link せず、runtime は external worker command に閉じる
- `FA-KWS-SPEC-017`: node は debug dump / WAV encode / PCM16 conversion を持たず、記録は `fa_record` へ分離する
- `FA-KWS-SPEC-018`: `PCM32LE/32` は float32 として解釈せず、`FLOAT32LE/32` のみを backend input として受け付ける
- `FA-KWS-SPEC-019`: empty `AudioFrame.data` は silent no-detection ではなく invalid frame として reject する
- `FA-KWS-SPEC-020`: backend public boundary は empty samples / sample-rate mismatch / missing stream を no-detection にせず fail closed する
- `FA-KWS-SPEC-021`: backend constructor は model path と numeric config を C API 初期化前に検証する
- `FA-KWS-SPEC-022`: node は `expected_source_id` と `expected_stream_id` により `AudioFrame` identity を bind し、別 source/stream の混入を reject する
- `FA-KWS-SPEC-024`: node / backend config struct は hidden runtime default を持たず、required ROS parameter 欠落時は起動失敗する
- `FA-KWS-SPEC-025`: `kws_wav_tool` は backend numeric config を CLI で明示させ、`SherpaOnnxKwsBackendConfig` 全 field を初期化してから backend を生成する
- `FA-KWS-SPEC-026`: launch の `node_name` / `config_file` は暗黙 default を持たない
- `FA-KWS-SPEC-027`: `expected_stream_id` は ROS topic 名と衝突してはならない
- `FA-KWS-SPEC-028`: audio / output QoS は必須 parameter で構成し、hidden `KeepLast` / reliability default を持たない
- `FA-KWS-SPEC-029`: `backend.command` / `backend.args` または `backend.stream_args` / `backend.health_args` / `backend.timeout_sec` / `backend.workspace_dir` は必須で、worker health check failure は起動失敗する
- `FA-KWS-SPEC-030`: `scripts/sherpa_onnx_kws_worker` は thin entrypoint、`fa_kws_py/backends/sherpa_onnx_kws_worker.py` は reference external worker として提供し、ROS2 node process に sherpa-onnx runtime を混入しない
- `FA-KWS-SPEC-031`: `backend.name` validation は ROS-free helper と C++ gtest で実行時検証し、node private string check だけに閉じない
- `FA-KWS-SPEC-032`: concrete KWS backend construction は ROS-free factory に閉じ、ROS2 node は `KwsBackend` interface と `KwsBackendSettings` だけを扱う
- `FA-KWS-SPEC-033`: backend worker payload は host endian の `float` memory dump ではなく、明示 raw float32le serialization として書く
- `FA-KWS-SPEC-034`: backend runtime exception は node callback 内で fatal log / shutdown により fail closed する
- `FA-KWS-SPEC-036`: persistent KWS stream worker は model を常駐保持し、node の `stream_gate` が VAD 条件に応じた audio chunk と reset だけを送る
