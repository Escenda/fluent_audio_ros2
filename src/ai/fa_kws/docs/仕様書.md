# fa_kws 仕様書

## 責務

`fa_kws` は audio frame と VAD state から wake word を検出する audio AI node です。backend を明示選択し、wake word event を publish します。

## 非責務

- VAD 推定そのもの
- ASR
- turn detection
- microphone source selection
- audio recording / dump file generation
- WAV / PCM encode
- model fallback

## Input / Output

| 種別 | 名前 | 型 |
| --- | --- | --- |
| Sub | `audio_topic` | `fa_interfaces/msg/AudioFrame` |
| Sub | `vad_topic` | `fa_interfaces/msg/VadState` |
| Pub | `output_topic` | `fa_interfaces/msg/WakeWordResult` |

## Backend Contract

- `backend.name` は必須。
- `backend.execution_provider` は必須。
- topic binding、sample rate、VAD gate、cooldown、model files、worker command、worker args、worker timeout、sherpa-onnx numeric config は ROS parameter として明示される必要がある。node code / backend config struct は実行時 default を持たず、config 欠落時は起動失敗する。
- audio / VAD / output topic QoS は `audio.qos.depth`、`audio.qos.reliable`、`vad.qos.depth`、`vad.qos.reliable`、`output.qos.depth`、`output.qos.reliable` として明示される必要がある。node code に hidden depth / reliability を置かない。
- backend は ROS2 topic/message を知らない。
- node が保持する backend 型は `KwsBackend` interface とし、sherpa-onnx 固有 runtime は `backend.command` で指定された external worker 側へ閉じる。
- backend name validation と concrete backend construction は `backends/factory` に閉じ、node は `KwsBackendSettings` と `KwsBackend` interface だけを扱う。
- backend input は mono float samples, source sample rate, VAD probability, timestamp。
- backend output は detected flag, keyword, score。
- `sherpa_onnx_kws` backend は validated mono float samples を明示的に raw float32le payload として serialize し、stdout の最終非空行として `NO_DETECTION` または `DETECTED<TAB>keyword<TAB>score<TAB>start_time_sec` だけを受け付ける。
- `expected_source_id` は必須。`AudioFrame.source_id` はこの値と一致しなければならない。
- `expected_stream_id` は必須。`AudioFrame.stream_id` と `VadState.stream_id` はこの値と一致しなければならない。
- `audio_topic` は ROS transport の接続点であり、frame identity ではない。`expected_stream_id` が `audio_topic` / `vad_topic` / `output_topic` と同一、または先頭 `/` の有無だけが違う同一 identity の場合は起動失敗にする。
- `VadState.source_id` は `expected_source_id` と一致しなければならない。一致しない VAD state は最新 state として扱わず、保持済み VAD state も clear する。
- VAD probability は最新 `VadState` から取得する。未受信または `vad.max_age_ms` 超過の stale state では audio frame を処理しない。
- `vad.probability_gate` は `(0.0, 1.0]` の有限値で必須。`0.0` や負値で VAD gating を無効化しない。
- `VadState.probability` は `[0.0, 1.0]` の有限値だけを fresh state として扱う。範囲外または NaN は invalid state として破棄し、次の valid state まで audio frame を処理しない。
- `VadState.probability` が gate 未満の場合、node は backend を呼び出さない。backend も防御的に sherpa-onnx stream へ波形を渡す前に gate を評価する。
- `VadState.probability` が gate 未満の場合、backend も worker を呼び出す前に gate を評価する。
- `kws_wav_tool` は offline 評価用であり、VAD topic を購読しない。使用時は `--assume-vad-speech` を明示し、canonical speech fixture として扱う意思を command line に残す。
- `kws_wav_tool` も backend config field を未初期化値や hidden default にしない。`--sample_rate`, `--num_threads`, `--max_active_paths`, `--num_trailing_blanks`, `--keywords_score`, `--threshold` を CLI で明示し、`SherpaOnnxKwsBackendConfig` 全 field を `build_backend_config()` で埋める。

## Fail Closed

- unknown backend
- required ROS parameter missing
- execution provider missing / unsupported
- worker command missing / not executable
- worker args / health_args missing required placeholder
- worker health check failure
- worker timeout / non-zero exit / invalid output
- model path missing / not a regular readable file
- tokens/keywords file missing / not a regular readable file
- invalid audio frame
- `expected_source_id` missing
- `expected_stream_id` missing
- `expected_stream_id` が ROS topic 名と衝突
- `AudioFrame.source_id` が `expected_source_id` と一致しない
- `AudioFrame.stream_id` が `expected_stream_id` と一致しない
- `VadState.source_id` / `stream_id` が `expected_source_id` / `expected_stream_id` と一致しない
- `kws_wav_tool` non-canonical WAV input
- VAD state missing / stale
- VAD probability NaN / out of range
- VAD probability が gate 未満
- `vad.probability_gate` が 0 以下、1 超過、または NaN
- `vad.max_age_ms` が 0 以下
- `audio.qos.depth` / `vad.qos.depth` / `output.qos.depth` が 0 以下
- `backend.command` / `backend.args` / `backend.health_args` / `backend.timeout_sec` / `backend.workspace_dir` が不正
- `kws_wav_tool` で VAD policy が未指定
- `kws_wav_tool` の backend numeric CLI が未指定または範囲外
- backend sample rate が configured `target_sample_rate` と一致しない
- backend が未初期化 spotter / stream で処理または reset される
- backend に空 sample window が渡される
- backend に non-finite または normalized range 外の sample window が渡される
- backend model / tokens / keywords path が regular readable file でない
- backend numeric config が範囲外または非有限値

backend runtime failure は callback 内で fatal log を出し、`rclcpp::shutdown()` により fail closed する。別 backend への fallback、no detection 扱い、audio frame drop 継続は行わない。

- `FA-KWS-SPEC-010`: `vad.probability_gate` は必須 gating threshold とし、0 以下や NaN で無効化しない

- `FA-KWS-SPEC-018`: `PCM32LE/32` は float32 として解釈せず、`FLOAT32LE/32` のみを backend input として受け付ける
- `FA-KWS-SPEC-019`: empty `AudioFrame.data` は silent no-detection ではなく invalid frame として reject する

- `FA-KWS-SPEC-027`: `expected_stream_id` は ROS topic 名と衝突してはならない
- `FA-KWS-SPEC-028`: audio / VAD / output QoS は必須 parameter で構成し、hidden `KeepLast` / reliability default を持たない
- `FA-KWS-SPEC-029`: `backend.command` / `backend.args` / `backend.health_args` / `backend.timeout_sec` / `backend.workspace_dir` は必須で、worker health check failure は起動失敗する
