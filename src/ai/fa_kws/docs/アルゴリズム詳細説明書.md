# fa_kws アルゴリズム詳細説明書

1. `AudioFrame` が mono / `FLOAT32LE` / bit depth 32 / normalized [-1.0, 1.0] / `target_sample_rate` と一致することを検証する。
2. 不一致の frame は backend に渡さず reject する。
3. 最新の `VadState.probability` を読む。
4. `VadState` が未受信、`vad.max_age_ms` より古い、または probability が `[0.0, 1.0]` の有限値でない場合は frame を処理せず skip する。
5. probability が `vad.probability_gate` 未満の場合は backend を呼ばず skip する。
6. backend `process()` に samples, sample rate, VAD probability, timestamp を渡す。
7. backend は防御的に VAD gate を再評価し、gate 未満なら sherpa-onnx stream に波形を渡す前に stream を reset して `nullopt` を返す。
8. detection があれば `WakeWordResult` を publish する。

VAD probability は gating signal です。`vad.probability_gate` は `(0.0, 1.0]` の有限値だけを受け付け、`0.0` や負値を「gating disabled」として扱いません。VAD 推定は `fa_vad` に分離します。
Resampling / downmix / bit-depth conversion は `fa_kws` では行いません。必要な変換は upstream の format/processing node で明示します。
音声記録、debug dump、WAV encode、PCM16 conversion は `fa_kws` の責務ではありません。後段確認用に音声を保存する場合は `fa_record` を pipeline に追加します。
`kws_wav_tool` も同じ責務境界に従います。tool 入力 WAV は `--sample_rate` と一致する mono / IEEE float32 / normalized [-1.0, 1.0] でなければならず、tool 内で resample、downmix、bit-depth conversion は行いません。
`kws_wav_tool` は VAD topic を購読しない offline 評価 tool なので、起動時に `--assume-vad-speech` を必須にし、backend には `vad_prob=1.0` と `vad_threshold=1.0` を明示して渡します。backend 側の VAD gate を `0.0` で無効化する経路は持ちません。
sherpa-onnx provider は `backend.execution_provider` で明示し、空値や未対応 provider は backend 生成前に reject します。
sherpa-onnx KWS の current C API result は confidence score を返さないため、backend は binary detection score `1.0` を明示的に返します。
