# fa_kws テスト設計

## Unit Test

- `FA-KWS-SPEC-001`: backend config validation。
- `FA-KWS-SPEC-002`: AudioFrame canonical format validation。
- `FA-KWS-SPEC-004`: worker detection result mapping。
- `FA-KWS-SPEC-005`: `backend.execution_provider` missing / unsupported validation。
- `FA-KWS-SPEC-007`: `channels != 1` / `sample_rate != target_sample_rate` / `encoding != FLOAT32LE` / `bit_depth != 32` を reject し、node 内変換しない。
- `FA-KWS-SPEC-008`: normalized range 外または non-finite samples を reject する。
- `FA-KWS-SPEC-009`: `kws_wav_tool` は non-canonical WAV を resample/downmix/bit-depth convert せず reject する。
- `FA-KWS-SPEC-012`: `kws_wav_tool` は `--assume-vad-speech` なしで起動しない。
- `FA-KWS-SPEC-014`: launch は `node_name` と `config_file` を必須で受け取り、backend/model/provider fallback を埋め込まない。
- `FA-KWS-SPEC-015`: backend construction は ROS-free C++ helper / factory で実行時検証し、node は `KwsBackend` interface だけを扱う。
- `FA-KWS-SPEC-016`: runtime は external worker command に閉じ、node process は worker protocol を明示的に呼び出す。
- `FA-KWS-SPEC-017`: backend public boundary は ROS2 topic/message を要求せず、raw float32le payload と typed request/result contract だけで動作する。
- `FA-KWS-SPEC-018`: node は debug dump / WAV encode / PCM16 conversion を持たず、記録は `fa_record` へ分離する。
- `FA-KWS-SPEC-019`: `PCM32LE/32` payload は float32 として解釈せず、実行テストで reject を確認する。
- `FA-KWS-SPEC-020`: empty `AudioFrame.data` は silent no-detection ではなく invalid frame として reject する。
- `FA-KWS-SPEC-021`: backend public boundary は empty samples / sample-rate mismatch / missing stream を no-detection にせず fail closed する。
- `FA-KWS-SPEC-022`: backend constructor は model path と numeric config を C API 初期化前に検証する。
- `FA-KWS-SPEC-023`: `expected_source_id` と `expected_stream_id` により `AudioFrame.source_id` / `stream_id` を bind し、不一致 frame を reject する。
- `FA-KWS-SPEC-025`: node / backend config struct は hidden runtime default を持たず、required ROS parameter 欠落時は起動失敗する。
- `FA-KWS-SPEC-026`: `kws_wav_tool` は backend numeric config を CLI で明示させ、`SherpaOnnxKwsBackendConfig` 全 field を初期化してから backend を生成する。
- `FA-KWS-SPEC-027`: `expected_stream_id` は ROS topic 名と衝突してはならない。
- `FA-KWS-SPEC-029`: `backend.command` / `backend.args` または `backend.stream_args` / `backend.health_args` / `backend.timeout_sec` / `backend.workspace_dir` は必須で、worker health check failure は起動失敗する。
- `FA-KWS-SPEC-030`: `scripts/sherpa_onnx_kws_worker` は thin entrypoint、`fa_kws_py/backends/sherpa_onnx_kws_worker.py` は reference external worker として提供し、ROS2 node process に sherpa-onnx runtime を混入しない。
- `FA-KWS-SPEC-031`: `backend.name` validation は ROS-free helper と C++ gtest で実行時検証し、node private string check だけに閉じない。
- `FA-KWS-SPEC-032`: concrete KWS backend construction は ROS-free factory に閉じ、ROS2 node は `KwsBackend` interface と `KwsBackendSettings` だけを扱う。
- `FA-KWS-SPEC-033`: backend worker payload は explicit raw float32le serialization であり、host endian memory dump や hidden sample conversion に戻さない。
- `FA-KWS-SPEC-034`: backend runtime exception は callback 内で fatal log / shutdown により fail closed し、no detection や継続運転にしない。
- `FA-KWS-SPEC-035`: VAD gate は ROS-free control object として検証し、`VoiceActivity` の identity / probability / stale update によって backend 呼び出しを許可または抑止する。
- `FA-KWS-SPEC-036`: persistent KWS stream worker は fake worker integration test で `STREAM_READY` / `PUSH` / `RESET` protocol と detection mapping を検証する。

## Integration Test

- `test/integration/test_fa_kws_external_worker_pipeline.py`
  - fake external worker で one-shot raw float32le payload と persistent stream `PUSH` protocol / stdout detection contract を確認する。
  - worker command、workspace、timeout、args / stream_args / health_args の明示 contract を検証する。
  - invalid worker output / backend exception を no-detection にせず fail closed として扱う。

## 未検証の native runtime integration

- canonical fixture WAV を `kws_wav_tool` に通し、backend が検出できることを確認する。
- sample rate / channel / bit depth が canonical でない fixture WAV は `kws_wav_tool` が backend 実行前に reject することを確認する。
- `kws_wav_tool` に `--assume-vad-speech` が無い場合は backend 実行前に reject することを確認する。
- `kws_wav_tool` が `--command`, `--workspace_dir`, `--timeout_sec`, `--sample_rate`, `--num_threads`, `--max_active_paths`, `--num_trailing_blanks`, `--keywords_score`, `--threshold` を explicit backend config として要求し、未初期化の `SherpaOnnxKwsBackendConfig` field を残さないことを確認する。

## Launch Test

- `fa_kws.launch.py` は `node_name` と `config_file` launch argument を必須で受け取り、launch file 内に default を持たない。
- `fa_kws.launch.py` は explicit `config_file` がなければ起動失敗し、package-local config を推測しない。
- launch file は backend 名、model path、execution provider を埋め込まない。
- missing model path config で起動失敗することを確認する。
- model / tokens / keywords path が directory または読み取り不能 file の場合に起動失敗することを確認する。
- missing execution provider config で起動失敗することを確認する。
- missing worker command / args / health_args / workspace / timeout config で起動失敗することを確認する。
- invalid `control.<id>.probability_gate` config で起動失敗することを確認する。
- `audio.qos.depth` / `control.<id>.qos.depth` / `output.qos.depth` が 0 以下の場合に起動失敗することを確認する。
- `AudioFrame.source_id != expected_source_id` または `AudioFrame.stream_id != expected_stream_id` の場合に backend へ渡らないことを確認する。

## Colcon Test

- `colcon test --packages-select fa_kws` で package-local contract test を実行する。
- sherpa-onnx runtime は external worker command 側の責務とし、node package は unavailable backend / dummy backend に fallback しない。
- test 実行時は ambient pytest plugin を無効化し、host/container 側の無関係な pytest plugin によって KWS 契約テストが skip されないことを保証する。
- `fa_kws_backend_config_validation_test` は ROS2 node を起動せず、missing / unknown / supported `backend.name` を C++ 実行テストとして検証する。
- `fa_kws_backend_factory_test` は ROS2 node を起動せず、missing / unknown `backend.name` と concrete backend config delegation を C++ 実行テストとして検証する。
