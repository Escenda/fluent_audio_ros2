# fa_audio_embedding 仕様書

## 1. 目的

`fa_audio_embedding` は canonical `AudioFrame` から audio embedding を生成する AI node である。
embedding model runtime は `backends/` 配下に分離し、ROS2 topic/message を知らない。

## 2. 入力/出力

入力は config と model capability が一致する canonical `AudioFrame` とする。
`source_id` と `stream_id` は config で明示した値に一致しなければならない。

supported input は以下に限定する。

- encoding: `FLOAT32LE`
- bit depth: `32`
- channels: `1`、mono
- layout: `interleaved`
- sample rate: config と capability が明示する supported sample rate
- window: config と capability が明示する supported window
- sample value: finite かつ normalized range `[-1.0, 1.0]`

`PCM16`、`PCM32`、stereo、planar、sample-rate mismatch、window mismatch、non-finite sample、normalized range violation は変換せず fail closed または frame reject とする。

出力は `AudioEmbeddingFrame` とし、embedding vector、model id、embedding dimension、source identity、stream identity、入力 sample rate、入力 sample count を持つ。

## 3. Model Capability

運用 config は model capability を明示する。

- model identity: `backend.model_id`、必要な場合のみ `backend.model_path`
- provider: `backend.name`
- worker command: `backend.command`
- payload encoding: `float32le_raw`
- supported sample rates
- supported window
- execution mode: batch embedding または streaming embedding
- output schema: `AudioEmbeddingFrame`
- embedding dimension

## 4. 責務外

- ASR / VAD / KWS
- speaker verification
- audio generation
- resample / channel conversion / gain
- feature stream からの embedding 生成
- missing model fallback

## 5. fail closed 条件

- `backend.name` 欠損
- `backend.command` 欠損
- `backend.model_id` 欠損
- `backend.args` 欠損または required placeholder 欠落
- `backend.model_path` 指定時に path が存在しない、読めない、または `backend.args` に `{model_path}` が無い
- `expected_source_id` / `expected_stream_id` 欠損
- model capability 欠損または config との矛盾
- embedding dimension mismatch
- input metadata mismatch
- backend output が finite vector でない
- normalized range 外の input sample
