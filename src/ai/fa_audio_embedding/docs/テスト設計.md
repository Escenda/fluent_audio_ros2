# fa_audio_embedding テスト設計

## Unit Test

- `FA-AUDIO-EMBEDDING-SPEC-001`: README、仕様書、アルゴリズム詳細説明書、テスト設計、backend docs が存在する。
- `FA-AUDIO-EMBEDDING-SPEC-002`: `package.xml`、`CMakeLists.txt`、launch、config、node executable が存在する。
- `FA-AUDIO-EMBEDDING-SPEC-003`: 仕様書が zero/stale embedding fallback を禁止する。
- `FA-AUDIO-EMBEDDING-SPEC-004`: backend docs が ROS2 topic/message dependency を禁止する。
- `FA-AUDIO-EMBEDDING-SPEC-005`: default config は `backend.name=external_worker` を明示するが、worker command / model / source / stream / dimension は暗黙選択しない。
- `FA-AUDIO-EMBEDDING-SPEC-006`: backend config は `backend.name`、`backend.command`、`backend.model_id`、required args placeholders、dimension を必須にする。
- `FA-AUDIO-EMBEDDING-SPEC-007`: production Python は `Any` / `dict[str, Any]` / `object` による曖昧型と ImportError 分岐を持たない。
- `FA-AUDIO-EMBEDDING-SPEC-008`: backend code は ROS2 / `fa_interfaces` を import しない。

## Integration Test

fake embedding backend で dimension、model id、finite vector contract、workspace cleanup を確認する。

## Launch Test

launch は explicit `config_file` を受け取り、default config は backend family だけを選択する。worker command、model、source identity、dimension は運用 config で明示し、欠落時は起動失敗として扱う。
missing backend、missing model id、dimension mismatch は node / backend 起動失敗として扱う。
