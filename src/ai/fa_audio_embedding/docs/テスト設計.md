# fa_audio_embedding テスト設計

## Unit Test

- `FA-AUDIO-EMBEDDING-SPEC-001`: default config は `backend.name`、worker command、model、source identity、stream identity、embedding dimension を暗黙選択しない。
- `FA-AUDIO-EMBEDDING-SPEC-002`: backend capability は `backend.name`、`backend.command`、`backend.model_id`、required args placeholders、payload encoding、timeout、dimension を必須にする。
- `FA-AUDIO-EMBEDDING-SPEC-003`: `backend.model_path` を指定する場合は path が存在し、読み取り可能で、`backend.args` に `{model_path}` を含む。
- `FA-AUDIO-EMBEDDING-SPEC-004`: accepted input は config と capability が明示する `AudioFrame` contract に限定する。`FLOAT32LE`、`bit_depth=32`、mono、interleaved、configured sample rate、configured window のみを受け付ける。
- `FA-AUDIO-EMBEDDING-SPEC-005`: unsupported `PCM16` / `PCM32` / stereo / planar / sample-rate mismatch / window mismatch / non-finite sample / normalized range violation は変換せず fail closed または frame reject とする。

## Integration Test

fake embedding worker で model id、finite `float32` vector、embedding dimension、workspace cleanup を確認する。
dimension mismatch、non-finite embedding、empty/non-vector output は backend failure として fail closed にする。

## Launch Test

launch は explicit `config_file` を受け取り、default config が worker/model/source identity/dimension を選択しないことを確認する。
運用 config では backend capability と accepted `AudioFrame` contract を明示し、欠落時は起動失敗として扱う。

### 追加が必要な behavioral test

- `AudioFrame` node contract: `FLOAT32LE` / `bit_depth=32` / mono / interleaved / configured sample rate / configured window を満たす frame だけ publish される。
- Unsupported input rejection: `PCM16`、`PCM32`、stereo、planar、sample-rate mismatch、window mismatch、non-finite sample、normalized range violation は conversion なしで reject される。
