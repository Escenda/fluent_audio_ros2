# fluent_audio_system 仕様書

## 責務

FluentAudio の system config を読み、enabled group / node を ROS2 launch action として展開します。

## 非責務

- backend/model の自動選択
- device 自動推測
- params file の生成・書き換え
- missing params file の fallback
- VLAbor profile 形式の解釈

## Input

| 種別 | 名前 | 内容 |
| --- | --- | --- |
| launch argument | `config` | FluentAudio system YAML の絶対パス |

## Config Schema

```yaml
system:
  default_start_delay: 0.2
  inter_group_delay: 0.5

groups:
  - id: io
    enable: true
    nodes:
      - id: fa_in
        enable: true
        package: fa_in
        exec: fa_in_node
        node_name: fa_in_node
        namespace: ""
        params_file: /path/to/fa_in.yaml
        parameters: {}
        remappings:
          audio/frame: robot/audio/frame
```

## Fail Closed

- `config` が空
- `config` が存在しない
- YAML root が mapping ではない
- `groups` が sequence ではない
- enabled node の `id`, `package`, `exec` が空
- `params_file` が指定されているが存在しない
- enabled node に `params_file` も `parameters` もない
- `remappings` の from / to が空

## Path Expansion

`config` と `params_file` は `${share:package_name}` を展開できます。package が見つからない場合は起動失敗します。

## Test IDs

- `FA-SYS-SPEC-001`: missing config で起動失敗する
- `FA-SYS-SPEC-002`: missing params file で起動失敗する
- `FA-SYS-SPEC-003`: enabled node を launch action に展開する
- `FA-SYS-SPEC-004`: disabled group / node を展開しない
- `FA-SYS-SPEC-005`: remappings を launch action に渡す
- `FA-SYS-SPEC-006`: `${share:...}` を path として展開する
