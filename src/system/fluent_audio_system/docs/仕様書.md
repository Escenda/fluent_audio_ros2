# fluent_audio_system 仕様書

## 責務

FluentAudio の system config を読み、enabled group / node を ROS2 launch action として展開します。
同じ config から、VLAbor / Docker build 側が必要な ROS2 package を解決するための package list も生成します。

## 非責務

- backend/model の自動選択
- device 自動推測
- params file の生成・書き換え
- missing params file の fallback
- VLAbor profile 形式の解釈

## Input

| 種別 | 名前 | 内容 |
| --- | --- | --- |
| launch argument | `config` | FluentAudio system YAML の絶対パス |
| launch argument | `fa_in_enabled` | profile で宣言された `fa_in` を site profile 側で有効化するか。`true` / `false` のみ。launch default は持たない |
| launch argument | `fa_out_enabled` | profile で宣言された `fa_out` を site profile 側で有効化するか。`true` / `false` のみ。launch default は持たない |
| launch argument | `fa_in_source_id` | profile 由来の raw ALSA capture source id |
| launch argument | `fa_out_sink_id` | profile 由来の raw ALSA playback sink id |
| CLI argument | `--config` | package list 生成対象の FluentAudio system YAML |

## Config Schema

YAML は読み込み境界で Pydantic v2 model に変換し、以降の launch 展開では型確定済みの config として扱います。未検証 dict を深い層へ流しません。

```yaml
system:
  default_start_delay: 0.2
  inter_group_delay: 0.5

groups:
  - id: io
    enable: true
    nodes:
      - id: fa_in
        enable: true
        package: fa_in
        exec: fa_in_node
        node_name: fa_in
        namespace: ""
        params_file: /path/to/fa_in.yaml
        parameters: {}
        remappings:
          audio/frame: robot/audio/frame
```

## Fail Closed

- `config` が空
- `config` が存在しない
- YAML root が mapping ではない
- `system` が存在しない、または mapping ではない
- `system.default_start_delay` / `system.inter_group_delay` が未指定
- `system.default_start_delay` / `system.inter_group_delay` が負数
- `groups` が存在しない
- `groups` が sequence ではない
- enabled / disabled に関わらず group の `enable` が未指定
- enabled / disabled に関わらず node の `enable` が未指定
- enabled group に `nodes` が存在しない
- enabled node の `id`, `package`, `exec` が空
- enabled node の `node_name` が空
- enabled node の `params_file` が空
- enabled node の `params_file` に `node_name` / `package` / `/**` のいずれの `ros__parameters` block も存在しない
- `analysis` group に VAD / KWS / ASR / Turn Detector / audio embedding などの buildable AI package が含まれる
- `package.xml` を持たない roadmap placeholder package が system config の `package` に指定されている
- system group category と package category が一致しない
- enabled node の `params_file` が存在しない
- `remappings` の from / to が空
- `remappings` が mapping ではない
- inline `parameters` が ROS parameter として扱えない型
- package 別 parameter contract で ROS topic と audio stream identity に同じ値を使っている
- package 別 parameter contract の topic / stream identity key に string または string list 以外の値を渡している
- runtime backend package の effective `backend.name` が未指定または空。対象は `fa_in` / `fa_out` / `fa_vad` / `fa_kws` / `fa_asr` / `fa_turn_detector` / `fa_audio_embedding` / `fa_tts`
- `${env:VAR_NAME}` 参照先の環境変数が未設定または空
- `fa_in_enabled` / `fa_out_enabled` が `true` / `false` 以外
- `fa_in_enabled=true` なのに `fa_in_source_id` が空
- `fa_out_enabled=true` なのに `fa_out_sink_id` が空

## Path Expansion

`config`、`params_file`、inline `parameters` の string / string list 値は `${share:package_name}` と `${env:VAR_NAME}` を展開できます。package が見つからない場合、または env が未設定 / 空の場合は起動失敗します。これは明示された path の展開であり、missing model / missing params_file の fallback ではありません。

`parameters` は `params_file` に対する明示 override としてのみ扱います。inline `parameters` だけで node を起動することは許可しません。

`params_file` は存在するだけでは不十分です。enabled node の `node_name`、`package`、または `/**` root のいずれかに `ros__parameters` block が必要です。空 file や別 node 用 params file の流用は、runtime default へ落ちる fallback として扱い、起動前に失敗させます。

## Topic / Stream Identity Contract

`input_topic` / `output_topic` / `audio_topic` / `vad_topic` / `input_topics` / `output_topics` などの ROS transport topic と、`input_stream_id` / `expected_stream_id` / `expected.stream_id` / `output.stream_id` / `input_stream_ids` などの `AudioFrame.stream_id` identity は別 contract です。

`fluent_audio_system` は package ごとの明示 parameter contract で `params_file` の `ros__parameters` と inline `parameters` を検証し、同じ値を topic と stream identity の両方に使う config を起動前に拒否します。`params_file` は node name、package name、`/**` の順に既知 ROS params block を読み、nested YAML は dotted parameter key に flatten して検証します。先頭 `/` の有無だけが違う値も同一 identity として扱います。suffix だけで任意 key を推測せず、supported package の既知 parameter key だけを検証対象にします。

## Required Package List

`ros2 run fluent_audio_system list_required_packages --config <system.yaml>` は、system config を build 解決用の validation で読み、build に必要な package を 1 行 1 package で出力します。

出力順は固定です。

1. `fa_interfaces`
2. `fluent_audio_system`
3. system config の enabled node package を group / node の展開順で重複排除したもの

disabled group / disabled node の package は出力しません。`config` 欠落、invalid schema、enabled node の `params_file` 欠落、category mismatch は CLI でも失敗扱いにし、部分的な package list は成功として返しません。

この CLI は pre-build package resolver なので、enabled node の `params_file` 実体確認、node params 内の `${share:...}` 解決、node params 内の `${env:...}` 解決は行いません。まだ build していない package の share directory が存在しない段階でも、system config から enabled package set を決定できる必要があります。launch 実行時は `load_system_config()` が `params_file` 実在と parameter path 展開を fail closed で検証します。

## Package-owned Profiles

`config/profiles/so101.yaml` は SO101 用の標準 profile であり、default enabled node は `fa_in` / `fa_out` に限定する。format pipeline は定義するが default では無効にし、site / launch 側が明示的に有効化する。

`config/profiles/so101_mic_frontend.yaml` は microphone frontend の明示 pipeline である。`fa_in` の default `PCM16LE/16/48kHz` stream を `fa_sample_format` に渡し、`FLOAT32LE/32/interleaved` に変換してから `fa_resample` へ渡す。`fa_resample` に PCM16 を直結しない。

`config/profiles/so101_tts_output.yaml` は TTS playback の明示 pipeline である。`fa_tts` は `audio/tts/frame` に `FLOAT32LE/32` を publish し、`fa_resample` が `audio/tts/48k_float32` へ sample rate を明示変換する。`fa_sample_format` が `PCM16LE/16` の `audio/tts/pcm16` へ変換し、その後 `fa_mix` が `audio/output/frame` へ routing し、`fa_out` が sink adapter として再生する。`fa_out` は resample、format conversion、gain、normalize を行わない。OpenJTalk dictionary path は `backend.openjtalk_dict_dir: ${env:FLUENT_AUDIO_OPENJTALK_DICT_DIR}` として明示し、未設定時は system config 展開で起動失敗する。

## Site Binding Override

VLAbor profile から渡す site 依存値は `fa_in_enabled`、`fa_out_enabled`、`fa_in_source_id`、`fa_out_sink_id` に限定します。これらは `fa_in` / `fa_out` の起動可否と raw ALSA id だけを上書きします。`fa_in_source_id` は `fa_in` の `audio.device_selector.mode=id` / `audio.device_selector.identifier=<fa_in_source_id>` として渡し、表示名解決を混ぜません。

`config`、`fa_in_enabled`、`fa_out_enabled`、`fa_in_source_id`、`fa_out_sink_id` はすべて明示 launch argument とし、launch file には default を置かない。system config で enabled な `fa_in` / `fa_out` を site binding default で黙って抑制しない。standalone debug で IO を起動しない場合は `fa_in_enabled:=false` / `fa_out_enabled:=false` を明示する。

`fa_in_enabled=true` の場合は `fa_in_source_id` を必須とし、`fa_out_enabled=true` の場合は `fa_out_sink_id` を必須とします。空の場合に system config 内の default device へ流すことは、site binding 欠落を隠す device fallback として扱い、起動失敗にします。

`fa_in_source_id` が指定された場合、source-bound audio AI node である `fa_vad` / `fa_kws` / `fa_asr` / `fa_turn_detector` / `fa_audio_embedding` には `expected_source_id` として同じ値を渡します。これは source identity の binding であり、backend/model/threshold を profile から渡す経路ではありません。

VAD frontend を同一 stream として構成する場合、system config 側で `fa_vad.input_stream_id`、`fa_kws.expected_stream_id`、`fa_turn_detector.expected_stream_id`、`fa_asr.expected_stream_id`、`fa_audio_embedding.expected_stream_id` を同じ `AudioFrame.stream_id` に揃えます。`fa_vad.input_topic`、`fa_kws.audio_topic`、`fa_turn_detector.audio_topic`、`fa_asr.audio_topic`、`fa_audio_embedding.input_topic` は ROS transport topic であり、frame identity ではありません。`fa_in_source_id` は source identity の site binding であり、stream identity は processing config で明示します。

backend/model 名、VAD / KWS / ASR / Turn Detector / audio embedding の model path、threshold、prompt、timeout、embedding dimension、external command は system config または各 node config に閉じます。VLAbor profile や `run.py` launch arguments から backend/model 詳細を渡しません。

## VLAbor Include Action Contract

VLAbor profile からの起動 action は Fluent Vision と同じ `type: include` 形式にします。`package` は `fluent_audio_system`、`launch` は `run.py` とし、`args` は `config` と site binding だけに限定します。

```yaml
- type: include
  package: fluent_audio_system
  launch: run.py
  args:
    config: ${fluent_audio_config}
    fa_in_enabled: ${fluent_audio_fa_in_enabled}
    fa_out_enabled: ${fluent_audio_fa_out_enabled}
    fa_in_source_id: ${fluent_audio_fa_in_source_id}
    fa_out_sink_id: ${fluent_audio_fa_out_sink_id}
  enabled: ${fluent_audio_enabled}
```

`test/fixtures/vlabor_include_action.yaml` はこの profile action の canonical fixture です。VLAbor 側の profile に実際に追加するときも、この fixture と同じ key set を使います。`backend.name`、model path、threshold、prompt、endpoint、secret はこの action の `args` に追加しません。
