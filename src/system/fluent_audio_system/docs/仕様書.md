# fluent_audio_system 仕様書

## 責務

FluentAudio の system config を読み、enabled group / node を ROS2 launch action として展開します。

## 非責務

- backend/model の自動選択
- device 自動推測
- params file の生成・書き換え
- missing params file の fallback
- VLAbor profile 形式の解釈

## Input

| 種別 | 名前 | 内容 |
| --- | --- | --- |
| launch argument | `config` | FluentAudio system YAML の絶対パス |
| launch argument | `fa_in_enabled` | profile 由来の `fa_in` 起動可否。`true` / `false` のみ |
| launch argument | `fa_out_enabled` | profile 由来の `fa_out` 起動可否。`true` / `false` のみ |
| launch argument | `fa_in_source_id` | profile 由来の raw ALSA capture source id |
| launch argument | `fa_out_sink_id` | profile 由来の raw ALSA playback sink id |

## Config Schema

```yaml
system:
  default_start_delay: 0.2
  inter_group_delay: 0.5

groups:
  - id: io
    enable: true
    nodes:
      - id: fa_in
        enable: true
        package: fa_in
        exec: fa_in_node
        node_name: fa_in_node
        namespace: ""
        params_file: /path/to/fa_in.yaml
        parameters: {}
        remappings:
          audio/frame: robot/audio/frame
```

## Fail Closed

- `config` が空
- `config` が存在しない
- YAML root が mapping ではない
- `system` が存在しない、または mapping ではない
- `system.default_start_delay` / `system.inter_group_delay` が未指定
- `system.default_start_delay` / `system.inter_group_delay` が負数
- `groups` が存在しない
- `groups` が sequence ではない
- enabled / disabled に関わらず group の `enable` が未指定
- enabled / disabled に関わらず node の `enable` が未指定
- enabled group に `nodes` が存在しない
- enabled node の `id`, `package`, `exec` が空
- enabled node の `params_file` が空
- enabled node の `params_file` が存在しない
- `remappings` の from / to が空
- `fa_in_enabled` / `fa_out_enabled` が `true` / `false` 以外

## Path Expansion

`config`、`params_file`、inline `parameters` の string / string list 値は `${share:package_name}` を展開できます。package が見つからない場合は起動失敗します。これは明示された path の展開であり、missing model / missing params_file の fallback ではありません。

`parameters` は `params_file` に対する明示 override としてのみ扱います。inline `parameters` だけで node を起動することは許可しません。

## Site Binding Override

VLAbor profile から渡す site 依存値は `fa_in_enabled`、`fa_out_enabled`、`fa_in_source_id`、`fa_out_sink_id` に限定します。これらは `fa_in` / `fa_out` の起動可否と raw ALSA id だけを上書きします。

backend/model 名、VAD / KWS / ASR / Turn Detector の model path、threshold、prompt、timeout は system config または各 node config に閉じます。profile から backend/model 詳細を渡しません。

## Test IDs

- `FA-SYS-SPEC-001`: missing config で起動失敗する
- `FA-SYS-SPEC-002`: missing params file で起動失敗する
- `FA-SYS-SPEC-003`: enabled node を launch action に展開する
- `FA-SYS-SPEC-004`: disabled group / node を展開しない
- `FA-SYS-SPEC-005`: remappings を launch action に渡す
- `FA-SYS-SPEC-006`: `${share:...}` を config / params_file / inline parameter path として展開する
- `FA-SYS-SPEC-007`: site binding launch args は `fa_in` / `fa_out` の enable と source/sink id だけを上書きする
