## Launch Test

- `ros2 launch fluent_audio_system run.py config:=... fa_in_enabled:=... fa_out_enabled:=... fa_in_source_id:=... fa_out_sink_id:=...` が system launcher を include する。
- VLAbor profile action fixture が `fluent_vision_system/run.py` と同じ `type: include` 形で `fluent_audio_system/run.py` を呼ぶ。
- `ros2 launch fluent_audio_system run.py config:=... fa_in_enabled:=true fa_out_enabled:=false fa_in_source_id:=hw:... fa_out_sink_id:=disabled` が `fa_in` に raw source id override を `audio.device_selector.mode=id` で渡す。
- `fa_in_source_id` が指定された場合に `fa_vad` / `fa_kws` / `fa_asr` / `fa_turn_detector` / `fa_audio_embedding` へ `expected_source_id` override を渡す。
- launch file は `Node` action に params/remappings を直接渡し、temporary YAML の生成や書き換えをしない。
- `setup.py` が `list_required_packages` console script を install する。
- `ros2 launch fluent_audio_system fluent_audio_system.launch.py config:=<generated-system-yaml>` で `fa_in(backend.name=pcm_file_reader) -> fa_out(backend.name=pcm_file_writer)` を接続し、raw PCM bytes が同一 payload として file sink に到達することを検証する。
- `ros2 launch fluent_audio_system run.py config:=<generated-system-yaml>` で同じ file source / file sink 経路を実行し、VLAbor が include する entrypoint 経由でも raw PCM bytes が到達することを検証する。
- `ros2 launch fluent_audio_system fluent_audio_system.launch.py config:=<generated-system-yaml>` で `fa_in(backend.name=network_pcm_receiver) -> fa_out(backend.name=network_pcm_sender)` を接続し、raw PCM UDP packet が同一 payload として network sink に到達することを検証する。
- `ros2 launch fluent_audio_system fluent_audio_system.launch.py config:=<generated-system-yaml>` で `fa_in(backend.name=pcm_file_reader) -> fa_sample_format -> fa_gain -> fa_out(backend.name=pcm_file_writer)` を接続し、PCM16 入力が `FLOAT32LE/32` へ明示変換され、gain 後の raw float32 bytes が file sink に到達することを検証する。
