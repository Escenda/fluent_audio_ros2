# fluent_audio_system テスト設計

## Unit Test

- `FA-SYS-SPEC-001`: missing config path で `RuntimeError`。
- `FA-SYS-SPEC-002`: YAML root が mapping でなければ `RuntimeError`。
- `FA-SYS-SPEC-003`: enabled node に `params_file` も `parameters` もなければ `RuntimeError`。
- `FA-SYS-SPEC-004`: missing params file で `RuntimeError`。
- `FA-SYS-SPEC-005`: remappings schema を検証する。
- `FA-SYS-SPEC-006`: `${share:...}` / `${env:...}` path を展開する。
- `FA-SYS-SPEC-007`: `fa_in_enabled` / `fa_out_enabled` が `true` / `false` 以外なら `RuntimeError`。
- `FA-SYS-SPEC-008`: site binding override は `fa_in` / `fa_out` の source/sink id と、source-bound audio AI node の `expected_source_id` だけを parameters に追加する。`fa_in_source_id` は `audio.device_selector.mode=id` として渡す。
- `FA-SYS-SPEC-009`: system config は Pydantic v2 model で境界検証する。
- `FA-SYS-SPEC-010`: `fa_in_enabled=true` かつ `fa_in_source_id` 空なら `RuntimeError`。
- `FA-SYS-SPEC-011`: `fa_out_enabled=true` かつ `fa_out_sink_id` 空なら `RuntimeError`。
- `FA-SYS-SPEC-012`: `remappings` は mapping 形式だけを受け付け、pair/list/object 形式を受け付けない。
- `FA-SYS-SPEC-013`: `analysis` group に AI package が含まれる場合は `RuntimeError`。
- `FA-SYS-SPEC-014`: system group category と package category が一致しない場合は `RuntimeError`。
- `FA-SYS-SPEC-015`: SO101 microphone frontend profile が `fa_in -> fa_sample_format -> fa_resample` の明示 format pipeline を展開する。
- `FA-SYS-SPEC-016`: SO101 TTS output profile が `fa_tts -> fa_resample -> fa_sample_format -> fa_mix -> fa_out` の明示 playback pipeline を展開する。
- `FA-SYS-SPEC-017`: `${env:...}` path を展開し、未設定または空なら `RuntimeError`。
- `FA-SYS-SPEC-018`: required package list は `fa_interfaces` / `fluent_audio_system` と enabled node package だけを出力し、disabled node と重複 package を含めない。
- `FA-SYS-SPEC-019`: required package list は pre-build resolver として node `params_file` の実体や node package share directory を要求しない。ただし enabled node の `params_file` 欠落は schema error として失敗する。
- `FA-SYS-SPEC-020`: `io` group は source / sink / utility IO packages を IO category として受け付ける。
- `FA-SYS-SPEC-021`: VAD frontend profile では `fa_vad.input_topic`、`fa_kws.audio_topic`、`fa_turn_detector.audio_topic`、`fa_asr.expected_stream_id` が同じ stream identity を指す。

## Integration Test

- sample config から `fa_in` node action が生成される。
- disabled `fa_out` は生成されない。
- remappings が launch node に渡される。
- generated system config で `fa_file_out -> fa_file_in` の順に raw PCM file sink/source node を展開し、同一 topic で接続できることを検証する。
- generated system config で `fa_network_out -> fa_network_in` の順に raw PCM UDP sink/source node を展開し、同一 topic で接続できることを検証する。
- generated system config で `fa_file_in -> fa_sample_format -> fa_gain -> fa_file_out` の明示 processing pipeline を接続し、source/sink adapter ではなく processing node で format conversion / gain が行われることを検証する。
- colcon test で package-local test が収集される。
- fixture YAML を `load_system_config()` で読み、`${share:fluent_audio_system}` 展開、missing params file、invalid schema を検証する。
- unit test で `${env:...}` が `config`、`params_file`、inline `parameters` の string / string list 値として展開されることを検証する。
- package-owned SO101 profile config を `load_system_config()` で読み、default enabled node が `fa_in` / `fa_out` だけであることを検証する。
- package-owned SO101 microphone frontend profile config を `load_system_config()` で読み、`fa_in -> fa_sample_format -> fa_resample` の順で enabled node が展開されることを検証する。
- package-owned SO101 TTS output profile config を `load_system_config()` で読み、`fa_tts -> fa_resample -> fa_sample_format -> fa_mix -> fa_out` の logical pipeline が展開され、TTS の `FLOAT32LE/32` が playback 前に 48kHz `PCM16LE/16` へ明示変換されること、OpenJTalk dictionary path が `${env:FLUENT_AUDIO_OPENJTALK_DICT_DIR}` から明示展開されることを検証する。
- package-owned SO101 TTS output profile config は `FLUENT_AUDIO_OPENJTALK_DICT_DIR` 未設定時に起動前 config 展開で `RuntimeError` になることを検証する。
- package-owned sample config が default enabled node を `fa_in` / `fa_out` / `fa_sample_format` / `fa_resample` にし、`PCM16LE` を `fa_resample` へ直結しないことを検証する。disabled TTS playback sample としては `fa_tts -> fa_resample_tts -> fa_sample_format_tts -> fa_mix` の明示変換 parameters を持つことを検証する。
- `colcon test --packages-select fluent_audio_system` は package-local `setup.cfg` により壊れた ambient pytest plugin を読まずに実行する。
- required package CLI が `--config` から system config を読み、1 行 1 package で base package と enabled node package を出力することを検証する。
- required package CLI が missing config / invalid schema / enabled node の `params_file` 欠落を成功扱いしないことを検証する。
- required package CLI が source-tree の package-owned profile から、未 build package の `${share:...}` params を解決せず enabled package list を出せることを検証する。
- `io` group が `fa_file_in` / `fa_network_in` / `fa_file_out` / `fa_network_out` を含む IO package を受け付けることを検証する。
- sample / SO101 profile の VAD frontend stream binding が `VadState.stream_id` 契約と一致することを検証する。

## Launch Test

- `ros2 launch fluent_audio_system run.py config:=...` が system launcher を include する。
- `ros2 launch fluent_audio_system run.py config:=... fa_in_enabled:=true fa_in_source_id:=hw:...` が `fa_in` に raw source id override を `audio.device_selector.mode=id` で渡す。
- `fa_in_source_id` が指定された場合に `fa_asr` / `fa_kws` / `fa_turn_detector` へ `expected_source_id` override を渡す。
- launch file は `Node` action に params/remappings を直接渡し、temporary YAML の生成や書き換えをしない。
- `setup.py` が `list_required_packages` console script を install する。
- `ros2 launch fluent_audio_system fluent_audio_system.launch.py config:=<generated-system-yaml>` で `fa_file_in -> fa_file_out` を接続し、raw PCM bytes が同一 payload として file sink に到達することを検証する。
- `ros2 launch fluent_audio_system fluent_audio_system.launch.py config:=<generated-system-yaml>` で `fa_network_in -> fa_network_out` を接続し、raw PCM UDP packet が同一 payload として network sink に到達することを検証する。
- `ros2 launch fluent_audio_system fluent_audio_system.launch.py config:=<generated-system-yaml>` で `fa_file_in -> fa_sample_format -> fa_gain -> fa_file_out` を接続し、PCM16 入力が `FLOAT32LE/32` へ明示変換され、gain 後の raw float32 bytes が file sink に到達することを検証する。
