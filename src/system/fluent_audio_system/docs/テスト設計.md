# fluent_audio_system テスト設計

## Unit Test

- `FA-SYS-SPEC-001`: missing config path で `RuntimeError`。
- `FA-SYS-SPEC-002`: YAML root が mapping でなければ `RuntimeError`。
- `FA-SYS-SPEC-003`: enabled node に `params_file` も `parameters` もなければ `RuntimeError`。
- `FA-SYS-SPEC-004`: missing params file で `RuntimeError`。
- `FA-SYS-SPEC-005`: remappings schema を検証する。
- `FA-SYS-SPEC-006`: `${share:...}` path を展開する。
- `FA-SYS-SPEC-007`: `fa_in_enabled` / `fa_out_enabled` が `true` / `false` 以外なら `RuntimeError`。
- `FA-SYS-SPEC-008`: site binding override は `fa_in` / `fa_out` の source/sink id だけを parameters に追加する。
- `FA-SYS-SPEC-009`: system config は Pydantic v2 model で境界検証する。

## Integration Test

- sample config から `fa_in` node action が生成される。
- disabled `fa_out` は生成されない。
- remappings が launch node に渡される。

## Launch Test

- `ros2 launch fluent_audio_system run.py config:=...` が system launcher を include する。
- `ros2 launch fluent_audio_system run.py config:=... fa_in_enabled:=true fa_in_source_id:=hw:...` が `fa_in` に source id override を渡す。

## Fixtures

- valid minimal system config
- missing params file config
- invalid schema config
- remapping config
