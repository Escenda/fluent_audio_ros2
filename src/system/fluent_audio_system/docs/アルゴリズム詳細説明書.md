# fluent_audio_system アルゴリズム詳細説明書

1. `config` launch argument を解決する。
2. site binding launch argument を検証する。`fa_in_enabled=true` なら `fa_in_source_id`、`fa_out_enabled=true` なら `fa_out_sink_id` を必須にする。
3. `${share:package}` / `${env:VAR_NAME}` を含む config path を展開し、YAML file の存在を確認する。env が未設定または空の場合は失敗する。
4. YAML root, `system`, `groups`, `nodes` を Pydantic v2 model として検証する。
5. `analysis` group に VAD / KWS / ASR / Turn Detector などの AI package が含まれていないことを検証する。
6. disabled group / node を skip する。
7. enabled node の `package`, `exec`, `node_name`, `namespace`, `params_file`, `parameters`, `remappings` を検証する。
8. `${share:package}` / `${env:VAR_NAME}` を含む `params_file` を展開し、指定されている場合は存在確認する。
9. inline `parameters` の string / string list 値に含まれる `${share:package}` / `${env:VAR_NAME}` を展開する。env が未設定または空の場合は失敗する。
10. `params_file` と inline `parameters` を `launch_ros.actions.Node.parameters` に渡す。
11. `remappings` を `launch_ros.actions.Node.remappings` に渡す。schema は `{source_topic: target_topic}` の mapping 形式だけを受け付ける。
12. `TimerAction` で node 起動間隔を付ける。

YAML を書き換えたり、一時 params file を生成したりしません。

## Required Package List

`list_required_packages` CLI は launch 展開と同じ `load_system_config()` を使います。これにより、invalid schema、missing `params_file`、`${share:...}` / `${env:...}` 解決失敗は package list 生成時にも fail closed になります。

package list 生成手順:

1. `--config` で指定された system YAML を検証済み `AudioSystemSpec` に変換する。
2. base package として `fa_interfaces`、`fluent_audio_system` を追加する。
3. enabled group / enabled node の展開順で `node.package` を走査する。
4. すでに出力済みの package は重複出力しない。
5. 1 行 1 package で標準出力へ書く。

disabled node は `AudioSystemSpec` に入らないため、build package list にも入りません。validation に失敗した場合は標準エラーへ理由を出し、exit code `2` で終了します。
