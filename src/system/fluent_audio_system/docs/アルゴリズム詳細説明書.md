# fluent_audio_system アルゴリズム詳細説明書

1. `config` launch argument を解決する。
2. site binding launch argument を検証する。`fa_in_enabled=true` なら `fa_in_source_id`、`fa_out_enabled=true` なら `fa_out_sink_id` を必須にする。
3. `${share:package}` を含む config path を展開し、YAML file の存在を確認する。
4. YAML root, `system`, `groups`, `nodes` を Pydantic v2 model として検証する。
5. disabled group / node を skip する。
6. enabled node の `package`, `exec`, `node_name`, `namespace`, `params_file`, `parameters`, `remappings` を検証する。
7. `${share:package}` を含む `params_file` を展開し、指定されている場合は存在確認する。
8. `params_file` と inline `parameters` を `launch_ros.actions.Node.parameters` に渡す。
9. `remappings` を `launch_ros.actions.Node.remappings` に渡す。`{"from": "...", "to": "..."}` 形式と `["from", "to"]` pair 形式を正規化して扱う。
10. `TimerAction` で node 起動間隔を付ける。

YAML を書き換えたり、一時 params file を生成したりしません。
