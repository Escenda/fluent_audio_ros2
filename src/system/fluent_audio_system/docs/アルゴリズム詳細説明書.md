# fluent_audio_system アルゴリズム詳細説明書

1. `config` launch argument を解決する。
2. site binding launch argument を検証する。`fa_in_enabled=true` なら `fa_in_source_id`、`fa_out_enabled=true` なら `fa_out_sink_id` を必須にする。
3. `${share:package}` / `${env:VAR_NAME}` を含む config path を展開し、YAML file の存在を確認する。env が未設定または空の場合は失敗する。
4. YAML root, `system`, `groups`, `nodes` を Pydantic v2 model として検証する。
5. `analysis` group に VAD / KWS / ASR / Turn Detector などの AI package が含まれていないことを検証する。
6. disabled group / node を skip する。
7. enabled node の `package`, `exec`, `node_name`, `namespace`, `params_file`, `parameters`, `remappings` を検証する。
8. enabled node の `params_file` は必須とし、`${share:package}` / `${env:VAR_NAME}` を展開して実体が存在しなければ失敗する。
9. `params_file` 内で `node_name`、`package`、`/**` の順に `ros__parameters` block を探す。どの root にも存在しない場合、空 file の場合、または別 node 用 params file だった場合は失敗する。
10. inline `parameters` の string / string list 値に含まれる `${share:package}` / `${env:VAR_NAME}` を展開する。env が未設定または空の場合は失敗する。
11. package 別 parameter contract で `params_file` の `ros__parameters` と inline `parameters` 内の ROS topic key / stream identity key を照合し、同じ値または先頭 `/` の有無だけが違う値を拒否する。
12. runtime backend package は `params_file` と inline `parameters` を重ねた effective `backend.name` を必須にし、未指定または空なら失敗する。対象は `fa_in` / `fa_out` / codec backend、correction backend、deterministic analysis backend、AI backend、TTS backend とする。
13. site binding 由来の `fa_in_source_id` を `fa_in` の `audio.device_selector.mode=id` / `audio.device_selector.identifier` override に渡す。
14. site binding 由来の `fa_in_source_id` を source-bound audio AI node の `expected_source_id` に渡す。
15. stream identity は system config の各 node parameters で固定し、`fa_vad.input_topic` と VAD consumer の audio / expected stream を同じ値に揃える。
16. `params_file` と inline `parameters`、site binding override を `launch_ros.actions.Node.parameters` に渡す。
17. `remappings` を `launch_ros.actions.Node.remappings` に渡す。schema は `{source_topic: target_topic}` の mapping 形式だけを受け付ける。
18. `TimerAction` で node 起動間隔を付ける。

YAML を書き換えたり、一時 params file を生成したりしません。

## Required Package List

`list_required_packages` CLI は pre-build package resolver です。launch 展開用の `load_system_config()` とは違い、enabled node の package set を決めるために必要な schema と category だけを検証し、node `params_file` の実体確認や node parameter 内の `${share:...}` / `${env:...}` 展開は行いません。

package list 生成手順:

1. `--config` で指定された system YAML の存在を確認して読み込む。
2. YAML root / `system` / `groups` / `nodes` を Pydantic v2 model として検証する。
3. enabled node に `package`, `exec`, `params_file` が明示されていることを検証する。
4. group category と package category を検証する。
5. base package として `fa_interfaces`、`fluent_audio_system` を追加する。
6. enabled group / enabled node の展開順で `node.package` を走査する。
7. すでに出力済みの package は重複出力しない。
8. 1 行 1 package で標準出力へ書く。

disabled node は build package list に入りません。validation に失敗した場合は標準エラーへ理由を出し、exit code `2` で終了します。
