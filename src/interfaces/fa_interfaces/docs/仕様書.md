# fa_interfaces 仕様書

## 目的

`fa_interfaces` は FluentAudio の ROS2 graph を横断する message / service 契約を定義する package である。音声 frame、VAD 状態、wake word、ASR、turn end、playback 完了、device 操作、録音、TTS の境界型をここに集約する。

## 仕様 ID

| ID | 内容 |
| --- | --- |
| `FA-INTERFACES-SPEC-001` | `fa_interfaces` は message / service 契約だけを提供し、runtime node、parameter、backend selection を持たない。 |
| `FA-INTERFACES-SPEC-002` | `AudioFrame` は PCM waveform payload と source / stream / format metadata 専用であり、RMS / peak / VAD / ASR / KWS / turn detector 結果を持たない。 |
| `FA-INTERFACES-SPEC-003` | `EncodedAudioChunk` は codec encode 済み payload 専用であり、PCM sample representation、bit depth、layout を推測で持ち込まない。 |
| `FA-INTERFACES-SPEC-004` | `LogMelFrame`、`CqtFrame`、`MfccFrame`、`StftFrame` は deterministic feature extraction output であり、AI 推論結果を表さない。 |
| `FA-INTERFACES-SPEC-005` | `LoudnessFrame`、`OnsetFrame`、`PitchFrame`、`TempoFrame` は deterministic measurement output であり、VAD / KWS / ASR / TD 結果を表さない。 |
| `FA-INTERFACES-SPEC-006` | `VadState` は `AudioFrame.source_id` / `AudioFrame.stream_id` と同じ identity を保持し、consumer は一致しない state を reject する。 |
| `FA-INTERFACES-SPEC-007` | `PlaybackControl` は `fa_out` の output lifecycle 制御だけを表し、音声処理 request field を持たない。 |
| `FA-INTERFACES-SPEC-008` | message / service の field は後段で勝手に補完せず、外部入力を受ける node が境界で検証し fail closed または frame drop する。 |
| `FA-INTERFACES-SPEC-009` | interface 定義は `rosidl_default_generators` に登録され、build 時に生成される。 |

## 入力 topic

なし。この package は interface 定義のみを提供し、node を起動しない。

## 出力 topic

なし。publish / subscribe は各 node package の責務である。

## 提供する message

| message | 用途 |
| --- | --- |
| `AudioFrame` | PCM 音声 chunk と source / stream / sample rate / channel / bit depth / encoding / layout metadata |
| `CqtFrame` | `AudioFrame` から抽出した constant-Q complex spectrum。AI 推論結果ではなく非 AI feature extraction の出力 |
| `EncodedAudioChunk` | codec encode 済み audio chunk と source / stream / codec / container / media timestamp metadata |
| `LoudnessFrame` | `AudioFrame` から測定した frame RMS / peak / dBFS。AI 推論結果ではなく非 AI measurement の出力 |
| `StftFrame` | `AudioFrame` から抽出した STFT complex spectrum。AI 推論結果ではなく非 AI feature extraction の出力 |
| `LogMelFrame` | `AudioFrame` から抽出した log-mel feature matrix。AI 推論結果ではなく非 AI feature extraction の出力 |
| `MfccFrame` | `AudioFrame` から抽出した MFCC feature matrix。AI 推論結果ではなく非 AI feature extraction の出力 |
| `OnsetFrame` | `AudioFrame` から測定した onset strength / onset flag。AI 推論結果ではなく非 AI event measurement の出力 |
| `PitchFrame` | `AudioFrame` から測定した fundamental frequency / voiced flag。AI 推論結果ではなく非 AI pitch measurement の出力 |
| `TempoFrame` | `AudioFrame` から測定した tempo / onset envelope。AI 推論結果ではなく非 AI tempo measurement の出力 |
| `VadState` | VAD の状態遷移、確信度、判定元 audio source / stream identity |
| `WakeWordResult` | KWS の検出結果 |
| `AsrResult` | ASR の partial / final transcript |
| `TurnContext` | session / turn 単位の音声処理 context |
| `TurnEnd` | turn detector の発話終了判定 |
| `PlaybackDone` | 出力再生完了通知 |

## 提供する service

| service | 用途 |
| --- | --- |
| `ListDevices` | source / sink backend の device 列挙。capability が安全に確定できない場合、`max_input_channels` / `default_sample_rates` は `0` |
| `PlaybackControl` | `fa_out` の stop / pause / resume 制御。stop は active epoch を進め、旧 epoch frame を stale として扱う |
| `SwitchDevice` | `target_selector_mode` で `id` / `name` / `index` を明示する device 切替 |
| `Record` | 録音開始 / 停止 |
| `Speak` | TTS 合成要求 |
| `TranscribeAudio` | numeric time range の音声を ASR-ready timeline から文字起こしする要求。自然言語 question は受け取らない |

## parameters

なし。interface package は runtime parameter を持たない。

## lifecycle

build 時に `rosidl_default_generators` で interface を生成する。runtime node は存在しない。

## QoS

QoS は message 型ではなく各 publisher / subscriber が決める。

## error handling

message / service の field を後段で勝手に補完しない。外部入力を受ける node は、境界で必須 field と format を検証し、不正な payload は fail closed または frame drop として扱う。

`AudioFrame` は波形 payload 専用である。RMS / peak / loudness / VAD / ASR / KWS / turn detector の結果は専用 message または専用 node の出力に分離する。

`EncodedAudioChunk` は codec encode 済み payload 専用である。`codec`、`container`、`payload_format` は空にせず、`sample_rate` / `channels` / `duration_ns` / `data` は後段 decoder が検証できる値を保持する。`header.stamp` は ROS graph 上の publish 時刻または capture 時刻、`media_time_ns` は `stream_id` と `epoch` 内の media timeline offset として扱う。PCM sample representation、bit depth、interleaved / planar layout は `AudioFrame` 側の契約であり、encoded payload へ推測で持ち込まない。

`LoudnessFrame` は測定値 payload 専用である。`rms` と `peak` は normalized full-scale amplitude、`rms_dbfs` と `peak_dbfs` は `db_floor` で下限を固定した dBFS 値、`crest_factor` は `peak / rms` である。これは LUFS / K-weighted loudness ではなく、`src/processing/analysis` の deterministic frame meter から publish される。

`VadState` は `AudioFrame.source_id` / `AudioFrame.stream_id` と同じ identity を保持する。VAD consumer は topic subscription だけを根拠に state を受け入れず、対応する audio stream と一致しない `VadState` を reject する。

`TranscribeAudio` は `time_range_spec` と `audio_scope` だけを request に持つ。`time_range_spec` の解決、ASR-ready timeline の保持、backend 呼び出し可否判断は service owner の責務であり、interface package は runtime fallback を持たない。failure は `error_code` に明示し、`window_not_found` は ASR-ready timeline が空で requested range を判定できない状態、`range_outside_window` は requested range が retention window 外の状態、`range_not_continuous` は保持済み frame はあるが gap により requested range を連続 coverage できない状態を表す。timeline failure は空 clip や別 range への置き換えを意味しない。

`LogMelFrame` は特徴量 payload 専用である。`layout` は `frames_by_mels` とし、`values` は frame-major row order、つまり `frame_count * n_mels` 個の `float32` を保持する。`LogMelFrame` は VAD / KWS / ASR / Turn Detector の推論結果を表さず、`src/processing/analysis` の deterministic feature extraction node から publish される。

`CqtFrame` は特徴量 payload 専用である。`layout` は `frames_by_bins`、`value_format` は `complex_cartesian` とし、`real` / `imag` は frame-major row order、つまりそれぞれ `frame_count * bin_count` 個の `float32` を保持する。`center_frequencies_hz` と `window_lengths` は各 CQT bin の kernel contract を保持する。`CqtFrame` は VAD / KWS / ASR / Turn Detector の推論結果を表さず、`src/processing/analysis` の deterministic feature extraction node から publish される。

`MfccFrame` は特徴量 payload 専用である。`layout` は `frames_by_coefficients` とし、`values` は frame-major row order、つまり `frame_count * n_mfcc` 個の `float32` を保持する。MFCC は log-mel power に DCT-II を適用した deterministic feature であり、VAD / KWS / ASR / Turn Detector の推論結果を表さない。

`OnsetFrame` は onset measurement payload 専用である。`layout` は `frames` とし、`frame_times_sec`、`strengths`、`detected` はいずれも `frame_count` 個の frame-major sequence を保持する。`detected` は deterministic spectral flux の閾値判定結果であり、VAD / KWS / ASR / Turn Detector の推論結果や確率を表さない。

`PitchFrame` は pitch measurement payload 専用である。`layout` は `frames` とし、`frame_times_sec`、`frequencies_hz`、`confidence`、`voiced` はいずれも `frame_count` 個の frame-major sequence を保持する。`confidence` は autocorrelation peak ratio であり、VAD / KWS / ASR / Turn Detector の推論結果や確率を表さない。

`TempoFrame` は tempo measurement payload 専用である。`layout` は `frames` とし、`frame_times_sec`、`onset_envelope`、`beats` はいずれも `frame_count` 個の frame-major sequence を保持する。`tempo_detected=false` は入力 frame 内で `confidence_threshold` を満たす tempo evidence が無いことを表し、device / format / backend failure の fallback ではない。

`StftFrame` は特徴量 payload 専用である。`layout` は `frames_by_bins`、`value_format` は `complex_cartesian` とし、`real` / `imag` は frame-major row order、つまりそれぞれ `frame_count * bin_count` 個の `float32` を保持する。`bin_count` は `n_fft / 2 + 1` である。`StftFrame` は VAD / KWS / ASR / Turn Detector の推論結果を表さず、`src/processing/analysis` の deterministic feature extraction node から publish される。

## backend selection

なし。backend は各 node package の責務であり、interface package は backend を知らない。

## non-goals

- node 起動
- audio processing
- backend runtime の依存管理
- package 固有 parameter の定義
