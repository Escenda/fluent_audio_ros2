# fa_interfaces テスト設計

## traceability

| test ID | 対応仕様 | 観点 |
| --- | --- | --- |
| `FA-INTERFACES-TC-001` -> `FA-INTERFACES-SPEC-002` | waveform boundary | `AudioFrame.msg` が waveform payload 専用 field だけを持つこと。 |
| `FA-INTERFACES-TC-002` -> `FA-INTERFACES-SPEC-003` | encoded payload boundary | `EncodedAudioChunk.msg` が codec payload 専用 field だけを持ち、PCM representation を持たないこと。 |
| `FA-INTERFACES-TC-003` -> `FA-INTERFACES-SPEC-011` | rosidl registration | interface 定義が `rosidl_default_generators` の生成対象に登録されていること。 |
| `FA-INTERFACES-TC-004` -> `FA-INTERFACES-SPEC-009` | playback lifecycle only | `PlaybackControl.srv` が output lifecycle 制御だけを表し、音声処理 request field を持たないこと。 |
| `FA-INTERFACES-TC-005` -> `FA-INTERFACES-SPEC-004` | deterministic feature messages | `LogMelFrame`、`CqtFrame`、`MfccFrame`、`StftFrame` が feature payload 専用で推論結果 field を持たないこと。 |
| `FA-INTERFACES-TC-006` -> `FA-INTERFACES-SPEC-005` | deterministic measurement messages | `LoudnessFrame`、`OnsetFrame`、`PitchFrame`、`TempoFrame` が measurement payload 専用で推論結果 field を持たないこと。 |
| `FA-INTERFACES-TC-007` -> `FA-INTERFACES-SPEC-006` | keyword spotting result | `WakeWordResult.msg` が keyword / score / detected の検出結果だけを持ち、audio payload を持たないこと。 |
| `FA-INTERFACES-TC-008` -> `FA-INTERFACES-SPEC-010` | downstream validation | sample rate / channel / data size の不整合や missing field を補完せず、各 node 側の validation で検出すること。 |
| `FA-INTERFACES-TC-009` -> `FA-INTERFACES-SPEC-001` | interface-only package | interface package 自体には launch test / runtime backend が存在しないこと。 |
| `FA-INTERFACES-TC-010` -> `FA-INTERFACES-SPEC-008` | audio window failure taxonomy | `ArchiveAudioWindow.srv` / `ExportAudioWindow.srv` が retained window / range / format failure を明示 error code として定義し、service owner が fallback なしに判定できること。`range_not_continuous` は現在 `ArchiveAudioWindow` 側の error code として扱う。 |

## unit test

- message / service 生成物が build できること。
- `AudioFrame.msg` が waveform payload field のみを持ち、RMS / peak / result field を持たないこと。
- `EncodedAudioChunk.msg` が codec payload field のみを持ち、PCM sample representation field を持たないこと。
- `PlaybackControl.srv` が `fa_out` の output lifecycle 制御だけを表し、音声処理 request field を持たないこと。
- `ArchiveAudioWindow.srv` / `ExportAudioWindow.srv` が空 window、retention miss、unsupported format を同じ failure に潰さないこと。
- `ArchiveAudioWindow.srv` が保持済み frame の gap を `range_not_continuous` として区別できること。
- `AudioFrame` の必須 field を使う downstream validation test を各 node 側に置くこと。

## integration test

- `AudioFrame` を publish / subscribe する最小 graph を作り、field が欠落せず伝搬すること。
- `fa_encode` / `fa_decode` 実装時に `EncodedAudioChunk` を publish / subscribe する最小 graph を作り、codec / container / media timestamp が欠落せず伝搬すること。
- KWS / turn message が system config の topic wiring と一致すること。
- audio window archive / export service が requested range と explicit error code を維持すること。

## launch test

interface package 自体には launch test は不要。interface を利用する node package の launch test で検証する。

## failure test

- sample rate / channel / data size の不整合を downstream node が検出できること。
- missing field を正常系に見せる補完を入れていないこと。

## latency test

interface package 単体では対象外。serialization cost が問題になる場合は graph-level benchmark で測る。

## real device smoke test

`fa_in` から `fa_out` までの smoke test で、`AudioFrame` contract が実デバイス経路でも成立することを確認する。
