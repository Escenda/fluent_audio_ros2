# fa_denoise アルゴリズム詳細説明書

## 処理手順

```text
AudioFrame
  ↓
validate 16kHz mono PCM
  ↓
decode to float32
  ↓
split into DTLN blocks
  ↓
run model_1 / model_2 ONNX inference
  ↓
overlap-add
  ↓
encode output AudioFrame
```

## DTLN 前提

公開 DTLN model の contract に合わせ、次を固定条件とする。

- sample rate: 16kHz
- channels: mono
- `block_len`: 512
- `block_shift`: 128

## frame / buffer 前提

input data は PCM16LE または float32 の frame boundary に一致すること。backend 入力は normalized float32 に変換する。

## known limitations

- DeepFIR model は未接続。
- `passthrough` は debug / wiring 用であり、denoise 代替ではない。
- current DTLN backend は 16kHz mono 専用。
