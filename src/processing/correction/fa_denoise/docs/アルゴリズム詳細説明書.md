# fa_denoise アルゴリズム詳細説明書

## 処理手順

```text
AudioFrame
  ↓
ROS adapter: validate header stamp, input_stream_id, 16kHz mono explicit encoding/bit-depth pair
  ↓
ROS-free backend input: AudioBuffer(bytes + AudioFormat)
  ↓
backend contract: decode PCM16LE/FLOAT32LE to normalized float32
  ↓
dtln_onnx backend: validate sample count aligned to dtln.block_shift
  ↓
split into DTLN blocks
  ↓
run model_1 / model_2 ONNX inference
  ↓
DTLN model-internal overlap-add reconstruction
  ↓
validate output sample count matches input sample count
  ↓
backend contract: validate output range and encode to configured output AudioFormat
  ↓
ROS adapter: publish AudioFrame with original header/source/epoch and output stream_id
```

この overlap-add は `dtln_onnx` backend 内部の model reconstruction である。normalized PCM samples を DTLN の `block_len` / `block_shift` contract で再構成し、同じ sample count の denoised PCM samples を返すだけで、transport stabilization ではない。overlapped `AudioFrame` chunk、timestamp / epoch / gap handling、jitter buffer、clock drift correction は扱わず、`src/streaming/fa_overlap_add` を置き換えない。

## DTLN 前提

公開 DTLN model の contract に合わせ、次を固定条件とする。

- sample rate: 16kHz
- channels: mono
- `block_len`: 512
- `block_shift`: 128
- ONNX model_1 の mask/state output shape は model_1 input shape と一致すること。
- ONNX model_2 の time/state output shape は model_2 input shape と一致すること。
- ONNX tensor shape は fully static / positive だけを許可する。dynamic dimension を `1` として推測しない。

## frame / buffer 前提

input `stream_id` は `input_stream_id` に一致すること。ROS topic 名は transport identity として扱い、AudioFrame の stream identity とは分離する。input data は `expected.encoding` / `expected.bit_depth` の pair に一致すること。`PCM16LE/16` または `FLOAT32LE/32` だけを受け付け、`PCM32LE/32` を float32 として解釈しない。ROS node は `AudioFrame` を ROS-free `AudioBuffer` へ写し、decode / encode は backend contract 側で行う。
input sample count は `dtln.block_shift` に揃える。現在の node は入力 header / epoch を維持して 1 input frame に対して 1 output frame を publish するため、DTLN output sample count が input sample count と一致しない場合は fail closed にする。
output data は `output.encoding` / `output.bit_depth` の pair に一致すること。model output が `[-1.0, 1.0]` を超える場合は clamp せず frame を drop し、range control は明示的な dynamics node に分離する。

## known limitations

- DeepFIR model は未接続。
- `passthrough` は debug / wiring 用であり、denoise 代替ではない。
- current DTLN backend は 16kHz mono 専用。
