# fa_denoise 仕様書

## 目的

`fa_denoise` は background noise を抑える correction node である。現在は DTLN ONNX backend を既定 backend とし、16kHz mono audio frame を denoise して publish する。

## 仕様 ID

| ID | 内容 |
| --- | --- |
| `FA-DENOISE-SPEC-001` | `backend.name` は必須で、`dtln_onnx` または明示 debug 用 `passthrough` だけを許可する |
| `FA-DENOISE-SPEC-002` | `dtln_onnx` は model path、ONNX Runtime availability、block length/shift、thread count、fully static / positive ONNX input/output tensor shape を起動時に検証する |
| `FA-DENOISE-SPEC-003` | 入力 frame は non-zero `header.stamp`、`input_topic` と同じ `stream_id`、16kHz mono、`PCM16LE/16` または `FLOAT32LE/32`、interleaved layout、非空 PCM frame boundary を満たす |
| `FA-DENOISE-SPEC-004` | `dtln_onnx` 入力は `dtln.block_shift` 境界に揃え、出力 sample count は入力 sample count と一致しなければならない |
| `FA-DENOISE-SPEC-005` | model output は finite な normalized `[-1.0, 1.0]` のみを許可し、hidden clamp は行わない |
| `FA-DENOISE-SPEC-006` | `enabled=false` は pass-through ではなく drop として扱う。node を使わない場合は system config 側で起動しない |
| `FA-DENOISE-SPEC-007` | publisher/backend 欠落など起動時 invariant の破損は runtime drop loop にせず fail closed する |

## 入力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `input_topic` | `audio/resample16k/mic` | denoise 対象 frame |

受信 frame の `stream_id` は `input_topic` と一致しなければならない。topic だけでなく frame 内の stream identity も一致させ、別経路の frame を同じ message 型だからという理由で処理しない。

## 出力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `output_topic` | `audio/denoise/frame` | denoise 後 frame |

## parameters

| parameter | 意味 |
| --- | --- |
| `enabled` | node 有効化。false の場合は frame を publish しない。system config では node 自体を disable する |
| `backend.name` | `dtln_onnx` or 明示 debug 用 `passthrough` |
| `expected_sample_rate` | 16kHz |
| `expected_channels` | 1 |
| `expected.encoding` | 入力 encoding。`PCM16LE` or `FLOAT32LE` |
| `expected.bit_depth` | 入力 bit depth。encoding と pair で検証 |
| `output.encoding` | 出力 encoding |
| `output.bit_depth` | 出力 bit depth |
| `dtln.block_len` | DTLN block length |
| `dtln.block_shift` | DTLN block shift |
| `dtln.model_1_path` | DTLN model 1 path。`dtln_onnx` では必須 |
| `dtln.model_2_path` | DTLN model 2 path。`dtln_onnx` では必須 |
| `dtln.*threads` | ONNX Runtime thread 設定 |
| `qos.*` | audio frame QoS |
| `diagnostics.publish_period_ms` | diagnostics publish 周期 |
| `diagnostics.qos.depth` | diagnostics QoS depth |
| `diagnostics.qos.reliable` | diagnostics QoS reliability |

## lifecycle

起動時に backend 名、topic、expected format、model path、ONNX Runtime availability、DTLN block 設定、ONNX input/output tensor shape を検証する。DTLN backend が選択されているのに model path が空、model file が存在しない、runtime が build に含まれていない、または model tensor shape が contract と一致しない場合は起動失敗する。空 model path から package share の model を推測しない。resolved `input_topic` と `output_topic` が同一の場合も、loopback を暗黙に作らず起動失敗する。

## QoS

input/output は同一 QoS で扱う。

## error handling

invalid frame、`enabled=false`、unsupported backend、missing model、ONNX inference failure、output range overflow、block_shift 不整合、output sample count mismatch は成功扱いにしない。encoding と bit depth の不一致、または `stream_id` と `input_topic` の不一致は推測変換・経路補正しない。publisher/backend 欠落は起動時 invariant の破損なので、drop loop にせず fail closed する。

## backend selection

`backend.name` parameter で選択する。backend は ROS2 topic / `AudioFrame` / diagnostics を知らず、`AudioFormat` と PCM bytes で構成される ROS-free `AudioBuffer` を受け取る。decode / encode、block alignment、output range validation は backend contract の責務である。`passthrough` は wiring / diagnostics 確認用であり、denoise backend の代替として default/system 経路では使わない。`passthrough` は入力と出力の sample rate / channels / encoding / bit depth が一致する場合だけ許可し、暗黙の format conversion として使わない。

`FA_DENOISE_ONNXRUNTIME=AUTO` では、ONNX Runtime が見つからなくても package build は成立する。この場合も `dtln_onnx` が選択された node は起動時に fail closed し、`passthrough` や別 backend へ暗黙に切り替えない。ONNX Runtime を必須にしたい build では `FA_DENOISE_ONNXRUNTIME=ON` を指定する。

## non-goals

- AEC
- beamforming
- VAD
- resampling
- online model download
- implicit model path fallback
- limiter / clipping / hidden range clamp
