# fa_denoise 仕様書

## 目的

`fa_denoise` は background noise を抑える correction node である。現在は DTLN ONNX backend を既定 backend とし、16kHz mono audio frame を denoise して publish する。

## 入力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `input_topic` | `audio/resample16k/mic` | denoise 対象 frame |

## 出力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `output_topic` | `audio/denoise/frame` | denoise 後 frame |

## parameters

| parameter | 意味 |
| --- | --- |
| `enabled` | node 有効化 |
| `backend` | `dtln_onnx` or `passthrough` |
| `expected_sample_rate` | 16kHz |
| `expected_channels` | 1 |
| `output.encoding` | 出力 encoding |
| `output.bit_depth` | 出力 bit depth |
| `dtln.block_len` | DTLN block length |
| `dtln.block_shift` | DTLN block shift |
| `dtln.model_1_path` | DTLN model 1 path。空なら package share の model を使う |
| `dtln.model_2_path` | DTLN model 2 path。空なら package share の model を使う |
| `dtln.*threads` | ONNX Runtime thread 設定 |
| `qos.*` | audio frame QoS |
| `diagnostics.publish_period_ms` | diagnostics publish 周期 |

## lifecycle

起動時に backend 名、topic、expected format、model path、ONNX Runtime availability を検証する。DTLN backend が選択されているのに model / runtime がない場合は起動失敗する。

## QoS

input/output は同一 QoS で扱う。

## error handling

invalid frame、unsupported backend、missing model、ONNX inference failure は成功扱いにしない。

## backend selection

`backend` parameter で選択する。backend は ROS2 topic を知らず、float32 PCM buffer と model config を受け取る。

## non-goals

- AEC
- beamforming
- VAD
- resampling
- online model download
