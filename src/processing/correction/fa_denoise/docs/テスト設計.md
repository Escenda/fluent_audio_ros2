# fa_denoise テスト設計

## unit test

- `backend.name` validation。
- empty model path / missing model file の起動失敗。
- default config は `dtln_onnx` で、model path は空のため明示設定なしでは起動失敗。
- `enabled=false` は pass-through ではなく drop。
- block size と overlap-add の境界条件。
- invalid sample rate / channel / bit depth の drop。
- input `encoding` / `bit_depth` pair が不一致なら drop。
- output range overflow は clamp せず drop。

## integration test

- `fa_resample -> fa_denoise` の topic wiring。
- DTLN model を使った short fixture の inference。

## launch test

- standalone launch は明示 launch argument の model path で起動できる。
- `config/default.yaml` 単体では model path 未指定として起動失敗する。
- ONNX Runtime がない build 環境では configure に失敗する。
- unknown backend で起動失敗する。
- 旧 `backend` key ではなく `backend.name` を使う。
- DTLN runtime は `include/fa_denoise/backends/` と `src/backends/` 配下に置き、ROS2 topic/message を含まない。
- ONNX Runtime 未導入の通常 workspace build では package configure が成功し、`backend.name=dtln_onnx` 選択時の node 起動で fail closed する。
- `FA_DENOISE_ONNXRUNTIME=ON` では ONNX Runtime 欠落時に configure で失敗する。
- `fa_denoise_backends` library を作り、node は backend source を直接含めずその library に link する。

## failure test

- model file missing。
- unsupported frame format。
- inference exception。

## latency test

block 単位の inference latency、queueing、diagnostics counter を測る。

## real device smoke test

実 microphone から `fa_resample -> fa_denoise` を起動し、output frame の sample rate / channel / RMS / drop counter を確認する。
