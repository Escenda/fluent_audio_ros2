# fa_denoise テスト設計

## unit test

| Test ID | 対応仕様 | 内容 |
| --- | --- | --- |
| `FA-DENOISE-TC-001` | `FA-DENOISE-SPEC-001` | missing backend / unknown backend で起動失敗 |
| `FA-DENOISE-TC-002` | `FA-DENOISE-SPEC-002` | empty model path / missing model file / ONNX Runtime 欠落 / invalid tensor shape で起動失敗 |
| `FA-DENOISE-TC-003` | `FA-DENOISE-SPEC-003` | invalid sample rate / channel / encoding / bit depth / input stream_id mismatch / empty data を drop |
| `FA-DENOISE-TC-004` | `FA-DENOISE-SPEC-004` | block_shift 不整合または output sample count mismatch を fail closed として扱う |
| `FA-DENOISE-TC-005` | `FA-DENOISE-SPEC-005` | output range overflow は clamp せず drop |
| `FA-DENOISE-TC-006` | `FA-DENOISE-SPEC-006` | `enabled=false` は publish せず drop |
| `FA-DENOISE-TC-007` | `FA-DENOISE-SPEC-007` | publisher/backend 欠落を runtime drop loop として扱わない |

## integration test

- `fa_resample -> fa_denoise` の topic wiring と `AudioFrame.stream_id` contract。
- DTLN model を使った short fixture の inference。

## launch test

| Test ID | 対応仕様 | 内容 |
| --- | --- | --- |
| `FA-DENOISE-LT-001` | `FA-DENOISE-SPEC-001` | standalone launch は backend などの処理契約を launch 引数で上書きしない |
| `FA-DENOISE-LT-002` | `FA-DENOISE-SPEC-001` | unknown backend で起動失敗する |
| `FA-DENOISE-LT-003` | `FA-DENOISE-SPEC-002` | `config/default.yaml` 単体では model path 未指定または ONNX Runtime 欠落として起動失敗する |
| `FA-DENOISE-LT-004` | `FA-DENOISE-SPEC-002` | `FA_DENOISE_ONNXRUNTIME=ON` では ONNX Runtime 欠落時に configure で失敗する |
| `FA-DENOISE-LT-005` | `FA-DENOISE-SPEC-002` | DTLN runtime は `include/fa_denoise/backends/` と `src/backends/` 配下に置き、ROS2 topic/message を含まない |
| `FA-DENOISE-LT-006` | `FA-DENOISE-SPEC-002` | `fa_denoise_backend_contract` と `fa_denoise_dtln_onnx_backend` library を分け、node は ROS-free backend interface に link する |
| `FA-DENOISE-LT-007` | `FA-DENOISE-SPEC-003` | node core を library 化し、C++ contract test で passthrough publish、stream mismatch drop、unsupported encoding drop、channel wildcard rejection、resolved topic loop rejection、topic/stream identity collision rejection を確認する |
| `FA-DENOISE-LT-008` | `FA-DENOISE-SPEC-003` `FA-DENOISE-SPEC-004` `FA-DENOISE-SPEC-005` | backend C++ contract test で PCM decode/encode、range reject、passthrough format mismatch、ROS-free identity processing を確認する |

## failure test

- model file missing。
- unsupported frame format。
- inference exception。

## latency test

block 単位の inference latency、queueing、diagnostics counter を測る。

## real device smoke test

実 microphone から `fa_resample -> fa_denoise` を起動し、output frame の sample rate / channel / RMS / drop counter を確認する。
