# アルゴリズム詳細説明書

## 責務分離

ROS node は `AudioFrame` metadata validation、topic 入出力、diagnostics、counter 更新だけを持つ。sample decode/encode、channel 別 mean 算出、sample 減算、finite 検証は `internal_frame_mean` backend が持つ。backend は ROS2 topic/message を知らない。

## 入力

入力は `FLOAT32LE` interleaved の `AudioFrame.data` である。`channels = C`、総 sample 数を `N` とすると、`N` は `C` で割り切れる。frame 数は `F = N / C` である。

## 手順

1. node が `AudioFrame` metadata と data size を検証する。
2. backend が `data` を `sizeof(float)` ごとに読み、各 sample を `float` として扱う。
3. 非有限の入力 sample があれば frame 全体を drop する。
4. sample index `i` の channel を `i % C` として、channel ごとの合計値を `double` で計算する。
5. 各 channel の平均値を `sum[channel] / F` で計算する。
6. 各 sample から該当 channel の平均値を差し引く。
7. 差し引き後の値が `double` と `float` の両方で finite であることを確認する。
8. `FLOAT32LE` byte列として一時 output vector に書き込む。
9. 全 sample の検証が完了してから output vector を置き換える。

## 数式

channel `c` の入力 sample を `x[c, t]` とする。

```text
mean[c] = (x[c, 0] + x[c, 1] + ... + x[c, F - 1]) / F
y[c, t] = x[c, t] - mean[c]
```

## 境界条件

- `F == 0` になる入力は、runtime frame contract の `data` 非空かつ `channels * sizeof(float)` 割り切り検証により処理前に reject される。
- 非有限入力、非有限平均、非有限出力は frame drop とする。
- clamp、normalize、gain 調整は行わない。
- backend 拒否時は output vector を更新しない。

## 状態

この backend は過去 frame の状態を持たない。DC offset は各 frame 内だけで推定する。
