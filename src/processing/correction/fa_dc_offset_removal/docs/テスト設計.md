# テスト設計

## Python contract tests

`test/unit/test_fa_dc_offset_removal_audio_frame_contract.py` は実装ファイルを検査し、以下を確認する。

- default config が `fa_dc_offset_removal/input` から `fa_dc_offset_removal/output` への ROS 搬送路と、`audio/sample_format/mic` から `audio/dc_offset_removed/mic` への stream identity 契約を分離して持つ
- device I/O、resampling、sample format conversion、gain、limiter、noise gate、high-pass、notch を持たない
- 起動時 config validation が fail closed である
- ROS2 の展開・remap 解決後に `input_topic` と `output_topic` が一致する設定を起動時に拒否し、feedback loop を作らない
- `input_stream_id` / `output.stream_id` の欠損、topic 名との衝突、相互衝突を起動時に拒否する
- `qos.depth` は起動時検証後に clamp されない
- runtime frame validation が source / stream / format / data size を検証する
- `stream_id` は `input_stream_id` と一致要求し、出力では `output.stream_id` へ更新する
- frame 内の channel 別平均計算と sample 減算が backend にあり、node からは消えている
- native `std::memcpy` による `FLOAT32LE` 処理は little-endian target に限定される
- backend が ROS2 topic/message を知らない
- 非有限入力・出力を backend が status で拒否する
- backend 拒否時に output vector を確定しない
- required layout と pytest / gtest registration が存在する

## C++ backend tests

`test/cpp/test_internal_frame_mean_backend.cpp` は ROS を使わず backend を直接検証する。

- 不正 config を constructor で拒否する
- channel ごとの frame 内平均を差し引く
- zero-mean input を変化させない
- 空入力、misaligned input、非 finite input を status で拒否する
- 拒否時に output vector を更新しない

## C++ graph smoke test

`test/cpp/test_dc_offset_removal_graph.cpp` は `FaDcOffsetRemovalNode` を `rclcpp::NodeOptions` で起動し、test publisher / subscriber と同じ executor に載せて検証する。

- `FLOAT32LE/32/interleaved` frame を input topic に publish する
- output topic に DC offset removal 後の frame が publish される
- `source_id`、sample rate、channels、encoding、bit depth、layout、epoch は保持される
- `stream_id` は `output.stream_id` に更新される
- channel 別平均が差し引かれた sample が届く
- 生文字列または ROS2 解決後に input/output topic が同一になる設定、`qos.depth <= 0` は起動時に拒否される

## Launch contract tests

`test/launch/test_fa_dc_offset_removal_launch_contract.py` は launch と default config の契約を検証する。

- launch file は `node_name` と `config_file` だけを受け取り、backend / expected format 詳細を launch argument に漏らさない
- default config は correction node として `FLOAT32LE/32/interleaved` 契約だけを持ち、gain / filter / resample を含まない
- `input_topic` 欠落、生文字列または ROS2 解決後に input/output topic が同一になる設定、`expected.encoding != FLOAT32LE` は launch 経路でも起動失敗する
