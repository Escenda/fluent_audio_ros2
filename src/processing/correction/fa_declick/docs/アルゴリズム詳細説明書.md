# fa_declick アルゴリズム詳細説明書

## 1. 処理対象

処理対象は FLOAT32LE interleaved `AudioFrame` の `data` のみである。metadata は `stream_id` を除いて保持する。

サンプル配列は以下のように解釈する。

```text
sample_index = frame_index * channels + channel_index
```

## 2. 前処理

処理前に frame contract と sample contract を検証する。

- frame metadata が設定値と一致しない場合は drop
- `data` が空、または frame size で割り切れない場合は drop
- 非 finite sample、または `[-1.0, 1.0]` 外の sample がある場合は drop

検証失敗時に zero、identity、latest sample、clamp、normalize は使わない。

## 3. 単一 sample click

default の `window.max_samples=1` では、各 channel の `frame_index=1` から `frame_count-2` までを走査する。

```text
previous = x[t - 1]
current  = x[t]
next     = x[t + 1]

is_click =
  abs(current - previous) > threshold.delta
  and abs(current - next) > threshold.delta
  and abs(previous - next) <= threshold.delta

if is_click:
  y[t] = (previous + next) / 2
else:
  y[t] = x[t]
```

先頭 sample と末尾 sample は previous / next が揃わないため補正対象にしない。

## 4. 連続 sample run

`window.max_samples > 1` の場合は、同じ channel 内の `1..window.max_samples` sample の run を候補にする。

run `[t, t+n-1]` は以下を満たす場合に click run とみなす。

- `n <= window.max_samples`
- `previous = x[t-1]` と `next = x[t+n]` が存在する
- `abs(previous - next) <= threshold.delta`
- run 内の全 sample `x[k]` について `abs(x[k] - previous) > threshold.delta`
- run 内の全 sample `x[k]` について `abs(x[k] - next) > threshold.delta`

補正値は run 全体で `(previous + next) / 2` とする。複数の run length が成立する場合は短い run を採用し、自然な隣接 sample の過剰補正を避ける。

## 5. 出力検証

補正値は書き込み前に finite かつ `[-1.0, 1.0]` 内であることを検証する。範囲外の場合は frame を drop し、clamp は行わない。

## 6. 計算量

`N` を frame 数、`C` を channel 数、`W` を `window.max_samples` とする。走査は概ね `O(N * C * W)` である。default の `W=1` では frame 数と channel 数に対して線形で動作する。

## 7. 境界

この実装は frame 内の previous / next のみを使う。frame をまたいだ状態は保持しない。境界 sample を推測して補正する処理は fallback になるため行わない。
