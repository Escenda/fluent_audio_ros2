# fa_declick テスト設計

## 1. 対象

`fa_declick` の package contract、設定 contract、AudioFrame contract、declick algorithm、ROS2 graph、launch fail-closed 境界を pytest と C++ gtest で検証する。

## 2. 単体テスト

`test/unit/test_fa_declick_audio_frame_contract.py` で以下を確認する。

- default config が FLOAT32LE interleaved contract を要求する
- `threshold.delta` と `window.max_samples` を起動時に fail closed で検証する
- `expected.*`, `qos.depth`, `diagnostics.publish_period_ms` を起動時に検証する
- resolved input/output topic が一致する場合は起動失敗する
- QoS depth を `std::max` などで補正しない
- 入力 frame の `source_id`, `stream_id`, encoding, sample_rate, channels, layout, data size を処理前に検証する
- backend が ROS2 topic/message を知らない
- sample が finite かつ `[-1.0, 1.0]` 内でない場合に backend が explicit status で reject する
- output は `out = in` から `stream_id` と `data` だけを変える
- clamp、normalize、resampling、device I/O を実装に含めない
- diagnostics が設定値、resolved topics、backend name、counters を含む
- package layout と colcon pytest / gtest hook が存在する

## 3. C++ backend テスト

`test/cpp/test_internal_impulse_declick_backend.cpp` で以下を確認する。

- invalid config は例外になる
- mono single sample click を `(previous + next) / 2` に補正する
- `window.max_samples` 内の連続 run を補正する
- stereo channel は独立に扱う
- empty / misaligned / non-finite / out-of-range input を explicit status で reject する
- reject 時に output buffer を上書きしない

## 4. ROS2 graph テスト

`test/cpp/test_fa_declick_graph.cpp` で以下を確認する。

- valid AudioFrame を publish したとき output topic に declick 済み frame が出る
- out-of-range sample が publish されない
- resolved input/output topic が同一の場合に node constructor が起動失敗する

## 5. launch テスト

`test/launch/test_fa_declick_launch_contract.py` で以下を確認する。

- `fa_declick.launch.py` は `node_name` と `config_file` だけを launch argument にし、しきい値や format contract を launch layer に漏らさない
- invalid config でプロセスが fail closed する

## 6. fixture

`test/fixtures` は将来の binary FLOAT32LE fixture 用に予約する。例:

- mono single-click frame
- stereo channel-isolated click frame
- out-of-range sample frame
