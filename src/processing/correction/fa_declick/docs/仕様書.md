# fa_declick 仕様書

## 1. 目的

`fa_declick` は、FluentAudio の FLOAT32LE interleaved 音声フレームに対し、単発または短い連続 impulse click を明示的に補正する。

この補正は失敗を隠す fallback ではない。入力契約を満たす音声サンプルのうち、前後サンプルとの差分条件を満たした click sample だけを変換し、それ以外の sample と metadata は保持する。

## 2. ノード

| 項目 | 値 |
| --- | --- |
| Package | `fa_declick` |
| Executable | `fa_declick_node` |
| Node | `fa_declick` |
| Subscribe | `input_topic` |
| Publish | `output_topic` |

## 3. 入出力契約

入力 `AudioFrame` は以下を満たす必要がある。

- `source_id` が空でない
- `stream_id` が空でない
- `stream_id == input_topic`
- `sample_rate > 0` かつ設定値と一致
- `channels > 0` かつ設定値と一致
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `layout == interleaved`
- `data` が空でない
- `data.size()` が `channels * sizeof(float)` で割り切れる
- 全サンプルが有限で `[-1.0, 1.0]` の範囲内

出力 `AudioFrame` は `header`, `source_id`, `sample_rate`, `channels`, `bit_depth`, `encoding`, `layout`, `epoch` を保持し、`stream_id` を `output_topic` に更新し、`data` を declick 済みサンプルへ置き換える。

## 4. click 判定

各 channel を独立に扱う。単一 sample の click は以下の条件で判定する。

- current sample に同一 channel の previous sample と next sample が存在する
- `abs(current - previous) > threshold.delta`
- `abs(current - next) > threshold.delta`
- `abs(previous - next) <= threshold.delta`

条件を満たした current sample は `(previous + next) / 2` で置き換える。

`window.max_samples` が `2` 以上の場合、同じ条件を連続 sample run に拡張する。run の両端にある previous / next が近く、run 内の全 sample が両端から `threshold.delta` より大きく離れている場合、run 内の sample をすべて `(previous + next) / 2` で置き換える。

## 5. パラメータ

| 名前 | 型 | 既定値 | 制約 |
| --- | --- | --- | --- |
| `input_topic` | string | `audio/noise_gated/mic` | 空不可 |
| `output_topic` | string | `audio/declicked/mic` | 空不可 |
| `threshold.delta` | double | `0.25` | finite かつ `(0.0, 2.0]` |
| `window.max_samples` | int | `1` | `> 0` |
| `expected.sample_rate` | int | `16000` | `> 0` |
| `expected.channels` | int | `1` | `> 0` |
| `expected.encoding` | string | `FLOAT32LE` | `FLOAT32LE` のみ |
| `expected.bit_depth` | int | `32` | `32` のみ |
| `expected.layout` | string | `interleaved` | `interleaved` のみ |
| `qos.depth` | int | `10` | `> 0` |
| `qos.reliable` | bool | `false` | true/false |
| `diagnostics.publish_period_ms` | int | `1000` | `> 0` |

## 6. エラー処理

起動時設定が不正な場合は例外で起動を停止する。ランタイムフレームが契約に違反した場合は warning を出し、そのフレームを drop する。サンプル値の clamp、normalize、暗黙変換は行わない。

出力補正値が有限でない、または `[-1.0, 1.0]` を外れる場合も frame を drop する。

## 7. Diagnostics

`diagnostics` topic に以下を含める。

- `threshold_delta`
- `window_max_samples`
- `input_topic`
- `output_topic`
- `expected_sample_rate`
- `expected_channels`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `samples_corrected`
- `click_runs_corrected`

## 8. 非目標

device I/O、resampling、sample format conversion、channel conversion、gain、normalize、limiter、filter、denoise、declip、decrackle、echo、reverb、legacy compatibility alias は提供しない。
