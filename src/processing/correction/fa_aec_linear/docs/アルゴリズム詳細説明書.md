# fa_aec_linear アルゴリズム詳細説明書

## 処理手順

```text
reference AudioFrame
  ↓
validate format, header.stamp, and stream_id == ref_topic, then cache reference by header.stamp

mic AudioFrame
  ↓
validate format, header.stamp, and stream_id == mic_topic
  ↓
validate expected encoding / bit-depth pair
  ↓
if enabled and reference timestamp is within contract:
      baseline_linear backend decodes mic/ref to float32
      baseline_linear backend subtracts cancel_gain * ref
      baseline_linear backend validates output range and encodes to original explicit format
  else:
      drop according to explicit enabled / timeout semantics
```

## cancellation

同じ sample layout の mic / ref を前提に、sample ごとに次を計算する。

```text
out[i] = mic[i] - cancel_gain * ref[i]
```

入力は `expected.encoding` / `expected.bit_depth` の pair と一致しなければならない。出力時は mic の explicit encoding / bit depth pair を維持する。`PCM16LE/16` と `FLOAT32LE/32` のみを受け付け、`PCM32LE/32` を float32 として解釈しない。計算結果が `[-1.0, 1.0]` を超える、または finite でない場合は backend が explicit status を返し、node が frame を drop する。range control は `fa_limiter` などの dynamics node を明示的に pipeline に入れる。

`baseline_linear` backend は ROS2 topic、ROS2 message、parameter file を知らない。入力は設定済みの `channels` / `encoding` / `bit_depth` / `cancel_gain` と mic/reference byte buffer だけである。

## frame / buffer 前提

- sample rate は 16kHz。
- `expected_channels` は 1 以上であり、channel 検査の wildcard はない。
- format pair は `expected.encoding` / `expected.bit_depth` に一致する `PCM16LE/16` または `FLOAT32LE/32`。
- mic/ref は同一 channel 数であること。
- mic/ref は同一 encoding / bit depth / sample count であること。
- mic `stream_id` は `mic_topic`、reference `stream_id` は `ref_topic` と一致すること。
- mic/reference の `header.stamp` は非ゼロで、reference は mic と同時刻または過去であり、mic timestamp との差分が `ref_timeout_ms` 内であること。
- resolved `mic_topic` / `ref_topic` / `output_topic` は相互に異なること。

## known limitations

- adaptive filter ではない。
- echo path delay 推定をしない。
- reference と mic の frame alignment がずれると効果は限定的。
