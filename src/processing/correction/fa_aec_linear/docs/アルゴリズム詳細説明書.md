# fa_aec_linear アルゴリズム詳細説明書

## 処理手順

```text
reference AudioFrame
  ↓
validate and cache latest reference

mic AudioFrame
  ↓
validate
  ↓
if enabled and fresh reference exists:
      decode mic/ref to float32
      subtract cancel_gain * ref
      validate output range and encode to original explicit format
  else:
      drop according to explicit enabled / timeout semantics
```

## cancellation

同じ sample layout の mic / ref を前提に、sample ごとに次を計算する。

```text
out[i] = mic[i] - cancel_gain * ref[i]
```

出力時は mic の explicit encoding / bit depth pair を維持する。`PCM16LE/16` と `FLOAT32LE/32` のみを受け付け、`PCM32LE/32` を float32 として解釈しない。計算結果が `[-1.0, 1.0]` を超える、または finite でない場合は clamp せず frame を drop する。range control は `fa_limiter` などの dynamics node を明示的に pipeline に入れる。

## frame / buffer 前提

- sample rate は 16kHz。
- format pair は `PCM16LE/16` または `FLOAT32LE/32`。
- mic/ref は同一 channel 数であること。
- mic/ref は同一 encoding / bit depth / sample count であること。
- reference は `ref_timeout_ms` 内であること。

## known limitations

- adaptive filter ではない。
- echo path delay 推定をしない。
- reference と mic の frame alignment がずれると効果は限定的。
