# fa_aec_linear 仕様書

## 目的

`fa_aec_linear` は mic stream から playback reference stream を線形に差し引く correction node である。実用 AEC の完成実装ではなく、AEC pipeline の topic contract、reference timing、diagnostics を固定するための baseline である。

## 仕様 ID

| ID | 内容 |
| --- | --- |
| `FA-AEC-LINEAR-SPEC-001` | 起動時に topic、sample rate、channel、encoding / bit depth、reference timeout、failure policy、cancel gain を検証する |
| `FA-AEC-LINEAR-SPEC-002` | mic frame の `stream_id` は `mic_topic`、reference frame の `stream_id` は `ref_topic` と一致しなければならない |
| `FA-AEC-LINEAR-SPEC-003` | mic/reference は同じ sample rate / channels / encoding / bit depth / sample count のときだけ処理する |
| `FA-AEC-LINEAR-SPEC-004` | reference 欠落、stale、format mismatch、decode / encode failure は mic pass-through せず drop する |
| `FA-AEC-LINEAR-SPEC-005` | output range overflow は hidden clamp せず drop し、range control は dynamics node に分離する |
| `FA-AEC-LINEAR-SPEC-006` | `enabled=false` は pass-through ではなく drop として扱う。node を使わない場合は system config 側で起動しない |
| `FA-AEC-LINEAR-SPEC-007` | publisher 欠落など起動時 invariant の破損は runtime drop loop にせず fail closed する |

## 入力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `mic_topic` | `audio/resample16k/mic` | echo を含む mic frame |
| `ref_topic` | `audio/resample16k/ref` | speaker output reference frame |

## 出力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `output_topic` | `audio/aec_linear/frame` | 線形 cancel 後の frame |

## parameters

| parameter | 意味 |
| --- | --- |
| `enabled` | false の場合は frame を publish しない。system config では node 自体を disable する |
| `expected_sample_rate` | 16kHz 固定 |
| `expected_channels` | 期待 channel 数。`1` 以上が必須。channel 検査の無効化は禁止 |
| `expected.encoding` | 入力 encoding。`PCM16LE` または `FLOAT32LE` |
| `expected.bit_depth` | 入力 bit depth。`PCM16LE/16` または `FLOAT32LE/32` の組だけ許可 |
| `ref_timeout_ms` | reference frame の有効期限 |
| `reference_failure_policy` | reference 欠落、stale、format mismatch、decode / encode failure 時の扱い。現在は `drop` のみ |
| `cancel_gain` | reference に掛ける係数 |
| `qos.*` | audio frame QoS |
| `diagnostics.publish_period_ms` | diagnostics publish 周期 |

## lifecycle

起動時に topic、expected sample rate、expected channel 数、expected encoding / bit depth、timeout、QoS、`cancel_gain` を検証する。不正 config は起動失敗する。

## QoS

mic/ref/output は同一 QoS で扱う。reference 欠落は hidden fallback ではなく、timeout 条件に基づいて frame を drop する。

## error handling

invalid frame は drop する。mic frame の `stream_id` は `mic_topic`、reference frame の `stream_id` は `ref_topic` と一致しなければならない。`enabled=false`、reference 欠落、stale reference、sample rate / channel / encoding / bit depth mismatch、sample count mismatch、decode failure、output range overflow は mic pass-through せず drop する。`PCM32LE/32` は float32 として解釈せず reject する。publisher 欠落は起動時 invariant の破損なので、drop loop にせず fail closed する。

## backend selection

なし。現在は node 内の baseline 処理である。実用 AEC engine を追加する場合は `fa_aec_nn` または dedicated backend package に切り出す。

## non-goals

- nonlinear echo suppression
- beamforming
- dereverberation
- device loopback capture
- limiter / clipping / hidden range clamp
- implicit PCM32LE-to-float32 conversion
