# fa_hum アルゴリズム詳細説明書

## 1. 概要

`fa_hum` は hum 基本周波数と整数倍の倍音に対して、二次 notch biquad を cascade する。各 channel は独立した state を持ち、interleaved sample の channel index は `sample_index % channels` で求める。

## 2. notch stage の生成

設定値:

- `f0 = hum.frequency_hz`
- `H = hum.harmonics`
- `Q = hum.q`
- `Fs = expected.sample_rate`
- `Nyquist = Fs / 2`

stage は `n = 1..H` に対して `fc = f0 * n` を計算し、`fc < Nyquist` のものだけを採用する。`fc >= Nyquist` になった時点で以降の harmonic も Nyquist 以上になるため stage 生成を終了する。

stage が 0 件の場合は設定エラーとして起動しない。

## 3. biquad 係数

各 stage の係数は Audio EQ Cookbook の notch filter 形式を使う。

```text
omega = 2 * pi * fc / Fs
alpha = sin(omega) / (2 * Q)
a0 = 1 + alpha

b0 = 1 / a0
b1 = -2 * cos(omega) / a0
b2 = 1 / a0
a1 = -2 * cos(omega) / a0
a2 = (1 - alpha) / a0
```

係数が non-finite になった場合は起動失敗とする。

## 4. sample 処理

1 sample に対して、全 stage を順に適用する。

```text
y = b0*x + b1*x1 + b2*x2 - a1*y1 - a2*y2
```

stage の出力 `y` を次 stage の入力 `x` とする。state 更新は stage ごとに行う。

```text
x2 <- x1
x1 <- x
y2 <- y1
y1 <- y
```

frame 処理中にエラーが見つかった場合、state は commit しない。処理開始時に state を copy し、frame 全体が成功した場合だけ `channel_states_` に反映する。

## 5. source_id 境界

IIR filter state は source 固有であり、別 source の履歴を混ぜると波形の意味が壊れる。そのため `source_id` が変わった frame を受け取った時点で、全 channel / 全 stage の state をゼロ初期化する。

これは fallback ではなく source 境界の明示処理である。source change は diagnostics の `source_resets` に反映する。

## 6. normalized FLOAT32LE contract

入力 sample は finite かつ `[-1.0, 1.0]` に入っている必要がある。出力 sample も同じ範囲でなければならない。

出力が範囲外になった場合は clamp しない。clamp は distortion を発生させ、後段に「正常な normalized stream」と見せてしまうため、frame を drop する。
