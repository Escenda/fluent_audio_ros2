# fa_hum テスト設計

## 1. 対象

`fa_hum` package の設定契約、frame contract、notch cascade 実装、source change state reset、drop policy を対象にする。

## 2. unit contract tests

`test/unit/test_fa_hum_audio_frame_contract.py` で以下を確認する。

| test | 確認内容 |
| --- | --- |
| default config | default YAML が `FLOAT32LE` interleaved contract と hum parameter 条件を満たす |
| responsibility boundary | resample, gain, IO, clamp など別責務を持たない |
| startup validation | 不正 config が `throw std::runtime_error` 経路を持つ |
| frame validation | `source_id`, `stream_id`, format, data size を処理前に検証する |
| source reset | `source_id` change を reject せず state reset する |
| metadata preservation | `stream_id` と `data` 以外を変更しない |
| cascade | harmonic loop, Nyquist cutoff, biquad 係数、per-channel cascade state を持つ |
| invalid sample drop | non-finite / range 外 sample と range 外 output を drop し、clamp しない |
| layout | 標準 processing package layout を満たす |
| colcon pytest | `ament_cmake_pytest` から pytest が実行される |

## 3. integration tests

`test/integration/` は ROS 2 実行環境での publish / subscribe smoke test を置く領域として予約する。

最低限の追加候補:

- valid `AudioFrame` を publish し、output topic に 1 frame が出ること
- invalid encoding / bit depth / data size が publish されないこと
- `source_id` 切替後も新 source の frame が publish されること

## 4. launch tests

`test/launch/` は `fa_hum.launch.py` の起動検証を置く領域として予約する。

最低限の追加候補:

- default config で node が起動すること
- `hum.frequency_hz >= Nyquist` で起動失敗すること
- `hum.harmonics < 1` で起動失敗すること

## 5. fixtures

`test/fixtures/` は deterministic FLOAT32LE frame fixture を置く領域として予約する。

fixtures 候補:

- 60 Hz sine + clean tone
- non-finite sample を含む frame
- `[-1.0, 1.0]` 外 sample を含む frame
