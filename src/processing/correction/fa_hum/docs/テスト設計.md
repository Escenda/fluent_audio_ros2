# fa_hum テスト設計

## 1. 対象

`fa_hum` package の設定契約、frame contract、notch cascade backend、source / epoch state reset、stale frame drop policy を対象にする。

## 2. unit contract tests

`test/unit/test_fa_hum_audio_frame_contract.py` で以下を確認する。

| test | 確認内容 |
| --- | --- |
| default config | default YAML が `FLOAT32LE` interleaved contract と hum parameter 条件を満たす |
| responsibility boundary | resample, gain, IO, clamp など別責務を持たない |
| startup validation | 不正 config が `throw std::runtime_error` 経路を持つ |
| frame validation | `source_id`, `input_stream_id` と一致する `stream_id`, format, data size を処理前に検証する |
| stream state | `source_id` / `epoch` / `header.stamp` の stateful IIR 契約を検証する |
| metadata preservation | `stream_id` を `output.stream_id` に更新し、`data` 以外の信号 metadata を変更しない |
| cascade boundary | harmonic loop, Nyquist cutoff, biquad 係数、per-channel cascade state が backend にあり node から消えている |
| invalid sample drop | non-finite / range 外 sample と range 外 output を drop し、clamp しない |
| layout | 標準 processing package layout を満たす |
| colcon pytest / gtest | `ament_cmake_pytest` と `ament_cmake_gtest` から tests が実行される |

## 3. C++ backend tests

`test/cpp/test_internal_notch_cascade_backend.cpp` は ROS を使わず backend を直接検証する。

- 不正 config を constructor で拒否する
- Nyquist 未満の harmonic stages を生成する
- normalized `FLOAT32LE` byte列を処理し、同じ byte length の output を返す
- `source_id` 変更、`epoch` 進行で state reset を返す
- stale epoch、空 source、空入力、misaligned input、non-finite / range 外 input を拒否する
- backend が ROS2 型を公開しない

## 4. C++ graph smoke tests

`test/cpp/test_fa_hum_graph.cpp` は `FaHumNode` を `rclcpp::NodeOptions` で起動し、test publisher / subscriber と同じ executor に載せて検証する。

- valid `AudioFrame` を publish し、output topic に 1 frame が出ること
- 同一 `source_id` / `epoch` で古い `header.stamp` の frame が publish されないこと
- 同一 `source_id` で古い `epoch` の frame が publish されないこと
- ROS2 解決後の input/output topic が同一なら起動失敗すること
- `AudioFrame.stream_id` が `input_stream_id` と異なる frame が publish されないこと
- `input_stream_id` / `output.stream_id` が ROS topic と同一、または相互に同一なら起動失敗すること

## 5. launch tests

`test/launch/test_fa_hum_launch_contract.py` は launch と default config の契約を検証する。

- launch file は `node_name` と `config_file` だけを受け取り、backend / expected format 詳細を launch argument に漏らさないこと
- `input_topic` 欠落で起動失敗すること
- ROS2 解決後の input/output topic が同一なら起動失敗すること
- `input_stream_id` 欠落で起動失敗すること
- `input_stream_id` / `output.stream_id` が ROS topic と同一なら起動失敗すること
- `hum.frequency_hz >= Nyquist` で起動失敗すること

## 6. fixtures

`test/fixtures/` は deterministic FLOAT32LE frame fixture を置く領域として予約する。

fixtures 候補:

- 60 Hz sine + clean tone
- non-finite sample を含む frame
- `[-1.0, 1.0]` 外 sample を含む frame
