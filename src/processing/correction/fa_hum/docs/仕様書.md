# fa_hum 仕様書

## 1. 目的

`fa_hum` は FluentAudio の correction stage として、`FLOAT32LE` interleaved `AudioFrame` に含まれる電源 hum 成分を除去する。対象は `hum.frequency_hz` と、その整数倍のうち Nyquist 周波数未満の倍音である。

## 2. 非目的

- 入力デバイス制御
- resample / bit-depth 変換 / channel 変換
- gain / normalize / limiter
- VAD / ASR / feature extraction
- 欠損値や範囲外値の補正

## 3. ROS 2 node

| 項目 | 内容 |
| --- | --- |
| package | `fa_hum` |
| executable | `fa_hum_node` |
| node name default | `fa_hum` |
| launch | `launch/fa_hum.launch.py` |
| config | `config/default.yaml` |

## 4. 入力

| 項目 | 条件 |
| --- | --- |
| message | `fa_interfaces/msg/AudioFrame` |
| topic | `input_topic` |
| encoding | `expected.encoding == "FLOAT32LE"` |
| bit depth | `expected.bit_depth == 32` |
| layout | `expected.layout == "interleaved"` |
| sample rate | `expected.sample_rate` と一致 |
| channels | `expected.channels` と一致 |
| sample range | `[-1.0, 1.0]` |

`source_id` と `stream_id` は必須である。`stream_id` は `input_topic` と一致しなければならない。

## 5. 出力

出力 message は入力 message をコピーし、以下のみ変更する。

| 項目 | 内容 |
| --- | --- |
| `stream_id` | `output_topic` |
| `data` | hum notch cascade 適用後の `FLOAT32LE` interleaved bytes |

`header`, `source_id`, `encoding`, `sample_rate`, `channels`, `bit_depth`, `layout`, `epoch` は変更しない。

## 6. 設定

| parameter | 型 | 条件 |
| --- | --- | --- |
| `input_topic` | string | 空文字不可 |
| `output_topic` | string | 空文字不可 |
| `hum.frequency_hz` | double | finite, `> 0`, 少なくとも 1 stage が Nyquist 未満 |
| `hum.harmonics` | int | `>= 1` |
| `hum.q` | double | finite, `> 0` |
| `expected.sample_rate` | int | `> 0` |
| `expected.channels` | int | `> 0` |
| `expected.encoding` | string | `FLOAT32LE` |
| `expected.bit_depth` | int | `32` |
| `expected.layout` | string | `interleaved` |
| `qos.depth` | int | `> 0` |
| `qos.reliable` | bool | `true` or `false` |
| `diagnostics.publish_period_ms` | int | `> 0` |

設定が不正な場合は node 起動時に例外を投げ、publisher / subscriber を作成しない。

## 7. 処理規則

1. frame contract を検証する。
2. `source_id` が前回処理 frame と異なる場合、全 channel / 全 notch stage の filter state をリセットする。
3. `frequency_hz * n < sample_rate / 2` を満たす harmonic stage を順に適用する。
4. 入力 sample が non-finite または `[-1.0, 1.0]` 外の場合、frame 全体を drop する。
5. 出力 sample が non-finite または `[-1.0, 1.0]` 外の場合、clamp せず frame 全体を drop する。
6. 処理成功時のみ output topic に publish する。

## 8. diagnostics

`diagnostics` topic に `DiagnosticArray` を定期 publish する。

主な値:

- `hum_frequency_hz`
- `hum_harmonics_requested`
- `hum_q`
- `notch_stage_count`
- `notch_stage_N_center_hz`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `source_resets`
- `active_source_id`
- `output_topic`
- `backend=internal_notch_cascade`

## 9. fail closed

この node は値の補正、default 推定、clamp、stream contract の吸収を行わない。不正な設定は起動失敗、不正な frame / sample は drop として扱う。
