# fa_hum 仕様書

## 1. 目的

`fa_hum` は FluentAudio の correction stage として、`FLOAT32LE` interleaved `AudioFrame` に含まれる電源 hum 成分を除去する。対象は `hum.frequency_hz` と、その整数倍のうち Nyquist 周波数未満の倍音である。

## 2. 非目的

- 入力デバイス制御
- resample / bit-depth 変換 / channel 変換
- gain / normalize / limiter
- VAD / ASR / feature extraction
- 欠損値や範囲外値の補正

## 3. ROS 2 node

| 項目 | 内容 |
| --- | --- |
| package | `fa_hum` |
| executable | `fa_hum_node` |
| node name default | `fa_hum` |
| launch | `launch/fa_hum.launch.py` |
| config | `config/default.yaml` |

## 4. 入力

| 項目 | 条件 |
| --- | --- |
| message | `fa_interfaces/msg/AudioFrame` |
| topic | `input_topic` |
| stream identity | `input_stream_id` |
| encoding | `expected.encoding == "FLOAT32LE"` |
| bit depth | `expected.bit_depth == 32` |
| layout | `expected.layout == "interleaved"` |
| sample rate | `expected.sample_rate` と一致 |
| channels | `expected.channels` と一致 |
| sample range | `[-1.0, 1.0]` |

`source_id` と `stream_id` は必須である。`stream_id` は `input_stream_id` と一致しなければならない。`input_topic` と `output_topic` は raw 値および ROS2 の展開・remap 解決後に異なる topic でなければならない。同一 topic は feedback loop になるため起動失敗とする。

ROS topic identity と `AudioFrame.stream_id` は別の識別子である。`input_stream_id` / `output.stream_id` は raw/resolved `input_topic` / `output_topic` のいずれとも一致してはならない。`input_stream_id` と `output.stream_id` も同一にできない。

## 5. 出力

出力 message は入力 message をコピーし、以下のみ変更する。

| 項目 | 内容 |
| --- | --- |
| `stream_id` | `output.stream_id` |
| `data` | hum notch cascade 適用後の `FLOAT32LE` interleaved bytes |

`header`, `source_id`, `encoding`, `sample_rate`, `channels`, `bit_depth`, `layout`, `epoch` は変更しない。

## 6. 設定

| parameter | 型 | 条件 |
| --- | --- | --- |
| `input_topic` | string | 空文字不可 |
| `output_topic` | string | 空文字不可 |
| `input_stream_id` | string | 空文字不可、raw/resolved ROS topic と一致不可 |
| `output.stream_id` | string | 空文字不可、raw/resolved ROS topic と一致不可、`input_stream_id` と一致不可 |
| `hum.frequency_hz` | double | finite, `> 0`, 少なくとも 1 stage が Nyquist 未満 |
| `hum.harmonics` | int | `>= 1` |
| `hum.q` | double | finite, `> 0` |
| `expected.sample_rate` | int | `> 0` |
| `expected.channels` | int | `> 0` |
| `expected.encoding` | string | `FLOAT32LE` |
| `expected.bit_depth` | int | `32` |
| `expected.layout` | string | `interleaved` |
| `qos.depth` | int | `> 0` |
| `qos.reliable` | bool | `true` or `false` |
| `diagnostics.publish_period_ms` | int | `> 0` |
| `diagnostics.qos.depth` | diagnostics QoS depth |
| `diagnostics.qos.reliable` | diagnostics QoS reliability |

設定が不正な場合は node 起動時に例外を投げ、publisher / subscriber を作成しない。

## 7. 処理規則

1. frame contract を検証する。
2. 同一 `source_id` / `epoch` で `header.stamp` が直近 accepted frame より古い場合、stale frame として drop する。
3. `source_id` が前回処理 frame と異なる場合、全 channel / 全 notch stage の filter state をリセットする。
4. 同一 `source_id` で `epoch` が進んだ場合、全 channel / 全 notch stage の filter state をリセットする。
5. 同一 `source_id` で `epoch` が前回より古い場合、stale epoch として drop する。
6. `frequency_hz * n < sample_rate / 2` を満たす harmonic stage を順に適用する。
7. 入力 sample が non-finite または `[-1.0, 1.0]` 外の場合、frame 全体を drop する。
8. 出力 sample が non-finite または `[-1.0, 1.0]` 外の場合、clamp せず frame 全体を drop する。
9. 処理成功時のみ output topic に publish する。

notch cascade 本体は ROS 非依存の `internal_notch_cascade` backend が持つ。ROS node は `AudioFrame` 契約検証、stale stamp 判定、topic 入出力、diagnostics、counter 更新だけを担当する。

## 8. diagnostics

`diagnostics` topic に `DiagnosticArray` を定期 publish する。

主な値:

- `hum_frequency_hz`
- `hum_harmonics_requested`
- `hum_q`
- `notch_stage_count`
- `notch_stage_N_center_hz`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `source_resets`
- `epoch_resets`
- `active_source_id`
- `active_epoch`
- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `resolved_input_topic`
- `resolved_output_topic`
- `backend.name=internal_notch_cascade`

## 9. fail closed

この node は値の補正、default 推定、clamp、stream contract の吸収を行わない。不正な設定は起動失敗、不正な frame / sample は drop として扱う。

## 10. 仕様 ID

| SPEC ID | 内容 |
| --- | --- |
| `FA-HUM-SPEC-001` | `fa_hum` は電源 hum と Nyquist 未満の整数倍 harmonic notch cascade のみを行い、device I/O、resample、format / channel conversion、gain、normalize、limiter、VAD、ASR、feature extraction、欠損補正を持たない。 |
| `FA-HUM-SPEC-002` | input frame は source/stream identity、FLOAT32LE/32/interleaved、sample rate、channels、finite normalized samples を満たす必要があり、topic identity と stream identity は分離する。 |
| `FA-HUM-SPEC-003` | output frame は `stream_id` と `data` だけを更新し、header、source_id、encoding、sample_rate、channels、bit_depth、layout、epoch を保持する。 |
| `FA-HUM-SPEC-004` | 起動時 config は topic、stream identity、hum frequency / harmonics / Q、expected format、QoS、diagnostics を fail closed で検証する。 |
| `FA-HUM-SPEC-005` | 同一 source/epoch で古い `header.stamp` の frame は stale frame として drop する。 |
| `FA-HUM-SPEC-006` | source_id 変更または epoch 進行では backend state を reset し、epoch が戻る frame は drop する。 |
| `FA-HUM-SPEC-007` | `frequency_hz * n < sample_rate / 2` を満たす harmonic stage だけを順に適用する。 |
| `FA-HUM-SPEC-008` | 入力または出力 sample が non-finite / normalized range 外の場合、clamp せず frame 全体を drop する。 |
| `FA-HUM-SPEC-009` | notch cascade 本体は ROS2 topic/message を知らない `internal_notch_cascade` backend が持つ。 |
| `FA-HUM-SPEC-010` | diagnostics は hum config、stage count、stage center、counters、active source/epoch、topic、stream identity、`backend.name=internal_notch_cascade` を含む。 |
| `FA-HUM-SPEC-011` | package は標準 processing layout、backend document、pytest、backend gtest、ROS graph gtest、launch contract test を持つ。 |
| `FA-HUM-SPEC-012` | launch layer は `node_name` と `config_file` だけを公開し、backend / expected format 詳細を launch argument に漏らさない。 |
