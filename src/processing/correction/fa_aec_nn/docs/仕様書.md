# fa_aec_nn 仕様書

## 目的

`fa_aec_nn` は `fa_aec_linear` 後段で残差 echo や nonlinear distortion を抑える correction node である。現在の実装は明示指定された `passthrough` backend だけを持つ contract skeleton であり、default config は `backend.name` 未指定として起動失敗にする。

## 仕様 ID

| ID | 内容 |
| --- | --- |
| `FA-AEC-NN-SPEC-001` | `backend.name` は必須で、未指定または unknown backend は起動失敗にする |
| `FA-AEC-NN-SPEC-002` | 入力 frame は `input_stream_id` と同じ `stream_id`、16kHz、明示 channel 数、`PCM16LE/16` または `FLOAT32LE/32`、interleaved layout、非空 PCM frame boundary を満たす |
| `FA-AEC-NN-SPEC-003` | backend は ROS2 topic/message を知らず、validated `AudioChunk` を受け取り `ProcessedAudioChunk` を返す |
| `FA-AEC-NN-SPEC-004` | backend 出力は入力と同じ sample rate / channels / encoding / bit depth / layout / frame count の PCM payload だけを許可する |
| `FA-AEC-NN-SPEC-005` | `enabled=false` は pass-through ではなく drop として扱う。node を使わない場合は system config 側で起動しない |
| `FA-AEC-NN-SPEC-006` | publisher/backend 欠落など起動時 invariant の破損は runtime drop loop にせず fail closed する |

## 入力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `input_topic` | `fa_aec_nn/input` | 線形 AEC 後の frame を受け取る ROS topic |
| `input_stream_id` | `audio/aec_linear/frame` | 線形 AEC 後の `AudioFrame.stream_id` |

## 出力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `output_topic` | `fa_aec_nn/output` | NN AEC 後の frame を publish する ROS topic |
| `output.stream_id` | `audio/aec/frame` | NN AEC 後の `AudioFrame.stream_id` |

## parameters

| parameter | 意味 |
| --- | --- |
| `enabled` | node 有効化。`false` の場合は publish せず drop する。通常は system config 側で node 自体を disable する |
| `backend.name` | 必須。現在は明示 `passthrough` のみ |
| `expected_sample_rate` | 16kHz |
| `expected_channels` | 期待 channel 数。`1` 以上が必須。channel 検査の無効化は禁止 |
| `expected.encoding` | 入力 encoding。現在は `PCM16LE` または `FLOAT32LE` |
| `expected.bit_depth` | 入力 bit depth。`PCM16LE/16` または `FLOAT32LE/32` の組だけ許可 |
| `qos.*` | audio frame QoS |
| `diagnostics.publish_period_ms` | diagnostics publish 周期 |
| `diagnostics.qos.depth` | diagnostics QoS depth |
| `diagnostics.qos.reliable` | diagnostics QoS reliability |

## lifecycle

起動時に backend 名、topic、expected sample rate、expected channel 数、expected encoding / bit depth、QoS を検証する。`backend.name` 空、または `passthrough` 以外は起動失敗にする。
`input_topic` / `output_topic` は ROS 搬送路、`input_stream_id` / `output.stream_id` は data plane の stream identity として扱う。stream identity が raw/resolved topic 名と衝突する設定、または input/output stream identity が同一の設定は起動失敗にする。

## QoS

input/output は同一 QoS で扱う。

## error handling

`enabled=false`、unknown backend、invalid frame、format mismatch、backend output contract violation は成功扱いにしない。入力 frame の `stream_id` は `input_stream_id` と一致しなければならない。`PCM32LE/32` など、明示された `expected.encoding` / `expected.bit_depth` と一致しない形式は drop する。publisher/backend 欠落は起動時 invariant の破損なので、drop loop にせず fail closed する。

## backend selection

`backend.name` parameter で選択する。backend は `include/fa_aec_nn/backends/` と `src/backends/` に置き、ROS2 topic、ROS message、`rclcpp` を知らず、validated `AudioChunk` を受け取る境界にする。backend は `ProcessedAudioChunk` を返すが、現在の node contract では入力と同じ sample rate / channels / encoding / bit depth / layout / frame count だけを publish 対象にする。backend が empty data、PCM frame boundary 不一致、format mismatch、frame count mismatch を返した場合は drop する。

## non-goals

- linear AEC
- reference capture
- device I/O
- implicit ONNX Runtime fallback
