# fa_aec_nn 仕様書

## 目的

`fa_aec_nn` は `fa_aec_linear` 後段で残差 echo や nonlinear distortion を抑える correction node である。現在の実装は `passthrough` backend だけを持つ contract skeleton であり、未実装 backend 名は起動失敗にする。

## 入力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `input_topic` | `audio/aec_linear/frame` | 線形 AEC 後の frame |

## 出力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `output_topic` | `audio/aec/frame` | NN AEC 後の frame |

## parameters

| parameter | 意味 |
| --- | --- |
| `enabled` | node 有効化 |
| `backend` | 現在は `passthrough` のみ |
| `expected_sample_rate` | 16kHz |
| `expected_channels` | 期待 channel 数 |
| `qos.*` | audio frame QoS |
| `diagnostics.publish_period_ms` | diagnostics publish 周期 |

## lifecycle

起動時に backend 名、topic、expected format を検証する。`passthrough` 以外は起動失敗にする。

## QoS

input/output は同一 QoS で扱う。

## error handling

unknown backend、invalid frame、format mismatch は成功扱いにしない。

## backend selection

`backend` parameter で選択する。backend は ROS2 topic を知らず、validated PCM chunk を受け取る境界にする。

## non-goals

- linear AEC
- reference capture
- device I/O
- implicit ONNX Runtime fallback
