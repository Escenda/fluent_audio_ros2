# fa_aec_nn アルゴリズム詳細説明書

## 処理手順

```text
AudioFrame
  ↓
validate expected format and stream_id == input_stream_id
  ↓
dispatch AudioChunk to backend
  ↓
validate ProcessedAudioChunk output contract
  ↓
publish processed frame
```

## passthrough backend

現在の `passthrough` backend は validated `AudioChunk` を受け取り、同じ sample rate / channels / encoding / bit depth / layout と byte payload を持つ `ProcessedAudioChunk` を返す。ROS2 topic、ROS message、`rclcpp` は知らない。これは未実装 NN を成功扱いにするものではなく、pipeline wiring と downstream contract の検証用 backend である。

## frame / buffer 前提

- sample rate は 16kHz。
- channels は `expected_channels` と一致すること。
- `expected_channels` は 1 以上であり、channel 検査の wildcard はない。
- encoding / bit depth は `PCM16LE/16` または `FLOAT32LE/32` の明示 pair と一致すること。
- `stream_id` は `input_stream_id` と一致すること。
- data は PCM frame boundary に一致すること。
- backend 出力は入力と同じ format contract と frame count を維持すること。
- backend が空 data、PCM frame boundary 不一致、format mismatch、frame count mismatch を返した場合は publish しない。
- publisher/backend 欠落は起動時 invariant の破損なので runtime drop loop にしない。

## known limitations

- 実際の neural AEC model は未接続。
- model backend を追加する場合、runtime dependency と model path validation を `docs/backends/` と launch config に明記する。
