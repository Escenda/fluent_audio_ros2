# fa_aec_nn テスト設計

## unit test

| Test ID | 対応仕様 | 内容 |
| --- | --- | --- |
| `FA-AEC-NN-TC-001` -> `FA-AEC-NN-SPEC-001` | `FA-AEC-NN-SPEC-001` | missing backend / unknown backend で起動失敗 |
| `FA-AEC-NN-TC-002` -> `FA-AEC-NN-SPEC-002` | `FA-AEC-NN-SPEC-002` | `stream_id` が `input_stream_id` と一致しない場合を含む invalid sample rate / channel / encoding / bit depth / empty data を drop |
| `FA-AEC-NN-TC-003` -> `FA-AEC-NN-SPEC-002` | `FA-AEC-NN-SPEC-002` | `expected_channels <= 0` は起動失敗し、channel wildcard を許可しない |
| `FA-AEC-NN-TC-004` -> `FA-AEC-NN-SPEC-003` | `FA-AEC-NN-SPEC-003` | backend files が ROS2 / message header に依存しない |
| `FA-AEC-NN-TC-005` -> `FA-AEC-NN-SPEC-004` | `FA-AEC-NN-SPEC-004` | backend output が empty / misaligned / format mismatch / frame count mismatch の場合は invalid contract として扱う |
| `FA-AEC-NN-TC-006` -> `FA-AEC-NN-SPEC-005` | `FA-AEC-NN-SPEC-005` | `enabled=false` の場合は publish せず drop |
| `FA-AEC-NN-TC-007` -> `FA-AEC-NN-SPEC-006` | `FA-AEC-NN-SPEC-006` | publisher/backend 欠落を runtime drop loop として扱わない |

## integration test

- `fa_aec_linear -> fa_aec_nn` の topic wiring。
- output metadata が input と一致すること。

## launch test

| Test ID | 対応仕様 | 内容 |
| --- | --- | --- |
| `FA-AEC-NN-LT-001` | `FA-AEC-NN-SPEC-001` | default config の `backend.name` は空で、明示 backend なしでは起動失敗する |
| `FA-AEC-NN-LT-002` | `FA-AEC-NN-SPEC-001` | `backend.name=unknown` で起動失敗する |
| `FA-AEC-NN-LT-003` | `FA-AEC-NN-SPEC-001` | debug config で `backend.name=passthrough` を明示した場合だけ起動できる |
| `FA-AEC-NN-LT-003A` | `FA-AEC-NN-SPEC-002` | `input_stream_id` / `output.stream_id` と ROS topic identity の衝突を起動失敗にする |
| `FA-AEC-NN-LT-004` | `FA-AEC-NN-SPEC-002` | `expected.encoding` / `expected.bit_depth` は `PCM16LE/16` または `FLOAT32LE/32` の明示 pair だけを受け付ける |
| `FA-AEC-NN-LT-005` | `FA-AEC-NN-SPEC-003` | backend implementation は `include/fa_aec_nn/backends/` と `src/backends/` に置く |
| `FA-AEC-NN-LT-006` | `FA-AEC-NN-SPEC-003` | backend implementation は `rclcpp` / `fa_interfaces` / ROS message header に依存しない |
| `FA-AEC-NN-LT-007` | `FA-AEC-NN-SPEC-003` | `fa_aec_nn_backends` library を作り、node は backend source を直接含めずその library に link する |

## failure test

- unsupported sample rate。
- unsupported channel。
- unsupported encoding / bit depth。
- stream id と `input_stream_id` の mismatch。
- empty data。
- `enabled=false` を pass-through として扱わないこと。

## latency test

passthrough baseline latency を測り、将来の NN backend の比較基準にする。

## real device smoke test

AEC pipeline に組み込み、diagnostics in/out/drop counter を確認する。
