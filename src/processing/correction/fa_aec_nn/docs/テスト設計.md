# fa_aec_nn テスト設計

## unit test

- `backend.name=passthrough` の frame passthrough。
- `enabled=false` の場合は publish せず drop。
- unknown backend で起動失敗。
- invalid format を drop。

## integration test

- `fa_aec_linear -> fa_aec_nn` の topic wiring。
- output metadata が input と一致すること。

## launch test

- default config の `backend.name` は空で、明示 backend なしでは起動失敗する。
- debug config で `backend.name=passthrough` を明示した場合だけ起動できる。
- `backend.name=unknown` で起動失敗する。
- backend implementation は `include/fa_aec_nn/backends/` と `src/backends/` に置く。
- backend implementation は `rclcpp` / `fa_interfaces` / ROS message header に依存しない。
- `fa_aec_nn_backends` library を作り、node は backend source を直接含めずその library に link する。

## failure test

- unsupported sample rate。
- unsupported channel。
- empty data。
- `enabled=false` を pass-through として扱わないこと。

## latency test

passthrough baseline latency を測り、将来の NN backend の比較基準にする。

## real device smoke test

AEC pipeline に組み込み、diagnostics in/out/drop counter を確認する。
