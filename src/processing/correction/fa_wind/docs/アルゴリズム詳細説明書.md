# fa_wind アルゴリズム詳細説明書

1. startup config で wind detector、suppression strength、expected format を検証する。
2. input `AudioFrame` の identity と format metadata を検証する。
3. backend で wind-like low-frequency burst / turbulence component を検出する。
4. 検出範囲に対して明示 suppression を適用する。
5. finite/normalized contract を検証し、契約を満たす場合だけ publish する。

wind detector が設定されていない場合に generic high-pass filter へ fallback しない。
high-pass だけが必要な pipeline では `fa_high_pass` を明示的に使う。

## Backend Boundary

backend は wind noise detection と suppression のみを担当し、ROS2 topic/message を知らない。
