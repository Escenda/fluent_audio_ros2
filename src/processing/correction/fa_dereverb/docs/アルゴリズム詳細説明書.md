# fa_dereverb アルゴリズム詳細説明書

1. startup config で backend、room profile/model、expected format を検証する。
2. input `AudioFrame` の identity と format metadata を検証する。
3. backend で reverberation suppression を適用する。
4. finite/normalized contract と frame count contract を検証する。
5. 契約を満たす場合だけ publish する。

room profile や model が欠損している場合に pass-through しない。
AEC や beamforming と混ぜる必要がある場合は、それぞれ専用 node を pipeline に明示する。

## Backend Boundary

backend は dereverberation algorithm のみを担当し、ROS2 topic/message を知らない。
Python version / venv が異なる neural backend は external worker / process / container 境界にする。
