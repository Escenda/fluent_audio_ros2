# fa_bit_depth 仕様書

## 1. 目的

`fa_bit_depth` は `AudioFrame` の PCM integer bit depth を専用に変換する processing node である。
入力 source/sink、codec、sample representation normalization、sample rate、channel count、音量、filter、noise reduction は責務外とする。

PCM integer bit-depth conversion は `internal_integer_bit_depth` backend が担当する。backend は ROS2 topic/message を知らず、node は parameter、QoS、runtime `AudioFrame` metadata 検証、publish/drop、diagnostics を担当する。

## 2. 入力

| 項目 | 内容 |
| --- | --- |
| topic | `input_topic` |
| message | `fa_interfaces/msg/AudioFrame` |
| stream_id | `input_stream_id` |
| encoding | `input.encoding` と一致 |
| bit depth | `input.bit_depth` と一致 |
| sample rate | `expected.sample_rate` と一致 |
| channels | `expected.channels` と一致 |
| layout | `expected.layout` と一致。初期実装では `interleaved` のみ |

`source_id` と `stream_id` は必須である。`stream_id` は `input_stream_id` と一致することを入力契約とする。
ROS topic と `AudioFrame.stream_id` は別 identity として扱う。`input_stream_id` は `input_topic` / `output_topic` の raw/resolved topic 名と一致してはならない。

## 3. 出力

| 項目 | 内容 |
| --- | --- |
| topic | `output_topic` |
| message | `fa_interfaces/msg/AudioFrame` |
| encoding | `output.encoding` |
| bit depth | `output.bit_depth` |
| stream_id | `output.stream_id` |

`source_id`、`header`、`sample_rate`、`channels`、`layout`、`epoch` は入力 frame から保持する。
`output.stream_id` は `input_topic` / `output_topic` の raw/resolved topic 名と一致してはならず、`input_stream_id` とも一致してはならない。

## 4. 対応変換

初期実装で対応する変換は次の通り。

| input | output |
| --- | --- |
| `PCM16LE` / 16 / `interleaved` | `PCM32LE` / 32 / `interleaved` |

上記以外の変換 config は起動時に失敗する。runtime frame の metadata が config と一致しない場合は warning を出して drop する。`PCM32LE/32` から `PCM16LE/16` への変換は下位 bit を破棄するため、この node では扱わない。

## 5. 非目標

- device を開くこと
- file / network / codec を扱うこと
- normalized float sample format への変換
- resample
- gain / loudness normalize / limiter
- filter / noise reduction
- channel count 変更
- non-interleaved layout 対応
- frame metadata による hidden auto-detection
- lossy down-conversion / truncation

## 6. エラー処理

startup config が不正な場合は例外で起動失敗する。`input_stream_id` / `output.stream_id` が空、topic と衝突、または相互に同一の場合も fail closed する。
runtime frame が契約に合わない場合は warning を出して drop し、変換結果を publish しない。
backend は `ProcessResult` と `FrameContractStatus` で失敗理由を返し、失敗時に出力 buffer を更新しない。

## 7. 仕様 ID

| SPEC ID | 内容 |
| --- | --- |
| `FA-BIT-DEPTH-SPEC-001` | `fa_bit_depth` は PCM integer bit depth conversion のみを行い、device I/O、codec、sample representation normalization、resample、gain、loudness normalize、limiter、filter、noise reduction、channel count 変更を持たない。 |
| `FA-BIT-DEPTH-SPEC-002` | default config は `PCM16LE/16/interleaved -> PCM32LE/32/interleaved` と expected sample rate / channels を明示する。 |
| `FA-BIT-DEPTH-SPEC-003` | ROS topic identity と `AudioFrame.stream_id` は分離し、空 stream ID、topic/stream 衝突、input/output stream 衝突は起動失敗にする。 |
| `FA-BIT-DEPTH-SPEC-004` | 対応変換は lossless `PCM16LE/16 -> PCM32LE/32` のみで、unknown conversion や lossy `PCM32LE/32 -> PCM16LE/16` は起動失敗にする。 |
| `FA-BIT-DEPTH-SPEC-005` | runtime frame は source/stream identity、layout、encoding、bit depth、sample rate、channels、byte count を処理前に検証する。 |
| `FA-BIT-DEPTH-SPEC-006` | 出力 frame は `source_id`、header、sample_rate、channels、layout、epoch を保持し、`stream_id`、encoding、bit_depth、data だけを出力契約に従って更新する。 |
| `FA-BIT-DEPTH-SPEC-007` | backend は `PCM16LE` word を `PCM32LE` high-word expansion として変換する。 |
| `FA-BIT-DEPTH-SPEC-008` | backend は ROS2 topic/message を知らず、typed status で失敗理由を返し、失敗時に output buffer を更新しない。 |
| `FA-BIT-DEPTH-SPEC-009` | package は標準 processing layout、backend document、pytest、backend gtest、node graph gtest を持つ。 |
