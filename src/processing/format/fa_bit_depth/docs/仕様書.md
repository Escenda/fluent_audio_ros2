# fa_bit_depth 仕様書

## 1. 目的

`fa_bit_depth` は `AudioFrame` の PCM integer bit depth を専用に変換する processing node である。
入力 source/sink、codec、sample representation normalization、sample rate、channel count、音量、filter、noise reduction は責務外とする。

PCM integer bit-depth conversion は `internal_integer_bit_depth` backend が担当する。backend は ROS2 topic/message を知らず、node は parameter、QoS、runtime `AudioFrame` metadata 検証、publish/drop、diagnostics を担当する。

## 2. 入力

| 項目 | 内容 |
| --- | --- |
| topic | `input_topic` |
| message | `fa_interfaces/msg/AudioFrame` |
| encoding | `input.encoding` と一致 |
| bit depth | `input.bit_depth` と一致 |
| sample rate | `expected.sample_rate` と一致 |
| channels | `expected.channels` と一致 |
| layout | `expected.layout` と一致。初期実装では `interleaved` のみ |

`source_id` と `stream_id` は必須である。`stream_id` は `input_topic` と一致することを入力契約とする。

## 3. 出力

| 項目 | 内容 |
| --- | --- |
| topic | `output_topic` |
| message | `fa_interfaces/msg/AudioFrame` |
| encoding | `output.encoding` |
| bit depth | `output.bit_depth` |
| stream_id | `output_topic` |

`source_id`、`header`、`sample_rate`、`channels`、`layout`、`epoch` は入力 frame から保持する。

## 4. 対応変換

初期実装で対応する変換は次の通り。

| input | output |
| --- | --- |
| `PCM16LE` / 16 / `interleaved` | `PCM32LE` / 32 / `interleaved` |

上記以外の変換 config は起動時に失敗する。runtime frame の metadata が config と一致しない場合は warning を出して drop する。`PCM32LE/32` から `PCM16LE/16` への変換は下位 bit を破棄するため、この node では扱わない。

## 5. 非目標

- device を開くこと
- file / network / codec を扱うこと
- normalized float sample format への変換
- resample
- gain / loudness normalize / limiter
- filter / noise reduction
- channel count 変更
- non-interleaved layout 対応
- frame metadata による hidden auto-detection
- lossy down-conversion / truncation

## 6. エラー処理

startup config が不正な場合は例外で起動失敗する。
runtime frame が契約に合わない場合は warning を出して drop し、変換結果を publish しない。
backend は `ProcessResult` と `FrameContractStatus` で失敗理由を返し、失敗時に出力 buffer を更新しない。
