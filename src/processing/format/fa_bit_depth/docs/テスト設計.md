# fa_bit_depth テスト設計

## 1. テスト対象

`fa_bit_depth` package の config、C++ node contract、package layout、integer bit-depth conversion algorithm を対象とする。

## 2. unit contract tests

`test/unit/test_fa_bit_depth_audio_frame_contract.py` で次を確認する。

- default config が `PCM16LE/16 -> PCM32LE/32`、`interleaved`、明示的な sample rate/channel 契約を持つこと
- source が device I/O、codec、resample、gain、normalized float conversion を扱わないこと
- startup config validation が `PCM16LE/16 -> PCM32LE/32` 以外を拒否すること
- runtime frame validation が metadata、layout、encoding、bit depth、rate、channel、byte count を確認すること
- output frame が `stream_id`、`encoding`、`bit_depth` 以外の metadata を保持すること
- PCM16LE から PCM32LE への変換が little-endian integer bit alignment として実装されていること
- lossy PCM32LE/32 -> PCM16LE/16 は fail closed し、下位 bit truncation path が存在しないこと
- colcon test から pytest contract tests が実行されること

## 3. integration tests

`test/cpp/test_fa_bit_depth_node_contract.cpp` で C++ node core を起動し、ROS2 pub/sub 経由で次を確認する。

- `PCM16LE/16` valid frame が `PCM32LE/32` high-word expansion として publish されること
- misaligned payload は publish されないこと
- runtime encoding / bit depth mismatch は publish されないこと
- lossy `PCM32LE/32 -> PCM16LE/16` config は起動時に reject されること

## 4. launch tests

今後の launch test では、`launch/fa_bit_depth.launch.py` が default config で node を起動できること、不正 config では起動失敗することを確認する。
