# fa_bit_depth アルゴリズム詳細説明書

## 1. PCM16LE から PCM32LE

入力 byte列を little-endian の signed 16 bit PCM として 2 byte ずつ読む。
変換は normalized float を経由せず、16 bit PCM word を 32 bit PCM word の上位 16 bit に配置する。

```text
raw16 = uint16 little-endian sample
raw32 = raw16 << 16
```

例:

| PCM16LE sample | PCM32LE sample |
| --- | --- |
| `0x0000` | `0x00000000` |
| `0x7fff` | `0x7fff0000` |
| `0x8000` | `0x80000000` |
| `0xffff` | `0xffff0000` |

この変換は 16 bit sample の順位と符号を保持し、追加される下位 16 bit は 0 で埋める。

## 2. PCM32LE から PCM16LE

入力 byte列を little-endian の signed 32 bit PCM として 4 byte ずつ読む。
変換は 32 bit PCM word の上位 16 bit を出力 16 bit PCM word として採用する。

```text
raw32 = uint32 little-endian sample
raw16 = (raw32 >> 16) & 0xffff
```

例:

| PCM32LE sample | PCM16LE sample |
| --- | --- |
| `0x00000000` | `0x0000` |
| `0x7fffffff` | `0x7fff` |
| `0x80000000` | `0x8000` |
| `0xffffffff` | `0xffff` |

下位 16 bit は破棄する。dither、rounding、clipping、normalized range conversion は行わない。

## 3. Byte count validation

`AudioFrame.data` は次を満たす必要がある。

```text
bytes_per_sample = input.bit_depth / 8
bytes_per_frame = expected.channels * bytes_per_sample
data.size > 0
data.size % bytes_per_frame == 0
```

不一致の場合、frame 境界が壊れているため publish せず drop する。

## 4. 変換しない項目

sample rate、channels、layout は変更しない。resample や channel 変換は別 package の責務である。
振幅加工も行わない。PCM integer bit depth の再配置だけを行い、gain、limiter、loudness normalize は行わない。
