# fa_sample_format テスト設計

## 1. Unit contract tests

`test/unit/test_fa_sample_format_audio_frame_contract.py` で package-local contract を検査する。

- default config が `PCM16LE/16 -> FLOAT32LE/32` を明示する
- startup で unsupported conversion を拒否する実装になっている
- runtime frame validation が source/stream、layout、encoding、bit depth、sample rate、channels、byte count を確認する
- 出力が `source_id` を保持し、`stream_id` / `encoding` / `bit_depth` を更新する
- device API、resample、gain、loudness normalize、limiter、filter、channel count 変更を含まない
- 標準 processing package layout を満たす
- CMake が pytest を `PYTEST_DISABLE_PLUGIN_AUTOLOAD=1` で登録する

## 2. Integration tests

現時点では placeholder のみ置く。今後、ROS2 graph 上で publisher/subscriber を起動し、PCM fixture が FLOAT32LE frame として publish されることを検証する。

## 3. Launch tests

現時点では placeholder のみ置く。今後、default launch が config file を読み、unsupported conversion config で fail closed になることを検証する。
