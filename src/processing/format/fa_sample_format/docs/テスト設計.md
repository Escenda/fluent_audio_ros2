# fa_sample_format テスト設計

## 1. Unit contract tests

`test/unit/test_fa_sample_format_audio_frame_contract.py` で package-local contract を検査する。

- default config が `PCM16LE/16 -> FLOAT32LE/32` を明示する
- startup で unsupported conversion を拒否する実装になっている
- runtime frame validation が source、`input_stream_id`、layout、encoding、bit depth、sample rate、channels、byte count を確認する
- 出力が `source_id` を保持し、`stream_id` を `output.stream_id` へ、`encoding` / `bit_depth` を出力形式へ更新する
- device API、resample、gain、loudness normalize、limiter、filter、channel count 変更を含まない
- 標準 processing package layout を満たす
- CMake が pytest を `PYTEST_DISABLE_PLUGIN_AUTOLOAD=1` で登録する

`test/cpp/test_internal_float32le_backend.cpp` で ROS-free backend contract を実 sample byte列で検査する。

- `PCM16LE/16 -> FLOAT32LE/32`、`PCM32LE/32 -> FLOAT32LE/32`、`FLOAT32LE/32 -> PCM16LE/16` だけを support する
- `-32768`, `0`, `32767` を `float32` little-endian bytes へ写像する
- `-2147483648`, `0`, `2147483647` を `float32` little-endian bytes へ写像する
- `-1.0`, `0.0`, `1.0` を `PCM16LE` little-endian bytes へ写像する
- `FLOAT32LE -> PCM16LE` で non-finite / range overflow を clamp せず拒否する
- empty / sample 幅で割り切れない payload は `FrameContractStatus` で拒否し、出力 buffer を更新しない
- CMake が gtest を登録する

`test/cpp/test_sample_format_graph.cpp` で ROS graph smoke test を実行する。

- test publisher が `PCM16LE/16` `AudioFrame` を `input_topic` へ publish し、`stream_id` を `input_stream_id` にする
- `FaSampleFormatNode` が実 subscription callback で変換する
- test subscriber が `output_topic` から `FLOAT32LE/32` `AudioFrame` を受信する
- `source_id` / `epoch` / sample rate / channels / layout は保持され、`stream_id` / encoding / bit depth だけが明示更新される

## 2. Integration tests

ROS2 graph smoke test は C++ gtest として実装する。外部 device や ROS launch を使わず、node を in-process executor に載せて data-plane contract を検証する。

## 3. Launch tests

`test/launch/test_fa_sample_format_launch_contract.py` で launch file と default config の境界を検査する。

- launch file が `node_name` と `config_file` だけを公開する。
- node parameter は `config/default.yaml` から読み、launch file に変換条件を直書きしない。
- launch file に `backend.name` を置かない。
- default config が `PCM16LE/16 -> FLOAT32LE/32`、`input_stream_id` / `output.stream_id`、`expected.sample_rate/channels/layout` を明示する。
- `ros2 launch` 経由で unsupported conversion config が起動失敗することを確認する。
- `ros2 launch` 経由で `input_topic` 欠落が起動失敗することを確認する。
- `ros2 launch` 経由で `output_topic` 欠落が起動失敗することを確認する。
- `ros2 launch` 経由で `expected.sample_rate <= 0` が起動失敗することを確認する。
- `ros2 launch` 経由で `expected.channels <= 0` が起動失敗することを確認する。
- `ros2 launch` 経由で `expected.layout != interleaved` が起動失敗することを確認する。

今後の runtime launch test では、実 graph 上の valid config startup を検証する。

## 4. 仕様とテストケース対応

### 4.1 仕様 ID

- `FA-SAMPLE-FORMAT-SPEC-001`: default config は `PCM16LE/16 -> FLOAT32LE/32`、`input_stream_id` / `output.stream_id`、期待 sample rate / channels / layout を明示する。
- `FA-SAMPLE-FORMAT-SPEC-002`: `fa_sample_format` は device I/O、resample、gain、loudness normalize、limiter、filter、channel count 変更を持たない。
- `FA-SAMPLE-FORMAT-SPEC-003`: startup は unsupported conversion や暗黙 conversion を fail closed で拒否する。
- `FA-SAMPLE-FORMAT-SPEC-004`: runtime frame validation は source、stream identity、layout、encoding、bit depth、sample rate、channels、byte count を検証してから処理する。
- `FA-SAMPLE-FORMAT-SPEC-005`: 出力は `source_id` / `epoch` / sample rate / channels / layout を保持し、`stream_id` / encoding / bit depth だけを出力契約に従って更新する。
- `FA-SAMPLE-FORMAT-SPEC-006`: ROS-free backend は `PCM16LE/16 -> FLOAT32LE/32`、`PCM32LE/32 -> FLOAT32LE/32`、`FLOAT32LE/32 -> PCM16LE/16` だけを support する。
- `FA-SAMPLE-FORMAT-SPEC-007`: non-finite、range overflow、empty、sample 幅で割り切れない payload は clamp や output mutation なしで reject する。
- `FA-SAMPLE-FORMAT-SPEC-008`: package は標準 processing layout を持ち、pytest と gtest を colcon test に登録する。
- `FA-SAMPLE-FORMAT-SPEC-009`: launch file は `node_name` / `config_file` だけを公開し、変換条件や backend 名を直書きしない。
- `FA-SAMPLE-FORMAT-SPEC-010`: ROS graph smoke test で `PCM16LE/16` input から `FLOAT32LE/32` output が publish される。
