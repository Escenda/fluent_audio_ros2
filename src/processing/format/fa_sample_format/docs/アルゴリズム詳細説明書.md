# fa_sample_format アルゴリズム詳細説明書

## 0. backend 境界

`internal_float32le` backend は ROS2 topic/message を知らない。入力は byte列と `FrameContract`、出力は byte列と `ProcessResult` である。
node は `AudioFrame` から `FrameContract` を作り、backend の成功時だけ `AudioFrame.data`、format metadata、`stream_id` を更新して publish する。
backend が失敗した場合、node は frame を drop し、古い出力を publish しない。
`input_topic` / `output_topic` は ROS transport topic、`input_stream_id` / `output.stream_id` は frame identity として扱い、topic 名と同一視しない。

## 1. PCM16LE から FLOAT32LE

入力 byte列を little-endian の signed 16 bit PCM として 2 byte ずつ読む。

```text
signed_value = int16 little-endian sample
float_sample = signed_value / 32768.0
```

これにより `-32768` は `-1.0`、`32767` は `1.0` 未満の値になる。

## 2. PCM32LE から FLOAT32LE

入力 byte列を little-endian の signed 32 bit PCM として 4 byte ずつ読む。

```text
signed_value = int32 little-endian sample
float_sample = signed_value / 2147483648.0
```

これにより `-2147483648` は `-1.0`、`2147483647` は `1.0` 以下の値になる。

## 3. FLOAT32LE から PCM16LE

入力 byte列を little-endian の 32 bit float として 4 byte ずつ読む。各 sample は finite かつ `[-1.0, 1.0]` に入っている必要がある。範囲外の sample は clamp せず、変換結果を publish しない。

```text
if sample < 0:
    pcm16 = round(sample * 32768.0)
else:
    pcm16 = round(sample * 32767.0)
```

これにより `-1.0` は `-32768`、`0.0` は `0`、`1.0` は `32767` になる。`std::clamp` や saturating cast は使わない。

## 4. Byte count validation

`AudioFrame.data` は次を満たす必要がある。

```text
bytes_per_sample = input.bit_depth / 8
bytes_per_frame = expected.channels * bytes_per_sample
data.size > 0
data.size % bytes_per_frame == 0
```

不一致の場合、frame 境界が壊れているため publish せず drop する。

`FrameContract` は encoding、bit depth、sample rate、channels、layout、data size を含む。backend は config と runtime contract が一致しない場合、`FrameContractStatus` で理由を返す。

## 5. 変換しない項目

sample rate、channels、layout は変更しない。resample や channel 変換は別 package の責務である。
振幅加工も行わない。整数 PCM を正規化 float sample 表現へ写像するだけで、gain、limiter、loudness normalize は行わない。
`source_id`、`header`、`epoch` は保持し、`stream_id` は `output.stream_id` へ明示更新する。
