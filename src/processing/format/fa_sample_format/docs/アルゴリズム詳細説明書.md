# fa_sample_format アルゴリズム詳細説明書

## 1. PCM16LE から FLOAT32LE

入力 byte列を little-endian の signed 16 bit PCM として 2 byte ずつ読む。

```text
signed_value = int16 little-endian sample
float_sample = signed_value / 32768.0
```

これにより `-32768` は `-1.0`、`32767` は `1.0` 未満の値になる。

## 2. PCM32LE から FLOAT32LE

入力 byte列を little-endian の signed 32 bit PCM として 4 byte ずつ読む。

```text
signed_value = int32 little-endian sample
float_sample = signed_value / 2147483648.0
```

これにより `-2147483648` は `-1.0`、`2147483647` は `1.0` 以下の値になる。

## 3. Byte count validation

`AudioFrame.data` は次を満たす必要がある。

```text
bytes_per_sample = input.bit_depth / 8
bytes_per_frame = expected.channels * bytes_per_sample
data.size > 0
data.size % bytes_per_frame == 0
```

不一致の場合、frame 境界が壊れているため publish せず drop する。

## 4. 変換しない項目

sample rate、channels、layout は変更しない。resample や channel 変換は別 package の責務である。
振幅加工も行わない。整数 PCM を正規化 float sample 表現へ写像するだけで、gain、limiter、loudness normalize は行わない。
