# fa_channel_convert テスト設計

## Contract test

`test/unit/test_fa_channel_convert_audio_frame_contract.py` は、package の config、source、layout、colcon pytest registration を検査する。

## Backend test

`test/cpp/test_internal_float32le_channel_convert_backend.cpp` は、ROS 非依存 backend の supported conversion、mono duplicate、stereo average、invalid FLOAT32LE sample、malformed contract、no output mutation を検査する。

## 検証項目

- default config が 16000Hz / FLOAT32LE / 32bit / interleaved / 1ch -> 2ch / `mono_to_stereo_duplicate` と topic/stream identity 分離を明示する
- device I/O、resampling、sample format conversion、gain、limiter、noise gate、filtering、denoise の責務を持たない
- 起動時に未対応 mode/channel combination を reject する
- runtime frame が `source_id`、`input_stream_id`、format、channel、data size を満たさない場合 drop する
- `stream_id` を `output.stream_id` に、`channels` を `output.channels` に更新し、他 metadata を保持する
- node が `FrameContract` / `ProcessResult` で backend に委譲する
- backend に mono duplicate と stereo average の実装証跡がある
- backend に normalized range と finite 検証がある
- backend failure で output buffer が更新されない
- `ament_cmake_pytest` で contract test が colcon から実行される
- `ament_cmake_gtest` で backend gtest が colcon から実行される

## 実行コマンド

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/format/fa_channel_convert/test/unit -q
```

```bash
docker compose -f docker/compose.yml run --rm --no-deps --entrypoint bash vlabor -lc 'source /opt/ros/humble/setup.bash && cd /ros2_ws && colcon --log-base /tmp/fa_channel_convert_log build --packages-select fa_interfaces fa_channel_convert --symlink-install --build-base /tmp/fa_channel_convert_build --install-base /tmp/fa_channel_convert_install --event-handlers console_direct+ && source /tmp/fa_channel_convert_install/setup.bash && colcon --log-base /tmp/fa_channel_convert_log test --packages-select fa_channel_convert --build-base /tmp/fa_channel_convert_build --install-base /tmp/fa_channel_convert_install --event-handlers console_direct+ && colcon --log-base /tmp/fa_channel_convert_log test-result --test-result-base /tmp/fa_channel_convert_build/fa_channel_convert --verbose'
```
