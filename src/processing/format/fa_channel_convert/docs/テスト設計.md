# fa_channel_convert テスト設計

## Contract test

`test/unit/test_fa_channel_convert_audio_frame_contract.py` は、config validation、runtime frame rejection、metadata update、unsupported capability rejection を検証する。

## Backend test

`test/cpp/test_internal_float32le_channel_convert_backend.cpp` は、ROS 非依存 backend の supported conversion、mono duplicate、stereo average、invalid FLOAT32LE sample、malformed contract、no output mutation を検査する。

## 検証項目

- default config が 16000Hz / FLOAT32LE / 32bit / interleaved / 1ch -> 2ch / `mono_to_stereo_duplicate` と topic/stream identity 分離を明示する
- device I/O、resampling、sample format conversion、gain、limiter、noise gate、filtering、denoise の責務を持たない
- 起動時に未対応 mode/channel combination を reject する
- runtime frame が `source_id`、`input_stream_id`、format、channel、data size を満たさない場合 drop する
- `stream_id` を `output.stream_id` に、`channels` を `output.channels` に更新し、他 metadata を保持する
- node が `FrameContract` / `ProcessResult` で backend に委譲する
- backend API test が mono duplicate と stereo average の数値出力を検証する
- backend に normalized range と finite 検証がある
- backend failure で output buffer が更新されない
- unsupported encoding / bit depth / sample rate / layout / channel combination は変換せず drop または起動失敗にする

## 仕様とテストケース対応

| --- | --- | --- | --- |
| `FA-CHANNEL-CONVERT-TC-001` -> `FA-CHANNEL-CONVERT-SPEC-001` | `FA-CHANNEL-CONVERT-SPEC-001` | behavior checks verify only explicit channel-count conversion, with no I/O, resample, format conversion, gain, limiter, gate, filter, or denoise | behavior boundary |
| `FA-CHANNEL-CONVERT-TC-002` -> `FA-CHANNEL-CONVERT-SPEC-002` | `FA-CHANNEL-CONVERT-SPEC-002` | config validation requires `FLOAT32LE/32/interleaved`, 16000Hz, explicit input/output channels, mode, and stream identity split | config validation |

| `FA-CHANNEL-CONVERT-TC-005` -> `FA-CHANNEL-CONVERT-SPEC-005` | `FA-CHANNEL-CONVERT-SPEC-005` | runtime validation and backend API reject unsupported format, sample rate, channels, data size, and invalid float samples before mutation | frame rejection |
| `FA-CHANNEL-CONVERT-TC-006` -> `FA-CHANNEL-CONVERT-SPEC-006` | `FA-CHANNEL-CONVERT-SPEC-006` | behavior checks preserve identity and update only `stream_id`, `channels`, and `data` | frame behavior |

| `FA-CHANNEL-CONVERT-TC-009` -> `FA-CHANNEL-CONVERT-SPEC-009` | `FA-CHANNEL-CONVERT-SPEC-009` | runtime frame validation hands a typed `FrameContract` to backend and rejects failures without publish | backend boundary |
| `FA-CHANNEL-CONVERT-TC-010` -> `FA-CHANNEL-CONVERT-SPEC-010` | `FA-CHANNEL-CONVERT-SPEC-010` | executable tests cover supported conversion, unsupported capability rejection, malformed frame rejection, and no output mutation on failure | executable coverage |
