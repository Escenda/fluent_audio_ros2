# fa_channel_convert 仕様書

## 目的

`fa_channel_convert` は、FLOAT32LE / 32bit / interleaved の `fa_interfaces/msg/AudioFrame` に対して、明示的に許可された channel-count conversion だけを行う。

## ノード

| 項目 | 値 |
| --- | --- |
| package | `fa_channel_convert` |
| executable | `fa_channel_convert_node` |
| node name | `fa_channel_convert` |
| subscribe | `input_topic` |
| publish | `output_topic` |
| diagnostics | `diagnostics` |

## パラメータ

| パラメータ | 意味 |
| --- | --- |
| `input_topic` | 入力 `AudioFrame` topic |
| `output_topic` | 出力 `AudioFrame` topic |
| `input_stream_id` | 受理する入力 `AudioFrame.stream_id` |
| `input.channels` | 期待する入力 channel 数 |
| `output.channels` | 出力 channel 数 |
| `output.stream_id` | 出力 `AudioFrame.stream_id` |
| `conversion.mode` | 変換モード |
| `expected.sample_rate` | 期待する sample rate |
| `expected.encoding` | `FLOAT32LE` 固定 |
| `expected.bit_depth` | `32` 固定 |
| `expected.layout` | `interleaved` 固定 |
| `qos.depth` | audio stream QoS depth |
| `qos.reliable` | true なら reliable、false なら best effort |
| `diagnostics.publish_period_ms` | diagnostics publish interval |
| `diagnostics.qos.depth` | diagnostics QoS depth |
| `diagnostics.qos.reliable` | diagnostics QoS reliability |

## 起動時検証

ROS node は topic、QoS、diagnostics interval を検証する。backend は sample rate、channel、encoding、bit depth、layout、conversion mode を検証する。

以下を満たさない設定は起動失敗とする。

- `input_topic` と `output_topic` が空でない
- `input_stream_id` と `output.stream_id` が空でない
- `input_stream_id` / `output.stream_id` が raw / resolved ROS topic と衝突しない
- `input_stream_id` と `output.stream_id` が同一でない
- `expected.sample_rate > 0`
- `input.channels > 0`
- `output.channels > 0`
- `expected.encoding == FLOAT32LE`
- `expected.bit_depth == 32`
- `expected.layout == interleaved`
- `conversion.mode == mono_to_stereo_duplicate` かつ `input.channels == 1` かつ `output.channels == 2`
- `conversion.mode == stereo_to_mono_average` かつ `input.channels == 2` かつ `output.channels == 1`
- `qos.depth > 0`
- `diagnostics.publish_period_ms > 0`
- `diagnostics.qos.depth > 0`
- `diagnostics.qos.reliable` は bool

## Runtime frame 検証

ROS node は `source_id` / `stream_id` と topic 境界を検証し、backend は `FrameContract` と byte buffer を検証する。

以下を満たさない frame は warning を出して drop する。

- `source_id` と `stream_id` が空でない
- `stream_id == input_stream_id`
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `layout == interleaved`
- `sample_rate == expected.sample_rate`
- `channels == input.channels`
- `data` が空でない
- `data.size()` が `input.channels * sizeof(float)` で割り切れる
- すべての入力 sample が finite かつ `[-1.0, 1.0]` 内
- すべての出力 sample が finite かつ `[-1.0, 1.0]` 内

backend は `FrameContractStatus` / `ProcessStatus` を返す。失敗時に出力 buffer は更新しない。

## Metadata 更新

出力 frame は入力 frame を基準にし、以下だけを更新する。

- `stream_id = output.stream_id`
- `channels = output.channels`
- `data = converted bytes`

`header`、`source_id`、`sample_rate`、`bit_depth`、`encoding`、`layout`、`epoch` は保持する。

## 仕様 ID

| SPEC ID | 内容 |
| --- | --- |
| `FA-CHANNEL-CONVERT-SPEC-001` | `fa_channel_convert` は明示 channel-count conversion のみを行い、device I/O、resampling、sample format conversion、gain、limiter、noise gate、filtering、denoise を持たない。 |
| `FA-CHANNEL-CONVERT-SPEC-002` | default config は `FLOAT32LE/32/interleaved`、16000Hz、`1ch -> 2ch`、`mono_to_stereo_duplicate`、topic/stream identity 分離を明示する。 |
| `FA-CHANNEL-CONVERT-SPEC-003` | 起動時 config は topic、stream identity、sample rate、input/output channels、expected format、conversion mode、QoS、diagnostics を fail closed で検証する。 |
| `FA-CHANNEL-CONVERT-SPEC-004` | 対応変換は `mono_to_stereo_duplicate` と `stereo_to_mono_average` の明示組み合わせだけである。 |
| `FA-CHANNEL-CONVERT-SPEC-005` | runtime frame は source/stream identity、format、sample rate、channel count、data size、finite normalized samples を処理前に検証する。 |
| `FA-CHANNEL-CONVERT-SPEC-006` | 出力 frame は `stream_id`、channels、data だけを更新し、header、source_id、sample_rate、bit_depth、encoding、layout、epoch を保持する。 |
| `FA-CHANNEL-CONVERT-SPEC-007` | backend は mono duplicate と stereo average algorithm を所有する。 |
| `FA-CHANNEL-CONVERT-SPEC-008` | backend failure では output buffer を更新せず、node は失敗 frame を publish しない。 |
| `FA-CHANNEL-CONVERT-SPEC-009` | backend は ROS2 topic/message を知らず、`FrameContract` / `ProcessResult` 境界で node に接続する。 |
| `FA-CHANNEL-CONVERT-SPEC-010` | executable tests は supported conversion、unsupported capability rejection、malformed frame rejection、failure 時の no output mutation を検証する。 |
