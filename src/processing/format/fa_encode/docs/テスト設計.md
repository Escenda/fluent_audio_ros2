# fa_encode テスト設計

## Unit Test

- `FA-ENCODE-SPEC-001`: package 化前でも README、仕様書、アルゴリズム詳細説明書、テスト設計、backend docs が存在する。
- `FA-ENCODE-SPEC-002`: `package.xml` と `CMakeLists.txt` を package 化前に置かない。
- `FA-ENCODE-SPEC-003`: 仕様書が `fa_out` に encode を隠さない責務分離を明記する。
- `FA-ENCODE-SPEC-004`: 仕様書が missing codec / backend / input format を fail closed として扱う。

## Integration Test

ROS2 package 化後に、fixture PCM `AudioFrame` を fake encoder backend に渡し、metadata 一致時だけ encoded payload が publish されることを確認する。

## Launch Test

ROS2 package 化後に、missing params file、missing backend、missing codec で起動失敗することを確認する。
