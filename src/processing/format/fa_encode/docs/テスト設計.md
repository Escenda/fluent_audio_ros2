# fa_encode テスト設計

## Unit Test

- `FA-ENCODE-SPEC-001`: README、仕様書、アルゴリズム詳細説明書、テスト設計、backend docs が存在する。
- `FA-ENCODE-SPEC-002`: package 化後に `package.xml` と `CMakeLists.txt` が存在する。
- `FA-ENCODE-SPEC-003`: 仕様書が `fa_out` に encode を隠さない責務分離を明記する。
- `FA-ENCODE-SPEC-004`: 仕様書が missing codec / backend / input format を fail closed として扱う。
- `FA-ENCODE-SPEC-005`: backend が ROS2 / `fa_interfaces` を include しない。
- `FA-ENCODE-SPEC-006`: external command timeout、empty output、oversized output を error として扱う。
- `FA-ENCODE-SPEC-007`: `input_stream_id` / `output.stream_id` は必須で、ROS topic identity と同値にしない。

## Integration Test

fixture PCM `AudioFrame` を explicit external command backend に渡し、`stream_id == input_stream_id` かつ declared output contract 一致時だけ `stream_id = output.stream_id` の `EncodedAudioChunk` が publish されることを確認する。

## Launch Test

missing params file、missing backend、missing codec、missing command executable で起動失敗することを確認する。
