# fa_encode 仕様書

## 1. 目的

`fa_encode` は PCM `AudioFrame` を encoded audio payload へ変換する format processing node である。
`fa_out` が出力 sink を開き、`fa_encode` が codec encode を担当することで、sink adapter と format conversion の責務を分離する。

## 2. 入力

入力は `fa_interfaces/msg/AudioFrame` とし、次の metadata を必須にする。

- `source_id`
- `stream_id`
- `sample_rate`
- `channels`
- `encoding`
- `bit_depth`
- `layout`
- `epoch`

入力 `stream_id` は `input_stream_id` と一致する必要がある。
入力 format は config と完全一致する必要がある。
`input_stream_id` と `output.stream_id` は ROS topic 名とは別の audio identity とし、
raw / resolved `input_topic` / `output_topic` と同値であってはならない。

## 3. 出力

出力は `fa_interfaces/msg/EncodedAudioChunk` とする。
container、codec、bitrate、frame duration、timestamp policy は startup config で明示する。
`stream_id` は `output.stream_id` に更新し、`source_id` は入力 `AudioFrame` から引き継ぐ。
`sequence` と `media_time_ns` は `epoch` 内で入力 PCM sample 数から単調に進める。

## 4. 責務外

- speaker / network / file sink を開くこと
- resample
- bit-depth conversion before encoder input
- channel count conversion
- gain / normalize / limiter
- VAD / ASR / embedding
- encoder failure 時の無音 payload 生成

## 5. fail closed 条件

- backend 未指定
- codec 未指定
- input format 未指定
- `input_stream_id` / `output.stream_id` 未指定、topic 名との同値、または相互同値
- input frame metadata が config と一致しない
- encoder が空 payload を返す
- encoded output contract が config と一致しない
- timestamp / source identity が欠損
