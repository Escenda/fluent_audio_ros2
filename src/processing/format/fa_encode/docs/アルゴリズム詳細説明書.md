# fa_encode アルゴリズム詳細説明書

1. startup config で `backend.name`、codec、input sample rate、input channels、input encoding、input layout を検証する。
2. `AudioFrame` の source identity、timestamp、format metadata を検証する。
3. backend に PCM samples と encode contract を渡す。
4. backend は external command の stdout bytes を encoded payload として返し、startup config の output contract を publish metadata として返す。
5. node は encoded payload が非空で、output contract が config と一致することを検証する。
6. 一致した場合だけ `EncodedAudioChunk` として publish する。

encode は PCM 表現を codec 表現へ圧縮する処理であり、音量補正やリサンプルではない。
必要な前処理は `fa_resample`、`fa_channel_convert`、`fa_sample_format`、`fa_bit_depth`、dynamics/frequency/correction node に明示的に接続する。

## Backend Boundary

backend は codec/container encode のみを担当し、ROS2 topic/message を知らない。
backend は unsupported codec、input format mismatch、encode failure を明示的な error として返す。
encode 失敗時に silence payload、empty packet、前回 packet の再利用はしない。

初期実装の `external_codec_encoder` は external command の stdin に PCM payload を渡し、stdout を encoded payload として受け取る。stdout は byte payload のみであり、codec/container/payload_format を stdout から推定しない。runtime metadata が必要な encoder では、stdout とは別の明示 protocol を追加し、推測や probe で補完しない。command は shell 経由では起動せず、`backend.command.executable` と `backend.command.arguments` を明示的に分離する。command timeout、最大 output byte 数、空 output は fail closed として扱う。
