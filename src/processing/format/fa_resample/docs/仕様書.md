# fa_resample 仕様書

## 目的

`fa_resample` は `AudioFrame` の sample rate だけを変換する format processing node である。現在の contract は `FLOAT32LE` / 32-bit / interleaved 入力を、`target_sample_rate` の `FLOAT32LE` / 32-bit / interleaved 出力へ変換する。

PCM16 / PCM32 から float32 への sample format 変換は `fa_sample_format` の責務であり、`fa_resample` は整数 PCM decode、bit depth 変換、clamp を行わない。

resampling 本体は ROS 非依存の `internal_linear_resampler` backend に置く。ROS node は topic routing、stream identity validation、publish/drop、diagnostics を担当する。

## 入力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `mic.input_topic` | `audio/frame` | microphone / source 側 frame |
| `ref.input_topic` | `audio/output/frame` | AEC reference 用 frame |

## 出力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `mic.output_topic` | `audio/resample16k/mic` | resampled mic stream |
| `ref.output_topic` | `audio/resample16k/ref` | resampled reference stream |

## stream identity

| parameter | default | 内容 |
| --- | --- | --- |
| `mic.input_stream_id` | `audio/float32/mic` | mic 入力として受け入れる `AudioFrame.stream_id` |
| `mic.output.stream_id` | `audio/preprocessed/mono16k` | resampled mic 出力の `AudioFrame.stream_id` |
| `ref.input_stream_id` | `audio/playback/main` | reference 入力として受け入れる `AudioFrame.stream_id` |
| `ref.output.stream_id` | `audio/preprocessed/ref16k` | resampled reference 出力の `AudioFrame.stream_id` |

`*.input_topic` / `*.output_topic` は ROS transport topic、`*.input_stream_id` / `*.output.stream_id` は frame identity である。enabled stream では topic 名と stream ID が同一視できる設定、または input/output stream ID が同一の設定を起動失敗にする。

## parameters

| parameter | 意味 |
| --- | --- |
| `target_sample_rate` | 出力 sample rate。`> 0` が必須 |
| `input.encoding` | `FLOAT32LE` 固定。別値は起動失敗 |
| `input.bit_depth` | `32` 固定。別値は起動失敗 |
| `input.layout` | `interleaved` 固定。別値は起動失敗 |
| `output.encoding` | `FLOAT32LE` 固定。別値は起動失敗 |
| `output.bit_depth` | `32` 固定。別値は起動失敗 |
| `mic.enabled` | mic stream の有効化 |
| `mic.input_stream_id` | mic 入力 stream identity |
| `mic.output.stream_id` | mic 出力 stream identity |
| `ref.enabled` | reference stream の有効化 |
| `ref.input_stream_id` | reference 入力 stream identity |
| `ref.output.stream_id` | reference 出力 stream identity |
| `qos.depth` | audio frame QoS depth |
| `qos.reliable` | reliable / best effort |
| `diagnostics.publish_period_ms` | diagnostics publish 周期 |
| `diagnostics.qos.depth` | diagnostics QoS depth |
| `diagnostics.qos.reliable` | diagnostics QoS reliability |

## lifecycle

起動時に parameter を検証し、enabled stream の publisher / subscriber を作る。topic 欠落、stream ID 欠落、topic と stream ID の衝突、`target_sample_rate <= 0`、`input.encoding != FLOAT32LE`、`input.bit_depth != 32`、`input.layout != interleaved`、`output.encoding != FLOAT32LE`、`output.bit_depth != 32` は起動失敗にする。

## QoS

audio frame topic と diagnostics topic は、それぞれ明示 parameter の QoS depth/reliability に従って reliable / best-effort を切り替える。

## error handling

不正 frame は drop し、diagnostics counter を増やす。sample rate / channel / data size / encoding / bit depth / layout の不整合を別 format として推測しない。

drop 条件:

- `source_id` / `stream_id` が空
- `stream_id` が対応する `mic.input_stream_id` / `ref.input_stream_id` と一致しない
- `sample_rate == 0` または `channels == 0`
- `encoding != FLOAT32LE`
- `bit_depth != 32`
- `layout != interleaved`
- `data` が空、または `channels * sizeof(float)` の倍数ではない
- sample が finite ではない、または normalized range `[-1.0, 1.0]` を外れる

## backend selection

現在は `internal_linear_resampler` backend を使う。高品質 resampler を追加する場合は backend として明示し、fallback では切り替えない。

backend は `ProcessStatus` と `FrameContractStatus` を返し、invalid input を空 vector だけで表現しない。拒否時は output vector を更新しない。

## non-goals

- device input
- codec decode
- sample format conversion
- bit depth conversion
- clipping / limiter / clamp
- channel remix
- denoise
- AGC
- VAD
