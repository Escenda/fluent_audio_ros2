# fa_resample 仕様書

## 目的

`fa_resample` は `AudioFrame.sample_rate` だけを変更する format processing node である。
入力音声の sample format、bit depth、channel count、interleave 形式、gain、noise、発話判定、文字起こしは変更しない。

現行 capability は `FLOAT32LE` / 32-bit / `interleaved` 入力を、同じ sample format / bit depth / channel count / layout のまま `target_sample_rate` へ変換することである。

`PCM16LE` / `PCM32LE` から `FLOAT32LE` への変換は `fa_sample_format` の責務である。channel 変換は channel conversion node の責務である。`fa_resample` は hidden conversion、clamp、normalize、denoise、VAD、ASR を持たない。

## 責務分離

ROS node は次を担当する。

- parameter validation
- mic/ref topic subscription と publication
- `source_id` / `stream_id` validation
- output `AudioFrame` mapping
- drop counter と warning log
- diagnostics publish

resampler backend は次を担当する。

- `FLOAT32LE` / 32-bit / `interleaved` frame contract validation
- finite normalized sample validation
- stream ごとの resampler state
- sample rate conversion
- backend metrics
- typed `ProcessStatus` / `FrameContractStatus`

backend は ROS2 topic、publisher/subscriber、`fa_interfaces/msg/AudioFrame` を知らない。

## 入力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `mic.input_topic` | `audio/frame` | microphone / source 側 frame |
| `ref.input_topic` | `audio/output/frame` | AEC reference 用 frame |

## 出力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `mic.output_topic` | `audio/resample16k/mic` | resampled mic stream |
| `ref.output_topic` | `audio/resample16k/ref` | resampled reference stream |

## stream identity

| parameter | default | 内容 |
| --- | --- | --- |
| `mic.input_stream_id` | `audio/float32/mic` | mic 入力として受け入れる `AudioFrame.stream_id` |
| `mic.output.stream_id` | `audio/preprocessed/mono16k` | resampled mic 出力の `AudioFrame.stream_id` |
| `ref.input_stream_id` | `audio/playback/main` | reference 入力として受け入れる `AudioFrame.stream_id` |
| `ref.output.stream_id` | `audio/preprocessed/ref16k` | resampled reference 出力の `AudioFrame.stream_id` |

`*.input_topic` / `*.output_topic` は ROS transport topic、`*.input_stream_id` / `*.output.stream_id` は frame identity である。
enabled stream では topic 名と stream ID が同一視できる設定、または input/output stream ID が同一の設定を起動失敗にする。

## parameters

| parameter | 意味 |
| --- | --- |
| `target_sample_rate` | 出力 sample rate。`> 0` が必須 |
| `backend.name` | 必須。`internal_linear_resampler`、`speexdsp`、`soxr` のいずれか |
| `backend.quality` | backend ごとの品質指定。下記参照 |
| `input.encoding` | `FLOAT32LE` 固定。別値は起動失敗 |
| `input.bit_depth` | `32` 固定。別値は起動失敗 |
| `input.layout` | `interleaved` 固定。別値は起動失敗 |
| `output.encoding` | `FLOAT32LE` 固定。別値は起動失敗 |
| `output.bit_depth` | `32` 固定。別値は起動失敗 |
| `mic.enabled` | mic stream の有効化 |
| `mic.input_stream_id` | mic 入力 stream identity |
| `mic.output.stream_id` | mic 出力 stream identity |
| `ref.enabled` | reference stream の有効化 |
| `ref.input_stream_id` | reference 入力 stream identity |
| `ref.output.stream_id` | reference 出力 stream identity |
| `qos.depth` | audio frame QoS depth。`> 0` が必須 |
| `qos.reliable` | reliable / best effort |
| `diagnostics.publish_period_ms` | diagnostics publish 周期。`> 0` が必須 |
| `diagnostics.qos.depth` | diagnostics QoS depth。`> 0` が必須 |
| `diagnostics.qos.reliable` | diagnostics QoS reliability |

## backend selection

`backend.name` は明示必須である。未知の backend 名は起動失敗にする。

| `backend.name` | `backend.quality` | 用途 |
| --- | --- | --- |
| `internal_linear_resampler` | 未設定または `debug_reference` | debug / reference 用の内部線形 backend |
| `speexdsp` | integer `0..10` | realtime voice pipeline 候補 |
| `soxr` | `QQ` / `LQ` / `MQ` / `HQ` / `VHQ` | high-quality resampling / 評価 / recording 用 |

`speexdsp` 選択時に `libspeexdsp.so.1` が見つからない場合、起動失敗にする。
`soxr` 選択時に `libsoxr.so.0` が見つからない場合、起動失敗にする。
選択した backend の runtime dependency が無い場合に `internal_linear_resampler` へ自動 fallback してはならない。

`package.xml` は runtime library の ROS package contract として `libspeexdsp1` と `libsoxr0` を
`exec_depend` に宣言する。C++ 実装は SpeexDSP / SoXR の header を直接 include せず、
`libspeexdsp.so.1` / `libsoxr.so.0` を `dlopen` するため、これらを build dependency や dev package
dependency として扱わない。

## input / output capability

入力として受け付ける `AudioFrame` contract は次に限定する。

| field | supported value |
| --- | --- |
| `source_id` | 空文字不可 |
| `stream_id` | 対応する `*.input_stream_id` と完全一致 |
| `encoding` | `FLOAT32LE` |
| `bit_depth` | `32` |
| `layout` | `interleaved` |
| `channels` | `>= 1` |
| `sample_rate` | `> 0` |
| `data` | non-empty、`channels * sizeof(float)` に整列 |
| sample value | finite、normalized range `[-1.0, 1.0]` |

出力 `AudioFrame` は次の contract を持つ。

| field | 出力値 |
| --- | --- |
| `source_id` | 入力から維持 |
| `stream_id` | 対応する `*.output.stream_id` |
| `encoding` | `FLOAT32LE` |
| `sample_rate` | `target_sample_rate` |
| `channels` | 入力と同一 |
| `bit_depth` | `32` |
| `layout` | `interleaved` |
| `epoch` | 入力から維持 |
| `header` | 入力から維持 |

capability 外の frame は backend state を変える前に rejection する。非対応の sample format、bit depth、channel count、layout、値域を補正して成功扱いしない。

## stream state

backend は stream ごとに state を保持する。state の map key は `stream_id` であり、作成時の stream contract は次を含む。

- `stream_id`
- frame contract: `encoding`、`sample_rate`、`channels`、`bit_depth`、`layout`
- `target_sample_rate`
- backend name
- backend quality

同一 `stream_id` の途中で frame contract、`target_sample_rate`、backend name、backend quality が変わった場合は stream contract violation とする。
backend を chunk ごとに hidden reinitialization してはならない。fractional phase、filter history、pending output、library state は stream state として保持する。

## lifecycle

起動時に parameter を検証し、選択 backend を生成し、enabled stream の publisher / subscriber と diagnostics publisher を作る。

起動失敗条件:

- required parameter が未設定
- `backend.name` が未知
- `backend.quality` が backend の許可範囲外
- 選択 backend の runtime dependency が無い
- `target_sample_rate <= 0`
- `input.encoding != FLOAT32LE`
- `input.bit_depth != 32`
- `input.layout != interleaved`
- `output.encoding != FLOAT32LE`
- `output.bit_depth != 32`
- QoS / diagnostics depth または diagnostics period が `<= 0`
- enabled stream の topic / stream ID が空
- topic と stream ID が同一視できる
- input/output stream ID が同一

## error handling

不正 frame は drop し、drop counter を増やす。backend は `ProcessStatus` と `FrameContractStatus` を返す。invalid input を空 vector だけで表現しない。

drop 条件:

- `source_id` / `stream_id` が空
- `stream_id` が対応する `mic.input_stream_id` / `ref.input_stream_id` と一致しない
- `sample_rate == 0` または `channels == 0`
- `encoding != FLOAT32LE`
- `bit_depth != 32`
- `layout != interleaved`
- `data` が空、または `channels * sizeof(float)` の倍数ではない
- sample が finite ではない、または normalized range `[-1.0, 1.0]` を外れる
- 同一 `stream_id` の stream contract が途中で変わる
- backend process / encode に失敗する

## diagnostics metrics

diagnostics には少なくとも次を出す。

| metric | 内容 |
| --- | --- |
| `algorithmic_delay_input_samples` | 入力 sample rate 基準の algorithmic delay |
| `algorithmic_delay_output_samples` | 出力 sample rate 基準の algorithmic delay |
| `algorithmic_delay_ms` | 出力 sample rate 基準の ms 換算 delay |
| `processing_time_mean_ms` | backend process call の平均処理時間 |
| `processing_time_max_ms` | backend process call の最大処理時間 |
| `input_frames_total` | backend が受け取った累積 input frame 数 |
| `output_frames_total` | backend が出した累積 output frame 数 |
| `expected_output_frames` | input/output sample rate 比から計算した期待累積 output frame 数 |
| `frame_count_error_samples` | `output_frames_total - expected_output_frames` の丸め差 |

## quality test XML metrics

`test/cpp/test_resampler_quality_metrics.cpp` は、backend を実際に処理して得た結果を GTest XML property として残す。

音質差の test-only metrics は production diagnostics ではない。対象は次である。

- `<prefix>_rms_error`
- `<prefix>_peak_error`
- `<prefix>_snr_db`
- `<prefix>_compared_samples`
- `reference_impulse_peak_offset_samples`

backend 別の delay / processing / frame count metrics は `BackendMetrics` 由来であり、音質差そのものではない。
品質比較テストでは、同じ backend prefix で次を GTest XML に記録する。

- `<prefix>_algorithmic_delay_input_samples`
- `<prefix>_algorithmic_delay_output_samples`
- `<prefix>_algorithmic_delay_ms`
- `<prefix>_processing_time_mean_ms`
- `<prefix>_processing_time_max_ms`
- `<prefix>_input_frames_total`
- `<prefix>_output_frames_total`
- `<prefix>_frame_count_error_samples`

## metrics reporter

`lib/fa_resample/fa_resample_metrics_report.py` は stdlib-only の metrics reporter である。
`fa_resample_backend_test.gtest.xml` 単体、または `fa_resample` GTest XML を含む directory を読み、quality metrics と backend metrics を人間が読める形に整形する。

reporter が扱う quality metrics は次である。

- `rms_error`
- `peak_error`
- `snr_db`
- `compared_samples`

reporter が扱う backend metrics は次である。

- `algorithmic_delay_input_samples`
- `algorithmic_delay_output_samples`
- `algorithmic_delay_ms`
- `processing_time_mean_ms`
- `processing_time_max_ms`
- `input_frames_total`
- `output_frames_total`
- `frame_count_error_samples`

reporter は incomplete metric group を成功扱いしない。欠けた metric group は失敗として扱い、欠損した品質・backend 指標を隠さない。

## non-goals

- device input
- codec decode / encode
- sample format conversion
- bit depth conversion
- channel conversion / remix / downmix
- interleaved / planar conversion
- gain / normalize / limiter / clamp
- denoise / AGC
- VAD / KWS / ASR / Turn Detector
