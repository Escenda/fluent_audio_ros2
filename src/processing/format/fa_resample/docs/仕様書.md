# fa_resample 仕様書

## 目的

`fa_resample` は `AudioFrame` を FluentAudio の解析系で扱いやすい sample rate へ変換する format processing node である。現在の contract は 16kHz 出力に固定し、VAD / ASR / denoise / AEC などの前段へ安定した stream を供給する。

## 入力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `mic.input_topic` | `audio/frame` | microphone / source 側 frame |
| `ref.input_topic` | `audio/output/frame` | AEC reference 用 frame |

## 出力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `mic.output_topic` | `audio/resample16k/mic` | 16kHz mic stream |
| `ref.output_topic` | `audio/resample16k/ref` | 16kHz reference stream |

## parameters

| parameter | 意味 |
| --- | --- |
| `target_sample_rate` | 16kHz 固定。別値は起動失敗 |
| `output.encoding` | 出力 encoding |
| `output.bit_depth` | 16 or 32 |
| `mic.enabled` | mic stream の有効化 |
| `ref.enabled` | reference stream の有効化 |
| `qos.depth` | audio frame QoS depth |
| `qos.reliable` | reliable / best effort |
| `diagnostics.publish_period_ms` | diagnostics publish 周期 |

## lifecycle

起動時に parameter を検証し、enabled stream の publisher / subscriber を作る。topic 欠落、`target_sample_rate` 不一致、invalid bit depth は起動失敗にする。

## QoS

audio frame topic は `qos.reliable` に従って reliable / best-effort を切り替える。diagnostics は `SystemDefaultsQoS` を使う。

## error handling

不正 frame は drop し、diagnostics counter を増やす。sample rate / channel / data size / bit depth の不整合を別 format として推測しない。

## backend selection

なし。現在は node 内の deterministic linear resampler を使う。高品質 resampler を追加する場合は `backends/` に engine を分離する。

## non-goals

- device input
- codec decode
- channel remix
- denoise
- AGC
- VAD
