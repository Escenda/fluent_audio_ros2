# fa_resample テスト設計

## 現在の自動テスト

### C++ backend test

- `target_sample_rate <= 0` を constructor で拒否する。
- 入力 sample rate と `target_sample_rate` から output frame 数を決める。
- 48kHz から 16kHz への downsample と 16kHz から 48kHz への upsample を検証する。
- `FLOAT32LE` / 32-bit / interleaved だけを受け付ける。
- PCM16LE / PCM32LE を reject し、backend 内で sample format conversion しない。
- NaN / Inf / normalized range 外 sample を reject し、clamp しない。
- invalid bit depth / zero channel / empty data / unaligned data を status で返す。
- backend が typed failure status を返し、拒否時に output vector を更新しない。

### Python static contract test

- default config が `input.FLOAT32LE/32/interleaved -> output.FLOAT32LE/32` を明示する。
- ROS node が `source_id`、`epoch` を維持し、`stream_id` を `*.output.stream_id` に更新する。
- default config が `*.input_topic` / `*.output_topic` と `*.input_stream_id` / `*.output.stream_id` を分ける。
- `internal_linear_resampler` が decode / validate / resample / encode を持ち、node からは消えている。
- PCM16 / PCM32 decode、PCM16 encode、`std::clamp` が戻っていないことを確認する。
- backend が ROS2 topic/message を知らないことを確認する。
- colcon test が C++ gtest と pytest の両方を実行することを確認する。

### ROS2 graph smoke test

- `FaResampleNode` を `NodeOptions` parameter override 付きで起動する。
- test publisher が `FLOAT32LE/32/interleaved`、48kHz、mono、`stream_id == mic.input_stream_id` の `AudioFrame` を publish する。
- test subscriber が `target_sample_rate`、`FLOAT32LE/32/interleaved`、mono の `AudioFrame` を受信する。
- `source_id`、`epoch` を維持し、`stream_id` を `mic.output.stream_id` に更新する。
- `stream_id` が対応する input stream ID と一致しない frame を reject する。

### Launch contract test

- `test/launch/test_fa_resample_launch_contract.py` で launch file が `node_name` と `config_file` だけを公開し、node parameter は `config/default.yaml` から読むことを確認する。
- launch file に `target_sample_rate` や `backend.name` を直書きしないことを確認する。
- default config が `FLOAT32LE/32/interleaved` から `FLOAT32LE/32` への resample node として、topic / stream ID を分けて明示していることを確認する。
- `ros2 launch` 経由で `target_sample_rate <= 0` の config が起動失敗することを確認する。
- `ros2 launch` 経由で enabled stream の topic が空なら起動失敗することを確認する。

## 未実装の runtime test

次は必要だが、現時点では package 内の自動テストとして未実装である。

### integration test

- `fa_in -> fa_resample -> fa_vad` の topic wiring を検証する。
- `fa_in -> fa_sample_format -> fa_resample -> fa_vad` の明示 format pipeline を検証する。
- mic/ref の enabled 切替で不要 topic を作らないこと。

### failure test

- data size が frame boundary に合わない。
- NaN / Inf sample。
- unsupported bit depth。
- unsupported encoding。
- range overflow sample。

### latency test

callback あたりの処理時間と diagnostics counter を測る。

### real device smoke test

実 microphone から frame を入力し、設定した `target_sample_rate` の output topic の sample rate / channel / duration が期待値と一致することを確認する。

## 仕様とテストケース対応

### 仕様 ID

- `FA-RESAMPLE-SPEC-001`: default config は `FLOAT32LE/32/interleaved` 入力と `FLOAT32LE/32` 出力を明示し、topic と stream ID を分離する。
- `FA-RESAMPLE-SPEC-002`: QoS depth は設定境界で検証し、暗黙補正や system default QoS 依存をしない。
- `FA-RESAMPLE-SPEC-003`: node は `source_id` / `epoch` を維持し、`stream_id` を `output.stream_id` に更新する。
- `FA-RESAMPLE-SPEC-004`: non-`FLOAT32LE`、未識別 stream、不正 layout / bit depth / channels / data size / sample は処理前に reject する。
- `FA-RESAMPLE-SPEC-005`: backend は decode / validate / resample / encode を所有し、ROS2 topic/message を知らず typed status を返す。
- `FA-RESAMPLE-SPEC-006`: backend は 48kHz -> 16kHz と 16kHz -> 48kHz の resample を channel count を変えずに実行する。
- `FA-RESAMPLE-SPEC-007`: package は C++ gtest と pytest を colcon test に登録する。
- `FA-RESAMPLE-SPEC-008`: launch file は `node_name` / `config_file` だけを公開し、target sample rate や backend 名を直書きしない。
- `FA-RESAMPLE-SPEC-009`: ROS graph smoke test で `FLOAT32LE/32/interleaved` input から target sample rate の output が publish される。
