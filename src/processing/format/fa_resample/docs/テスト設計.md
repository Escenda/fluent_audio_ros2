# fa_resample テスト設計

## テスト方針

`fa_resample` のテストは source text や Markdown の文字列検査ではなく、backend contract、startup failure、frame rejection、stream state、quality metrics、ROS graph behavior を実行経路で確認する。

書類は実装完了の証拠ではない。検証済み、未検証、未実施を分けて扱う。

## 現在の自動テスト観点

### backend selection / startup validation

- `backend.name` が `internal_linear_resampler`、`speexdsp`、`soxr` のいずれかに限定される。
- unknown backend は起動失敗になる。
- `speexdsp` の `backend.quality` は integer `0..10` だけを受け付ける。
- `soxr` の `backend.quality` は `QQ` / `LQ` / `MQ` / `HQ` / `VHQ` だけを受け付ける。
- `speexdsp` / `soxr` 選択時に `internal_linear_resampler` へ fallback しない。
- 選択 backend の runtime dependency が無い場合は、その backend の startup failure または optional runtime unavailable skip として扱う。
- `package.xml` は runtime library contract として `libspeexdsp1` / `libsoxr0` を `exec_depend` に宣言する。

対象:

- `test/cpp/test_internal_linear_resampler_backend.cpp`
- `test/cpp/test_resample_graph.cpp`

### backend capability validation

- `FLOAT32LE` / 32-bit / interleaved だけを受け付ける。
- `channels >= 1` と `sample_rate > 0` を要求する。
- `data.size()` が `channels * sizeof(float)` に整列していることを要求する。
- PCM16LE / PCM32LE を reject し、backend 内で sample format conversion しない。
- NaN / Inf / normalized range 外 sample を reject し、clamp しない。
- invalid frame は typed `ProcessStatus` / `FrameContractStatus` を返し、caller-owned output vector を失敗表現に使わない。

対象:

- `test/cpp/test_internal_linear_resampler_backend.cpp`

### stream state / frame count metrics

- `internal_linear_resampler` は stream state を持ち、chunk 境界をまたいで累積 output frame index と input tail buffer を維持する。
- `algorithmic_delay_input_samples`、`algorithmic_delay_output_samples`、`algorithmic_delay_ms`、processing time metrics、frame count metrics を確認する。
- frame count metrics は `input_frames_total`、`output_frames_total`、`expected_output_frames`、`frame_count_error_samples` を含む。
- 同じ stream の contract が途中で変わる場合は、stream contract violation として扱う必要がある。

対象:

- `test/cpp/test_internal_linear_resampler_backend.cpp`
- `test/cpp/test_resample_graph.cpp`

### ROS2 graph behavior

- `FaResampleNode` を `NodeOptions` parameter override 付きで起動する。
- test publisher が `FLOAT32LE/32/interleaved`、48kHz、mono、`stream_id == mic.input_stream_id` の `AudioFrame` を publish する。
- test subscriber が `target_sample_rate`、`FLOAT32LE/32/interleaved`、同一 channel count の `AudioFrame` を受信する。
- `source_id` と `epoch` を維持し、`stream_id` を `mic.output.stream_id` に更新する。
- `stream_id` が対応する input stream ID と一致しない frame を reject し、publish しない。
- diagnostics に backend name / quality と delay / processing / frame count metrics が出ることを確認する。
- SoXR graph smoke は `backend.name=soxr`、`backend.quality=MQ` で `FaResampleNode` を起動し、
  `AudioFrame` の publish / subscribe と `/diagnostics` を確認する。
- SpeexDSP graph smoke は `backend.name=speexdsp`、`backend.quality=6` で `FaResampleNode` を起動し、
  `AudioFrame` の publish / subscribe と `/diagnostics` を確認する。
- SoXR / SpeexDSP graph smoke の diagnostics 確認対象は `backend.name`、`backend.quality`、
  `algorithmic_delay_ms`、`processing_time_mean_ms`、`input_frames_total`、`output_frames_total`、
  `expected_output_frames`、`frame_count_error_samples` である。
- backend 選択 graph smoke は実機 live path ではなく、ROS graph 上の backend selection / diagnostics contract 検証である。
- graph smoke の入力は低振幅 multi-tone とする。高振幅 ramp で `FLOAT32LE` encoding failed になった履歴があるが、
  production contract は弱めず、test input だけを contract 内の値域へ合わせる。

対象:

- `test/cpp/test_resample_graph.cpp`

### sample_format -> resample integration

- `fa_sample_format` が PCM16LE input を `FLOAT32LE/32/interleaved` へ明示変換する。
- `fa_resample` がその `FLOAT32LE` stream を受け、16kHz の `FLOAT32LE` stream を publish する。
- `source_id`、`epoch`、channel count、layout を維持し、`stream_id` と `sample_rate` を contract 通りに更新する。
- PCM16LE を `fa_resample` に直結しない。

対象:

- `test/integration/test_sample_format_resample_graph.py`

### quality metrics

quality metrics は test-only evidence として扱う。production diagnostics ではない。

音質差として確認する test-only metric:

- `rms_error`
- `peak_error`
- `snr_db`
- `impulse_peak_offset_samples`

信号:

- passband multi-tone
- out-of-band alias probe
- impulse

比較:

- SoXR `VHQ` を golden reference にする。
- SoXR `MQ` / `HQ` と `internal_linear_resampler` を比較する。
- SoXR `MQ` / `HQ` は passband と alias probe で `internal_linear_resampler` より良い metric を要求する。
- SpeexDSP quality `6` は runtime library が使える場合に SoXR `VHQ` reference と比較する。
- SpeexDSP runtime library が無い場合、この metric test は optional skip とする。

GTest XML には、音質差とは別に、実際に処理した backend の `BackendMetrics` も同じ prefix で記録する。

backend 別に XML へ残す metric:

- `<prefix>_algorithmic_delay_input_samples`
- `<prefix>_algorithmic_delay_output_samples`
- `<prefix>_algorithmic_delay_ms`
- `<prefix>_processing_time_mean_ms`
- `<prefix>_processing_time_max_ms`
- `<prefix>_input_frames_total`
- `<prefix>_output_frames_total`
- `<prefix>_frame_count_error_samples`

これは source 文字列検査ではなく、backend を実行したあとに取得した metrics の `RecordProperty` である。

対象:

- `test/cpp/test_resampler_quality_metrics.cpp`

## 仕様 ID

- `FA-RESAMPLE-SPEC-001`: `fa_resample` は sample rate conversion だけを担当し、format / bit depth / channel / gain / denoise / VAD / ASR を隠さない。
- `FA-RESAMPLE-SPEC-002`: `backend.name` は明示必須で、unknown backend は startup failure になる。
- `FA-RESAMPLE-SPEC-003`: `backend.quality` は backend ごとの許可値だけを受け付ける。
- `FA-RESAMPLE-SPEC-004`: 選択 backend の runtime dependency が無い場合は startup failure であり、fallback しない。
- `FA-RESAMPLE-SPEC-005`: input capability は `FLOAT32LE/32/interleaved/channels>=1/sample_rate>0` に限定する。
- `FA-RESAMPLE-SPEC-006`: output は input と同じ format / bit depth / channel count / layout で、sample rate だけ `target_sample_rate` へ変わる。
- `FA-RESAMPLE-SPEC-007`: backend は stream state を `stream_id` と stream contract で管理し、mid-stream contract change を violation とする。
- `FA-RESAMPLE-SPEC-008`: diagnostics は delay、processing time、frame count metrics を publish する。
- `FA-RESAMPLE-SPEC-009`: SoXR `VHQ` reference に対する test-only quality metrics で backend 品質を比較する。
- `FA-RESAMPLE-SPEC-010`: sample format conversion は `fa_sample_format` の責務であり、integration test は明示 pipeline を通す。
- `FA-RESAMPLE-SPEC-011`: metrics reporter は GTest XML の quality / backend metrics を出力し、incomplete metric group を失敗扱いする。
