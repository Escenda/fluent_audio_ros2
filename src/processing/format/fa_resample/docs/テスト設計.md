# fa_resample テスト設計

## 現在の自動テスト

### unit / core test

- 48kHz / 16kHz / 8kHz の入力を 16kHz frame 数へ変換する。
- `FLOAT32LE` / 32-bit / interleaved だけを受け付ける。
- PCM16LE / PCM32LE を reject し、node 内で sample format conversion しない。
- NaN / Inf / normalized range 外 sample を reject し、clamp しない。
- invalid bit depth / zero channel / empty data / unaligned data を drop する。

### static contract test

- default config が `input.FLOAT32LE/32/interleaved -> output.FLOAT32LE/32` を明示する。
- `fa_resample_node.cpp` に PCM16 / PCM32 decode、PCM16 encode、`std::clamp` が戻っていないことを確認する。
- colcon test が C++ gtest と pytest の両方を実行することを確認する。

## 未実装の runtime test

次は必要だが、現時点では package 内の自動テストとして未実装である。

### integration test

- `fa_in -> fa_resample -> fa_vad` の topic wiring を検証する。
- `fa_in -> fa_sample_format -> fa_resample -> fa_vad` の明示 format pipeline を検証する。
- mic/ref の enabled 切替で不要 topic を作らないこと。

### launch test

- `config/default.yaml` で起動できる。
- `target_sample_rate != 16000` で起動失敗する。
- enabled stream の topic が空なら起動失敗する。

### failure test

- data size が frame boundary に合わない。
- NaN / Inf sample。
- unsupported bit depth。
- unsupported encoding。
- range overflow sample。

### latency test

callback あたりの処理時間と diagnostics counter を測る。

### real device smoke test

実 microphone から 48kHz frame を入力し、16kHz output topic の sample rate / channel / duration が期待値と一致することを確認する。
