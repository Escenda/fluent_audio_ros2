# fa_resample アルゴリズム詳細説明書

## 責務分離

ROS node は parameter、QoS、mic/ref topic、stream identity、publish/drop、diagnostics、`AudioFrame` mapping を担当する。`FLOAT32LE` contract validation、sample decode/encode、finite normalized sample enforcement、linear resampling は `internal_linear_resampler` backend が担当する。backend は ROS2 topic/message を知らない。

## 処理手順

```text
AudioFrame
  ↓
node: source_id / stream_id validation
  ↓
backend: validate metadata and FLOAT32LE/32/interleaved contract
  ↓
backend: decode raw FLOAT32LE bytes to interleaved float32 samples
  ↓
backend: reject non-finite or out-of-range samples
  ↓
backend: linear interpolation resample
  ↓
backend: encode raw FLOAT32LE bytes
  ↓
node: publish AudioFrame
```

## resampling

出力 frame 数は次で決める。

```text
out_frames = round(in_frames * target_sample_rate / input_sample_rate)
```

各出力 sample は入力位置 `src_pos = out_index * input_sample_rate / target_sample_rate` を使い、隣接 2 sample の線形補間で求める。

```text
y[i] = (1 - frac) * x[floor(src_pos)] + frac * x[min(floor(src_pos)+1, last)]
```

## frame / buffer 前提

- 入力は `FLOAT32LE` / 32-bit / `interleaved` PCM。
- `stream_id` は node 側で対応する `mic.input_stream_id` / `ref.input_stream_id` と一致する。
- `data.size()` は `channels * sizeof(float)` の倍数。
- sample は finite かつ normalized range `[-1.0, 1.0]` に入っている。
- 出力 sample rate は `target_sample_rate` で明示する。
- 出力 `stream_id` は対応する `mic.output.stream_id` / `ref.output.stream_id` に更新する。

`fa_resample` は PCM16 / PCM32 を float32 に変換しない。整数 PCM を扱う場合は前段に `fa_sample_format` を明示して挟む。出力も `FLOAT32LE` / 32-bit 固定であり、PCM16 へ戻す処理や clipping 防止の clamp はここに置かない。

## known limitations

- 現在の resampler は線形補間であり、band-limited resampling ではない。
- channel 数は変えない。channel conversion は `processing/format/fa_channel_convert` の責務にする。
- sample format / bit depth conversion は `fa_sample_format` / `fa_bit_depth` の責務にする。
- per-frame rounding なので、長時間 streaming の sample-count drift は将来の stateful streaming resampler backend で扱う。
