# fa_format 仕様書

## 1. 目的

`fa_format` は format conversion pipeline を明示的に構成する wrapper / orchestration node である。
単一 backend で変換をまとめて隠すのではなく、`fa_decode`、`fa_sample_format`、`fa_bit_depth`、`fa_channel_convert`、`fa_interleave`、`fa_resample`、`fa_encode` などの専用段を設定順に接続する。

## 2. 入力

入力は upstream node からの audio stream とする。
`fa_format` 自体は device、file、network を開かない。

## 3. 出力

出力は configured pipeline の最後の stage が publish する stream とする。
各 stage は input/output topic、source identity、stream identity、format metadata を明示する。

## 4. Pipeline Contract

pipeline config は ordered stage list として定義する。
各 stage は次を必須にする。

- `stage.name`
- `stage.package`
- `stage.launch_file`
- `stage.params_file`
- `stage.input_topic`
- `stage.output_topic`
- expected input format
- expected output format

`fa_format` は stage 間の hidden negotiation をしない。
前段 output と後段 input が一致しない config は起動失敗にする。

## 5. 責務外

- device/source/sink adapter
- dynamics / frequency / correction / analysis / generation 処理
- format mismatch の自動補正
- missing stage の skip
- unsupported conversion の fallback

## 6. fail closed 条件

- stage list が空
- stage package / launch / params_file 欠落
- adjacent stage の output/input format mismatch
- topic identity が循環または重複して ambiguous
- unsupported stage type
