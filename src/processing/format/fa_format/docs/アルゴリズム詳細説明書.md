# fa_format アルゴリズム詳細説明書

1. startup config の ordered stage list を読み込む。
2. 各 stage の package、launch file、params file、input/output topic、input/output format contract を検証する。
3. 隣接 stage の output format と input format が一致することを検証する。
4. source/sink stage と processing stage が混在していないことを検証する。
5. 検証に通った stage だけを launch include / component composition の候補にする。
6. 欠損または不一致がある場合は起動失敗し、stage skip や自動補完はしない。

`fa_format` は audio samples を直接編集する DSP node ではない。
個々の変換は専用 package が担当し、`fa_format` は順序と契約整合性を管理する。

## Boundary

backend / composer は stage config validation と expansion のみを担当し、ROS2 message payload の変換はしない。
runtime audio frame の metadata validation は各 stage node の責務である。
