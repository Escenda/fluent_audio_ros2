# fa_format テスト設計

## Unit Test

- `FA-FORMAT-SPEC-001`: package 化前でも README、仕様書、アルゴリズム詳細説明書、テスト設計、backend docs が存在する。
- `FA-FORMAT-SPEC-002`: `package.xml` と `CMakeLists.txt` を package 化前に置かない。
- `FA-FORMAT-SPEC-003`: 仕様書が hidden format negotiation を禁止する。
- `FA-FORMAT-SPEC-004`: 仕様書が adjacent stage mismatch を fail closed として扱う。

## Integration Test

ROS2 package 化後に、valid stage list から include action 群を生成できること、stage mismatch で起動失敗することを確認する。

## Launch Test

ROS2 package 化後に、missing params file、empty stage list、unknown stage type で起動失敗することを確認する。
