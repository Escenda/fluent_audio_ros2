# fa_interleave アルゴリズム詳細説明書

## backend 境界

`internal_layout_reorder` backend は ROS2 topic/message を知らない。入力は byte列と `FrameContract`、出力は byte列と `ProcessResult` である。
node は `AudioFrame` から `FrameContract` を作り、backend の成功時だけ `AudioFrame.data` と layout metadata を更新して publish する。
backend が失敗した場合、node は frame を drop し、古い出力を publish しない。

## 入力表現

`AudioFrame.data` は raw little-endian sample byte sequence として扱う。`fa_interleave` は sample 値を数値として解釈せず、`bytes_per_sample` 単位で byte block を並べ替える。

対応する `bytes_per_sample` は次の通り。

| encoding | bit depth | bytes |
| --- | --- | --- |
| `PCM16LE` | 16 | 2 |
| `PCM32LE` | 32 | 4 |
| `FLOAT32LE` | 32 | 4 |

## Interleaved to Planar

interleaved は frame ごとに channel sample が並ぶ。

```text
in:  [c0f0, c1f0, c0f1, c1f1, c0f2, c1f2]
out: [c0f0, c0f1, c0f2, c1f0, c1f1, c1f2]
```

一般化すると、入力 offset は次の式で決まる。

```text
input_sample_index = frame_index * channels + channel_index
```

## Planar to Interleaved

planar は channel ごとに連続した frame sample が並ぶ。

```text
in:  [c0f0, c0f1, c0f2, c1f0, c1f1, c1f2]
out: [c0f0, c1f0, c0f1, c1f1, c0f2, c1f2]
```

一般化すると、入力 offset は次の式で決まる。

```text
input_sample_index = channel_index * frame_count + frame_index
```

## 不正 frame の扱い

`data.size()` が `channels * bytes_per_sample` の倍数でない場合、frame 境界または planar channel 境界が成立しないため、その runtime frame 全体を drop する。zero fill、truncate、padding、latest value reuse は行わない。
`FrameContract` は layout、encoding、bit depth、sample rate、channels、data size を含む。backend は config と runtime contract が一致しない場合、`FrameContractStatus` で理由を返す。

## 非対象処理

本 package は layout 以外の意味を変更しない。sample format conversion、bit-depth conversion、resampling、channel count conversion、gain、limiter、noise gate、filtering、denoise、device I/O は別 package の責務とする。
