# fa_interleave 仕様書

## 1. 目的

`fa_interleave` は `AudioFrame.data` の layout を `interleaved` と `planar` の間で明示的に変換する processing node である。
sample format、bit depth、channel count、sample rate、音量、filter、source/sink は責務外とする。

layout reorder は `internal_layout_reorder` backend が担当する。backend は ROS2 topic/message を知らず、node は parameter、QoS、runtime `AudioFrame` metadata 検証、publish/drop、diagnostics を担当する。

## 2. ノード

| 項目 | 値 |
| --- | --- |
| package | `fa_interleave` |
| executable | `fa_interleave_node` |
| node name | `fa_interleave` |
| subscribe | `input_topic` |
| publish | `output_topic` |
| diagnostics | `diagnostics` |

## 3. パラメータ

| パラメータ | 意味 |
| --- | --- |
| `input_topic` | 入力 `AudioFrame` topic |
| `output_topic` | 出力 `AudioFrame` topic |
| `input_stream_id` | 受理する入力 `AudioFrame.stream_id` |
| `input.layout` | 入力 layout。`interleaved` または `planar` |
| `output.stream_id` | 出力 `AudioFrame.stream_id` |
| `output.layout` | 出力 layout。`interleaved` または `planar` |
| `expected.sample_rate` | 期待する sample rate |
| `expected.channels` | 期待する channel 数。`> 0` |
| `expected.encoding` | `FLOAT32LE`、`PCM16LE`、`PCM32LE` のいずれか |
| `expected.bit_depth` | encoding と一致する bit depth |
| `qos.depth` | audio stream QoS depth |
| `qos.reliable` | true なら reliable、false なら best effort |
| `diagnostics.publish_period_ms` | diagnostics publish interval |
| `diagnostics.qos.depth` | diagnostics QoS depth |
| `diagnostics.qos.reliable` | diagnostics QoS reliability |

## 4. 起動時検証

以下を満たさない設定は起動失敗とする。

- `input_topic` と `output_topic` が空でない
- `input_stream_id` と `output.stream_id` が空でない
- `input_stream_id` / `output.stream_id` が raw / resolved ROS topic と衝突しない
- `input_stream_id` と `output.stream_id` が同一でない
- `input.layout` と `output.layout` が `interleaved` または `planar`
- `input.layout != output.layout`
- `expected.sample_rate > 0`
- `expected.channels > 0`
- `expected.encoding == FLOAT32LE` かつ `expected.bit_depth == 32`
- `expected.encoding == PCM16LE` かつ `expected.bit_depth == 16`
- `expected.encoding == PCM32LE` かつ `expected.bit_depth == 32`
- `qos.depth > 0`
- `diagnostics.publish_period_ms > 0`
- `diagnostics.qos.depth > 0`
- `diagnostics.qos.reliable` は bool

## 5. Runtime frame 検証

以下を満たさない frame は warning を出して drop する。

- `source_id` と `stream_id` が空でない
- `stream_id == input_stream_id`
- `layout == input.layout`
- `encoding == expected.encoding`
- `bit_depth == expected.bit_depth`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `data` が空でない
- `data.size()` が `expected.channels * bytes_per_sample` で割り切れる

backend は `FrameContractStatus` で不一致理由を返し、node は warning を出して drop する。

## 6. Metadata 更新

出力 frame は入力 frame を基準にし、以下だけを更新する。

- `stream_id = output.stream_id`
- `layout = output.layout`
- `data = reordered bytes`

`header`、`source_id`、`sample_rate`、`channels`、`encoding`、`bit_depth`、`epoch` は保持する。
backend が失敗した場合は出力 buffer を更新せず、古い変換結果を publish しない。

## 7. 非目標

- device を開くこと
- file / network / codec を扱うこと
- resample
- sample format conversion
- bit-depth conversion
- channel count conversion
- gain / loudness normalize / limiter
- filter / noise reduction
- frame metadata による hidden auto-detection

## 8. 仕様 ID

| SPEC ID | 内容 |
| --- | --- |
| `FA-INTERLEAVE-SPEC-001` | `fa_interleave` は layout reorder のみを行い、device I/O、file/network/codec、resample、sample format conversion、bit-depth conversion、channel count conversion、gain、loudness normalize、limiter、filter、noise reduction を持たない。 |
| `FA-INTERLEAVE-SPEC-002` | default config は 16000Hz、2ch、`FLOAT32LE/32`、`interleaved -> planar`、topic/stream identity 分離を明示する。 |
| `FA-INTERLEAVE-SPEC-003` | 起動時 config は topic、stream identity、input/output layout、expected sample rate / channels / encoding / bit depth、QoS、diagnostics を fail closed で検証する。 |
| `FA-INTERLEAVE-SPEC-004` | 対応 layout conversion は `interleaved -> planar` と `planar -> interleaved` の明示組み合わせだけで、hidden auto-detection はしない。 |
| `FA-INTERLEAVE-SPEC-005` | runtime frame は source/stream identity、layout、encoding、bit depth、sample rate、channels、byte count を処理前に検証する。 |
| `FA-INTERLEAVE-SPEC-006` | 出力 frame は `stream_id`、layout、data だけを更新し、header、source_id、sample_rate、channels、encoding、bit_depth、epoch を保持する。 |
| `FA-INTERLEAVE-SPEC-007` | backend は sample byte width に基づく byte reorder を行い、numeric sample conversion を行わない。 |
| `FA-INTERLEAVE-SPEC-008` | backend failure では output buffer を更新せず、node は失敗 frame を publish しない。 |
| `FA-INTERLEAVE-SPEC-009` | backend は ROS2 topic/message を知らず、typed `FrameContractStatus` で失敗理由を返す。 |
| `FA-INTERLEAVE-SPEC-010` | executable tests は supported layout conversion、unsupported capability rejection、malformed frame rejection、failure 時の no output mutation を検証する。 |
