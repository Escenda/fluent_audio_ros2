# fa_decode テスト設計

## Unit Test

- `FA-DECODE-SPEC-001`: config は codec/container、expected output encoding / bit depth / sample rate / channels / layout、input/output stream identity を明示する。
- `FA-DECODE-SPEC-002`: node validation は unsupported codec/container/backend protocol と unsupported output AudioFrame capability を fail closed で拒否する。
- `FA-DECODE-SPEC-003`: 仕様書が `fa_in` に decode を隠さない責務分離を明記する。
- `FA-DECODE-SPEC-004`: 仕様書が missing codec / backend / metadata を fail closed として扱う。
- `FA-DECODE-SPEC-005`: external worker / fake backend protocol は encoded chunk を受け、declared output contract と一致する PCM だけを成功として返す。
- `FA-DECODE-SPEC-006`: external command timeout、empty output、oversized output を error として扱う。
- `FA-DECODE-SPEC-007`: `input_stream_id` / `output.stream_id` は必須で、ROS topic identity と同値にしない。

## Integration Test

fixture encoded chunk を explicit external command backend に渡し、`stream_id == input_stream_id` かつ declared output contract 一致時だけ `stream_id = output.stream_id` の `AudioFrame` が publish されることを確認する。

## Launch Test

missing params file、missing backend、missing codec、missing command executable で起動失敗することを確認する。

## 仕様とテストケース対応

### 実装済み

- `FA-DECODE-TC-001` -> `FA-DECODE-SPEC-001`, `FA-DECODE-SPEC-002`: config validation rejects missing codec/container/output capability and unsupported output `AudioFrame` metadata.
- `FA-DECODE-TC-002` -> `FA-DECODE-SPEC-003`, `FA-DECODE-SPEC-004`: launch fail-closed tests reject missing backend, missing codec, missing command, and missing metadata.
- `FA-DECODE-TC-003` -> `FA-DECODE-SPEC-005`: fake backend protocol test verifies declared PCM output contract and explicit failure result.
- `FA-DECODE-TC-004` -> `FA-DECODE-SPEC-006`: backend API test verifies timeout, empty output, oversized output, and output metadata mismatch are errors.
- `FA-DECODE-TC-005` -> `FA-DECODE-SPEC-007`: ROS graph / file pipeline tests publish decoded `AudioFrame` only when input stream and output contract match.
- `FA-DECODE-TC-006` -> `FA-DECODE-SPEC-004`: launch fail-closed test covers invalid params file and backend startup failure.
