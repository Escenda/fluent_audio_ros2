# fa_decode テスト設計

## Unit Test

- `FA-DECODE-SPEC-001`: README、仕様書、アルゴリズム詳細説明書、テスト設計、backend docs が存在する。
- `FA-DECODE-SPEC-002`: package 化後に `package.xml` と `CMakeLists.txt` が存在する。
- `FA-DECODE-SPEC-003`: 仕様書が `fa_in` に decode を隠さない責務分離を明記する。
- `FA-DECODE-SPEC-004`: 仕様書が missing codec / backend / metadata を fail closed として扱う。
- `FA-DECODE-SPEC-005`: backend が ROS2 / `fa_interfaces` を include しない。
- `FA-DECODE-SPEC-006`: external command timeout、empty output、oversized output を error として扱う。
- `FA-DECODE-SPEC-007`: `input_stream_id` / `output.stream_id` は必須で、ROS topic identity と同値にしない。

## Integration Test

fixture encoded chunk を explicit external command backend に渡し、`stream_id == input_stream_id` かつ declared output contract 一致時だけ `stream_id = output.stream_id` の `AudioFrame` が publish されることを確認する。

## Launch Test
