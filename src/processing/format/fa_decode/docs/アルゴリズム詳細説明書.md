# fa_decode アルゴリズム詳細説明書

1. startup config で `backend.name`、`input_stream_id`、`output.stream_id`、input codec/container/payload format、input sample rate/channels、output encoding、output layout を検証する。
2. `input_stream_id` / `output.stream_id` が raw / resolved ROS topic identity と同値ではなく、相互にも異なることを検証する。
3. encoded chunk の source identity、timestamp、`stream_id == input_stream_id`、input contract を検証する。
4. backend に encoded payload と decode contract を渡す。
5. backend は external command の stdout bytes を decoded PCM samples として返し、startup config の output contract を publish metadata として返す。
6. node は decoded samples が非空で、output contract が config と一致することを検証する。
7. 一致した場合だけ `stream_id = output.stream_id` の `AudioFrame` を publish する。

decode は audio content を PCM 表現へ展開する処理であり、音量補正やノイズ除去ではない。
decoder backend が native sample rate や channel count を返した場合でも、node はそれを後段向けに自動変換しない。
必要な変換は `fa_resample`、`fa_channel_convert`、`fa_sample_format`、`fa_bit_depth` に明示的に接続する。

## Backend Boundary

backend は codec/container decode のみを担当し、ROS2 topic/message を知らない。
backend は入力欠損、unsupported codec、decode failure を明示的な error として返す。
decode 失敗時に silence、zero-filled samples、前回 frame の再利用はしない。
初期実装の `external_codec_decoder` は external command の stdin に encoded payload を渡し、stdout を decoded PCM payload として受け取る。stdout は byte payload のみであり、sample rate、channel count、encoding、bit depth、layout を stdout から推定しない。runtime metadata が必要な decoder では、stdout とは別の明示 protocol を追加し、推測や probe で補完しない。
