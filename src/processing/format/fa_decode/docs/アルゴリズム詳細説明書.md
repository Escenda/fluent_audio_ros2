# fa_decode アルゴリズム詳細説明書

1. startup config で `backend.name`、codec、expected sample rate、expected channels、output encoding、output layout を検証する。
2. encoded chunk の source identity と timestamp を検証する。
3. backend に encoded payload と decode contract を渡す。
4. backend は decoded samples と actual format metadata を返す。
5. node は actual metadata が config と一致することを検証する。
6. 一致した場合だけ `AudioFrame` を publish する。

decode は audio content を PCM 表現へ展開する処理であり、音量補正やノイズ除去ではない。
decoder backend が native sample rate や channel count を返した場合でも、node はそれを後段向けに自動変換しない。
必要な変換は `fa_resample`、`fa_channel_convert`、`fa_sample_format`、`fa_bit_depth` に明示的に接続する。

## Backend Boundary

backend は codec/container decode のみを担当し、ROS2 topic/message を知らない。
backend は入力欠損、unsupported codec、decode failure を明示的な error として返す。
decode 失敗時に silence、zero-filled samples、前回 frame の再利用はしない。
