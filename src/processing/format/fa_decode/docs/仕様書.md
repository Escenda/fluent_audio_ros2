# fa_decode 仕様書

## 1. 目的

`fa_decode` は encoded audio payload を PCM `fa_interfaces/msg/AudioFrame` へ変換する format processing node である。
`fa_in` が入力 source を開き、`fa_decode` が codec decode を担当することで、source adapter と format conversion の責務を分離する。

## 2. 入力

入力は `fa_interfaces/msg/EncodedAudioChunk` とする。
入力 contract は codec、container、payload format、sample rate、channel count、duration、timestamp、source identity を明示する必要がある。

## 3. 出力

出力は `AudioFrame` とし、次の metadata を必須にする。

- `source_id`
- `stream_id`
- `sample_rate`
- `channels`
- `encoding`
- `bit_depth`
- `layout`
- `epoch`

decode 後の sample representation は backend config で明示し、hidden resample、hidden gain、hidden channel conversion は行わない。

## 4. 責務外

- 入力 device を開くこと
- system audio / application audio capture
- resample
- bit-depth conversion beyond decoder output contract
- channel count conversion
- gain / normalize / limiter
- VAD / ASR / embedding
- encoded payload が欠損した場合の推測補完

## 5. fail closed 条件

- codec 未指定
- backend 未指定
- backend command / model / library contract 不正
- sample rate または channel count 未指定
- decode result が空
- decoded output contract が config と一致しない
- timestamp / source identity が欠損
