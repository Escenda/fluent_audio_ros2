# fa_decode 仕様書

## 1. 目的

`fa_decode` は encoded audio payload を PCM `fa_interfaces/msg/AudioFrame` へ変換する format processing node である。
`fa_in` が入力 source を開き、`fa_decode` が codec decode を担当することで、source adapter と format conversion の責務を分離する。

## 2. 入力

入力は `fa_interfaces/msg/EncodedAudioChunk` とする。
入力 contract は codec、container、payload format、sample rate、channel count、duration、timestamp、source identity を明示する必要がある。
入力 `stream_id` は `input_stream_id` と一致する必要がある。
`input_stream_id` と `output.stream_id` は ROS topic 名とは別の audio identity とし、
raw / resolved `input_topic` / `output_topic` と同値であってはならない。

## 3. 出力

出力は `AudioFrame` とし、次の metadata を必須にする。

- `source_id`
- `stream_id`
- `sample_rate`
- `channels`
- `encoding`
- `bit_depth`
- `layout`
- `epoch`

decode 後の sample representation は backend config で明示し、hidden resample、hidden gain、hidden channel conversion は行わない。
出力 `stream_id` は `output.stream_id` に設定する。

## 3.1 Capability contract

supported codec / container と、出力 `encoding` / `bit_depth` / `sample_rate` / `channels` / `layout` は config で明示する。
external worker / fake backend protocol は decoded PCM と実際の出力 metadata を返す。metadata が config と一致しない場合は fail closed とし、node 側で resample、bit depth 変換、channel conversion、layout conversion を補完しない。
unsupported codec、unsupported backend protocol、または unsupported output capability は publish せず error とする。

## 4. 責務外

- 入力 device を開くこと
- system audio / application audio capture
- resample
- bit-depth conversion beyond decoder output contract
- channel count conversion
- gain / normalize / limiter
- VAD / ASR / embedding
- encoded payload が欠損した場合の推測補完

## 5. fail closed 条件

- codec 未指定
- backend 未指定
- backend command / model / library contract 不正
- sample rate または channel count 未指定
- `input_stream_id` / `output.stream_id` 未指定、topic 名との同値、または相互同値
- decode result が空
- decoded output contract が config と一致しない
- timestamp / source identity が欠損
