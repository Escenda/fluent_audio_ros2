# fa_ducking アルゴリズム詳細説明書

## 1. 入力

`fa_ducking` は program stream と sidechain stream の 2 系統を受け取る。両方とも `FLOAT32LE`、32 bit、interleaved、正規化済み `[-1.0, 1.0]` sample を前提とする。

node は ROS parameter、topic、`AudioFrame` metadata、diagnostics、出力 `stream_id` 更新だけを扱う。sample loop、RMS、sidechain state、gain smoothing は ROS-free backend が担当する。

## 2. Sidechain RMS

sidechain frame の byte列を `float` sample として読み出し、以下で frame RMS を計算する。

```text
rms = sqrt(sum(sample^2) / sample_count)
```

RMS が finite ではない、または `[0.0, 1.0]` を外れる場合、その sidechain frame は不正として drop し、recent sidechain state を無効化する。

## 3. Sidechain active 判定

program frame の処理時刻 `now` と recent sidechain 受信時刻 `received_at` から age を計算する。

```text
age_ms = (now - received_at) / 1_000_000
```

active sidechain は以下をすべて満たす状態である。

- valid sidechain state が存在する
- `0 <= age_ms <= sidechain.max_age_ms`
- `sidechain_rms >= sidechain.threshold_rms`

条件を満たさない場合、sidechain は inactive とする。inactive は ducking target を `1.0` に戻す状態であり、不正 sidechain frame を再利用する fallback ではない。

## 4. Gain target

`ducking.gain_db` は以下で linear gain に変換する。

```text
duck_gain = pow(10, ducking.gain_db / 20)
```

active sidechain の target gain は `duck_gain`、inactive sidechain の target gain は `1.0` である。`ducking.gain_db` は `[-96.0, 0.0)` の範囲だけを許可するため、active target は必ず attenuation になる。

## 5. Smoothing

frame duration は program frame の sample 数、channel 数、sample rate から計算する。

```text
frame_seconds = (sample_count / channels) / sample_rate
alpha = 1 - exp(-frame_seconds / time_constant_seconds)
next_gain = current_gain + alpha * (target_gain - current_gain)
```

target gain が current gain より小さい場合は attack、target gain が current gain 以上の場合は release を使う。`next_gain` は `[duck_gain, 1.0]` に収める。

## 6. Program sample 変換

program sample ごとに smoothed gain を乗算する。

```text
output_sample = program_sample * next_gain
```

出力 sample が finite ではない、または `[-1.0, 1.0]` を外れる場合、その program frame は drop する。clamp、normalize、limiter は行わない。成功した frame だけ `current_gain` と diagnostics counter を更新する。

## 7. Metadata

出力 frame は program 入力 frame をコピーしてから `stream_id` と `data` のみを更新する。`stream_id` は `output.stream_id` に更新する。sidechain の metadata は output へ引き継がない。
