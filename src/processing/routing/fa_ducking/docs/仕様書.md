# fa_ducking 仕様書

## 1. 目的

`fa_ducking` は、program `AudioFrame` stream に対し、sidechain `AudioFrame` stream の RMS を制御信号とする ducking を行う。

対象は routing/dynamics boundary の sidechain ducking であり、sidechain の混合、resampling、format conversion、limiter、compressor、normalize、output device playback は責務外である。

## 2. Package / Executable / Node

| 項目 | 値 |
| --- | --- |
| package | `fa_ducking` |
| executable | `fa_ducking_node` |
| node default name | `fa_ducking_node` |

## 3. 入出力

| 種別 | パラメータ | Message |
| --- | --- | --- |
| subscribe | `program_topic` | `fa_interfaces/msg/AudioFrame` |
| subscribe | `sidechain_topic` | `fa_interfaces/msg/AudioFrame` |
| publish | `output_topic` | `fa_interfaces/msg/AudioFrame` |

sidechain frame は RMS state の更新にのみ使う。sidechain sample を output data へ加算しない。

## 4. AudioFrame 契約

program / sidechain の入力 frame は以下を満たす必要がある。

| Field | 条件 |
| --- | --- |
| `source_id` | 非空 |
| `stream_id` | 非空、かつ program は `program_topic`、sidechain は `sidechain_topic` と一致 |
| `sample_rate` | `expected.sample_rate` と一致、かつ `> 0` |
| `channels` | `expected.channels` と一致、かつ `> 0` |
| `encoding` | `FLOAT32LE` |
| `bit_depth` | `32` |
| `layout` | `interleaved` |
| `data` | 非空、かつ `channels * sizeof(float)` の倍数 |

入力 sample は finite かつ `[-1.0, 1.0]` の正規化範囲内でなければならない。出力 sample が finite でない、または正規化範囲を超える場合、その program frame は drop する。clamp や limiter による補正は行わない。

## 5. Parameter

| Parameter | Default | 条件 |
| --- | ---: | --- |
| `program_topic` | `audio/program/frame` | 非空 |
| `sidechain_topic` | `audio/sidechain/frame` | 非空 |
| `output_topic` | `audio/ducked/frame` | 非空 |
| `sidechain.threshold_rms` | `0.05` | finite、`> 0.0`、`<= 1.0` |
| `sidechain.max_age_ms` | `100` | `> 0` |
| `ducking.gain_db` | `-12.0` | finite、`>= -96.0`、`< 0.0` |
| `ducking.attack_ms` | `10.0` | finite、`> 0.0` |
| `ducking.release_ms` | `250.0` | finite、`> 0.0` |
| `expected.sample_rate` | `16000` | `> 0` |
| `expected.channels` | `1` | `> 0` |
| `expected.encoding` | `FLOAT32LE` | 固定 |
| `expected.bit_depth` | `32` | 固定 |
| `expected.layout` | `interleaved` | 固定 |
| `qos.depth` | `10` | `> 0` |
| `qos.reliable` | `false` | bool |
| `diagnostics.publish_period_ms` | `1000` | `> 0` |

## 6. Sidechain state

valid sidechain frame を受信すると、frame RMS と受信時刻を recent sidechain state として保持する。program frame 処理時に、受信時刻からの経過が `sidechain.max_age_ms` 以内で、かつ RMS が `sidechain.threshold_rms` 以上の場合だけ active sidechain とする。

sidechain が未受信、stale、または threshold 未満の場合、target gain は `1.0` とする。これは sidechain inactive state の明示仕様であり、壊れた frame を正常扱いする fallback ではない。sidechain frame が契約違反または不正 sample を含む場合、その frame を drop し、recent sidechain state を無効化する。

## 7. Ducking state

起動直後の gain は `1.0` とする。active sidechain では `ducking.gain_db` を linear gain に変換した値を target gain とする。inactive sidechain では `1.0` を target gain とする。

target gain が current gain より小さい場合は `ducking.attack_ms`、大きい場合は `ducking.release_ms` から frame duration に基づく smoothing coefficient を計算する。program frame へは smoothed current gain を乗算する。

## 10. 失敗時動作

- 起動時設定不正: 例外により node 起動を失敗させる。
- program frame 契約違反: warning を出して frame を drop する。
- sidechain frame 契約違反: warning を出して frame を drop し、recent sidechain state を無効化する。
- 入力 sample 不正: warning を出して対象 frame を drop する。
- 出力 sample 範囲超過: warning を出して program frame を drop する。この場合、candidate gain は current gain として確定しない。

意味を変える fallback、legacy alias、暗黙の補正は行わない。
