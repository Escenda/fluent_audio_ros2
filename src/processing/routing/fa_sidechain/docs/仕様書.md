# fa_sidechain 仕様書

## 1. 目的

`fa_sidechain` は、sidechain 音声 stream から明示的な gain-control stream を生成する routing / analysis bridge node である。

program audio の購読、変更、mix、ducking は行わない。下流 node が control stream を購読し、自身の責務として program audio へ反映する。

## 2. 非目標

- program audio sample の変更
- resample
- normalize
- clamp
- compressor / limiter / ducking の実行
- 欠損 frame の補完
- 古い stream 名や旧 key の互換

## 3. ROS2 Interface

| direction | topic parameter | message |
| --- | --- | --- |
| subscribe | `sidechain_topic` | `fa_interfaces/msg/AudioFrame` |
| publish | `control_topic` | `fa_interfaces/msg/AudioFrame` |
| publish | `diagnostics` | `diagnostic_msgs/msg/DiagnosticArray` |

## 4. Parameters

| name | default | requirement |
| --- | --- | --- |
| `sidechain_topic` | `audio/sidechain/frame` | 空文字不可 |
| `control_topic` | `audio/sidechain/control` | 空文字不可、`sidechain_topic` と別 topic |
| `detector.threshold_rms` | `0.05` | finite、`(0.0, 1.0]` |
| `detector.active_gain_db` | `-12.0` | finite、linear gain が `[0.0, 4.0]` |
| `detector.inactive_gain_db` | `0.0` | finite、linear gain が `[0.0, 4.0]` |
| `control.sample_rate` | `1000` | `> 0` |
| `expected.sample_rate` | `16000` | `> 0` |
| `expected.channels` | `1` | `> 0` |
| `expected.encoding` | `FLOAT32LE` | `FLOAT32LE` のみ |
| `expected.bit_depth` | `32` | `32` のみ |
| `expected.layout` | `interleaved` | `interleaved` のみ |
| `qos.depth` | `10` | `> 0` |
| `qos.reliable` | `false` | boolean |
| `diagnostics.publish_period_ms` | `1000` | `> 0` |

startup config が不正な場合は例外で起動を停止する。

## 5. Input Validation

sidechain input frame は以下をすべて満たす必要がある。

- `source_id` が空でない
- `stream_id == sidechain_topic`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == expected.encoding`
- `bit_depth == expected.bit_depth`
- `layout == expected.layout`
- `data` が空でない
- `data.size()` が `expected.channels * sizeof(float)` で割り切れる
- 全 sample が finite
- 全 sample が `[-1.0, 1.0]`

不正 frame は publish せず、`frames_dropped` を増やす。値の補完や丸めは行わない。

## 6. Output

出力 `AudioFrame` は次の通り。

- `header = input.header`
- `source_id = input.source_id`
- `stream_id = control_topic`
- `sample_rate = control.sample_rate`
- `channels = 1`
- `encoding = FLOAT32LE`
- `bit_depth = 32`
- `layout = interleaved`
- `epoch = input.epoch`
- `data = target_gain_linear` の float32 1 sample

`target_gain_linear` は finite かつ `[0.0, 4.0]` でなければならない。

## 7. Diagnostics

`diagnostics` には最低限以下を含める。

- `frames_in`
- `frames_out`
- `frames_dropped`
- `last_rms`
- `last_gain_linear`
- `active_frames`
- `inactive_frames`

設定値も diagnostics に含め、実行中の control 契約を確認できるようにする。
