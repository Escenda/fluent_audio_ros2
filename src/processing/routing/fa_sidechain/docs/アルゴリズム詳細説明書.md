# fa_sidechain アルゴリズム詳細説明書

## 1. Data Flow

```text
sidechain AudioFrame
  -> node metadata validation
  -> backend FLOAT32LE sample decode
  -> backend RMS calculation
  -> backend threshold decision
  -> backend dB to linear gain
  -> backend mono FLOAT32LE control payload
  -> node AudioFrame wrapping
```

この flow は control signal generation であり、program audio processing ではない。

## 2. RMS

入力 sample 列を `x[i]`、sample 数を `N` とする。

```text
rms = sqrt(sum(x[i] * x[i]) / N)
```

前提:

- `N > 0`
- 全 sample は finite
- 全 sample は `[-1.0, 1.0]`

この前提は RMS 計算前の validation で保証する。

## 3. Gain Decision

```text
if rms >= detector.threshold_rms:
  target_gain_linear = pow(10, detector.active_gain_db / 20)
else:
  target_gain_linear = pow(10, detector.inactive_gain_db / 20)
```

`rms == threshold` は active とする。

## 4. Output Encoding

control frame の `data` は backend が `target_gain_linear` を IEEE754 float32 little-endian として 1 sample だけ格納した payload を使う。

出力 frame は音声波形ではなく control stream なので、`channels=1`、`sample_rate=control.sample_rate` とする。

## 5. Error Handling

startup config の不正は fail closed として node 起動を停止する。

runtime frame の不正は frame drop とし、warning と diagnostics counter で検出できるようにする。resample、normalize、clamp、欠損値補完は行わない。

backend が不正 sample や表現不能 gain を返した場合、node は publish せず `frames_dropped` を増やす。backend は成功時だけ last state と control payload を更新する。
