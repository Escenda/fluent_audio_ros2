# fa_sidechain テスト設計

## 1. Scope

この package のテストは、package layout、設定契約、backend 境界、C++ 実装上の fail closed / frame drop 契約、RMS と gain 変換の数値仕様を確認する。

## 2. Unit-Style Tests

`test/unit/test_fa_sidechain_audio_frame_contract.py`

- default config が topic と stream ID を分離していること
- package launch の `node_name` / `config_file` に runtime default がないこと
- node parameter がすべて required であり、topic / stream ID distinctness を検証すること
- node が metadata validation の後で backend を呼ぶこと
- backend が ROS-free で、sample decode / RMS / gain bytes を所有すること
- backend が不正 sample を drop し、成功時だけ state / output を commit すること
- diagnostics に backend state、counter、diagnostics QoS が含まれること
- package layout と colcon pytest / backend gtest contract があること

## 3. C++ Backend Tests

`test/cpp/test_internal_sidechain_detector_backend.cpp`

- threshold 以上の RMS で active gain を出すこと
- threshold 未満で inactive gain を出すこと
- invalid input で state / output を更新しないこと
- invalid config が例外で fail closed すること
- unknown status が `logic_error` になること
