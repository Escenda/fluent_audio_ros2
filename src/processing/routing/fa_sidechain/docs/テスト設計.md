# fa_sidechain テスト設計

## 1. Scope

この package のテストは、package layout、設定契約、C++ 実装上の fail closed / frame drop 契約、RMS と gain 変換の数値仕様を確認する。

## 2. Unit-Style Tests

`test/unit/test_fa_sidechain_audio_frame_contract.py`

- default config が required topic と FLOAT32LE interleaved 契約を持つこと
- startup validation が不正 config を例外で止めること
- runtime validation が不正 frame を drop する分岐を持つこと
- sample decode が non-finite と範囲外 sample を拒否すること
- RMS と threshold decision が仕様通りであること
- output frame が mono FLOAT32LE 1 sample の control stream になること
- diagnostics に required counters が含まれること
- package layout と colcon pytest contract があること

## 3. Integration Tests

現時点では `test/integration/.gitkeep` のみを置く。親 agent 側で ROS2 runtime を含む統合起動確認を追加する想定。

## 4. Launch Tests

現時点では `test/launch/.gitkeep` のみを置く。launch testing pattern が共有された時点で、default launch が node と config file を解決できることを確認する。

## 5. Fixtures

現時点では `test/fixtures/.gitkeep` のみを置く。binary `AudioFrame.data` fixture が必要になった場合に追加する。
