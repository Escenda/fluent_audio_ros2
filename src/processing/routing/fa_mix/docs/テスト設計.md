# fa_mix テスト設計

## traceability

| test ID | 対応仕様 | 観点 |
| --- | --- | --- |
| `FA-MIX-TC-001` -> `FA-MIX-SPEC-001` | mixing boundary | 複数入力を 1 output stream に混ぜる明示 mixing point であること。 |
| `FA-MIX-TC-002` -> `FA-MIX-SPEC-002` | topic / stream identity | topic と stream ID を分離し、衝突を fail closed すること。 |
| `FA-MIX-TC-003` -> `FA-MIX-SPEC-003` | input set config | input 数、gain 数、master index を検証し、空 gain を implicit 0 dB にしないこと。 |
| `FA-MIX-TC-004` -> `FA-MIX-SPEC-004` | required format config | expected PCM16LE format、QoS、diagnostics parameter を required とし、不正値で起動失敗すること。 |
| `FA-MIX-TC-005` -> `FA-MIX-SPEC-005` | complete frame set | master trigger 時に全入力の valid latest frame が揃う場合だけ publish すること。 |
| `FA-MIX-TC-006` -> `FA-MIX-SPEC-006` | no partial mix | zero timestamp、stale frame、missing input、length mismatch を partial mix にしないこと。 |
| `FA-MIX-TC-007` -> `FA-MIX-SPEC-007` | backend boundary | PCM16LE decode / gain / summing / encode を ROS-free backend に閉じること。 |
| `FA-MIX-TC-008` -> `FA-MIX-SPEC-008` | no hidden limiter | overflow を clamp / limiter で丸めず error とすること。 |
| `FA-MIX-TC-009` -> `FA-MIX-SPEC-009` | output metadata | output metadata が master frame と output stream config から作られること。 |
| `FA-MIX-TC-010` -> `FA-MIX-SPEC-010` | non-goals | resampling、channel conversion、limiter、ducking、device playback を持たないこと。 |
| `FA-MIX-TC-011` -> `FA-MIX-SPEC-011` | launch contract | launch file が `node_name` / `config_file` だけを要求し、package default を推測しないこと。 |
| `FA-MIX-TC-012` -> `FA-MIX-SPEC-012` | colcon coverage | pytest と backend gTest が `colcon test` から実行されること。 |

## unit test

- 1 入力 passthrough 相当の mix。
- 複数入力の dB gain 適用。
- master index validation。
- topic と stream ID が分離されていること。
- package launch の `node_name` / `config_file` に runtime default がないこと。
- node parameter がすべて required であり、topic / stream ID distinctness を検証すること。
- backend が ROS-free で PCM16LE decode / mix / encode を所有すること。
- configured input が欠ける場合は partial mix を publish せず mixed frame 全体を drop する。
- master frame の `header.stamp` から見て stale frame がある場合は partial mix を publish せず mixed frame 全体を drop する。
- zero timestamp frame は valid latest frame として使わない。
- decoded sample count が master と異なる場合は shorter buffer に切り詰めず mixed frame 全体を drop する。
- `input_gains_db` が空の場合は implicit 0 dB fallback にせず起動失敗する。
- mixed sample が `[-1.0, 1.0]` を超える場合は clamp せず drop する。

## C++ backend gtest

- matching PCM16 input を mix すること。
- sample count mismatch で output を更新しないこと。
- overflow を clamp せず error status にすること。
- invalid config が例外で fail closed すること。
- unknown status が `logic_error` になること。

## integration test

- `fa_tts -> fa_mix -> fa_out` の topic wiring。
- 複数 TTS / monitor source の mix。

## launch test

- 明示された `node_name` と `config_file` で起動できる。
- `input_gains_db` の空配列または長さ不一致で起動失敗する。
- unsupported bit depth で起動失敗する。

## failure test

- format mismatch。
- empty data。
- frame length mismatch。

## latency test

入力数ごとの mix latency を測る。

## real device smoke test

TTS と別 source を mix し、`fa_out` から output されることを確認する。
