# fa_mix テスト設計

## unit test

- 1 入力 passthrough 相当の mix。
- 複数入力の dB gain 適用。
- master index validation。
- stale frame exclusion。

## integration test

- `fa_tts -> fa_mix -> fa_out` の topic wiring。
- 複数 TTS / monitor source の mix。

## launch test

- `config/default.yaml` で起動できる。
- `input_gains_db` の長さ不一致で起動失敗する。
- unsupported bit depth で起動失敗する。

## failure test

- format mismatch。
- empty data。
- frame length mismatch。

## latency test

入力数ごとの mix latency を測る。

## real device smoke test

TTS と別 source を mix し、`fa_out` から output されることを確認する。
