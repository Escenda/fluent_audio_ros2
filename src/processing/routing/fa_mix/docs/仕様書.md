# fa_mix 仕様書

## 目的

`fa_mix` は複数の `AudioFrame` stream を 1 つの output stream へ混ぜる routing node である。TTS、monitor、loopback など複数経路を扱うための明示的な mixing point として使う。

## 入力 topic

| parameter | example | 内容 |
| --- | --- | --- |
| `input_topics` | `["fa_mix/tts"]` | mix 対象 ROS topic 群 |
| `input_stream_ids` | `["audio/tts/frame"]` | 入力 `AudioFrame.stream_id` 群 |

## 出力 topic

| parameter | example | 内容 |
| --- | --- | --- |
| `output_topic` | `fa_mix/output` | mixed audio frame を publish する ROS topic |
| `output.stream_id` | `audio/mix/output` | 出力 `AudioFrame.stream_id` |

topic 名と `AudioFrame.stream_id` は別契約である。`input_topics` / `output_topic` を stream ID として使ってはならない。

## parameters

| parameter | 意味 |
| --- | --- |
| `input_topics` | 入力 topic 一覧 |
| `input_stream_ids` | 入力 stream ID 一覧。`input_topics` と同じ長さで、topic 名と一致不可 |
| `input_gains_db` | 入力ごとの gain。1 要素なら全入力共通。空配列による implicit 0 dB fallback は禁止 |
| `master_index` | publish trigger とする入力 index |
| `output.stream_id` | 出力 frame の stream ID。topic 名および入力 stream ID と一致不可 |
| `expected.*` | 入力 format contract |
| `max_frame_age_ms` | master 以外の latest frame 有効期限 |
| `qos.*` | audio frame QoS |
| `diagnostics.qos.*` | diagnostics QoS |
| `diagnostics.publish_period_ms` | diagnostics publish 周期 |

## lifecycle

起動時に input/output topic、input/output stream ID、gain 数、gain の finite 性、master index、expected PCM16LE interleaved format、QoS を検証する。不正 config は起動失敗する。

すべて required であり、コード上に有効な runtime default はない。package launch の `node_name` と `config_file` も明示必須である。

## QoS

input/output は同一 QoS で扱う。diagnostics は `diagnostics.qos.*` の明示 QoS で publish する。

## error handling

format mismatch、missing input、stale frame、decode failure、frame length mismatch、mixed sample の PCM16 range overflow は drop counter に反映し、別 format へ推測変換しない。overflow を limiter として丸めない。

master input が届いた時点で、configured input のすべてに valid frame が存在し、master frame の `header.stamp` から見て `max_frame_age_ms` 内であり、master と同じ sample count である場合だけ publish する。入力欠落や長さ不一致を skip して partial mix を publish することは禁止する。

`header.stamp` が zero、負時刻、または nanosecond range 外の frame は valid frame として扱わない。callback 受信時刻で stale 判定を代用しない。

## backend 境界

PCM16LE decode、gain application、sample summing、overflow/range detection、PCM16LE encode は ROS-free backend `internal_pcm16_mixer` に閉じる。backend は ROS2 topic、`fa_interfaces/msg/AudioFrame`、diagnostics、timestamp cache を知らない。

ROS node は topic subscription/publication、latest frame cache、timestamp stale 判定、AudioFrame metadata validation、`stream_id` 更新、diagnostics を担当する。

## non-goals

- resampling
- channel conversion
- limiter
- clipping / hidden range clamp
- ducking
- output device playback
