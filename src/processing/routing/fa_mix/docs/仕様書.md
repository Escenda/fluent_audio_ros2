# fa_mix 仕様書

## 目的

`fa_mix` は複数の `AudioFrame` stream を 1 つの output stream へ混ぜる routing node である。TTS、monitor、loopback など複数経路を扱うための明示的な mixing point として使う。

## 入力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `input_topics` | `["audio/tts/frame"]` | mix 対象 topic 群 |

## 出力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `output_topic` | `audio/output/frame` | mixed audio frame |

## parameters

| parameter | 意味 |
| --- | --- |
| `input_topics` | 入力 topic 一覧 |
| `input_gains_db` | 入力ごとの gain。1 要素なら全入力共通 |
| `master_index` | publish trigger とする入力 index |
| `expected.*` | 入力 format contract |
| `max_frame_age_ms` | master 以外の latest frame 有効期限 |
| `qos.*` | audio frame QoS |
| `diagnostics.publish_period_ms` | diagnostics publish 周期 |

## lifecycle

起動時に input/output topic、gain 数、master index、expected format、QoS を検証する。不正 config は起動失敗する。

## QoS

input/output は同一 QoS で扱う。default は reliable。

## error handling

format mismatch、stale frame、decode failure、mixed sample の PCM16 range overflow は drop counter に反映し、別 format へ推測変換しない。overflow を limiter として丸めない。

## backend selection

なし。現在は node 内の PCM16LE mixer を使う。高度な routing engine が必要な場合は dedicated backend に分離する。

## non-goals

- resampling
- channel conversion
- limiter
- clipping / hidden range clamp
- ducking
- output device playback
