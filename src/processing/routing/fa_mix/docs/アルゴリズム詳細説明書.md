# fa_mix アルゴリズム詳細説明書

## 処理手順

```text
AudioFrame from any input
  ↓
validate metadata / expected format in node
  ↓
store latest frame in node
  ↓
if frame is from master input:
      collect fresh frames for every configured input
      fail closed when any input is missing or stale
      pass byte payloads to backend
      backend decodes PCM16LE to float32
      backend applies input gain
      backend sums samples
      backend validates normalized PCM16 range
      backend encodes PCM16LE
      node wraps backend payload in output AudioFrame
```

## mixing

入力 `n` の gain は dB から線形値へ変換する。

```text
linear_gain = 10 ^ (gain_db / 20)
mix[i] = sum(input_n[i] * linear_gain_n)
```

出力は backend が `[-1.0, 1.0]` の範囲内だけ PCM16LE へ戻す。範囲外の mixed sample が出た場合は typed status を返し、node は frame を drop する。`input_gains_db` を下げるか、前段に明示的な limiter node を追加する。`fa_mix` 自体は hidden limiter として clamp しない。

`input_gains_db` は必須であり、1 要素または入力数と同数だけを受け付ける。空配列を暗黙の 0 dB として扱わない。

## frame / buffer 前提

- 全入力は同一 sample rate / channel / bit depth / encoding。
- 現在は PCM16LE のみ。
- master 以外の frame は master frame の `header.stamp` から見て `max_frame_age_ms` 内であること。受信時刻で古い frame を新しい観測として扱わない。
- zero / negative / range 外の `header.stamp` は frame contract 違反として drop する。
- master と master 以外の decoded sample count は完全一致すること。一致しない場合は backend が error status を返し、shorter buffer へ切り詰めず、mixed frame 全体を drop する。

## backend commit policy

backend は成功時だけ output payload と `last_sample_count` を更新する。入力数不一致、empty input、misaligned input、sample count mismatch、overflow では output payload を変更しない。

## known limitations

- limiter / compressor は未実装。必要なら `processing/dynamics` に置く。
- resampling と channel conversion は別 node に分離する。
