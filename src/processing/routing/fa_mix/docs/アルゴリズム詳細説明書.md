# fa_mix アルゴリズム詳細説明書

## 処理手順

```text
AudioFrame from any input
  ↓
validate expected format
  ↓
store latest frame
  ↓
if frame is from master input:
      collect fresh frames for every configured input
      fail closed when any input is missing, stale, undecodable, or length-mismatched
      decode PCM16LE to float32
      apply input gain
      sum samples
      validate normalized PCM16 range
      encode PCM16LE
      publish mixed frame
```

## mixing

入力 `n` の gain は dB から線形値へ変換する。

```text
linear_gain = 10 ^ (gain_db / 20)
mix[i] = sum(input_n[i] * linear_gain_n)
```

出力は `[-1.0, 1.0]` の範囲内だけ PCM16LE へ戻す。範囲外の mixed sample が出た場合は frame を drop し、`input_gains_db` を下げるか、前段に明示的な limiter node を追加する。`fa_mix` 自体は hidden limiter として clamp しない。

`input_gains_db` は必須であり、1 要素または入力数と同数だけを受け付ける。空配列を暗黙の 0 dB として扱わない。

## frame / buffer 前提

- 全入力は同一 sample rate / channel / bit depth / encoding。
- 現在は PCM16LE のみ。
- master 以外の frame は `max_frame_age_ms` 内であること。
- master と master 以外の decoded sample count は完全一致すること。一致しない場合は shorter buffer へ切り詰めず、mixed frame 全体を drop する。

## known limitations

- limiter / compressor は未実装。必要なら `processing/dynamics` に置く。
- resampling と channel conversion は別 node に分離する。
