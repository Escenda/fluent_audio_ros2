# fa_patchbay アルゴリズム詳細説明書

## 処理概要

`fa_patchbay` は起動時に route table を固定し、topic ごとに subscription を作成する。受信した `AudioFrame` は format と metadata を検証し、同じ input topic に紐づく route へ 1 copy ずつ publish する。

## 起動時処理

1. ROS parameter を宣言する。
2. `input_topics` と `output_topics` を読み込む。
3. array 長、空 topic、重複 route、self route を検証する。
4. expected format と QoS / diagnostics 設定を検証する。
5. route ごとに publisher を作る。
6. unique input topic ごとに subscription を作る。
7. diagnostics timer を作る。

無効 config は補完しない。起動時に `std::runtime_error` を投げ、node を終了させる。

## Runtime Pipeline

```text
AudioFrame on input topic
  -> increment frames_in
  -> validate source_id / stream_id / expected format / data alignment
  -> invalid: increment frames_dropped and return
  -> valid: copy frame per route
  -> set copied stream_id to output topic
  -> publish
  -> increment copies_out
```

## Frame Validation

`stream_id` は callback が受け取った input topic と一致しなければならない。これは route 推定や stale stream の混入を防ぐためである。

data alignment は次で判定する。

```text
bytes_per_frame = expected.channels * (expected.bit_depth / 8)
data.size() % bytes_per_frame == 0
```

`data` が空の場合は drop する。

## Copy Policy

frame copy は `fa_interfaces::msg::AudioFrame out = msg;` で作成し、`out.stream_id` のみを route output topic に変更する。

`fa_patchbay` は PCM payload を解釈しない。byte order、sample 値、振幅、channel 数、sample rate は変更しない。

## Diagnostics

diagnostics timer は `diagnostics.publish_period_ms` ごとに現在の route table と counter を publish する。counter は `std::atomic<uint64_t>` で保持する。

## Backend Interface

runtime backend は存在しない。device、file、network、DSP library とは接続しない。
