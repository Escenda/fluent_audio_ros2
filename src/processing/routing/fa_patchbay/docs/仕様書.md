# fa_patchbay 仕様書

## 目的

`fa_patchbay` は、明示設定された route table に従って `AudioFrame` stream を静的に複製する routing node である。

## 非目標

- DSP
- format conversion
- gain / normalize
- mixing
- device I/O
- route の推定
- 欠損 config の補完

## 所属分類

`src/processing/routing/fa_patchbay`

## Node

| 項目 | 値 |
| --- | --- |
| package | `fa_patchbay` |
| executable | `fa_patchbay_node` |
| default node name | `fa_patchbay` |
| default config key | `fa_patchbay` |

## 入力 / 出力

`input_topics` と `output_topics` は同じ長さの string array とする。

各 index が 1 route を表す。

```text
input_topics[i] -> output_topics[i]
```

複数 route が同じ input topic を共有してよい。route pair は重複してはならない。`input_topics[i] == output_topics[i]` は禁止する。

## Message Contract

対象 message は `fa_interfaces/msg/AudioFrame` である。

受信 frame は次を満たす必要がある。

- `source_id` が空でない
- `stream_id` が受信した input topic と一致する
- `sample_rate` が `expected.sample_rate` と一致する
- `channels` が `expected.channels` と一致する
- `encoding` が `expected.encoding` と一致する
- `bit_depth` が `expected.bit_depth` と一致する
- `layout` が `expected.layout` と一致する
- `data` が空でない
- `data.size()` が `channels * bytes_per_sample` の倍数である

有効 frame は、route ごとに 1 copy publish する。`header`、`source_id`、`encoding`、`sample_rate`、`channels`、`bit_depth`、`layout`、`data`、`epoch` は変更しない。変更する field は `stream_id` のみで、route の output topic を設定する。

## Parameters

| parameter | 型 | 必須条件 |
| --- | --- | --- |
| `input_topics` | string array | 1件以上、空文字禁止 |
| `output_topics` | string array | 1件以上、空文字禁止、`input_topics` と同じ長さ |
| `expected.sample_rate` | int | `> 0` |
| `expected.channels` | int | `> 0` |
| `expected.encoding` | string | `PCM16LE`, `PCM32LE`, `FLOAT32LE` |
| `expected.bit_depth` | int | encoding と対応する byte aligned bit depth |
| `expected.layout` | string | `interleaved` |
| `qos.depth` | int | `> 0` |
| `qos.reliable` | bool | `true` なら reliable、`false` なら best effort |
| `diagnostics.publish_period_ms` | int | `> 0` |
| `diagnostics.qos.depth` | int | `> 0` |
| `diagnostics.qos.reliable` | bool | `true` なら reliable、`false` なら best effort |

対応 format は次のみである。

| encoding | bit depth | layout |
| --- | --- | --- |
| `PCM16LE` | `16` | `interleaved` |
| `PCM32LE` | `32` | `interleaved` |
| `FLOAT32LE` | `32` | `interleaved` |

## Fail Closed

次の場合、起動を失敗させる。

- route array が空
- route array 長が一致しない
- route topic が空
- route pair が重複
- output topic が input topic と同じ
- expected format が不正または未対応
- QoS depth が不正
- diagnostics period が不正

runtime frame が不正な場合は publish せず drop し、warning と diagnostics counter を残す。

## Diagnostics

`diagnostics` topic に `diagnostic_msgs/msg/DiagnosticArray` を publish する。

必須 key:

- `route_count`
- `unique_input_count`
- `input_topic.N`
- `output_topic.N`
- `frames_in`
- `frames_dropped`
- `copies_out`

## Backend

runtime backend は持たない。詳細は `docs/backends/no_runtime_backend.md` を参照する。
