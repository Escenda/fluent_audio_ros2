# fa_monitor_mix アルゴリズム詳細説明書

## 1. 概要

`fa_monitor_mix` は latest valid frame を入力ごとに保持し、master input の到着を trigger として
monitor bus を生成する。入力不足や stale は音声意味を変えるため、silence では補完しない。

## 2. 入力検証

各 subscription callback は、受信 frame を保存する前に contract を検証する。

1. `source_id` と `stream_id` を検証する。
2. format fields が `expected.*` と一致することを検証する。
3. `FLOAT32LE` byte列として channel frame 境界に整列していることを検証する。
4. 全 sample が finite で `[-1.0, 1.0]` に収まることを検証する。

検証に失敗した frame は latest state に保存しない。

## 3. Mix trigger

`master_index` の入力が valid の場合だけ mix を試行する。master 以外の入力は、最後に受信した
valid frame の受信時刻を `max_frame_age_ms` と比較する。

## 4. Mix 処理

各入力 sample に `input_gains_db` から変換した linear gain を掛け、同じ sample index ごとに加算する。
入力 byte length が master と異なる場合は frame 境界が揃っていないため drop する。

mix 中または master gain 適用後に sample が finite でない、または `[-1.0, 1.0]` を超えた場合は
publish しない。clamp、limiter、normalize は別 node の責務である。

## 5. 出力生成

output frame は master frame の `header` と `epoch` を引き継ぐ。`source_id` は `output.source_id`、
`stream_id` は `output_topic` に置き換える。format fields は `expected.*` を設定する。

## 6. 計算量

sample 数を `N`、入力数を `M` とした場合、mix の計算量は `O(N * M)`、一時 buffer は
入力 frame decode と mix buffer のため `O(N)` である。
