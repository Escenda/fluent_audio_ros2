# fa_monitor_mix 仕様書

## 1. 目的

`fa_monitor_mix` は、複数の `AudioFrame` stream から monitor bus 用の独立した mix stream を生成する
routing node である。

ROS2 node は topic subscription / publish / diagnostics だけを担当し、sample validation と mix は
ROS-free `internal_monitor_mix` backend が担当する。backend は ROS2 topic や `AudioFrame` を知らない。

## 1.1. 仕様 ID

| ID | 内容 |
| --- | --- |
| `FA-MONITOR-MIX-SPEC-001` | `fa_monitor_mix` は複数 `AudioFrame` stream から monitor bus 用の独立 mix stream を生成する routing node である。 |
| `FA-MONITOR-MIX-SPEC-002` | ROS2 node は topic subscription / publish / diagnostics を担当し、sample validation と mix は ROS-free backend に閉じる。 |
| `FA-MONITOR-MIX-SPEC-003` | `input_topics` / `output_topic` と `input_stream_ids` / `output.stream_id` は別契約であり、topic identity と stream identity の衝突を起動時に拒否する。 |
| `FA-MONITOR-MIX-SPEC-004` | `input_topics`、`input_stream_ids`、`input_gains_db`、`master_index`、output identity、expected format、QoS、diagnostics parameter は required で、不正値は fail closed する。 |
| `FA-MONITOR-MIX-SPEC-005` | 入力 frame は `FLOAT32LE` / 32 bit / interleaved、expected sample rate / channels、finite normalized sample を満たす必要がある。 |
| `FA-MONITOR-MIX-SPEC-006` | master trigger 時に全入力の latest valid frame が揃い、stale でなく、byte length が一致する場合だけ publish する。 |
| `FA-MONITOR-MIX-SPEC-007` | missing / stale / byte length mismatch / range overflow は monitor frame を publish せず drop counter に反映する。 |
| `FA-MONITOR-MIX-SPEC-008` | output metadata は master frame と monitor output config から作り、`source_id` / `stream_id` は output config を使う。 |
| `FA-MONITOR-MIX-SPEC-009` | device 出力、resample、format conversion、limiter / compressor / normalize、silence 合成、stale frame reuse は行わない。 |
| `FA-MONITOR-MIX-SPEC-010` | diagnostics は frame counter、drop counter、backend / output state を publish する。 |
| `FA-MONITOR-MIX-SPEC-011` | launch file は `node_name` / `config_file` を明示引数とし、package-local default config を推測しない。 |
| `FA-MONITOR-MIX-SPEC-012` | pytest / backend gTest contract は `colcon test` から実行される。 |

## 2. 非目標

- device 出力
- resample
- format conversion
- limiter / compressor / normalize
- 欠損入力への silence 合成
- stale frame の再利用

## 3. 入力

`input_topics` で指定された各 topic から `fa_interfaces/msg/AudioFrame` を購読する。domain stream identity は
`input_stream_ids` で別に指定し、topic 名と同一にしない。

入力 frame は次の条件をすべて満たす必要がある。

- `source_id` が空でない
- `stream_id` が対応する `input_stream_ids` と一致する
- `sample_rate` / `channels` / `encoding` / `bit_depth` / `layout` が `expected.*` と一致する
- `encoding=FLOAT32LE`
- `bit_depth=32`
- `layout=interleaved`
- `data` が空でなく、channel frame 境界に整列している
- 全 sample が finite かつ `[-1.0, 1.0]` の範囲内

## 4. 出力

master input frame を trigger として、全入力の latest valid frame が揃っている場合だけ
`output_topic` へ `AudioFrame` を publish する。

出力 frame は次の値を持つ。

- `header`: master frame 由来
- `epoch`: master frame 由来
- `source_id`: `output.source_id`
- `stream_id`: `output.stream_id`
- format fields: `expected.*`
- `data`: `FLOAT32LE` interleaved sample bytes

## 5. Parameter

| Parameter | 必須 | Default | 条件 |
| --- | --- | --- | --- |
| `input_topics` | yes | `["audio/program/frame", "audio/tts/frame"]` | 非空、各 topic 非空 |
| `input_stream_ids` | yes | `["audio/program_bus", "audio/tts_bus"]` | `input_topics` と同数、各 stream_id 非空かつ一意。raw / resolved topic 名と一致不可 |
| `input_gains_db` | yes | `[0.0]` | size 1 または `input_topics` と同数 |
| `master_index` | yes | `0` | `input_topics` の範囲内 |
| `output_topic` | yes | `audio/monitor/frame` | 非空 |
| `output.stream_id` | yes | `audio/monitor_bus` | 非空。raw / resolved topic 名と `input_stream_ids` と一致不可 |
| `output.source_id` | yes | `monitor_mix` | 非空 |
| `expected.sample_rate` | yes | `48000` | 正の整数 |
| `expected.channels` | yes | `2` | 正の整数 |
| `expected.encoding` | yes | `FLOAT32LE` | `FLOAT32LE` のみ |
| `expected.bit_depth` | yes | `32` | `32` のみ |
| `expected.layout` | yes | `interleaved` | `interleaved` のみ |
| `max_frame_age_ms` | yes | `100` | 正の整数 |
| `qos.depth` | yes | `10` | 正の整数 |
| `qos.reliable` | yes | `false` | boolean |
| `diagnostics.publish_period_ms` | yes | `1000` | 正の整数 |
| `diagnostics.qos.depth` | yes | `10` | 正の整数 |
| `diagnostics.qos.reliable` | yes | `true` | boolean |

## 6. Drop 条件

次の場合は output frame を publish しない。

- 入力 frame contract 違反
- master frame 到着時にいずれかの入力 frame が未受信
- いずれかの入力 frame が `max_frame_age_ms` を超過
- master と他入力の byte length が異なる
- mix 結果 sample が finite でない、または `[-1.0, 1.0]` を超える

resolved `input_topics` は一意でなければならない。resolved `output_topic` がいずれかの input topic と一致する場合は、loopback を暗黙に作らず起動失敗する。
`input_stream_ids` と `output.stream_id` は ROS topic identity とは別の frame identity として扱い、raw / resolved `input_topics` / `output_topic` と同値の場合は起動失敗する。

## 7. Diagnostics

`diagnostics` topic に次の counter を publish する。

- `frames_in`
- `frames_valid`
- `frames_dropped`
- `mix_frames_out`
- `mix_frames_dropped`
- `stale_frame_drops`
- `missing_frame_drops`
- `range_drops`
