# fa_loopback アルゴリズム詳細説明書

## startup

1. `input_topic`、`output_topic`、`loopback.require_distinct_topics` を読む。
2. expected format、QoS、diagnostics 周期を読む。
3. 必須値が空でないことを確認する。
4. `loopback.require_distinct_topics=true` かつ `input_topic == output_topic` の場合は起動を止める。
5. expected format が byte-aligned interleaved PCM/FLOAT の明示契約であることを確認する。
6. publisher、subscription、diagnostics timer を作る。

## frame path

1. subscription callback で `frames_in` を増やす。
2. null frame は drop する。
3. `AudioFrame` contract を検証する。
4. 検証に失敗した frame は publish せず、`frames_dropped` を増やす。
5. 検証に成功した frame を copy する。
6. copy の `stream_id` を `output_topic` に更新する。
7. publish 後に `frames_out` を増やす。

## routing identity

`fa_loopback` は音声 bytes を処理しない。`data` は入力 frame の copy によって保持される。sample rate、channel 数、encoding、bit depth、layout、epoch は routing の前後で同一である。

この node が変えるのは `stream_id` だけである。これは output topic が新しい stream boundary であることを `AudioFrame` 内にも明示するためである。

## invalid frame handling

invalid frame は fail closed として drop する。drop は仕様化された degraded behavior であり、以下を満たす。

- warning log を出す
- `frames_dropped` を増やす
- invalid bytes を publish しない
- format 推測や補正を行わない

## same-topic passthrough

default では same-topic loopback を禁止する。`loopback.require_distinct_topics=false` の場合だけ許可する。この設定は operator が ROS2 graph 上の loopback を明示的に構成していることを示す制御であり、node が自動 fallback として使うものではない。
