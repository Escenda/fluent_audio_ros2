# fa_loopback アルゴリズム詳細説明書

## startup

1. `input_topic`、`output_topic`、`input_stream_id`、`output.stream_id` を required parameter として読む。
2. expected format、audio QoS、diagnostics QoS、diagnostics 周期を required parameter として読む。
3. required parameter が未設定または型不一致の場合は起動を止める。
4. `input_topic` と `output_topic` が解決後に同じ場合は起動を止める。
5. `input_stream_id` と `output.stream_id` が同じ場合は起動を止める。
6. stream ID が raw/resolved topic 名と一致する場合は起動を止める。
7. expected format が byte-aligned interleaved PCM/FLOAT の明示契約であることを確認する。
8. publisher、subscription、diagnostics timer を作る。

## frame path

1. subscription callback で `frames_in` を増やす。
2. null frame は drop する。
3. `AudioFrame` contract を検証する。
4. 検証に失敗した frame は publish せず、`frames_dropped` を増やす。
5. 検証に成功した frame を copy する。
6. copy の `stream_id` を `output.stream_id` に更新する。
7. publish 後に `frames_out` を増やす。

## routing identity

`fa_loopback` は音声 bytes を処理しない。`data` は入力 frame の copy によって保持される。sample rate、channel 数、encoding、bit depth、layout、epoch は routing の前後で同一である。

この node が変えるのは `stream_id` だけである。`output_topic` は ROS2 graph 上の配送先、`output.stream_id` は `AudioFrame` 内の新しい stream boundary である。

## invalid frame handling

invalid frame は fail closed として drop する。drop は仕様化された degraded behavior であり、以下を満たす。

- warning log を出す
- `frames_dropped` を増やす
- invalid bytes を publish しない
- format 推測や補正を行わない

## same-topic guard

same-topic loopback は常に禁止する。`input_topic` と `output_topic` が解決後に同じ場合は startup error とする。same-topic を許可する旧 operator escape hatch は残さない。

## no runtime backend

`fa_loopback` は byte-for-byte routing node であり、音声処理 engine を持たない。DSP、model runtime、device API、cloud API への backend 接続はしない。
