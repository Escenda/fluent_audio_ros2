# fa_loopback テスト設計

## 目的

`fa_loopback` が routing node として、AudioFrame 契約、startup validation、topic/stream identity 分離、metadata/data 保持、self-loop guard、diagnostics を満たすことを確認する。

## traceability

| test ID | 対応仕様 | 観点 |
| --- | --- | --- |
| `FA-LOOPBACK-TC-001` -> `FA-LOOPBACK-SPEC-001` | routing boundary | loopback が routing node であり、DSP / format / device adapter でないこと。 |
| `FA-LOOPBACK-TC-002` -> `FA-LOOPBACK-SPEC-002` | topic / stream identity | topic と stream ID を分離し、衝突を fail closed すること。 |
| `FA-LOOPBACK-TC-003` -> `FA-LOOPBACK-SPEC-003` | self-loop guard | 解決後の same-topic config を拒否すること。 |
| `FA-LOOPBACK-TC-004` -> `FA-LOOPBACK-SPEC-004` | required startup config | expected format、QoS、diagnostics parameter を required とし、不正値で起動失敗すること。 |
| `FA-LOOPBACK-TC-005` -> `FA-LOOPBACK-SPEC-005` | runtime frame validation | source / stream identity、format、data alignment を publish 前に検証すること。 |
| `FA-LOOPBACK-TC-006` -> `FA-LOOPBACK-SPEC-006` | copy-only output | 出力が入力 copy で、`stream_id` だけを更新すること。 |
| `FA-LOOPBACK-TC-007` -> `FA-LOOPBACK-SPEC-007` | no audio processing | resample、decode、encode、gain、clamp、normalize を持たないこと。 |
| `FA-LOOPBACK-TC-008` -> `FA-LOOPBACK-SPEC-008` | diagnostics | config 値と frame counter が diagnostics に含まれること。 |
| `FA-LOOPBACK-TC-009` -> `FA-LOOPBACK-SPEC-009` | launch contract | launch file が `node_name` / `config_file` だけを要求し、package default を推測しないこと。 |
| `FA-LOOPBACK-TC-010` -> `FA-LOOPBACK-SPEC-010` | colcon coverage | package の pytest contract が `colcon test` から実行されること。 |

## unit-style contract tests

- default config が明示的な loopback routing contract を持つ
- package launch が `node_name` と `config_file` を必須にし、package default config path へ fallback しない
- startup validation が不正設定で fail closed する
- 解決後に同じ `input_topic` / `output_topic` を拒否する
- `input_stream_id` / `output.stream_id` と topic 名の混同を拒否する
- runtime frame validation が不正 frame を publish 前に drop する
- output は入力 frame の copy であり、`stream_id` だけを更新する
- resample、decode、encode、gain、clamp、normalize を実装していない
- diagnostics に config 値と `frames_in` / `frames_out` / `frames_dropped` が含まれる
- required package layout を満たす
- colcon から pytest contract を実行できる

## integration / launch

`test/integration` と `test/launch` は package layout として用意する。ROS2 launch test pattern が共有側で固まった後、以下を追加する。

- valid frame publish 後に output topic へ同一 bytes が届く
- invalid format frame が output topic へ届かない
- diagnostics counter が publish される
- same-topic config の startup failure を確認する
- topic 名と stream ID を同一にした config の startup failure を確認する
