# fa_loopback 仕様書

## 1. 目的

`fa_loopback` は、既存の `AudioFrame` stream を別 topic へ routing し、output を別の input として扱えるようにする node である。

この package は routing node であり、DSP、format conversion、device adapter ではない。

## 2. 入出力

| 種別 | パラメータ | 型 |
| --- | --- | --- |
| subscribe | `input_topic` | `fa_interfaces/msg/AudioFrame` |
| publish | `output_topic` | `fa_interfaces/msg/AudioFrame` |
| publish | `diagnostics` | `diagnostic_msgs/msg/DiagnosticArray` |

`input_topic` / `output_topic` は ROS2 topic 名である。`input_stream_id` / `output.stream_id` は `AudioFrame.stream_id` として扱う stream identity であり、topic 名とは別の値として設定する。

## 3. パラメータ

launch から渡す `node_name` と `config_file` は必須である。package launch は default config path を推測しない。

node parameter はすべて required であり、未設定の場合は startup error とする。`config/default.yaml` は単体デバッグ用の明示例であり、runtime fallback ではない。

| name | required | validation |
| --- | --- | --- |
| `input_topic` | yes | non-empty |
| `output_topic` | yes | non-empty |
| `input_stream_id` | yes | non-empty、raw/resolved topic 名と一致しない |
| `output.stream_id` | yes | non-empty、`input_stream_id` と異なる、raw/resolved topic 名と一致しない |
| `expected.sample_rate` | yes | `1..384000` |
| `expected.channels` | yes | `1..64` |
| `expected.encoding` | yes | `PCM16LE`, `PCM32LE`, `FLOAT32LE` |
| `expected.bit_depth` | yes | byte-aligned and matches encoding |
| `expected.layout` | yes | `interleaved` |
| `qos.depth` | yes | `> 0` |
| `qos.reliable` | yes | boolean |
| `diagnostics.qos.depth` | yes | `> 0` |
| `diagnostics.qos.reliable` | yes | boolean |
| `diagnostics.publish_period_ms` | yes | `> 0` |

`input_topic` と `output_topic` が解決後に同じ場合は startup error とする。same-topic を許可する旧 operator escape hatch は持たない。

## 4. AudioFrame 契約

入力 frame は以下をすべて満たす必要がある。

- `source_id` が空ではない
- `stream_id` が空ではない
- `stream_id == input_stream_id`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == expected.encoding`
- `bit_depth == expected.bit_depth`
- `layout == expected.layout`
- `data` が空ではない
- `data.size()` が `channels * bytes_per_sample` で割り切れる

違反した frame は drop する。欠損値の補完、zero 埋め、latest 値の使用、format 推測は行わない。

## 5. 出力契約

出力は入力 `AudioFrame` の copy である。ただし、新しい stream boundary を明示するために `stream_id` だけを `output.stream_id` に更新する。

変更しない field:

- `header`
- `source_id`
- `encoding`
- `sample_rate`
- `channels`
- `bit_depth`
- `layout`
- `data`
- `epoch`

## 6. diagnostics

`diagnostics` topic に以下を含める。

- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `expected.sample_rate`
- `expected.channels`
- `expected.encoding`
- `expected.bit_depth`
- `expected.layout`
- `qos.depth`
- `qos.reliable`
- `diagnostics.qos.depth`
- `diagnostics.qos.reliable`
- `frames_in`
- `frames_out`
- `frames_dropped`

## 7. fail closed

startup config が不正な場合は例外で起動を止める。runtime frame が不正な場合は publish せず drop し、warning と diagnostics counter に反映する。
