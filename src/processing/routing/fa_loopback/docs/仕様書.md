# fa_loopback 仕様書

## 1. 目的

`fa_loopback` は、既存の `AudioFrame` stream を別 topic へ routing し、output を別の input として扱えるようにする node である。

この package は routing node であり、DSP、format conversion、device adapter ではない。

## 2. 入出力

| 種別 | パラメータ | 型 |
| --- | --- | --- |
| subscribe | `input_topic` | `fa_interfaces/msg/AudioFrame` |
| publish | `output_topic` | `fa_interfaces/msg/AudioFrame` |
| publish | `diagnostics` | `diagnostic_msgs/msg/DiagnosticArray` |

## 3. パラメータ

| name | default | validation |
| --- | --- | --- |
| `input_topic` | `audio/output/frame` | non-empty |
| `output_topic` | `audio/loopback/frame` | non-empty |
| `loopback.require_distinct_topics` | `true` | boolean |
| `expected.sample_rate` | `48000` | `> 0` |
| `expected.channels` | `1` | `> 0` |
| `expected.encoding` | `PCM16LE` | `PCM16LE`, `PCM32LE`, `FLOAT32LE` |
| `expected.bit_depth` | `16` | byte-aligned and matches encoding |
| `expected.layout` | `interleaved` | `interleaved` |
| `qos.depth` | `10` | `> 0` |
| `qos.reliable` | `true` | boolean |
| `diagnostics.publish_period_ms` | `1000` | `> 0` |

`loopback.require_distinct_topics=true` の場合、`input_topic == output_topic` は startup error とする。`false` の場合のみ、operator-controlled graph loopback として同一 topic passthrough を許可する。

## 4. AudioFrame 契約

入力 frame は以下をすべて満たす必要がある。

- `source_id` が空ではない
- `stream_id` が空ではない
- `stream_id == input_topic`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == expected.encoding`
- `bit_depth == expected.bit_depth`
- `layout == expected.layout`
- `data` が空ではない
- `data.size()` が `channels * bytes_per_sample` で割り切れる

違反した frame は drop する。欠損値の補完、zero 埋め、latest 値の使用、format 推測は行わない。

## 5. 出力契約

出力は入力 `AudioFrame` の copy である。ただし、publish 先を明示するために `stream_id` だけを `output_topic` に更新する。

変更しない field:

- `header`
- `source_id`
- `encoding`
- `sample_rate`
- `channels`
- `bit_depth`
- `layout`
- `data`
- `epoch`

## 6. diagnostics

`diagnostics` topic に以下を含める。

- `input_topic`
- `output_topic`
- `loopback.require_distinct_topics`
- `expected.sample_rate`
- `expected.channels`
- `expected.encoding`
- `expected.bit_depth`
- `expected.layout`
- `qos.depth`
- `qos.reliable`
- `frames_in`
- `frames_out`
- `frames_dropped`

## 7. fail closed

startup config が不正な場合は例外で起動を止める。runtime frame が不正な場合は publish せず drop し、warning と diagnostics counter に反映する。
