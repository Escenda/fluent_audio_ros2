# fa_bus_router アルゴリズム詳細説明書

## 概要

`fa_bus_router` は `AudioFrame` の transport routing だけを行う。音声 sample、sample rate、channel 数、encoding、bit depth、layout は変換しない。

## 起動時検証

1. `input_topic` が空でないことを確認する。
2. `output_topics` が空でなく、各 topic が空でないことを確認する。
3. `output_topics` が unique であり、`input_topic` を含まないことを確認する。
4. `expected.*` が正の sample rate / channels、対応済み encoding / bit depth、`interleaved` layout であることを確認する。
5. `qos.depth` と `diagnostics.publish_period_ms` が正であることを確認する。
6. 不正な設定は例外で起動失敗にする。

## runtime 処理手順

1. 入力 frame 数を count する。
2. null frame は drop する。
3. `AudioFrame` 契約を検証する。
4. 不正 frame は drop し、warning を出す。
5. 有効 frame について、`output_topics` の数だけ message copy を作る。
6. 各 copy の `stream_id` を publish 先 topic に設定する。
7. `data` を含む他 field は copy 元のまま publish する。
8. publish copy 数を `copies_out` に反映する。

## routing identity

入力 `stream_id` は `input_topic` と一致する必要がある。出力 `stream_id` は各 output topic と一致する。これにより、topic と message metadata の対応が崩れた frame を後段へ流さない。

## drop 方針

runtime frame validation の失敗は、壊れた stream contract を検出した状態である。値の推定、直近値の再利用、zero padding、format conversion は行わず、その frame だけを drop する。
