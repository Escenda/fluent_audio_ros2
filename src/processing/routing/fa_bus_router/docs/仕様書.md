# fa_bus_router 仕様書

## 目的

`fa_bus_router` は、1本の `AudioFrame` stream を明示された複数の bus topic へ複製する routing node である。音声 sample は変更せず、publish 先を表すために `stream_id` のみを output topic に更新する。

## ノード

| 項目 | 値 |
| --- | --- |
| package | `fa_bus_router` |
| executable | `fa_bus_router_node` |
| node name | `fa_bus_router` |
| input | `input_topic` |
| outputs | `output_topics` |

## 設定契約

| parameter | default | 条件 |
| --- | --- | --- |
| `input_topic` | `audio/frame` | non-empty |
| `output_topics` | `[audio/output/frame]` | non-empty、全要素 non-empty、unique、`input_topic` を含まない |
| `expected.sample_rate` | `48000` | `> 0` |
| `expected.channels` | `1` | `> 0` |
| `expected.encoding` | `PCM16LE` | `PCM16LE`、`PCM32LE`、`FLOAT32LE` のいずれか |
| `expected.bit_depth` | `16` | encoding と一致する byte-aligned depth |
| `expected.layout` | `interleaved` | `interleaved` |
| `qos.depth` | `10` | `> 0` |
| `qos.reliable` | `true` | reliable なら reliable QoS、false なら best effort |
| `diagnostics.publish_period_ms` | `1000` | `> 0` |

起動時設定が不正な場合、node は `std::runtime_error` で fail closed する。

## 入力契約

入力 frame は以下をすべて満たす必要がある。

| 項目 | 条件 |
| --- | --- |
| `source_id` | non-empty |
| `stream_id` | non-empty かつ `input_topic` と一致 |
| `sample_rate` | `expected.sample_rate` と一致 |
| `channels` | `expected.channels` と一致 |
| `encoding` | `expected.encoding` と一致 |
| `bit_depth` | `expected.bit_depth` と一致 |
| `layout` | `expected.layout` と一致 |
| `data` | non-empty かつ `channels * bit_depth / 8` で割り切れる |

runtime frame が不正な場合、その frame を drop し、warning を出す。欠損値の補完、zero 埋め、format 推定は行わない。

## 出力契約

各 output topic には、入力 frame の copy を publish する。

- `stream_id == output topic`
- `header` は入力 frame と同一
- `source_id` は入力 frame と同一
- `sample_rate`、`channels`、`encoding`、`bit_depth`、`layout` は入力 frame と同一
- `data` は入力 frame と同一 byte sequence
- `epoch` は入力 frame と同一

## diagnostics

`diagnostics` topic に以下を含める。

- `input_topic`
- `output_count`
- `output_topic.N`
- `expected.sample_rate`
- `expected.channels`
- `expected.encoding`
- `expected.bit_depth`
- `expected.layout`
- `frames_in`
- `frames_dropped`
- `copies_out`
