## unit contract tests

`test/unit/test_fa_bus_router_audio_frame_contract.py` は package files を静的に検査し、以下を確認する。

- default config が explicit bus routing contract を持つ
- 起動時設定の invalid 値を `std::runtime_error` で fail closed する
- runtime frame validation が `source_id`、`input_stream_id` と一致する `stream_id`、format、data size を検査する
- 出力は入力 `AudioFrame` の copy であり、`stream_id` だけを `output.stream_ids[index]` に更新する
- node が mixing、resampling、gain、sample conversion を実装していない
- diagnostics に routing 状態と counter が出る
- package layout と colcon pytest registration が存在する

## integration tests

今後の integration test では、ROS2 node を launch し、以下を確認する。

- 有効 frame がすべての output topic へ publish される
- output ごとに `stream_id` が対応する `output.stream_ids[index]` と一致する
- `data` byte sequence が入力と一致する
- `sample_rate`、`channels`、`encoding`、`bit_depth`、`layout` が不一致の frame は publish されない
- `output_topics` が重複する config では node が起動しない
- topic と stream identity が衝突する config では node が起動しない

## launch tests

launch test では `launch/fa_bus_router.launch.py` が default config を読み、node name と executable が契約どおりになることを確認する。
