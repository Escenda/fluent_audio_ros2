# fa_latency_compensation テスト設計

## Unit Contract Tests

`test/unit/test_fa_latency_compensation_audio_frame_contract.py` は package の静的 contract を検査します。

- default config が required parameter を明示する
- node が device I/O や audio sample editing token を含まない
- startup validation が fail-closed である
- runtime validation が `source_id` / `stream_id` / format / data alignment を確認する
- compensation が `header.stamp` と `stream_id` だけを更新する
- 負 timestamp と timestamp overflow を drop し、diagnostics counter に反映する
- package layout が required files を含む
- CMake が pytest と `ament_lint_auto` を登録する

## Integration Tests

現時点では launch / ROS graph を実行する integration fixture は持ちません。親 system 登録後に、`input_topic` へ known `AudioFrame` を publish し、`output_topic` の `header.stamp` と `data` byte 同一性を確認する launch test を追加します。

## Non-Goals

この package-local test は actual audio device、speaker、microphone、sample processing quality を検証しません。
