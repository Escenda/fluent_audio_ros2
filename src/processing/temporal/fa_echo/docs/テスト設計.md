## 1. Unit contract test

`test/unit/test_fa_echo_audio_frame_contract.py` で package 内の静的 contract を検証する。

- example config が required parameters と FLOAT32LE interleaved contract を持つこと
- launch の `config_file` が明示必須で、package default を持たないこと
- CMake が pytest integration を持つこと
- `package.xml` が pytest/yaml test dependency を持つこと
- node 実装が device I/O、clip、normalize、resample を含まないこと
- startup parameter が fail closed で検証されること
- frame contract が処理前に検証されること
- ROS topic と stream identity が別 parameter として扱われること
- node が sample validation と delay state mutation を backend へ委譲すること
- backend が echo recurrence を指定式通りに実装すること
- backend が invalid input/output/state を status で返し、clip しないこと
- backend が source_id change で delay state をリセットすること
- diagnostics に設定と counter が出ること

## 2. Backend C++ tests

- canonical sample に dry/wet echo を適用する。
- channel ごとの delay line が独立していることを確認する。
- source_id change で旧 source の delayed sample が混入しないことを確認する。
- empty source / empty input / byte alignment / NaN / normalized range violation を `ProcessStatus` で返す。
- output range violation や representable でない output を reject し、前回 output buffer を上書きしない。
- 未定義の backend status message は fallback 文字列へ丸めず例外にする。

## 3. C++ graph smoke test

- `FaEchoNode` を `rclcpp::NodeOptions` で起動し、test publisher / subscriber と同じ executor に載せる。
- ROS topic と `AudioFrame.stream_id` を別値にした状態で publish できることを確認する。
- output topic に echo 後の frame が publish される。
- `source_id`、sample rate、channels、encoding、bit depth、layout、epoch は保持される。
- `stream_id` は `output.stream_id` に更新される。

## 4. Integration test 方針

親 orchestration で package 登録後に、ROS2 runtime 上で以下を確認する。

- `ros2 launch fa_echo fa_echo.launch.py config_file:=...` で起動する
- `config_file` 未指定では package default を使わず起動失敗になる
- contract に合う `AudioFrame` を publish すると `output_topic` に frame が出る
- `stream_id` は `output.stream_id` に変わり、`source_id` は保持される
- `source_id` を切り替えると旧 source の delayed sample が混入しない
- 不正 format frame は publish されず diagnostics drop counter が増える
