# fa_echo アルゴリズム詳細説明書

## 1. 処理対象

処理対象は `FLOAT32LE` interleaved の `AudioFrame.data` である。sample は `[-1.0, 1.0]` の正規化 float として扱う。

## 2. Delay sample 数

`echo.delay_ms` は `expected.sample_rate` から sample 数へ変換する。

```text
raw_delay_samples = echo.delay_ms * expected.sample_rate / 1000.0
delay_samples = llround(raw_delay_samples)
```

`delay_samples == 0` は起動時エラーである。

## 3. 内部状態

ROS 非依存の `internal_feedback_echo` backend が、各 channel に長さ `delay_samples` の
ring delay line を持つ。初期値は silence sample `0.0` である。ring の write/read 位置は
channel ごとに保持する。ROS node は `AudioFrame` metadata と backend 入出力 byte列の受け渡しだけを行う。

## 4. Echo recurrence

各 input sample `x[n]` と delayed sample `d[n]` から、次の値を計算する。

```text
y[n] = dry_gain * x[n] + wet_gain * d[n]
s[n] = x[n] + feedback_gain * d[n]
```

`y[n]` を出力 sample として書き込み、`s[n]` を ring delay line の現在位置へ保存する。その後 ring 位置を 1 sample 進める。

## 5. 検証と drop

以下のいずれかを満たす frame は publish しない。

- `AudioFrame` の format contract が一致しない
- 入力 sample が finite でない
- 入力 sample が `[-1.0, 1.0]` 外
- echo output が finite でない
- echo output が `[-1.0, 1.0]` 外
- 次 delay state が finite でない
- 次 delay state が `[-1.0, 1.0]` 外

output/state が不正な frame では、計算途中の delay state と output buffer を commit しない。

## 6. Source 切替

`source_id` が前回 commit 済み source と異なる場合、処理前の候補 state を silence 初期化する。frame が正常に処理できた場合のみ、新しい source と delay state を commit する。
