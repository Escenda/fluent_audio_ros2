# fa_echo 仕様書

## 1. 目的

`fa_echo` は FluentAudio の `AudioFrame` stream に、明示設定された内部 feedback echo を適用する processing node である。device capture、playback、file I/O、format 変換、clip、limiter、normalize は扱わない。

## 2. 入出力

| 項目 | 内容 |
| --- | --- |
| 入力 topic | `input_topic` parameter で指定する。空文字は禁止 |
| 出力 topic | `output_topic` parameter で指定する。空文字は禁止 |
| 入力 message | `fa_interfaces/msg/AudioFrame` |
| 出力 message | `fa_interfaces/msg/AudioFrame` |

出力 message は入力 metadata を保持し、`stream_id` のみ `output.stream_id` に更新する。`source_id` は変更しない。
ROS topic 名と `AudioFrame.stream_id` は別の identity として扱う。topic は transport、
stream_id は音声 stream contract であり、設定では `input_stream_id` / `output.stream_id`
として明示する。

## 3. 対応 AudioFrame contract

| field | 必須値 |
| --- | --- |
| `encoding` | `FLOAT32LE` |
| `bit_depth` | `32` |
| `layout` | `interleaved` |
| `sample_rate` | `expected.sample_rate` と一致 |
| `channels` | `expected.channels` と一致 |

`source_id` と `stream_id` は空であってはならない。`stream_id` は `input_stream_id` と一致しなければならない。

## 4. Parameter

| parameter | 内容 | 検証 |
| --- | --- | --- |
| `input_topic` | 購読 topic | 必須、空文字禁止 |
| `output_topic` | publish topic | 必須、空文字禁止 |
| `input_stream_id` | 入力 frame identity | 必須、空文字禁止 |
| `output.stream_id` | 出力 frame identity | 必須、空文字禁止 |
| `echo.delay_ms` | echo delay time | finite かつ `> 0.0`、1 sample 以上に変換できること |
| `echo.feedback_gain` | delay line に戻す delayed 成分の gain | finite かつ `abs(value) < 1.0` |
| `echo.wet_gain` | delayed 成分の出力 gain | finite |
| `echo.dry_gain` | 入力成分の出力 gain | finite |
| `expected.sample_rate` | 期待 sample rate | `> 0` |
| `expected.channels` | 期待 channel 数 | `> 0` |
| `qos.depth` | AudioFrame QoS depth | `> 0` |
| `qos.reliable` | reliable QoS を使うか | boolean |
| `diagnostics.publish_period_ms` | diagnostics publish 周期 | `> 0` |

gain は合計値で正規化しない。設定値の意味を変える fallback は行わない。

## 5. エラー処理

起動時 parameter が不正な場合は例外で起動失敗する。`input_topic == output_topic`、
`input_stream_id == output.stream_id` は自己 feedback や identity 混線を作るため起動失敗する。
frame contract 不一致、入力 sample 不正、echo output/state 不正の場合は該当 frame を
publish せず drop し、diagnostics counter を更新する。clip や silent replacement は行わない。

## 6. 状態管理

delay line は ROS 非依存 backend が channel ごとに独立して保持する。`source_id` が変わった場合は、
旧 source の delayed sample を新 source へ混入させないため delay state を silence 初期値へリセットする。

## 7. Diagnostics

`diagnostics` topic に `DiagnosticArray` を publish する。少なくとも delay/gain 設定、現在の source、入出力/drop counter、source reset counter を含める。
