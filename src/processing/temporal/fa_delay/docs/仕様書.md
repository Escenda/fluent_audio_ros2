# fa_delay 仕様書

## 1. 目的

`fa_delay` は、FluentAudio の FLOAT32LE interleaved 音声フレームに対し、設定した時間だけ音声を後ろへずらす intentional temporal delay を適用する。

## 1.1. 仕様 ID

| ID | 内容 |
| --- | --- |
| `FA-DELAY-SPEC-001` | `fa_delay` は `FLOAT32LE` interleaved 音声 frame に intentional temporal delay を適用する temporal processing node である。 |
| `FA-DELAY-SPEC-002` | `input_topic` / `output_topic` と `input_stream_id` / `output.stream_id` は別契約であり、topic identity と stream identity の衝突を起動時に拒否する。 |
| `FA-DELAY-SPEC-003` | `delay.ms`、expected format、QoS、diagnostics parameter は required で、不正値は fail closed する。 |
| `FA-DELAY-SPEC-004` | runtime frame は source / stream identity、FLOAT32LE format、data alignment、finite normalized sample を検証する。 |
| `FA-DELAY-SPEC-005` | `delay.ms` は `expected.sample_rate` から whole samples に変換し、各 channel に delay buffer を適用する。 |
| `FA-DELAY-SPEC-006` | stream 先頭と `source_id` 変更時の silence 初期値は delay effect の定義であり、失敗補完ではない。 |
| `FA-DELAY-SPEC-007` | output metadata は入力を保持し、`stream_id` と `data` だけを更新する。 |
| `FA-DELAY-SPEC-008` | invalid sample は clamp / normalize / 暗黙変換で補正せず drop する。 |
| `FA-DELAY-SPEC-009` | diagnostics は delay 設定、source、stream identity、counter を publish する。 |
| `FA-DELAY-SPEC-010` | device I/O、resampling、format conversion、channel conversion、gain、normalize、limiter、filter、denoise、echo、reverb、legacy alias を提供しない。 |
| `FA-DELAY-SPEC-011` | launch file は `node_name` / `config_file` を明示引数とし、package-local default config を推測しない。 |
| `FA-DELAY-SPEC-012` | pytest / backend gTest contract は `colcon test` から実行され、`ament_lint_auto` dependency を package manifest に明示する。 |

## 3. 入出力契約

入力 `AudioFrame` は以下を満たす必要がある。

- `source_id` が空でない
- `stream_id` が空でない
- `stream_id == input_stream_id`
- `sample_rate > 0` かつ設定値と一致
- `channels > 0` かつ設定値と一致
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `layout == interleaved`
- `data` が空でない
- `data.size()` が `channels * sizeof(float)` で割り切れる
- 全サンプルが有限で `[-1.0, 1.0]` の範囲内

出力 `AudioFrame` は `header`, `source_id`, `sample_rate`, `channels`, `bit_depth`, `encoding`, `layout`, `epoch` を保持し、`stream_id` を `output.stream_id` に更新し、`data` を delay 済みサンプルへ置き換える。

## 4. delay の意味

`delay.ms` は `expected.sample_rate` を使って whole samples に変換する。変換後の `delay_samples` は各 channel に個別に適用する。

stream の先頭、または accepted frame の `source_id` が変わった直後は、各 channel の delay buffer を `delay_samples` 個の silence sample `0.0F` で初期化する。この silence は「失敗時の補完」ではなく、delay effect の定義そのものである。

## 5. パラメータ

`fa_delay_node` はすべてのパラメータを required として扱う。`fa_delay.launch.py` の `config_file` も必須 launch argument であり、package 内の設定ファイルを暗黙には読み込まない。`config/default.yaml` は debug / example 用の明示 config であり、runtime default ではない。

| 名前 | 型 | 設定例 | 制約 |
| --- | --- | --- | --- |
| `input_topic` | string | `fa_delay/input` | 空不可、出力 topic と resolved identity が異なる |
| `output_topic` | string | `fa_delay/output` | 空不可、入力 topic と resolved identity が異なる |
| `input_stream_id` | string | `audio/buffered/mic` | 空不可、raw / resolved ROS topic identity と異なる |
| `output.stream_id` | string | `audio/delayed/mic` | 空不可、raw / resolved ROS topic identity と異なり、入力 stream id と異なる |
| `delay.ms` | double | `250.0` | finite かつ `> 0`、whole samples 変換後 `>= 1` |
| `expected.sample_rate` | int | `16000` | `0 < value <= 384000` |
| `expected.channels` | int | `1` | `0 < value <= 64` |
| `expected.encoding` | string | `FLOAT32LE` | `FLOAT32LE` のみ |
| `expected.bit_depth` | int | `32` | `32` のみ |
| `expected.layout` | string | `interleaved` | `interleaved` のみ |
| `qos.depth` | int | `10` | `> 0` |
| `qos.reliable` | bool | `false` | true/false |
| `diagnostics.publish_period_ms` | int | `1000` | `> 0` |
| `diagnostics.qos.depth` | int | `10` | `> 0` |
| `diagnostics.qos.reliable` | bool | `true` | true/false |

## 6. エラー処理

起動時設定が不正な場合は例外で起動を停止する。`input_topic == output_topic`、`input_stream_id == output.stream_id`、topic と stream id の同値は起動失敗する。ランタイムフレームが契約に違反した場合は warning を出し、そのフレームを drop する。サンプル値の clamp、normalize、暗黙変換は行わない。

## 7. Diagnostics

`diagnostics` topic に以下を含める。

- `delay_ms`
- `delay_samples`
- `current_source_id`
- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `messages_in`
- `messages_out`
- `messages_dropped`
- `source_resets`
- `expected_channels`

## 8. 非目標

device I/O、resampling、sample format conversion、channel conversion、gain、normalize、limiter、filter、denoise、echo、reverb、legacy compatibility alias は提供しない。
