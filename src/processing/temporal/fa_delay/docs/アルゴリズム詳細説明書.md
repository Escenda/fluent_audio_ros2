# fa_delay アルゴリズム詳細説明書

## 1. 入力モデル

入力は FLOAT32LE interleaved `AudioFrame` である。1 フレームのバイト数は `channels * sizeof(float)` とする。

## 2. delay sample 数

`delay_samples = round(delay.ms * expected.sample_rate / 1000.0)` とする。`expected.sample_rate` は 1 channel あたりの sample rate なので、`delay_samples` は各 channel の delay line 長である。

変換結果が `0` の設定は無効として起動を停止する。効果を持たない delay を silently pass-through しない。

## 3. 状態

ノードは accepted stream の `current_source_id_` と、channel ごとの delay buffer を持つ。

```text
delay_buffers_[channel] = deque<float>(delay_samples, 0.0F)
```

buffer の初期 `0.0F` は configured delay を表す silence であり、欠損データを隠す fallback ではない。

## 4. source reset

accepted frame の `source_id` が現在の `current_source_id_` と異なる場合、全 channel の delay buffer を初期化する。これにより別 source の過去 sample が新しい source へ混入しない。

## 5. サンプル処理

1. `AudioFrame` 契約を検証する。
2. 全 input sample が有限で `[-1.0, 1.0]` の範囲内であることを確認する。
3. 必要なら source state を初期化する。
4. `out = in` とし、`out.stream_id = output_topic` に更新する。
5. 各 interleaved frame / channel について以下を行う。

```text
input_sample = input[channel]
delayed_sample = delay_buffers_[channel].front()
delay_buffers_[channel].pop_front()
delay_buffers_[channel].push_back(input_sample)
output[channel] = delayed_sample
```

ランタイムフレームが不正な場合は state を進めず、publish もしない。

## 6. 出力長

出力 `data.size()` は入力 `data.size()` と同じである。streaming node は終端を知らないため、末尾の flush frame は生成しない。
