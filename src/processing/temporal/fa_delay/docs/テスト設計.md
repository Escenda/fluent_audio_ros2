# fa_delay テスト設計

## 1. 対象

`fa_delay` の package layout、設定契約、C++ 実装上の境界検証、delay sample 変換、source reset、metadata 保持、diagnostics、colcon pytest 登録を対象とする。

## 2. 単体 contract test

`test/unit/test_fa_delay_audio_frame_contract.py` で以下を確認する。

- default config が topic と stream id を分離し、delay parameter、FLOAT32LE interleaved 契約を持つ
- package layout が標準構造と一致する
- 起動時パラメータが fail closed で範囲検証される
- `delay.ms` が `expected.sample_rate` から whole samples へ変換される
- runtime frame が source/stream/format/data size を検証される
- 不正 sample は backend が検出し、state mutation 前に drop される
- `stream_id == input_stream_id` を要求し、出力で `stream_id = output.stream_id` に更新する
- channel ごとの delay buffer が silence で初期化される
- `source_id` 変更で delay state が reset される
- device I/O、resampling、format conversion、channel conversion、normalize、limiter、filter、denoise、echo、reverb を含まない
- diagnostics に delay 設定、source、counters が含まれる
- backend gTest と `ament_cmake_pytest` で colcon からテストが実行される
- 未定義の backend status message は fallback 文字列へ丸めず例外にする

## 3. 統合テスト

現時点では統合テストは `.gitkeep` のみ配置する。ROS2 runtime を使う publish/subscribe 統合テストは、fa_interfaces と実行環境の fixture が整った段階で追加する。

## 4. 実行コマンド

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/temporal/fa_delay/test/unit -q
```
