# fa_reverb 仕様書

## 1. 目的

`fa_reverb` は `FLOAT32LE` interleaved `AudioFrame` に残響を付与する temporal processing node である。
source / sink adapter、format conversion、resample、limiter、normalize は責務に含めない。

## 2. 所属分類

- FluentAudio 分類: `processing/temporal`
- Package: `fa_reverb`
- Node: `fa_reverb`
- Backend: `internal_feedback_delay`

## 3. 入出力

| 種別 | 名前 | 型 |
| --- | --- | --- |
| Subscribe | `input_topic` parameter で指定 | `fa_interfaces/msg/AudioFrame` |
| Publish | `output_topic` parameter で指定 | `fa_interfaces/msg/AudioFrame` |
| Publish | `diagnostics` | `diagnostic_msgs/msg/DiagnosticArray` |

出力 `AudioFrame` は入力 metadata を保持し、`stream_id` のみ `output.stream_id` に更新する。
ROS topic 名と `AudioFrame.stream_id` は別の identity として扱う。topic は transport、
stream_id は音声 stream contract であり、設定では `input_stream_id` / `output.stream_id`
として明示する。

## 4. Message contract

入力 frame は次を満たす必要がある。

- `source_id` が空でない
- `stream_id == input_stream_id`
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `layout == interleaved`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `data` が空でなく、`channels * sizeof(float)` で割り切れる
- 全 sample が finite かつ `[-1.0, 1.0]`

違反した frame は publish せず drop する。

## 5. Parameters

すべて required であり、コード上に有効な runtime default はない。

| Parameter | 型 | 条件 |
| --- | --- | --- |
| `input_topic` | string | 非空 |
| `output_topic` | string | 非空 |
| `input_stream_id` | string | 非空 |
| `output.stream_id` | string | 非空 |
| `reverb.room_size` | double | finite, `0.0 <= value <= 1.0` |
| `reverb.damping` | double | finite, `0.0 <= value <= 1.0` |
| `reverb.wet_gain` | double | finite, `0.0 <= value <= 1.0` |
| `reverb.dry_gain` | double | finite, `0.0 <= value <= 1.0` |
| `expected.sample_rate` | int | `0 < value <= 384000` |
| `expected.channels` | int | `0 < value <= 64` |
| `expected.encoding` | string | `FLOAT32LE` |
| `expected.bit_depth` | int | `32` |
| `expected.layout` | string | `interleaved` |
| `qos.depth` | int | `> 0` |
| `qos.reliable` | bool | 明示指定 |
| `diagnostics.publish_period_ms` | int | `> 0` |

追加条件として `reverb.dry_gain + reverb.wet_gain <= 1.0` を満たす必要がある。

## 6. Fail closed 条件

起動時 config が欠落または不正な場合、node は例外で起動を中止する。
`input_topic == output_topic`、`input_stream_id == output.stream_id` は自己 feedback や
identity 混線を作るため起動失敗する。
`input_stream_id` または `output.stream_id` が `input_topic` / `output_topic` のどちらかと
一致する場合も、transport と stream contract の結合を作るため起動失敗する。
runtime frame が不正な場合、その frame は drop し、内部 delay state は更新しない。
処理結果または feedback state が finite でない、または `[-1.0, 1.0]` を超える場合も frame を drop し、clip / clamp は行わない。

## 7. Diagnostics

`diagnostics` へ次の情報を publish する。

- `room_size`
- `damping`
- `wet_gain`
- `dry_gain`
- `effective_feedback_gain`
- `delay_lines`
- `current_source_id`
- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `messages_in`
- `messages_out`
- `messages_dropped`
- `source_resets`
- `expected_channels`
