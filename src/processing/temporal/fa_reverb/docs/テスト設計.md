# fa_reverb テスト設計

## 1. Scope

初期実装では unit pytest contract により package layout、required parameter 宣言、frame validation、DSP state 更新の fail-closed 条件を検証する。
ROS2 launch 結合と実音声 fixture は後続の integration / launch test で扱う。

## 2. 仕様 ID

| ID | 仕様 |
| --- | --- |
| FA-REVERB-SPEC-001 | package layout が標準構成を満たす |
| FA-REVERB-SPEC-002 | required parameter に有効な runtime default を持たない |
| FA-REVERB-SPEC-003 | config parameter を finite / range checked する |
| FA-REVERB-SPEC-004 | `FLOAT32LE` interleaved `AudioFrame` contract を検証する |
| FA-REVERB-SPEC-005 | input sample が finite / normalized でない場合 drop する |
| FA-REVERB-SPEC-006 | multi-tap feedback delay state を channel ごとに持つ |
| FA-REVERB-SPEC-007 | output / feedback state が finite / normalized でない場合 clamp せず drop する |
| FA-REVERB-SPEC-008 | `source_id` 変化時に state を reset する |
| FA-REVERB-SPEC-009 | diagnostics に config と counters を出す |
| FA-REVERB-SPEC-010 | colcon から pytest contract を実行できる |

## 3. Test case 対応

| Test case | 対応仕様 |
| --- | --- |
| `test_package_layout_matches_standard_processing_layout` | FA-REVERB-SPEC-001 |
| `test_reverb_parameters_are_declared_without_runtime_defaults` | FA-REVERB-SPEC-002 |
| `test_reverb_parameters_are_range_checked` | FA-REVERB-SPEC-003 |
| `test_reverb_validates_float32_interleaved_frame_contract` | FA-REVERB-SPEC-004 |
| `test_reverb_drops_invalid_samples_before_state_mutation` | FA-REVERB-SPEC-005 |
| `test_reverb_uses_per_channel_multi_tap_feedback_delay_state` | FA-REVERB-SPEC-006 |
| `test_reverb_rejects_invalid_output_or_state_without_clipping` | FA-REVERB-SPEC-007 |
| `test_reverb_resets_state_when_source_id_changes` | FA-REVERB-SPEC-008 |
| `test_diagnostics_include_reverb_source_and_counters` | FA-REVERB-SPEC-009 |
| `test_colcon_runs_pytest_contracts` | FA-REVERB-SPEC-010 |

## 4. 未検証範囲
