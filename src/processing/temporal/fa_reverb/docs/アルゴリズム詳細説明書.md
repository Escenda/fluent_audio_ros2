# fa_reverb アルゴリズム詳細説明書

## 1. Processing pipeline

1. `AudioFrame` の identity / format / payload alignment を検証する。
2. 全 `FLOAT32LE` sample が finite かつ `[-1.0, 1.0]` であることを検証する。
3. `source_id` が初回または変化した場合、delay state を無音で初期化する。
4. channel ごとに 4 本の feedback delay line を更新する。
5. 各 delay line の出力を平均し、wet signal とする。
6. `dry_gain * input + wet_gain * wet` を出力 sample とする。
7. 出力 sample と次 state が finite かつ `[-1.0, 1.0]` の場合だけ state を commit して publish する。

## 2. Delay line

delay length は sample rate から固定 ms 値を sample 数へ変換する。

| line | base delay |
| --- | --- |
| 0 | 29.7 ms |
| 1 | 37.1 ms |
| 2 | 41.1 ms |
| 3 | 43.7 ms |

各 channel は独立した delay line 群を持つ。interleaved payload から channel sample を取り出し、同じ channel の state だけを更新する。

## 3. Feedback と damping

`room_size` は effective feedback gain を決める。

```text
feedback_gain = 0.20 + 0.65 * room_size
input_gain = 1.0 - feedback_gain
```

`damping` は delay output に対する one-pole low-pass feedback として使う。

```text
filtered = (1.0 - damping) * delayed + damping * previous_filter_state
next_state = input_gain * input + feedback_gain * filtered
```

`input`、`filtered` が `[-1, 1]` にある限り、`next_state` は convex combination になり、理論上 `[-1, 1]` に収まる。
実装では浮動小数演算の結果も再検証し、範囲外なら state を commit しない。

## 4. Normalization

wet signal は 4 本の delay output の算術平均である。

```text
wet = (delay_0 + delay_1 + delay_2 + delay_3) / 4
output = dry_gain * input + wet_gain * wet
```

config validation で `dry_gain + wet_gain <= 1.0` を要求するため、input と wet が正規化範囲にある限り output も正規化範囲に収まる。
それでも runtime では output を検証し、範囲外なら clamp せず drop する。

## 5. State commit

処理は transactional に行う。callback は既存 delay state を copy した `next_state` を更新し、frame 全体の処理が成功した場合だけ member state へ代入する。
途中で非 finite sample、範囲外 output、不正な delay state を検出した場合、その frame は publish せず、既存 state も更新しない。

## 6. Drop 条件

- config parameter 欠落または範囲外
- `AudioFrame` contract 不一致
- input sample が finite でない
- input sample が `[-1.0, 1.0]` の外
- delay state の channel 数または line 数が不正
- delay buffer が空
- output または next feedback state が finite でない
- output または next feedback state が `[-1.0, 1.0]` の外

drop は warning と `messages_dropped` counter で観測可能にする。
