# fa_time_stretch アルゴリズム詳細説明書

1. startup config で stretch ratio、frame length、hop length、window type、expected format を検証する。
2. input `AudioFrame` の identity と format metadata を検証する。
3. backend で phase vocoder または明示選択された algorithm を適用する。
4. time stretch 後の sample count が ratio から計算される expected range と一致することを検証する。
5. finite/normalized contract を検証し、契約を満たす場合だけ publish する。

pitch を保つことが contract なので、resample による duration change はこの node の責務外である。
入力長が algorithm contract と一致しない場合は hidden trim / padding をせず drop する。

## Backend Boundary

backend は time-scale modification のみを担当し、ROS2 topic/message を知らない。
timestamp compensation や streaming jitter absorption は `src/streaming` の専用 node が担当する。
