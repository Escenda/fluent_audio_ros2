# fa_crossfade 仕様書

## 1. 目的

`fa_crossfade` は2つの audio segment を、明示的な overlap 長と fade curve に従って接続する temporal processing node である。

初期実装では `AudioFrame` 1件を完全な segment として扱う。streaming chunk の境界推測や segment 欠損補完は行わない。

## 2. 入力

| 種別 | 名前 | 型 |
| --- | --- | --- |
| Subscribe | `input_a_topic` parameter で指定 | `fa_interfaces/msg/AudioFrame` |
| Subscribe | `input_b_topic` parameter で指定 | `fa_interfaces/msg/AudioFrame` |

各入力 topic は隣接 segment を1件ずつ受け取る。2つの segment は同じ `epoch` の pair として扱われる。

入力 segment は以下を満たす必要がある。

- `source_id` が空でない
- `input_a.stream_id == input_a_stream_id`
- `input_b.stream_id == input_b_stream_id`
- `input_a.epoch == input_b.epoch`
- `source_id`, sample rate, channels, encoding, bit depth, layout が両入力で一致
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `layout == interleaved`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `data` が空でなく、`channels * sizeof(float)` で割り切れる
- 全 sample が finite かつ `[-1.0, 1.0]`

## 3. 出力

出力は入力と同じ sample rate、channels、encoding、bit depth、layout を保持する。
`header`, `source_id`, `epoch` は `input_a` を保持し、`stream_id` は `output.stream_id` に更新する。`data` は `input_a` の prefix、crossfade overlap、`input_b` の suffix を連結した FLOAT32LE bytes になる。

ROS topic と stream identity は別概念であり、`input_a_stream_id` / `input_b_stream_id` / `output.stream_id` は `input_a_topic` / `input_b_topic` / `output_topic` と一致してはならない。

## 4. Parameters

すべて required であり、コード上に有効な runtime default はない。package launch の `config_file` も明示必須である。

| Parameter | 型 | 条件 |
| --- | --- | --- |
| `input_a_topic` | string | 非空、他 topic と distinct |
| `input_b_topic` | string | 非空、他 topic と distinct |
| `output_topic` | string | 非空、他 topic と distinct |
| `input_a_stream_id` | string | 非空、topic 名と一致不可 |
| `input_b_stream_id` | string | 非空、topic 名および他 stream と一致不可 |
| `output.stream_id` | string | 非空、topic 名および入力 stream と一致不可 |
| `crossfade.overlap_frames` | int | `> 0` |
| `crossfade.curve` | string | `linear` または `equal_power` |
| `expected.sample_rate` | int | `0 < value <= 384000` |
| `expected.channels` | int | `0 < value <= 64` |
| `expected.encoding` | string | `FLOAT32LE` |
| `expected.bit_depth` | int | `32` |
| `expected.layout` | string | `interleaved` |
| `qos.depth` | int | `> 0` |
| `qos.reliable` | bool | 明示指定 |
| `diagnostics.qos.depth` | int | `> 0` |
| `diagnostics.qos.reliable` | bool | 明示指定 |
| `diagnostics.publish_period_ms` | int | `> 0` |

## 5. 責務外

- device / file / network I/O
- resample / channel conversion / bit-depth conversion
- gain normalize / limiter
- routing mixer / bus routing
- missing segment の補完
- segment boundary 欠損時の推測 crossfade

## 6. fail closed 条件

- overlap length が未指定または不正
- fade curve が unsupported
- adjacent segment の format metadata mismatch
- adjacent segment の epoch mismatch
- segment boundary 欠損
- non-finite sample
- output が normalized range を超える
