# fa_crossfade アルゴリズム詳細説明書

1. startup config で overlap length、fade curve、expected format を検証する。
2. segment boundary と adjacent segment metadata を検証する。
3. overlap 区間に fade-out curve と fade-in curve を適用する。
4. overlap sample を加算し、finite/normalized contract を検証する。
5. pre-overlap、crossfade、post-overlap を連結して publish する。

boundary が欠損している場合に、無音検出や VAD で境界を推測しない。
crossfade 後に clipping が発生する場合も limiter を隠さず fail closed で drop する。

## Backend Boundary

backend は samples と curve parameter だけを扱い、ROS2 topic/message を知らない。
segment selection、topic subscription、diagnostics は node が担当する。
