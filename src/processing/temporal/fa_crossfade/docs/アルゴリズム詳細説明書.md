# fa_crossfade アルゴリズム詳細説明書

1. startup config で topics、stream ids、overlap length、fade curve、expected format を検証する。
2. `input_a_topic` と `input_b_topic` から完全な `AudioFrame` segment を受け取る。
3. node が segment metadata、`stream_id`、format、`epoch` pair を検証する。
4. backend が FLOAT32LE sample contract を検証する。
5. overlap 区間に fade-out curve と fade-in curve を適用する。
6. overlap sample を加算し、finite/normalized contract を検証する。
7. `input_a` prefix、crossfade overlap、`input_b` suffix を連結して publish する。

boundary が欠損している場合に、無音検出や VAD で境界を推測しない。
crossfade 後に clipping が発生する場合も limiter を隠さず fail closed で drop する。

## Fade Curves

overlap index `i` は `0 <= i < overlap_frames` とし、正規化位置は次で計算する。

```text
position = (i + 1) / (overlap_frames + 1)
```

`linear`:

```text
a_gain = 1.0 - position
b_gain = position
```

`equal_power`:

```text
a_gain = cos(position * pi / 2)
b_gain = sin(position * pi / 2)
```

## Backend Boundary

backend は2つの FLOAT32LE byte列、channel 数、overlap frames、curve parameter だけを扱い、ROS2 topic/message を知らない。
segment pair selection、topic subscription、diagnostics、`stream_id` 更新は node が担当する。
