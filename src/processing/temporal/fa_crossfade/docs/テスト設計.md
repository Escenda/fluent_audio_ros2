# fa_crossfade テスト設計

## Unit contract test

- `FA-CROSSFADE-SPEC-001`: package layout が標準構成を満たす。
- `FA-CROSSFADE-SPEC-002`: launch の `config_file` は明示必須で package default を持たない。
- `FA-CROSSFADE-SPEC-003`: example config が required parameters と FLOAT32LE interleaved contract を持つ。
- `FA-CROSSFADE-SPEC-004`: node が ROS topic と stream identity を別 parameter として扱う。
- `FA-CROSSFADE-SPEC-005`: node が2入力 segment を同一 `epoch` pair として検証する。
- `FA-CROSSFADE-SPEC-006`: backend は ROS-free で sample loop と curve を担当する。
- `FA-CROSSFADE-SPEC-007`: invalid input/output を clamp / normalize / limiter で隠さない。
- `FA-CROSSFADE-SPEC-008`: hidden limiter / boundary guessing / missing segment 補完を禁止する。

## Backend C++ test

- linear crossfade の deterministic output を確認する。
- equal-power curve が `cos/sin` gain を使うことを確認する。
- input byte alignment、短すぎる segment、NaN、normalized range violation を `ProcessStatus` で返す。
- output range violation を reject し、output buffer を上書きしない。
- unsupported fade curve / backend status message は fallback 文字列へ丸めず例外にする。

## C++ graph smoke test

- `FaCrossfadeNode` を `rclcpp::NodeOptions` で起動し、2つの input publisher と output subscriber を同じ executor に載せる。
- AudioFrame QoS と diagnostics QoS が明示 parameter で構成される。
- 誤った `input_a_stream_id` の frame と同一 epoch の `input_b` を送っても publish されない。
- epoch mismatch pair が publish されない。
- 同一 epoch かつ metadata 一致の pair だけが publish される。
- output frame は `source_id`、format metadata、`epoch` を保持し、`stream_id` を `output.stream_id` に更新する。

## Integration Test

ROS2 runtime 上で、同一 `epoch` の2つの deterministic segment fixture を crossfade し、metadata mismatch、epoch mismatch、clipping case が publish されないことを確認する。

## Launch Test

missing params file、unsupported fade curve、invalid overlap length で起動失敗することを確認する。
