# fa_pitch_shift アルゴリズム詳細説明書

1. startup config で pitch ratio、frame length、hop length、window type、expected format を検証する。
2. input `AudioFrame` の identity と format metadata を検証する。
3. backend で phase vocoder または明示選択された algorithm を適用する。
4. pitch shift 後の sample count が入力 frame count と一致することを検証する。
5. finite/normalized contract を検証し、契約を満たす場合だけ publish する。

duration を保つことが contract なので、出力長が変わった場合は hidden trim / pad せず drop する。
resample による pitch change はこの node の責務外である。

## Backend Boundary

backend は pitch transformation のみを担当し、ROS2 topic/message を知らない。
topic subscription、metadata validation、diagnostics は node が担当する。
