# fa_fade 仕様書

## 1. 目的

`fa_fade` は、FluentAudio の FLOAT32LE interleaved 音声フレームに対し、フレーム位置に基づく線形 fade-in または fade-out を適用する。

## 2. ノード

| 項目 | 値 |
| --- | --- |
| Package | `fa_fade` |
| Executable | `fa_fade_node` |
| Node | `fa_fade` |
| Subscribe | `input_topic` |
| Publish | `output_topic` |

## 3. 入出力契約

入力 `AudioFrame` は以下を満たす必要がある。

- `source_id` が空でない
- `stream_id` が空でない
- `stream_id == input_stream_id`
- `sample_rate > 0` かつ設定値と一致
- `channels > 0` かつ設定値と一致
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `layout == interleaved`
- `data` が空でない
- `data.size()` が `channels * sizeof(float)` で割り切れる
- 全サンプルが有限で `[-1.0, 1.0]` の範囲内

出力 `AudioFrame` は `header`, `source_id`, `sample_rate`, `channels`, `bit_depth`, `encoding`, `layout`, `epoch` を保持し、`stream_id` を `output.stream_id` に更新し、`data` を fade 済みサンプルへ置き換える。

## 4. パラメータ

`fa_fade_node` はすべてのパラメータを required として扱う。`fa_fade.launch.py` の `config_file` も必須 launch argument であり、package 内の設定ファイルを暗黙には読み込まない。`config/default.yaml` は debug / example 用の明示 config であり、runtime default ではない。

| 名前 | 型 | 設定例 | 制約 |
| --- | --- | --- | --- |
| `input_topic` | string | `fa_fade/input` | 空不可、出力 topic と resolved identity が異なる |
| `output_topic` | string | `fa_fade/output` | 空不可、入力 topic と resolved identity が異なる |
| `input_stream_id` | string | `audio/buffered/mic` | 空不可、raw / resolved ROS topic identity と異なる |
| `output.stream_id` | string | `audio/faded/mic` | 空不可、raw / resolved ROS topic identity と異なり、入力 stream id と異なる |
| `fade.mode` | string | `fade_in` | `fade_in` または `fade_out` |
| `fade.duration_frames` | int | `16000` | `> 0` |
| `fade.initial_position_frames` | int | `0` | `>= 0` |
| `expected.sample_rate` | int | `16000` | `0 < value <= 384000` |
| `expected.channels` | int | `1` | `0 < value <= 64` |
| `expected.encoding` | string | `FLOAT32LE` | `FLOAT32LE` のみ |
| `expected.bit_depth` | int | `32` | `32` のみ |
| `expected.layout` | string | `interleaved` | `interleaved` のみ |
| `qos.depth` | int | `10` | `> 0` |
| `qos.reliable` | bool | `false` | true/false |
| `diagnostics.publish_period_ms` | int | `1000` | `> 0` |
| `diagnostics.qos.depth` | int | `10` | `> 0` |
| `diagnostics.qos.reliable` | bool | `true` | true/false |

## 5. エラー処理

起動時設定が不正な場合は例外で起動を停止する。`input_topic == output_topic`、`input_stream_id == output.stream_id`、topic と stream id の同値は起動失敗する。ランタイムフレームが契約に違反した場合は warning を出し、そのフレームを drop する。サンプル値の clamp、normalize、暗黙変換は行わない。

## 6. Diagnostics

`diagnostics` topic に以下を含める。

- `mode`
- `duration_frames`
- `current_position_frames`
- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `expected_channels`

## 7. 非目標

device I/O、resampling、sample format conversion、channel conversion、gain、normalize、limiter、filter、denoise、legacy compatibility alias は提供しない。
