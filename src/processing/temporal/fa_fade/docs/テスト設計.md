# fa_fade テスト設計

## 1. 対象

`fa_fade` の package layout、設定契約、C++ node と ROS-free backend の責務境界、fade アルゴリズム、diagnostics、colcon pytest / gTest 登録を対象とする。

## 1.1. traceability

| test ID | 対応仕様 | 観点 |
| --- | --- | --- |
| `FA-FADE-TC-001` -> `FA-FADE-SPEC-001` | fade boundary | frame position に基づく linear fade-in / fade-out を適用すること。 |
| `FA-FADE-TC-002` -> `FA-FADE-SPEC-002` | topic / stream identity | topic と stream ID を分離し、衝突を fail closed すること。 |
| `FA-FADE-TC-003` -> `FA-FADE-SPEC-003` | required startup config | fade mode / duration / position、expected format、QoS、diagnostics parameter を required とすること。 |
| `FA-FADE-TC-004` -> `FA-FADE-SPEC-004` | runtime frame validation | source / stream identity、format、data alignment、sample range を検証すること。 |
| `FA-FADE-TC-005` -> `FA-FADE-SPEC-005` | backend boundary | fade gain 式と position state を ROS-free backend に閉じること。 |
| `FA-FADE-TC-006` -> `FA-FADE-SPEC-006` | output metadata | output が `stream_id` と `data` だけを更新すること。 |
| `FA-FADE-TC-007` -> `FA-FADE-SPEC-007` | no hidden correction | sample / output / position error を clamp / normalize で補正しないこと。 |
| `FA-FADE-TC-008` -> `FA-FADE-SPEC-008` | diagnostics | mode、duration、position、counter が diagnostics に含まれること。 |
| `FA-FADE-TC-009` -> `FA-FADE-SPEC-009` | non-goals | device I/O、format conversion、normalize、limiter、legacy alias を持たないこと。 |
| `FA-FADE-TC-010` -> `FA-FADE-SPEC-010` | launch contract | launch file が `node_name` / `config_file` だけを要求し、package default を推測しないこと。 |
| `FA-FADE-TC-011` -> `FA-FADE-SPEC-011` | colcon coverage | pytest / backend gTest と lint dependency が colcon manifest 上で成立すること。 |

## 2. 単体 contract test

`test/unit/test_fa_fade_audio_frame_contract.py` で以下を確認する。

- example config が指定された topic / stream identity、fade parameter、FLOAT32LE interleaved 契約を持つ
- package layout が標準構造と一致する
- 起動時パラメータが fail closed で範囲検証される
- runtime frame が source/stream/format/data size を検証される
- `stream_id == input_stream_id` を要求し、出力で `stream_id = output.stream_id` に更新する
- fade-in/fade-out の gain 式と position state が backend にある
- サンプル範囲外、非有限入力、非有限出力、position overflow を drop する
- device I/O、resampling、format conversion、channel conversion、normalize、limiter、filter、denoise を含まない
- diagnostics に mode、duration、current position、stream identity、counters が含まれる
- `ament_cmake_pytest` と `ament_cmake_gtest` で colcon から contract test が実行される

## 3. 統合テスト

現時点では統合テストは `.gitkeep` のみ配置する。ROS2 runtime を使う publish/subscribe 統合テストは、fa_interfaces と実行環境の fixture が整った段階で追加する。
