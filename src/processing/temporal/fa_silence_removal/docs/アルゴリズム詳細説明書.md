# fa_silence_removal アルゴリズム詳細説明書

## 1. 入力単位

処理単位は 1 つの `AudioFrame` message である。この仕様書では、この message を chunk と呼ぶ。

`fa_silence_removal` は chunk を分割しない。non-silent / silent の判定結果は chunk 全体に適用する。

ROS node は `AudioFrame` の契約検証、topic 入出力、diagnostics、出力 `stream_id` 更新を担当する。RMS 計算と hangover state は ROS-free backend `internal_rms_silence_removal` が担当する。

## 2. 前提条件

入力は以下の形式に固定する。

- `encoding=FLOAT32LE`
- `bit_depth=32`
- `layout=interleaved`
- `stream_id == input_stream_id`
- `channels == expected.channels`
- `sample_rate == expected.sample_rate`

この前提に合わない frame は drop する。別形式への変換、channel 数の推定、byte order の推定、stream identity の推定は行わない。

## 3. RMS 計算

`data` を `float` sample 配列として 4 byte ずつ読み出す。channel ごとの RMS ではなく、chunk 内の全 sample を対象に 1 つの RMS を計算する。

```text
sum_squares = Σ(sample_i * sample_i)
rms = sqrt(sum_squares / sample_count)
```

各 sample は `[-1.0, 1.0]` の正規化 FLOAT32 として扱う。以下の sample を含む chunk は invalid として drop する。

- NaN
- inf / -inf
- `-1.0` 未満
- `1.0` より大きい

invalid sample を clamp したり、0 に置き換えたり、normalize して継続したりしない。

## 4. publish / drop 判定

判定順序は以下の通り。

1. node が frame contract を検証する
2. backend が sample contract を検証しながら RMS を計算する
3. backend が `rms >= threshold.rms` の場合に `accepted_active` を返す
4. backend が `rms < threshold.rms` かつ hangover 残量がある場合に `accepted_hangover` を返す
5. backend が `rms < threshold.rms` かつ hangover 残量がない場合に `dropped_silent` を返す
6. node が accepted decision だけを publish する

publish する場合は入力 frame をコピーし、`stream_id` のみ `output.stream_id` に更新する。`data` は書き換えない。

## 5. hangover 更新

`hangover_ms` は以下で sample 数へ変換する。

```text
hangover_samples = ceil(hangover_ms * expected.sample_rate / 1000.0)
```

non-silent chunk が publish されるたびに、`hangover_samples_remaining` は `hangover_samples` にリセットされる。

silent chunk が hangover によって publish された場合は、chunk の frame 数を `hangover_samples_remaining` から引く。frame 数は以下で計算する。

```text
frame_count = data.size() / (expected.channels * sizeof(float))
```

残量より大きい chunk を publish した場合、残量は 0 にする。chunk 内部で切って前半だけ publish することはしない。

## 6. 状態

保持する処理状態は以下のみである。

- backend: `hangover_samples`, `hangover_samples_remaining`, `last_rms`
- node: diagnostics counters

source ごとの履歴や external VAD state は持たない。必要になった場合は別 package または backend として設計する。

## 7. 数値境界

RMS は double で計算する。入力 sample は正規化範囲に限定されるため、`sum_squares / sample_count` は `[0.0, 1.0]` に収まる。計算結果が有限でない、または `[0.0, 1.0]` を外れた場合は invalid chunk として drop する。

backend は invalid input を受けた場合に `last_rms` や hangover state を更新しない。これにより壊れた frame が以後の silent 判定へ影響しない。

## 8. fail closed

backend の `Decision` と `ProcessStatus` は switch で全列挙子を明示的に処理する。未知の列挙値が発生した場合は `std::logic_error` を投げ、silent / accepted のどちらかに丸めない。
