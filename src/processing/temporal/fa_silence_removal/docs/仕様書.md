# fa_silence_removal 仕様書

## 1. 目的

`fa_silence_removal` は、FluentAudio の FLOAT32LE interleaved 音声 chunk に対し、RMS 閾値で silent / non-silent を判定し、silent chunk を出力 stream から取り除く。

このパッケージは sample 値を書き換えない。silent chunk に対して zeroing、gain gate、noise gate を適用するのではなく、publish しないことで取り除く。

## 2. ノード

| 項目 | 値 |
| --- | --- |
| Package | `fa_silence_removal` |
| Executable | `fa_silence_removal_node` |
| Node | `fa_silence_removal` |
| Subscribe | `input_topic` |
| Publish | `output_topic` |

## 3. 入出力契約

入力 `AudioFrame` は以下を満たす必要がある。

- `source_id` が空でない
- `stream_id` が空でない
- `stream_id == input_stream_id`
- `sample_rate > 0` かつ設定値と一致
- `channels > 0` かつ設定値と一致
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `layout == interleaved`
- `data` が空でない
- `data.size()` が `channels * sizeof(float)` で割り切れる
- 全サンプルが有限で `[-1.0, 1.0]` の範囲内

出力 `AudioFrame` は `header`, `source_id`, `sample_rate`, `channels`, `bit_depth`, `encoding`, `layout`, `data`, `epoch` を保持し、`stream_id` だけを `output.stream_id` に更新する。ROS topic と stream identity は別概念であり、`input_stream_id` / `output.stream_id` は `input_topic` / `output_topic` と一致してはならない。

## 4. silent 判定

`data` 全体を FLOAT32LE sample 列として読み、全 channel を含めた chunk RMS を計算する。

```text
rms = sqrt(sum(sample^2) / sample_count)
```

- `rms >= threshold.rms`: non-silent として publish
- `rms < threshold.rms`: silent として drop。ただし hangover 中は publish

`threshold.rms` は正規化 FLOAT32 の RMS 閾値であり、有限かつ `[0.0, 1.0]` の範囲に限る。

## 5. hangover

`hangover_ms` は `expected.sample_rate` を使って sample 数へ変換する。activity chunk が publish された直後に hangover 残量を `hangover_samples` に設定する。

silent chunk 受信時に hangover 残量がある場合、その chunk は publish し、chunk の frame 数だけ残量を減らす。残量が 0 になった後の silent chunk は publish しない。

このパッケージは chunk を分割しない。したがって hangover の終端は chunk 境界に丸められ、最後に publish される silent chunk が設定時間を最大 1 chunk 分だけ超える場合がある。これは silent chunk removal の粒度仕様であり、欠損値補完や fallback ではない。

## 6. パラメータ

package launch の `config_file` は必須であり、package 内 default file を暗黙に読み込まない。以下の値は `config/default.yaml` の設定例であって、node 内 runtime default ではない。node は全パラメータを必須として読み、未設定または型不一致を起動失敗にする。

| 名前 | 型 | 設定例 | 制約 |
| --- | --- | --- | --- |
| `input_topic` | string | `fa_silence_removal/input` | 空不可。`output_topic` と resolved topic が異なる |
| `output_topic` | string | `fa_silence_removal/output` | 空不可。`input_topic` と resolved topic が異なる |
| `input_stream_id` | string | `audio/buffered/mic` | 空不可。topic 名と一致不可 |
| `output.stream_id` | string | `audio/silence_removed/mic` | 空不可。topic 名および `input_stream_id` と一致不可 |
| `threshold.rms` | double | `0.02` | finite かつ `[0.0, 1.0]` |
| `hangover_ms` | double | `200.0` | finite かつ `>= 0` |
| `expected.sample_rate` | int | `16000` | `0 < value <= 384000` |
| `expected.channels` | int | `1` | `0 < value <= 64` |
| `expected.encoding` | string | `FLOAT32LE` | `FLOAT32LE` のみ |
| `expected.bit_depth` | int | `32` | `32` のみ |
| `expected.layout` | string | `interleaved` | `interleaved` のみ |
| `qos.depth` | int | `10` | `> 0` |
| `qos.reliable` | bool | `false` | true/false |
| `diagnostics.publish_period_ms` | int | `1000` | `> 0` |

## 7. エラー処理

起動時設定が不正な場合は例外で起動を停止する。ランタイムフレームが契約に違反した場合、または sample が有限な正規化 FLOAT32 でない場合は warning を出し、その frame を drop する。

不正 sample に対する clamp、normalize、zero 補完、format 推定、stream_id 推定は行わない。

backend が返す `Decision` / `ProcessStatus` は全列挙子を明示的に扱う。未知の列挙値は正常系として解釈せず、`std::logic_error` で fail closed する。

`diagnostics` topic に以下を含める。

- `threshold_rms`
- `hangover_ms`
- `hangover_samples`
- `hangover_samples_remaining`
- `last_rms`
- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `expected_sample_rate`
- `expected_channels`

- `messages_out`
- `messages_dropped`
- `invalid_frames_dropped`
- `silent_frames_dropped`
- `hangover_frames`
- `active_frames`

## 9. 非目標

device I/O、resampling、sample format conversion、channel conversion、gain、noise gate、zeroing、normalize、limiter、filter、denoise、VAD state publication、ASR、legacy compatibility alias は提供しない。

## 10. backend 境界

RMS 計算と hangover state は ROS-free C++ backend `internal_rms_silence_removal` に閉じる。backend は `fa_interfaces/msg/AudioFrame`、ROS topic、diagnostics、publisher/subscriber を知らない。ROS node は frame contract、topic subscription/publication、diagnostics、`stream_id` 更新を担当する。
