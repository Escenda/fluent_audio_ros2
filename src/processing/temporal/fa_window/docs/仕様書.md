# fa_window 仕様書

## 目的

`fa_window` は、FLOAT32LE interleaved の `AudioFrame` に per-frame の Hann または Hamming 解析窓を適用する。

## 仕様 ID

| ID | 内容 |
| --- | --- |
| `FA-WINDOW-SPEC-001` | `fa_window` は `FLOAT32LE` interleaved `AudioFrame` に per-frame の Hann または Hamming 解析窓を適用する temporal processing node である。 |
| `FA-WINDOW-SPEC-002` | device I/O、resampling、sample format conversion、channel conversion、gain、normalize、limiter、filter、denoise を行わない。 |
| `FA-WINDOW-SPEC-003` | `input_topic` / `output_topic` と `input_stream_id` / `output.stream_id` は別契約であり、topic identity と stream identity の衝突を起動時に拒否する。 |
| `FA-WINDOW-SPEC-004` | window type / expected frames / strict frame count、expected format、QoS、diagnostics parameter は required で、不正値は fail closed する。 |
| `FA-WINDOW-SPEC-005` | runtime frame は source / stream identity、FLOAT32LE format、data alignment、finite normalized sample を検証する。 |
| `FA-WINDOW-SPEC-006` | `strict_frame_count=true` では frame count 一致を必須にし、false では実入力 frame count で係数を計算する明示仕様とする。 |
| `FA-WINDOW-SPEC-007` | frame count policy、sample validation、Hann/Hamming window 適用は ROS-free backend に閉じる。 |
| `FA-WINDOW-SPEC-008` | output metadata は入力を保持し、`stream_id` と `data` だけを更新する。 |
| `FA-WINDOW-SPEC-009` | unknown `WindowType` / `ProcessStatus` は `std::logic_error` で fail closed する。 |
| `FA-WINDOW-SPEC-010` | diagnostics は window type、expected frames、strict policy、last frame count、stream identity、counter を publish する。 |
| `FA-WINDOW-SPEC-011` | launch file は `node_name` / `config_file` を明示引数とし、package-local default config を推測しない。 |
| `FA-WINDOW-SPEC-012` | pytest / backend gTest contract は `colcon test` から実行され、`ament_lint_auto` dependency を package manifest に明示する。 |

## 非目標

device I/O、resampling、sample format conversion、channel conversion、gain、normalize、limiter、filter、denoise は行わない。

| 項目 | 内容 |

| package | `fa_window` |

## 入力契約

入力 `AudioFrame` は以下を満たす必要がある。

- `source_id` が空でない
- `stream_id` が空でなく、`input_stream_id` と一致する
- `sample_rate > 0` かつ設定値と一致する
- `channels > 0` かつ設定値と一致する
- `encoding == "FLOAT32LE"`
- `bit_depth == 32`
- `layout == "interleaved"`
- `data` が空でなく、`channels * sizeof(float)` で割り切れる
- sample は finite かつ normalized range `[-1, 1]` に収まる

契約違反の runtime frame は warning を出して破棄する。

## 出力契約

出力 `AudioFrame` は入力から以下を保持する。

- `header`
- `source_id`
- `sample_rate`
- `channels`
- `bit_depth`
- `encoding`
- `layout`
- `epoch`

`stream_id` は `output.stream_id` に更新し、`data` は window 適用済み FLOAT32LE interleaved bytes に更新する。

## Startup Validation

以下の設定不備は起動時に例外で fail closed する。

- `input_topic` / `output_topic` が空
- `input_topic` と `output_topic` の resolved identity が同一
- `input_stream_id` / `output.stream_id` が空
- `input_stream_id` / `output.stream_id` が raw / resolved ROS topic identity と同一
- `input_stream_id` と `output.stream_id` が同一
- `window.type` が `hann` / `hamming` 以外
- `window.expected_frames <= 1`
- `expected.sample_rate <= 0` または `> 384000`
- `expected.channels <= 0` または `> 64`
- `expected.encoding != "FLOAT32LE"`
- `expected.bit_depth != 32`
- `expected.layout != "interleaved"`
- `qos.depth <= 0`
- `diagnostics.publish_period_ms <= 0`
- `diagnostics.qos.depth <= 0`

## Frame Count

`window.strict_frame_count=true` の場合、入力 frame count は `window.expected_frames` と一致する必要がある。一致しない frame は破棄する。

`window.strict_frame_count=false` の場合、実際の入力 frame count で係数を計算する。ただし `frame_count > 1` は必須であり、満たさない frame は破棄する。この挙動は明示設定であり fallback ではない。

## Diagnostics

diagnostics は以下を含む。

- `window_type`
- `expected_frames`
- `strict_frame_count`
- `last_frame_count`
- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `expected_channels`

## Backend

`internal_window_function` backend は ROS2 topic/message を知らず、FLOAT32LE interleaved byte列の frame count policy、sample validation、Hann/Hamming window 適用のみを担当する。未知の `WindowType` / `ProcessStatus` は `std::logic_error` で fail closed する。
