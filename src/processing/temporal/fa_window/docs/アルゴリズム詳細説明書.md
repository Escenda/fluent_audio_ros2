# fa_window アルゴリズム詳細説明書

## 入力配列

ROS node は `AudioFrame` の topic/message/diagnostics 境界だけを扱う。`AudioFrame.data` は ROS-free backend `internal_window_function` に FLOAT32LE interleaved bytes として渡す。1 frame の byte 数は `channels * sizeof(float)` である。

## 係数

frame count を `N`、frame index を `n` とする。`N > 1` が必須である。

Hann:

```text
w[n] = 0.5 * (1 - cos(2 * pi * n / (N - 1)))
```

Hamming:

```text
w[n] = 0.54 - 0.46 * cos(2 * pi * n / (N - 1))
```

## 適用

sample index `i` の frame index は `i / channels` である。同じ frame index に属する全 channel に同じ係数を掛ける。

```text
out[i] = in[i] * w[i / channels]
```

入力 sample と出力 sample は finite かつ `[-1, 1]` に収まる必要がある。範囲外または非 finite の sample を含む frame は破棄する。clamp や normalize は行わない。
backend は全 sample の検証と変換が成功した場合だけ output payload を commit する。未知の `WindowType` / `ProcessStatus` は `std::logic_error` で fail closed する。
