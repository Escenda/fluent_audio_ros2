# fa_window テスト設計

## Unit Contract Tests

`test/unit/test_fa_window_audio_frame_contract.py` は、実行環境に ROS2 node を起動せず、config と source を検査して package contract を固定する。

確認項目:

- example config が topic / stream identity と FLOAT32LE interleaved 契約を表す
- window が device I/O、resampling、format conversion、channel conversion、gain、limiter、filter、denoise を持たない
- 起動時 validation が fail closed になっている
- runtime frame validation が source/stream identity、format、data size を確認する
- strict frame count の true/false 挙動、Hann/Hamming 係数式、invalid sample drop を backend が担当する
- stream identity が `output.stream_id` に更新される
- diagnostics に設定値、stream identity、last frame count、counter が含まれる
- colcon pytest / backend gTest registration が存在する

## Integration / Launch

`test/integration` と `test/launch` は将来の ROS2 graph test 用の配置を確保する。現時点では `.gitkeep` のみを置く。
