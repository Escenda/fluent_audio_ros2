# fa_trim 仕様書

## 1. 目的

`fa_trim` は `AudioFrame` ごとに先頭と末尾の sample frame を削除する時間軸編集 node である。
固定長 window の端を落とす、前後に付いた guard sample を削除する、短い synthetic fixture を整形するなど、
frame 内で完結する trim を扱う。

## 2. 非目標

- resample、bit depth 変換、channel 変換
- silence removal、VAD、fade、windowing
- device / file / network I/O
- 複数 frame をまたぐ buffering
- 欠損 frame を補う fallback

## 3. 所属分類

`processing/temporal`

## 4. Topic

| direction | type | parameter |
| --- | --- | --- |
| subscribe | `fa_interfaces/msg/AudioFrame` | `input_topic` |
| publish | `fa_interfaces/msg/AudioFrame` | `output_topic` |
| publish | `diagnostic_msgs/msg/DiagnosticArray` | fixed `diagnostics` |

## 5. Message Contract

| ID | 仕様 |
| --- | --- |
| FA-TRIM-SPEC-001 | 入力 `AudioFrame` は `source_id` と `stream_id` を必須とする。 |
| FA-TRIM-SPEC-002 | 入力 `stream_id` は `input_stream_id` と一致しなければならない。 |
| FA-TRIM-SPEC-003 | 入力 format は `expected.sample_rate`、`expected.channels`、`FLOAT32LE`、32bit、`interleaved` と一致しなければならない。 |
| FA-TRIM-SPEC-004 | `data` は空でなく、`channels * sizeof(float)` の倍数でなければならない。 |
| FA-TRIM-SPEC-005 | 出力 `stream_id` は `output.stream_id` に更新する。`source_id`、`header`、`encoding`、`sample_rate`、`channels`、`bit_depth`、`layout` は入力を継承する。 |
| FA-TRIM-SPEC-006 | 出力 `epoch` は `input.epoch + 1` とする。`UINT32_MAX` の入力は wrap を避けるため drop する。 |
| FA-TRIM-SPEC-007 | trim 後に sample frame が 0 個になる frame は publish せず、diagnostics counter に記録する。 |
| FA-TRIM-SPEC-008 | `FLOAT32LE` sample が finite でなく、または normalized range `[-1.0, 1.0]` 外の場合は drop する。 |
| FA-TRIM-SPEC-009 | `input_topic` / `output_topic` は ROS topic identity、`input_stream_id` / `output.stream_id` は audio stream identity として分離する。raw / resolved topic と stream id が同一の場合は起動失敗する。 |

`header` は入力 frame の capture boundary として保持する。先頭 sample の時刻へ再解釈しない。
sample-accurate retimestamp が必要な pipeline では、`fa_latency_compensation` など timestamp 専用 node を明示的に挟む。

## 6. Parameters

すべて required parameter として宣言し、コード側に有効な runtime default は持たない。`fa_trim.launch.py` の `config_file` も必須 launch argument であり、package 内の設定ファイルを暗黙には読み込まない。

| parameter | type | validation |
| --- | --- | --- |
| `input_topic` | string | non-empty、出力 topic と resolved identity が異なる |
| `output_topic` | string | non-empty、入力 topic と resolved identity が異なる |
| `input_stream_id` | string | non-empty、raw / resolved ROS topic identity と異なる |
| `output.stream_id` | string | non-empty、raw / resolved ROS topic identity と異なり、入力 stream id と異なる |
| `trim.leading_frames` | int | `>= 0` |
| `trim.trailing_frames` | int | `>= 0` |
| `expected.sample_rate` | int | `0 < value <= 384000` |
| `expected.channels` | int | `0 < value <= 64` |
| `expected.encoding` | string | `FLOAT32LE` |
| `expected.bit_depth` | int | `32` |
| `expected.layout` | string | `interleaved` |
| `qos.depth` | int | `> 0` |
| `qos.reliable` | bool | required |
| `diagnostics.publish_period_ms` | int | `> 0` |

`trim.leading_frames` と `trim.trailing_frames` が両方 0 の設定は no-op になるため起動失敗とする。

## 7. Lifecycle / State

通常の `rclcpp::Node` として起動する。node 内に frame 間 state は持たない。
起動時 parameter validation に失敗した場合は例外で fail closed する。

## 8. Diagnostics

`diagnostics` に以下を publish する。

- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `leading_frames`
- `trailing_frames`
- `expected_channels`
- `last_input_frame_count`
- `last_output_frame_count`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `contract_drops`
- `invalid_sample_drops`
- `trim_exhausted_drops`
- `epoch_overflow_drops`

## 9. Performance

処理は入力 frame の contract validation と、残す byte range の copy のみである。
frame 間 buffering は行わない。

## 10. Backend

`internal_frame_trim` backend は ROS2 topic/message を知らず、FLOAT32LE interleaved byte列の sample validation と byte range trim のみを担当する。詳細は `docs/backends/internal_frame_trim.md` を参照する。
