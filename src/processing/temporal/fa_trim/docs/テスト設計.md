# fa_trim テスト設計

## 1. 方針

`fa_trim` の unit test は package contract を pytest で検証する。
C++ node の起動・ROS2 pub/sub 結合は launch / integration test に分けるが、初期実装では `.gitkeep` のみ置く。

## 2. 仕様とテストケース

| Test ID | Spec ID | 内容 | 実装 |
| --- | --- | --- | --- |
| FA-TRIM-TC-001 | FA-TRIM-SPEC-001..004 | default config が FLOAT32LE interleaved 契約を満たす | `test_default_config_requires_float32_interleaved_contract` |
| FA-TRIM-TC-002 | FA-TRIM-SPEC-001..004 | required parameter が runtime default なしで宣言され、範囲検証される | `test_parameters_are_required_without_runtime_defaults_and_range_checked` |
| FA-TRIM-TC-003 | FA-TRIM-SPEC-001..004 | frame contract を処理前に検証する | `test_runtime_validates_audio_frame_contract_before_trimming` |
| FA-TRIM-TC-004 | FA-TRIM-SPEC-008 | invalid FLOAT32 sample を clamp せず drop する | `test_invalid_samples_are_dropped_without_clamping_or_normalizing` |
| FA-TRIM-TC-005 | FA-TRIM-SPEC-005..007 | trim 後の metadata、epoch、payload range を検証する | `test_trim_preserves_declared_metadata_updates_stream_epoch_and_payload` |
| FA-TRIM-TC-006 | FA-TRIM-SPEC-007 | sample frame が残らない場合は empty publish しない | `test_trim_drops_frames_that_would_be_empty` |
| FA-TRIM-TC-007 | FA-TRIM-SPEC-006 | epoch wrap を drop する | `test_epoch_increment_wrap_is_dropped` |
| FA-TRIM-TC-008 | FA-TRIM-SPEC-007 | diagnostics に trim/drop counter が含まれる | `test_diagnostics_include_trim_policy_and_drop_counters` |
| FA-TRIM-TC-009 | package layout | 標準 layout と `.gitkeep` を検証する | `test_package_layout_matches_required_processing_layout` |
| FA-TRIM-TC-010 | build contract | colcon から pytest contract が実行される | `test_colcon_runs_pytest_contracts_and_lint_dependencies` |

## 3. 未実装テスト

- launch test: required parameter 欠落時に process が fail closed することを確認する。
