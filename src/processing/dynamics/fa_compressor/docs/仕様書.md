# fa_compressor 仕様書

## 1. 目的

`fa_compressor` は、正規化済み FLOAT32LE interleaved `AudioFrame` stream に対し、静的な per-sample dynamic range compression を行う。

## 2. Package / Executable / Node

| 項目 | 値 |
| --- | --- |
| package | `fa_compressor` |
| executable | `fa_compressor_node` |
| node default name | `fa_compressor` |
| node core | `fa_compressor_node_core` |

## 3. 入出力

| 種別 | パラメータ | Message |
| --- | --- | --- |
| subscribe | `input_topic` | `fa_interfaces/msg/AudioFrame` |
| publish | `output_topic` | `fa_interfaces/msg/AudioFrame` |

## 4. AudioFrame 契約

入力 frame は以下を満たす必要がある。

| Field | 条件 |
| --- | --- |
| `source_id` | 非空 |
| `stream_id` | 非空、かつ `input_topic` と一致 |
| `sample_rate` | `expected.sample_rate` と一致、かつ `> 0` |
| `channels` | `expected.channels` と一致、かつ `> 0` |
| `encoding` | `FLOAT32LE` |
| `bit_depth` | `32` |
| `layout` | `interleaved` |
| `data` | 非空、かつ `channels * sizeof(float)` の倍数 |

入力 sample は finite かつ `[-1.0, 1.0]` の正規化範囲内でなければならない。出力 sample が finite でない、または正規化範囲を超える場合、その frame は drop する。clamp や limiter による補正は行わない。

## 5. Parameter

| Parameter | Default | 条件 |
| --- | ---: | --- |
| `input_topic` | `audio/normalized/mic` | 非空 |
| `output_topic` | `audio/compressed/mic` | 非空 |
| `compressor.threshold_linear` | `0.5` | finite、`> 0.0`、`< 1.0` |
| `compressor.ratio` | `4.0` | finite、`> 1.0` |
| `compressor.makeup_gain_linear` | `1.0` | finite、`> 0.0`、`<= 4.0` |
| `expected.sample_rate` | `16000` | `> 0` |
| `expected.channels` | `1` | `> 0` |
| `expected.encoding` | `FLOAT32LE` | 固定 |
| `expected.bit_depth` | `32` | 固定 |
| `expected.layout` | `interleaved` | 固定 |
| `qos.depth` | `10` | `> 0` |
| `qos.reliable` | `false` | bool |
| `diagnostics.publish_period_ms` | `1000` | `> 0` |

## 6. 出力 metadata

出力 frame は入力 frame を基準に以下を保持・更新する。

| Field | 扱い |
| --- | --- |
| `header` | 入力を保持 |
| `source_id` | 入力を保持 |
| `sample_rate` | 入力を保持 |
| `channels` | 入力を保持 |
| `bit_depth` | 入力を保持 |
| `encoding` | 入力を保持 |
| `layout` | 入力を保持 |
| `epoch` | 入力を保持 |
| `stream_id` | `output_topic` に更新 |
| `data` | 圧縮後 FLOAT32LE sample bytes |

## 7. Diagnostics

`diagnostics` topic に `diagnostic_msgs/msg/DiagnosticArray` を publish する。status values には少なくとも以下を含める。

- `threshold_linear`
- `ratio`
- `makeup_gain_linear`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `samples_compressed`
- `output_topic`

## 8. 失敗時動作

- 起動時設定不正: 例外により node 起動を失敗させる。
- null `AudioFrame` pointer: warning を出して frame を drop する。
- runtime frame 契約違反: warning を出して frame を drop する。
- 入力 sample 不正: warning を出して frame を drop する。
- 出力 sample 範囲超過: warning を出して frame を drop する。

意味を変える fallback、legacy alias、暗黙の補正は行わない。

## 9. Launch Contract

- `launch/fa_compressor.launch.py` は `node_name` と `config_file` だけを launch argument として公開する。
- `compressor.*`、`expected.*`、QoS、diagnostics は config file で指定する。
- `FaCompressorNode` は `rclcpp::NodeOptions` を受け取り、ROS graph test は node core を直接生成して parameter override できる。
- `compressor.threshold_linear <= 0.0`、`compressor.ratio <= 1.0`、`compressor.makeup_gain_linear > 4.0`、format contract 不一致は launch 経路でも fail closed する。

## 10. Traceability

- `FA-COMPRESSOR-SPEC-001`: `fa_compressor` は device I/O や unrelated processing を行わない。
- `FA-COMPRESSOR-SPEC-002`: `fa_compressor` は `FLOAT32LE` / 32bit / interleaved のみ扱う。
- `FA-COMPRESSOR-SPEC-003`: 起動時 config は範囲検証し、不正値は fail closed する。
- `FA-COMPRESSOR-SPEC-004`: threshold 超過 sample は static curve で圧縮する。
- `FA-COMPRESSOR-SPEC-005`: 出力 `source_id` は保持し、`stream_id` は `output_topic` にする。
- `FA-COMPRESSOR-SPEC-006`: 入力 frame の `stream_id` は `input_topic` と一致していなければならない。
- `FA-COMPRESSOR-SPEC-007`: null frame pointer は drop する。
- `FA-COMPRESSOR-SPEC-008`: launch は `node_name` / `config_file` だけを公開し、node 固有 parameter は config file に閉じる。
- `FA-COMPRESSOR-SPEC-009`: ROS graph 経路で valid frame は publish され、stream mismatch frame は publish されない。
