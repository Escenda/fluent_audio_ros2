# fa_compressor 仕様書

## 1. 目的

`fa_compressor` は、正規化済み FLOAT32LE interleaved `AudioFrame` stream に対し、静的な per-sample dynamic range compression を行う。

## 2. Package / Executable / Node

| 項目 | 値 |
| --- | --- |
| package | `fa_compressor` |
| executable | `fa_compressor_node` |
| node canonical name | `fa_compressor` |
| node core | `fa_compressor_node_core` |

## 3. 入出力

| 種別 | パラメータ | Message |
| --- | --- | --- |
| subscribe | `input_topic` | `fa_interfaces/msg/AudioFrame` |
| publish | `output_topic` | `fa_interfaces/msg/AudioFrame` |

ROS topic 名は転送経路、`AudioFrame.stream_id` は音声 stream identity として扱う。両者を同一視しない。`input_topic` / `output_topic` と `input_stream_id` / `output.stream_id` はそれぞれ非空かつ互いに衝突してはならない。

## 4. AudioFrame 契約

入力 frame は以下を満たす必要がある。

| Field | 条件 |
| --- | --- |
| `source_id` | 非空 |
| `stream_id` | 非空、かつ `input_stream_id` と一致 |
| `sample_rate` | `expected.sample_rate` と一致、かつ `> 0` |
| `channels` | `expected.channels` と一致、かつ `> 0` |
| `encoding` | `FLOAT32LE` |
| `bit_depth` | `32` |
| `layout` | `interleaved` |
| `data` | 非空、かつ `channels * sizeof(float)` の倍数 |

入力 sample は finite かつ `[-1.0, 1.0]` の正規化範囲内でなければならない。出力 sample が finite でない、または正規化範囲を超える場合、その frame は drop する。clamp や limiter による補正は行わない。

## 5. Parameter

全 parameter は launch で渡された config file から明示設定する。node 内の暗黙値や package-provided config fallback は持たない。

| Parameter | 必須 | 条件 |
| --- | --- | --- |
| `input_topic` | yes | 非空、`output_topic` と別の ROS topic に解決される |
| `output_topic` | yes | 非空、`input_topic` と別の ROS topic に解決される |
| `input_stream_id` | yes | 非空、ROS topic 名と衝突しない、`output.stream_id` と別 |
| `output.stream_id` | yes | 非空、ROS topic 名と衝突しない、`input_stream_id` と別 |
| `compressor.threshold_linear` | yes | finite、`> 0.0`、`< 1.0` |
| `compressor.ratio` | yes | finite、`> 1.0` |
| `compressor.makeup_gain_linear` | yes | finite、`> 0.0`、`<= 4.0` |
| `expected.sample_rate` | yes | `0 < value <= 384000` |
| `expected.channels` | yes | `0 < value <= 64` |
| `expected.encoding` | yes | `FLOAT32LE` |
| `expected.bit_depth` | yes | `32` |
| `expected.layout` | yes | `interleaved` |
| `qos.depth` | yes | `> 0` |
| `qos.reliable` | yes | bool |
| `diagnostics.qos.depth` | yes | `> 0` |
| `diagnostics.qos.reliable` | yes | bool |
| `diagnostics.publish_period_ms` | yes | `> 0` |

## 6. 出力 metadata

出力 frame は入力 frame を基準に以下を保持・更新する。

| Field | 扱い |
| --- | --- |
| `header` | 入力を保持 |
| `source_id` | 入力を保持 |
| `sample_rate` | 入力を保持 |
| `channels` | 入力を保持 |
| `bit_depth` | 入力を保持 |
| `encoding` | 入力を保持 |
| `layout` | 入力を保持 |
| `epoch` | 入力を保持 |
| `stream_id` | `output.stream_id` に更新 |
| `data` | 圧縮後 FLOAT32LE sample bytes |

## 7. Diagnostics

`diagnostics` topic に `diagnostic_msgs/msg/DiagnosticArray` を publish する。status values には少なくとも以下を含める。

- `threshold_linear`
- `ratio`
- `makeup_gain_linear`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `samples_compressed`
- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `qos.depth`
- `diagnostics.qos.depth`
- `diagnostics.qos.reliable`

## 8. 失敗時動作

- 起動時設定不正: 例外により node 起動を失敗させる。
- null `AudioFrame` pointer: warning を出して frame を drop する。
- runtime frame 契約違反: warning を出して frame を drop する。
- 入力 sample 不正: warning を出して frame を drop する。
- 出力 sample 範囲超過: warning を出して frame を drop する。

意味を変える fallback、legacy alias、暗黙の補正は行わない。

## 9. Launch Contract

- `launch/fa_compressor.launch.py` は `node_name` と `config_file` だけを launch argument として公開する。
- `node_name` と `config_file` に暗黙値を持たせない。呼び出し側が必ず明示する。
- package share 内の config file を暗黙に探す fallback は持たない。
- `compressor.*`、`expected.*`、QoS、diagnostics は config file で指定する。
- `FaCompressorNode` は `rclcpp::NodeOptions` を受け取り、ROS graph test は node core を直接生成して parameter override できる。
- `compressor.threshold_linear <= 0.0`、`compressor.ratio <= 1.0`、`compressor.makeup_gain_linear > 4.0`、format contract 不一致は launch 経路でも fail closed する。

## 10. Traceability

- `FA-COMPRESSOR-SPEC-001`: `fa_compressor` は device I/O や unrelated processing を行わない。
- `FA-COMPRESSOR-SPEC-002`: `fa_compressor` は `FLOAT32LE` / 32bit / interleaved のみ扱う。
- `FA-COMPRESSOR-SPEC-003`: 起動時 config は範囲検証し、不正値は fail closed する。
- `FA-COMPRESSOR-SPEC-004`: threshold 超過 sample は static curve で圧縮する。
- `FA-COMPRESSOR-SPEC-005`: 出力 `source_id` は保持し、`stream_id` は `output.stream_id` にする。
- `FA-COMPRESSOR-SPEC-006`: 入力 frame の `stream_id` は `input_stream_id` と一致していなければならない。
- `FA-COMPRESSOR-SPEC-007`: null frame pointer は drop する。
- `FA-COMPRESSOR-SPEC-008`: launch は `node_name` / `config_file` だけを公開し、node 固有 parameter は config file に閉じる。
- `FA-COMPRESSOR-SPEC-009`: ROS graph 経路で valid frame は publish され、stream mismatch frame は publish されない。
- `FA-COMPRESSOR-SPEC-010`: ROS topic と AudioFrame stream identity を分離し、衝突する config は fail closed する。
- `FA-COMPRESSOR-SPEC-011`: diagnostics QoS は node 固有 config として明示し、system QoS fallback に依存しない。
