# fa_compressor テスト設計

## 1. Unit contract test

`test/unit/test_fa_compressor_audio_frame_contract.py` は source/config/CMake を検査し、以下を確認する。

- default config が FLOAT32LE interleaved contract と指定 default 値を持つ。
- compressor が device I/O、resampling、format conversion、channel conversion、limiter、gate、normalize、filter、denoise を持たない。
- 起動時 config validation が fail closed になっている。
- runtime `AudioFrame` validation が source/stream identity と format/data 契約を確認する。
- `stream_id` は `output_topic` に更新し、それ以外の metadata は入力を保持する。
- static compression curve が仕様通りに実装されている。
- 出力範囲超過時に clamp / limit せず drop する。
- diagnostics が parameter と frame/sample counter を含む。
- package layout と colcon pytest registration が存在する。

## 2. Integration test 方針

実 ROS graph での integration test では、`AudioFrame` publisher から `input_topic` に known sample bytes を送信し、`output_topic` の bytes と metadata を比較する。

## 3. Launch test 方針

launch test では default config で node が起動できること、不正 config で fail closed すること、diagnostics timer が publish されることを確認する。
