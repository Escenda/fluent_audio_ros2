# fa_compressor テスト設計

## 1. Unit contract test

`test/unit/test_fa_compressor_audio_frame_contract.py` は source/config/CMake を検査し、以下を確認する。

- example config が FLOAT32LE interleaved contract と明示指定値を持つ。
- compressor node/backend が device I/O、resampling、format conversion、channel conversion、limiter、gate、normalize、filter、denoise を持たない。
- 起動時 config validation が fail closed になっている。
- runtime `AudioFrame` validation が source/stream identity と format/data 契約を確認する。
- ROS topic と `AudioFrame.stream_id` を分離し、衝突する設定を fail closed する。
- diagnostics QoS を明示 parameter として扱い、system QoS fallback に依存しない。
- node が `NodeOptions` を受け取り、`main()` が `src/main.cpp` に分離されている。
- `stream_id` は `output.stream_id` に更新し、それ以外の metadata は入力を保持する。
- backend が static compression curve を仕様通りに実装する。
- backend が `ProcessStatus` で empty / misaligned / non-finite / out-of-range input / invalid output を返す。
- 出力範囲超過時に clamp / limit せず drop し、output と counter を commit しない。
- diagnostics が parameter と frame/sample counter を含む。
- package layout と colcon pytest / backend gTest / ROS graph gTest registration が存在する。

## 2. Backend C++ tests

`test/cpp/test_internal_static_curve_backend.cpp` で以下を確認する。

- threshold 以下の sample は保持する。
- threshold 超過 sample に knee なし compression curve を適用し、符号を保持する。
- makeup gain を static curve 後に適用する。
- empty / byte alignment / NaN / normalized range violation / output range violation を `ProcessStatus` で返す。
- rejected frame では output buffer を上書きしない。

## 3. ROS graph tests

`test/cpp/test_compressor_graph.cpp` は `NodeOptions` で `fa_compressor` を起動し、
ROS2 executor 上で publisher/subscriber を接続して以下を確認する。

- valid `FLOAT32LE` frame が publish される。
- output frame は `source_id`、format metadata、`epoch` を保持し、`stream_id` を `output.stream_id` に更新する。
- threshold 超過 sample に static compression curve が適用される。
- `stream_id` が `input_stream_id` と一致しない frame は publish されない。

## 4. Launch tests

`test/launch/test_fa_compressor_launch_contract.py` は以下を確認する。

- launch argument が `node_name` / `config_file` のみに閉じている。
- example config が compressor の責務に閉じ、resample / limiter / normalize を含まない。
- `compressor.threshold_linear <= 0.0` で launch 経路が fail closed する。
- `compressor.ratio <= 1.0` で launch 経路が fail closed する。
- `compressor.makeup_gain_linear > 4.0` で launch 経路が fail closed する。
- wrong `expected.encoding` で launch 経路が fail closed する。
- `input_stream_id` / `output.stream_id` が衝突する config で launch 経路が fail closed する。
- stream identity が ROS topic と衝突する config で launch 経路が fail closed する。
