# fa_noise_gate テスト設計

## 1. 方針

unit test は package file を inspection し、起動設定、runtime contract、責務境界、
algorithm evidence、colcon pytest 登録を確認する。ROS graph を起動しない contract test
として、既存 processing package と同じ構成にする。

## 2. 対象

| テスト | 確認内容 |
| --- | --- |
| default config | topic、gate parameter、expected FLOAT32LE interleaved contract |
| unrelated responsibilities | device I/O、resample、format conversion、limiter、normalize、filter、denoise を含まない |
| startup validation | gate parameter と expected contract が fail closed で検証される |
| frame validation | source/stream、stream_id policy、format、data size を処理前に検証する |
| identity update | 入力 frame を preserve し、stream_id と data だけを明示更新する |
| algorithm evidence | `abs(sample) < threshold` のとき closed gain、その他 preserve |
| invalid sample handling | 非有限・範囲外 input/output を drop し、clamp しない |
| package layout | 指定された docs/config/launch/include/src/test layout |
| colcon pytest | `ament_cmake_pytest` で test directory を登録する |

## 3. 実行コマンド

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/dynamics/fa_noise_gate/test/unit -q
```

## 4. 未対象

この unit test では ROS2 runtime の publish/subscribe integration は起動しない。
integration test は `test/integration` に追加する。
