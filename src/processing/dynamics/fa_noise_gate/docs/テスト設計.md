# fa_noise_gate テスト設計

## 1. 方針

unit test は package file を inspection し、起動設定、runtime contract、backend 境界、
algorithm evidence、colcon pytest/gTest 登録を確認する。ROS graph test と launch test で
executor 上の publish/drop と fail-closed 起動も確認する。

## 2. 対象

| テスト | 確認内容 |
| --- | --- |
| example config | topic / stream identity、gate parameter、expected FLOAT32LE interleaved contract |
| unrelated responsibilities | device I/O、resample、format conversion、limiter、normalize、filter、denoise を含まない |
| startup validation | gate parameter と expected contract が fail closed で検証される |
| frame validation | source/stream、`input_stream_id` policy、format、data size を処理前に検証する |
| identity update | 入力 frame を preserve し、`output.stream_id` と data だけを明示更新する |
| topic/stream split | ROS topic と `AudioFrame.stream_id` を分離し、衝突する設定を fail closed する |
| diagnostics QoS | diagnostics QoS を明示 parameter として扱い、system QoS fallback に依存しない |
| algorithm evidence | backend が `abs(sample) < threshold` のとき closed gain、その他 preserve |
| invalid sample handling | backend が非有限・範囲外 input/output を `ProcessStatus` で返し、clamp しない |
| backend boundary | backend が `rclcpp` / `fa_interfaces` / `AudioFrame` を知らない |
| node core | node が `NodeOptions` を受け取り、`main()` が `src/main.cpp` に分離されている |
| package layout | 指定された docs/config/launch/include/src/test layout |
| colcon pytest/gTest | `ament_cmake_pytest`、backend gtest、graph gtest を登録する |

## 3. Backend C++ tests

- threshold 以上の sample を preserve する。
- threshold 未満の sample に closed gain を適用し、gated sample count を返す。
- `closed_gain_linear=0.0` を明示 mute として扱う。
- empty / byte alignment / NaN / normalized range violation を `ProcessStatus` で返す。
- rejected frame では output buffer を上書きしない。

## 4. ROS graph tests

`test/cpp/test_noise_gate_graph.cpp` は `NodeOptions` で `fa_noise_gate` を起動し、ROS2 executor 上で publisher/subscriber を接続する。

- valid frame が publish されること
- threshold 未満 sample に `closed_gain_linear` が適用されること
- `source_id`、format、`epoch` が保持され、`stream_id` が `output.stream_id` に更新されること
- `stream_id != input_stream_id` の frame が publish されないこと

## 5. Launch tests

`test/launch/test_fa_noise_gate_launch_contract.py` は launch file と example config を検査し、ROS2 が利用可能な環境では不正 config の起動失敗を確認する。

- launch argument は `node_name` / `config_file` のみ
- example config は noise gate 専用の dynamics node として resample / normalize / compressor / limiter を含まない
- `gate.threshold_linear < 0.0` は起動失敗
- `gate.closed_gain_linear > 1.0` は起動失敗
- `expected.encoding != FLOAT32LE` は起動失敗
- stream identity 衝突、stream identity と ROS topic の衝突は起動失敗

## 6. 実行コマンド

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/dynamics/fa_noise_gate/test/unit src/processing/dynamics/fa_noise_gate/test/launch -q
```
