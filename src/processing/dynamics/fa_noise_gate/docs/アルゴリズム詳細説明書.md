# fa_noise_gate アルゴリズム詳細説明書

## 1. 入力表現

入力は `FLOAT32LE` interleaved の validated sample bytes である。1 sample は
little-endian の 32bit float として backend が読み出す。処理前に frame contract を検証し、
`data.size()` が `channels * sizeof(float)` で割り切れない frame は drop する。

## 2. 正規化範囲

各 sample は finite かつ `[-1.0, 1.0]` に入っていなければならない。
範囲外 sample は clip や normalize の対象ではなく、frame 全体を無効として drop する。

## 3. Threshold gate

パラメータは以下の 2 つである。

- `gate.threshold_linear`
- `gate.closed_gain_linear`

各 sample `x` に対して以下を計算する。

```text
if abs(x) < gate.threshold_linear:
    y = x * gate.closed_gain_linear
else:
    y = x
```

`abs(x) == gate.threshold_linear` の場合は gate を閉じず、入力 sample を preserve する。
`gate.threshold_linear == 0.0` の場合、正規化範囲内の sample はすべて preserve される。
`gate.closed_gain_linear == 0.0` の場合、threshold 未満の sample は 0.0 になる。
これは明示的な mute であり、欠損値の fallback ではない。

## 4. 出力検証

計算結果 `y` も finite かつ `[-1.0, 1.0]` に入っていなければならない。
不正な出力が発生した場合は frame 全体を drop する。clamp や別ゲインへの fallback は行わない。
backend は失敗 frame の output buffer と `samples_gated` counter を commit しない。

## 5. 状態

このアルゴリズムは frame 間状態を持たない。node は source_id を preserve し、stream_id を
`output_topic` に更新する。node は sample loop を持たない。
