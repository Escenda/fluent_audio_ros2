# fa_gain テスト設計

## Unit

- default config が `FLOAT32LE` / 32bit / interleaved を要求する。
- backend API、ROS graph、launch tests で clamp、clip、normalize、resample、channel conversion を行わないことを検証する。
- `validateFrame` が source identity、layout、encoding、bit depth、sample rate、channels、data size を検証する。
- `validateFrame` が ROS topic ではなく `input_stream_id` で frame identity を検証する。
- `applyGain` が `source_id` を保持し、`stream_id` を `output.stream_id` に更新し、backend status で reject する。
- `input_topic == output_topic`、`input_stream_id == output.stream_id`、topic と stream id の同値、QoS depth の暗黙補正がないことを確認する。
- diagnostics QoS が required parameter で、system default QoS に依存しないことを確認する。
- backend が `ProcessStatus` で empty / misaligned / non-finite / out-of-range input / invalid output を返す。
- backend API、ROS graph、launch fail-closed の executable coverage を維持する。

## Backend C++ tests

- `gain.linear` を canonical samples に適用する。
- interleaved channel alignment を検証する。
- empty / byte alignment / NaN / normalized range violation を `ProcessStatus` で返す。
- gain 後に `[-1.0, 1.0]` を超える出力を reject する。
- representable でない出力を reject し、前回 output buffer を上書きしない。
- 未定義の backend status message は fallback 文字列へ丸めず例外にする。

## C++ graph smoke test

- `FaGainNode` を `rclcpp::NodeOptions` で起動し、test publisher / subscriber と同じ executor に載せる。
- 正規化済み `FLOAT32LE/32/interleaved` frame を input topic に publish する。
- output topic に gain 後の frame が publish される。
- `source_id`、sample rate、channels、encoding、bit depth、layout、epoch は保持される。
- `stream_id` は `output.stream_id` に更新される。
- `gain.linear` が sample に適用され、範囲外 clipping / limiter / normalize は行われない。

## Integration

- `fa_resample -> fa_gain -> fa_vad` の topic wiring を確認する。
- gain 後に範囲外になる frame が publish されず、drop counter が増えることを確認する。

## Launch

- launch file は `node_name` と `config_file` だけを必須 argument として受け取り、package default config へ fallback しない。
- default config は dynamics node として `FLOAT32LE/32/interleaved` 契約と `gain.linear` だけを持ち、resample / normalize / limiter を含まない。
- `gain.linear < 0`、`expected.encoding != FLOAT32LE` は launch 経路でも起動失敗する。
- `input_topic == output_topic`、`input_stream_id == output.stream_id`、topic と stream id の同値は launch 経路でも起動失敗する。

## Fixtures

- 正規化済み `FLOAT32LE` mono 16kHz frame。
- gain 後に範囲外になる frame。

## 仕様とテストケース対応

### 仕様 ID

- `FA-GAIN-SPEC-001`: default config は `FLOAT32LE/32/interleaved` 入力と明示的な `gain.linear` を要求する。
- `FA-GAIN-SPEC-002`: `fa_gain` は clamp、clip、normalize、resample、channel conversion、I/O を持たない。
- `FA-GAIN-SPEC-003`: frame validation は source identity、stream identity、layout、encoding、bit depth、sample rate、channels、data size を検証する。
- `FA-GAIN-SPEC-004`: gain parameter、transport topic、stream identity、diagnostics QoS は required parameter として検証し、暗黙 default に依存しない。
- `FA-GAIN-SPEC-005`: output は `source_id`、sample rate、channels、encoding、bit depth、layout、epoch を保持し、`stream_id` だけを `output.stream_id` に更新する。
- `FA-GAIN-SPEC-006`: ROS-free backend は interleaved `FLOAT32LE` sample に `gain.linear` を適用する。
- `FA-GAIN-SPEC-007`: empty、misaligned、non-finite、normalized range 外 input、gain 後 range overflow は reject し、limiter / normalize へ逃がさない。
- `FA-GAIN-SPEC-008`: transport topic と stream identity は別契約で、topic/stream の同値や入出力同値を起動時に拒否する。
- `FA-GAIN-SPEC-009`: executable tests は backend API、ROS graph publish/drop、launch fail-closed 経路を検証する。
- `FA-GAIN-SPEC-010`: launch file は `node_name` / `config_file` だけを公開し、package default config fallback や inline parameter を持たない。
- `FA-GAIN-SPEC-011`: ROS graph smoke test で正規化済み `FLOAT32LE/32/interleaved` input から gain 適用済み output が publish される。
