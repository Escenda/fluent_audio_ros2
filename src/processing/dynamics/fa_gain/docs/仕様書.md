# fa_gain 仕様書

`fa_gain` は音量・ダイナミクス系の processing node である。入力された正規化済み `FLOAT32LE` `AudioFrame` に明示的な linear gain を掛け、別 topic へ publish する。

## 責務

- `input_topic` から `AudioFrame` を購読する。
- `input_stream_id` と一致する frame identity だけを処理する。
- `gain.linear` を全 sample に適用する。
- `output_topic` へ同じ format の `AudioFrame` を publish する。
- 出力 frame の `stream_id` は `output.stream_id` に設定する。
- diagnostics に入出力 frame 数と drop 数を publish する。

## 非責務

- device I/O
- resample
- bit depth 変換
- channel 変換
- normalize
- limiter / clipping
- compressor / AGC
- VAD / ASR / feature extraction

gain 後に `[-1.0, 1.0]` を超える場合、`fa_gain` は limiter として飽和処理しない。その frame は drop し、limiter が必要な場合は `processing/dynamics/fa_limiter` を pipeline に追加する。

## Parameters

| parameter | required | 内容 |
| --- | --- | --- |
| `input_topic` | yes | 入力 `AudioFrame` topic |
| `output_topic` | yes | 出力 `AudioFrame` topic |
| `input_stream_id` | yes | 入力 `AudioFrame.stream_id` と一致すべき stream identity |
| `output.stream_id` | yes | 出力 `AudioFrame.stream_id` に設定する stream identity |
| `gain.linear` | yes | `>= 0.0` の有限値 |
| `expected.sample_rate` | yes | 期待 sample rate |
| `expected.channels` | yes | 期待 channel 数 |
| `expected.encoding` | yes | `FLOAT32LE` 固定 |
| `expected.bit_depth` | yes | `32` 固定 |
| `expected.layout` | yes | `interleaved` 固定 |
| `qos.depth` | yes | ROS topic queue depth |
| `qos.reliable` | yes | reliable / best effort |
| `diagnostics.publish_period_ms` | yes | diagnostics publish interval |

## Error Handling

- 起動時 parameter が不正な場合は例外で起動失敗する。
- `input_topic == output_topic`、`input_stream_id == output.stream_id` は自己 feedback や identity 混線を作るため起動失敗する。
- frame identity、`stream_id == input_stream_id`、format、layout、data size が不正な frame は publish せず drop する。
- 入力 sample が `[-1.0, 1.0]` 外、または gain 後に範囲外になる frame は publish せず drop する。
- clipping、limiting、normalize による暗黙補正は行わない。

## Traceability

- `FA-GAIN-SPEC-001`: `fa_gain` は device I/O を行わない。
- `FA-GAIN-SPEC-002`: `fa_gain` は `FLOAT32LE` / 32bit / interleaved のみ扱う。
- `FA-GAIN-SPEC-003`: `gain.linear` は有限値かつ `>= 0.0`。
- `FA-GAIN-SPEC-004`: clipping / normalize / limiter を実装しない。
- `FA-GAIN-SPEC-005`: 出力 `source_id` は保持し、`stream_id` は `output.stream_id` にする。
- `FA-GAIN-SPEC-006`: 入力 frame の `stream_id` は `input_stream_id` と一致していなければならない。
- `FA-GAIN-SPEC-007`: ROS transport topic と frame stream identity は別 parameter として扱う。
- `FA-GAIN-SPEC-008`: `input_topic == output_topic` または `input_stream_id == output.stream_id` は起動時に拒否する。
