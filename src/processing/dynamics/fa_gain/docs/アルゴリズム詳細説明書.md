# fa_gain アルゴリズム詳細説明書

## 入力検証

1. `source_id` と `stream_id` が空でないことを確認する。
2. `stream_id == input_stream_id` を確認する。
3. `layout == interleaved` を確認する。
4. `encoding == FLOAT32LE`、`bit_depth == 32` を確認する。
5. `sample_rate` と `channels` が config の `expected.*` と一致することを確認する。
6. `data.size()` が `channels * sizeof(float)` の倍数であることを確認する。

不一致は frame skip として扱う。別 format を暗黙変換すると format node の責務を侵食するためである。

## Gain 適用

`internal_gain` backend が sample 単位の gain 処理を行う。node は ROS parameter、
AudioFrame metadata、diagnostics だけを扱い、sample loop を持たない。

1. validated sample bytes を 4 byte ごとに `float` として読み出す。
2. 入力 sample が finite かつ `[-1.0, 1.0]` 内であることを確認する。
3. `sample * gain.linear` を計算する。
4. 結果が finite かつ `[-1.0, 1.0]` 内であることを確認する。
5. 出力 bytes へ `float` として書き戻す。

clamp helper や saturating cast は使わない。これは limiter の責務だからである。
backend は失敗 frame の output buffer を commit せず、node へ `ProcessStatus` を返す。

## 出力

- `header`、`source_id`、sample format metadata は入力から保持する。
- `stream_id` は `output.stream_id` に更新する。
- diagnostics には `frames_in`、`frames_out`、`frames_dropped`、`gain_linear`、QoS 契約を出す。
