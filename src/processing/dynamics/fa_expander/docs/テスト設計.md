# fa_expander テスト設計

## 1. Python unit contract tests

`test/unit/test_fa_expander_audio_frame_contract.py` は package source を検査し、以下を確認する。

- example config が `FLOAT32LE` / 32-bit / interleaved 契約を満たす
- expander parameter の明示値と範囲
- device I/O、device gain、resampling、format conversion、channel conversion、compressor、limiter、gate、filter、denoise を含まない
- 起動時設定不正を `std::runtime_error` で fail closed する
- runtime `AudioFrame` contract を処理前に検証する
- `stream_id == input_stream_id` を要求し、出力 `stream_id` を `output.stream_id` に更新する
- ROS topic と `AudioFrame.stream_id` を分離し、衝突する設定を fail closed する
- diagnostics QoS を明示 parameter として扱い、system QoS fallback に依存しない
- 静的 expansion 曲線が backend にあり、node からは消えている
- backend が ROS2 topic/message を知らない
- input/output sample の finite と normalized range を backend が検証し、clamp/zero fallback しない
- output 拒否時は output vector を確定しない
- diagnostics が backend config と counter を publish する
- node が `NodeOptions` を受け取り、`main()` が `src/main.cpp` に分離されている
- package layout が `src/main.cpp`、graph test、launch test を含む標準 layout を満たす
- colcon pytest、backend gtest、graph gtest registration が存在する

## 2. C++ backend tests

`test/cpp/test_internal_static_expander_backend.cpp` は ROS を使わず backend を直接検証する。

- 不正 config を constructor で拒否する
- threshold 以上の sample は変化させない
- threshold 未満の sample は符号を保ったまま下向き expansion する
- 空入力、misaligned input、非 finite input、範囲外 input を status で拒否する
- 拒否時に output vector を更新しない

## 3. Integration tests

`test/cpp/test_expander_graph.cpp` は `NodeOptions` で `fa_expander` を起動し、ROS2 executor 上で publisher/subscriber を接続する。

- valid frame が publish されること
- threshold 未満 sample が下向き expansion されること
- `source_id`、format、`epoch` が保持され、`stream_id` が `output.stream_id` に更新されること
- `stream_id != input_stream_id` の frame が publish されないこと

## 4. Launch tests

`test/launch/test_fa_expander_launch_contract.py` は launch file と example config を検査し、ROS2 が利用可能な環境では不正 config の起動失敗を確認する。

- launch argument は `node_name` / `config_file` のみ
- example config は expander 専用の dynamics node として resample / normalize / compressor / limiter を含まない
- `expander.threshold_linear <= 0.0` は起動失敗
- `expander.ratio <= 1.0` は起動失敗
- `expected.encoding != FLOAT32LE` は起動失敗
- stream identity 衝突、stream identity と ROS topic の衝突は起動失敗

## 5. Manual validation command

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/dynamics/fa_expander/test/unit src/processing/dynamics/fa_expander/test/launch -q
```
