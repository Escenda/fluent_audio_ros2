# アルゴリズム詳細説明書

## 責務分離

ROS node は `AudioFrame` metadata validation、topic 入出力、diagnostics、counter 更新だけを持つ。静的下向き expansion 曲線、sample decode/encode、sample 値検証は `internal_static_expander` backend が持つ。backend は ROS2 topic/message を知らない。

## 処理内容

`fa_expander` は静的な下向きエキスパンションのみを行う。attack、release、RMS 検出、ピーク正規化、しきい値以下のゼロ化、clamp は行わない。

各サンプル `x` について、`a = abs(x)` とする。

- `a >= threshold` の場合: `y = x`
- `a < threshold` の場合: `expanded_abs = threshold * pow(a / threshold, ratio)`、`y = sign(x) * expanded_abs`

`ratio > 1.0` なので、しきい値未満の絶対値はより小さい絶対値へ写像される。`x == 0.0` は `0.0` になる。

## 妥当性検査

node は `AudioFrame` metadata と byte length を検証する。backend は入力サンプルが finite かつ `[-1.0, 1.0]` の範囲内であることを検証する。出力計算結果も double と float の両方で finite かつ `[-1.0, 1.0]` の範囲内であることを確認する。

backend は出力全体を一時 vector に生成し、全 sample の検証が完了してから output vector を置き換える。不正な値が見つかった場合は status を返し、output vector を変更しない。

## カウンタ

- `frames_in`: 受信フレーム数
- `frames_out`: publish したフレーム数
- `frames_dropped`: 契約違反または計算結果不正により drop したフレーム数
- `samples_expanded`: `abs(x) < threshold` により展開処理を受けた累積サンプル数

`samples_expanded` は backend result を node が成功時に反映する。backend は ROS counter を直接持たない。
