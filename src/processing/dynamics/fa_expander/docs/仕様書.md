# fa_expander 仕様書

## 目的

`fa_expander` は、FLOAT32LE interleaved `AudioFrame` ストリームに静的な下向きエキスパンションを適用する。ノイズゲート、コンプレッサ、リミッタ、正規化、フィルタ、デノイズ、リサンプリング、デバイス I/O、チャンネル変換は行わない。

DSP 本体は ROS 非依存の `internal_static_expander` backend に置く。ROS node は `AudioFrame` 契約検証、topic 入出力、diagnostics、counter 更新を担当する。

## ノード

- package: `fa_expander`
- executable: `fa_expander_node`
- node: `fa_expander`
- node core: `fa_expander_node_core`

## 入力契約

入力フレームは以下を満たす必要がある。

- `source_id` が非空
- `stream_id` が非空
- `stream_id == input_stream_id`
- `sample_rate > 0` かつ設定値と一致
- `channels > 0` かつ設定値と一致
- `encoding == "FLOAT32LE"`
- `bit_depth == 32`
- `layout == "interleaved"`
- `data` が非空
- `data.size()` が `channels * sizeof(float)` で割り切れる
- 全サンプルが finite かつ `[-1.0, 1.0]`

契約違反の実行時フレームは warning を出して drop する。

## 出力契約

出力フレームは入力から以下を保持する。

- `header`
- `source_id`
- `sample_rate`
- `channels`
- `bit_depth`
- `encoding`
- `layout`
- `epoch`

更新するフィールドは以下のみ。

- `stream_id`: `output.stream_id`
- `data`: 下向きエキスパンション後の FLOAT32LE interleaved samples

## パラメータ

全 parameter は launch で渡された config file から明示設定する。node 内の暗黙値や package-provided config fallback は持たない。

| 名前 | 必須 | 制約 |
| --- | --- | --- |
| `input_topic` | yes | 非空、`output_topic` と別の ROS topic に解決される |
| `output_topic` | yes | 非空、`input_topic` と別の ROS topic に解決される |
| `input_stream_id` | yes | 非空、ROS topic 名と衝突しない、`output.stream_id` と別 |
| `output.stream_id` | yes | 非空、ROS topic 名と衝突しない、`input_stream_id` と別 |
| `expander.threshold_linear` | yes | finite, `> 0.0`, `< 1.0` |
| `expander.ratio` | yes | finite, `> 1.0` |
| `expected.sample_rate` | yes | `0 < value <= 384000` |
| `expected.channels` | yes | `0 < value <= 64` |
| `expected.encoding` | yes | `FLOAT32LE` |
| `expected.bit_depth` | yes | `32` |
| `expected.layout` | yes | `interleaved` |
| `qos.depth` | yes | `> 0` |
| `qos.reliable` | yes | bool |
| `diagnostics.qos.depth` | yes | `> 0` |
| `diagnostics.qos.reliable` | yes | bool |
| `diagnostics.publish_period_ms` | yes | `> 0` |

不正な設定では起動を継続しない。

## Diagnostics

`diagnostics` topic に `diagnostic_msgs/msg/DiagnosticArray` を publish する。status values には少なくとも以下を含める。

- `expander_threshold_linear`
- `expander_ratio`
- `expected_sample_rate`
- `expected_channels`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `samples_expanded`
- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `qos.depth`
- `diagnostics.qos.depth`
- `diagnostics.qos.reliable`

## 失敗時動作

- 起動時設定不正: 例外により node 起動を失敗させる。
- runtime frame 契約違反: warning を出して frame を drop する。
- 入力 sample 不正: backend status を warning に含め、frame を drop する。
- 出力 sample 不正: backend status を warning に含め、frame を drop する。この場合、backend は output vector を更新しない。

## Launch Contract

`launch/fa_expander.launch.py` が受け取る launch argument は `node_name` と `config_file` のみとする。`node_name` と `config_file` に暗黙値は持たせない。DSP parameter は launch argument として露出せず、必ず `config_file` から読む。

node 実装は `fa_expander_node_core` と `src/main.cpp` に分離し、ROS graph test から `NodeOptions` で parameter override できる。

## Traceability

| ID | 契約 |
| --- | --- |
| `FA-EXPANDER-SPEC-001` | `fa_expander` は静的下向き expansion のみを行い、I/O、resample、filter、gate、compressor、limiter、normalize を持たない |
| `FA-EXPANDER-SPEC-002` | 起動時 config は必須 topic / stream identity、expander range、expected format、QoS、diagnostics period を fail closed で検証する |
| `FA-EXPANDER-SPEC-003` | backend は ROS2 topic/message を知らず、静的 expansion 曲線と sample 検証を所有する |
| `FA-EXPANDER-SPEC-004` | node は runtime `AudioFrame` metadata と byte layout を処理前に検証し、違反 frame を publish しない |
| `FA-EXPANDER-SPEC-005` | 出力 frame は identity metadata を保持し、`stream_id` だけを `output.stream_id` に更新する |
| `FA-EXPANDER-SPEC-006` | backend 拒否時は output vector を確定せず、node は frame を drop する |
| `FA-EXPANDER-SPEC-007` | diagnostics は backend config と frame/sample counter を publish する |
| `FA-EXPANDER-SPEC-008` | launch は `node_name` と `config_file` のみを受け、config 不正時は起動失敗する |
| `FA-EXPANDER-SPEC-009` | node core は `NodeOptions` で生成でき、graph test から parameter override できる |
| `FA-EXPANDER-SPEC-010` | ROS topic と AudioFrame stream identity を分離し、衝突する config は fail closed する |
| `FA-EXPANDER-SPEC-011` | diagnostics QoS は node 固有 config として明示し、system QoS fallback に依存しない |
