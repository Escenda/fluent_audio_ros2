# fa_expander 仕様書

## 目的

`fa_expander` は、FLOAT32LE interleaved `AudioFrame` ストリームに静的な下向きエキスパンションを適用する。ノイズゲート、コンプレッサ、リミッタ、正規化、フィルタ、デノイズ、リサンプリング、デバイス I/O、チャンネル変換は行わない。

DSP 本体は ROS 非依存の `internal_static_expander` backend に置く。ROS node は `AudioFrame` 契約検証、topic 入出力、diagnostics、counter 更新を担当する。

## ノード

- package: `fa_expander`
- executable: `fa_expander_node`
- node: `fa_expander_node`

## 入力契約

入力フレームは以下を満たす必要がある。

- `source_id` が非空
- `stream_id` が非空
- `stream_id == input_topic`
- `sample_rate > 0` かつ設定値と一致
- `channels > 0` かつ設定値と一致
- `encoding == "FLOAT32LE"`
- `bit_depth == 32`
- `layout == "interleaved"`
- `data` が非空
- `data.size()` が `channels * sizeof(float)` で割り切れる
- 全サンプルが finite かつ `[-1.0, 1.0]`

契約違反の実行時フレームは warning を出して drop する。

## 出力契約

出力フレームは入力から以下を保持する。

- `header`
- `source_id`
- `sample_rate`
- `channels`
- `bit_depth`
- `encoding`
- `layout`
- `epoch`

更新するフィールドは以下のみ。

- `stream_id`: `output_topic`
- `data`: 下向きエキスパンション後の FLOAT32LE interleaved samples

## パラメータ

| 名前 | 既定値 | 制約 |
| --- | ---: | --- |
| `input_topic` | `audio/noise_gated/mic` | 非空 |
| `output_topic` | `audio/expanded/mic` | 非空 |
| `expander.threshold_linear` | `0.05` | finite, `> 0.0`, `< 1.0` |
| `expander.ratio` | `2.0` | finite, `> 1.0` |
| `expected.sample_rate` | `16000` | `> 0` |
| `expected.channels` | `1` | `> 0` |
| `expected.encoding` | `FLOAT32LE` | 固定 |
| `expected.bit_depth` | `32` | 固定 |
| `expected.layout` | `interleaved` | 固定 |
| `qos.depth` | `10` | `> 0` |
| `qos.reliable` | `false` | bool |
| `diagnostics.publish_period_ms` | `1000` | `> 0` |

不正な設定では起動を継続しない。

## Diagnostics

`diagnostics` topic に `diagnostic_msgs/msg/DiagnosticArray` を publish する。status values には少なくとも以下を含める。

- `expander_threshold_linear`
- `expander_ratio`
- `expected_sample_rate`
- `expected_channels`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `samples_expanded`
- `output_topic`

## 失敗時動作

- 起動時設定不正: 例外により node 起動を失敗させる。
- runtime frame 契約違反: warning を出して frame を drop する。
- 入力 sample 不正: backend status を warning に含め、frame を drop する。
- 出力 sample 不正: backend status を warning に含め、frame を drop する。この場合、backend は output vector を更新しない。
