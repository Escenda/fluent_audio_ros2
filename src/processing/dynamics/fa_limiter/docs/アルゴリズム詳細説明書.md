# fa_limiter アルゴリズム詳細説明書

## 入力検証

1. `source_id` と `stream_id` が空でないことを確認する。
2. `stream_id == input_topic` を確認する。
3. `layout == interleaved` を確認する。
4. `encoding == FLOAT32LE`、`bit_depth == 32` を確認する。
5. `sample_rate` と `channels` が config の `expected.*` と一致することを確認する。
6. `data.size()` が `channels * sizeof(float)` の倍数であることを確認する。

不一致は frame skip として扱う。別 format を暗黙変換すると format node の責務を侵食するためである。

## Hard Limiter

1. `AudioFrame.data` を 4 byte ごとに `float` として読み出す。
2. 入力 sample が finite であることを確認する。
3. `sample > threshold.linear` の場合は `threshold.linear` にする。
4. `sample < -threshold.linear` の場合は `-threshold.linear` にする。
5. それ以外の sample は変更しない。
6. 出力 `AudioFrame.data` へ `float` として書き戻す。

`fa_limiter` の clamp は limiter の明示責務である。sample rate、channel 数、layout、bit depth、encoding は変更しない。

## 出力

- `header`、`source_id`、sample format metadata は入力から保持する。
- `stream_id` は `output_topic` に更新する。
- diagnostics には `frames_in`、`frames_out`、`frames_dropped`、`samples_limited`、`threshold_linear` を出す。
