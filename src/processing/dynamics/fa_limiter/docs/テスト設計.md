# fa_limiter テスト設計

## Unit

- default config が `FLOAT32LE` / 32bit / interleaved を要求する。
- default config が `threshold.linear` を必須構造として持ち、値が `(0.0, 1.0]` である。
- node/backend source code に device API、resample、normalize、gain、channel conversion が存在しない。
- `validateFrame` が source identity、layout、encoding、bit depth、sample rate、channels、data size を検証する。
- backend が sample を `[-threshold.linear, +threshold.linear]` に制限し、limited sample count を返す。
- `applyLimiter` が `source_id` を保持し、`stream_id` を `output_topic` に更新する。
- backend が `ProcessStatus` で empty / misaligned / non-finite input を返す。
- CMake が pytest と backend gTest を colcon test に登録する。
- package layout が標準 layout を満たす。

## Backend C++ tests

- threshold 未満の sample を保持する。
- threshold 超過 sample を明示 threshold に制限する。
- `[-1.0, 1.0]` を超える finite sample も limiter semantics として threshold に制限する。
- empty / byte alignment / non-finite input を `ProcessStatus` で返す。
- rejected frame では output buffer を上書きしない。

## Integration

- `fa_gain -> fa_limiter -> fa_out` の topic wiring を確認する。
- threshold を超えた sample が publish 前に clamp され、`samples_limited` が増えることを確認する。

## Launch

- `ros2 launch fa_limiter fa_limiter.launch.py` が `config/default.yaml` で起動する。

## Fixtures

- `threshold.linear` 未満の `FLOAT32LE` mono 16kHz frame。
- `threshold.linear` を超える `FLOAT32LE` mono 16kHz frame。
