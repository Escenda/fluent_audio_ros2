# fa_limiter テスト設計

## Unit

- example config が `FLOAT32LE` / 32bit / interleaved を要求する。
- example config が `threshold.linear` を必須構造として持ち、値が `(0.0, 1.0]` である。
- node/backend source code に device API、resample、normalize、gain、channel conversion が存在しない。
- node が `NodeOptions` を受け取り、`main()` が `src/main.cpp` に分離されている。
- `validateFrame` が null pointer、source identity、layout、encoding、bit depth、sample rate、channels、data size を検証する。
- ROS topic と `AudioFrame.stream_id` を分離し、衝突する設定を fail closed する。
- diagnostics QoS を明示 parameter として扱い、system QoS fallback に依存しない。
- backend が sample を `[-threshold.linear, +threshold.linear]` に制限し、limited sample count を返す。
- `applyLimiter` が `source_id` を保持し、`stream_id` を `output.stream_id` に更新する。
- backend が `ProcessStatus` で empty / misaligned / non-finite input を返す。
- CMake が pytest、backend gTest、ROS graph gTest を colcon test に登録する。
- package layout が `src/main.cpp`、graph test、launch test を含む標準 layout を満たす。

## Backend C++ tests

- threshold 未満の sample を保持する。
- threshold 超過 sample を明示 threshold に制限する。
- `[-1.0, 1.0]` を超える finite sample も limiter semantics として threshold に制限する。
- empty / byte alignment / non-finite input を `ProcessStatus` で返す。
- rejected frame では output buffer を上書きしない。

## Integration

- `test/cpp/test_limiter_graph.cpp` で `NodeOptions` により `fa_limiter` を起動する。
- valid `FLOAT32LE` frame が ROS graph 経路で publish されることを確認する。
- output frame は `source_id`、format metadata、`epoch` を保持し、`stream_id` を `output.stream_id` に更新することを確認する。
- threshold を超えた sample が publish 前に clamp されることを確認する。
- `stream_id` が `input_stream_id` と一致しない frame は publish されないことを確認する。

## Launch

- `test/launch/test_fa_limiter_launch_contract.py` で launch argument が `node_name` / `config_file` のみに閉じていることを確認する。
- example config が dynamics node の責務に閉じ、resample / normalize / gain を含まないことを確認する。
- `threshold.linear <= 0.0`、`threshold.linear > 1.0`、wrong `expected.encoding` で launch 経路が fail closed することを確認する。
- `input_stream_id` / `output.stream_id` が衝突する config で launch 経路が fail closed することを確認する。
- stream identity が ROS topic と衝突する config で launch 経路が fail closed することを確認する。

## Fixtures

- `threshold.linear` 未満の `FLOAT32LE` mono 16kHz frame。
- `threshold.linear` を超える `FLOAT32LE` mono 16kHz frame。
