# fa_limiter 仕様書

`fa_limiter` は音量・ダイナミクス系の processing node である。入力された `FLOAT32LE` `AudioFrame` に明示的な hard limiter を適用し、別 topic へ publish する。

## 責務

- `input_topic` から `AudioFrame` を購読する。
- `threshold.linear` を上限として sample を `[-threshold.linear, +threshold.linear]` に制限する。
- `output_topic` へ同じ format の `AudioFrame` を publish する。
- `AudioFrame.stream_id` は `input_stream_id` で受け、出力では `output.stream_id` に更新する。
- diagnostics に入出力 frame 数、drop 数、limiter 適用 sample 数を publish する。

## 非責務

- device I/O
- resample
- bit depth 変換
- channel 変換
- interleave / planar 変換
- normalize
- gain
- compressor / AGC
- VAD / ASR / feature extraction

`fa_limiter` は limiter として sample を clamp する。これは normalize や gain ではなく、明示された threshold を超えた sample の上限制御である。
ROS topic 名は転送経路、`AudioFrame.stream_id` は音声 stream identity として扱う。`input_topic` / `output_topic` と `input_stream_id` / `output.stream_id` は非空かつ互いに衝突してはならない。

## Parameters

| parameter | required | 内容 |
| --- | --- | --- |
| `input_topic` | yes | 入力 `AudioFrame` topic。`output_topic` と別の ROS topic に解決される |
| `output_topic` | yes | 出力 `AudioFrame` topic。`input_topic` と別の ROS topic に解決される |
| `input_stream_id` | yes | 入力 `AudioFrame.stream_id`。ROS topic 名と衝突しない |
| `output.stream_id` | yes | 出力 `AudioFrame.stream_id`。ROS topic 名と衝突しない |
| `threshold.linear` | yes | `(0.0, 1.0]` の有限値 |
| `expected.sample_rate` | yes | `0 < value <= 384000` |
| `expected.channels` | yes | `0 < value <= 64` |
| `expected.encoding` | yes | `FLOAT32LE` 固定 |
| `expected.bit_depth` | yes | `32` 固定 |
| `expected.layout` | yes | `interleaved` 固定 |
| `qos.depth` | yes | ROS topic queue depth |
| `qos.reliable` | yes | reliable / best effort |
| `diagnostics.qos.depth` | yes | diagnostics topic queue depth |
| `diagnostics.qos.reliable` | yes | diagnostics reliable / best effort |
| `diagnostics.publish_period_ms` | yes | diagnostics publish interval |

## Error Handling

- 起動時 parameter が不正な場合は例外で起動失敗する。
- `AudioFrame` pointer が null の場合は publish せず drop する。
- frame identity、`stream_id == input_stream_id`、format、layout、data size が不正な frame は publish せず drop する。
- 入力 sample が finite でない frame は publish せず drop する。
- format 変換、resample、channel 変換、normalize、gain による暗黙補正は行わない。

## Launch Contract

- `launch/fa_limiter.launch.py` は `node_name` と `config_file` だけを launch argument として公開する。
- `node_name` と `config_file` に暗黙値を持たせない。呼び出し側が必ず明示する。
- package share 内の config file を暗黙に探す fallback は持たない。
- `threshold.linear`、`expected.*`、QoS、diagnostics は config file で指定する。
- node 実装は `fa_limiter_node_core` と `src/main.cpp` に分離し、ROS graph test から `NodeOptions` で parameter override できる。
- `threshold.linear <= 0.0`、`threshold.linear > 1.0`、format contract 不一致は launch 経路でも fail closed する。

## Traceability

- `FA-LIMITER-SPEC-001`: `fa_limiter` は device I/O を行わない。
- `FA-LIMITER-SPEC-002`: `fa_limiter` は `FLOAT32LE` / 32bit / interleaved のみ扱う。
- `FA-LIMITER-SPEC-003`: `threshold.linear` は有限値かつ `(0.0, 1.0]`。
- `FA-LIMITER-SPEC-004`: limiter は sample を `[-threshold.linear, +threshold.linear]` に clamp する。
- `FA-LIMITER-SPEC-005`: 出力 `source_id` は保持し、`stream_id` は `output.stream_id` にする。
- `FA-LIMITER-SPEC-006`: 入力 frame の `stream_id` は `input_stream_id` と一致していなければならない。
- `FA-LIMITER-SPEC-007`: null frame pointer は drop する。
- `FA-LIMITER-SPEC-008`: launch は `node_name` / `config_file` だけを公開し、node 固有 parameter は config file に閉じる。
- `FA-LIMITER-SPEC-009`: ROS graph 経路で valid frame は publish され、stream mismatch frame は publish されない。
- `FA-LIMITER-SPEC-010`: ROS topic と AudioFrame stream identity を分離し、衝突する config は fail closed する。
- `FA-LIMITER-SPEC-011`: diagnostics QoS は node 固有 config として明示し、system QoS fallback に依存しない。
