# fa_limiter 仕様書

`fa_limiter` は音量・ダイナミクス系の processing node である。入力された `FLOAT32LE` `AudioFrame` に明示的な hard limiter を適用し、別 topic へ publish する。

## 責務

- `input_topic` から `AudioFrame` を購読する。
- `threshold.linear` を上限として sample を `[-threshold.linear, +threshold.linear]` に制限する。
- `output_topic` へ同じ format の `AudioFrame` を publish する。
- diagnostics に入出力 frame 数、drop 数、limiter 適用 sample 数を publish する。

## 非責務

- device I/O
- resample
- bit depth 変換
- channel 変換
- interleave / planar 変換
- normalize
- gain
- compressor / AGC
- VAD / ASR / feature extraction

`fa_limiter` は limiter として sample を clamp する。これは normalize や gain ではなく、明示された threshold を超えた sample の上限制御である。

## Parameters

| parameter | required | 内容 |
| --- | --- | --- |
| `input_topic` | yes | 入力 `AudioFrame` topic |
| `output_topic` | yes | 出力 `AudioFrame` topic |
| `threshold.linear` | yes | `(0.0, 1.0]` の有限値 |
| `expected.sample_rate` | yes | 期待 sample rate |
| `expected.channels` | yes | 期待 channel 数 |
| `expected.encoding` | yes | `FLOAT32LE` 固定 |
| `expected.bit_depth` | yes | `32` 固定 |
| `expected.layout` | yes | `interleaved` 固定 |
| `qos.depth` | yes | ROS topic queue depth |
| `qos.reliable` | yes | reliable / best effort |
| `diagnostics.publish_period_ms` | yes | diagnostics publish interval |

## Error Handling

- 起動時 parameter が不正な場合は例外で起動失敗する。
- frame identity、`stream_id == input_topic`、format、layout、data size が不正な frame は publish せず drop する。
- 入力 sample が finite でない frame は publish せず drop する。
- format 変換、resample、channel 変換、normalize、gain による暗黙補正は行わない。

## Traceability

- `FA-LIMITER-SPEC-001`: `fa_limiter` は device I/O を行わない。
- `FA-LIMITER-SPEC-002`: `fa_limiter` は `FLOAT32LE` / 32bit / interleaved のみ扱う。
- `FA-LIMITER-SPEC-003`: `threshold.linear` は有限値かつ `(0.0, 1.0]`。
- `FA-LIMITER-SPEC-004`: limiter は sample を `[-threshold.linear, +threshold.linear]` に clamp する。
- `FA-LIMITER-SPEC-005`: 出力 `source_id` は保持し、`stream_id` は `output_topic` にする。
- `FA-LIMITER-SPEC-006`: 入力 frame の `stream_id` は `input_topic` と一致していなければならない。
