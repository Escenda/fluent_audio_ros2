# fa_normalize アルゴリズム詳細説明書

## 1. 入力

入力は `FLOAT32LE` interleaved の `AudioFrame.data` である。`channels = C`、総 sample 数を `N` とすると、`N` は `C` で割り切れる。frame 数は `F = N / C` である。

## 2. 検証

処理前に `AudioFrame` metadata と data size を検証する。さらに各 sample を `float` として読み、finite かつ normalized `[-1.0, 1.0]` であることを確認する。

検証失敗時はその frame を drop し、出力を publish しない。

## 3. Peak 算出

全 sample の absolute value を走査し、frame peak を求める。

```text
peak = max(abs(sample[i])) for i in 0..N-1
```

## 4. Silence pass-through

`peak < normalize.silence_threshold_linear` の場合は無音相当 frame として扱う。この場合、sample 値は変更せず、`stream_id` のみ `output_topic` に更新する。

この分岐は無音付近を無理に増幅しないための明示的な処理であり、欠損値や不正値を正常扱いする fallback ではない。適用条件は `peak < silence_threshold` に限定され、入力 sample の検証は事前に完了している。

## 5. Peak normalization

`peak >= normalize.silence_threshold_linear` の場合は以下の gain を使う。

```text
gain = normalize.target_peak_linear / peak
output[i] = input[i] * gain
```

全 output sample は finite かつ normalized `[-1.0, 1.0]` である必要がある。範囲外になる場合は frame を drop する。clamp は行わない。

## 6. 出力 metadata

出力 `AudioFrame` は入力をコピーしてから `stream_id` と `data` のみ更新する。`header`、`source_id`、`sample_rate`、`channels`、`bit_depth`、`encoding`、`layout`、`epoch` は保持する。

## 7. Diagnostics

処理状態として `target_peak_linear`、`silence_threshold_linear`、`last_gain`、入力/出力/drop counter、silence pass-through counter、normalized counter を publish する。
