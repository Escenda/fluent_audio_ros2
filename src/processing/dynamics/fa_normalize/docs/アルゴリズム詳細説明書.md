# fa_normalize アルゴリズム詳細説明書

## 1. 責務分離

ROS node は `AudioFrame` metadata validation、topic 入出力、diagnostics、counter 更新だけを持つ。Peak 検出、silence 判定、gain 計算、sample decode/encode、sample 値検証は `internal_peak_normalize` backend が持つ。backend は ROS2 topic/message を知らない。

## 2. 入力

入力は `FLOAT32LE` interleaved の `AudioFrame.data` である。`channels = C`、総 sample 数を `N` とすると、`N` は `C` で割り切れる。frame 数は `F = N / C` である。

## 3. 検証

node は subscription callback で null `AudioFrame` pointer を drop し、処理前に
`AudioFrame` metadata と data size を検証する。backend は各 sample を `float`
として読み、finite かつ normalized `[-1.0, 1.0]` であることを確認する。

検証失敗時はその frame を drop し、出力を publish しない。backend は拒否時に output vector を更新しない。

## 4. Peak 算出

全 sample の absolute value を走査し、frame peak を求める。

```text
peak = max(abs(sample[i])) for i in 0..N-1
```

## 5. Silence pass-through

`peak < normalize.silence_threshold_linear` の場合は無音相当 frame として扱う。この場合、sample 値は変更せず、backend は input bytes と同一の output bytes、`ProcessMode::kSilencePassthrough`、`gain = 1.0` を返す。

この分岐は無音付近を無理に増幅しないための明示的な処理であり、欠損値や不正値を正常扱いする fallback ではない。適用条件は `peak < silence_threshold` に限定され、入力 sample の検証は事前に完了している。

## 6. Peak normalization

`peak >= normalize.silence_threshold_linear` の場合は以下の gain を使う。

```text
gain = normalize.target_peak_linear / peak
output[i] = input[i] * gain
```

全 output sample は finite かつ normalized `[-1.0, 1.0]` である必要がある。範囲外になる場合は frame を drop する。clamp は行わない。

backend は出力全体を一時 vector に生成し、全 sample の検証が完了してから output vector を置き換える。途中で拒否された frame は output vector と node 側の `last_gain` を変更しない。

## 7. 出力 metadata

出力 `AudioFrame` は入力をコピーしてから `stream_id` と `data` のみ更新する。`stream_id` は `output.stream_id` に設定する。`header`、`source_id`、`sample_rate`、`channels`、`bit_depth`、`encoding`、`layout`、`epoch` は保持する。

## 8. Diagnostics

node は backend accessor から `target_peak_linear`、`silence_threshold_linear` を読み、`last_gain`、入力/出力/drop counter、silence pass-through counter、normalized counter、topic/stream identity、QoS 設定と合わせて publish する。`last_gain` は backend success 後にだけ commit する。diagnostics QoS は `diagnostics.qos.*` で明示し、system default QoS へ暗黙 fallback しない。

## 9. Node Core

`FaNormalizeNode` は `rclcpp::NodeOptions` を受け取る node core として実装する。
executable entrypoint は `src/main.cpp` に分離し、graph test は node core を直接生成して
publisher / subscriber 経路を検証する。これにより launch file だけでなく ROS graph 上の
metadata 保持、`stream_id` 更新、normalize 結果、drop 条件を実行可能な test で確認する。
