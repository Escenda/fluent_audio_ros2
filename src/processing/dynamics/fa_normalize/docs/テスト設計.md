# fa_normalize テスト設計

## 1. Python unit contract tests

`test/unit/test_fa_normalize_audio_frame_contract.py` は package source を検査し、以下を確認する。

- example config が `FLOAT32LE` / 32-bit / interleaved 契約を満たす
- ROS topic と `AudioFrame.stream_id` を分離し、衝突する設定を fail closed する
- `target_peak_linear` と `silence_threshold_linear` の値と範囲
- device I/O、resampling、format conversion、channel conversion、compressor、limiter、gate、filter、denoise を含まない
- 起動時設定不正を `std::runtime_error` で fail closed する
- runtime `AudioFrame` contract を処理前に検証する
- node が `NodeOptions` を受け取り、`main()` が `src/main.cpp` に分離されている
- `stream_id == input_stream_id` を要求し、出力 `stream_id` を `output.stream_id` に更新する
- peak/gain algorithm が backend にあり、node からは消えている
- silence threshold 未満では backend が data を pass-through する
- backend が ROS2 topic/message を知らない
- input/output sample の finite と normalized range を backend が検証し、clamp しない
- 拒否時は output vector と node の `last_gain` を commit しない
- diagnostics が target、threshold、last_gain、counter、topic/stream identity、QoS 設定を publish する
- runtime config / backend config が意味のある default を持たず、launch default や system default QoS へ fallback しない
- package layout と colcon pytest / backend gtest / ROS graph gtest registration が存在する

## 2. C++ backend tests

`test/cpp/test_internal_peak_normalize_backend.cpp` は ROS を使わず backend を直接検証する。

- 不正 config を constructor で拒否する
- valid frame の peak を target peak に正規化する
- silence frame は unity gain で pass-through する
- 空入力、misaligned input、非 finite input、範囲外 input を status で拒否する
- 拒否時に output vector を更新しない

## 3. ROS graph tests

`test/cpp/test_normalize_graph.cpp` は `NodeOptions` で `fa_normalize` を起動し、
ROS2 executor 上で publisher/subscriber を接続して以下を確認する。

- valid `FLOAT32LE` frame が publish される
- output frame は `source_id`、format metadata、`epoch` を保持し、`stream_id` を `output.stream_id` に更新する
- peak normalize により frame peak が `normalize.target_peak_linear` になる
- `stream_id` が `input_stream_id` と一致しない frame は publish されない

## 4. Launch tests

`test/launch/test_fa_normalize_launch_contract.py` は以下を確認する。

- launch argument が `node_name` / `config_file` のみに閉じている
- default config が dynamics node の責務に閉じ、resample / limiter / gain を含まない
- `normalize.target_peak_linear <= 0.0` で launch 経路が fail closed する
- `normalize.silence_threshold_linear >= normalize.target_peak_linear` で launch 経路が fail closed する
- wrong `expected.encoding` で launch 経路が fail closed する
- `input_stream_id` / `output.stream_id` が同一、または topic 名と衝突する設定で launch 経路が fail closed する

## 5. Manual validation command

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/dynamics/fa_normalize/test/unit src/processing/dynamics/fa_normalize/test/launch -q
```
