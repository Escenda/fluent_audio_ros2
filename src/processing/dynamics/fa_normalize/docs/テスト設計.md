# fa_normalize テスト設計

## 1. Unit contract tests

`test/unit/test_fa_normalize_audio_frame_contract.py` は package source を検査し、以下を確認する。

- default config が `FLOAT32LE` / 32-bit / interleaved 契約を満たす
- `target_peak_linear` と `silence_threshold_linear` の default と範囲
- device I/O、resampling、format conversion、channel conversion、compressor、limiter、gate、filter、denoise を含まない
- 起動時設定不正を `std::runtime_error` で fail closed する
- runtime `AudioFrame` contract を処理前に検証する
- `stream_id == input_topic` を要求し、出力 `stream_id` を `output_topic` に更新する
- peak/gain algorithm が `target_peak / peak` である
- silence threshold 未満では data を pass-through する
- input/output sample の finite と normalized range を検証し、clamp しない
- diagnostics が target、threshold、last_gain、counter を publish する
- package layout と colcon pytest registration が存在する

## 2. Integration tests

将来の integration test では ROS2 executor 上で publisher/subscriber を接続し、valid frame が publish されること、invalid frame が publish されないこと、diagnostics が定期 publish されることを確認する。

## 3. Launch tests

将来の launch test では `launch/fa_normalize.launch.py` が default config を読み、node 名と parameter file を launch argument で変更できることを確認する。

## 4. Manual validation command

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/dynamics/fa_normalize/test/unit -q
```
