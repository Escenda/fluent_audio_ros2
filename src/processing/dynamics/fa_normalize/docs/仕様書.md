# fa_normalize 仕様書

## 1. 目的

`fa_normalize` は音量・ダイナミクス系の processing node である。入力された `FLOAT32LE` interleaved `AudioFrame` の frame 内 peak absolute value を基準に、per-frame peak normalization を適用して別 topic へ publish する。

この node は peak normalize 専用であり、LUFS / loudness normalization ではない。

DSP 本体は ROS 非依存の `internal_peak_normalize` backend に置く。ROS node は `AudioFrame` 契約検証、topic 入出力、diagnostics、counter 更新を担当する。

## 2. 非目的

- device I/O
- resampling
- sample format conversion
- channel conversion
- compressor / limiter / gate
- filter / EQ
- denoise
- clamp による範囲補正
- legacy / compatibility alias

## 3. Node

| item | value |
| --- | --- |
| package | `fa_normalize` |
| executable | `fa_normalize_node` |
| node default name | `fa_normalize` |
| node core | `fa_normalize_node_core` |

## 4. Topic

| direction | parameter | type | default |
| --- | --- | --- | --- |
| subscribe | `input_topic` | `fa_interfaces/msg/AudioFrame` | `audio/noise_gated/mic` |
| publish | `output_topic` | `fa_interfaces/msg/AudioFrame` | `audio/normalized/mic` |

## 5. 起動時設定契約

| parameter | rule |
| --- | --- |
| `input_topic` | non-empty |
| `output_topic` | non-empty |
| `normalize.target_peak_linear` | finite, `> 0.0`, `<= 1.0` |
| `normalize.silence_threshold_linear` | finite, `>= 0.0`, `< normalize.target_peak_linear` |
| `expected.sample_rate` | `> 0` |
| `expected.channels` | `> 0` |
| `expected.encoding` | `FLOAT32LE` |
| `expected.bit_depth` | `32` |
| `expected.layout` | `interleaved` |
| `qos.depth` | `> 0` |
| `diagnostics.publish_period_ms` | `> 0` |

不正な起動時設定は `std::runtime_error` で fail closed する。値の補正や default への差し替えは行わない。

## 6. Runtime AudioFrame 契約

入力 frame は以下を満たす必要がある。

- `source_id` が non-empty
- `stream_id` が non-empty
- `stream_id == input_topic`
- `sample_rate` が設定値と一致
- `channels` が設定値と一致
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `layout == interleaved`
- `data` が non-empty
- `data.size()` が `channels * sizeof(float)` の倍数
- 全 input sample が finite
- 全 input sample が normalized `[-1.0, 1.0]`

不正 frame は warning を出して drop する。

## 7. 出力契約

出力 frame は入力 frame の以下を保持する。

- `header`
- `source_id`
- `sample_rate`
- `channels`
- `bit_depth`
- `encoding`
- `layout`
- `epoch`

`stream_id` は `output_topic` に更新する。`data` は silence pass-through の場合は入力と同一 byte列、normalize の場合は peak normalize 後の `FLOAT32LE` byte列とする。

## 8. Diagnostics

`diagnostics` topic に `diagnostic_msgs/msg/DiagnosticArray` を publish する。status values には少なくとも以下を含める。

- `target_peak_linear`
- `silence_threshold_linear`
- `last_gain`
- `expected_sample_rate`
- `expected_channels`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `frames_silence_passthrough`
- `frames_normalized`
- `output_topic`

`last_gain` は backend が成功した frame でのみ更新する。silence pass-through は `1.0`、normalize は計算 gain を commit する。拒否された frame は直前の `last_gain` を保持する。

## 9. Error Handling

- 起動時設定不正: fail closed
- null `AudioFrame` pointer: warning + drop
- Runtime frame contract 違反: warning + drop
- input sample が finite でない、または `[-1.0, 1.0]` 外: warning + drop
- normalize 出力が finite でない、または `[-1.0, 1.0]` 外: warning + drop

出力値の clamp は行わない。backend が拒否した場合、output vector と `last_gain` は更新しない。

## 10. Launch Contract

- `launch/fa_normalize.launch.py` は `node_name` と `config_file` だけを launch argument として公開する。
- `normalize.*`、`expected.*`、QoS、diagnostics は config file で指定する。
- `FaNormalizeNode` は `rclcpp::NodeOptions` を受け取り、ROS graph test は node core を直接生成して parameter override できる。
- `normalize.target_peak_linear <= 0.0`、`normalize.silence_threshold_linear >= normalize.target_peak_linear`、format contract 不一致は launch 経路でも fail closed する。

## 11. Traceability

- `FA-NORMALIZE-SPEC-001`: `fa_normalize` は device I/O や format conversion を行わない。
- `FA-NORMALIZE-SPEC-002`: `fa_normalize` は `FLOAT32LE` / 32bit / interleaved のみ扱う。
- `FA-NORMALIZE-SPEC-003`: 起動時 config は範囲検証し、不正値は fail closed する。
- `FA-NORMALIZE-SPEC-004`: valid frame は peak が `normalize.target_peak_linear` になるよう per-frame normalize する。
- `FA-NORMALIZE-SPEC-005`: silence frame は sample data を増幅せず pass-through する。
- `FA-NORMALIZE-SPEC-006`: 出力 `source_id` は保持し、`stream_id` は `output_topic` にする。
- `FA-NORMALIZE-SPEC-007`: 入力 frame の `stream_id` は `input_topic` と一致していなければならない。
- `FA-NORMALIZE-SPEC-008`: null frame pointer は drop する。
- `FA-NORMALIZE-SPEC-009`: launch は `node_name` / `config_file` だけを公開し、node 固有 parameter は config file に閉じる。
- `FA-NORMALIZE-SPEC-010`: ROS graph 経路で valid frame は publish され、stream mismatch frame は publish されない。
