# fa_normalize 仕様書

## 1. 目的

`fa_normalize` は音量・ダイナミクス系の processing node である。入力された `FLOAT32LE` interleaved `AudioFrame` の frame 内 peak absolute value を基準に、per-frame peak normalization を適用して別 topic へ publish する。

この node は peak normalize 専用であり、LUFS / loudness normalization ではない。

## 2. 非目的

- device I/O
- resampling
- sample format conversion
- channel conversion
- compressor / limiter / gate
- filter / EQ
- denoise
- clamp による範囲補正
- legacy / compatibility alias

## 3. Node

| item | value |
| --- | --- |
| package | `fa_normalize` |
| executable | `fa_normalize_node` |
| node default name | `fa_normalize` |

## 4. Topic

| direction | parameter | type | default |
| --- | --- | --- | --- |
| subscribe | `input_topic` | `fa_interfaces/msg/AudioFrame` | `audio/noise_gated/mic` |
| publish | `output_topic` | `fa_interfaces/msg/AudioFrame` | `audio/normalized/mic` |

## 5. 起動時設定契約

| parameter | rule |
| --- | --- |
| `input_topic` | non-empty |
| `output_topic` | non-empty |
| `normalize.target_peak_linear` | finite, `> 0.0`, `<= 1.0` |
| `normalize.silence_threshold_linear` | finite, `>= 0.0`, `< normalize.target_peak_linear` |
| `expected.sample_rate` | `> 0` |
| `expected.channels` | `> 0` |
| `expected.encoding` | `FLOAT32LE` |
| `expected.bit_depth` | `32` |
| `expected.layout` | `interleaved` |
| `qos.depth` | `> 0` |
| `diagnostics.publish_period_ms` | `> 0` |

不正な起動時設定は `std::runtime_error` で fail closed する。値の補正や default への差し替えは行わない。

## 6. Runtime AudioFrame 契約

入力 frame は以下を満たす必要がある。

- `source_id` が non-empty
- `stream_id` が non-empty
- `stream_id == input_topic`
- `sample_rate` が設定値と一致
- `channels` が設定値と一致
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `layout == interleaved`
- `data` が non-empty
- `data.size()` が `channels * sizeof(float)` の倍数
- 全 input sample が finite
- 全 input sample が normalized `[-1.0, 1.0]`

不正 frame は warning を出して drop する。

## 7. 出力契約

出力 frame は入力 frame の以下を保持する。

- `header`
- `source_id`
- `sample_rate`
- `channels`
- `bit_depth`
- `encoding`
- `layout`
- `epoch`

`stream_id` は `output_topic` に更新する。`data` は silence pass-through の場合は入力と同一 byte列、normalize の場合は peak normalize 後の `FLOAT32LE` byte列とする。

## 8. Error Handling

- 起動時設定不正: fail closed
- Runtime frame contract 違反: warning + drop
- input sample が finite でない、または `[-1.0, 1.0]` 外: warning + drop
- normalize 出力が finite でない、または `[-1.0, 1.0]` 外: warning + drop

出力値の clamp は行わない。
