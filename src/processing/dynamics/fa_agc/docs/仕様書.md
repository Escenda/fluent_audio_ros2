# fa_agc 仕様書

## 1. 目的

`fa_agc` は、正規化済み FLOAT32LE interleaved `AudioFrame` stream に対し、frame RMS に基づく automatic gain control を行う。

対象は stream data の digital gain 制御であり、device gain、`fa_in`、resampling、format conversion、limiter、compressor、normalize は責務外である。

## 2. Package / Executable / Node

| 項目 | 値 |
| --- | --- |
| package | `fa_agc` |
| executable | `fa_agc_node` |
| node default name | `fa_agc` |
| node core | `fa_agc_node_core` |

## 3. 入出力

| 種別 | パラメータ | Message |
| --- | --- | --- |
| subscribe | `input_topic` | `fa_interfaces/msg/AudioFrame` |
| publish | `output_topic` | `fa_interfaces/msg/AudioFrame` |

`input_topic` / `output_topic` は ROS2 transport topic、`input_stream_id` / `output.stream_id` は `AudioFrame.stream_id` として扱う stream identity である。topic 名を stream identity として代用しない。

## 4. AudioFrame 契約

入力 frame は以下を満たす必要がある。

| Field | 条件 |
| --- | --- |
| `source_id` | 非空 |
| `stream_id` | 非空、かつ `input_stream_id` と一致 |
| `sample_rate` | `expected.sample_rate` と一致、かつ `> 0` |
| `channels` | `expected.channels` と一致、かつ `> 0` |
| `encoding` | `FLOAT32LE` |
| `bit_depth` | `32` |
| `layout` | `interleaved` |
| `data` | 非空、かつ `channels * sizeof(float)` の倍数 |

入力 sample は finite かつ `[-1.0, 1.0]` の正規化範囲内でなければならない。出力 sample が finite でない、または正規化範囲を超える場合、その frame は drop する。clamp や limiter による補正は行わない。

## 5. Parameter

| Parameter | Required | 条件 |
| --- | --- | --- |
| `input_topic` | yes | 非空 |
| `output_topic` | yes | 非空、`input_topic` と解決後に異なる |
| `input_stream_id` | yes | 非空、raw/resolved topic 名と一致しない |
| `output.stream_id` | yes | 非空、`input_stream_id` と異なる、raw/resolved topic 名と一致しない |
| `agc.target_rms` | yes | finite、`> 0.0`、`<= 1.0` |
| `agc.min_gain` | yes | finite、`> 0.0`、`<= 1.0` |
| `agc.max_gain` | yes | finite、`>= agc.min_gain`、`>= 1.0` |
| `agc.attack_ms` | yes | finite、`> 0.0` |
| `agc.release_ms` | yes | finite、`> 0.0` |
| `expected.sample_rate` | yes | `0 < value <= 384000` |
| `expected.channels` | yes | `0 < value <= 64` |
| `expected.encoding` | yes | `FLOAT32LE` 固定 |
| `expected.bit_depth` | yes | `32` 固定 |
| `expected.layout` | yes | `interleaved` 固定 |
| `qos.depth` | yes | `> 0` |
| `qos.reliable` | yes | bool |
| `diagnostics.qos.depth` | yes | `> 0` |
| `diagnostics.qos.reliable` | yes | bool |
| `diagnostics.publish_period_ms` | yes | `> 0` |

## 6. AGC state

起動直後の gain は `1.0` とする。そのため `agc.min_gain <= 1.0 <= agc.max_gain` を起動時に要求する。

各 valid frame で frame RMS を計算し、`target_rms / frame_rms` を gain target として `min_gain` から `max_gain` の範囲に制限する。`frame_rms == 0.0` の場合は `target_rms / frame_rms` の極限として `max_gain` を target とする。これは silence を valid audio frame として扱うための明示仕様であり、不正 sample や欠損 data を正常扱いする fallback ではない。

gain を下げる場合は `attack_ms`、gain を上げる場合は `release_ms` から frame duration に基づく smoothing coefficient を計算する。

## 7. 出力 metadata

出力 frame は入力 frame を基準に以下を保持・更新する。

| Field | 扱い |
| --- | --- |
| `header` | 入力を保持 |
| `source_id` | 入力を保持 |
| `sample_rate` | 入力を保持 |
| `channels` | 入力を保持 |
| `bit_depth` | 入力を保持 |
| `encoding` | 入力を保持 |
| `layout` | 入力を保持 |
| `epoch` | 入力を保持 |
| `stream_id` | `output.stream_id` に更新 |
| `data` | AGC 後 FLOAT32LE sample bytes |

## 8. Diagnostics

`diagnostics` topic に `diagnostic_msgs/msg/DiagnosticArray` を publish する。status values には少なくとも以下を含める。

- `target_rms`
- `min_gain`
- `max_gain`
- `attack_ms`
- `release_ms`
- `current_gain`
- `last_frame_rms`
- `last_target_gain`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `gain_reductions`
- `gain_increases`
- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `qos.depth`
- `diagnostics.qos.depth`
- `diagnostics.qos.reliable`

## 9. 失敗時動作

- 起動時設定不正: 例外により node 起動を失敗させる。
- runtime frame 契約違反: warning を出して frame を drop する。
- 入力 sample 不正: warning を出して frame を drop する。
- 出力 sample 範囲超過: warning を出して frame を drop する。この場合、candidate gain は current gain として確定しない。

意味を変える fallback、legacy alias、暗黙の補正は行わない。

## 10. Launch Contract

`launch/fa_agc.launch.py` が受け取る launch argument は required `node_name` と required `config_file` のみとする。package default config への fallback は持たない。AGC parameter、expected format、QoS は launch argument として露出せず、必ず `config_file` から読む。

node 実装は `fa_agc_node_core` と `src/main.cpp` に分離し、ROS graph test から `NodeOptions` で parameter override できる。

## 11. Traceability

| ID | 契約 |
| --- | --- |
| `FA-AGC-SPEC-001` | `fa_agc` は frame RMS based AGC のみを行い、I/O、device gain、resample、format conversion、filter、denoise、compressor、limiter、normalize を持たない |
| `FA-AGC-SPEC-002` | 起動時 config は必須 topic、AGC range、initial gain range、expected format、QoS、diagnostics period を fail closed で検証する |
| `FA-AGC-SPEC-003` | backend は ROS2 topic/message を知らず、frame RMS、target gain、attack/release smoothing、sample 検証を所有する |
| `FA-AGC-SPEC-004` | node は runtime `AudioFrame` metadata と byte layout を処理前に検証し、違反 frame を publish しない |
| `FA-AGC-SPEC-005` | 出力 frame は identity metadata を保持し、`stream_id` だけを `output.stream_id` に更新する |
| `FA-AGC-SPEC-006` | backend 拒否時は candidate gain と output vector を確定せず、node は frame を drop する |
| `FA-AGC-SPEC-007` | diagnostics は backend state、AGC parameter、frame counter、gain direction counter を publish する |
| `FA-AGC-SPEC-008` | launch は `node_name` と `config_file` のみを受け、config 不正時は起動失敗する |
| `FA-AGC-SPEC-009` | node core は `NodeOptions` で生成でき、graph test から parameter override できる |
| `FA-AGC-SPEC-010` | ROS transport topic と frame stream identity は別 parameter として扱う |
| `FA-AGC-SPEC-011` | diagnostics QoS は required parameter とし、暗黙 default を使わない |
