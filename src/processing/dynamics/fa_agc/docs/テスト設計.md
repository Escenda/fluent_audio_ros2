# fa_agc テスト設計

## 1. Python unit contract tests

`test/unit/test_fa_agc_audio_frame_contract.py` は package source を検査し、以下を確認する。

- example config が `FLOAT32LE` / 32-bit / interleaved 契約を満たす
- AGC parameter の明示例と範囲
- C++ config type が AGC parameter の意味ある runtime default を持たない
- device I/O、device gain、`fa_in`、resampling、format conversion、channel conversion、compressor、limiter、gate、filter、denoise を含まない
- 起動時設定不正を `std::runtime_error` で fail closed する
- runtime `AudioFrame` contract を処理前に検証する
- `stream_id == input_stream_id` を要求し、出力 `stream_id` を `output.stream_id` に更新する
- topic 名と stream identity の混同を起動時に拒否する
- diagnostics QoS が required parameter で、system default QoS に依存しないことを確認する
- frame RMS、bounded target gain、attack/release smoothing algorithm が backend にある
- backend が ROS2 topic/message を知らない
- input/output sample の finite と normalized range を backend が検証し、clamp しない
- output 範囲超過時は publish せず、candidate gain と output を確定しない
- diagnostics が backend state、AGC parameter、counter を publish する
- node が `NodeOptions` を受け取り、`main()` が `src/main.cpp` に分離されている
- package layout が `src/main.cpp`、graph test、launch test を含む標準 layout を満たす
- colcon pytest、backend gtest、graph gtest registration が存在する

## 2. C++ backend tests

`test/cpp/test_internal_rms_agc_backend.cpp` は ROS を使わず backend を直接検証する。

- 不正 config を constructor で拒否する
- target RMS と frame RMS が一致する場合は unity gain を維持する
- silence は valid input として `max_gain` を target にする
- 空入力、misaligned input、非 finite input、範囲外 input を status で拒否する
- attack と release で異なる smoothing time constant を使う
- output 範囲超過時に backend state と output vector を更新しない

## 3. Integration tests

`test/cpp/test_agc_graph.cpp` は `NodeOptions` で `fa_agc` を起動し、ROS2 executor 上で publisher/subscriber を接続する。

- valid frame が publish されること
- AGC gain が data に反映されること
- `source_id`、format、`epoch` が保持され、`stream_id` が `output.stream_id` に更新されること
- `stream_id != input_stream_id` の frame が publish されないこと

## 4. Launch tests

`test/launch/test_fa_agc_launch_contract.py` は launch file と example config fixture を検査し、ROS2 が利用可能な環境では不正 config の起動失敗を確認する。

- launch argument は required `node_name` / required `config_file` のみ
- package default config path へ fallback しない
- example config fixture は AGC 専用の dynamics node として resample / normalize / compressor / limiter を含まない
- `agc.target_rms <= 0.0` は起動失敗
- `agc.min_gain <= 1.0 <= agc.max_gain` を満たさない config は起動失敗
- `agc.attack_ms <= 0.0` は起動失敗
- `expected.encoding != FLOAT32LE` は起動失敗
- `input_stream_id == output.stream_id` は起動失敗
- stream identity が raw/resolved topic 名と一致する場合は起動失敗

## 5. Manual validation command

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/dynamics/fa_agc/test/unit src/processing/dynamics/fa_agc/test/launch -q
```
