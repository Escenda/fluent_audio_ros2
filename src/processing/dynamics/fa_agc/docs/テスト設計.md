# fa_agc テスト設計

## 1. Python unit contract tests

`test/unit/test_fa_agc_audio_frame_contract.py` は package source を検査し、以下を確認する。

- default config が `FLOAT32LE` / 32-bit / interleaved 契約を満たす
- AGC parameter の default と範囲
- device I/O、device gain、`fa_in`、resampling、format conversion、channel conversion、compressor、limiter、gate、filter、denoise を含まない
- 起動時設定不正を `std::runtime_error` で fail closed する
- runtime `AudioFrame` contract を処理前に検証する
- `stream_id == input_topic` を要求し、出力 `stream_id` を `output_topic` に更新する
- frame RMS、bounded target gain、attack/release smoothing algorithm が backend にある
- backend が ROS2 topic/message を知らない
- input/output sample の finite と normalized range を backend が検証し、clamp しない
- output 範囲超過時は publish せず、candidate gain と output を確定しない
- diagnostics が backend state、AGC parameter、counter を publish する
- package layout と colcon pytest / gtest registration が存在する

## 2. C++ backend tests

`test/cpp/test_internal_rms_agc_backend.cpp` は ROS を使わず backend を直接検証する。

- 不正 config を constructor で拒否する
- target RMS と frame RMS が一致する場合は unity gain を維持する
- silence は valid input として `max_gain` を target にする
- 空入力、misaligned input、非 finite input、範囲外 input を status で拒否する
- attack と release で異なる smoothing time constant を使う
- output 範囲超過時に backend state と output vector を更新しない

## 3. Integration tests

将来の integration test では ROS2 executor 上で publisher/subscriber を接続し、valid frame が publish されること、invalid frame が publish されないこと、diagnostics が定期 publish されることを確認する。

## 4. Launch tests

将来の launch test では `launch/fa_agc.launch.py` が default config を読み、node 名と parameter file を launch argument で変更できることを確認する。
