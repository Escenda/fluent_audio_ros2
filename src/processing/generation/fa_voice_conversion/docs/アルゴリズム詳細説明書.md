# fa_voice_conversion アルゴリズム詳細説明書

1. startup config で backend、model path/model id、target voice condition、expected format を検証する。
2. input `AudioFrame` の identity と format metadata を検証する。
3. backend に canonical speech payload と target voice condition を渡す。
4. backend output の sample rate、channels、finite/normalized contract を検証する。
5. 契約を満たす場合だけ publish する。

target voice condition が欠損している場合に default voice へ fallback しない。
Python version / venv が異なる backend は external worker / process / container 境界にする。

## Backend Boundary

backend は voice conversion inference のみを担当し、ROS2 topic/message を知らない。
