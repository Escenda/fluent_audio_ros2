# fa_neural_vocoder アルゴリズム詳細説明書

1. startup config で backend、model path/model id、feature schema、expected output format を検証する。
2. input feature payload の shape、dtype、hop length、conditioning を検証する。
3. backend に canonical feature tensor を渡す。
4. backend output waveform の sample rate、channels、finite/normalized contract を検証する。
5. 契約を満たす場合だけ `AudioFrame` として publish する。

feature schema が一致しない場合に reshape や default conditioning を隠さない。
Python version / venv が異なる backend は external worker / process / container 境界にする。

## Backend Boundary

backend は neural vocoder inference のみを担当し、ROS2 topic/message を知らない。
