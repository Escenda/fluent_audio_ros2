# fa_music_source_separation アルゴリズム詳細説明書

1. startup config で target stem list、backend、model path/model id、expected format を検証する。
2. input `AudioFrame` の identity と format metadata を検証する。
3. backend に canonical audio tensor と target stem list を渡す。
4. backend output labels と count が target stem list と一致することを検証する。
5. 各 output の finite/normalized contract を検証し、契約を満たす場合だけ publish する。

unknown stem を推測して topic を増減しない。
mixing や monitor routing は `src/processing/routing` の専用 node が担当する。

## Backend Boundary

backend は music separation inference のみを担当し、ROS2 topic/message を知らない。
Python version / venv が異なる backend は external worker / process / container 境界にする。
