# fa_tts アルゴリズム詳細説明書

## 処理手順

```text
Speak request
  ↓
validate text / voice
  ↓
resolve cache key
  ↓
load PCM cache or synthesize with pyopenjtalk
  ↓
normalize waveform scale
  ↓
encode PCM16LE AudioFrame
  ↓
publish output
```

## waveform normalization

pyopenjtalk の戻り値は環境によって振幅 scale が異なることがある。絶対値最大が 1.0 を超える場合は 16bit scale とみなし、`32768.0` で正規化する。

## frame / buffer 前提

- 出力は PCM16LE。
- 現在は mono。
- `AudioFrame.epoch` は playback routing の責務なので `fa_tts` では設定しない。

## rejected request fields

- `request.play=true` は playback routing の責務なので拒否する。
- `request.volume_db != 0.0` は dynamics / routing 側の gain 責務なので拒否する。

## known limitations

- backend は pyopenjtalk のみ。
- cache metadata は簡易 text file であり、境界 validation を強化する余地がある。
