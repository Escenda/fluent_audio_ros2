# fa_tts テスト設計

## unit test

- empty text を拒否する。
- cache key が text / voice / volume で変わる。
- waveform scale normalization。
- `stop_topic` 受信で epoch が進む。

## integration test

- `Speak` service から `audio/tts/frame` が publish される。
- `request.play=true` で playback topic にも publish される。

## launch test

- `config/default.yaml` で起動できる環境では起動する。
- pyopenjtalk がない環境では起動失敗する。

## failure test

- synthesis failure。
- cache directory write failure。
- invalid voice。

## latency test

cache miss / cache hit の service latency を分けて測る。

## real device smoke test

`fa_tts -> fa_out` で実 speaker から音声が出ること、stop 後の古い epoch が後段で破棄されることを確認する。
