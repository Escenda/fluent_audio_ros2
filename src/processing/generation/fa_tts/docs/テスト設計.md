# fa_tts テスト設計

## traceability

| test ID | 対応仕様 | 観点 |
| --- | --- | --- |
| `FA-TTS-TC-001` -> `FA-TTS-SPEC-001` | generation boundary | default config と node source に playback topic / stop topic / gain parameter を持たないこと。 |
| `FA-TTS-TC-002` -> `FA-TTS-SPEC-002` | required startup config | `backend.name`、output topic / identity、QoS が typed required parameter として宣言され、runtime default を持たないこと。 |
| `FA-TTS-TC-003` -> `FA-TTS-SPEC-003` | explicit backend selection | missing / unknown `backend.name` を拒否し、cloud fallback を選択しないこと。 |
| `FA-TTS-TC-004` -> `FA-TTS-SPEC-004` | OpenJTalk dictionary contract | `pyopenjtalk` backend が concrete dictionary directory を要求し、home directory fallback を使わないこと。 |
| `FA-TTS-TC-005` -> `FA-TTS-SPEC-005` | backend boundary | backend が ROS2 topic / message を import せず、node が `build_tts_backend` で backend を選択すること。 |
| `FA-TTS-TC-006` -> `FA-TTS-SPEC-006` | waveform scale contract | normalized 範囲外の waveform を scale 推測 / clip / PCM16 encode で吸収しないこと。 |
| `FA-TTS-TC-007` -> `FA-TTS-SPEC-007` | no playback or gain request | `request.play=true` と `request.volume_db != 0.0` を拒否すること。 |
| `FA-TTS-TC-008` -> `FA-TTS-SPEC-008` | cache key safety | `cache_key` を 40 lowercase hex characters だけに制限し、path traversal を許可しないこと。 |
| `FA-TTS-TC-009` -> `FA-TTS-SPEC-009` | cache metadata validation | cache metadata の encoding / sample rate / channels / bit depth を検証し、invalid cache を payload として扱わないこと。 |
| `FA-TTS-TC-010` -> `FA-TTS-SPEC-010` | failure before publish | synthesis / cache write / empty text / invalid voice を response error とし、成功 publish に見せないこと。 |
| `FA-TTS-TC-011` -> `FA-TTS-SPEC-011` | launch contract | launch file が `node_name` / `config_file` だけを要求し、package-local default config を推測しないこと。 |
| `FA-TTS-TC-012` -> `FA-TTS-SPEC-012` | colcon coverage | package の pytest contract が `colcon test` から実行されること。 |

## unit test

- empty text を拒否する。
- cache key が text / voice で変わる。
- waveform scale contract validation。
- `backend.openjtalk_dict_dir` missing で起動失敗。
- pyopenjtalk waveform が normalized 範囲外の場合に scale 推測 / clip せず failure。
- cache metadata に encoding が無い場合は invalid cache として扱う。
- backend は `FLOAT32LE/32` payload を返し、PCM16 encode を行わない。
- launch は `node_name` / `config_file` の runtime default を持たない。
- node parameter は typed required declaration にし、runtime default を持たない。
- `output.stream_id` は `output_topic` / `output.source_id` と同一なら起動失敗する。
- `request.play=true` を拒否する。
- `request.volume_db != 0.0` を拒否する。

## integration test

- `Speak` service から `audio/tts/frame` が publish される。
- `fa_tts -> format conversion -> fa_mix -> fa_out` の routing で playback される。

## launch test

- `node_name` と `config_file` を明示したときだけ launch action が作成される。
- launch file は `FindPackageShare` / `PathJoinSubstitution` で package-local config を推測しない。
- pyopenjtalk がない環境では起動失敗する。

## failure test

- synthesis failure。
- cache directory write failure。
- invalid voice。

## latency test

cache miss / cache hit の service latency を分けて測る。

## real device smoke test

`fa_tts -> format conversion -> fa_mix -> fa_out` で実 speaker から音声が出ること、stop 後の古い epoch が後段で破棄されることを確認する。
