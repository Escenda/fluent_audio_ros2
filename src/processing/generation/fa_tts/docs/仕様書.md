# fa_tts 仕様書

## 目的

`fa_tts` は text-to-speech generation node である。`Speak` service request を受け、合成した `AudioFrame` を `audio/tts/frame` へ publish する。現在の backend は `pyopenjtalk` であり、OpenAI などの cloud TTS には依存しない。

## 入力 topic

なし。

## service

| service | type | 内容 |
| --- | --- | --- |
| `speak` | `fa_interfaces/srv/Speak` | text / voice 指定を受けて音声合成する |

## 出力 topic

| parameter | default | 内容 |
| --- | --- | --- |
| `output_topic` | `audio/tts/frame` | 合成済み音声 |

## parameters

| parameter | 意味 |
| --- | --- |
| `default_voice` | request voice が空の場合に使う voice |
| `output_topic` | TTS frame publish topic |
| `cache_dir` | PCM cache directory。既定は `~/.cache/fluent_audio/tts`。空なら disk cache 無効 |

## lifecycle

起動時に pyopenjtalk runtime と parameter を検証する。runtime がない場合は import failure で起動失敗する。

## QoS

TTS output は reliable QoS を使う。audio frame を command state と混同しない。

## error handling

空 text、`request.play=true`、`request.volume_db != 0.0`、synthesis failure、cache write failure は response に明示する。runtime import failure を握りつぶさない。

## backend selection

現在は `pyopenjtalk` backend のみ。別 backend を追加する場合は `backends/` に分離し、`backend.name` を明示 config にする。

## non-goals

- ASR
- playback device I/O
- playback routing
- gain / normalize / limiter
- dialogue policy
- cloud API fallback
