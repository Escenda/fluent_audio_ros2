# fa_speech_translation アルゴリズム詳細説明書

1. startup config で source language、target language、backend、model path/model id、expected format を検証する。
2. input `AudioFrame` の identity と format metadata を検証する。
3. backend に canonical speech payload と language contract を渡す。
4. backend output の target language、sample rate、channels、finite/normalized contract を検証する。
5. 契約を満たす場合だけ publish する。

ASR-only や TTS-only の fallback に切り替えない。
Python version / venv が異なる backend は external worker / process / container 境界にする。

## Backend Boundary

backend は speech-to-speech translation のみを担当し、ROS2 topic/message を知らない。
