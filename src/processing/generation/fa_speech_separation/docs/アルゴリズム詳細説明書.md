# fa_speech_separation アルゴリズム詳細説明書

1. startup config で source count、target labels、backend、model path/model id、expected format を検証する。
2. input `AudioFrame` の identity と format metadata を検証する。
3. backend に canonical audio tensor と target labels を渡す。
4. backend output count と labels が config と一致することを検証する。
5. 各 output の finite/normalized contract を検証し、契約を満たす場合だけ publish する。

unknown source count を推測して topic を増減しない。
speaker identity は別の AI package が担当する。

## Backend Boundary

backend は speech separation inference のみを担当し、ROS2 topic/message を知らない。
Python version / venv が異なる backend は external worker / process / container 境界にする。
