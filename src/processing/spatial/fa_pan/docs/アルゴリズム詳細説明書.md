# fa_pan アルゴリズム詳細説明書

## 入力表現

入力は FLOAT32LE interleaved stereo `AudioFrame` である。sample 配列は以下の順序で並ぶ。

```text
L0, R0, L1, R1, ...
```

`data.size()` は `2 * sizeof(float)` の倍数でなければならない。

## Constant-Power Pan

sample loop、FLOAT32LE decode/encode、normalized range validation は ROS-free backend
`internal_constant_power_pan` が担当する。ROS node は topic、`AudioFrame` metadata validation、
diagnostics、`stream_id` 更新だけを担当する。

`pan.position` を `[-1.0, 1.0]` とし、以下を計算する。

```text
angle = (position + 1) * pi / 4
left_gain = cos(angle)
right_gain = sin(angle)
```

各 stereo pair に対して以下を適用する。

```text
out_left = in_left * left_gain
out_right = in_right * right_gain
```

`position=-1.0` では `left_gain=1.0`, `right_gain=0.0`、`position=1.0` では `left_gain=0.0`, `right_gain=1.0` になる。`position=0.0` では両 channel が `sqrt(0.5)` 相当の gain になる。

## Validation

入力 sample は finite かつ normalized `[-1, 1]` でなければならない。出力 sample も同じ範囲で検証する。違反時は frame 全体を drop する。

`fa_pan` は spatial placement であり、以下は行わない。

- clamp
- normalize
- channel-count conversion
- sample format conversion
- resampling
- limiter / filter / denoise

## Byte Order

FLOAT32LE sample は backend 内で 4 byte little-endian として読み書きする。byte 列から
`uint32_t` を組み立て、`std::memcpy` で `float` と相互変換する。
