# fa_pan テスト設計

## Unit Contract Tests

`test/unit/test_fa_pan_audio_frame_contract.py` は package contract を静的に検査する。

- default config が stereo FLOAT32LE interleaved contract を持つこと
- node が unrelated responsibilities を持たないこと
- invalid config で fail closed する validation があること
- runtime frame contract を処理前に検証すること
- ROS topic と `AudioFrame.stream_id` を分離し、`input_stream_id` / `output.stream_id` を使うこと
- `stream_id` を `output.stream_id` に更新し、それ以外の metadata を不要に書き換えないこと
- constant-power gain の式が実装されていること
- finite normalized `[-1, 1]` 違反を drop すること
- diagnostics が `pan.position`、左右 gain、counter を含むこと
- standard processing package layout を満たすこと
- `ament_cmake_pytest` で colcon から pytest が登録されること

## Integration Tests

将来の integration test では、実 ROS2 graph 上で `fa_pan/input` から `fa_pan/output` への publish を確認する。入力 frame の `stream_id` が `input_stream_id` と一致すること、出力 frame の `stream_id` が `output.stream_id` に更新されること、`epoch` と source identity が保持されること、invalid frame が publish されないことを確認対象にする。

## Launch Tests

将来の launch test では `launch/fa_pan.launch.py` が default config を読み込み、`fa_pan_node` を起動できることを確認する。
