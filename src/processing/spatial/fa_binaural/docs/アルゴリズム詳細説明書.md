# fa_binaural アルゴリズム詳細説明書

1. startup config で HRTF profile、input layout、output format を検証する。
2. input `AudioFrame` の identity と format metadata を検証する。
3. backend で channel ごとの HRTF convolution または明示 renderer を適用する。
4. 2ch output と finite/normalized contract を検証する。
5. 契約を満たす場合だけ publish する。

HRTF profile が欠損している場合に generic stereo downmix へ fallback しない。
speaker output は `fa_out` が担当する。

## Backend Boundary

backend は binaural rendering のみを担当し、ROS2 topic/message を知らない。
