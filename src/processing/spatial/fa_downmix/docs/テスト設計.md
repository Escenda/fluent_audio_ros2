# fa_downmix テスト設計

## traceability

| test ID | 対応仕様 | 観点 |
| --- | --- | --- |
| `FA-DOWNMIX-TC-001` -> `FA-DOWNMIX-SPEC-001` | downmix boundary | 明示許可された N-channel to mono/stereo downmix だけを行うこと。 |
| `FA-DOWNMIX-TC-002` -> `FA-DOWNMIX-SPEC-002` | non-goals | upmix、resampling、format conversion、gain、limiter、pan、filtering、denoise、device I/O を持たないこと。 |
| `FA-DOWNMIX-TC-003` -> `FA-DOWNMIX-SPEC-003` | topic / stream identity | topic と stream ID を分離し、衝突を fail closed すること。 |
| `FA-DOWNMIX-TC-004` -> `FA-DOWNMIX-SPEC-004` | required startup config | input/output channels、mode、expected format、QoS、diagnostics parameter を required とすること。 |
| `FA-DOWNMIX-TC-005` -> `FA-DOWNMIX-SPEC-005` | mode contract | `average_to_mono` / `pair_average_to_stereo` だけを許可し、non-downmix を拒否すること。 |
| `FA-DOWNMIX-TC-006` -> `FA-DOWNMIX-SPEC-006` | runtime frame validation | source / stream identity、format、channel、data alignment、sample range を検証すること。 |
| `FA-DOWNMIX-TC-007` -> `FA-DOWNMIX-SPEC-007` | output metadata | output が `stream_id`、`channels`、`data` だけを更新すること。 |
| `FA-DOWNMIX-TC-008` -> `FA-DOWNMIX-SPEC-008` | no hidden correction | invalid sample を clamp / normalize で補正せず drop すること。 |
| `FA-DOWNMIX-TC-009` -> `FA-DOWNMIX-SPEC-009` | diagnostics | mode、format、counter が diagnostics に含まれること。 |
| `FA-DOWNMIX-TC-010` -> `FA-DOWNMIX-SPEC-010` | launch contract | launch file が `node_name` / `config_file` だけを要求し、package default を推測しないこと。 |
| `FA-DOWNMIX-TC-011` -> `FA-DOWNMIX-SPEC-011` | colcon coverage | pytest と lint dependency が colcon manifest 上で成立すること。 |

## Contract test

`test/unit/test_fa_downmix_audio_frame_contract.py` は、package の config、source、layout、colcon pytest registration を検査する。

## 検証項目

- default config が 16000Hz / FLOAT32LE / 32bit / interleaved / 4ch -> 2ch / `pair_average_to_stereo` と topic/stream identity 分離を明示する
- device I/O、resampling、sample format conversion、gain、limiter、pan、filtering、denoise の責務を持たない
- 起動時に未対応 mode/channel combination と upmix/non-downmix を reject する
- runtime frame が `source_id`、`input_stream_id`、format、channel、data size を満たさない場合 drop する
- `stream_id` を `output.stream_id` に、`channels` を `output.channels` に更新し、他 metadata を保持する
- all-channel average mono と pair-average stereo の実装証跡がある
- normalized range と finite 検証がある
- `ament_cmake_pytest` で contract test が colcon から実行される

## 実行コマンド

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/spatial/fa_downmix/test/unit -q
```
