# fa_upmix 仕様書

## 1. 目的

`fa_upmix` は FluentAudio の FLOAT32LE interleaved `AudioFrame` stream に対して、明示された mode に基づくチャンネル複製のみを行う processing node である。

## 1.1. 仕様 ID

| ID | 内容 |
| --- | --- |
| `FA-UPMIX-SPEC-001` | `fa_upmix` は `FLOAT32LE` interleaved `AudioFrame` に対して明示 mode に基づく channel duplication だけを行う spatial processing node である。 |
| `FA-UPMIX-SPEC-002` | device I/O、resample、bit depth / encoding conversion、gain、normalization、limiter、pan、metadata 補完、unsupported mode fallback を行わない。 |
| `FA-UPMIX-SPEC-003` | `input_topic` / `output_topic` と `input_stream_id` / `output.stream_id` は別契約であり、topic identity と stream identity の衝突を起動時に拒否する。 |
| `FA-UPMIX-SPEC-004` | expected format、input/output channel、mode、QoS、diagnostics parameter は required で、不正値は fail closed する。 |
| `FA-UPMIX-SPEC-005` | `mono_duplicate` は 1 channel 入力を `N > 1` channel へ複製し、`stereo_duplicate_pairs` は stereo pair を偶数 channel へ複製する。 |
| `FA-UPMIX-SPEC-006` | runtime frame は source / stream identity、format、data alignment、finite normalized sample を検証し、違反 frame は drop する。 |
| `FA-UPMIX-SPEC-007` | 出力 metadata は入力を原則維持し、`stream_id`、`channels`、`data` だけを更新する。 |
| `FA-UPMIX-SPEC-008` | invalid sample は clamp / normalize で補正せず drop する。 |
| `FA-UPMIX-SPEC-009` | diagnostics は topic、stream id、mode、expected format、output channels、frame counter を publish する。 |
| `FA-UPMIX-SPEC-010` | launch file は `node_name` / `config_file` を明示引数とし、package-local default config を推測しない。 |
| `FA-UPMIX-SPEC-011` | pytest contract は `colcon test` から実行される。 |

## 2. 非目標

- 音声デバイス、ファイル、ネットワークへの入出力
- resample、bit depth 変換、encoding 変換
- gain、normalization、limiter、pan
- 欠損 metadata の補完
- 未対応 mode への fallback

## 3. 起動パラメータ

| 名前 | 必須 | 内容 |
| --- | --- | --- |
| `input_topic` | yes | 入力 `AudioFrame` topic。空文字は禁止 |
| `output_topic` | yes | 出力 `AudioFrame` topic。空文字は禁止 |
| `input_stream_id` | yes | 受理する入力 `AudioFrame.stream_id`。空文字、raw/resolved topic 名との一致は禁止 |
| `expected.sample_rate` | yes | 入力 sample rate。正の整数 |
| `expected.input_channels` | yes | 入力 channel 数。mode と一致する値 |
| `expected.encoding` | yes | `FLOAT32LE` のみ |
| `expected.bit_depth` | yes | `32` のみ |
| `expected.layout` | yes | `interleaved` のみ |
| `output.stream_id` | yes | 出力 `AudioFrame.stream_id`。空文字、raw/resolved topic 名、`input_stream_id` との一致は禁止 |
| `output.channels` | yes | 出力 channel 数。mode と一致する値 |
| `mode` | yes | `mono_duplicate` または `stereo_duplicate_pairs` |
| `qos.depth` | yes | 正の整数 |
| `qos.reliable` | yes | `true` なら reliable、`false` なら best effort |
| `diagnostics.publish_period_ms` | yes | diagnostics publish 間隔。正の整数 |
| `diagnostics.qos.depth` | yes | diagnostics QoS depth。正の整数 |
| `diagnostics.qos.reliable` | yes | diagnostics QoS reliability |

## 4. 対応 mode

### 4.1 `mono_duplicate`

- 入力 channel 数: `1`
- 出力 channel 数: `N > 1`
- 各 frame の mono sample を全出力 channel に同じ順序で書き込む

### 4.2 `stereo_duplicate_pairs`

- 入力 channel 数: `2`
- 出力 channel 数: `N > 2` かつ偶数
- 各 frame の `L,R` を出力 pair 数だけ `L,R,L,R...` の順序で書き込む

## 5. 起動時 fail closed

以下の場合は node 起動を失敗させる。

- 必須 topic が空
- `input_stream_id` または `output.stream_id` が空
- `input_stream_id` または `output.stream_id` が raw/resolved ROS topic 名と一致する
- `input_stream_id` と `output.stream_id` が同一
- sample rate、channel 数、QoS depth、diagnostics period が不正
- encoding / bit depth / layout が契約外
- mode と input/output channel の組み合わせが未対応

## 6. runtime frame validation

受信 frame は publish 前に以下を検証する。

- `source_id` と `stream_id` が空でない
- `stream_id == input_stream_id`
- encoding、bit depth、layout、sample rate、channels が起動時 contract と一致する
- `data` が空でない
- `data.size()` が `expected.input_channels * sizeof(float)` の倍数
- 全 input sample が有限かつ `[-1.0, 1.0]`

検証失敗時は frame を drop し、出力しない。

## 7. 出力 contract

出力 `AudioFrame` は入力 metadata を原則維持し、以下のみ更新する。

- `stream_id = output.stream_id`
- `channels = output.channels`
- `data = upmix 後の FLOAT32LE interleaved bytes`

encoding、bit depth、layout、sample rate、source_id、header、epoch は変更しない。

## 8. diagnostics

`diagnostics` topic に `diagnostic_msgs/msg/DiagnosticArray` を publish する。status には以下を含める。

- input/output topic
- input/output stream id
- mode
- expected sample rate / input channels / encoding / bit depth / layout
- output channels
- `frames.in`
- `frames.out`
- `frames.drop`
