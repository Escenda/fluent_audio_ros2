# fa_upmix テスト設計

## 1. テスト方針

`fa_upmix` は device I/O を持たない processing node である。package-local pytest では、設定 contract、fail closed 条件、runtime validation、metadata 更新範囲、diagnostics、CMake への pytest 登録を静的に確認する。

## 1.1. traceability

| test ID | 対応仕様 | 観点 |
| --- | --- | --- |
| `FA-UPMIX-TC-001` -> `FA-UPMIX-SPEC-001` | upmix boundary | channel duplication だけを行う spatial processing node であること。 |
| `FA-UPMIX-TC-002` -> `FA-UPMIX-SPEC-002` | non-goals | device I/O、format conversion、gain、pan、fallback を持たないこと。 |
| `FA-UPMIX-TC-003` -> `FA-UPMIX-SPEC-003` | topic / stream identity | topic と stream ID を分離し、衝突を fail closed すること。 |

| `FA-UPMIX-TC-005` -> `FA-UPMIX-SPEC-005` | mode algorithm | `mono_duplicate` と `stereo_duplicate_pairs` の channel 複製 contract を満たすこと。 |
| `FA-UPMIX-TC-006` -> `FA-UPMIX-SPEC-006` | runtime frame validation | source / stream identity、format、data alignment、sample range を検証すること。 |
| `FA-UPMIX-TC-007` -> `FA-UPMIX-SPEC-007` | output metadata | output が `stream_id`、`channels`、`data` だけを更新すること。 |

## 2. 単体テスト

### 2.1 default config contract

- `input_topic` / `output_topic` が明示されている
- `input_stream_id` / `output.stream_id` が ROS topic と別 identity として明示されている
- `expected.encoding=FLOAT32LE`
- `expected.bit_depth=32`
- `expected.layout=interleaved`
- default mode が `mono_duplicate`
- default input/output channel が `1 -> 2`

### 2.2 startup validation

- 必須 topic の空文字を拒否する
- `input_stream_id` / `output.stream_id` の空文字を拒否する
- `input_stream_id` / `output.stream_id` と raw/resolved topic 名の衝突を拒否する
- `input_stream_id` と `output.stream_id` の一致を拒否する
- sample rate / channel / QoS / diagnostics period の不正値を拒否する
- encoding / bit depth / layout の不一致を拒否する
- 未対応 mode/channel 組み合わせを拒否する

### 2.3 runtime frame validation

- `source_id` / `stream_id` の欠損を drop
- `stream_id` と `input_stream_id` の mismatch を drop
- encoding / bit depth / layout / sample rate / channels mismatch を drop
- data size 不正を drop

### 2.4 algorithm contract

- `mono_duplicate` が input sample を全 output channel に複製する
- `stereo_duplicate_pairs` が LR pair を偶数出力 channel に複製する
- sample が invalid な場合に clamp / normalize せず drop する

### 2.5 output metadata

- `stream_id` を `output.stream_id` に更新する
- `channels` を `output.channels` に更新する
- `data` を upmix 後の bytes に更新する
- encoding / bit depth / sample rate / layout / epoch は更新しない

## 3. integration / launch

現時点では `.gitkeep` のみ配置する。親 package 登録後に、launch 経由で node を起動し、valid frame が publish され invalid frame が drop されることを ROS2 経路で確認する。
