# fa_upmix アルゴリズム詳細説明書

## 1. 入力 decode

`AudioFrame.data` は `FLOAT32LE` の byte array として扱う。sample index から 4 byte を little endian で `uint32_t` に組み立て、`std::memcpy` で `float` として読む。

byte order の推定や host endian への暗黙依存は行わない。

## 2. normalized sample validation

読み出した sample は処理前に次を満たす必要がある。

```text
std::isfinite(sample) && sample >= -1.0 && sample <= 1.0
```

NaN、Inf、normalized range 外の値は clipping や normalization で補正しない。frame 全体を drop する。

## 3. `mono_duplicate`

入力 channel 数は 1、出力 channel 数は `N > 1`。

各 frame について、mono sample `x` を読み出し、出力 channel 数だけ同じ値を書き込む。

```text
in:  [x0] [x1] [x2]
out: [x0 x0 ... x0] [x1 x1 ... x1] [x2 x2 ... x2]
```

## 4. `stereo_duplicate_pairs`

入力 channel 数は 2、出力 channel 数は `N > 2` かつ偶数。

各 frame について、入力 `L,R` を読み出し、出力 pair 数だけ同じ LR pair を書き込む。

```text
in:  [L0 R0] [L1 R1]
out: [L0 R0 L0 R0 ...] [L1 R1 L1 R1 ...]
```

## 5. 出力 encode

出力 sample は append 前にも normalized finite check を通す。duplicate 処理は値を変えないため通常は input validation と同じ結果になるが、出力 byte 列に invalid sample を入れない contract をコード上で明示する。

`std::memcpy` で `float` から `uint32_t` へ変換し、little endian byte order で `AudioFrame.data` に append する。

## 6. 失敗時の扱い

未対応 mode は起動時に拒否する。runtime で契約違反 frame または invalid sample を検出した場合は、その frame を drop する。意味を変える fallback、clamp、zero fill、channel 推定は行わない。
