# fa_source_separation アルゴリズム詳細説明書

1. startup config で target source list、backend、model path/model id、expected format を検証する。
2. input `AudioFrame` の identity と format metadata を検証する。
3. backend に canonical audio tensor と target source list を渡す。
4. backend output count と labels が target source list と一致することを検証する。
5. 各 output の finite/normalized contract を検証し、契約を満たす場合だけ publish する。

unknown source count を推測して topic を増減しない。
speaker identity や diarization は専用 AI package の責務であり、この node に混ぜない。

## Backend Boundary

backend は source separation inference のみを担当し、ROS2 topic/message を知らない。
Python version / venv が異なる backend は external worker / process / container 境界にする。
