# fa_beamforming テスト設計

## 仕様 ID

| ID | 仕様 |
| --- | --- |
| FA-BEAMFORMING-SPEC-001 | default config は明示的な FLOAT32LE interleaved multi-channel 入力、mono 出力契約、topic/stream identity 分離を持つ |
| FA-BEAMFORMING-SPEC-002 | node は必須 parameter を runtime default なしで宣言する |
| FA-BEAMFORMING-SPEC-003 | 起動時に topic、stream identity、expected、output、QoS、diagnostics、weights を fail closed で検証する |
| FA-BEAMFORMING-SPEC-004 | weights length は `expected.channels` と一致し、finite かつ絶対値和 nonzero でなければならない |
| FA-BEAMFORMING-SPEC-005 | runtime frame は処理前に `AudioFrame` contract を検証する |
| FA-BEAMFORMING-SPEC-006 | 出力は mono FLOAT32LE interleaved で `stream_id=output.stream_id` になる |
| FA-BEAMFORMING-SPEC-007 | algorithm は明示 weights の weighted sum であり、equal weights を推定しない |
| FA-BEAMFORMING-SPEC-008 | invalid sample / invalid output は clamp や normalize せず drop する |
| FA-BEAMFORMING-SPEC-009 | diagnostics は config と frame counter を含む |
| FA-BEAMFORMING-SPEC-010 | package は標準 processing layout を満たす |

## Test Case 対応

| Test case | 対応仕様 |
| --- | --- |
| `test_default_config_requires_explicit_float32le_beamforming_contract` | FA-BEAMFORMING-SPEC-001 |
| `test_required_parameters_are_declared_without_runtime_defaults` | FA-BEAMFORMING-SPEC-002 |
| `test_startup_validation_fails_closed_for_invalid_config` | FA-BEAMFORMING-SPEC-003 |
| `test_weight_validation_requires_explicit_finite_nonzero_channel_weights` | FA-BEAMFORMING-SPEC-004 |
| `test_beamforming_validates_frame_contract_before_processing` | FA-BEAMFORMING-SPEC-005 |
| `test_beamforming_outputs_mono_float32le_interleaved_stream` | FA-BEAMFORMING-SPEC-006 |
| `test_static_delay_sum_algorithm_uses_explicit_weights` | FA-BEAMFORMING-SPEC-007 |
| `test_beamforming_drops_invalid_runtime_samples_instead_of_clamping_or_normalizing` | FA-BEAMFORMING-SPEC-008 |
| `test_diagnostics_include_beamforming_config_and_counters` | FA-BEAMFORMING-SPEC-009 |
| `test_package_layout_matches_standard_processing_layout` | FA-BEAMFORMING-SPEC-010 |
