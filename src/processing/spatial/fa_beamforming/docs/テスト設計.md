# fa_beamforming テスト設計

## traceability

| test ID | 対応仕様 | 観点 |
| --- | --- | --- |
| `FA-BEAMFORMING-TC-001` -> `FA-BEAMFORMING-SPEC-001` | beamforming boundary | fixed-weight beamforming で mono `AudioFrame` を publish すること。 |
| `FA-BEAMFORMING-TC-002` -> `FA-BEAMFORMING-SPEC-002` | non-goals | weight 推定、fallback、format conversion、gain normalize、AEC、denoise、I/O を持たないこと。 |
| `FA-BEAMFORMING-TC-003` -> `FA-BEAMFORMING-SPEC-003` | topic / stream identity | topic と stream ID を分離し、衝突を fail closed すること。 |
| `FA-BEAMFORMING-TC-004` -> `FA-BEAMFORMING-SPEC-004` | weight validation | weights length、finite、nonzero 絶対値和を検証すること。 |
| `FA-BEAMFORMING-TC-005` -> `FA-BEAMFORMING-SPEC-005` | required startup config | expected format、output mono contract、QoS、diagnostics parameter を fail closed で検証すること。 |
| `FA-BEAMFORMING-TC-006` -> `FA-BEAMFORMING-SPEC-006` | runtime frame validation | source / stream identity、format、channel、data alignment、sample range を処理前に検証すること。 |
| `FA-BEAMFORMING-TC-007` -> `FA-BEAMFORMING-SPEC-007` | weighted sum algorithm | 明示 weights の weighted sum を使い、equal weights を推定しないこと。 |
| `FA-BEAMFORMING-TC-008` -> `FA-BEAMFORMING-SPEC-008` | mono output metadata | output が mono FLOAT32LE interleaved で `stream_id=output.stream_id` になること。 |
| `FA-BEAMFORMING-TC-009` -> `FA-BEAMFORMING-SPEC-009` | no hidden correction | invalid input / output を clamp / normalize / zero fill せず drop すること。 |
| `FA-BEAMFORMING-TC-010` -> `FA-BEAMFORMING-SPEC-010` | diagnostics | config と frame counter が diagnostics に含まれること。 |
| `FA-BEAMFORMING-TC-011` -> `FA-BEAMFORMING-SPEC-011` | launch contract | launch file が `node_name` / `config_file` だけを要求し、package default を推測しないこと。 |
| `FA-BEAMFORMING-TC-012` -> `FA-BEAMFORMING-SPEC-012` | colcon coverage | package の pytest contract が `colcon test` から実行されること。 |

## 実行

```bash
pytest ros2_ws/src/fluent_audio_ros2/src/processing/spatial/fa_beamforming/test/unit
```

colcon 経路では `ament_cmake_pytest` から同じ pytest contract を実行する。
