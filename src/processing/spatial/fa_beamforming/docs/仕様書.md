# fa_beamforming 仕様書

## 目的

`fa_beamforming` は、multi-channel microphone array 由来の `AudioFrame` に、起動時 config で明示された固定 weight を適用し、mono の `AudioFrame` を publish する spatial processing node である。

## 非目標

- weight 推定、DOA 推定、adaptive beamforming
- equal weight fallback
- resampling、bit-depth conversion、interleave conversion
- gain normalize、limiter、clipping 補正
- AEC、denoise、source separation
- device / file / network I/O

## 所属分類

- `src/processing/spatial`
- package: `fa_beamforming`

## Topic

| 種別 | 既定 config 例 | message |
| --- | --- | --- |
| subscribe | `fa_beamforming/input` | `fa_interfaces/msg/AudioFrame` |
| publish | `fa_beamforming/output` | `fa_interfaces/msg/AudioFrame` |
| publish | `diagnostics` | `diagnostic_msgs/msg/DiagnosticArray` |

## Message Contract

入力 `AudioFrame` は次を満たす必要がある。

- `source_id` が空でない
- `stream_id == input_stream_id`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `layout == interleaved`
- `data` が空でなく、`expected.channels * sizeof(float)` の倍数
- 各 sample が finite かつ `[-1.0, 1.0]`

出力 `AudioFrame` は次を満たす。

- `header`、`source_id`、`sample_rate`、`epoch` は入力を保持
- `stream_id == output.stream_id`
- `channels == 1`
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `layout == interleaved`
- `data` は frame ごとの weighted sum

## Parameters

すべての parameter は必須である。node は runtime default を持たず、未指定・型不一致・値不正の場合は起動を中止する。

| parameter | 型 | 条件 |
| --- | --- | --- |
| `input_topic` | string | 空でない |
| `output_topic` | string | 空でない |
| `input_stream_id` | string | 空でない。raw/resolved `input_topic` / `output_topic` と一致不可 |
| `output.stream_id` | string | 空でない。raw/resolved `input_topic` / `output_topic` および `input_stream_id` と一致不可 |
| `beamforming.weights` | double array | 長さが `expected.channels` と一致、全要素 finite、絶対値和が 0 より大きい |
| `output.channels` | integer | `1` |
| `expected.sample_rate` | integer | `> 0` |
| `expected.channels` | integer | `> 0` |
| `expected.encoding` | string | `FLOAT32LE` |
| `expected.bit_depth` | integer | `32` |
| `expected.layout` | string | `interleaved` |
| `qos.depth` | integer | `> 0` |
| `qos.reliable` | bool | 明示指定 |
| `diagnostics.publish_period_ms` | integer | `> 0` |
| `diagnostics.qos.depth` | yes | diagnostics QoS depth。正の整数 |
| `diagnostics.qos.reliable` | yes | diagnostics QoS reliability |

## Remapping

topic 名は parameter で指定する。`input_topic` / `output_topic` は ROS 搬送路であり、`AudioFrame.stream_id` ではない。`stream_id` は data plane 上の stream contract として扱い、入力では `input_stream_id`、出力では `output.stream_id` と一致させる。

## Lifecycle / State

Lifecycle node ではない。起動時に parameter を検証し、検証成功後に publisher、subscriber、diagnostics timer を作成する。

## Error / Fail Closed

- 必須 parameter がない場合は起動失敗
- `input_stream_id` / `output.stream_id` が空、topic と衝突、または相互に同一の場合は起動失敗
- weight が不正な場合は起動失敗
- 入力 frame contract が不一致の場合は warning を出して frame drop
- 入力 sample または出力 weighted sum が finite normalized range 外の場合は warning を出して frame drop
- 不正 frame を補正、clamp、normalize、zero 埋めして publish しない

## Diagnostics

`diagnostics` に次を publish する。

- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `beamforming.weights`
- `beamforming.weights_sum_abs`
- `expected.*`
- `output.channels`
- `frames.in`
- `frames.out`
- `frames.drop`

## Performance

処理量は `frame_count * expected.channels` の線形時間である。追加 buffer は出力 `data` 分のみ確保する。

## Config 例

```yaml
fa_beamforming:
  ros__parameters:
    input_topic: "fa_beamforming/input"
    output_topic: "fa_beamforming/output"
    input_stream_id: "audio/spatial/mic"
    beamforming:
      weights: [1.0, 0.0, 0.0, 0.0]
    expected:
      sample_rate: 16000
      channels: 4
      encoding: "FLOAT32LE"
      bit_depth: 32
      layout: "interleaved"
    output:
      channels: 1
      stream_id: "audio/beamformed/mic"
    qos:
      depth: 10
      reliable: false
    diagnostics:
      publish_period_ms: 1000
```

## Backend 選択仕様

初期実装は package 内部の `static_delay_sum` backend 相当のみを提供する。backend 選択 parameter は持たない。
