# fa_ambisonics アルゴリズム詳細説明書

1. startup config で ambisonic order、normalization、channel ordering、coordinate convention を検証する。
2. input `AudioFrame` の identity と format metadata を検証する。
3. backend で明示 transform matrix を構築する。
4. channel matrix transform を適用する。
5. output metadata と finite/normalized contract を検証する。

channel ordering が不明な場合に推測しない。
renderer が必要な場合は `fa_binaural` などの専用 node を pipeline に明示する。

## Backend Boundary

backend は channel transform のみを担当し、ROS2 topic/message を知らない。
