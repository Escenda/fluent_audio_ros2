# fa_stereo_widening アルゴリズム詳細説明書

## 入力表現

入力は FLOAT32LE interleaved stereo `AudioFrame` である。sample 配列は以下の順序で並ぶ。

```text
L0, R0, L1, R1, ...
```

`data.size()` は `2 * sizeof(float)` の倍数でなければならない。

ROS topic は配送経路、`AudioFrame.stream_id` は stream identity である。node は
`input_topic` で購読した frame の `stream_id` が `input_stream_id` と一致する場合だけ処理し、
出力 frame の `stream_id` を `output.stream_id` に更新して `output_topic` へ publish する。
stream identity が raw/resolved topic 名と衝突する設定、または input/output stream identity が
同一の設定では起動しない。

## Mid/Side Stereo Width

`width` を `[0.0, 4.0]` とし、各 stereo pair に対して以下を計算する。

```text
mid = (L + R) / 2
side = (L - R) / 2 * width
out_L = mid + side
out_R = mid - side
```

`width=1.0` では入力 stereo pair と同じ値になる。`width=0.0` では左右が同じ mid 値になり、`width>1.0` では side 成分が増幅される。

## Validation

入力 sample は finite かつ normalized `[-1, 1]` でなければならない。出力 sample も同じ範囲で検証する。違反時は frame 全体を drop する。

`fa_stereo_widening` は spatial width のみを扱い、以下は行わない。

- clamp
- normalize
- channel-count conversion
- sample format conversion
- resampling
- limiter / filter / denoise
- pan

## Byte Order

FLOAT32LE sample は 4 byte little-endian として読み書きする。byte 列から `uint32_t` を組み立て、`std::memcpy` で `float` と相互変換する。
