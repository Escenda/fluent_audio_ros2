## Unit Contract Tests

`test/unit/test_fa_stereo_widening_audio_frame_contract.py` は package contract を静的に検査する。

- default config が stereo FLOAT32LE interleaved contract と `width` default を持つこと
- default config が ROS topic と `AudioFrame.stream_id` を別 identity として持つこと
- node が unrelated responsibilities を持たないこと
- invalid config、empty stream identity、topic と衝突する stream identity、input/output 同一 stream identity で fail closed する validation があること
- runtime frame contract を処理前に検証すること
- 入力 `stream_id` を `input_stream_id` で検証すること
- `stream_id` を `output.stream_id` に更新し、それ以外の metadata を不要に書き換えないこと
- mid/side stereo width の式が実装されていること
- finite normalized `[-1, 1]` 違反を drop すること
- diagnostics が `width` と counter を含むこと
- standard processing package layout を満たすこと
- `ament_cmake_pytest` で colcon から pytest が登録されること

## Integration Tests

将来の integration test では、実 ROS2 graph 上で `fa_stereo_widening/input` から `fa_stereo_widening/output` への publish を確認する。入力 frame の `stream_id` が `audio/spatial/mic` と一致する frame だけを受理し、出力 frame の `stream_id` が `audio/stereo_widened/mic` に更新されること、`epoch` と identity metadata が保持されること、invalid frame が publish されないことを確認対象にする。
