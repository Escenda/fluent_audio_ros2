# fa_stereo_widening テスト設計

## Unit Contract Tests

`test/unit/test_fa_stereo_widening_audio_frame_contract.py` は package contract を静的に検査する。

- default config が stereo FLOAT32LE interleaved contract と `width` default を持つこと
- node が unrelated responsibilities を持たないこと
- invalid config で fail closed する validation があること
- runtime frame contract を処理前に検証すること
- `stream_id` を `output_topic` に更新し、それ以外の metadata を不要に書き換えないこと
- mid/side stereo width の式が実装されていること
- finite normalized `[-1, 1]` 違反を drop すること
- diagnostics が `width` と counter を含むこと
- standard processing package layout を満たすこと
- `ament_cmake_pytest` で colcon から pytest が登録されること

## Integration Tests

将来の integration test では、実 ROS2 graph 上で `audio/spatial/mic` から `audio/stereo_widened/mic` への publish を確認する。入力 frame の `epoch` と identity metadata が保持されること、invalid frame が publish されないことを確認対象にする。

## Launch Tests

将来の launch test では `launch/fa_stereo_widening.launch.py` が default config を読み込み、`fa_stereo_widening_node` を起動できることを確認する。
