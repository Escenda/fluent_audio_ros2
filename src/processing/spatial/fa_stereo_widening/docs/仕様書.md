# fa_stereo_widening 仕様書

## 目的

`fa_stereo_widening` は FluentAudio の stereo widening processing package である。FLOAT32LE interleaved stereo `AudioFrame` に mid/side transform を適用し、stereo width 変更後の stereo frame を publish する。

## 仕様 ID

| ID | 内容 |
| --- | --- |
| `FA-STEREO-WIDENING-SPEC-001` | `fa_stereo_widening` は `FLOAT32LE` interleaved stereo `AudioFrame` に mid/side transform を適用する spatial processing node である。 |
| `FA-STEREO-WIDENING-SPEC-002` | device I/O、resampling、sample format / channel conversion、limiter、filter、denoise、pan、analysis metadata 生成を行わない。 |
| `FA-STEREO-WIDENING-SPEC-003` | `input_topic` / `output_topic` と `input_stream_id` / `output.stream_id` は別契約であり、topic identity と stream identity の衝突を起動時に拒否する。 |
| `FA-STEREO-WIDENING-SPEC-004` | `width`、expected stereo FLOAT32LE format、QoS、diagnostics parameter は required で、不正値は fail closed する。 |
| `FA-STEREO-WIDENING-SPEC-005` | runtime frame は source / stream identity、stereo FLOAT32LE format、data alignment、finite normalized input / output sample を検証する。 |
| `FA-STEREO-WIDENING-SPEC-006` | output metadata は入力を基準にし、`stream_id` と `data` だけを更新する。 |
| `FA-STEREO-WIDENING-SPEC-007` | mid/side stereo width は `width` parameter により制御し、invalid output を clamp / normalize せず drop する。 |
| `FA-STEREO-WIDENING-SPEC-008` | diagnostics は width、format contract、frame counter を publish する。 |
| `FA-STEREO-WIDENING-SPEC-009` | launch file は `node_name` / `config_file` を明示引数とし、package-local default config を推測しない。 |
| `FA-STEREO-WIDENING-SPEC-010` | pytest contract は `colcon test` から実行され、`ament_lint_auto` dependency を package manifest に明示する。 |

## Scope

対象:

- `fa_interfaces/msg/AudioFrame` の購読と publish
- `width` による mid/side stereo widening
- 起動時 config validation
- runtime frame validation
- diagnostics publish

対象外:

- device I/O
- resampling
- sample format conversion
- channel-count conversion
- limiter / filter / denoise
- pan
- RMS / peak / VAD 等の解析 metadata 生成

## ROS2 Interface

| 種別 | 名前 | 型 | 内容 |
| --- | --- | --- | --- |
| subscribe | `input_topic` | `fa_interfaces/msg/AudioFrame` | stereo widening 対象 stereo frame |
| publish | `output_topic` | `fa_interfaces/msg/AudioFrame` | stereo widening 後の stereo frame |
| publish | `diagnostics` | `diagnostic_msgs/msg/DiagnosticArray` | width、format contract、counter |

## Parameters

| Parameter | Default | Constraint |
| --- | --- | --- |
| `input_topic` | `fa_stereo_widening/input` | non-empty |
| `output_topic` | `fa_stereo_widening/output` | non-empty |
| `input_stream_id` | `audio/spatial/mic` | non-empty、raw/resolved topic 名と一致不可 |
| `output.stream_id` | `audio/stereo_widened/mic` | non-empty、raw/resolved topic 名および `input_stream_id` と一致不可 |
| `width` | `1.0` | finite, `0.0 <= value <= 4.0` |
| `expected.sample_rate` | `16000` | `> 0` |
| `expected.channels` | `2` | exactly `2` |
| `expected.encoding` | `FLOAT32LE` | exactly `FLOAT32LE` |
| `expected.bit_depth` | `32` | exactly `32` |
| `expected.layout` | `interleaved` | exactly `interleaved` |
| `qos.depth` | `10` | `> 0` |
| `qos.reliable` | `false` | boolean |
| `diagnostics.publish_period_ms` | `1000` | `> 0` |
| `diagnostics.qos.depth` | `10` | `> 0` |
| `diagnostics.qos.reliable` | `true` | bool |

Invalid config は起動時に `std::runtime_error` で fail closed する。

## AudioFrame Contract

Runtime frame は以下を満たす必要がある。

- `source_id` が non-empty
- `stream_id` が non-empty
- `stream_id == input_stream_id`
- `sample_rate == expected.sample_rate`
- `channels == 2`
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `layout == interleaved`
- `data` が non-empty
- `data.size()` が `2 * sizeof(float)` の倍数
- 全入力 sample が finite normalized `[-1, 1]`
- 全出力 sample が finite normalized `[-1, 1]`

違反した runtime frame は warning を出して drop し、publish しない。clamp、normalize、default 補完は行わない。

## Output Preservation

出力 frame は入力 frame を基準に以下のみ更新する。

- `stream_id = output.stream_id`
- `data = stereo widening 後の FLOAT32LE interleaved bytes`

`header`, `source_id`, `sample_rate`, `channels`, `bit_depth`, `encoding`, `layout`, `epoch` は保持する。
