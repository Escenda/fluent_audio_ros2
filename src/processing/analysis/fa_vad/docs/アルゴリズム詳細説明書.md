# fa_vad アルゴリズム詳細説明書

1. `AudioFrame` が mono / float32 / normalized [-1.0, 1.0] / `target_sample_rate` と一致することを検証する。
2. 不一致の frame は backend に渡さず reject する。
3. backend adapter 用に float samples を PCM16 bytes へ serialize する。
4. backend `update()` に PCM16 bytes を渡す。
5. `silero` backend は window を一時 WAV file に書き、`backend.command` を external process として実行する。
6. external process の stdout から probability float を読む。
7. probability / is_speech / start / end を `VadState` として publish する。
8. speech state が変化した場合のみ Bool topic を publish する。

Resampling / downmix / bit-depth conversion は `fa_vad` では行いません。必要な変換は upstream の format/processing node で明示します。
Silero backend は `backend.model_path` の local torch.hub repository だけを external worker に渡し、online source や torch cache path を推測しません。推論 device は `backend.execution_provider` として external worker へ明示的に渡します。ROS2 node process 内で PyTorch を import しません。
