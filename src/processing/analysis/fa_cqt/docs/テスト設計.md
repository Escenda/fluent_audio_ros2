# fa_cqt テスト設計

## 方針

`fa_cqt` は AI node ではないため、テストは model inference ではなく feature extraction contract を対象にする。

## 仕様とテストケース対応

| TC ID | 対応 SPEC | 観点 |
| --- | --- | --- |
| `FA-CQT-TC-001` -> `FA-CQT-SPEC-001` | `FA-CQT-SPEC-001` | package が `src/processing/analysis` にあり、README / docs が AI node ではないことを明記する |
| `FA-CQT-TC-002` -> `FA-CQT-SPEC-002` | `FA-CQT-SPEC-002` | backend source が ROS2 / `fa_interfaces` を import しない |
| `FA-CQT-TC-003` -> `FA-CQT-SPEC-003` | `FA-CQT-SPEC-003` | `FLOAT32LE` mono normalized input だけを扱う |
| `FA-CQT-TC-004` -> `FA-CQT-SPEC-004` | `FA-CQT-SPEC-004` | backend output が finite で `frame_count x bin_count` shape を持つ |
| `FA-CQT-TC-005` -> `FA-CQT-SPEC-005` | `FA-CQT-SPEC-005` | default config は `backend.name=internal_cqt` を明示する |
| `FA-CQT-TC-006` -> `FA-CQT-SPEC-006` | `FA-CQT-SPEC-006` | ROS parameter は default で補完せず、YAML/launch から明示されない場合は起動失敗する |
| `FA-CQT-TC-007` -> `FA-CQT-SPEC-007` | `FA-CQT-SPEC-007` | input `stream_id` が `expected.stream_id` と一致しない frame は publish しない |
| `FA-CQT-TC-008` -> `FA-CQT-SPEC-008` | `FA-CQT-SPEC-008` | sample count が `feature.frame_length + N * feature.hop_length` に一致しない場合は truncate せず drop する |
| `FA-CQT-TC-009` -> `FA-CQT-SPEC-009` | `FA-CQT-SPEC-009` | zero signal は real / imag とも zero になる |
| `FA-CQT-TC-010` -> `FA-CQT-SPEC-010` | `FA-CQT-SPEC-010` | sample system config は CQT 用 long-window input を明示する |
