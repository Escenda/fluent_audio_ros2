# fa_cqt アルゴリズム詳細説明書

## 入力契約

入力は `FLOAT32LE`、mono、`interleaved`、normalized `[-1.0, 1.0]` の `AudioFrame` に限定する。`fa_cqt` 内で resample、downmix、bit depth conversion、normalization は行わない。

## 処理手順

1. `AudioFrame` metadata と payload を検証する。`stream_id` は `input_topic` と一致する必要がある。
2. `float32` 1 次元配列として samples を読む。
3. `feature.frame_length` と `feature.hop_length` に従い、padding なしで frame 分割する。入力長は `feature.frame_length + N * feature.hop_length` に一致する必要があり、末尾 sample は truncate しない。
4. `feature.f_min_hz * 2 ** (bin_index / bins_per_octave)` で各 bin の center frequency を決める。
5. `Q = 1 / (2 ** (1 / bins_per_octave) - 1)` とし、各 bin の kernel length を `ceil(Q * sample_rate / center_frequency)` とする。
6. 各 kernel は Hann window と complex sinusoid から作り、L2 norm で正規化する。
7. 各 frame 中央の kernel length 区間に complex kernel を掛け、complex response を計算する。
8. `frames_by_bins` / `complex_cartesian` layout の `CqtFrame` として publish する。

入力長が `feature.frame_length` 未満、または `feature.frame_length + N * feature.hop_length` に一致しない場合は zero padding / truncate せず frame drop とする。これは後段の時間幅契約を変えないためである。

## 数値仕様

- center frequencies: `f_min_hz * 2 ** (bin_index / bins_per_octave)`。
- Q factor: `1 / (2 ** (1 / bins_per_octave) - 1)`。
- kernel length: `ceil(Q * sample_rate / center_frequency)`。
- window: `numpy.hanning(kernel_length)`。
- kernel: `window * exp(-2j * pi * center_frequency * time)`。
- normalization: complex kernel の L2 norm。
- response: `vdot(kernel, centered_frame_segment)`。

## Backend Boundary

`InternalCqtBackend` は ROS2 topic、ROS message、`rclpy` を参照しない。ROS2 node は parameter、message validation、publish / subscribe の adapter としてだけ振る舞う。
