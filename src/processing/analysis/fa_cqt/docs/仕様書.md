# fa_cqt 仕様書

## 目的

`fa_cqt` は `FLOAT32LE` mono `AudioFrame` を complex constant-Q feature matrix に変換し、`fa_interfaces/msg/CqtFrame` として publish する非 AI analysis node である。

## 責務

- `AudioFrame` の source / stream / format metadata を検証する。
- 入力 frame の `stream_id` が `expected.stream_id` と一致することを検証する。
- `internal_cqt` backend で deterministic complex CQT feature を計算する。
- `CqtFrame.real` / `CqtFrame.imag` に `frames_by_bins` layout の frame-major matrix を publish する。

## 非責務

- VAD / KWS / ASR / Turn Detector
- speaker / audio embedding
- model inference
- resample / bit depth conversion / channel conversion
- magnitude / log-power 変換
- gain / normalize / denoise
- device I/O

AI model node は `src/ai` に置く。`fa_cqt` は model input や音楽寄り解析に使える特徴量を作るだけで、認識結果や分類結果を生成しない。

## Parameters

| parameter | required | 内容 |
| --- | --- | --- |
| `input_topic` | yes | 入力 `AudioFrame` topic |
| `output_topic` | yes | 出力 `CqtFrame` topic |
| `expected.stream_id` | yes | 入力 `AudioFrame.stream_id` と照合する stream identity。ROS topic 名とは分離する |
| `output.stream_id` | yes | 出力 feature frame の `stream_id`。ROS topic 名とは分離する |
| `backend.name` | yes | `internal_cqt` 固定。未知 backend は起動失敗 |
| `expected.sample_rate` | yes | 入力 sample rate |
| `expected.channels` | yes | `1` 固定 |
| `expected.encoding` | yes | `FLOAT32LE` 固定 |
| `expected.bit_depth` | yes | `32` 固定 |
| `expected.layout` | yes | `interleaved` 固定 |
| `feature.frame_length` | yes | analysis frame length。最長 CQT kernel 以上であること |
| `feature.hop_length` | yes | frame hop length |
| `feature.bin_count` | yes | 出力 bin 数 |
| `feature.bins_per_octave` | yes | octave あたりの bin 数 |
| `feature.f_min_hz` | yes | minimum center frequency |
| `feature.window` | yes | `hann` 固定 |
| `feature.normalization` | yes | `l2` 固定 |
| `qos.depth` | yes | ROS topic queue depth |
| `qos.reliable` | yes | reliable / best effort |

## Error Handling

- 起動時 config が不正な場合は起動失敗する。
- frame が expected format と一致しない場合は publish せず drop する。
- frame の `stream_id` が `expected.stream_id` と一致しない場合は publish せず drop する。
- sample が non-finite または `[-1.0, 1.0]` 外の場合は publish せず drop する。
- 入力長が `feature.frame_length` 未満の場合は publish せず drop する。
- 入力長が `feature.frame_length + N * feature.hop_length` に一致しない場合は、末尾を暗黙に truncate せず drop する。chunk 連結や overlap は専用 streaming/temporal node で行う。
- 最長 CQT kernel が `feature.frame_length` を超える config は起動失敗する。
- backend invariant failure は invalid frame として握りつぶさず node failure として扱う。
- resample、downmix、zero fill、padding、clamp による暗黙補正は行わない。

## Traceability

- `FA-CQT-SPEC-001`: `fa_cqt` は `src/processing/analysis` の非 AI feature package である。
- `FA-CQT-SPEC-002`: backend は ROS2 topic/message を知らない。
- `FA-CQT-SPEC-003`: non-canonical `AudioFrame` は hidden conversion せず drop する。
- `FA-CQT-SPEC-004`: `CqtFrame.real` / `CqtFrame.imag` は `frames_by_bins` layout である。
