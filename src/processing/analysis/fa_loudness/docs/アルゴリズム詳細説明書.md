# fa_loudness アルゴリズム詳細説明書

## Pipeline

1. `AudioFrame` の source / stream / format / byte alignment を検証する。
2. `AudioFrame.data` を little-endian float32 の 1D sample array として読む。
3. sample が finite かつ normalized `[-1.0, 1.0]` であることを検証する。
4. `rms = sqrt(mean(samples^2))` を double precision の intermediate で計算する。
5. `peak = max(abs(samples))` を計算する。
6. `meter.db_floor` から lower-bound amplitude を計算し、`rms_dbfs` / `peak_dbfs` を求める。
7. `rms == 0.0` なら `crest_factor = 0.0`、それ以外は `peak / rms` とする。
8. output が finite であることを検証し、`LoudnessFrame` を publish する。

## dBFS

`dbfs(value)` は次の仕様とする。

```text
floor_amplitude = 10 ** (meter.db_floor / 20)
if value <= floor_amplitude:
  meter.db_floor
else:
  20 * log10(value)
```

zero input を `-inf` にせず `meter.db_floor` へ固定する。これは表示 / monitoring の下限仕様であり、入力欠損を補う fallback ではない。

## Backend Boundary

`InternalFrameMeterBackend` は `FrameMeterConfig` と `numpy.ndarray` だけを扱う。ROS2 parameter、topic 名、message 型、QoS、logging は node 側の責務である。backend で ROS2 import や `fa_interfaces` import を行わない。

## Drop Conditions

次の場合は frame を publish しない。

- `source_id` / `stream_id` が空
- `stream_id` が `expected.stream_id` と一致しない
- encoding / bit depth / channels / layout / sample rate が expected contract と一致しない
- byte length が float32 に整列しない
- sample が non-finite または normalized range 外

この node は loudness measurement 専用であり、normalize、gain correction、limiter、clip、format conversion、resample を行わない。
