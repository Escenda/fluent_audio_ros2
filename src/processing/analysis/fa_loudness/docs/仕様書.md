# fa_loudness 仕様書

## 目的

`fa_loudness` は `AudioFrame` の canonical PCM stream から frame RMS、peak、dBFS、crest factor を測定し、`fa_interfaces/msg/LoudnessFrame` として publish する非 AI analysis node である。

## 非目標

- LUFS / K-weighted loudness
- gain / normalize / compressor / limiter
- resample / channel convert / bit depth convert / sample format convert
- VAD / KWS / ASR / Turn Detector などの AI 推論
- device I/O

## 所属分類

`src/processing/analysis/fa_loudness`

## 入力

| Topic | 型 | 内容 |
| --- | --- | --- |
| `input_topic` | `fa_interfaces/msg/AudioFrame` | mono `FLOAT32LE` PCM frame |

## 出力

| Topic | 型 | 内容 |
| --- | --- | --- |
| `output_topic` | `fa_interfaces/msg/LoudnessFrame` | frame RMS / peak / dBFS meter |

## Message Contract

入力 `AudioFrame` は次を満たす必要がある。

- `source_id` / `stream_id` が空でない
- `stream_id == input_topic`
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `channels == 1`
- `layout == interleaved`
- `sample_rate == expected.sample_rate`
- `data` が non-empty かつ float32 byte aligned
- sample は finite かつ normalized `[-1.0, 1.0]`

出力 `LoudnessFrame` は次を満たす。

- `source_id` は入力を保持する
- `stream_id == output_topic`
- `rms` と `peak` は normalized full-scale amplitude
- `rms_dbfs` と `peak_dbfs` は `meter.db_floor` で下限固定された dBFS
- `crest_factor == peak / rms`。`rms == 0.0` の場合は明示仕様として `0.0`

## Parameters

| Parameter | 必須 | 内容 |
| --- | --- | --- |
| `input_topic` | yes | 入力 `AudioFrame` topic |
| `output_topic` | yes | 出力 `LoudnessFrame` topic |
| `backend.name` | yes | `internal_frame_meter` のみ |
| `expected.sample_rate` | yes | 入力 sample rate |
| `expected.channels` | yes | 現在は `1` のみ |
| `expected.encoding` | yes | `FLOAT32LE` |
| `expected.bit_depth` | yes | `32` |
| `expected.layout` | yes | `interleaved` |
| `meter.db_floor` | yes | zero / very small value の dBFS lower bound。有限かつ `[-300.0, 0.0)` |
| `qos.depth` | yes | pub/sub QoS depth |
| `qos.reliable` | yes | reliable QoS を使うか |

## Fail Closed / Drop

起動時 parameter が不正な場合は例外で起動失敗する。入力 frame contract が不一致、sample が non-finite / normalized range 外、または byte alignment が壊れている場合は publish せず drop する。zero fill、clip、resample、format conversion、gain correction は行わない。

## Backend

runtime backend は `internal_frame_meter` のみ。backend は ROS2 topic / message を知らず、`numpy.float32` の 1D sample array だけを受け取る。詳細は `docs/backends/internal_frame_meter.md` を参照する。

## Test IDs

- `FA-LOUDNESS-SPEC-001`: package layout が標準構造を満たす
- `FA-LOUDNESS-SPEC-002`: default config が canonical `FLOAT32LE` analysis contract を持つ
- `FA-LOUDNESS-SPEC-003`: backend は RMS / peak / dBFS / crest factor を決定的に算出する
- `FA-LOUDNESS-SPEC-004`: backend は dtype / ndim / range 不一致を reject する
- `FA-LOUDNESS-SPEC-005`: backend は ROS2 topic / message を知らない
- `FA-LOUDNESS-SPEC-006`: node は runtime default 補完をせず、明示 parameter を要求する
- `FA-LOUDNESS-SPEC-007`: node は `AudioFrame.stream_id == input_topic` を検証する
