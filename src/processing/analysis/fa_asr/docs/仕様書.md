# fa_asr 仕様書

## 責務

`fa_asr` は発話区間の audio buffer を ASR backend に渡し、文字起こし結果を publish する analysis node です。

## 非責務

- VAD 推定
- KWS
- turn detection
- device source selection
- ASR backend fallback

## Input / Output

| 種別 | 名前 | 型 |
| --- | --- | --- |
| Sub | `audio_topic` | `fa_interfaces/msg/AudioFrame` |
| Sub | `vad_topic` | `fa_interfaces/msg/VadState` |
| Sub | `turn_context_topic` | `fa_interfaces/msg/TurnContext` |
| Pub | `result_topic` | `fa_interfaces/msg/AsrResult` |

## Backend Contract

- `backend.name` は必須。
- default config は `backend.name` を暗黙指定しない。
- backend は ROS2 topic/message を知らない。
- backend input は mono float samples と sample rate。
- backend output は transcript text。

## Supported Backend Slots

- `local_command`
- `whisper_cpp`
- `parakeet_worker`
- `openai_realtime`
- `openai_transcriptions`

各 backend は個別 class と loader を持ち、`LocalCommandAsrBackend` の名前差し替え alias として扱いません。共通 subprocess 実行は内部 helper に閉じます。

`local_command` と `whisper_cpp` は local process backend として `backend.model_path` を要求します。`parakeet_worker`、`openai_realtime`、`openai_transcriptions` は external worker / process / container backend として `backend.model` を要求します。OpenAI SDK/API は `fa_asr` に import せず、worker 側の責務にします。

未知 backend、backend command 欠落、model / model_path 欠落、invalid working directory は起動失敗にします。
`backend.args` は backend ごとの worker/CLI contract なので default config では空にし、ASR を有効化する profile または system config で必ず明示します。空配列、`{audio}` 欠落、`{model}` 欠落は起動失敗です。

## Test IDs

- `FA-ASR-SPEC-001`: backend.name 空で起動失敗
- `FA-ASR-SPEC-002`: unknown backend で起動失敗
- `FA-ASR-SPEC-003`: backend ごとに専用 class / required parameter を使う
- `FA-ASR-SPEC-004`: VAD end で buffer を finalize する
- `FA-ASR-SPEC-005`: backend error を AsrResult error として publish する
