# fa_mfcc アルゴリズム詳細説明書

## 入力契約

入力は `FLOAT32LE`、mono、`interleaved`、normalized `[-1.0, 1.0]` の `AudioFrame` に限定する。`fa_mfcc` 内で resample、downmix、bit depth conversion、normalization は行わない。

## 処理手順

1. `AudioFrame` metadata と payload を検証する。`stream_id` は `input_topic` と一致する必要がある。
2. `float32` 1 次元配列として samples を読む。
3. `feature.n_fft` と `feature.hop_length` に従い、padding なしで frame 分割する。入力長は `feature.n_fft + N * feature.hop_length` に一致する必要があり、末尾 sample は truncate しない。
4. Hann window を適用する。
5. `rfft` で power spectrum を計算する。
6. triangular mel filterbank を掛ける。
7. `feature.log_floor` を下限として natural log を取る。
8. log-mel matrix に DCT-II basis を掛け、MFCC matrix を計算する。
9. `frames_by_coefficients` layout の `MfccFrame` として publish する。

入力長が `feature.n_fft` 未満、または `feature.n_fft + N * feature.hop_length` に一致しない場合は zero padding / truncate せず frame drop とする。これは後段の時間幅契約を変えないためである。

## 数値仕様

- window: `numpy.hanning(feature.n_fft)`。
- spectrum: `numpy.fft.rfft(windowed_frame)`。
- power: `abs(spectrum) ** 2`。FFT normalization は行わない。
- mel scale: HTK 形式 `2595 * log10(1 + hz / 700)`。
- mel filterbank: triangular filters。各 filter は `2 / (right_hz - left_hz)` で area normalization する。
- log: `numpy.log(max(mel_power, feature.log_floor))` の natural log。
- DCT: DCT-II。
- DCT normalization: `ortho` 固定。0 番係数は `sqrt(1 / n_mels)`、1 番以降は `sqrt(2 / n_mels)` を掛ける。

## Backend Boundary

`InternalMfccBackend` は ROS2 topic、ROS message、`rclpy` を参照しない。ROS2 node は parameter、message validation、publish / subscribe の adapter としてだけ振る舞う。
