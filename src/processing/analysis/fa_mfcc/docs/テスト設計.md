# fa_mfcc テスト設計

## 方針

`fa_mfcc` は AI node ではないため、テストは model inference ではなく feature extraction contract を対象にする。

## Test IDs

| ID | 観点 |
| --- | --- |
| `FA-MFCC-SPEC-001` | package が `src/processing/analysis` にあり、README / docs が AI node ではないことを明記する |
| `FA-MFCC-SPEC-002` | backend source が ROS2 / `fa_interfaces` を import しない |
| `FA-MFCC-SPEC-003` | `FLOAT32LE` mono normalized input だけを扱う |
| `FA-MFCC-SPEC-004` | backend output が finite で `frame_count x n_mfcc` shape を持つ |
| `FA-MFCC-SPEC-005` | default config は `backend.name=internal_mfcc` を明示する |
| `FA-MFCC-SPEC-006` | ROS parameter は default で補完せず、YAML/launch から明示されない場合は起動失敗する |
| `FA-MFCC-SPEC-007` | input `stream_id` が `expected.stream_id` と一致しない frame は publish しない |
| `FA-MFCC-SPEC-008` | sample count が `n_fft + N * hop_length` に一致しない場合は truncate せず drop する |
| `FA-MFCC-SPEC-009` | zero signal の golden vector は DCT-II ortho の constant log-mel 入力と一致する |
| `FA-MFCC-SPEC-010` | sample system config は 20ms 16kHz mic chunk と整合する `feature.n_fft=320` を明示する |
