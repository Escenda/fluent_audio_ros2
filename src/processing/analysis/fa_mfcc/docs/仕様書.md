# fa_mfcc 仕様書

## 目的

`fa_mfcc` は `FLOAT32LE` mono `AudioFrame` を MFCC feature matrix に変換し、`fa_interfaces/msg/MfccFrame` として publish する非 AI analysis node である。

## 責務

- `AudioFrame` の source / stream / format metadata を検証する。
- 入力 frame の `stream_id` が `expected.stream_id` と一致することを検証する。
- `internal_mfcc` backend で deterministic MFCC feature を計算する。
- `MfccFrame.values` に `frames_by_coefficients` layout の frame-major matrix を publish する。

## 非責務

- VAD / KWS / ASR / Turn Detector
- speaker / audio embedding
- model inference
- resample / bit depth conversion / channel conversion
- gain / normalize / denoise
- device I/O

AI model node は `src/ai` に置く。`fa_mfcc` は model input に使える特徴量を作るだけで、認識結果や分類結果を生成しない。

## Parameters

| parameter | required | 内容 |
| --- | --- | --- |
| `input_topic` | yes | 入力 `AudioFrame` topic |
| `output_topic` | yes | 出力 `MfccFrame` topic |
| `expected.stream_id` | yes | 入力 `AudioFrame.stream_id` と照合する stream identity。ROS topic 名とは分離する |
| `output.stream_id` | yes | 出力 feature frame の `stream_id`。ROS topic 名とは分離する |
| `backend.name` | yes | `internal_mfcc` 固定。未知 backend は起動失敗 |
| `expected.sample_rate` | yes | 入力 sample rate |
| `expected.channels` | yes | `1` 固定 |
| `expected.encoding` | yes | `FLOAT32LE` 固定 |
| `expected.bit_depth` | yes | `32` 固定 |
| `expected.layout` | yes | `interleaved` 固定 |
| `feature.n_fft` | yes | STFT frame size |
| `feature.hop_length` | yes | STFT hop length |
| `feature.n_mels` | yes | mel bin count |
| `feature.n_mfcc` | yes | MFCC coefficient count |
| `feature.f_min_hz` | yes | mel minimum frequency |
| `feature.f_max_hz` | yes | mel maximum frequency |
| `feature.log_floor` | yes | log input lower bound |
| `feature.dct_type` | yes | `dct2` 固定 |
| `feature.normalization` | yes | `ortho` 固定 |
| `qos.depth` | yes | ROS topic queue depth |
| `qos.reliable` | yes | reliable / best effort |

## Error Handling

- 起動時 config が不正な場合は起動失敗する。
- frame が expected format と一致しない場合は publish せず drop する。
- frame の `stream_id` が `expected.stream_id` と一致しない場合は publish せず drop する。
- sample が non-finite または `[-1.0, 1.0]` 外の場合は publish せず drop する。
- 入力長が `feature.n_fft` 未満の場合は publish せず drop する。
- 入力長が `feature.n_fft + N * feature.hop_length` に一致しない場合は、末尾を暗黙に truncate せず drop する。chunk 連結や overlap は専用 streaming/temporal node で行う。
- backend invariant failure は invalid frame として握りつぶさず node failure として扱う。
- resample、downmix、zero fill、padding、clamp による暗黙補正は行わない。

## Traceability

- `FA-MFCC-SPEC-001`: `fa_mfcc` は `src/processing/analysis` の非 AI feature package である。
- `FA-MFCC-SPEC-002`: backend は ROS2 topic/message を知らない。
- `FA-MFCC-SPEC-003`: non-canonical `AudioFrame` は hidden conversion せず drop する。
- `FA-MFCC-SPEC-004`: `MfccFrame.values` は `frames_by_coefficients` layout である。
- `FA-MFCC-SPEC-005`: default config は `backend.name=internal_mfcc` を明示する。
- `FA-MFCC-SPEC-006`: ROS parameter は default で補完せず、YAML/launch から明示されない場合は起動失敗する。
- `FA-MFCC-SPEC-007`: input `stream_id` が `expected.stream_id` と一致しない frame は publish しない。
- `FA-MFCC-SPEC-008`: sample count が `feature.n_fft + N * feature.hop_length` に一致しない場合は truncate せず drop する。
- `FA-MFCC-SPEC-009`: zero signal の golden vector は DCT-II ortho の constant log-mel 入力と一致する。
- `FA-MFCC-SPEC-010`: sample system config は 20ms 16kHz mic chunk と整合する `feature.n_fft=320` を明示する。
