# fa_stft アルゴリズム詳細説明書

## Pipeline

1. `AudioFrame` の source / stream / format / byte alignment を検証する。
2. `AudioFrame.data` を little-endian float32 の 1D sample array として読む。
3. sample が finite かつ normalized `[-1.0, 1.0]` であることを検証する。
4. `sample_count >= n_fft` かつ `(sample_count - n_fft) % hop_length == 0` を検証する。
5. `hop_length` 間隔で frame-major matrix を作る。
6. `feature.window` の窓関数を掛ける。
7. `np.fft.rfft` で `n_fft / 2 + 1` bin の complex spectrum を得る。
8. `real` / `imag` を float32 に変換し、non-finite output を reject する。
9. `StftFrame` へ metadata と flattened `real` / `imag` を publish する。

## Window

- `hann`: `numpy.hanning(n_fft)` を float32 として使う。
- `rectangular`: 全サンプル 1.0 の窓を使う。

## Backend Boundary

`InternalStftBackend` は `StftConfig` と `numpy.ndarray` だけを扱う。ROS2 parameter、topic 名、message 型、QoS、logging は node 側の責務である。backend で ROS2 import や `fa_interfaces` import を行わない。

## Drop Conditions

次の場合は frame を publish しない。

- `source_id` / `stream_id` が空
- `stream_id` が `input_topic` と一致しない
- encoding / bit depth / channels / layout / sample rate が expected contract と一致しない
- byte length が float32 に整列しない
- sample が non-finite または normalized range 外
- sample count が `n_fft` 未満
- sample count が `n_fft` / `hop_length` に整列しない

この node は padding、zero fill、clip、format conversion、resample を行わない。frame length を整えたい場合は upstream の `fa_frame_buffer` / `fa_window` / `fa_chunk_overlap` などを system config に明示して挟む。
