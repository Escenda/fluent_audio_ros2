# fa_stft 仕様書

## 目的

`fa_stft` は `AudioFrame` の canonical PCM stream から STFT complex spectrum を抽出し、`fa_interfaces/msg/StftFrame` として publish する非 AI analysis node である。

## 非目標

- VAD / KWS / ASR / Turn Detector などの AI 推論
- resample / channel convert / bit depth convert / sample format convert
- denoise / gain / normalize / filter
- device I/O
- model runtime

## 所属分類

`src/processing/analysis/fa_stft`

## 入力

| Topic | 型 | 内容 |
| --- | --- | --- |
| `input_topic` | `fa_interfaces/msg/AudioFrame` | mono `FLOAT32LE` PCM frame |

## 出力

| Topic | 型 | 内容 |
| --- | --- | --- |
| `output_topic` | `fa_interfaces/msg/StftFrame` | frame-major STFT complex matrix |

## Message Contract

入力 `AudioFrame` は次を満たす必要がある。

- `source_id` / `stream_id` が空でない
- `stream_id == input_topic`
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `channels == 1`
- `layout == interleaved`
- `sample_rate == expected.sample_rate`
- `data` が non-empty かつ float32 byte aligned
- sample は finite かつ normalized `[-1.0, 1.0]`

出力 `StftFrame` は次を満たす。

- `source_id` は入力を保持する
- `stream_id == output_topic`
- `layout == frames_by_bins`
- `value_format == complex_cartesian`
- `bin_count == n_fft / 2 + 1`
- `real` / `imag` はそれぞれ `frame_count * bin_count` 個の frame-major row order

## Parameters

| Parameter | 必須 | 内容 |
| --- | --- | --- |
| `input_topic` | yes | 入力 `AudioFrame` topic |
| `output_topic` | yes | 出力 `StftFrame` topic |
| `backend.name` | yes | `internal_stft` のみ |
| `expected.sample_rate` | yes | 入力 sample rate |
| `expected.channels` | yes | 現在は `1` のみ |
| `expected.encoding` | yes | `FLOAT32LE` |
| `expected.bit_depth` | yes | `32` |
| `expected.layout` | yes | `interleaved` |
| `feature.n_fft` | yes | FFT size |
| `feature.hop_length` | yes | hop samples |
| `feature.window` | yes | `hann` または `rectangular` |
| `qos.depth` | yes | pub/sub QoS depth |
| `qos.reliable` | yes | reliable QoS を使うか |

## Fail Closed / Drop

起動時 parameter が不正な場合は例外で起動失敗する。入力 frame contract が不一致、sample が non-finite / normalized range 外、または frame length が `n_fft` / `hop_length` に整列しない場合は publish せず drop する。zero fill、clip、resample、padding は行わない。

## Backend

runtime backend は `internal_stft` のみ。backend は ROS2 topic / message を知らず、`numpy.float32` の 1D sample array だけを受け取る。詳細は `docs/backends/internal_stft.md` を参照する。

## Test IDs

- `FA-STFT-SPEC-001`: package layout が標準構造を満たす
- `FA-STFT-SPEC-002`: default config が canonical `FLOAT32LE` analysis contract を持つ
- `FA-STFT-SPEC-003`: backend は finite な complex spectrum を frame-major で返す
- `FA-STFT-SPEC-004`: backend は dtype / ndim / range / alignment 不一致を reject する
- `FA-STFT-SPEC-005`: backend は ROS2 topic / message を知らない
- `FA-STFT-SPEC-006`: node は runtime default 補完をせず、明示 parameter を要求する
- `FA-STFT-SPEC-007`: node は `AudioFrame.stream_id == input_topic` を検証する
