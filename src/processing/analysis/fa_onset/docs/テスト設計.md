# fa_onset テスト設計

## 方針

`fa_onset` は AI node ではないため、テストは model inference ではなく onset measurement contract を対象にする。

## Test IDs

| ID | 観点 |
| --- | --- |
| `FA-ONSET-SPEC-001` | package が `src/processing/analysis` にあり、README / docs が AI node ではないことを明記する |
| `FA-ONSET-SPEC-002` | backend source が ROS2 / `fa_interfaces` を import しない |
| `FA-ONSET-SPEC-003` | `FLOAT32LE` mono normalized input だけを扱う |
| `FA-ONSET-SPEC-004` | backend output が finite で `frame_count` と同じ長さの onset sequence を持つ |
| `FA-ONSET-SPEC-005` | default config は `backend.name=internal_spectral_flux` を明示する |
| `FA-ONSET-SPEC-006` | ROS parameter は default で補完せず、YAML/launch から明示されない場合は起動失敗する |
| `FA-ONSET-SPEC-007` | input `stream_id` が `input_topic` と一致しない frame は publish しない |
| `FA-ONSET-SPEC-008` | sample count が `n_fft + N * hop_length` に一致しない場合は truncate せず drop する |
| `FA-ONSET-SPEC-009` | zero signal は strength zero かつ detected false になる |
| `FA-ONSET-SPEC-010` | sample system config は 20ms 16kHz mic chunk と整合する `feature.n_fft=320` を明示する |
