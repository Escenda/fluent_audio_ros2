# fa_onset アルゴリズム詳細説明書

## 入力契約

入力は `FLOAT32LE`、mono、`interleaved`、normalized `[-1.0, 1.0]` の `AudioFrame` に限定する。`fa_onset` 内で resample、downmix、bit depth conversion、normalization は行わない。

## 処理手順

1. `AudioFrame` metadata と payload を検証する。`stream_id` は `expected.stream_id` と一致する必要がある。
2. `float32` 1 次元配列として samples を読む。
3. `feature.n_fft` と `feature.hop_length` に従い、padding なしで frame 分割する。入力長は `feature.n_fft + N * feature.hop_length` に一致する必要があり、末尾 sample は truncate しない。
4. Hann window を適用する。
5. `rfft` で magnitude spectrum を計算する。
6. 前 frame との差分の正成分だけを合計し、spectral flux onset strength とする。最初の frame は zero spectrum との差分として扱う。
7. `detector.threshold` 以上かつ局所ピークで、`detector.min_interval_sec` を満たす frame を onset とする。
8. `frames` layout の `OnsetFrame` として publish する。

入力長が `feature.n_fft` 未満、または `feature.n_fft + N * feature.hop_length` に一致しない場合は zero padding / truncate せず frame drop とする。これは後段の時間幅契約を変えないためである。

## 数値仕様

- window: `numpy.hanning(feature.n_fft)`。
- spectrum: `numpy.fft.rfft(windowed_frame)`。
- magnitude: `abs(spectrum)`。FFT normalization は行わない。
- strength: `sum(max(magnitude[t] - magnitude[t - 1], 0))`。
- frame time: `frame_index * hop_length / sample_rate + n_fft / (2 * sample_rate)`。
- local peak: `strength[t] >= strength[t - 1]` かつ `strength[t] >= strength[t + 1]`。端の frame は存在しない隣接値を `-inf` として扱う。
- minimum interval: `ceil(min_interval_sec * sample_rate / hop_length)` frame。

## Backend Boundary

`InternalSpectralFluxBackend` は ROS2 topic、ROS message、`rclpy` を参照しない。ROS2 node は parameter、message validation、publish / subscribe の adapter としてだけ振る舞う。
