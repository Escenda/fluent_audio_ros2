# fa_pitch テスト設計

## 方針

`fa_pitch` は AI node ではないため、テストは model inference ではなく pitch measurement contract を対象にする。

## 仕様とテストケース対応

| TC ID | 対応 SPEC | 観点 |
| --- | --- | --- |
| `FA-PITCH-TC-001` -> `FA-PITCH-SPEC-001` | `FA-PITCH-SPEC-001` | package が `src/processing/analysis` にあり、README / docs が AI node ではないことを明記する |
| `FA-PITCH-TC-002` -> `FA-PITCH-SPEC-002` | `FA-PITCH-SPEC-002` | backend source が ROS2 / `fa_interfaces` を import しない |
| `FA-PITCH-TC-003` -> `FA-PITCH-SPEC-003` | `FA-PITCH-SPEC-003` | `FLOAT32LE` mono normalized input だけを扱う |
| `FA-PITCH-TC-004` -> `FA-PITCH-SPEC-004` | `FA-PITCH-SPEC-004` | backend output が finite で `frame_count` と同じ長さの pitch sequence を持つ |
| `FA-PITCH-TC-005` -> `FA-PITCH-SPEC-005` | `FA-PITCH-SPEC-005` | default config は `backend.name=internal_autocorrelation` を明示する |
| `FA-PITCH-TC-006` -> `FA-PITCH-SPEC-006` | `FA-PITCH-SPEC-006` | ROS parameter は default で補完せず、YAML/launch から明示されない場合は起動失敗する |
| `FA-PITCH-TC-007` -> `FA-PITCH-SPEC-007` | `FA-PITCH-SPEC-007` | input `stream_id` が `expected.stream_id` と一致しない frame は publish しない |
| `FA-PITCH-TC-008` -> `FA-PITCH-SPEC-008` | `FA-PITCH-SPEC-008` | sample count が `feature.n_fft + N * feature.hop_length` に一致しない場合は truncate せず drop する |
| `FA-PITCH-TC-009` -> `FA-PITCH-SPEC-009` | `FA-PITCH-SPEC-009` | silence は frequency zero、confidence zero、voiced false になる |
| `FA-PITCH-TC-010` -> `FA-PITCH-SPEC-010` | `FA-PITCH-SPEC-010` | sample system config は 20ms 16kHz mic chunk と整合する `feature.n_fft=320` を明示する |
