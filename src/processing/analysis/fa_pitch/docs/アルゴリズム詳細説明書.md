# fa_pitch アルゴリズム詳細説明書

## 入力契約

入力は `FLOAT32LE`、mono、`interleaved`、normalized `[-1.0, 1.0]` の `AudioFrame` に限定する。`fa_pitch` 内で resample、downmix、bit depth conversion、normalization は行わない。

## 処理手順

1. `AudioFrame` metadata と payload を検証する。`stream_id` は `input_topic` と一致する必要がある。
2. `float32` 1 次元配列として samples を読む。
3. `feature.n_fft` と `feature.hop_length` に従い、padding なしで frame 分割する。入力長は `feature.n_fft + N * feature.hop_length` に一致する必要があり、末尾 sample は truncate しない。
4. frame ごとに平均値を引いて Hann window を適用する。
5. autocorrelation を計算する。
6. `feature.f_min_hz` / `feature.f_max_hz` から決まる lag 範囲で最大 autocorrelation peak を探す。
7. `sample_rate / best_lag` を pitch frequency とする。
8. `autocorr[best_lag] / autocorr[0]` を confidence とし、`detector.confidence_threshold` 以上なら voiced とする。
9. `frames` layout の `PitchFrame` として publish する。

入力長が `feature.n_fft` 未満、または `feature.n_fft + N * feature.hop_length` に一致しない場合は zero padding / truncate せず frame drop とする。これは後段の時間幅契約を変えないためである。

## 数値仕様

- window: `numpy.hanning(feature.n_fft)`。
- autocorrelation: `numpy.correlate(windowed_frame, windowed_frame, mode="full")` の正 lag 側。
- search lag: `floor(sample_rate / f_max_hz)` から `ceil(sample_rate / f_min_hz)` まで。
- confidence: `max(autocorr[best_lag] / autocorr[0], 0.0)` を `1.0` 上限で clamp する。
- unvoiced / silence: frame energy が `1.0e-12` 以下の場合、frequency `0.0`、confidence `0.0`、voiced `false` とする。
- frame time: `frame_index * hop_length / sample_rate + n_fft / (2 * sample_rate)`。

## Backend Boundary

`InternalAutocorrelationBackend` は ROS2 topic、ROS message、`rclpy` を参照しない。ROS2 node は parameter、message validation、publish / subscribe の adapter としてだけ振る舞う。
