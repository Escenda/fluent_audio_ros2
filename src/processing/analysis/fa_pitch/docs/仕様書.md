# fa_pitch 仕様書

## 目的

`fa_pitch` は `FLOAT32LE` mono `AudioFrame` を autocorrelation pitch measurement に変換し、`fa_interfaces/msg/PitchFrame` として publish する非 AI analysis node である。

## 責務

- `AudioFrame` の source / stream / format metadata を検証する。
- 入力 frame の `stream_id` が `input_topic` と一致することを検証する。
- `internal_autocorrelation` backend で deterministic fundamental frequency、confidence、voiced flag を計算する。
- `PitchFrame.frame_times_sec`、`PitchFrame.frequencies_hz`、`PitchFrame.confidence`、`PitchFrame.voiced` に `frames` layout の frame-major sequence を publish する。

## 非責務

- VAD / KWS / ASR / Turn Detector
- speaker / audio embedding
- model inference
- resample / bit depth conversion / channel conversion
- gain / normalize / denoise
- device I/O

AI model node は `src/ai` に置く。`fa_pitch` は pitch track の deterministic measurement を作るだけで、音声区間、wake word、文字起こし、turn end を生成しない。

## Parameters

| parameter | required | 内容 |
| --- | --- | --- |
| `input_topic` | yes | 入力 `AudioFrame` topic |
| `output_topic` | yes | 出力 `PitchFrame` topic |
| `backend.name` | yes | `internal_autocorrelation` 固定。未知 backend は起動失敗 |
| `expected.sample_rate` | yes | 入力 sample rate |
| `expected.channels` | yes | `1` 固定 |
| `expected.encoding` | yes | `FLOAT32LE` 固定 |
| `expected.bit_depth` | yes | `32` 固定 |
| `expected.layout` | yes | `interleaved` 固定 |
| `feature.n_fft` | yes | analysis frame size |
| `feature.hop_length` | yes | frame hop length |
| `feature.method` | yes | `autocorrelation` 固定 |
| `feature.f_min_hz` | yes | 探索する minimum pitch |
| `feature.f_max_hz` | yes | 探索する maximum pitch |
| `detector.confidence_threshold` | yes | voiced 判定 threshold |
| `qos.depth` | yes | ROS topic queue depth |
| `qos.reliable` | yes | reliable / best effort |

## Error Handling

- 起動時 config が不正な場合は起動失敗する。
- frame が expected format と一致しない場合は publish せず drop する。
- frame の `stream_id` が `input_topic` と一致しない場合は publish せず drop する。
- sample が non-finite または `[-1.0, 1.0]` 外の場合は publish せず drop する。
- 入力長が `feature.n_fft` 未満の場合は publish せず drop する。
- 入力長が `feature.n_fft + N * feature.hop_length` に一致しない場合は、末尾を暗黙に truncate せず drop する。chunk 連結や overlap は専用 streaming/temporal node で行う。
- backend invariant failure は invalid frame として握りつぶさず node failure として扱う。
- resample、downmix、zero fill、padding、clamp による暗黙補正は行わない。

## Traceability

- `FA-PITCH-SPEC-001`: `fa_pitch` は `src/processing/analysis` の非 AI measurement package である。
- `FA-PITCH-SPEC-002`: backend は ROS2 topic/message を知らない。
- `FA-PITCH-SPEC-003`: non-canonical `AudioFrame` は hidden conversion せず drop する。
- `FA-PITCH-SPEC-004`: `PitchFrame` の frame sequence は `frames` layout である。
