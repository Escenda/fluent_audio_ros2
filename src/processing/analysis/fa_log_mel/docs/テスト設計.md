# fa_log_mel テスト設計

## 方針

`fa_log_mel` は AI node ではないため、テストは model inference ではなく feature extraction contract を対象にする。

## 仕様とテストケース対応

| TC ID | 対応 SPEC | 観点 |
| --- | --- | --- |
| `FA-LOG-MEL-TC-001` -> `FA-LOG-MEL-SPEC-001` | `FA-LOG-MEL-SPEC-001` | package が `src/processing/analysis` にあり、README / docs が AI node ではないことを明記する |
| `FA-LOG-MEL-TC-002` -> `FA-LOG-MEL-SPEC-002` | `FA-LOG-MEL-SPEC-002` | backend source が ROS2 / `fa_interfaces` を import しない |
| `FA-LOG-MEL-TC-003` -> `FA-LOG-MEL-SPEC-003` | `FA-LOG-MEL-SPEC-003` | `FLOAT32LE` mono normalized input だけを扱う |
| `FA-LOG-MEL-TC-004` -> `FA-LOG-MEL-SPEC-004` | `FA-LOG-MEL-SPEC-004` | backend output が finite で `frame_count x n_mels` shape を持つ |
| `FA-LOG-MEL-TC-005` -> `FA-LOG-MEL-SPEC-005` | `FA-LOG-MEL-SPEC-005` | default config は `backend.name=internal_log_mel` を明示する |
| `FA-LOG-MEL-TC-006` -> `FA-LOG-MEL-SPEC-006` | `FA-LOG-MEL-SPEC-006` | ROS parameter は default で補完せず、YAML/launch から明示されない場合は起動失敗する |
| `FA-LOG-MEL-TC-007` -> `FA-LOG-MEL-SPEC-007` | `FA-LOG-MEL-SPEC-007` | input `stream_id` が `expected.stream_id` と一致しない frame は publish しない |
| `FA-LOG-MEL-TC-008` -> `FA-LOG-MEL-SPEC-008` | `FA-LOG-MEL-SPEC-008` | sample count が `feature.n_fft + N * feature.hop_length` に一致しない場合は truncate せず drop する |
| `FA-LOG-MEL-TC-009` -> `FA-LOG-MEL-SPEC-009` | `FA-LOG-MEL-SPEC-009` | zero signal の golden vector が `log(feature.log_floor)` と一致する |
| `FA-LOG-MEL-TC-010` -> `FA-LOG-MEL-SPEC-010` | `FA-LOG-MEL-SPEC-010` | sample system config は 20ms 16kHz mic chunk と整合する `feature.n_fft=320` を明示する |
