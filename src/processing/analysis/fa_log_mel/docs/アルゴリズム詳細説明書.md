# fa_log_mel アルゴリズム詳細説明書

## 入力契約

入力は `FLOAT32LE`、mono、`interleaved`、normalized `[-1.0, 1.0]` の `AudioFrame` に限定する。`fa_log_mel` 内で resample、downmix、bit depth conversion、normalization は行わない。

## 処理手順

1. `AudioFrame` metadata と payload を検証する。
2. `float32` 1 次元配列として samples を読む。
3. `feature.n_fft` と `feature.hop_length` に従い、padding なしで frame 分割する。
4. Hann window を適用する。
5. `rfft` で power spectrum を計算する。
6. triangular mel filterbank を掛ける。
7. `feature.log_floor` を下限として natural log を取る。
8. `frames_by_mels` layout の `LogMelFrame` として publish する。

入力長が `feature.n_fft` 未満の場合は zero padding せず frame drop とする。これは後段の時間幅契約を変えないためである。

## Backend Boundary

`InternalLogMelBackend` は ROS2 topic、ROS message、`rclpy` を参照しない。ROS2 node は parameter、message validation、publish / subscribe の adapter としてだけ振る舞う。
