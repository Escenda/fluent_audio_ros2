# fa_tempo テスト設計

## 方針

`fa_tempo` は AI node ではないため、テストは model inference ではなく tempo measurement contract を対象にする。

## 仕様とテストケース対応

| TC ID | 対応 SPEC | 観点 |
| --- | --- | --- |
| `FA-TEMPO-TC-001` -> `FA-TEMPO-SPEC-001` | `FA-TEMPO-SPEC-001` | package が `src/processing/analysis` にあり、README / docs が AI node ではないことを明記する |
| `FA-TEMPO-TC-002` -> `FA-TEMPO-SPEC-002` | `FA-TEMPO-SPEC-002` | backend source が ROS2 / `fa_interfaces` を import しない |
| `FA-TEMPO-TC-003` -> `FA-TEMPO-SPEC-003` | `FA-TEMPO-SPEC-003` | `FLOAT32LE` mono normalized input だけを扱う |
| `FA-TEMPO-TC-004` -> `FA-TEMPO-SPEC-004` | `FA-TEMPO-SPEC-004` | backend output が finite で `frame_count` と同じ長さの frame time / onset envelope / beat sequence を持つ |
| `FA-TEMPO-TC-005` -> `FA-TEMPO-SPEC-005` | `FA-TEMPO-SPEC-005` | default config は `backend.name=internal_onset_autocorrelation` を明示する |
| `FA-TEMPO-TC-006` -> `FA-TEMPO-SPEC-006` | `FA-TEMPO-SPEC-006` | ROS parameter は default で補完せず、YAML/launch から明示されない場合は起動失敗する |
| `FA-TEMPO-TC-007` -> `FA-TEMPO-SPEC-007` | `FA-TEMPO-SPEC-007` | input `stream_id` が `expected.stream_id` と一致しない frame は publish しない |
| `FA-TEMPO-TC-008` -> `FA-TEMPO-SPEC-008` | `FA-TEMPO-SPEC-008` | sample count が `feature.n_fft + N * feature.hop_length` に一致しない場合は truncate せず drop する |
| `FA-TEMPO-TC-009` -> `FA-TEMPO-SPEC-009` | `FA-TEMPO-SPEC-009` | silence は tempo_detected false、tempo zero、confidence zero になる |
| `FA-TEMPO-TC-010` -> `FA-TEMPO-SPEC-010` | `FA-TEMPO-SPEC-010` | sample system config は 20ms 16kHz mic chunk と整合する `feature.n_fft=320` を明示する |
