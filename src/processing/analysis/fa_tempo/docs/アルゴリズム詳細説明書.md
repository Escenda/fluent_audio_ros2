# fa_tempo アルゴリズム詳細説明書

## 入力契約

入力は `FLOAT32LE`、mono、`interleaved`、normalized `[-1.0, 1.0]` の `AudioFrame` に限定する。`fa_tempo` 内で resample、downmix、bit depth conversion、normalization は行わない。

## 処理手順

1. `AudioFrame` metadata と payload を検証する。`stream_id` は `input_topic` と一致する必要がある。
2. `float32` 1 次元配列として samples を読む。
3. `feature.n_fft` と `feature.hop_length` に従い、padding なしで frame 分割する。入力長は `feature.n_fft + N * feature.hop_length` に一致する必要があり、末尾 sample は truncate しない。
4. Hann window を適用する。
5. `rfft` で magnitude spectrum を計算する。
6. 前 frame との差分の正成分だけを合計し、onset envelope とする。最初の frame は zero spectrum との差分として扱う。
7. onset envelope から平均値を引き、tempo BPM range に対応する lag 範囲で autocorrelation を計算する。
8. 最大 autocorrelation lag を beat period とし、`60 * sample_rate / (beat_period_frames * hop_length)` を tempo BPM とする。
9. normalized autocorrelation peak を confidence とし、`tempo.confidence_threshold` 以上なら `tempo_detected=true` とする。
10. `tempo_detected=true` の場合、最初の beat period 内の最大 onset envelope を anchor とし、`beat_period_frames` 間隔で `beats=true` を付ける。
11. `frames` layout の `TempoFrame` として publish する。

入力長が `feature.n_fft` 未満、または `feature.n_fft + N * feature.hop_length` に一致しない場合は zero padding / truncate せず frame drop とする。これは後段の時間幅契約を変えないためである。

## 数値仕様

- window: `numpy.hanning(feature.n_fft)`。
- spectrum: `numpy.fft.rfft(windowed_frame)`。
- magnitude: `abs(spectrum)`。FFT normalization は行わない。
- onset envelope: `sum(max(magnitude[t] - magnitude[t - 1], 0))`。
- search lag: `ceil(60 * sample_rate / (bpm_max * hop_length))` から `floor(60 * sample_rate / (bpm_min * hop_length))` まで。
- autocorrelation score: `dot(envelope[:-lag], envelope[lag:]) / dot(envelope, envelope)`。envelope は平均値を引いてから使う。
- confidence: 最大 autocorrelation score を `0.0` 以上 `1.0` 以下に制限した値。
- frame time: `frame_index * hop_length / sample_rate + n_fft / (2 * sample_rate)`。

## Backend Boundary

`InternalOnsetAutocorrelationBackend` は ROS2 topic、ROS message、`rclpy` を参照しない。ROS2 node は parameter、message validation、publish / subscribe の adapter としてだけ振る舞う。
