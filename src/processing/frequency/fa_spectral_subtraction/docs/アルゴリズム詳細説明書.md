# fa_spectral_subtraction アルゴリズム詳細説明書

1. startup config で frame length、hop length、window type、subtraction floor、noise profile を検証する。
2. input `AudioFrame` の identity と format metadata を検証する。
3. channel ごとに windowing と STFT を適用する。
4. magnitude spectrum から noise profile を差し引き、floor を適用する。
5. original phase と修正 magnitude から inverse STFT を行う。
6. overlap-add で frame を再構成し、finite/normalized contract を検証する。
7. 契約を満たす場合だけ publish する。

noise profile がない場合に pass-through しない。
入力長が frame/hop contract と一致しない場合は、zero padding や truncate を隠さず drop する。

## Backend Boundary

backend は STFT、noise subtraction、inverse STFT のみを担当し、ROS2 topic/message を知らない。
noise profile load/calibration の境界は明示 config または専用 calibration flow に分ける。
