# fa_eq テスト設計

## Scope

`fa_eq` の package-local contract を pytest と C++ gtest で検証する。ROS graph は C++ gtest、launch fail-closed は `test/launch` の pytest で検証する。

## Unit Contract

- `FA-EQ-SPEC-001`: device I/O、resample、format conversion、compressor、limiter、normalize、denoise を行わない。
- `FA-EQ-SPEC-002`: `FLOAT32LE` / 32bit / interleaved のみ扱う。
- `FA-EQ-SPEC-003`: `low.cutoff_hz` を finite かつ `> 0.0` として検証する。
- `FA-EQ-SPEC-004`: `high.cutoff_hz` を finite、`> low.cutoff_hz`、Nyquist 未満として検証する。
- `FA-EQ-SPEC-005`: `gains.*_db` を finite として検証する。
- `FA-EQ-SPEC-006`: low / mid / high split と dB gain mix を channel ごとに適用する。
- `FA-EQ-SPEC-007`: 出力 `source_id` は保持し、`stream_id` は `output.stream_id` にする。
- `FA-EQ-SPEC-008`: null frame pointer を dereference せず drop する。
- `FA-EQ-SPEC-009`: `source_id` と monotonic `epoch` により state の適用範囲を固定する。
- `FA-EQ-SPEC-010`: forward `epoch` gap は fresh EQ state で処理し、reset 数を diagnostics に出す。
- `FA-EQ-SPEC-011`: launch は `node_name` と `config_file` だけを公開し、config 不正時は fail closed する。
- `FA-EQ-SPEC-012`: ROS topic identity と `AudioFrame.stream_id` を分離し、衝突する設定は fail closed する。
- `fa_eq_backend_test`: backend の数値出力、拒否 status、拒否時 state 非更新、reset state commit 条件を C++ gtest で検査する。

補助 contract:

- non-finite input、range 外 input、range 外 output は clamp せず frame drop とする。
- diagnostics が cutoff、gain、state、counter を publish する。
- package layout と backend gtest / graph gtest / pytest / lint auto wiring が標準 processing layout に一致する。

## C++ Graph Tests

- `PublishesThreeBandEqFloat32Frame`: NodeOptions で node を起動し、publish/subscribe 経路で EQ 出力、metadata 保持、duplicate epoch drop を検証する。
- `ResetsFilterStateOnForwardEpochGap`: forward epoch gap 後の出力が fresh EQ state 由来になることを検証する。

## Launch Tests

- launch argument が `node_name` / `config_file` だけであること
- default config が EQ node の責務だけを持つこと
- invalid `low.cutoff_hz` で起動失敗すること
- invalid `high.cutoff_hz` で起動失敗すること
- invalid gain で起動失敗すること
- wrong encoding で起動失敗すること

## Command

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/frequency/fa_eq/test/unit src/processing/frequency/fa_eq/test/launch -q
```
