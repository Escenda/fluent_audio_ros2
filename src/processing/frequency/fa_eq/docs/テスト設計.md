# fa_eq テスト設計

## Scope

`fa_eq` の package-local contract を pytest で検証する。C++ runtime integration は colcon build と launch/integration test の拡張で扱う。

## Unit Contract

- `FA-EQ-SPEC-001`: default config が FLOAT32LE interleaved 3-band EQ 契約を表す。
- `FA-EQ-SPEC-002`: startup config は `low.cutoff_hz < high.cutoff_hz < Nyquist` と finite gains を fail closed で検証する。
- `FA-EQ-SPEC-003`: runtime frame は source / stream / format / byte count を処理前に検証する。
- `FA-EQ-SPEC-004`: 出力 metadata は `stream_id` と `data` 以外を保存する。
- `FA-EQ-SPEC-005`: low / high split と mid residual、dB to linear gain、band mix を明示的に実装する。
- `FA-EQ-SPEC-006`: non-finite input、range 外 input、range 外 output は clamp せず frame drop とする。
- `FA-EQ-SPEC-007`: source change で channel state を reset する。
- `FA-EQ-SPEC-008`: diagnostics が cutoff、gain、state、counter を publish する。
- `FA-EQ-SPEC-009`: package layout と pytest wiring が標準 processing layout に一致する。

## Integration

`test/integration` は `AudioFrame` publish/subscribe の runtime smoke test 用に確保する。追加時は fixture で FLOAT32LE interleaved sample を生成し、実際の ROS2 graph で publish/drop の両方を確認する。

## Launch

`test/launch` は `fa_eq.launch.py` の launch 引数と config 解決を確認する test 用に確保する。
