# fa_eq テスト設計

## Scope

`fa_eq` の package-local contract を pytest と C++ gtest で検証する。ROS graph は C++ gtest、launch fail-closed は `test/launch` の pytest で検証する。

## Unit Contract

- `FA-EQ-SPEC-001`: default config が FLOAT32LE interleaved 3-band EQ 契約を表す。
- `FA-EQ-SPEC-002`: device I/O、resample、normalize、limiter、denoise を持たない。
- `FA-EQ-SPEC-003`: startup config は `low.cutoff_hz < high.cutoff_hz < Nyquist` と finite gains を fail closed で検証する。
- `FA-EQ-SPEC-004`: runtime frame は source / epoch / stream / format / byte count を処理前に検証する。
- `FA-EQ-SPEC-005`: null frame pointer を dereference 前に drop する。
- `FA-EQ-SPEC-006`: 出力 metadata は `stream_id` と `data` 以外を保存する。
- `FA-EQ-SPEC-007`: backend accept 後だけ `source_id` を bind する。
- `FA-EQ-SPEC-008`: forward epoch gap だけ fresh state を要求し、backend accept 後だけ reset counter / epoch を更新する。
- `FA-EQ-SPEC-009`: low / high split と mid residual、dB to linear gain、band mix を明示的に実装する。
- `FA-EQ-SPEC-010`: non-finite input、range 外 input、range 外 output は clamp せず frame drop とする。
- `FA-EQ-SPEC-011`: diagnostics が cutoff、gain、state、counter を publish する。
- `FA-EQ-SPEC-012`: package layout と backend gtest / graph gtest / pytest / lint auto wiring が標準 processing layout に一致する。
- `fa_eq_backend_test`: backend の数値出力、拒否 status、拒否時 state 非更新、reset state commit 条件を C++ gtest で検査する。

## C++ Graph Tests

- `PublishesThreeBandEqFloat32Frame`: NodeOptions で node を起動し、publish/subscribe 経路で EQ 出力、metadata 保持、duplicate epoch drop を検証する。
- `ResetsFilterStateOnForwardEpochGap`: forward epoch gap 後の出力が fresh EQ state 由来になることを検証する。

## Launch Tests

- launch argument が `node_name` / `config_file` だけであること
- default config が EQ node の責務だけを持つこと
- invalid `low.cutoff_hz` で起動失敗すること
- invalid `high.cutoff_hz` で起動失敗すること
- invalid gain で起動失敗すること
- wrong encoding で起動失敗すること

## Command

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/frequency/fa_eq/test/unit src/processing/frequency/fa_eq/test/launch -q
```
