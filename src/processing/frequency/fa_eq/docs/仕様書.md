# fa_eq 仕様書

## 目的

`fa_eq` は、FluentAudio の FLOAT32LE interleaved 音声 stream に対して、一次 low-pass と一次 high-pass で low / mid / high の 3 band に分割し、各 band に dB gain を適用して再合成する周波数処理 node である。

## 対象外

- device I/O
- resampling
- sample format conversion
- compressor、limiter、normalize、denoise
- multi-band dynamic EQ
- legacy alias、互換 path
- 欠損値や不正値を補う fallback

## ROS2 Interface

- package: `fa_eq`
- executable: `fa_eq_node`
- node name default: `fa_eq`
- subscribe: `input_topic`
- publish: `output_topic`
- message type: `fa_interfaces/msg/AudioFrame`

## Parameters

| name | default | constraint |
| --- | --- | --- |
| `input_topic` | `audio/sample_format/mic` | non-empty |
| `output_topic` | `audio/eq/mic` | non-empty |
| `low.cutoff_hz` | `250.0` | finite, `> 0` |
| `high.cutoff_hz` | `4000.0` | finite, `> low.cutoff_hz`, `< sample_rate / 2` |
| `gains.low_db` | `0.0` | finite |
| `gains.mid_db` | `0.0` | finite |
| `gains.high_db` | `0.0` | finite |
| `expected.sample_rate` | `16000` | `> 0` |
| `expected.channels` | `1` | `> 0` |
| `expected.encoding` | `FLOAT32LE` | exactly `FLOAT32LE` |
| `expected.bit_depth` | `32` | exactly `32` |
| `expected.layout` | `interleaved` | exactly `interleaved` |
| `qos.depth` | `10` | `> 0` |
| `qos.reliable` | `false` | bool |
| `diagnostics.publish_period_ms` | `1000` | `> 0` |

不正な parameter では node 起動を継続しない。

## Runtime Frame Contract

入力 frame は以下を満たす必要がある。

- `source_id` is non-empty
- `stream_id` is non-empty and equals `input_topic`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == expected.encoding`
- `bit_depth == expected.bit_depth`
- `layout == expected.layout`
- `data` is non-empty
- `data.size()` is divisible by `channels * sizeof(float)`
- every input sample is finite and normalized in `[-1, 1]`

不正な runtime frame は warning を出して drop する。値の推測、clamp、normalize、zero 埋めは行わない。

## Output Contract

出力 frame は入力 frame から以下を保存する。

- `header`
- `source_id`
- `sample_rate`
- `channels`
- `bit_depth`
- `encoding`
- `layout`
- `epoch`

`stream_id` は `output_topic` に更新し、`data` は EQ 後の FLOAT32LE interleaved samples に置き換える。出力 sample が finite normalized `[-1, 1]` を満たさない場合は frame を drop する。

## Band Definition

- low: `low.cutoff_hz` の一次 low-pass output
- high: `high.cutoff_hz` の一次 high-pass output
- mid: `input - low - high`

最終出力は `low * gain_low + mid * gain_mid + high * gain_high` とする。dB gain は `pow(10, db / 20)` で linear gain に変換する。

## State

filter state は channel ごとに保持する。accepted frame の `source_id` が変わった場合、状態を reset し、最初の sample から新しい source として初期化する。

## Diagnostics

diagnostics には cutoff、filter coefficient、band gain、state source、reset count、frame counters を含める。
