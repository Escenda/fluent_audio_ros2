# fa_eq 仕様書

## 目的

`fa_eq` は、FluentAudio の FLOAT32LE interleaved 音声 stream に対して、一次 low-pass と一次 high-pass で low / mid / high の 3 band に分割し、各 band に dB gain を適用して再合成する周波数処理 node である。

## 対象外

- device I/O
- resampling
- sample format conversion
- compressor、limiter、normalize、denoise
- multi-band dynamic EQ
- legacy alias、互換 path
- 欠損値や不正値を補う fallback

## ROS2 Interface

- package: `fa_eq`
- executable: `fa_eq_node`
- node name default: `fa_eq`
- subscribe: `input_topic`
- publish: `output_topic`
- message type: `fa_interfaces/msg/AudioFrame`

## Parameters

| name | default | constraint |
| --- | --- | --- |
| `input_topic` | `fa_eq/input` | non-empty |
| `output_topic` | `fa_eq/output` | non-empty |
| `input_stream_id` | `audio/sample_format/mic` | non-empty, distinct from raw / resolved ROS topic identity |
| `output.stream_id` | `audio/eq/mic` | non-empty, distinct from raw / resolved ROS topic identity and `input_stream_id` |
| `low.cutoff_hz` | `250.0` | finite, `> 0` |
| `high.cutoff_hz` | `4000.0` | finite, `> low.cutoff_hz`, `< sample_rate / 2` |
| `gains.low_db` | `0.0` | finite |
| `gains.mid_db` | `0.0` | finite |
| `gains.high_db` | `0.0` | finite |
| `expected.sample_rate` | `16000` | `> 0` |
| `expected.channels` | `1` | `> 0` |
| `expected.encoding` | `FLOAT32LE` | exactly `FLOAT32LE` |
| `expected.bit_depth` | `32` | exactly `32` |
| `expected.layout` | `interleaved` | exactly `interleaved` |
| `qos.depth` | `10` | `> 0` |
| `qos.reliable` | `false` | bool |
| `diagnostics.publish_period_ms` | `1000` | `> 0` |
| `diagnostics.qos.depth` | `10` | `> 0` |
| `diagnostics.qos.reliable` | `true` | bool |

不正な parameter では node 起動を継続しない。

## Runtime Frame Contract

入力 frame は以下を満たす必要がある。

- `source_id` is non-empty
- `stream_id` is non-empty and equals `input_stream_id`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == expected.encoding`
- `bit_depth == expected.bit_depth`
- `layout == expected.layout`
- `data` is non-empty
- `data.size()` is divisible by `channels * sizeof(float)`
- every input sample is finite and normalized in `[-1, 1]`
- after the first accepted frame, `source_id` equals the bound state source
- after the first accepted frame, `epoch` is strictly greater than the previous accepted epoch

不正な runtime frame は warning を出して drop する。値の推測、clamp、normalize、zero 埋めは行わない。

## Output Contract

出力 frame は入力 frame から以下を保存する。

- `header`
- `source_id`
- `sample_rate`
- `channels`
- `bit_depth`
- `encoding`
- `layout`
- `epoch`

`stream_id` は `output.stream_id` に更新し、`data` は EQ 後の FLOAT32LE interleaved samples に置き換える。出力 sample が finite normalized `[-1, 1]` を満たさない場合は frame を drop する。

## Band Definition

- low: `low.cutoff_hz` の一次 low-pass output
- high: `high.cutoff_hz` の一次 high-pass output
- mid: `input - low - high`

最終出力は `low * gain_low + mid * gain_mid + high * gain_high` とする。dB gain は `pow(10, db / 20)` で linear gain に変換する。

## State

filter state は channel ごとに保持する。最初に backend が受理した frame の `source_id` を state source として bind し、それ以降に異なる `source_id` の frame が来た場合は drop する。`epoch` が前回以下の frame は duplicate / regression として drop する。`epoch` が前回 + 1 ではない forward gap の場合は、古い low / high split state を使わず fresh state で処理する。

backend が frame を拒否した場合、`source_id` bind、`epoch` 更新、state reset counter 更新は行わない。

## Diagnostics

diagnostics には topic/stream identity、cutoff、filter coefficient、band gain、state source、state reset count、frame counters を含める。

## Launch Contract

`fa_eq.launch.py` が公開する launch argument は `node_name` と `config_file` のみである。cutoff、gain、expected format、QoS、diagnostics interval は config file にだけ置く。invalid config は node 起動失敗として扱う。

## Traceability

- `FA-EQ-SPEC-001`: `fa_eq` は device I/O、resample、format conversion、compressor、limiter、normalize、denoise を行わない。
- `FA-EQ-SPEC-002`: `fa_eq` は `FLOAT32LE` / 32bit / interleaved のみ扱う。
- `FA-EQ-SPEC-003`: `low.cutoff_hz` は finite かつ `> 0.0` である。
- `FA-EQ-SPEC-004`: `high.cutoff_hz` は finite、`> low.cutoff_hz`、Nyquist 未満である。
- `FA-EQ-SPEC-005`: `gains.*_db` は finite である。
- `FA-EQ-SPEC-006`: low / mid / high split と dB gain mix を channel ごとに適用する。
- `FA-EQ-SPEC-007`: 出力 `source_id` は保持し、`stream_id` は `output.stream_id` にする。
- `FA-EQ-SPEC-008`: null frame pointer は dereference せず drop する。
- `FA-EQ-SPEC-009`: `source_id` と monotonic `epoch` により state の適用範囲を固定する。
- `FA-EQ-SPEC-010`: forward `epoch` gap は fresh EQ state で処理し、reset 数を diagnostics に出す。
- `FA-EQ-SPEC-011`: launch は `node_name` と `config_file` だけを公開し、config 不正時は fail closed する。
- `FA-EQ-SPEC-012`: ROS topic identity と `AudioFrame.stream_id` を分離し、衝突する設定は fail closed する。
