# fa_eq アルゴリズム詳細説明書

## 概要

`fa_eq` は、各 channel 独立に一次 low-pass と一次 high-pass を入力 sample へ並列適用する。mid band は入力から low と high を引いた残差として定義する。入力 layout は FLOAT32LE interleaved のみを受け付ける。

## 係数

`dt = 1 / sample_rate`

low split:

```text
low_rc = 1 / (2 * pi * low_cutoff_hz)
low_alpha = dt / (low_rc + dt)
```

high split:

```text
high_rc = 1 / (2 * pi * high_cutoff_hz)
high_alpha = high_rc / (high_rc + dt)
```

`low_cutoff_hz` は finite かつ `> 0`、`high_cutoff_hz` は finite かつ `> low_cutoff_hz` かつ Nyquist 未満でなければならない。

## Gain

各 band の dB gain は以下で linear gain に変換する。

```text
gain_linear = pow(10, gain_db / 20)
```

`gains.low_db`、`gains.mid_db`、`gains.high_db` は finite でなければならない。変換後の linear gain が finite かつ `> 0` でない場合は起動失敗とする。

## Recurrence

sample `x[n]` に対して、channel state は `prev_y_low`、`prev_x_hp`、`prev_y_hp` を持つ。

```text
low[n] = prev_y_low + low_alpha * (x[n] - prev_y_low)
high[n] = high_alpha * (prev_y_hp + x[n] - prev_x_hp)
mid[n] = x[n] - low[n] - high[n]
out[n] = low[n] * gain_low + mid[n] * gain_mid + high[n] * gain_high
```

source ごとの最初の accepted sample は state 初期化に使う。この初回 sample では `low = x[n]`、`high = 0`、`mid = 0` として扱う。

## Source Reset

`source_id` が変わった accepted frame では、全 channel state を reset する。reset 後は新しい source の最初の sample から state を初期化し、diagnostics の `source_resets` を increment する。

## Invalid Frame Handling

以下は frame 全体を drop する。

- contract 不一致
- 空 data
- frame byte size が `channels * sizeof(float)` で割り切れない
- 入力 sample が finite normalized `[-1, 1]` ではない
- split または mix 計算結果が non-finite
- final output sample が finite normalized `[-1, 1]` ではない

drop 時に clamp、normalize、resampling、zero 補完は行わない。
