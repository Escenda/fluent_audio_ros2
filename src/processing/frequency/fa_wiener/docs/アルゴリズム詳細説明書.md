# fa_wiener アルゴリズム詳細説明書

1. startup config で frame length、hop length、window type、gain floor、noise estimate source を検証する。
2. input `AudioFrame` の identity と format metadata を検証する。
3. channel ごとに windowing と STFT を適用する。
4. signal/noise power estimate から Wiener gain を計算する。
5. gain floor を適用し、original phase と修正 magnitude から inverse STFT を行う。
6. overlap-add で frame を再構成し、finite/normalized contract を検証する。
7. 契約を満たす場合だけ publish する。

noise estimate がない場合に pass-through しない。
入力長が frame/hop contract と一致しない場合は、zero padding や truncate を隠さず drop する。

## Backend Boundary

backend は STFT、Wiener gain computation、inverse STFT のみを担当し、ROS2 topic/message を知らない。
noise estimate source は明示 config または専用 estimator node で管理する。
