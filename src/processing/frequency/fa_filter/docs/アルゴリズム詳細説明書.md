# fa_filter アルゴリズム詳細説明書

1. startup config の ordered filter stage list を読み込む。
2. 各 stage の package、launch file、params file、input/output topic、frequency-specific parameters を検証する。
3. 隣接 stage の output format と input format が一致することを検証する。
4. stage が frequency category に属することを検証する。
5. 検証に通った stage だけを launch include / component composition の候補にする。
6. 欠損または不一致がある場合は起動失敗し、stage skip や自動補完はしない。

`fa_filter` 自体は samples を直接 filter する DSP backend ではない。
実際の filter は `fa_high_pass`、`fa_low_pass`、`fa_notch` などの専用 package が担当し、`fa_filter` は順序と契約整合性を管理する。
