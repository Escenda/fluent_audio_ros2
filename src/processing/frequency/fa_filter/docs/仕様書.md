# fa_filter 仕様書

## 1. 目的

`fa_filter` は frequency processing pipeline を明示的に構成する wrapper / orchestration node である。
high-pass、low-pass、band-pass、notch、EQ、de-esser、spectral subtraction、Wiener filtering を設定順に接続し、周波数編集の責務を `fa_in` / `fa_out` / AI node に混ぜない。

## 2. Pipeline Contract

pipeline config は ordered stage list として定義する。
各 stage は次を必須にする。

- `stage.name`
- `stage.package`
- `stage.launch_file`
- `stage.params_file`
- `stage.input_topic`
- `stage.output_topic`
- `stage.input_stream_id`
- `stage.output.stream_id`
- expected input format
- expected output format

`stage.input_topic` / `stage.output_topic` は ROS 搬送路の identity であり、
`AudioFrame.stream_id` ではない。隣接 stage の output/input contract が一致しない場合は起動失敗にする。
隣接 stage では前段の `stage.output.stream_id` と後段の `stage.input_stream_id`
が一致する必要がある。

## 3. 責務外

- device/source/sink adapter
- resample
- bit-depth conversion
- channel count conversion
- dynamics processing
- VAD / ASR / embedding
- stage mismatch の自動補正

## 4. fail closed 条件

- stage list が空
- stage package / launch / params_file 欠落
- adjacent stage の output/input format mismatch
- adjacent stage の stream identity mismatch
- stage topic と stream identity の衝突
- unsupported stage type
- missing cutoff/frequency/band config
