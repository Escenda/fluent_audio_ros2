# fa_deesser テスト設計

## 1. 対象

`fa_deesser` package の設定契約、AudioFrame 契約、backend 境界、アルゴリズム、
diagnostics、package layout、pytest/gTest/CMake 連携を対象とする。

## 2. Unit contract tests

`test/unit/test_fa_deesser_audio_frame_contract.py` で以下を確認する。

- default config が `FLOAT32LE` / `32bit` / `interleaved`、reliable QoS、topic/stream identity 分離を要求する
- startup validation が不正設定と stream identity 衝突を fallback せず `std::runtime_error` にする
- frame validation が stream identity、metadata、data size を検証する
- de-esser node が ROS 境界、backend が low/high split、threshold、attenuation gain、recombine を行う
- `source_id` 変更で per-channel filter state を reset する
- 不正 sample / output を clamp や normalize せず drop する
- diagnostics に topic、stream identity、設定値、counters を publish する
- backend が `ProcessStatus` で拒否理由を返し、ROS 型に依存しない
- package-local pytest と backend gTest が CMake から実行される

## 3. Backend C++ tests

`test/cpp/test_internal_split_band_deesser_backend.cpp` で以下を確認する。

- threshold 未満の入力は sample を維持する
- threshold 以上の high band だけを attenuate し、attenuated sample count を返す
- empty / byte alignment / NaN / normalized range violation を `ProcessStatus` で返す
- rejected frame では reset state を commit しない
- reset state 指定時は次 frame を fresh state から処理する

## 4. 未実装の integration tests

`test/integration` と `test/launch` は package layout と将来の ROS2 launch/integration
検証用に確保する。現時点では `.gitkeep` のみを置く。
