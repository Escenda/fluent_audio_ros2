# fa_deesser テスト設計

## 1. 対象

`fa_deesser` package の設定契約、AudioFrame 契約、アルゴリズム境界、diagnostics、
package layout、pytest/CMake 連携を対象とする。

## 2. Unit contract tests

`test/unit/test_fa_deesser_audio_frame_contract.py` で以下を確認する。

- default config が `FLOAT32LE` / `32bit` / `interleaved` と reliable QoS を要求する
- startup validation が不正設定を fallback せず `std::runtime_error` にする
- frame validation が metadata と data size を検証する
- de-esser が low/high split、threshold、attenuation gain、recombine を行う
- `source_id` 変更で per-channel filter state を reset する
- 不正 sample / output を clamp や normalize せず drop する
- diagnostics に設定値と counters を publish する
- package-local pytest が CMake から実行される

## 3. 未実装の integration tests

`test/integration` と `test/launch` は package layout と将来の ROS2 launch/integration
検証用に確保する。現時点では `.gitkeep` のみを置く。
