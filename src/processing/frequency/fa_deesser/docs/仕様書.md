# fa_deesser 仕様書

## 1. 目的

`fa_deesser` は FluentAudio の `AudioFrame` ストリームに対して、歯擦音などの
高域成分だけを検出して減衰する frequency processing node である。

## 2. 非目的

この package は以下を行わない。

- device I/O
- resampling
- sample format conversion
- channel count conversion
- compressor / limiter / normalize
- denoise / VAD / ASR

## 3. ノード

| 項目 | 値 |
| --- | --- |
| package | `fa_deesser` |
| executable | `fa_deesser_node` |
| node name | `fa_deesser` |

## 4. Topic

| 種別 | parameter | default |
| --- | --- | --- |
| subscribe | `input_topic` | `fa_deesser/input` |
| publish | `output_topic` | `fa_deesser/output` |

`input_topic` と `output_topic` は ROS 搬送路の identity である。`AudioFrame.stream_id`
とは分離する。空文字は起動失敗とする。

## 5. 起動時パラメータ

| parameter | default | 条件 |
| --- | ---: | --- |
| `input_stream_id` | `audio/normalized/mic` | non-empty、ROS topic と異なる、`output.stream_id` と異なる |
| `output.stream_id` | `audio/deessed/mic` | non-empty、ROS topic と異なる、`input_stream_id` と異なる |
| `detector.cutoff_hz` | `4500.0` | finite, `0.0 < value < Nyquist` |
| `detector.threshold` | `0.08` | finite, `0.0 <= value <= 1.0` |
| `detector.attenuation_db` | `-9.0` | finite, `-120.0 <= value <= 0.0` |
| `expected.sample_rate` | `16000` | `> 0` |
| `expected.channels` | `1` | `> 0` |
| `expected.encoding` | `FLOAT32LE` | `FLOAT32LE` のみ |
| `expected.bit_depth` | `32` | `32` のみ |
| `expected.layout` | `interleaved` | `interleaved` のみ |
| `qos.depth` | `10` | `> 0` |
| `qos.reliable` | `true` | `true` なら reliable、`false` なら best effort |
| `diagnostics.publish_period_ms` | `1000` | `> 0` |
| `diagnostics.qos.depth` | `10` | `> 0` |
| `diagnostics.qos.reliable` | `true` | bool |

不正な起動時パラメータは fallback せず、例外で起動を失敗させる。

## 6. 入力フレーム契約

入力 `AudioFrame` は以下を満たす必要がある。

- `source_id` が非空
- `stream_id` が非空、かつ `input_stream_id` と一致
- `sample_rate` が `expected.sample_rate` と一致
- `channels` が `expected.channels` と一致
- `encoding == FLOAT32LE`
- `bit_depth == 32`
- `layout == interleaved`
- `data` が非空
- `data.size()` が `channels * sizeof(float)` で割り切れる
- 全サンプルが finite
- 全サンプルが `[-1.0, 1.0]` の正規化範囲内

実行時に契約違反を検出した frame は publish せず、警告ログを出して drop する。
値の補完、clamp、normalize、別形式としての解釈は行わない。

## 7. 出力フレーム契約

出力 `AudioFrame` は入力から以下を preserve する。

- header
- source_id
- sample_rate
- channels
- bit_depth
- encoding
- layout
- epoch

`stream_id` は `output.stream_id` に更新し、`data` は de-esser 適用後の `FLOAT32LE`
interleaved bytes に更新する。

## 8. 状態

first-order low/high split の low-band state は channel ごとに保持する。
`source_id` が前回処理 frame と異なる場合は、全 channel の filter state を `0.0`
に reset する。別 source の履歴を混ぜた処理は行わない。

## 9. Diagnostics

`diagnostics` topic に `DiagnosticArray` を publish する。status name は
`fa_deesser` とし、設定値と counters を含める。

- `detector_cutoff_hz`
- `detector_threshold`
- `detector_attenuation_db`
- `detector_attenuation_gain`
- `input_topic`
- `output_topic`
- `input_stream_id`
- `output_stream_id`
- `expected_sample_rate`
- `expected_channels`
- `frames_in`
- `frames_out`
- `frames_dropped`
- `samples_attenuated`
- `filter_resets`

## 10. 仕様 ID

| SPEC ID | 内容 |
| --- | --- |
| `FA-DEESSER-SPEC-001` | `fa_deesser` は de-esser frequency processing のみを行い、I/O、resample、format conversion、channel conversion、compressor、limiter、normalize、denoise、VAD、ASR を持たない。 |
| `FA-DEESSER-SPEC-002` | ROS topic identity と `AudioFrame.stream_id` は分離し、空 topic、空 stream ID、topic/stream 衝突、input/output stream 衝突は起動失敗にする。 |
| `FA-DEESSER-SPEC-003` | 起動時 config は detector range、expected format、QoS、diagnostics period を fail closed で検証する。 |
| `FA-DEESSER-SPEC-004` | runtime `AudioFrame` は source/stream identity、sample rate、channels、encoding、bit depth、layout、data size を処理前に検証する。 |
| `FA-DEESSER-SPEC-005` | 出力 frame は header、source_id、sample_rate、channels、bit_depth、encoding、layout、epoch を保持し、`stream_id` だけを `output.stream_id` に更新する。 |
| `FA-DEESSER-SPEC-006` | backend は first-order low/high split、threshold 判定、attenuation gain、recombine を所有する。 |
| `FA-DEESSER-SPEC-007` | `source_id` が変わった場合は per-channel filter state を reset し、別 source の履歴を混ぜない。 |
| `FA-DEESSER-SPEC-008` | non-finite sample、normalized range 外 input/output、empty、misaligned payload は clamp / normalize / reinterpret せず reject する。 |
| `FA-DEESSER-SPEC-009` | diagnostics は detector config、topic/stream identity、expected format、frame/sample counters、filter reset count を publish する。 |
| `FA-DEESSER-SPEC-010` | backend は ROS2 topic/message を知らず、typed `ProcessStatus` と message conversion boundary を持つ。 |
| `FA-DEESSER-SPEC-011` | package は標準 processing layout、backend document、pytest、backend gtest、launch/integration placeholder を持つ。 |
