# fa_deesser アルゴリズム詳細説明書

## 1. 入力

入力は `FLOAT32LE` の正規化済み interleaved `AudioFrame` に限定する。
サンプル値は `[-1.0, 1.0]` の有限値でなければならない。

## 2. Split-band

channel ごとに first-order low-pass state を持つ。

```text
alpha = 1 - exp(-2 * pi * cutoff_hz / sample_rate)
low[n] = low[n-1] + alpha * (x[n] - low[n-1])
high[n] = x[n] - low[n]
```

`cutoff_hz` は起動時に `0 < cutoff_hz < Nyquist` を検証する。
`alpha` が有限で `(0, 1)` に入らない場合、その frame は drop する。

## 3. De-essing

高域成分の絶対値が `detector.threshold` 以上の場合だけ、高域成分に
`detector.attenuation_db` から変換した linear gain を掛ける。

```text
gain = 10 ^ (attenuation_db / 20)
processed_high = high * gain  if abs(high) >= threshold
processed_high = high         otherwise
y = low + processed_high
```

低域成分は直接減衰しない。

## 4. 状態 reset

`source_id` が変わった frame を受け取った場合、全 channel の low-pass state を
`0.0` に reset してから処理する。source 間で filter 履歴を共有しない。

## 5. 不正値の扱い

入力 sample、filter coefficient、出力 sample のいずれかが不正な場合は、その frame
全体を drop する。clamp、normalize、zero fill、別形式への解釈は行わない。
