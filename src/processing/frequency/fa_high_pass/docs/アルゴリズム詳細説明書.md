# fa_high_pass アルゴリズム詳細説明書

## 入力検証

1. `source_id` と `stream_id` が空でないことを確認する。
2. 2 frame 目以降は、最初に受理した `source_id` と一致することを確認する。
3. `stream_id == input_stream_id` を確認する。
4. `layout == interleaved` を確認する。
5. `encoding == FLOAT32LE`、`bit_depth == 32` を確認する。
6. `sample_rate` と `channels` が config の `expected.*` と一致することを確認する。
7. `data.size()` が `channels * sizeof(float)` の倍数であることを確認する。

不一致は frame skip として扱う。別 format を暗黙変換すると format node の責務を侵食し、別 stream の state を再利用すると filter 出力の意味が壊れるためである。

## Cutoff Validation

`filter.cutoff_hz` は起動時に次の条件を満たす必要がある。

- finite
- `> 0.0`
- `< expected.sample_rate / 2.0`

不正な cutoff は起動失敗とする。Nyquist 以上の cutoff を補正して継続する処理は行わない。

## First-Order High-Pass Filter

起動時に次を計算する。

```text
dt = 1 / sample_rate
RC = 1 / (2 * pi * cutoff_hz)
alpha = RC / (RC + dt)
```

各 channel は独立した `previous_input`、`previous_output`、`initialized` を持つ。
初回 sample では state を初期化し、出力 sample は `0.0` とする。2 sample 目以降は次式を適用する。

```text
y[n] = alpha * (y[n-1] + x[n] - x[n-1])
```

`AudioFrame.data` は interleaved FLOAT32LE として 4 byte ごとに読み書きする。出力値は high-pass filter の結果そのものであり、normalize、gain、limit、clamp は行わない。

backend は入力 frame 全体を一時 state に対して処理し、全 sample が成功した場合だけ channel state を更新する。途中で non-finite input、FLOAT32LE として表現できない output、または byte 長不整合を検出した場合、その frame は拒否し、前回までの filter state を保持する。

## 出力

- `header`、`source_id`、sample format metadata は入力から保持する。
- `stream_id` は `output.stream_id` に更新する。
- diagnostics には `frames_in`、`frames_out`、`frames_dropped`、`filter_cutoff_hz`、`filter_alpha` を出す。
