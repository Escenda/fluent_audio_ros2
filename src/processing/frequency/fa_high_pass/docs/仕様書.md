# fa_high_pass 仕様書

`fa_high_pass` は周波数処理系の processing node である。入力された `FLOAT32LE`
`AudioFrame` に一次 high-pass filter を適用し、別 topic へ publish する。

## 責務

- `input_topic` から `AudioFrame` を購読する。
- `filter.cutoff_hz` を cutoff として一次 high-pass filter を適用する。
- channel ごとの filter state を node 内に保持する。
- filter state は `source_id` と連続する `epoch` にだけ紐づける。
- `output_topic` へ同じ format の `AudioFrame` を publish する。
- diagnostics に入出力 frame 数、drop 数、cutoff、filter alpha を publish する。

## 非責務

- device I/O
- resample
- bit depth 変換
- channel 変換
- interleave / planar 変換
- normalize
- gain
- limiter / compressor / AGC
- VAD / ASR / feature extraction

## Parameters

| parameter | required | 内容 |
| --- | --- | --- |
| `input_topic` | yes | 入力 `AudioFrame` topic。入力 frame の `stream_id` と一致すること |
| `output_topic` | yes | 出力 `AudioFrame` topic |
| `filter.cutoff_hz` | yes | 有限値、`> 0.0`、かつ `< expected.sample_rate / 2.0` |
| `expected.sample_rate` | yes | 期待 sample rate |
| `expected.channels` | yes | 期待 channel 数 |
| `expected.encoding` | yes | `FLOAT32LE` 固定 |
| `expected.bit_depth` | yes | `32` 固定 |
| `expected.layout` | yes | `interleaved` 固定 |
| `qos.depth` | yes | ROS topic queue depth |
| `qos.reliable` | yes | reliable / best effort |
| `diagnostics.publish_period_ms` | yes | diagnostics publish interval |
| `diagnostics.qos.depth` | yes | diagnostics topic queue depth |
| `diagnostics.qos.reliable` | yes | diagnostics reliable / best effort |

## Error Handling

- 起動時 parameter が不正な場合は例外で起動失敗する。
- frame identity、format、layout、data size が不正な frame は publish せず drop する。
- null frame pointer は dereference せず drop する。
- 最初に受理した `source_id` と異なる frame は、既存 state を流用せず drop する。
- 直前に publish した frame 以下の `epoch` を持つ frame は、duplicate / regression として drop する。
- 直前に publish した frame から `epoch` が 1 より大きく進んだ場合は、欠落による stream discontinuity として古い filter state を破棄し、fresh state で処理する。
- 入力 sample または filter 出力が finite でない frame は publish せず drop する。
- format 変換、resample、channel 変換、normalize、gain、limit による暗黙補正は行わない。

## Traceability

- `FA-HIGH-PASS-SPEC-001`: `fa_high_pass` は device I/O を行わない。
- `FA-HIGH-PASS-SPEC-002`: `fa_high_pass` は `FLOAT32LE` / 32bit / interleaved のみ扱う。
- `FA-HIGH-PASS-SPEC-003`: `filter.cutoff_hz` は有限値、`> 0.0`、Nyquist 未満である。
- `FA-HIGH-PASS-SPEC-004`: channel ごとの一次 high-pass filter state を保持する。
- `FA-HIGH-PASS-SPEC-005`: 出力 `source_id` は保持し、`stream_id` は `output_topic` にする。
- `FA-HIGH-PASS-SPEC-006`: launch は `node_name` と `config_file` だけを受け取り、node 固有 parameter は config file で fail closed 検証する。
- `FA-HIGH-PASS-SPEC-007`: filter state は同一 `source_id` かつ連続する `epoch` でだけ継続し、duplicate / regression は drop、forward gap は fresh state で処理する。
