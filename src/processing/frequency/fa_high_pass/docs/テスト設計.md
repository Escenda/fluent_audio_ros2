# fa_high_pass テスト設計

## Unit Contract Tests

| test | 対応仕様 | 確認内容 |
| --- | --- | --- |
| `test_default_config_requires_float32_interleaved_contract` | `FA-HIGH-PASS-SPEC-002`, `003` | default config が明示 format と valid cutoff を持つ |
| `test_high_pass_does_not_hide_other_processing_or_io_responsibilities` | `FA-HIGH-PASS-SPEC-001` | device API、resample、normalize、gain、limit を持たない |
| `test_high_pass_validates_frame_contract_before_processing` | `FA-HIGH-PASS-SPEC-002` | identity、format、data size を処理前に検証する |
| `test_high_pass_preserves_source_identity_and_updates_stream_identity` | `FA-HIGH-PASS-SPEC-005` | `source_id` を保持し、`stream_id` を `output_topic` にする |
| `test_high_pass_binds_source_only_after_backend_accepts_frame` | `FA-HIGH-PASS-SPEC-005` | backend が拒否した初回 frame で source を固定しない |
| `test_high_pass_uses_first_order_per_channel_state` | `FA-HIGH-PASS-SPEC-004` | channel ごとの previous input/output state と一次 HPF 式を使う |
| `fa_high_pass_backend_test` | `FA-HIGH-PASS-SPEC-004` | backend の数値出力、拒否 status、拒否時 state 非更新を C++ gtest で検証する |
| `test_cutoff_parameter_is_required_and_range_checked` | `FA-HIGH-PASS-SPEC-003` | cutoff が有限、正、Nyquist 未満で検証される |
| `test_package_layout_matches_standard_processing_layout` | layout | 標準 processing package layout を満たす |
| `test_colcon_runs_pytest_contracts` | test runner | CMake が gtest / pytest を登録し plugin autoload を無効化する |

## Integration / Launch

`test/integration` と `test/launch` は directory contract を先に固定する。ROS graph を使った publish/subscribe smoke test は、親側の system integration で package wiring が追加された後に実装する。
