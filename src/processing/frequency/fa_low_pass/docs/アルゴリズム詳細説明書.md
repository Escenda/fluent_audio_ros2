# fa_low_pass アルゴリズム詳細説明書

## Filter

一次 low-pass filter を channel ごとに適用する。

```text
rc = 1 / (2 * pi * cutoff_hz)
dt = 1 / sample_rate
alpha = dt / (rc + dt)
y[n] = y[n-1] + alpha * (x[n] - y[n-1])
```

`cutoff_hz` は finite, `> 0`, `< Nyquist` を起動時に検証する。`alpha` が finite
かつ `(0, 1)` に入らない場合は起動しない。

## State Initialization

state は channel ごとに `previous_output` と `initialized` を持つ。各 channel の
最初の accepted sample はそのまま output し、同時に `previous_output` として保持する。
以降の sample から recurrence を適用する。

## Source Reset

`source_id` が前回 accepted frame と異なる場合、node は backend に `reset_state=true`
を渡す。backend は一時 state を未初期化状態へ戻してから処理し、frame 全体が成功した
場合だけ reset 後 state を commit する。これにより異なる microphone/source の履歴が
混ざらず、かつ不正な source 切り替え frame で既存 state が破壊されない。

## Frame Atomicity

入力 sample または出力 sample が non-finite または normalized range `[-1, 1]` 外の
場合、その frame 全体を drop する。処理途中の state は commit しない。

## Byte Layout

入力 `data` は `FLOAT32LE` interleaved bytes として扱う。`std::memcpy` で `float`
へ読み出し、同じ layout で出力 `data` に書き戻す。format conversion は行わない。
