# fa_low_pass 仕様書

## 目的

`fa_low_pass` は FluentAudio の周波数処理パッケージであり、`FLOAT32LE`
interleaved の `fa_interfaces/msg/AudioFrame` に一次 low-pass filter を適用する。

## ノード

- package: `fa_low_pass`
- executable: `fa_low_pass_node`
- node name default: `fa_low_pass`

## 入出力

- subscribe: `input_topic`
- publish: `output_topic`
- diagnostics: `diagnostics`

出力フレームは入力フレームの `header`, `source_id`, `sample_rate`, `channels`,
`bit_depth`, `encoding`, `layout`, `epoch` を保持し、`stream_id` を `output_topic`
へ更新する。`data` は low-pass 適用後の `FLOAT32LE` interleaved bytes へ更新する。

## パラメータ

- `input_topic`: 入力 `AudioFrame` topic。空文字は禁止。
- `output_topic`: 出力 `AudioFrame` topic。空文字は禁止。
- `filter.cutoff_hz`: finite, `> 0`, `< sample_rate / 2`。既定値は `3400.0`。
- `expected.sample_rate`: `> 0`。既定値は `16000`。
- `expected.channels`: `> 0`。既定値は `1`。
- `expected.encoding`: `FLOAT32LE` 固定。
- `expected.bit_depth`: `32` 固定。
- `expected.layout`: `interleaved` 固定。
- `qos.depth`: `> 0`。
- `qos.reliable`: `false` なら best effort、`true` なら reliable。
- `diagnostics.publish_period_ms`: `> 0`。

起動時設定が不正な場合は例外で fail closed する。

## Runtime Frame Contract

処理対象フレームは以下を満たす必要がある。

- `source_id` が空でない。
- `stream_id` が空でない。
- `stream_id == input_topic`。
- `sample_rate == expected.sample_rate`。
- `channels == expected.channels`。
- `encoding == FLOAT32LE`。
- `bit_depth == 32`。
- `layout == interleaved`。
- `data` が空でない。
- `data.size()` が `channels * sizeof(float)` で割り切れる。
- 全入力 sample が finite かつ normalized range `[-1.0, 1.0]` に入る。

不正な runtime frame は warning を出して drop する。値の clamp、normalize、
format conversion、補完は行わない。

## Source State

filter state は channel ごとに保持する。`source_id` が変わった場合、古い source の
state を新しい source へ混ぜないため、state を reset して新しい source の先頭 sample
から再初期化する。

## 非責務

この package は device I/O、resampling、sample format conversion、channel conversion、
gain、normalize、limiter、denoise、VAD、feature extraction を行わない。
