# fa_low_pass テスト設計

## Scope

`test/unit/test_fa_low_pass_audio_frame_contract.py` は package の静的 contract を検査する。
`test/cpp/test_low_pass_graph.cpp` は ROS graph 上の publish/subscribe contract を検査する。
`test/launch/test_fa_low_pass_launch_contract.py` は launch file と fail-closed 起動経路を検査する。
実 audio device は扱わない。

## 検査項目

- default config が `FLOAT32LE` / 32-bit / interleaved / 16 kHz / 1 ch を要求する。
- default config は post-resample の `audio/resample16k/mic` を入力 stream identity にする。
- default config は ROS topic と `AudioFrame.stream_id` を分離する。
- `filter.cutoff_hz` の既定値が `3400.0` で Nyquist 未満である。
- device I/O、resampling、format conversion、gain、normalize、limiter、denoise を持たない。
- 起動時 parameter validation が fail closed である。
- runtime frame validation が source/stream/format/data size を検査する。
- stream identity が ROS topic と衝突する設定を fail closed にする。
- null frame pointer を dereference せず drop する。
- `stream_id` を `output.stream_id` へ更新し、それ以外の identity を保持する。
- low-pass coefficient と recurrence が仕様式に一致する。
- C++ gtest が backend の数値出力、拒否 status、reset state commit 条件を検査する。
- ROS graph test が連続 epoch の state carry、forward epoch gap の state reset、duplicate / regression drop を検査する。
- diagnostics が cutoff、alpha、state source、counter を含む。
- launch test が config file 以外に node 固有 parameter を expose しないこと、invalid cutoff / encoding が起動失敗することを検査する。
- package layout と colcon backend gtest / graph gtest / pytest registration が標準形に一致する。

## 実行

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/frequency/fa_low_pass/test/unit src/processing/frequency/fa_low_pass/test/launch -q
```
