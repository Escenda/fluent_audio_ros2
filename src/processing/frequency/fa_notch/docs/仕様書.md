# fa_notch 仕様書

`fa_notch` は周波数処理系の processing node である。入力された `FLOAT32LE`
`AudioFrame` に二次 notch filter を適用し、別 topic へ publish する。

## 責務

- `input_topic` から `AudioFrame` を購読する。
- `filter.center_hz` を中心周波数、`filter.q` を Q 値として二次 notch filter を適用する。
- channel ごとの biquad filter state を node 内に保持する。
- `source_id` と `epoch` により、state を適用してよい連続 stream だけを処理する。
- `output_topic` へ同じ format の `AudioFrame` を publish する。
- diagnostics に入出力 frame 数、drop 数、center、Q、正規化済み係数、state source、state reset 数を publish する。

## 非責務

- device I/O
- resample
- bit depth 変換
- channel 変換
- interleave / planar 変換
- normalize
- gain
- limiter / compressor / AGC
- VAD / ASR / feature extraction

## Parameters

| parameter | required | 内容 |
| --- | --- | --- |
| `input_topic` | yes | 入力 `AudioFrame` topic。入力 frame の `stream_id` と一致すること |
| `output_topic` | yes | 出力 `AudioFrame` topic |
| `filter.center_hz` | yes | 有限値、`> 0.0`、かつ `< expected.sample_rate / 2.0` |
| `filter.q` | yes | 有限値、`> 0.0` |
| `expected.sample_rate` | yes | 期待 sample rate |
| `expected.channels` | yes | 期待 channel 数 |
| `expected.encoding` | yes | `FLOAT32LE` 固定 |
| `expected.bit_depth` | yes | `32` 固定 |
| `expected.layout` | yes | `interleaved` 固定 |
| `qos.depth` | yes | ROS topic queue depth |
| `qos.reliable` | yes | reliable / best effort |
| `diagnostics.publish_period_ms` | yes | diagnostics publish interval |

## Launch Contract

`fa_notch.launch.py` が公開する launch argument は次の 2 つだけである。

- `node_name`
- `config_file`

filter parameter、expected format、QoS、diagnostics interval は config file にだけ置く。
launch argument で個別 parameter を上書きできる形にはしない。profile や system 側からは
config file を選ぶだけにし、現場依存 binding と DSP parameter の境界を混ぜない。

## Error Handling

- 起動時 parameter が不正な場合は例外で起動失敗する。
- null `AudioFrame` pointer は dereference せず drop する。
- frame identity、format、layout、data size が不正な frame は publish せず drop する。
- 最初に受理した `source_id` と異なる frame は、既存 state を流用せず drop する。
- `epoch` が最後に受理した frame 以下の場合は、duplicate / regression として drop する。
- `epoch` が前回 + 1 ではない forward gap の場合は、古い biquad state を捨て、fresh state で処理する。
- 入力 sample または filter 出力が finite でない frame は publish せず drop する。
- backend が frame を拒否した場合、`source_id` bind、`epoch` 更新、state reset counter 更新は行わない。
- format 変換、resample、channel 変換、normalize、gain、limit による暗黙補正は行わない。

## Traceability

- `FA-NOTCH-SPEC-001`: `fa_notch` は device I/O を行わない。
- `FA-NOTCH-SPEC-002`: `fa_notch` は `FLOAT32LE` / 32bit / interleaved のみ扱う。
- `FA-NOTCH-SPEC-003`: `filter.center_hz` は有限値、`> 0.0`、Nyquist 未満である。
- `FA-NOTCH-SPEC-004`: `filter.q` は有限値、`> 0.0` である。
- `FA-NOTCH-SPEC-005`: channel ごとの二次 biquad filter state を保持する。
- `FA-NOTCH-SPEC-006`: 出力 `source_id` は保持し、`stream_id` は `output_topic` にする。
- `FA-NOTCH-SPEC-007`: null frame pointer は dereference せず drop する。
- `FA-NOTCH-SPEC-008`: `source_id` と monotonic `epoch` により biquad state の適用範囲を固定する。
- `FA-NOTCH-SPEC-009`: forward `epoch` gap は fresh biquad state で処理し、reset 数を diagnostics に出す。
- `FA-NOTCH-SPEC-010`: launch は `node_name` と `config_file` だけを公開し、config 不正時は fail closed する。
