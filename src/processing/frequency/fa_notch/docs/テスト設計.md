# fa_notch テスト設計

## Unit Contract Tests

| test | 対応仕様 | 確認内容 |
| --- | --- | --- |
| `test_default_config_requires_float32_interleaved_contract` | `FA-NOTCH-SPEC-002`, `003`, `004` | default config が明示 format と valid center / Q を持つ |
| `test_notch_does_not_hide_other_processing_or_io_responsibilities` | `FA-NOTCH-SPEC-001` | device API、resample、normalize、gain、limit を持たない |
| `test_notch_validates_frame_contract_before_processing` | `FA-NOTCH-SPEC-002` | identity、format、data size を処理前に検証する |
| `test_notch_preserves_source_identity_and_updates_stream_identity` | `FA-NOTCH-SPEC-006` | `source_id` を保持し、`stream_id` を `output_topic` にする |
| `test_notch_uses_second_order_biquad_per_channel_state` | `FA-NOTCH-SPEC-005` | channel ごとの二次 biquad state と notch 式を使う |
| `test_filter_parameters_are_required_and_range_checked` | `FA-NOTCH-SPEC-003`, `004` | center と Q が有限、正、Nyquist 条件で検証される |
| `test_package_layout_matches_standard_processing_layout` | layout | 標準 processing package layout を満たす |
| `test_colcon_runs_pytest_contracts` | test runner | CMake が pytest を登録し plugin autoload を無効化する |

## Integration / Launch

`test/integration` と `test/launch` は directory contract を先に固定する。ROS graph を使った publish/subscribe smoke test は、親側の system integration で package wiring が追加された後に実装する。
