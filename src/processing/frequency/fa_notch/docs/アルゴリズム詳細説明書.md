# fa_notch アルゴリズム詳細説明書

## 入力検証

1. subscription callback 入口で null `AudioFrame` pointer でないことを確認する。
2. `source_id` と `stream_id` が空でないことを確認する。
3. 2 frame 目以降は、最初に受理した `source_id` と一致することを確認する。
4. `epoch` が最後に受理した frame より大きいことを確認する。
5. `stream_id == input_stream_id` を確認する。
6. `layout == interleaved` を確認する。
7. `encoding == FLOAT32LE`、`bit_depth == 32` を確認する。
8. `sample_rate` と `channels` が config の `expected.*` と一致することを確認する。
9. `data.size()` が `channels * sizeof(float)` の倍数であることを確認する。

不一致は frame skip として扱う。別 format を暗黙変換すると format node の責務を侵食し、別 stream の state を再利用すると filter 出力の意味が壊れるためである。

## Filter Parameter Validation

`filter.center_hz` は起動時に次の条件を満たす必要がある。

- finite
- `> 0.0`
- `< expected.sample_rate / 2.0`

`filter.q` は起動時に次の条件を満たす必要がある。

- finite
- `> 0.0`

不正な parameter は起動失敗とする。Nyquist 以上の center や非正 Q を補正して継続する処理は行わない。

## Second-Order Notch Biquad

起動時に標準 biquad notch 係数を計算する。

```text
w0 = 2 * pi * center_hz / sample_rate
alpha = sin(w0) / (2 * q)

b0 = 1
b1 = -2 * cos(w0)
b2 = 1
a0 = 1 + alpha
a1 = -2 * cos(w0)
a2 = 1 - alpha
```

処理前にすべての係数を `a0` で正規化する。

```text
b0 = b0 / a0
b1 = b1 / a0
b2 = b2 / a0
a1 = a1 / a0
a2 = a2 / a0
```

各 channel は独立した `previous_input_1`、`previous_input_2`、`previous_output_1`、`previous_output_2` を持つ。
state は node 起動時に zero initial condition として初期化する。forward `epoch` gap を検出した場合も、古い IIR state を使わず zero initial condition から処理する。各 sample には次式を適用する。

```text
y[n] = b0*x[n] + b1*x[n-1] + b2*x[n-2] - a1*y[n-1] - a2*y[n-2]
```

`AudioFrame.data` は interleaved FLOAT32LE として 4 byte ごとに読み書きする。出力値は notch filter の結果そのものであり、normalize、gain、limit、clamp は行わない。

backend は入力 frame 全体を一時 state に対して処理し、全 sample が成功した場合だけ channel state を更新する。`reset_state=true` の場合も、fresh state を一時 state として使い、成功した場合だけ fresh state 由来の結果を commit する。frame 内に非 finite sample、非 finite 出力、または FLOAT32LE として表現できない出力があった場合、その frame は publish せず、filter state も更新しない。

notch は IIR filter であり、過去入力と過去出力が狭帯域の除去特性に直接影響する。別 source の混入、duplicate / regression epoch、欠落 frame 後の古い state 再利用は ringing や履歴汚染として後段に伝わるため、source mismatch は drop、non-monotonic epoch は drop、forward gap は reset として扱う。

## 出力

- `header`、`source_id`、sample format metadata は入力から保持する。
- `stream_id` は `output.stream_id` に更新する。
- diagnostics には `input_topic`、`output_topic`、`input_stream_id`、`output_stream_id`、`frames_in`、`frames_out`、`frames_dropped`、`filter_center_hz`、`filter_q`、`state_source_id`、`state_resets`、正規化済み biquad 係数を出す。
