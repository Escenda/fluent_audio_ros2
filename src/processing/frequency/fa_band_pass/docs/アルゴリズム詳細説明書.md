# fa_band_pass アルゴリズム詳細説明書

## 概要

`fa_band_pass` は、各 channel 独立に一次 high-pass filter を適用し、その出力を一次 low-pass filter に入力する。入力 layout は FLOAT32LE interleaved のみを受け付ける。

## 係数

`dt = 1 / sample_rate`

high-pass:

```text
rc_hp = 1 / (2 * pi * low_cut_hz)
hp_alpha = rc_hp / (rc_hp + dt)
```

low-pass:

```text
rc_lp = 1 / (2 * pi * high_cut_hz)
lp_alpha = dt / (rc_lp + dt)
```

`low_cut_hz` は finite かつ `> 0`、`high_cut_hz` は finite かつ `> low_cut_hz` かつ Nyquist 未満でなければならない。

## Recurrence

sample `x[n]` に対して、channel state は `prev_x_hp`、`prev_y_hp`、`prev_y_lp` を持つ。

```text
y_hp[n] = hp_alpha * (prev_y_hp + x[n] - prev_x_hp)
y_lp[n] = prev_y_lp + lp_alpha * (y_hp[n] - prev_y_lp)
```

source ごとの最初の accepted sample は state 初期化に使い、出力は `0.0` とする。以降の sample から上記 recurrence を適用する。

## State Boundary

node は subscription callback 入口で null `AudioFrame` pointer を drop する。次に `source_id`、`stream_id == input_stream_id`、`epoch`、format、layout、byte length を検証する。

最初に backend が受理した frame の `source_id` を state source として bind する。以降の frame で `source_id` が異なる場合は、別 stream の履歴を cascade state に混ぜないため drop する。`epoch` が最後に受理した frame 以下の場合は duplicate / regression として drop する。

`epoch` が最後に受理した値 + 1 ではない forward gap の場合、node は backend に `reset_state=true` を渡す。backend は一時 state を未初期化状態へ戻してから処理し、frame 全体が成功した場合だけ reset 後 state を commit する。reset 後は現在 frame の最初の sample から state を初期化し、diagnostics の `state_resets` を increment する。backend が拒否した場合は、`source_id` bind、`epoch` 更新、reset counter 更新は行わない。

band-pass は高域遮断と低域遮断の IIR state を連結しているため、古い source や欠落前の state をそのまま使うと過去入力と過去出力の履歴が後段へ伝わる。source mismatch は drop、forward gap は fresh state として扱う。

## Invalid Frame Handling

以下は frame 全体を drop する。

- contract 不一致
- 空 data
- frame byte size が `channels * sizeof(float)` で割り切れない
- 入力 sample が finite normalized `[-1, 1]` ではない
- filter 計算結果が non-finite または FLOAT32LE として表現できない
- final output sample が finite normalized `[-1, 1]` ではない

drop 時に clamp、normalize、resampling、zero 補完は行わない。

## Output Identity

ROS topic は `input_topic` / `output_topic` で指定する搬送路であり、`AudioFrame.stream_id` ではない。出力 frame は入力の `header`、`source_id`、format metadata、`epoch` を保持し、`stream_id` だけを `output.stream_id` に更新する。
