# fa_band_pass テスト設計

## Scope

unit contract tests は package 内の config、C++ source、header、CMake/package metadata を検査する。ROS graph は C++ gtest、launch fail-closed は `test/launch` の pytest で検証する。

## Unit Tests

- default config が FLOAT32LE interleaved band-pass contract を表すこと
- device I/O、resampling、format conversion、gain、normalize、limiter、denoise を持たないこと
- startup config validation が fail closed であること
- runtime frame validation が source / epoch / format 処理前に実行されること
- null frame pointer を dereference 前に drop すること
- `stream_id` が input topic から output topic に更新され、他の identity が保存されること
- backend accept 後だけ `source_id` を bind すること
- forward epoch gap だけ fresh state を要求し、backend accept 後だけ reset counter / epoch を更新すること
- high-pass と low-pass の係数式と recurrence が source に含まれること
- C++ gtest が backend の数値出力、拒否 status、reset state commit 条件を検査すること
- channel ごとの state を持つこと
- non-finite または範囲外 sample を clamp せず drop すること
- diagnostics が filter、state、counter を含むこと
- required package layout が揃っていること
- colcon backend gtest / graph gtest / pytest / lint auto registration があること

## C++ Graph Tests

- `PublishesBandPassFilteredFloat32Frame`: NodeOptions で node を起動し、publish/subscribe 経路で cascade 出力、metadata 保持、duplicate epoch drop を検証する。
- `ResetsFilterStateOnForwardEpochGap`: forward epoch gap 後の出力が fresh cascade state 由来になることを検証する。

## Launch Tests

- launch argument が `node_name` / `config_file` だけであること
- default config が frequency node の責務だけを持つこと
- invalid `filter.low_cut_hz` で起動失敗すること
- invalid `filter.high_cut_hz` で起動失敗すること
- wrong encoding で起動失敗すること

## Command

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/frequency/fa_band_pass/test/unit src/processing/frequency/fa_band_pass/test/launch -q
```
