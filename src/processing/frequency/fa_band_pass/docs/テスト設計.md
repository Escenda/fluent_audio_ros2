# fa_band_pass テスト設計

## Scope

unit contract tests は package 内の config、C++ source、header、CMake/package metadata を検査する。runtime ROS graph の結合試験は `test/integration` と `test/launch` に分離する。

## Unit Tests

- default config が FLOAT32LE interleaved band-pass contract を表すこと
- device I/O、resampling、format conversion、gain、normalize、limiter、denoise を持たないこと
- startup config validation が fail closed であること
- runtime frame validation が処理前に実行されること
- `stream_id` が input topic から output topic に更新され、他の identity が保存されること
- high-pass と low-pass の係数式と recurrence が source に含まれること
- channel ごとの state を持つこと
- non-finite または範囲外 sample を clamp せず drop すること
- source change で state reset と counter update を行うこと
- diagnostics が filter、state、counter を含むこと
- required package layout が揃っていること
- colcon pytest registration があること

## Command

```bash
env PYTHONDONTWRITEBYTECODE=1 PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python3 -m pytest -p no:cacheprovider src/processing/frequency/fa_band_pass/test/unit -q
```
