# fa_band_pass 仕様書

## 目的

`fa_band_pass` は、FluentAudio の FLOAT32LE interleaved 音声 stream に対して、一次 high-pass filter と一次 low-pass filter をこの順序で適用する周波数処理 node である。

## 対象外

- device I/O
- resampling
- sample format conversion
- gain、normalize、limiter、denoise
- legacy alias、互換 path
- 欠損値や不正値を補う fallback

## ROS2 Interface

- package: `fa_band_pass`
- executable: `fa_band_pass_node`
- node name default: `fa_band_pass`
- subscribe: `input_topic`
- publish: `output_topic`
- message type: `fa_interfaces/msg/AudioFrame`

## Parameters

| name | default | constraint |
| --- | --- | --- |
| `input_topic` | `audio/sample_format/mic` | non-empty |
| `output_topic` | `audio/band_pass/mic` | non-empty |
| `filter.low_cut_hz` | `80.0` | finite, `> 0` |
| `filter.high_cut_hz` | `3400.0` | finite, `> low_cut_hz`, `< sample_rate / 2` |
| `expected.sample_rate` | `16000` | `> 0` |
| `expected.channels` | `1` | `> 0` |
| `expected.encoding` | `FLOAT32LE` | exactly `FLOAT32LE` |
| `expected.bit_depth` | `32` | exactly `32` |
| `expected.layout` | `interleaved` | exactly `interleaved` |
| `qos.depth` | `10` | `> 0` |
| `qos.reliable` | `false` | bool |
| `diagnostics.publish_period_ms` | `1000` | `> 0` |

不正な parameter では node 起動を継続しない。

## Runtime Frame Contract

入力 frame は以下を満たす必要がある。

- `source_id` is non-empty
- `stream_id` is non-empty and equals `input_topic`
- `sample_rate == expected.sample_rate`
- `channels == expected.channels`
- `encoding == expected.encoding`
- `bit_depth == expected.bit_depth`
- `layout == expected.layout`
- `data` is non-empty
- `data.size()` is divisible by `channels * sizeof(float)`
- every input sample is finite and normalized in `[-1, 1]`
- after the first accepted frame, `source_id` equals the bound state source
- after the first accepted frame, `epoch` is strictly greater than the previous accepted epoch

不正な runtime frame は warning を出して drop する。値の推測、clamp、normalize、zero 埋めは行わない。

## Output Contract

出力 frame は入力 frame から以下を保存する。

- `header`
- `source_id`
- `sample_rate`
- `channels`
- `bit_depth`
- `encoding`
- `layout`
- `epoch`

`stream_id` は `output_topic` に更新し、`data` は band-pass 後の FLOAT32LE interleaved samples に置き換える。出力 sample が finite normalized `[-1, 1]` を満たさない場合は frame を drop する。

## State

filter state は channel ごとに保持する。最初に backend が受理した frame の `source_id` を state source として bind し、それ以降に異なる `source_id` の frame が来た場合は drop する。`epoch` が前回以下の frame は duplicate / regression として drop する。`epoch` が前回 + 1 ではない forward gap の場合は、古い HP/LP state を使わず fresh state で処理する。

backend が frame を拒否した場合、`source_id` bind、`epoch` 更新、state reset counter 更新は行わない。

## Diagnostics

diagnostics には `low_cut`、`high_cut`、`hp_alpha`、`lp_alpha`、state source、state reset count、frame counters を含める。

## Launch Contract

`fa_band_pass.launch.py` が公開する launch argument は `node_name` と `config_file` のみである。filter parameter、expected format、QoS、diagnostics interval は config file にだけ置く。invalid config は node 起動失敗として扱う。

## Traceability

- `FA-BAND-PASS-SPEC-001`: `fa_band_pass` は device I/O、resample、format conversion、gain、normalize、limiter、denoise を行わない。
- `FA-BAND-PASS-SPEC-002`: `fa_band_pass` は `FLOAT32LE` / 32bit / interleaved のみ扱う。
- `FA-BAND-PASS-SPEC-003`: `filter.low_cut_hz` は finite かつ `> 0.0` である。
- `FA-BAND-PASS-SPEC-004`: `filter.high_cut_hz` は finite、`> filter.low_cut_hz`、Nyquist 未満である。
- `FA-BAND-PASS-SPEC-005`: channel ごとに一次 HP + 一次 LP cascade state を保持する。
- `FA-BAND-PASS-SPEC-006`: 出力 `source_id` は保持し、`stream_id` は `output_topic` にする。
- `FA-BAND-PASS-SPEC-007`: null frame pointer は dereference せず drop する。
- `FA-BAND-PASS-SPEC-008`: `source_id` と monotonic `epoch` により state の適用範囲を固定する。
- `FA-BAND-PASS-SPEC-009`: forward `epoch` gap は fresh cascade state で処理し、reset 数を diagnostics に出す。
- `FA-BAND-PASS-SPEC-010`: launch は `node_name` と `config_file` だけを公開し、config 不正時は fail closed する。
