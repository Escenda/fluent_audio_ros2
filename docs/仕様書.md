# FluentAudio 仕様書

## 1. 位置づけ

FluentAudio は ROS2 上の音声入出力、音声 processing、音声 AI node、音声 app node を構成するための package 集合です。

本 repository root の仕様書は package 横断の責務境界を定義します。各 node の topic、parameter、failure behavior は各 package 配下の `docs/仕様書.md` を正とします。
README-only の roadmap placeholder は `docs/roadmap_placeholders.md` で分類します。`package.xml` を持たない directory は機能実装済み package として数えず、system config から参照しません。

## 2. レイヤー責務

| layer | 責務 | 禁止事項 |
| --- | --- | --- |
| `src/interfaces` | message / service contract | runtime backend 実装 |
| `src/io` | source / sink adapter | resample、gain、filter、VAD、ASR、mixing |
| `src/processing` | format、dynamics、frequency、temporal、correction、spatial、analysis、generation、routing | device I/O、VLAbor profile 解釈、認識 / 分類 / 対話方針 AI runtime |
| `src/ai` | VAD、KWS、ASR、Turn Detector、speaker / audio embedding、SED などの AI model node | device I/O、DSP 変換、transport buffering、application policy |
| `src/streaming` | buffering、jitter buffer、clock drift correction、PLC、latency compensation、time alignment、overlap-add | device I/O、AI model runtime、application policy |
| `src/apps` | voice command、dialogue、safety などの application orchestration | model runtime の直接実装 |
| `src/system` | system config validation、node 展開、topic wiring | device 推測、model fallback、temporary YAML 書き換え |

## 3. I/O 契約

`fa_in` は入力 source adapter です。microphone、system audio、application audio、loopback、file、network などの source 境界を backend として扱いますが、音声内容は編集しません。

`fa_out` は出力 sink adapter です。speaker、loopback、file、network などの sink 境界を backend として扱いますが、音声内容は編集しません。

format convert、resample、gain、AGC、filter、noise reduction、AEC、mixing は `src/processing` の専用 node を pipeline に明示的に置きます。buffering、jitter buffer、clock drift correction などのリアルタイム伝送安定化は `src/streaming` の専用 node に置きます。

## 4. Backend 契約

Backend は ROS2 topic / message を知りません。ROS2 node は parameter validation、message conversion、topic publication/subscription、lifecycle を担当し、backend は device access、DSP library、model runtime、external worker、cloud API との境界だけを担当します。

VAD / KWS / ASR / Turn Detector は `src/ai` に置き、engine は package 内の `backends/` に分離します。`src/processing/analysis` は STFT、Mel spectrogram、MFCC、loudness などの非 AI feature extraction だけを扱い、AI model node の置き場にはしません。streaming node も `src/processing` ではなく `src/streaming` に置きます。

`src/processing/generation` は音声 data-plane の生成 / 変換 node の置き場です。TTS、voice conversion、speech enhancement、neural codec、vocoder のように「入力または request から別の audio representation を作る」処理を扱います。認識、分類、会話制御、LLM / MCP / VLM orchestration、dialogue policy は `src/processing/generation` に置かず、AI model node は `src/ai`、application orchestration は `src/apps` に置きます。generation backend が neural model、cloud API、external worker を使う場合も、backend は明示選択で、ROS-free な `backends/` に分離します。

`backend.name` は必須です。unknown backend、missing model、missing executable、invalid config は起動失敗にします。OpenAI backend は明示的に選ばれた backend であり、local backend の fallback ではありません。

各 backend は supported capability を明示します。最低限、`AudioFrame.encoding`、bit depth、sample rate、channel count、layout、payload byte alignment、値域、model/provider、外部 worker protocol、output schema を、扱う backend ごとに仕様化します。未対応 capability は変換で吸収しません。

起動時に判定できる非対応 backend / model / provider / payload encoding / sample rate / executable / credential は startup failure とします。入力 frame ごとにしか判定できない非対応 format、stream identity 不一致、malformed payload、non-finite sample、normalized range 外 sample は backend state を触る前に frame rejection とします。必要 backend の runtime failure は、message contract が explicit error result を定める場合を除いて runtime fatal shutdown とします。

## 5. System Config 契約

`fluent_audio_system` は system YAML を読み、enabled group / node を ROS2 launch action に展開します。

必須条件:

- `config` launch argument は必須
- `system.default_start_delay` と `system.inter_group_delay` は必須
- group / node の `enable` は必須
- enabled node の `params_file` は必須
- `params_file` は実在必須

禁止事項:

- missing `params_file` を inline parameter だけで補うこと
- device や model path を推測すること
- temporary YAML を書き換えて起動すること
- missing backend を別 backend へ置き換えること

## 6. Failure Policy

必要条件が満たせない場合は fail closed します。

許可される挙動:

- 起動時 validation error
- runtime fatal error
- 明示条件に基づく frame drop
- 明示 degraded backend として仕様化された挙動

禁止される挙動:

- 別 device / model / backend への暗黙 fallback
- missing data を zero / empty / default で補うこと
- format mismatch を node 内で隠れて変換すること
- runtime import error を機能分岐として扱うこと
- 未対応 capability を、source text 検査や命名規則だけで禁止扱いにすること。非対応は config validation、frame validation、backend API、launch、ROS graph、external worker protocol のいずれかで実行時 contract として観測可能にする。
