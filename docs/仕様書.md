# FluentAudio 仕様書

## 1. 位置づけ

FluentAudio は ROS2 上の音声入出力、音声 processing、音声 AI node、音声 app node を構成するための package 集合です。

本 repository root の仕様書は package 横断の責務境界を定義します。各 node の topic、parameter、failure behavior は各 package 配下の `docs/仕様書.md` を正とします。

## 2. レイヤー責務

| layer | 責務 | 禁止事項 |
| --- | --- | --- |
| `src/interfaces` | message / service contract | runtime backend 実装 |
| `src/io` | source / sink adapter | resample、gain、filter、VAD、ASR、mixing |
| `src/processing` | format、dynamics、frequency、temporal、correction、spatial、analysis、generation、routing、streaming | device I/O、VLAbor profile 解釈 |
| `src/apps` | voice command、dialogue、safety などの application orchestration | model runtime の直接実装 |
| `src/system` | system config validation、node 展開、topic wiring | device 推測、model fallback、temporary YAML 書き換え |

## 3. I/O 契約

`fa_in` は入力 source adapter です。microphone、system audio、application audio、loopback、file、network などの source 境界を backend として扱いますが、音声内容は編集しません。

`fa_out` は出力 sink adapter です。speaker、loopback、file、network などの sink 境界を backend として扱いますが、音声内容は編集しません。

format convert、resample、gain、AGC、filter、noise reduction、AEC、mixing、buffering は `src/processing` の専用 node を pipeline に明示的に置きます。

## 4. Backend 契約

Backend は ROS2 topic / message を知りません。ROS2 node は parameter validation、message conversion、topic publication/subscription、lifecycle を担当し、backend は device access、DSP library、model runtime、external worker、cloud API との境界だけを担当します。

VAD / KWS / ASR / Turn Detector は `src/processing/analysis` に置き、engine は package 内の `backends/` に分離します。

`backend.name` は必須です。unknown backend、missing model、missing executable、invalid config は起動失敗にします。OpenAI backend は明示的に選ばれた backend であり、local backend の fallback ではありません。

## 5. System Config 契約

`fluent_audio_system` は system YAML を読み、enabled group / node を ROS2 launch action に展開します。

必須条件:

- `config` launch argument は必須
- `system.default_start_delay` と `system.inter_group_delay` は必須
- group / node の `enable` は必須
- enabled node の `params_file` は必須
- `params_file` は実在必須

禁止事項:

- missing `params_file` を inline parameter だけで補うこと
- device や model path を推測すること
- temporary YAML を書き換えて起動すること
- missing backend を別 backend へ置き換えること

## 6. Failure Policy

必要条件が満たせない場合は fail closed します。

許可される挙動:

- 起動時 validation error
- runtime fatal error
- 明示条件に基づく frame drop
- 明示 degraded backend として仕様化された挙動

禁止される挙動:

- 別 device / model / backend への暗黙 fallback
- missing data を zero / empty / default で補うこと
- format mismatch を node 内で隠れて変換すること
- runtime import error を機能分岐として扱うこと
