# FluentAudio アルゴリズム詳細説明書

## 1. 全体 pipeline

FluentAudio の標準 pipeline は次の層で構成します。

```text
source backend
  -> fa_in
  -> format / correction / dynamics / frequency / temporal / spatial nodes
  -> streaming nodes
  -> non-AI analysis / ai / generation / routing nodes
  -> fa_out
  -> sink backend
```

`fa_in` と `fa_out` は source / sink adapter であり、音声編集を実行しません。音声変換や解析は必ず dedicated processing node に分離します。

## 2. 処理分類

`src/processing` の直下は次の 9 分類で固定します。AI model node とリアルタイム伝送 node は責務が大きく異なるため、top-level の `src/ai` と `src/streaming` に分離します。

| category | 内容 |
| --- | --- |
| `format` | resample、bit depth、channel、interleave、codec、value range |
| `dynamics` | gain、normalize、compressor、limiter、expander、gate、AGC |
| `frequency` | EQ、low-pass、high-pass、band-pass、notch、de-esser、spectral subtraction、Wiener |
| `temporal` | trim、silence removal、time stretch、pitch shift、delay、reverb、fade、windowing |
| `correction` | denoise、AEC、dereverb、declip、declick、hum removal、DC offset |
| `spatial` | pan、downmix、upmix、beamforming、source separation |
| `analysis` | STFT、mel、MFCC、loudness measurement などの非 AI feature extraction |
| `generation` | TTS、voice conversion、speech enhancement、neural codec、vocoder |
| `routing` | mixer、bus routing、sidechain、ducking、monitor、loopback、patchbay |

Top-level の追加分類:

| directory | 内容 |
| --- | --- |
| `src/ai` | VAD、KWS、ASR、Turn Detector、speaker embedding、audio embedding、SED などの AI model node |
| `src/streaming` | buffering、jitter buffer、clock drift、packet loss concealment、latency compensation、time alignment、chunk overlap、overlap-add |

`src/processing/analysis` は AI package の置き場ではありません。VAD、KWS、ASR、Turn Detector、SED、speaker / audio embedding のように model inference または認識結果生成が主責務の node は `src/ai` に置きます。streaming node も processing の下には置かず、top-level の `src/streaming` に置きます。

## 3. Node / Backend 分離

ROS2 node の処理手順:

1. parameter を読み、必須値を検証する。
2. backend config を明示型へ変換する。
3. backend を `backend.name` で構築する。
4. input frame の sample rate、channel、encoding、bit depth、value range を検証する。
5. backend に ROS2 非依存の byte / float buffer と metadata を渡す。
6. backend result を ROS2 message に変換して publish する。

Backend は ROS2 topic、namespace、launch、parameter server を参照しません。

## 4. External Engine

Python version、venv、GPU runtime、native library が ROS2 node と一致しない engine は external worker / process / container 境界に置きます。

ASR backend の例:

- `whisper_cpp`: local executable process
- `parakeet_worker`: external worker process
- `local_command`: configured local command
- `openai_realtime`: explicit cloud API backend

External backend は command、model path、endpoint、credential contract を起動時に検証します。条件を満たせない場合は起動失敗にします。

## 5. Format Handling

Hidden format negotiation は禁止します。analysis backend が必要とする canonical format は node config に明示し、入力 frame が一致しない場合は reject / fail closed します。

Resample、downmix、bit depth conversion、normalized range conversion が必要な場合は `src/processing/format` の node を pipeline に挿入します。
